# 基础卡片插件目录 AGENTS

## 角色

- 本目录按子目录管理多个基础卡片插件工程；每个子目录都应被视为独立插件项目。

## 通用规则

- 每个基础卡片插件都必须维护完整的 `manifest`、查看态渲染、编辑态面板、配置 schema、测试、i18n 和必要资源文件。
- 系统能力访问只能通过 Host / Bridge / `chips-sdk` 正式链路，不得直连 Host 内部实现。
- 基础卡片配置、卡片文件格式、资源引用和公共插件契约属于生态公共信息，需优先对齐 `生态共用技术文档/`。
- 如果某个插件需要新增生态级字段、资源规范或公共编辑协议，先更新共用文档或登记工单，不要只在单插件里私自定规则。
- 需要移除的历史内容移动到本目录或子项目自己的 `归档/`，不要直接删除。

## 验证

- 优先使用各插件自己的 `chipsdev` 正式脚本与单元/集成测试做验证。
