# 图片基础卡片插件 - 需求规格说明书（vNext）

版本：0.1.0  
状态：正式版  

---

## 1. 产品概述

- 插件类型：基础卡片插件（`type: card`）
- 插件标识：`chips.basecard.image`
- 支持卡片类型：`base.image`
- 运行环境：
  - Host 通用查看链路
  - Host 托管编辑器链路
  - 官方 `Chips-EditingEngine` 本地基础卡片运行时
- 正式入口：
  - `renderBasecardView`
  - `renderBasecardEditor`
  - `basecardDefinition`

本插件负责在新架构下完整迁移旧版图片基础卡片能力，正式支持图片资源在卡片根目录中的存储、渲染、编辑、排序和布局配置，并复刻旧版项目的核心交互能力。

## 2. 功能范围

### 2.1 渲染组件（查看模式）

插件在查看态根据配置渲染图片内容，支持以下正式布局：

- 单张图片（`single`）
  - 当图片数量小于等于 1 时自动生效
  - 支持宽度百分比调节
  - 支持左 / 中 / 右对齐
- 网格布局（`grid`）
  - `2x2`
  - `3x3`
  - `3-column-infinite`
  - `2x2` / `3x3` 超出显示上限时，仅最后一个可见图片格显示 `+N` 叠层，不得覆盖整个网格容器
- 长图拼接（`long-scroll`）
  - 正式行为为纵向完整展开，不提供嵌套固定窗口裁切
- 横向滑动（`horizontal-scroll`）
  - 支持鼠标滚轮映射为横向滚动

查看态图片来源：

- 卡片内部资源：`source = file`，正式记录相对于卡片根目录的 `file_path`
- 外部 URL：`source = url`，正式记录 `url`

### 2.2 编辑组件（编辑模式）

插件在编辑态提供以下正式能力：

- 图片导入
  - 点击或拖拽上传一个或多个本地图片文件
  - 通过 URL 添加外部图片
- 图片列表管理
  - 图片列表正式采用 3 列无限缩略图插槽编排，不能依赖生硬的固定 CSS Grid 排位
  - 列表项只显示缩略图本体，不在图片上方叠加名称或操作按钮
  - 列表末尾必须提供灰色加号图片卡片，点击后在列表下方展开图片添加区域
  - 拖拽排序采用占位插入式预览：开始拖动时原位置保留空位，移动到目标位置时列表自动让位并插入
  - 拖动过程中鼠标旁必须显示当前图片的浮动缩略图预览
  - 拖拽过程中其他图片必须以平滑动画完成让位与吸附，交互效果应接近手机桌面图标整理
  - 拖拽到底部固定删除投放区可删除单张图片
  - 清空全部图片
  - 编辑器采用分区式面板，不设置顶部固定工具栏
  - 当前编辑器正式不提供每张图片的标题 / 替代文本表单编辑入口
- 布局配置
  - 切换布局类型
  - 当卡片从单图扩展为多图时，默认自动切换到网格布局
  - 多图状态不提供单图布局选项；如输入非法多图 `single` 配置，归一化后强制改写为 `grid`
  - 网格模式切换
  - 长图模式切换
  - 单图宽度百分比与对齐方式调整
  - 图片间距模式切换，仅提供 `none` / `comfortable` 两个正式选项
  - `none` 时图片零间距且无圆角，`comfortable` 时图片保留适当间距并使用小圆角
- 历史操作
  - 撤销
  - 重做

### 2.3 资源链路约束

- 卡片内部资源路径统一从卡片文件根目录开始记录，不相对于 `content/`
- 本地导入图片必须通过宿主 `importResource(...)` 写入卡片根目录
- 删除卡片内部图片必须通过宿主 `deleteResource(...)` 执行
- 编辑态缩略图和预览图对 `source = file` 的资源访问必须通过宿主 `resolveResourceUrl(...)`
- 宿主返回的临时资源 URL 必须通过 `releaseResourceUrl(...)` 释放
- 当资源解析因切卡、临时 URL 失效等原因短暂失败时，插件必须自动重试一次，避免把瞬时故障直接暴露给用户
- 运行时 `blob:` / `data:` / 系统绝对路径不得写回正式 YAML
- 插件正式不设置图片数量上限和图片大小上限；若宿主存在环境级限制，应由宿主单独拦截并提示

## 3. 数据结构要求

正式配置字段：

- `card_type: "ImageCard"`
- `theme?: string`
- `images: ImageItem[]`
- `layout_type: "single" | "grid" | "long-scroll" | "horizontal-scroll"`
- `layout_options?:`
  - `grid_mode?: "2x2" | "3x3" | "3-column-infinite"`
  - `single_width_percent?: number`
  - `single_alignment?: "left" | "center" | "right"`
  - `spacing_mode?: "none" | "comfortable"`

图片项 `ImageItem`：

- `id: string`
- `source: "file" | "url"`
- `file_path?: string`
- `url?: string`
- `alt?: string`
- `title?: string`

补充约束：

- `images` 允许为空，用于创建空白卡片和执行“清空全部”后的正式保存
- `source = "file"` 时，`file_path` 必须始终指向卡片根目录下的资源文件
- 当 `images.length > 1` 时，正式多图布局不允许使用 `single`

## 4. 非功能需求

- 技术栈：React
- 样式要求：统一使用 `--chips-*` token 和主题变量，不硬编码整套视觉风格
- 国际化：至少提供 `zh-CN` / `en-US`
- 稳定性：编辑器对无效输入、图片加载失败、资源导入失败要有明确降级提示
- 交互一致性：拖拽删除区必须固定在编辑器底部；滚动到底部时页面仍需保留足够留白，避免删除区遮挡最后一行图片
- 可测试性：
  - 单元测试覆盖配置归一、校验、渲染布局、编辑器主要交互
  - 集成测试覆盖“编辑修改 -> 视图更新”的正式主链路

## 5. 明确不允许的实现

- 不允许把旧版 Vue、旧版 Core、旧版路由协议直接搬入新项目
- 不允许保留任何兼容旧架构的分支代码
- 不允许在插件内直接访问文件系统
- 不允许使用临时占位编辑器、简化版图片列表或临时 Data URL 持久化方案
