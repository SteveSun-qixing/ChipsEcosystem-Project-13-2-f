# 图片基础卡片插件 - 数据模型设计

版本：0.1.0  
状态：正式版  

---

## 1. 正式配置模型

```ts
type ImageSourceType = "file" | "url";

interface ImageItem {
  id: string;
  source: ImageSourceType;
  file_path?: string;
  url?: string;
  alt?: string;
  title?: string;
}

type LayoutType = "single" | "grid" | "long-scroll" | "horizontal-scroll";
type GridMode = "2x2" | "3x3" | "3-column-infinite";
type SingleAlignment = "left" | "center" | "right";
type SpacingMode = "none" | "comfortable";

interface LayoutOptions {
  grid_mode?: GridMode;
  single_width_percent?: number;
  single_alignment?: SingleAlignment;
  spacing_mode?: SpacingMode;
}

interface ImageCardConfig {
  card_type: "ImageCard";
  theme?: string;
  images: ImageItem[];
  layout_type: LayoutType;
  layout_options?: LayoutOptions;
}
```

## 2. 默认值

- `card_type = "ImageCard"`
- `theme = ""`
- `images = []`
- `layout_type = "single"`
- `layout_options.grid_mode = "2x2"`
- `layout_options.single_width_percent = 100`
- `layout_options.single_alignment = "center"`
- `layout_options.spacing_mode = "comfortable"`

## 3. 约束规则

### 3.1 图片项

- `id` 必须为非空字符串
- `source = "file"` 时必须提供 `file_path`
- `source = "url"` 时必须提供 `url`
- `file_path` 记录相对于卡片根目录的路径
- `images` 允许为空，用于空白卡片和“清空全部”后的正式保存
- `title` / `alt` 仍属于正式可选字段，但当前编辑器图片网格不提供逐张表单编辑入口

### 3.2 布局

- `layout_type` 只能是 `single | grid | long-scroll | horizontal-scroll`
- 当 `images.length > 1` 且输入 `layout_type = "single"` 时，归一化结果强制改为 `grid`
- `grid_mode` 只能是 `2x2 | 3x3 | 3-column-infinite`
- `single_width_percent` 范围为 `10-100`
- `spacing_mode` 只能是 `none | comfortable`

## 4. 渲染派生模型

### 4.1 有效布局类型

```ts
effectiveLayoutType = images.length <= 1
  ? "single"
  : config.layout_type === "single"
    ? "grid"
    : config.layout_type;
```

### 4.2 网格显示数量

- `2x2` 最多展示 4 张
- `3x3` 最多展示 9 张
- `3-column-infinite` 展示全部

### 4.3 图片显示源

- `source = "file"`：
  - 查看态：直接使用 `file_path`
  - 编辑态：通过 `resolveResourceUrl(file_path)` 解析预览 URL
  - 资源解析失败时，查看态和编辑态都会自动进行一次重试恢复
  - 长图布局不再创建固定高度子滚动容器，图片按顺序完整展开
- `source = "url"`：
  - 查看态 / 编辑态都直接使用 `url`

### 4.4 编辑态列表派生

- 编辑器图片列表固定为 `3` 列无限缩略图插槽编排
- 列表插槽由编辑器根据容器宽度计算绝对定位坐标，并以统一动画曲线驱动图片让位与吸附
- 网格末尾始终附加一个“添加图片”灰色加号卡片，用于展开列表底部添加模块
- 删除动作不再通过逐项按钮完成，而是通过底部固定删除投放区处理
- 插件本身不设置图片数量上限和文件大小上限

### 4.5 间距视觉派生

- `spacing_mode = "none"`：
  - `gap = 0`
  - `radius = 0`
- `spacing_mode = "comfortable"`：
  - `gap = 10`
  - `radius = 12`

## 5. 历史记录模型

```ts
type HistoryActionType =
  | "add"
  | "remove"
  | "move"
  | "update"
  | "layout"
  | "batch"
  | "clear";

interface HistoryEntry {
  type: HistoryActionType;
  before: ImageCardConfig;
  after: ImageCardConfig;
  timestamp: number;
}
```

历史栈只保存结构性编辑后的草稿快照，不保存运行时资源 URL。

## 6. EditingEngine 对接要求

- `normalizeConfig(...)` 输出正式 `ImageCardConfig`
- `validateConfig(...)` 输出编辑态错误映射
- `collectResourcePaths(...)` 只收集 `source = "file"` 的 `file_path`
- `createInitialConfig(...)` 生成空图片列表和正式默认布局
