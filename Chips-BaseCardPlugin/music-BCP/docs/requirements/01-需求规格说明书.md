# 音乐基础卡片插件需求规格说明书

版本：0.1.0
状态：正式版

## 1. 目标

音乐基础卡片用于在复合卡片中保存音乐文件及其相关信息，并通过统一资源打开链路把播放交给音乐播放器应用。

## 2. 功能需求

- 支持上传一个必填音频文件，并将资源保存到卡片根目录。
- 支持编辑歌曲名、专辑封面、歌词文件、制作团队、发行日期、专辑名字、语种、流派。
- 制作团队必须支持多个身份，每个身份支持多个成员。
- 查看态应显示长条状卡片：
  - 左侧为专辑封面；
  - 右侧上方为歌曲名；
  - 右侧下方为较小字体的歌手名。
- 点击查看态卡片时，应向外层容器表达资源打开意图，由宿主统一调用 `resource.open`。
- 用户上传音频文件后，应自动解析音频内嵌元数据：
  - 歌曲名
  - 歌手信息
  - 专辑名
  - 专辑封面
  - 歌词文本
- 若解析到封面或歌词，插件应自动生成正式资源文件并填入配置，其中封面必须以独立图片文件形式保存。
- 对音频内嵌封面，插件必须按真实图片字节头识别实际格式，不能只信任音频标签声明的 MIME。
- 若内嵌封面真实格式为 TIFF，插件必须先提取源封面，再通过正式宿主转换链路把它转换为 PNG，并让配置只引用最终 PNG 资源。
- 编辑页采用与视频基础卡片一致的“分组标题 + 列表行 + 资源预览项”布局，不使用多层卡片嵌套。
- 资源存在时显示资源预览项；资源缺失时切换为上传面。

## 3. 非功能要求

- 所有内部资源路径必须记录为相对于卡片根目录的路径。
- 不得把 `blob:`、`data:`、系统绝对路径写入正式配置。
- 查看态与编辑态都必须使用统一主题 token 与多语言系统。
- 自动补全逻辑必须通过正式宿主资源桥完成资源导入、删除与 TIFF -> PNG 转换。
- 编辑器必须在窄窗口与移动端比例面板中完整显示并可操作。
- 文本字段与资源行在窄窗口下必须自动折叠为单列。
- 音频文件上传前，其他可选字段仍可先编辑为当前草稿配置；正式校验仍以 `audio_file` 为唯一必填字段。
- 变更后需通过插件级 `lint/test/build/validate` 验证。
