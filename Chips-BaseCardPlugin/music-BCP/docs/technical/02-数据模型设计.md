# 音乐基础卡片插件数据模型设计

版本：0.1.0
状态：正式版

## 1. 正式配置模型

```ts
interface ProductionTeamRole {
  id: string;
  role: string;
  people: string[];
}

interface MusicCardConfig {
  card_type: "MusicCard";
  theme?: string;
  audio_file: string;
  music_name: string;
  album_cover: string;
  lyrics_file: string;
  production_team: ProductionTeamRole[];
  release_date: string;
  album_name: string;
  language: string;
  genre: string;
}
```

## 2. 默认值

- `card_type = "MusicCard"`
- `theme = ""`
- `audio_file = ""`
- `music_name = ""`
- `album_cover = ""`
- `lyrics_file = ""`
- `production_team = []`
- `release_date = ""`
- `album_name = ""`
- `language = ""`
- `genre = ""`

## 3. 约束规则

### 3.1 资源字段

- `audio_file` 必填，必须是卡片根目录相对路径。
- `album_cover` 可空；非空时必须是卡片根目录相对路径。
- `lyrics_file` 可空；非空时必须是卡片根目录相对路径。
- 当 `album_cover = ""` 时，运行时使用插件包内置默认封面图占位；该默认图不写入配置，也不计入卡片资源清单。
- 若音频内嵌封面真实格式为 TIFF，正式 `album_cover` 必须记录转换后 PNG 文件的相对路径；源 TIFF 只允许作为中间转换资源存在于导入流程中。

### 3.2 制作团队

- `production_team` 是列表；
- 每个条目必须包含稳定 `id`；
- `role` 为空时仍允许保留为空白草稿；
- `people` 是去重前保持顺序的字符串数组；
- 编辑态允许用户先创建空白身份卡片，再按需补充内容。

### 3.3 其他字段

- `release_date` 推荐使用 `YYYY-MM-DD`；
- `music_name`、`album_name`、`language`、`genre` 都允许为空字符串。

## 4. 派生模型

### 4.1 查看态歌曲名

```ts
displayTitle = music_name || stripExtension(audio_file.fileName) || "未命名歌曲"
```

说明：

- `music_name` 字段本身允许为空；
- 当它为空时，运行时会从 `audio_file` 文件名派生展示标题，但不会把该派生值反写进配置，除非当前音频导入链路显式执行了自动补全。

### 4.2 查看态歌手名

优先从 `production_team` 中角色名包含以下关键字的条目派生：

- `歌手`
- `演唱`
- `主唱`
- `artist`
- `singer`
- `vocal`

若不存在上述角色，则回退到第一个包含成员的身份条目。

### 4.3 资源清单收集

`collectResourcePaths(...)` 只收集：

- `audio_file`
- `album_cover`
- `lyrics_file`

并在宿主持久化阶段用于重建 `structure.yaml manifest.resources`。

## 5. 播放打开上下文

当查看态把音乐卡片交给正式 `resource.open` 时，除了主音频资源，还会同步构造 `MusicCardOpenPayload`：

```ts
interface MusicCardOpenPayload {
  kind: "chips.music-card";
  version: "1.0.0";
  cardType: "base.music";
  config: MusicCardConfig;
  resources: {
    audio: { resourceId: string; relativePath: string; fileName?: string; mimeType?: string };
    cover?: { resourceId: string; relativePath: string; fileName?: string; mimeType?: string };
    lyrics?: { resourceId: string; relativePath: string; fileName?: string; mimeType?: string };
  };
  display: {
    title: string;
    artist: string;
  };
}
```

正式要求：

1. `config` 保留音乐基础卡片正式字段口径，不改写成另一套私有结构；
2. `resources.audio` 必填，且 `resourceId` 必须对应当前查看态已解析好的正式资源；
3. `resources.cover / resources.lyrics` 只有在查看态已拿到稳定可打开资源时才注入；
4. `display.title / display.artist` 用于保证播放器与查看态在展示标题和主歌手上保持一致。
