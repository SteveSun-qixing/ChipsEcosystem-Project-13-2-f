# Phase 1 - 契约与数据模型重构

状态：待开发

## 目标

把富文本基础卡片的正式内容模型从 HTML 切换到 Markdown，并为后续 Milkdown 接入建立统一配置基线。

## 任务列表

1. 重构 schema
   - 把 `body` 模型替换为 `content_format/content_source/content_text/content_file`
   - 更新 `normalizeConfig/validateConfig/createInitialConfig`

2. 清理旧 HTML 假设
   - 移除模板、类型、测试夹具中的 `body: "<p>...</p>"` 口径
   - 清理对 HTML 真源的默认依赖

3. 建立计数与存储策略模块
   - 新增正文可见文本长度计算
   - 新增 inline / file 分流判定逻辑

4. 同步内部文档与公共同步项
   - 确认插件内部文档口径
   - 列出需更新的生态公共文档位置

## 验收标准

- 新版配置类型可独立归一与校验
- 旧版 HTML 正文字段不再出现在主配置模型中
- 200 字分流逻辑具备独立可测试实现
