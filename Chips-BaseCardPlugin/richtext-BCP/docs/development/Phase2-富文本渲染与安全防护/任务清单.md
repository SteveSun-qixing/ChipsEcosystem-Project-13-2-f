# Phase 2 - 富文本渲染与安全防护（已完成）

## 目标

完成富文本基础卡片在查看模式下的渲染实现，并在新架构下补齐必要的 HTML 安全过滤能力。

## 任务列表

1. 渲染 DOM 结构设计
   - 在 `src/render/view.tsx` 中实现：
     - 外层容器：`chips-basecard chips-basecard--richtext`
     - 标题元素：`chips-basecard__title`
     - 内容容器：`chips-basecard__body chips-basecard__body--richtext`
   - 保证标题使用纯文本填充，内容使用 HTML 渲染
   - 状态：已完成

2. 运行时挂载与主题占位
   - 在 `src/render/runtime.ts` 中实现 `mountBasecardView`：
     - 清空容器
     - 可选注入 `themeCssText` 样式文本
     - 挂载渲染根节点并返回卸载函数
   - 保持与 Host 的主题系统解耦
   - 状态：已完成

3. HTML 安全过滤
   - 在 `src/shared/utils.ts` 中实现：
     - `sanitizeRichTextHtml(html: string): string`
     - 白名单标签、属性与样式
     - URL 协议过滤与 `<a>` 的 `noopener` 自动补全
   - 在渲染阶段使用过滤后的 HTML：
     - `createBasecardViewRoot` 中对 `config.body` 进行过滤后再写入 DOM
   - 状态：已完成

4. 默认配置与模板
   - `src/schema/card-config.ts` 定义 `BasecardConfig` 与默认值
   - `templates/default-card-config.yaml` 提供示例富文本内容
   - `templates/parameters.md` 说明各字段含义
   - 状态：已完成

5. 渲染层测试
   - `tests/unit/render-view.test.tsx`：
     - 验证标题与内容渲染
   - `tests/unit/sanitizer.test.ts`：
     - 验证脚本标签与危险链接的过滤行为
   - 状态：已完成

> 阶段结论：富文本查看链路在新架构下已具备完整能力，安全过滤策略可在后续版本按需要拓展。
