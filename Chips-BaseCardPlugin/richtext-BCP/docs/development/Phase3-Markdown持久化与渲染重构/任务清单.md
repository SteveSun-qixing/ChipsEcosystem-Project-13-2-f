# Phase 3 - Markdown 持久化与渲染重构

状态：待开发

## 目标

完成 Markdown inline / file 双存储能力，并将查看态改造成基于 Markdown 正式内容源的静态渲染链路。

## 任务列表

1. 实现正文读取链路
   - inline 模式直接读取 `content_text`
   - file 模式通过正式资源 URL 读取 `.md`

2. 实现正文写入链路
   - `<= 200` 写入 inline 配置
   - `> 200` 通过 `importResource` 写入根目录 Markdown 文件
   - 切回 inline 时登记旧文件删除

3. 重构查看态
   - 以 Markdown 为正式输入
   - 输出静态只读 DOM
   - 保持透明背景与内容驱动高度

4. 建立错误态与清理逻辑
   - 文件读取失败隔离
   - 资源 URL 释放
   - 重复挂载与切换稳定性

## 验收标准

- inline / file 两类配置都能正确打开、编辑和重新保存
- 200 字阈值可稳定驱动模式切换
- 查看态不挂完整编辑器且可正确显示内容
