# Phase 3 - 编辑器功能与校验（已完成）

## 目标

实现富文本编辑器 UI 与核心交互能力，在不引入撤销/重做等额外复杂度的前提下，确保配置数据和校验逻辑完整可靠。

## 任务列表

1. 标题与内容编辑区
   - 在 `src/editor/panel.tsx` 中构建编辑表单结构：
     - 标题输入框：`.chips-basecard-editor__input`
     - 富文本编辑区：`.chips-basecard-editor__richtext`
     - 错误展示区域：`.chips-basecard-editor__errors`
   - 状态：已完成

2. 富文本工具栏
   - 实现以下工具按钮与行为：
     - 文本格式：加粗、斜体、下划线、删除线、上标、下标、行内代码
     - 段落格式：正文 / H1-H6、有序列表、无序列表、引用块
     - 样式：前景色、背景高亮色、字体大小、左/中/右/两端对齐
     - 插入：链接、图片、水平线
     - 清除格式：清除选区格式
   - 使用 `document.execCommand` 与辅助函数 `wrapSelectionWithTag` / `wrapSelectionWithStyle` 封装操作
   - 状态：已完成

3. 键盘快捷键
   - 在编辑区上监听 `keydown`：
     - `Ctrl/Cmd + B`：加粗
     - `Ctrl/Cmd + I`：斜体
     - `Ctrl/Cmd + U`：下划线
   - 快捷键触发后同步执行校验与回调
   - 状态：已完成

4. 校验与回调链路
   - 使用 `validateAndEmit` 统一封装：
     - 从 DOM 读取标题与 HTML 内容
     - 通过 `sanitizeRichTextHtml` 过滤 HTML，并在有变化时回写编辑区
     - 以 `defaultBasecardConfig` 为基线构造新的 `BasecardConfig`
     - 检查标题与内容是否为空，错误写入 `.chips-basecard-editor__errors`
     - 校验通过时调用 `props.onChange(nextConfig)`
   - 状态：已完成

5. 国际化接入
   - 在 `src/shared/i18n.ts` 中实现 `createTranslator(locale?)`
   - 在编辑器中替换：
     - 标题/内容标签文案
     - 标题/内容必填错误提示
   - 支持根据 `BasecardConfig.locale` 或 `navigator.language` 选择中英文
   - 状态：已完成

6. 集成测试
   - `tests/unit/editor-panel.test.tsx`：
     - 验证编辑器在字段变化时正确触发 `onChange`
   - `tests/integration/card-flow.test.ts`：
     - 同时挂载渲染器与编辑器
     - 模拟在编辑器中修改标题，验证查看器中的标题同步更新
   - 状态：已完成

> 阶段结论：编辑链路已具备完整的富文本编辑能力与最小校验闭环，可在后续阶段按需要扩展高级功能（撤销/重做、字数统计等）。
