# 富文本基础卡片插件需求文档（Milkdown 版）

版本：2026-03-25  
状态：已实现并对齐当前代码

---

## 1. 文档定位

- 本文档描述 `Chips-BaseCardPlugin/richtext-BCP` 当前正式需求口径。
- 当前正式实现已完成从旧版 HTML + `contenteditable` 到 Milkdown + Markdown 的重构。
- 旧版 HTML 方案文档已归档到 `归档/20260324-富文本基础卡片旧版HTML方案/`。

---

## 2. 重构目标

### 2.1 核心目标

1. 富文本编辑内核统一采用 **Milkdown 官方生态**。
2. 顶部固定工具栏与旧操作窗口全部移除，改为：
   - 文本选区触发的**紧凑图标悬浮工具框**
   - 编辑区右键触发的**上下文菜单**
3. 富文本正式内容源统一为 **Markdown**。
4. 基于正文可见文本长度，在 `content/*.yaml` 内联存储与卡片根目录 Markdown 文件引用之间自动切换。
5. 查看态渲染统一基于 Markdown 正式内容源，提升显示一致性与长文本稳定性。

### 2.2 约束目标

- 插件正式身份保持不变：
  - `pluginId = chips.basecard.richtext`
  - `cardType = base.richtext`
  - `card_type = RichTextCard`
- 继续遵循 Host / SDK / 编辑引擎既有正式插件导出契约，不新增私有运行时入口。
- 当前项目未发布第一版，不为旧版 HTML 配置保留兼容实现。

---

## 3. 功能需求

### 3.1 编辑内核与交互

#### 3.1.1 编辑器内核

- 编辑态必须使用 Milkdown 官方生态作为唯一正式编辑内核。
- 编辑器内部正式数据源为 Markdown 字符串，不再以 HTML DOM 作为持久化真源。
- 不再使用 `document.execCommand` 作为正式编辑能力实现基础。

#### 3.1.2 选区图标悬浮工具框

- 用户在富文本编辑区内选中非空文本后，编辑器必须显示悬浮工具框。
- 悬浮工具框必须锚定到当前选区附近位置；当选区变化时同步跟随。
- 悬浮工具框仅展示**文本处理相关操作**。
- 悬浮工具框按钮必须使用图标表达，当前实现允许使用占位图标，但交互结构必须稳定为图标按钮。
- 图标按钮必须使用紧凑白底视觉，优先保证在小尺寸编辑区中完整显示；热区与视觉尺寸需要兼顾可点击性和不遮挡正文。
- 鼠标悬停或键盘聚焦图标按钮时，必须出现 Tooltip 显示操作文字。
- 当出现以下任一情况时，悬浮工具框必须隐藏：
  - 选区折叠为光标；
  - 选区离开当前富文本编辑区；
  - 编辑器失焦；
  - 当前节点进入不可编辑或错误态。
- 悬浮工具框点击后必须保留并恢复原选区，避免按钮点击导致格式作用点漂移。
- 不再保留独立“操作工具组件窗口”、顶部固定工具栏或旁侧工具面板作为正式路径。

#### 3.1.3 悬浮工具框首期能力

首期仅提供以下行内能力：

- 加粗
- 斜体
- 行内代码
- 链接
- 清除当前选区行内格式

#### 3.1.4 编辑区右键菜单

- 用户在编辑区右键时，必须出现上下文菜单承载其他编辑操作。
- 右键菜单必须采用薯片生态 `menu` 组件契约：
  - `data-scope="menu"`
  - `data-part="root|content|item"`
  - `content` 使用 `role="menu"`
  - `item` 使用 `role="menuitem"`
- 右键菜单中的操作项必须使用图标 + 文本标签组合呈现。
- 右键菜单必须支持键盘导航：
  - `ArrowUp / ArrowDown`
  - `Home / End`
  - `Escape`

#### 3.1.5 首期正式支持的格式能力

首期正式支持 **Markdown 原生/主流基础能力**，包括：

- 行内格式：加粗、斜体、行内代码、链接、删除线、高亮、下划线、上标、下标。
- 段落结构：正文段落、标题（H1-H3）、引用块、有序列表、无序列表、任务列表、分隔线。
- 高级结构：表格、代码块、数学公式。
- 基础快捷键：
  - `Ctrl / Cmd + B`：加粗
  - `Ctrl / Cmd + I`：斜体
- 选区清理：移除当前选区的行内格式。

以下内容不作为本轮正式能力：

- 字号面板、字体家族面板、评论批注
- 图片、附件、外链媒体嵌入
- Slash Command、协同编辑

#### 3.1.6 输入稳定性

- 必须正确处理中文输入法（IME）组合输入，组合过程中不得强制提交或重建编辑器。
- 编辑过程中不得因局部状态更新导致光标跳首行、段落错位或选区丢失。
- 编辑过程中不得因为自动保存、资源导入或宿主回调身份变化导致编辑器重挂载、输入失焦或选区重置。
- 编辑区正文必须显示文本插入光标，不能出现按钮式手型光标。
- 用户仍在编辑面板内部输入时，系统不得把草稿高频提交到正式配置与落盘链路。
- 编辑态需要区分“预览刷新”和“正式保存”两种输出：复合卡片预览应近实时更新，但卡片目录写盘只能在离开编辑面板或显式保存时发生。
- 粘贴行为必须先经过正式清洗与归一，再进入编辑器文档状态。
- 当用户粘贴 Markdown 文本时，系统必须直接按 Markdown 语法生成结构化节点，不能先以普通字符插入再依赖后续手动编辑触发语法刷新。

### 3.2 持久化与存储策略

#### 3.2.1 正式持久化格式

- 富文本卡片的正式内容格式统一为 Markdown。
- 内容配置文件继续位于 `content/{baseCardId}.yaml`。
- 当内容以内联方式保存时，Markdown 直接写入内容配置。
- 当内容以文件方式保存时，Markdown 文档写入卡片根目录，内容配置仅保存相对路径引用。

#### 3.2.2 200 字分流规则

- 编辑器在每次输出正式配置前，必须基于**正文可见文本长度**执行存储分流判断。
- 当正文长度 **小于或等于 200 个字符** 时：
  - Markdown 内容直接保存到 `content/{baseCardId}.yaml`。
- 当正文长度 **大于 200 个字符** 时：
  - Markdown 内容保存为卡片根目录中的 `.md` 文件；
  - `content/{baseCardId}.yaml` 中仅保存根目录相对路径引用。

#### 3.2.3 存储切换规则

- 当正文从 `<= 200` 增长为 `> 200` 时，系统必须自动从内联模式切换为文件模式。
- 当正文从 `> 200` 缩短为 `<= 200` 时，系统必须自动从文件模式切回内联模式。
- 长文本文件模式的自动保存必须以“离开编辑面板时 flush”为正式落盘时机，不能把每次按键都升级为一次正式文件写入与预览刷新。
- 在用户仍处于编辑中的时间窗内，系统可以使用运行时 inline Markdown 草稿驱动复合卡片预览，但不能把这份临时预览草稿直接当作最终磁盘存储结果。
- 切换模式时必须通过宿主正式资源链路完成文件写入、替换与删除，不得绕开 `importResource/deleteResource` 或私自写磁盘。
- 内容配置中不得写入：
  - 系统绝对路径
  - 临时 `blob:` URL
  - 编辑态临时缓存键

### 3.3 查看态渲染

#### 3.3.1 渲染目标

- 查看态必须基于同一份 Markdown 正式内容源渲染。
- 查看态不再复用旧版 HTML 富文本正文直出方案。
- 查看态不应挂载完整可编辑实例，只输出只读内容渲染结果。

#### 3.3.2 显示要求

- 内容高度必须由正文真实内容驱动，不允许再出现额外固定空白区。
- 根容器与正文背景保持透明，不额外绘制独立卡片壳层。
- 标题、段落、列表、引用块、行内代码等基础排版必须在查看态稳定呈现。
- 删除线、高亮、下划线、上标、下标、表格、代码块与数学公式必须在查看态与编辑态保持一致语义。
- 长文本场景下，初次渲染与后续刷新不得频繁闪烁或重复整棵重建。

#### 3.3.3 一致性要求

- 编辑态输出的 Markdown 与查看态渲染结果必须保持语义一致。
- 当内容来源为文件引用时，查看态必须能通过正式资源解析链路读取对应 Markdown 文档并渲染。
- 当宿主提供文本资源读取能力时，富文本正文必须优先通过该正式能力读取，不得把正文读取退化为 URL 请求。
- 当文件引用在编辑引擎本地运行时被解析为 `file://` 时，且当前运行时未直接提供文本资源读取能力，富文本正文必须通过正式文件服务读取，不得直接对 `file://` 执行 `fetch(...)`。

### 3.4 主题与多语言

- 所有用户可见文案继续走 `i18n/`。
- 编辑态和查看态继续消费 Host 注入的主题 token 与样式变量。
- Tooltip 与右键菜单必须遵循薯片生态 `tooltip/menu` 组件契约与 token 命名。
- 插件不得私自引入独立视觉系统，不得在代码中硬编码整套色板。

## 4. 非功能需求

### 4.1 架构与依赖

- 编辑器内核仍以 Milkdown 官方生态为主，不引入额外富文本内核。
- 允许围绕 Markdown 语法扩展引入与 Milkdown 装配层直接相关的 remark / micromark / KaTeX 依赖，但必须统一收口到插件内部 Markdown 扩展层，且与 `@milkdown/*` 版本治理一起纳入测试。
- 插件仍需遵循：
  - `renderBasecardView`
  - `renderBasecardEditor`
  - `basecardDefinition`
  三个正式入口契约。

### 4.2 质量与稳定性

- 不允许保留未闭环的临时主流程方案。
- 不允许在主流程中为富文本类型增加宿主特判。
- 编辑态、查看态、schema、测试与文档必须同步收口。

### 4.3 测试要求

必须至少覆盖以下验证：

- 选区出现/隐藏图标悬浮工具框
- Tooltip 显示与隐藏
- 编辑区右键菜单打开与键盘导航
- Markdown 编辑结果正确序列化
- `<= 200` 内联保存与 `> 200` 文件保存
- 文件模式与内联模式来回切换
- 查看态对 inline / file 两类配置均可正确渲染
- IME、快捷键、粘贴与失焦等关键交互稳定

---

## 5. 本轮明确不做

- 旧版 HTML 数据兼容迁移
- 评论、协同编辑
- 图片上传、媒体嵌入、附件引用
- 独立属性面板、右栏设置区、顶部固定工具栏回退方案
- 基于字符阈值之外的复杂分级存储策略

---

## 6. 验收口径

需满足以下结果：

1. 选中正文后出现紧凑图标悬浮工具框，取消选区后自动隐藏。
2. 图标按钮悬停或聚焦时出现气泡提示；编辑区右键出现块级操作菜单。
3. 编辑器正式输出 Markdown，而非 HTML。
4. 200 字以内内容写入 `content/*.yaml`；超过 200 字自动改写为卡片根目录 Markdown 文件引用。
5. 查看态可正确显示 inline / file 两种内容来源，并保持主题与透明背景约束。
6. 所有关联测试通过，旧版 HTML 方案文档继续保留在归档目录，不再作为当前口径引用。
