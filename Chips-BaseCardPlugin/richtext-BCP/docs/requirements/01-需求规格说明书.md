# 富文本基础卡片插件重构需求文档（Milkdown 版）

版本：2026-03-24  
状态：设计已确认，待开发  

---

## 1. 文档定位

- 本文档描述 `Chips-BaseCardPlugin/richtext-BCP` 下一轮正式重构目标。
- 当前仓库中的富文本插件仍然是基于 HTML + `contenteditable` 的旧实现；本文件描述的是**下一版目标行为**，不是当前已上线行为。
- 本轮任务只交付新版需求文档、技术文档和开发计划，并将旧版方案文档归档；实现与测试改造属于后续开发任务。

---

## 2. 重构目标

### 2.1 核心目标

1. 富文本编辑内核统一改为 **Milkdown 官方生态**。
2. 取消现有固定工具栏/操作工具组件窗口，改为**文本选区触发的悬浮工具框**。
3. 富文本正式内容源统一改为 **Markdown**，不再把 HTML 作为正式持久化格式。
4. 根据正文长度自动在 `content/*.yaml` 内联保存与卡片根目录 Markdown 文件引用之间切换。
5. 重构查看态渲染链路，提升显示一致性、内容高度稳定性和长文本渲染性能。

### 2.2 约束目标

- 保持插件正式身份不变：
  - `pluginId = chips.basecard.richtext`
  - `cardType = base.richtext`
  - `card_type = RichTextCard`
- 继续遵循 Host / SDK / 编辑引擎既有正式插件导出契约，不新增私有运行时入口。
- 不为旧版 HTML 配置保留兼容实现；当前项目尚未发布第一版，本轮按新契约直接重构。

---

## 3. 功能需求

### 3.1 编辑内核与交互

#### 3.1.1 编辑器内核

- 编辑态必须使用 Milkdown 官方生态作为唯一正式编辑内核。
- 编辑器内部正式数据源为 Markdown 字符串，不再以 HTML DOM 作为持久化真源。
- 不再使用 `document.execCommand` 作为正式编辑能力实现基础。

#### 3.1.2 悬浮工具框

- 用户在富文本编辑区内选中非空文本后，编辑器必须显示悬浮工具框。
- 悬浮工具框必须锚定到当前选区附近位置；当选区变化时同步跟随。
- 当出现以下任一情况时，悬浮工具框必须隐藏：
  - 选区折叠为光标；
  - 选区离开当前富文本编辑区；
  - 编辑器失焦；
  - 当前节点进入不可编辑或错误态。
- 悬浮工具框点击后必须保留并恢复原选区，避免按钮点击导致格式作用点漂移。
- 不再保留独立“操作工具组件窗口”、顶部固定工具栏或旁侧工具面板作为正式路径。

#### 3.1.3 首期正式支持的格式能力

首期正式支持 **Markdown 原生/主流基础能力**，包括：

- 行内格式：加粗、斜体、删除线、行内代码、链接。
- 段落结构：正文段落、标题（H1-H3）、引用块、有序列表、无序列表、分隔线。
- 基础快捷键：
  - `Ctrl / Cmd + B`：加粗
  - `Ctrl / Cmd + I`：斜体
- 选区清理：移除当前选区的行内格式。

以下内容不作为本轮正式能力：

- 下划线、高亮色、字号面板、字体家族面板
- 表格、任务列表、数学公式、评论批注
- 图片、附件、外链媒体嵌入
- Slash Command、右键命令面板、协同编辑

#### 3.1.4 输入稳定性

- 必须正确处理中文输入法（IME）组合输入，组合过程中不得强制提交或重建编辑器。
- 编辑过程中不得因局部状态更新导致光标跳首行、段落错位或选区丢失。
- 粘贴行为必须先经过正式清洗与归一，再进入编辑器文档状态。

### 3.2 持久化与存储策略

#### 3.2.1 正式持久化格式

- 富文本卡片的正式内容格式统一为 Markdown。
- 内容配置文件继续位于 `content/{baseCardId}.yaml`。
- 当内容以内联方式保存时，Markdown 直接写入内容配置。
- 当内容以文件方式保存时，Markdown 文档写入卡片根目录，内容配置仅保存相对路径引用。

#### 3.2.2 200 字分流规则

- 编辑器在每次输出正式配置前，必须基于**正文可见文本长度**执行存储分流判断。
- 当正文长度 **小于或等于 200 个字符** 时：
  - Markdown 内容直接保存到 `content/{baseCardId}.yaml`。
- 当正文长度 **大于 200 个字符** 时：
  - Markdown 内容保存为卡片根目录中的 `.md` 文件；
  - `content/{baseCardId}.yaml` 中仅保存根目录相对路径引用。

#### 3.2.3 存储切换规则

- 当正文从 `<= 200` 增长为 `> 200` 时，系统必须自动从内联模式切换为文件模式。
- 当正文从 `> 200` 缩短为 `<= 200` 时，系统必须自动从文件模式切回内联模式。
- 切换模式时必须通过宿主正式资源链路完成文件写入、替换与删除，不得绕开 `importResource/deleteResource` 或私自写磁盘。
- 内容配置中不得写入：
  - 系统绝对路径
  - 临时 `blob:` URL
  - 编辑态临时缓存键

### 3.3 查看态渲染

#### 3.3.1 渲染目标

- 查看态必须基于同一份 Markdown 正式内容源渲染。
- 查看态不再复用旧版 HTML 富文本正文直出方案。
- 查看态不应挂载完整可编辑实例，只输出只读内容渲染结果。

#### 3.3.2 显示要求

- 内容高度必须由正文真实内容驱动，不允许再出现额外固定空白区。
- 根容器与正文背景保持透明，不额外绘制独立卡片壳层。
- 标题、段落、列表、引用块、行内代码等基础排版必须在查看态稳定呈现。
- 长文本场景下，初次渲染与后续刷新不得频繁闪烁或重复整棵重建。

#### 3.3.3 一致性要求

- 编辑态输出的 Markdown 与查看态渲染结果必须保持语义一致。
- 当内容来源为文件引用时，查看态必须能通过正式资源解析链路读取对应 Markdown 文档并渲染。

### 3.4 主题与多语言

- 所有用户可见文案继续走 `i18n/`。
- 编辑态和查看态继续消费 Host 注入的主题 token 与样式变量。
- 插件不得私自引入独立视觉系统，不得在代码中硬编码整套色板。

---

## 4. 非功能需求

### 4.1 架构与依赖

- 只允许引入 Milkdown 官方生态包作为本轮新增编辑器依赖。
- 不引入未确认的第三方富文本库、Markdown 运行时或样式系统。
- 插件仍需遵循：
  - `renderBasecardView`
  - `renderBasecardEditor`
  - `basecardDefinition`
  三个正式入口契约。

### 4.2 质量与稳定性

- 不允许保留临时版、占位版或 TODO 方案。
- 不允许在主流程中为富文本类型增加宿主特判。
- 编辑态、查看态、schema、测试与文档必须同步收口。

### 4.3 测试要求

后续实现必须至少覆盖以下验证：

- 选区出现/隐藏悬浮工具框
- Markdown 编辑结果正确序列化
- `<= 200` 内联保存与 `> 200` 文件保存
- 文件模式与内联模式来回切换
- 查看态对 inline / file 两类配置均可正确渲染
- IME、快捷键、粘贴与失焦等关键交互稳定

---

## 5. 本轮明确不做

- 旧版 HTML 数据兼容迁移
- 表格、公式、任务列表、评论、协同编辑
- 图片上传、媒体嵌入、附件引用
- 独立属性面板、右栏设置区、顶部固定工具栏回退方案
- 基于字符阈值之外的复杂分级存储策略

---

## 6. 验收口径

后续开发完成后，需满足以下结果：

1. 选中正文后出现悬浮工具框，取消选区后自动隐藏。
2. 编辑器正式输出 Markdown，而非 HTML。
3. 200 字以内内容写入 `content/*.yaml`；超过 200 字自动改写为卡片根目录 Markdown 文件引用。
4. 查看态可正确显示 inline / file 两种内容来源，并保持主题与透明背景约束。
5. 所有关联测试通过，旧版 HTML 方案文档继续保留在归档目录，不再作为当前口径引用。
