# 富文本基础卡片插件 - 需求规格说明书（vNext）

版本：0.1.0  
状态：正式版（新架构重构版）  

---

## 1. 产品概述

- 插件类型：基础卡片插件（`type: card`）  
- 插件标识：`chips.basecard.richtext`  
- 支持卡片类型：`base.richtext`  
- 运行环境：由薯片主机在 iframe / 编辑面板中加载运行  

本插件提供富文本内容的渲染与编辑能力，用于在复合卡片中承载格式化文本内容，是文本类基础卡片中的核心类型之一。

---

## 2. 功能范围

### 2.1 渲染组件（查看模式）

在查看器或通过 SDK 的统一显示接口被调用时，插件根据 `BasecardConfig` 渲染一个只读的富文本卡片页面：

- 标题区域
  - 渲染 `config.title`（纯文本）
  - 使用类名：`chips-basecard__title`
- 内容区域
  - 将 `config.body` 作为 HTML 渲染
  - 使用类名：`chips-basecard__body chips-basecard__body--richtext`
  - 支持的富文本结构至少包括：
    - 文本内联格式：加粗、斜体、下划线、删除线、上标、下标、行内代码
    - 段落与结构：段落、H1-H6 标题、有序/无序列表、引用块、水平线、换行
    - 样式：文字颜色、背景高亮、字体大小、对齐方式
    - 链接与图片：`<a>` 超链接、`<img>` 图片
- 安全处理
  - 渲染前对 HTML 做安全过滤：
    - 仅保留白名单标签与属性
    - 移除 `javascript:`、`data:` 等危险协议
    - 为 `<a>` 自动补全 `target="_blank" rel="noopener"`

### 2.2 编辑组件（编辑模式）

在编辑引擎中，插件作为基础卡片编辑器嵌入编辑面板，提供所见即所得的富文本编辑能力：

- 标题输入
  - 文本输入框编辑 `title`
  - 校验：必填、非空字符串
- 富文本编辑区
  - `contenteditable` 容器用于编辑 `body` 的 HTML 内容
  - 工具栏提供常用格式化命令：
    - 文本格式：加粗 / 斜体 / 下划线 / 删除线 / 上标 / 下标 / 行内代码
    - 段落格式：正文 / H1-H6、有序列表、无序列表、引用块
    - 样式：文字颜色、背景高亮、字体大小（px），左/中/右/两端 对齐
    - 插入：链接、图片、水平线
    - 清除格式：移除内联样式与基础格式
  - 键盘快捷键：
    - `Ctrl / Cmd + B`：加粗
    - `Ctrl / Cmd + I`：斜体
    - `Ctrl / Cmd + U`：下划线
- 校验与回调
  - 标题与内容均不能为空
  - 校验失败时在编辑面板内展示错误列表
  - 校验通过后，通过 `onChange(BasecardConfig)` 向上层提交完整配置
- 国际化
  - 基于 `i18n/zh-CN.json` 与 `i18n/en-US.json` 实现中英文文案
  - 使用 `BasecardConfig.locale` 或浏览器语言决定当前语言

## 3. 非功能需求

- 与主机架构解耦：
  - 不直接依赖旧版 `ChipsCore`、不内嵌主题加载、资源加载逻辑
  - 仅通过标准插件入口 `renderBasecardView` / `renderBasecardEditor` 与 Host 集成
- 安全性：
  - 对渲染 HTML 进行最小但明确的 XSS 防护
  - 不执行脚本、不保留危险协议与事件属性
- 可测试性：
  - 提供基于 Vitest + jsdom 的单元测试与集成测试
  - 覆盖渲染、编辑器基础回调以及 HTML 清洗逻辑

---

## 4. 明确不在本次实现范围内的需求

以下能力在旧版插件中存在或在原需求文档中定义，但**根据当前用户要求暂不实现**，后续如需恢复将通过独立任务处理：

- 撤销 / 重做历史栈与可视化历史 UI
- 链接点击后的外部行为（如统一在查看器中打开）
- 图片点击后的外部行为（如调用系统图片查看器）
- 字数统计 UI 与 `maxLength` 限制
- 基于文件的内容来源切换（`content_source: file`）与卡片文件资源读取
- 主题加载与响应式断点布局逻辑（交由主题与宿主负责）

这些能力在旧版文档中保留为生态长期目标，本插件在 vNext 架构下实现的是“核心富文本编辑与渲染”子集，并保证后续可以在不破坏当前数据模型的前提下扩展。
