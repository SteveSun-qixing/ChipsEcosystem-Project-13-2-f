# 富文本基础卡片插件 - 架构设计（vNext）

版本：0.1.0  
状态：正式版  

---

## 1. 架构定位

- 插件类型：基础卡片插件（card）
- 职责边界：
  - 负责富文本卡片在查看器中的展示（渲染组件）
  - 负责富文本卡片在编辑引擎中的编辑交互（编辑组件）
  - 不负责卡片文件读取、主题加载、SDK 显示链路等上层职责
- 集成方式：
  - Host 在 iframe 中加载插件打包产物（`dist/src/index.js`）
  - 通过导出的 `renderBasecardView` / `renderBasecardEditor` 进行调用

---

## 2. 模块划分

目录结构（仅列出核心模块）：

- `src/index.ts`
  - 插件对 Host 暴露的唯一入口文件
  - 定义 `BasecardRenderContext` / `BasecardEditorContext`
  - 导出：
    - `renderBasecardView(ctx)`：挂载渲染组件
    - `renderBasecardEditor(ctx)`：挂载编辑组件
- `src/schema/card-config.ts`
  - 定义 `BasecardConfig`：
    - `id: string`
    - `title: string`
    - `body: string`（HTML）
    - `locale?: string`
  - 提供 `defaultBasecardConfig` 作为配置合并基线
- `src/render/view.tsx`
  - 纯 DOM 渲染模块，不依赖第三方框架
  - 负责根据 `BasecardConfig` 构建渲染用的 DOM 结构
- `src/render/runtime.ts`
  - 提供 `mountBasecardView(ctx)`：
    - 清空容器
    - 可选注入 `themeCssText`
    - 调用 `createBasecardViewRoot` 并挂载
    - 返回卸载函数
- `src/editor/panel.tsx`
  - 纯 DOM 实现的富文本编辑面板
  - 包含表单结构、工具栏、编辑区、校验逻辑和回调封装
- `src/editor/runtime.ts`
  - 提供 `mountBasecardEditor(ctx)`：
    - 清空容器
    - 调用 `createBasecardEditorRoot` 并挂载
    - 返回卸载函数
- `src/shared/utils.ts`
  - 通用工具函数：
    - `isNonEmptyString`
    - `sanitizeRichTextHtml`：富文本 HTML 安全过滤
- `src/shared/i18n.ts`
  - 轻量级 i18n 封装：
    - `createTranslator(locale?)` 创建基于 `locale` 的取词函数
- `config/logging.ts`
  - `createLogger` 帮助函数，统一前缀为 `[basecard-richtext]`

## 3. 渲染链路设计

### 3.1 调用入口

上层 Host / SDK 调用流程：

```ts
import { renderBasecardView } from "chips.basecard.richtext";

const dispose = renderBasecardView({
  container: iframeDocument.body,
  config: basecardConfig,
  themeCssText, // 可选，由主题系统注入
});
```

### 3.2 渲染流程

1. `mountBasecardView` 清空容器
2. 如提供 `themeCssText`，在容器内插入 `<style>` 标签
3. 调用 `createBasecardViewRoot(config)` 创建根 DOM 节点：
   - 外层容器：`<div class="chips-basecard chips-basecard--richtext">`
   - 标题：`<h2 class="chips-basecard__title">`
   - 内容：`<div class="chips-basecard__body chips-basecard__body--richtext">`
4. 使用 `sanitizeRichTextHtml` 对 `config.body` 进行 HTML 过滤后塞入内容区域
5. 返回卸载函数，用于 Host 在 iframe 生命周期结束时清理 DOM

## 4. 编辑链路设计

### 4.1 调用入口

编辑引擎在编辑面板内调用：

```ts
import { renderBasecardEditor } from "chips.basecard.richtext";

const dispose = renderBasecardEditor({
  container: editorPanelElement,
  initialConfig: basecardConfig,
  onChange(nextConfig) {
    // 将 BasecardConfig 写回卡片文件
  },
});
```

### 4.2 编辑器内部结构

`createBasecardEditorRoot` 构建 DOM：

- 根容器：`.chips-basecard-editor.chips-basecard-editor--richtext`
- 表单：`.chips-basecard-editor__form`
  - 标题区域：
    - `<label>` + `<input type="text" class="chips-basecard-editor__input">`
  - 内容区域：
    - `<label>` 文本
    - 工具栏：`.chips-basecard-editor__toolbar`
    - 编辑区：`<div contenteditable class="chips-basecard-editor__richtext">`
  - 错误展示区域：`.chips-basecard-editor__errors`

### 4.3 数据与校验

- 每次输入或执行格式化命令后调用 `validateAndEmit`：
  - 从 DOM 读取当前标题与编辑区 HTML
  - 使用 `sanitizeRichTextHtml` 过滤 HTML，必要时回写到编辑区
  - 构造新的 `BasecardConfig`：
    - 以 `defaultBasecardConfig` 为基线
    - 合并 `props.initialConfig`
    - 覆盖 `title` 与 `body`
  - 校验规则：
    - `title` 必须是非空字符串
    - `body` 的 `textContent` 不能为空
  - 校验通过时调用 `props.onChange(nextConfig)`

## 5. 与旧版插件的差异

- **不再依赖 Vue 与 ChipsCore**：
  - 旧版通过 Vue 组件和 Core 服务完成渲染、资源加载和主题应用；
  - 新版仅在插件内部使用原生 DOM，资源与主题由 Host 统一处理。
- **配置模型收敛**：
  - 旧版 `RichTextCardConfig` 包含 `content_source`、`content_file`、`layout` 等字段；
  - 新版 `BasecardConfig` 将所有外部依赖移交给上层，仅保留：
    - `id`、`title`、`body`、`locale`
- **安全策略调整**：
  - 旧版在加载 HTML 后进行过滤；
  - 新版在编辑与渲染两个环节统一通过 `sanitizeRichTextHtml` 处理。
- **功能范围收敛**：
  - 按用户要求暂不实现撤销/重做、字数统计、`maxLength` 等高级功能；
  - 保持 API 与内部结构可扩展，为未来补齐这些能力预留空间。
