# 富文本基础卡片插件 - 架构设计（vNext）

版本：0.1.0  
状态：正式版  

---

## 1. 架构定位

- 插件类型：基础卡片插件（card）
- 职责边界：
  - 负责富文本卡片在查看器中的展示（渲染组件）
  - 负责富文本卡片在编辑引擎中的编辑交互（编辑组件）
  - 不负责卡片文件读取、主题加载、SDK 显示链路等上层职责
- 集成方式：
  - Host 在 iframe 中加载插件打包产物（`dist/index.mjs`）
  - 通过导出的 `renderBasecardView` / `renderBasecardEditor` 进行调用

---

## 2. 模块划分

目录结构（仅列出核心模块）：

- `src/index.ts`
  - 插件对 Host 暴露的唯一入口文件
  - 定义 `BasecardRenderContext` / `BasecardEditorContext`
  - 导出：
    - `renderBasecardView(ctx)`：挂载渲染组件
    - `renderBasecardEditor(ctx)`：挂载编辑组件
- `src/schema/card-config.ts`
  - 定义 `BasecardConfig`：
    - `id: string`
    - `body: string`（HTML）
    - `locale?: string`
  - 提供 `defaultBasecardConfig` 作为配置合并基线
  - 官方编辑引擎在新建 `base.richtext` 实例时，会由描述符层注入 `<p>123456789</p>` 作为正式初始正文
- `src/render/view.tsx`
  - 纯 DOM 渲染模块，不依赖第三方框架
  - 负责根据 `BasecardConfig` 构建渲染用的 DOM 结构
- `src/render/runtime.ts`
  - 提供 `mountBasecardView(ctx)`：
    - 清空容器
    - 可选注入 `themeCssText`
    - 调用 `createBasecardViewRoot` 并挂载
    - 返回卸载函数
- `src/editor/panel.tsx`
  - 纯 DOM 实现的富文本编辑面板
  - 包含表单结构、工具栏、编辑区、校验逻辑和回调封装
- `src/editor/runtime.ts`
  - 提供 `mountBasecardEditor(ctx)`：
    - 清空容器
    - 调用 `createBasecardEditorRoot` 并挂载
    - 返回卸载函数
- `src/shared/utils.ts`
  - 通用工具函数：
    - `isNonEmptyString`
    - `sanitizeRichTextHtml`：富文本 HTML 安全过滤
- `src/shared/i18n.ts`
  - 轻量级 i18n 封装：
    - `createTranslator(locale?)` 创建基于 `locale` 的取词函数
- `config/logging.ts`
  - `createLogger` 帮助函数，统一前缀为 `[basecard-richtext]`

## 3. 渲染链路设计

### 3.1 调用入口

上层 Host / SDK 调用流程：

```ts
import { renderBasecardView } from "chips.basecard.richtext";

const dispose = renderBasecardView({
  container: iframeDocument.body,
  config: basecardConfig,
  themeCssText, // 可选，由主题系统注入
});
```

### 3.2 渲染流程

1. `mountBasecardView` 清空容器
2. 如提供 `themeCssText`，在容器内插入 `<style>` 标签
3. 调用 `createBasecardViewRoot(config)` 创建根 DOM 节点：
   - 外层容器：`<div class="chips-basecard chips-basecard--richtext">`
   - 内容：`<div class="chips-basecard__body chips-basecard__body--richtext">`
4. 使用 `sanitizeRichTextHtml` 对 `config.body` 进行 HTML 过滤后塞入内容区域
5. 返回卸载函数，用于 Host 在 iframe 生命周期结束时清理 DOM

## 4. 编辑链路设计

### 4.1 调用入口

编辑引擎在编辑面板内调用：

```ts
import { renderBasecardEditor } from "chips.basecard.richtext";

const dispose = renderBasecardEditor({
  container: editorPanelElement,
  initialConfig: basecardConfig,
  onChange(nextConfig) {
    // 将 BasecardConfig 写回卡片文件
  },
});
```

### 4.2 编辑器内部结构

`createBasecardEditorRoot` 构建 DOM：

- 根容器：`.chips-basecard-editor.chips-basecard-editor--richtext`
- 内联样式：编辑器根节点内注入 `<style>`，确保 iframe 内工具栏、滚动区和错误区样式闭环
- 工具栏壳层：`.chips-basecard-editor__toolbar-shell`
- 工具栏面板：`.chips-basecard-editor__toolbar-panel`
- 工具栏头部：`.chips-basecard-editor__toolbar-header`
- 工具栏切换按钮：`.chips-basecard-editor__toolbar-toggle`
- 工具栏内容区：`.chips-basecard-editor__toolbar-content`
- 滚动壳层：`.chips-basecard-editor__surface-frame`
- 独立滚动区：`.chips-basecard-editor__surface`
- 编辑区：`<div contenteditable class="chips-basecard-editor__richtext">`
- 错误展示区域：`.chips-basecard-editor__errors`

结构约束：

- 编辑器不再提供标题输入区域，顶部首个可见区域必须是工具栏；
- 编辑器结构固定为“上工具栏区 + 下正文滚动区”两层，不再使用悬浮工具栏方案；
- 工具栏必须固定在编辑器顶部，不与正文滚动区绑定；
- 工具栏宽度与正文滚动区宽度保持一致，避免额外悬浮宽度与左右留白；
- 工具栏允许折叠成紧凑条带，折叠态只保留头部与展开按钮；
- `html/body/#chips-basecard-editor-root` 与容器节点必须形成完整 `height: 100%` 链，并禁止外层整页滚动；
- 正文编辑只允许滚动 `.chips-basecard-editor__surface`；
- `.chips-basecard-editor__surface` 必须直接承担正文滚动，滚动时顶部工具栏保持原位；
- 工具栏按钮、颜色选择器、段落样式选择器都要通过保存/恢复选区后再执行命令，避免点击工具栏导致格式作用到错误位置。

### 4.3 数据与校验

- 每次输入或执行格式化命令后调用 `validateAndEmit`：
  - 从 DOM 读取编辑区 HTML
  - 使用 `sanitizeRichTextHtml` 过滤 HTML，必要时回写到编辑区
  - 构造新的 `BasecardConfig`：
    - 显式保留 `id`
    - 覆盖 `body`
    - 透传或回退 `locale`
  - 校验规则：
    - `body` 的有效内容不能为空；纯空白文本、被清洗后为空的 HTML 都视为无效
  - 仅当配置签名变化时调用 `props.onChange(nextConfig)`，避免无效重复上报

交互细节：

- 粘贴时优先消费 `text/html`，统一经过 `sanitizeRichTextHtml` 过滤后再插入；
- 如果剪贴板只有纯文本，则转换成段落 / 换行对应的 HTML 结构；
- 常规输入阶段只生成安全配置草稿，不立即把清洗后的 HTML 回写到编辑区 DOM；DOM 归一化只在失焦等安全时机执行，避免光标漂移；
- 输入法组合输入通过 `compositionstart/compositionend` 单独管理，组合过程中暂停提交，结束后再统一发出变更；
- `Enter` / `Shift + Enter` 必须优先使用编辑区当前实时选区执行插入，而不是依赖工具栏保存选区；
- 编辑区在首次聚焦与回车插入前固定 `defaultParagraphSeparator = "p"`，确保段落分裂始终落到正式 `p` 结构；
- `Ctrl/Cmd + B/I/U` 走与工具栏一致的正式命令链，保持行为一致；
- 链接与图片插入在 prompt 交互结束后必须恢复原始选区，再写回经过清洗的 HTML。

## 5. 与旧版插件的差异

- **不再依赖 Vue 与 ChipsCore**：
  - 旧版通过 Vue 组件和 Core 服务完成渲染、资源加载和主题应用；
  - 新版仅在插件内部使用原生 DOM，资源与主题由 Host 统一处理。
- **配置模型收敛**：
  - 旧版 `RichTextCardConfig` 包含 `content_source`、`content_file`、`layout` 等字段；
  - 新版 `BasecardConfig` 将所有外部依赖移交给上层，仅保留：
    - `id`、`body`、`locale`
- **安全策略调整**：
  - 旧版在加载 HTML 后进行过滤；
  - 新版在编辑与渲染两个环节统一通过 `sanitizeRichTextHtml` 处理。
- **功能范围收敛**：
  - 按用户要求暂不实现撤销/重做、字数统计、`maxLength` 等高级功能；
  - 保持 API 与内部结构可扩展，为未来补齐这些能力预留空间。
