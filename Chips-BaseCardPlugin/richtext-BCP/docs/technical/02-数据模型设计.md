# 富文本基础卡片插件 - 数据模型设计

版本：0.1.0  
状态：正式版  

---

## 1. BasecardConfig

存储位置：复合卡片文件的内容配置段，由上层卡片服务负责读写。插件在运行时只接收解析后的 `BasecardConfig` 对象。

```ts
export interface BasecardConfig {
  /**
   * 基础卡片实例 ID，由复合卡片结构与卡片服务负责生成与管理。
   */
  id: string;

  /**
   * 富文本内容，使用 HTML 字符串保存。
   */
  body: string;

  /**
   * 语言代码，例如 zh-CN / en-US。
   */
  locale?: string;
}
```

默认配置：

```ts
export const defaultBasecardConfig: BasecardConfig = {
  id: "",
  body: "",
  locale: "zh-CN",
};
```

说明：

- `defaultBasecardConfig` 仅作为插件内部配置合并基线，不直接代表“新建基础卡片实例”的最终初始正文；
- 官方编辑引擎当前通过描述符层创建新实例时，会注入 `<p>123456789</p>` 作为正式初始内容；
- 已存在卡片若正文为空，归一化链路仍保持空正文语义，不会在打开旧数据时自动补写默认文本。

---

## 2. 字段说明

| 字段    | 类型   | 必填 | 默认值 | 说明                                                         |
|---------|--------|------|--------|--------------------------------------------------------------|
| `id`    | string | 是   | `""`   | 基础卡片实例标识，由卡片服务统一分配，不在插件内部生成     |
| `body`  | string | 是   | `""`   | 富文本内容，HTML 字符串，仅允许受支持标签和样式             |
| `locale`| string | 否   | `"zh-CN"` | 语言代码，遵循 BCP-47 规范，如 `zh-CN` / `en-US` 等      |

## 3. 富文本 HTML 约束

### 3.1 支持的标签

插件在编辑和渲染环节通过 `sanitizeRichTextHtml` 统一约束可用标签：

- 文本格式：
  - `<strong>`, `<b>`：加粗
  - `<em>`, `<i>`：斜体
  - `<u>`：下划线
  - `<s>`, `<del>`：删除线
  - `<sup>`：上标
  - `<sub>`：下标
  - `<code>`：行内代码
- 段落结构：
  - `<p>`：段落
  - `<h1>`-`<h6>`：标题
  - `<ul>`, `<ol>`, `<li>`：列表
  - `<blockquote>`：引用块
  - `<hr>`：分隔线
  - `<br>`：换行
  - `<span>`：内联容器
- 链接与图片：
  - `<a>`：超链接
  - `<img>`：图片

不在白名单中的标签会被“展开”为纯内容（移除标签但保留子节点）。

### 3.2 支持的属性

- 全局允许：
  - `title`
  - `style`（仅允许安全样式属性）
- `<a>` 允许：
  - `href`（安全协议）
  - `target`（自动补全 `_blank`）
  - `rel`（自动补全 `noopener`）
- `<img>` 允许：
  - `src`（安全协议或相对路径）
  - `alt`
  - `width`
  - `height`

### 3.3 样式属性白名单

`style` 中仅保留以下属性：

- `color`
- `background-color`
- `font-size`
- `text-align`
- `font-weight`
- `font-style`
- `text-decoration`

其它样式属性会被移除。

### 3.4 URL 安全规则

在 `href` / `src` 中禁止以下协议：

- `javascript:`
- `data:`
- `vbscript:`

允许：

- `http:`
- `https:`
- `mailto:`
- 无协议的相对路径（如 `./path`、`../path`、`/path`）

## 4. 校验规则

编辑器在调用 `onChange` 之前会做最小化校验：

- 内容：
  - `textContent` 去除前后空白后长度必须大于 0
  - HTML 内容保存前先经过 `sanitizeRichTextHtml` 处理

上层如果需要更多业务规则（例如最大字数限制、敏感词过滤等），应在卡片服务层或复合卡片层实现，插件不内嵌这些业务逻辑。

---

## 5. 国际化模型

- 内部使用 `locale?: string` 决定编辑器使用的语言：
  - 优先使用 `BasecardConfig.locale`
  - 其次尝试浏览器环境的 `navigator.language`
  - 默认回退到中文 `zh-CN`
- 文案来源：
  - `i18n/zh-CN.json`
  - `i18n/en-US.json`
- 取词方式：
  - 通过 `createTranslator(locale)` 创建 `t(key)` 函数
  - 当前实现主要覆盖：
    - 工具栏、占位符与内容标签
    - 内容为空的校验提示
