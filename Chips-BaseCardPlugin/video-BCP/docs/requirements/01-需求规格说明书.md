# 视频基础卡片插件需求规格说明书

版本：0.1.0  
状态：正式版

## 1. 产品定位

- 插件类型：基础卡片插件（`type: card`）
- 插件标识：`chips.basecard.video`
- 支持卡片类型：`base.video`
- 兼容别名：`VideoCard`

本插件用于在复合卡片中承载视频内容，负责视频文件、封面和基础元信息的查看与编辑，并把视频打开动作统一交给 Host `resource.open` 路由。

## 2. 功能范围

### 2.1 查看态

- 卡片主体显示视频封面。
- 若 `cover_image` 未填写，则直接回退显示视频第一帧。
- 鼠标悬浮时，视频区域显示黑色半透明遮罩和播放图标。
- 当填写了 `video_title` 时，在视频区域下方显示标题。
- 当填写了 `creator` 和/或 `publish_time` 时，在标题下方用更小字体显示。
- 点击视频区域时，不在基础卡片内直接启动播放器，而是通过宿主提供的 `openResource(...)` 上抛打开意图。

### 2.2 编辑态

- 上传视频文件并导入到卡片根目录。
- 视频与封面都支持点击或拖拽上传本地文件。
- 视频与封面都支持通过 URL 导入，导入后仍写入卡片根目录正式资源。
- 上传视频后自动提取第一帧，生成默认封面资源。
- 上传成功后，上传区替换为视频/封面预览项；鼠标悬浮时显示删除按钮。
- 手动上传、替换和移除封面。
- 编辑页采用标题分组 + 列表行布局，不使用多层卡片嵌套。
- 窄窗口与移动端比例窗口下，资源行与字段行必须自动折叠为单列，保证完整显示。
- 视频与封面操作按钮采用纵向排布，避免编辑面板拥挤。
- 编辑以下可选文本字段：
  - `video_title`
  - `creator`
  - `publish_time`
- 文本字段在编辑期间只更新本地草稿；当用户离开当前输入区后，再统一向宿主提交刷新。

## 3. 数据结构

正式配置字段：

- `card_type: "VideoCard"`
- `theme?: string`
- `video_file: string`
- `cover_image: string`
- `video_title: string`
- `publish_time: string`
- `creator: string`

约束：

- `video_file` 必填，且必须是相对于卡片根目录的资源路径。
- `cover_image` 可为空；为空时查看态回退到视频第一帧。
- 不得把 `blob:`、`data:`、绝对路径或宿主临时 URL 写回正式 YAML。

## 4. 非功能要求

- 技术栈：React
- 样式：统一使用 `--chips-*` token，不私自引入新的主题体系。
- 国际化：至少提供 `zh-CN` / `en-US`
- 资源链路：视频与封面统一通过宿主 `importResource(...)`、`resolveResourceUrl(...)`、`deleteResource(...)`
- 编辑稳定性：文本编辑不得在每次按键时触发复合卡片刷新，避免闪烁。
- 可测试性：
  - 单元测试覆盖 schema、查看态点击打开、文本失焦提交与编辑态视频导入链路
  - 集成测试覆盖“编辑修改 -> 失焦提交 -> 查看态更新”主流程
