# 视频基础卡片插件架构设计

版本：0.1.0  
状态：正式版

## 1. 设计目标

本插件采用“纯插件导出 + 宿主资源桥 + React UI + 浏览器首帧抽帧”架构，在不引入额外第三方库的前提下，为视频基础卡片提供正式的查看、编辑和资源生命周期能力。

核心目标：

- 与图片、音乐基础卡片保持一致的卡片根目录资源模型
- 在查看态通过统一资源打开契约唤起视频播放器
- 在编辑态完成视频导入、默认封面生成与补充信息编辑
- 让配置 schema、查看态、编辑态、模板和测试共用同一套字段口径

## 2. 模块划分

### 2.1 `src/schema/card-config.ts`

负责：

- 视频卡片正式配置类型定义
- 默认配置
- 配置归一函数
- 配置校验函数

### 2.2 `src/shared/utils.ts`

负责：

- 卡片根目录相对路径归一
- 视频 MIME 推导
- 内部资源路径收集
- 文件名与展示标题辅助逻辑

### 2.3 `src/shared/video-cover.ts`

负责：

- 基于浏览器 `video + canvas` 提取视频首帧
- 生成可导入宿主资源桥的默认封面 `File`

### 2.4 `src/render/view.tsx`

负责：

- 查看态 React 组件
- 视频与封面资源 URL 解析
- 封面显示、首帧回退、悬浮遮罩与播放图标
- 点击视频时通过 `openResource(...)` 把打开意图转交给外层宿主

### 2.5 `src/editor/panel.tsx`

负责：

- 标题分组 + 列表行式编辑布局与响应式排版
- 视频导入、URL 下载导入与默认封面生成
- 封面上传、替换、移除
- 视频标题、创作者、发布时间编辑
- 文本输入本地草稿与失焦后统一提交
- 错误提示与资源状态反馈

## 3. 宿主契约

### 3.1 查看态

- `renderBasecardView({ container, config, themeCssText, resolveResourceUrl, releaseResourceUrl, openResource })`
- `video_file` 与 `cover_image` 优先通过宿主提供的 `resolveResourceUrl(...)` 解析。
- 当 `cover_image` 为空时，查看态直接使用 `<video>` 的首帧作为回退封面。
- 当宿主提供 `openResource(...)` 时，点击视频卡片会把视频资源意图上抛给外层统一走 `resource.open`。

### 3.2 编辑态

- `renderBasecardEditor({ container, initialConfig, onChange, resolveResourceUrl, releaseResourceUrl, importResource, deleteResource })`
- 编辑页不内嵌卡片预览，也不使用多层卡片壳布局，避免窄侧栏里产生拥挤层级。
- 文本字段先写入编辑器本地草稿；当用户离开元信息输入区后，再统一调用 `onChange(...)` 输出正式配置，减少复合卡片刷新闪烁。
- 资源区提供“点击/拖拽上传”与“URL 输入导入”两条正式入口；URL 导入会先下载成 `File`，再通过 `importResource(...)` 写入卡片根目录。
- 视频上传链路：
  1. 选择本地视频文件
  2. 调用 `importResource(...)` 写入卡片根目录
  3. 调用 `extractVideoCoverFile(...)` 生成默认封面
  4. 若默认封面生成成功，再次通过 `importResource(...)` 写入封面资源
  5. 通过 `onChange(...)` 输出新的正式配置
- 替换视频或封面时，旧资源通过 `deleteResource(...)` 登记删除。

## 4. 关键设计决策

- `video_file` 是唯一必填正式资源字段，其他字段全部允许为空。
- 查看态标题和次信息只在用户填写时显示，不自动占位。
- 默认封面生成失败时不阻断视频导入；查看态会继续回退到视频第一帧。
- 资源打开不在基础卡片内部直接启动视频播放器，而是统一交给宿主资源打开路由。
- 编辑页采用居中收窄的设置面板式结构，保证在编辑器侧栏中仍有稳定留白和清晰层级。
- 编辑页内部只保留分组标题与列表行，不再叠加额外信息说明块和嵌套卡片容器。
- 资源操作保持立即提交；文本元信息改为失焦提交，以平衡输入稳定性和宿主刷新成本。
- 当资源已经存在时，上传区切换为资源预览项；视频显示视频画面，封面显示图片，删除动作通过悬浮显露的按钮触发。
