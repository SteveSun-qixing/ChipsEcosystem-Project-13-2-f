# 网页基础卡片需求规格说明书

## 目标

交付一个新的基础卡片插件“网页基础卡片”，用于在复合卡片中显示网页。

## 正式需求

- 支持远程网址作为网页来源。
- 支持网页 ZIP 压缩包作为网页来源，入口文件必须为 `index.html`。
- 编辑器需要通过正式宿主能力解压 ZIP，并将整个网页目录写入复合卡片根目录。
- 显示区域支持两种模式：
  - 固定比例：固定为高:宽 `16:7`；
  - 自由展开：按网页高度展开。
- 网页基础卡片外层显示区域不得额外绘制描边或圆角。
- 查看态必须把当前显示区域正式传递给嵌入页，供网页按卡片视口做自适应。
- 固定比例模式下，网页在固定窗口内滚动查看。
- 自由展开模式下，高度最高不能超过当前宽度的 `20` 倍；超过后在窗口内滚动查看。
- 当网页禁止被 iframe 嵌入时，查看态需要显示明确错误提示。
- 在编辑引擎编辑态中，网页基础卡片必须屏蔽网页内部交互，点击网页区域用于选中该基础卡片并打开编辑面板。
- 编辑器面板采用“比例切换 -> 网页模式切换 -> 条件输入”的正式结构：
  - 比例区使用切换式选项按钮；
  - 网页模式支持“链接 / 上传压缩包”切换；
  - 选择链接后显示网址输入框，并在输入合法网址后自动刷新；
  - 选择文件后显示支持拖拽和点击的 ZIP 上传区域；
  - 切换到新来源时不要求用户先手动清空旧来源。
