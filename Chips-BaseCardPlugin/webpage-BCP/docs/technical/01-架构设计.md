# 网页基础卡片架构设计

## 正式链路

- 查看态：`renderBasecardView -> 嵌套 iframe -> 网页内容`
- 编辑态：`renderBasecardEditor -> importArchiveBundle/deleteResource -> onChange`

## 关键实现

- 远程网址直接使用嵌套 iframe 加载。
- 网页 ZIP 包通过正式 `importArchiveBundle` 宿主能力导入到卡片根目录。
- 官方编辑引擎在导入网页 ZIP 时，若当前运行环境无法直接暴露所选 `File` 的真实磁盘路径，会先通过正式文件服务把该文件暂存到卡片会话导入目录，再继续复用 `zip.list/zip.extract` 宿主链路。
- ZIP 导入成功后，配置中只保留：
  - `bundle_root`
  - `entry_file`
  - `resource_paths`
- 查看态读取 `index.html` 文本并注入 `<base href>`，使网页包内部的相对资源继续按卡片根目录解析。
- 查看态壳层保持无描边、无圆角，由网页内容自己决定视觉边缘。
- 查看态显示模式分为两类：
  - `fixed`：固定为宽:高 `7:16` 视口，网页在该视口内滚动；
  - `free`：按网页内容高度展开，但最高不超过当前宽度的 `20` 倍。
- 查看态通过正式网页视口协议把稳定视口信息传递给嵌入页：
  - 父层向网页发送 `chips:webpage-card:viewport`；
  - 网页包和同源网页可读取 `window.__CHIPS_WEBPAGE_CARD_VIEWPORT__`、CSS 变量 `--chips-webpage-card-*` 与 `chips:webpage-card:viewport-change` 事件；
  - 嵌入页可通过 `chips:webpage-card:content-height` 向父层回传内容高度。
- 协议中的 `height` 表示传递给网页内部的稳定测量视口高度：
  - `fixed`：等于固定比例 `7:16` 对应的正式视口高度；
  - `free`：优先对齐宿主当前真实窗口高度，并限制不超过固定比例高度；卡片外层最终展开高度只用于宿主布局，不能再次回写给网页内部，否则会形成高度测量反馈，导致网页把整张长卡误判成窗口高度。
- 协议中的 `baseHeight` 表示宿主在内容暂不可测或需要回退时采用的最小外层高度。
- 对网页 ZIP 包，查看态会额外注入稳定虚拟视口层：把常见 `vh/dvh/svh/lvh` 高度单位和 `window.innerHeight` 这类原生读取尽量映射到协议视口高度，降低长网页在 `free` 模式下因 iframe 被拉长而误判窗口高度的风险。
- 同源网页与实现了高度回传协议的网页会按当前显示模式计算卡片高度；跨源且未回传高度的场景回退到当前模式的初始视口。
- 编辑器采用“比例切换 + 模式切换 + 条件输入”结构，网址自动刷新，ZIP 支持拖拽和点击上传。
- 编辑引擎预览态通过描述符声明的透明选择层屏蔽网页内部点击，保证点击网页区域时优先进入基础卡片编辑链路。
