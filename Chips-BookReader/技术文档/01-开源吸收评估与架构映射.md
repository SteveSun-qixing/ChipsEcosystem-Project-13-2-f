## 2. 仓库结构评估

- Thorium 当前源码本体是 Electron 桌面应用，不是可直接嵌入的纯前端阅读内核。
- 阅读器能力散落在 `src/main`、`src/renderer`、`src/common` 与 Readium 相关子树中，依赖 Electron 主进程、IPC、Redux 状态同步和桌面应用生命周期。
- 真正可借鉴的是阅读域模型、章节定位、目录组织、阅读器交互分工；不应整体吸收其 Electron 应用壳。

## 6. 吸收路径选择

本项目采用组合策略：

- 路径 A：参考方案，自主重写
  - 适用于应用壳、启动链路、React UI、主题、多语言、manifest、Host 适配
- 路径 B：抽取能力，内部封装
  - 适用于 EPUB 解包、OPF/nav 理解、目录与章节组织、ReadiumCSS 注入、分页测量和阅读定位内核

未选择路径 C 的原因：

- Thorium 的 Electron / Redux / Readium 链路与薯片生态 Host 承载模型不一致
- 直接深度二开会把上游桌面应用语义泄漏到生态公共接口
- 长期维护成本高，且容易被上游仓库结构绑架

## 7. 薯片生态内架构映射

### 目标仓库职责

- `src/domain/epub/`
  - EPUB ZIP 读取
  - OPF / nav / ncx 解析
  - Publication / Link 内部适配层
  - HTML 章节变换器
  - 章节内容文档渲染
- `src/domain/readium/assets/`
  - 保留 ReadiumCSS 与相关字体资源，作为阅读引擎的正式样式资源来源
- `src/domain/readium/归档/`
  - 存放被新引擎替代的旧单体 `runtime.ts / layout.ts`
- `src/engine/`
  - 当前正式阅读运行时所在目录
  - 负责分页、滚动、布局、ReadiumCSS、位置追踪、动画、搜索与文档控制器
  - 继续沿用 Readium / Thorium 的排版与分页思路，但收口为薯片生态内部的纯 TypeScript 契约
- `src/interaction/`
  - 负责键盘、点击、滚轮、触摸等输入意图映射
- `src/hooks/`
  - 负责把 React 生命周期与阅读引擎、交互系统、进度和书签持久化对齐
- `src/components/`
  - 阅读器 UI 组件拆分后的正式组合层
  - 章节 iframe 承载、控制层、面板、进度条、反馈提示
- `src/styles/`
  - 阅读器壳层、控制层、面板、进度与动画样式模块
- `src/utils/`
  - 启动上下文恢复
  - 二进制归一
  - 书籍源目标归一
- `manifest.yaml`
  - app 插件正式声明
  - `resource-handler:view:application/epub+zip`
  - `file-handler:.epub/.epub3`

### 与参考应用对齐点

- 启动上下文恢复参考图片查看器 / 音乐播放器 / 视频播放器正式模式
- 运行时承载、主题、多语言和 Host 调用遵循薯片 app 插件正式模型

## 8. 当前结论

- `Chips-BookReader` 不是薄壳应用，可以有完整内部模块。
- 但这些内部模块必须是薯片生态自己的阅读域模块，而不是把 Thorium 桌面应用链路整体迁入。
- 当前正式落地采用了“React 应用壳 + 内部阅读引擎层”的多层结构：
  - 以内部 `Publication/Link` 模型收口 OPF / spine / toc / resources
  - 以 HTML Transformer 收口章节资源改写、样式内联和内部链接归一
  - 以 `src/engine/document-controller.ts` 组合 Thorium / Readium 吸收来的版式注入、分页测量、定位和交互逻辑
- 当前新引擎已经正式替代旧单体实现，旧 `runtime.ts / layout.ts` 已归档。
- 阅读引擎当前的正式子模块包括：
  - `pagination-engine`：spread 级绝对定位与无漂移翻页
  - `scroll-engine`：滚动主轴度量与边界检测
  - `readium-css-manager`：ReadiumCSS profile、样式注入与 CSS 变量治理
  - `layout-calculator`：断点化布局度量
  - `position-tracker`：锚点恢复与阅读进度
  - `animation-controller`：平滑翻页动画
  - `search-engine`：跨章节增量搜索与 iframe 高亮
  - `document-controller`：正式组合协调器
- UI 侧已经同步完成组件化拆分：
  - `ReaderShell` 负责状态编排
  - `ReaderChrome / ProgressBar / *Panel` 负责具体交互界面
  - `interaction/` 目录负责输入意图层
- 对外仍只暴露薯片生态阅读器应用自己的状态、UI 和 Host 契约，不暴露 Thorium 内部事件名或 Electron 语义。
