# 书籍阅读器实现说明

## 1. 当前工程结构

本轮重构后，阅读器主链已经从旧的单体 `runtime.ts + ReaderShell.tsx` 切换为「应用壳层 + 引擎层 + 交互层 + Hooks 层 + 组件层 + 样式层」分层结构。

### 1.1 根组件与应用壳层

- `src/App.tsx`
  - 负责启动参数恢复、文件/链接打开、主题和语言接入、章节渲染调度。
  - 负责读取本地文件、Host 资源路由资源和远程 EPUB URL。
  - 负责把 `EpubBook`、当前章节、阅读偏好、主题色板传递给阅读器壳层。

### 1.2 EPUB 解析层

- `src/domain/epub/archive.ts`
  - 负责无第三方依赖的 ZIP 条目读取，支持 `store / deflate`。
- `src/domain/epub/package.ts`
  - 负责 `container.xml`、OPF、NAV、NCX 解析与阅读顺序构建。
- `src/domain/epub/publication.ts`
  - 负责内部 `Publication / Link` 适配层。
- `src/domain/epub/markup.ts`
  - 负责章节 HTML 变换、资源内联、内部链接改写与最终 `srcdoc` 生成。

### 1.3 阅读引擎层

- `src/engine/types.ts`
  - 收口分页、滚动、布局、位置、动画、进度的正式类型契约。
- `src/engine/pagination-engine.ts`
  - 负责分页模式下的度量、绝对偏移定位、spread 对齐与边界判断。
- `src/engine/scroll-engine.ts`
  - 负责滚动模式下的滚动度量、边界检测和 fragment 定位。
- `src/engine/readium-css-manager.ts`
  - 负责 ReadiumCSS profile 识别、资源加载、补丁样式和 CSS 变量注入。
- `src/engine/layout-calculator.ts`
  - 负责窗口断点、单双页 spread、页边距、列间距、有效内容宽度计算。
- `src/engine/position-tracker.ts`
  - 负责分页/滚动锚点捕获与恢复，以及章节内/全书进度计算。
- `src/engine/animation-controller.ts`
  - 负责分页模式下的平滑滚动动画。
- `src/engine/search-engine.ts`
  - 负责跨章节增量搜索、结果摘录构建、iframe 内高亮与清除。
- `src/engine/document-controller.ts`
  - 作为组合协调器，把 CSS 管理、布局、分页、滚动、位置和动画拼成正式阅读运行时。

## 2. 打开与启动链路

### 2.1 本地文件

- 通过 `client.platform.openFile(...)` 选择本地 EPUB。
- 通过窗口拖拽文件打开 EPUB。
- 通过启动参数中的本地路径恢复阅读目标。

### 2.2 Host 资源路由

- 读取 `client.platform.getLaunchContext()`。
- 通过 `resolveLaunchBookTarget(...)` 归一化 `resourceOpen` / `targetPath` 等启动信息。
- 若存在 `filePath` 则走 `client.file.read(..., { encoding: "binary" })`。
- 否则回退到 `client.resource.readBinary(...)`。

### 2.3 远程链接

- 支持直接输入远程 EPUB URL。
- 通过标准 `fetch` 下载远程资源，再交给 EPUB 解析层。

### 2.4 统一加载结果

- 所有入口都会归一为 `LaunchBookTarget`。
- `App.tsx` 调用 `loadEpubBook(...)` 生成 `EpubBook`。
- 当前章节通过 `renderSectionDocument(...)` 生成 `RenderedSectionDocument`。
- `ReaderShell.tsx` 使用 `useReaderEngine(...)` 把 `RenderedSectionDocument` 装载到 iframe。

## 3. 渲染引擎主链

### 3.1 DocumentController 的职责

`DocumentController` 是当前正式阅读运行时的协调中心，负责：

- 在章节 iframe 中注入 ReadiumCSS 和补丁样式。
- 读取当前阅读偏好与主题色板并同步到文档根节点。
- 按阅读模式分派给 `PaginationEngine` 或 `ScrollEngine`。
- 在窗口变化、布局变化、章节切换后恢复阅读位置。
- 对外发出：
  - `page-changed`
  - `scroll-changed`
  - `layout-changed`
  - `boundary-reached`
  - `progress-updated`

### 3.2 分页模式

分页模式不再使用旧实现中的“相对偏移累加”策略，而是改为基于 spread 索引的绝对定位：

- 旧问题：`currentOffset ± viewportWidth` 连续取整后会累积误差，导致越翻越歪。
- 当前策略：`targetOffset = spreadIndex × spreadAdvancePx`。
  - 单页模式下：`spreadAdvancePx = spreadWidthPx`。
  - 双页 spread 下：`spreadAdvancePx = spreadWidthPx + columnGapPx`。
  - 如果双页场景仍按 `spreadWidthPx` 前进，就会每次少走一个 `columnGapPx`，翻页次数越多错位越明显。
- 结果：
  - 连续翻页不会累积漂移。
  - 章节末尾不再出现“大半页被截掉”。
  - `snapToSpread(...)` 与 `computeOffsetForSpread(...)` 都围绕绝对 spread 索引工作。

### 3.3 滚动模式

滚动模式由 `ScrollEngine` 独立负责：

- 基于主轴滚动偏移量计算 `scrollFraction`。
- 在章节头尾维持边界检测。
- 在 fragment 跳转时根据元素几何位置换算目标偏移。

### 3.4 布局与样式

`LayoutCalculator` 与 `ReadiumCssManager` 共同决定正文排版：

- `LayoutCalculator`
  - 根据窗口宽度、高度、字体缩放、版心宽度、章节类型决定单双页 spread。
  - 输出 `pageWidthPx / pageGutterPx / columnGapPx / effectiveContentWidth / windowBreakpoint`。
- `ReadiumCssManager`
  - 根据文档语言和方向性选择 `default / rtl / cjk-horizontal / cjk-vertical` profile。
  - 把字体、字号、背景色、文本色、链接色、列数等偏好注入 ReadiumCSS 变量。

### 3.5 位置恢复与进度

`PositionTracker` 负责两类能力：

- 阅读锚点
  - 分页模式记录 `spreadIndex / spreadFraction / scrollFraction`
  - 滚动模式记录 `scrollFraction`
  - 窗口变化后按锚点映射到新视口
- 阅读进度
  - 章节内进度 `sectionFraction`
  - 全书进度 `bookFraction / bookPercentage`
  - 分页模式下的 `currentPage / totalPages`

## 4. React 与运行时桥接

### 4.1 useReaderEngine

`useReaderEngine(...)` 的职责是把 React 生命周期和引擎生命周期对齐：

- iframe 加载完成后创建 `DocumentController`。
- 章节切换时销毁旧控制器、挂载新控制器。
- 阅读偏好/主题变化时仅调用 `controller.update(...)`，不重建整章 DOM。
- 初始定位优先级：
  1. `currentFragment`
  2. `pendingBoundary`
  3. 章节起始位置

### 4.2 useReaderInteraction

`useReaderInteraction(...)` 负责：

- 把 `InteractionManager` 挂到宿主窗口。
- 把点击、滚轮、触摸监听挂到章节 iframe。
- 在交互事件中回调：
  - 阅读导航
  - 边界跳转
  - 控制层显隐
  - 面板关闭
  - EPUB 内部链接跳章
  - 外部链接打开
  - 阅读偏好微调

### 4.3 useReaderProgress

- 监听 `DocumentController` 的 `progress-updated`。
- 节流后输出给 `ProgressBar` 和 `ReaderChrome`。
- 避免在滚动/翻页阶段产生过密的 React 重渲染。

### 4.4 useBookmarks

- 按 `book.source.sourceId` 对书签分组隔离。
- 使用 `client.config.get / set` 读写书签数组。
- 支持当前位置匹配：
  - 分页模式优先用 `spreadIndex`
  - 跨阅读模式时回退为 `scrollFraction` 容差匹配

## 5. 交互系统设计

### 5.1 键盘

- `ArrowLeft / ArrowUp / PageUp`：上一页
- `ArrowRight / ArrowDown / PageDown`：下一页
- `Space / Shift+Space`：下一页 / 上一页
- `Home / End`：跳到章节首尾
- `[` / `]`：缩小/放大字号
- `- / +`：缩窄/放宽版心
- `Escape`：关闭当前面板

输入框、文本域、选择框和 `contenteditable` 区域不会触发阅读快捷键。

### 5.2 点击

- 阅读区域左 26%：上一页
- 阅读区域右 26%：下一页
- 中间区域：切换控制层显隐
- 若当前有打开面板，中间区域点击会优先关闭面板
- iframe 内部改写后的 EPUB 链接会回路由到章节切换
- 外部链接统一走 `platform.openExternal(...)`

### 5.3 滚轮 / 触摸板

- 分页模式：
  - 离散鼠标滚轮按单次输入直接翻一页。
  - 即使鼠标在 Chromium / Electron 下以 `DOM_DELTA_PIXEL` 上报，只要表现出离散刻度特征，也走“一刻度一页”的独立分支。
  - 连续触摸板像素流先累计到分页阈值，再把整次手势消费为一次翻页。
  - 一次触摸板惯性滚动在 idle 窗口结束前不会重复触发多页跳转。
  - 即使在导航锁窗口内，也会继续 `preventDefault()`，避免浏览器把分页文档偷偷滚偏。
- 滚动模式：
  - 正常章节中维持自然滚动。
  - 只有接近章节边界时才把继续滚动解释为越章。

### 5.4 触摸

- 分页模式：横向滑动翻页。
- 滚动模式：在章节边界处通过纵向滑动越章。

### 5.5 导航锁

`InteractionManager` 内部维护导航锁窗口，避免：

- 同一次鼠标/触摸/键盘输入触发重复跳转
- 某些键盘长按导致重复翻页过快
- 触摸手势在单次滑动中重复翻页

## 7. 进度、书签与搜索

### 7.1 阅读进度

- 进度数据来自 `DocumentController -> PositionTracker -> useReaderProgress`。
- `ProgressBar` 以章节内位置为拖动基准。
- `ReaderChrome` 同步显示章节位置、页码或全书百分比。

### 7.2 书签系统

书签结构：

- `sectionIndex`
- `sectionTitle`
- `spreadIndex`
- `scrollFraction`
- `readingMode`
- `createdAt`

正式行为：

- 点击当前书签按钮可在当前位置添加/删除书签。
- 书签通过 `client.config` 持久化。
- 切换书籍时按 `bookSourceId` 隔离。
- 打开书签时，若当前阅读模式与创建模式不同，会自动使用 `scrollFraction` 回退定位。

### 7.3 搜索系统

`SearchEngine` 当前提供：

- 跨章节纯文本增量搜索
- 每章搜索完成后让出主线程
- 结果摘录构建
- 在当前章节 iframe 中插入 `<mark>` 高亮
- 当前选中结果的聚焦和滚动对齐

搜索 UI 行为：

- 输入即触发增量搜索
- 结果按章节分组显示
- 选中结果后跳转到对应章节，并在 iframe 内保持高亮

当前正式实现已经完成以下重构目标：

- 用分层引擎替代旧单体阅读运行时主链。
- 用多组件组合替代旧的超大 `ReaderShell.tsx`。
- 完成进度条、搜索、书签、阅读偏好、面板系统和动画样式重构。

当前仓库内仍保留 EPUB 解析层和 ReadiumCSS 资源层作为正式依赖，但主阅读运行时已经完全切换到新的 `engine / interaction / hooks / components / styles` 架构。
