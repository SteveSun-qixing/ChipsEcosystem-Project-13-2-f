# 电子书阅读器界面渲染与交互重构 — 总纲

## 1. 重构背景

当前电子书阅读器存在以下核心问题：

### 1.1 渲染引擎缺陷

- **分页累积漂移**：CSS multi-column 分页模式下，`scrollWidth` 不总是 `offsetWidth` 的整数倍。`snapToUnitByDirection` 使用 ceil/floor 取整后，每次翻页都引入 ±1px 的对齐误差。连续翻页 20–30 次后，累积误差可达半页以上，表现为正文内容逐渐被截断。
- **页面截断**：当章节总内容宽度不是视口宽度的精确整数倍时，最后一页或最后几页的内容会被不完整地裁切掉。
- **翻页效果生硬**：分页模式下通过 `scrollTo({ behavior: 'smooth' })` 做翻页动画，受浏览器滚动动画的简陋实现限制，视觉效果不可控，且在部分平台上 `smooth` 不生效或卡顿。
- **窗口缩放后跳页**：`captureProgressionRatio` / `restoreProgressionRatio` 使用全局进度比值做锚点恢复，当新旧视口的列数发生变化时，恢复位置可能跳到错误页面。

### 1.2 架构缺陷

- **ReaderShell.tsx 巨型组件**：1156 行代码，混合了 UI 渲染、事件处理、引擎控制、面板状态、拖拽处理等全部逻辑，违背单一职责原则。
- **交互逻辑与渲染引擎深度耦合**：滚轮 / 键盘 / 触摸等交互代码直接写在 ReaderShell 的 useEffect 回调中，无法单独测试、复用或替换。
- **引擎层职责混乱**：`runtime.ts`（1301 行）同时负责 ReadiumCSS 加载、CSS 变量注入、分页测量、滚动控制、布局计算、窗口缩放处理，缺乏内聚边界。

### 1.3 功能缺失

- **无阅读进度显示**：没有进度条、页码指示器、百分比等阅读进度可视化。
- **无书签功能**：无法标记和回溯阅读位置。
- **无搜索功能**：无法在电子书内容中搜索文本。
- **无翻页动画**：分页模式缺少精美的翻页过渡效果。
- **UI 过渡不够精致**：面板打开/关闭、控制层显隐缺少流畅的动画过渡。

## 2. 重构目标

将电子书查看器的使用体验对标主流优秀电子书阅读器（Apple Books、Kobo、Google Play Books）：

1. **渲染精确**：彻底消除分页累积漂移和页面截断问题。
2. **交互流畅**：翻页、滚动、缩放、模式切换等操作体验丝滑。
3. **架构清晰**：引擎层、交互层、UI 层三层分离，各自内聚，独立可测。
4. **功能完整**：具备阅读进度、书签、搜索等现代阅读器标准功能。
5. **视觉精美**：控件、面板、翻页动画等 UI 细节达到高品质标准。

## 3. 技术约束

- 不引入新的第三方库（foliate-js 已评估，决定不引入）。
- 继续使用 ReadiumCSS 作为排版基础，在其之上深度修复和增强渲染引擎。
- 保留现有 EPUB 解析层（`src/domain/epub/`）不变。
- React 技术栈，所有文案走 i18n，所有样式走主题 token。
- 通过 `chips-sdk` / `window.chips.*` 访问系统能力。
