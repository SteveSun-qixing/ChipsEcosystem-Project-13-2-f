# 阶段 02 — 分页引擎重构

## 目标

实现全新的分页引擎，彻底修复连续翻页时的累积漂移问题和页面截断问题。

## 背景

当前分页逻辑存在于 `runtime.ts` 的 `ReadiumDocumentController` 中，核心缺陷如下：

### 累积漂移根因

1. CSS multi-column 布局中 `scrollWidth` 不一定是 `offsetWidth` 的精确整数倍。例如视口宽度 1440px，scrollWidth 可能为 95611px（95611 / 1440 = 66.396…），最后一个 spread 只有约 0.4 个 spread 宽度的有效内容。
2. `snapToUnitByDirection` 通过 `Math.ceil` / `Math.floor` 取整，每次翻页引入的对齐误差是 `value % unit` 的余量。
3. `move()` 方法先计算 `current + unit`，再对结果做 snap 取整。如果当前位置已经因前一次 snap 偏离了"精确整数倍位置"，那么 `current + unit` 的结果也会偏离，snap 之后的误差与原偏差叠加。
4. 经过 N 次翻页后，累积误差 = N × 单次平均误差 ≈ N × 0.5px。翻页 30 次后误差约 15px，翻页 100 次后约 50px，已经可以看到明显的内容截断。

### 修复策略

**改用基于 spread 索引的绝对定位，而非相对偏移累加。**

关键思路：
- 始终将目标滚动位置计算为 `spreadIndex × spreadWidth`，而非 `currentOffset ± viewportWidth`。
- 翻页操作只是 `spreadIndex ± 1`，然后从索引算出绝对像素位置。
- `spreadWidth` 就是 `scrollingElement.offsetWidth`，即 iframe 可视区域宽度（CSS column 分页的单位宽度就是容器的 offsetWidth）。
- 这样每次翻页的目标位置都是从 0 开始的整数倍，不存在舍入误差累积。

### 关键实现细节

#### 度量函数 `measure()`

```typescript
function measure(document: Document, dir: DocumentDirectionality): PaginationMetrics {
  const scrollEl = resolveScrollingElement(document);
  const offsetWidth = scrollEl.offsetWidth;    // spread 宽度 = 容器可视宽度
  const scrollWidth = scrollEl.scrollWidth;    // 内容总宽度
  
  // 计算实际列数
  const bodyStyle = document.defaultView!.getComputedStyle(document.body!);
  const colCount = parseInt(bodyStyle.columnCount, 10) || 1;
  const colGap = parseFloat(bodyStyle.columnGap) || 0;
  
  // 单列宽度 = (offsetWidth - (colCount - 1) * colGap) / colCount
  const colWidth = (offsetWidth - (colCount - 1) * colGap) / colCount;
  
  // spread 总数 = ceil(scrollWidth / offsetWidth)
  // 注意：最后一个 spread 可能不满
  const spreadCount = Math.max(1, Math.ceil(scrollWidth / Math.max(1, offsetWidth)));
  
  // 最大可滚动偏移
  const maxScrollOffset = Math.max(0, scrollWidth - offsetWidth);
  
  return {
    totalColumns: colCount * spreadCount,   // 非精确，CSS column 自动分配
    columnsPerSpread: colCount,
    spreadCount,
    currentSpreadIndex: 0,                  // 调用方负责填充
    spreadWidthPx: offsetWidth,
    columnWidthPx: colWidth,
    columnGapPx: colGap,
    totalScrollableWidth: scrollWidth,
    maxScrollOffset,
  };
}
```

#### 翻页函数（基于索引）

```typescript
function computeTargetOffset(
  currentIndex: number,
  direction: PageDirection,
  metrics: PaginationMetrics,
): { targetIndex: number; targetOffset: number } {
  const targetIndex = direction === "next"
    ? Math.min(currentIndex + 1, metrics.spreadCount - 1)
    : Math.max(currentIndex - 1, 0);
  
  // 绝对定位：targetOffset = targetIndex × spreadWidth
  // 但要 clamp 到 maxScrollOffset，避免超出
  const targetOffset = Math.min(
    targetIndex * metrics.spreadWidthPx,
    metrics.maxScrollOffset,
  );
  
  return { targetIndex, targetOffset };
}
```

#### 从当前滚动位置反推 spread 索引

```typescript
function resolveSpreadIndex(scrollOffset: number, spreadWidth: number): number {
  if (spreadWidth <= 0) return 0;
  // 使用四舍五入，因为 scrollOffset 可能因子像素渲染有微小偏移
  return Math.round(scrollOffset / spreadWidth);
}
```

### 与旧实现的关键区别

| 维度 | 旧实现 | 新实现 |
|------|--------|--------|
| 翻页计算 | `current + viewportUnit`，然后 snap | `spreadIndex ± 1`，然后 `index × spreadWidth` |
| 对齐方式 | ceil/floor 取整（方向敏感） | `Math.round`（当前位置） + 精确乘法（目标位置） |
| 误差累积 | 每次 ±0.5px 累积 | 0（每次都从索引计算绝对位置） |
| 边界处理 | 复杂的 adjusted/unadjusted maxShift | 简单的 `maxScrollOffset = scrollWidth - offsetWidth` |
| 双页 spread | 独立的 `isTwoPageSpread` 函数 | 统一在 metrics 中表达 |

## 验收标准

1. 连续翻页 100 次后，滚动位置与理论值（`spreadIndex × spreadWidth`）的误差不超过 1px。
2. 最后一页的内容不被截断。
3. 章节 scrollWidth 不是 offsetWidth 整数倍时，正常处理。
4. 双页 spread 模式下翻页正确。
5. RTL / 竖排模式下翻页方向正确。
6. 所有纯函数都有独立单元测试。
7. 现有 `readium-layout.test.ts` 中的 `snapToUnitByDirection` 测试仍通过（兼容导出）。
