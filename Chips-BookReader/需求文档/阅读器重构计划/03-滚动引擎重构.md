# 阶段 03 — 滚动引擎重构

## 目标

实现独立的滚动模式引擎，处理纵向自然滚动、章节边界越章、滚动进度计算。

## 背景

当前滚动模式逻辑散落在 `ReadiumDocumentController` 和 `ReaderShell` 中，滚动和分页共用同一套 `move()` / `isNearBoundary()` 方法，通过 `if (mode === "paginated")` 分支处理差异。这导致两种模式的逻辑相互纠缠，难以独立优化。

### 当前问题

1. 滚动模式下的越章阈值通过 `resolveWheelNavigationThreshold` 全局控制，与分页模式共享数据结构。
2. `isNearBoundary` 只检查滚动偏移量是否接近首尾，不够精确——没有考虑用户的滚动速度和方向。
3. 缺少平滑的越章过渡：到达章末后直接跳到下一章，没有视觉上的连续感。

## 具体实现

### 核心类 `ScrollEngine`

```typescript
export class ScrollEngine {
  // ─── 度量 ───
  measure(document: Document, directionality: DocumentDirectionality): ScrollMetrics;
  
  // ─── 滚动进度 ───
  computeScrollFraction(scrollOffset: number, maxOffset: number): number;
  
  // ─── 边界检测（更精确） ───
  isNearBoundary(metrics: ScrollMetrics, direction: PageDirection, thresholdPx?: number): boolean;
  
  // ─── 是否到达边界 ───
  isAtBoundary(metrics: ScrollMetrics, boundary: ReadingBoundary): boolean;
  
  // ─── 滚动到指定位置 ───
  computeScrollTarget(
    current: number,
    delta: number,
    maxOffset: number,
  ): { target: number; reachedBoundary: ReadingBoundary | null };
  
  // ─── 滚动到边界 ───
  computeBoundaryOffset(boundary: ReadingBoundary, maxOffset: number): number;
  
  // ─── 从 fragment 计算滚动位置 ───
  resolveFragmentOffset(element: Element, directionality: DocumentDirectionality): number;
}
```

### 关键实现细节

#### 边界检测

```typescript
// 边界阈值：滚动模式下使用视口高度的 4% 作为阈值
const SCROLL_BOUNDARY_THRESHOLD_RATIO = 0.04;
const MIN_BOUNDARY_THRESHOLD_PX = 24;

function isNearBoundary(
  metrics: ScrollMetrics,
  direction: PageDirection,
  thresholdPx?: number,
): boolean {
  const threshold = thresholdPx ?? Math.max(
    MIN_BOUNDARY_THRESHOLD_PX,
    Math.round(metrics.viewportSize * SCROLL_BOUNDARY_THRESHOLD_RATIO),
  );
  
  if (direction === "previous") {
    return metrics.scrollOffset <= threshold;
  }
  
  return metrics.scrollOffset >= Math.max(0, metrics.maxScrollOffset - threshold);
}
```

#### 滚动目标计算

```typescript
function computeScrollTarget(
  current: number,
  delta: number,
  maxOffset: number,
): { target: number; reachedBoundary: ReadingBoundary | null } {
  const target = Math.max(0, Math.min(current + delta, maxOffset));
  
  let reachedBoundary: ReadingBoundary | null = null;
  if (target <= 0 && delta < 0) {
    reachedBoundary = "start";
  } else if (target >= maxOffset && delta > 0) {
    reachedBoundary = "end";
  }
  
  return { target, reachedBoundary };
}
```

### 滚动模式与分页模式的职责边界

| 职责 | ScrollEngine | PaginationEngine |
|------|-------------|-----------------|
| 滚动位置计算 | 纵向连续滚动 | 横向离散翻页 |
| 边界检测 | scrollTop 接近首尾 | spreadIndex 到达首尾 |
| 进度计算 | scrollTop / maxScroll | spreadIndex / spreadCount |
| 越章触发 | 在边界继续滚动 | 在边界继续翻页 |
| 对齐策略 | 不需要对齐 | spread 对齐 |

## 验收标准

1. 滚动模式下阅读体验流畅。
2. 到达章节底部后继续向下滚动可自然触发越章。
3. 到达章节顶部后继续向上滚动可自然触发返回上一章。
4. 滚动进度计算准确（0–1 区间，线性）。
5. 竖排书写模式下滚动方向正确。
6. 所有纯函数都有单元测试。
