# 阶段 05 — 布局计算器增强

## 目标

增强现有的 `layout.ts` 响应式布局计算模块，使其能够为分页引擎和滚动引擎提供更精确的布局参数。

## 背景

当前 `layout.ts` 的 `computeResponsiveReadiumLayout()` 函数已经实现了基础的响应式布局计算，包括单/双页 spread 判断、页边距、栏间距等。需要增强的部分包括：

1. 为分页引擎提供精确的 spread 宽度计算。
2. 为不同章节类型（chapter / cover / illustration）提供差异化的布局参数。
3. 增加窗口断点逻辑，在不同宽度区间提供更精细的排版体验。

## 具体增强内容

### 新增功能

1. **精确 spread 宽度计算**：考虑滚动条、边框等对实际可用宽度的影响。
2. **窗口断点系统**：
   - `compact`：< 640px（移动端）
   - `medium`：640–1024px（平板）
   - `expanded`：1024–1440px（小桌面）
   - `large`：> 1440px（大桌面）
3. **章节类型感知**：不同章节类型有不同的布局策略。
4. **输出完整的布局描述**：不仅输出 CSS 变量值，还输出供引擎使用的度量值。

### 增强后的函数签名

```typescript
export interface LayoutCalculatorInput {
  viewportWidth: number;
  viewportHeight: number;        // 新增：高度（用于纵横比判断）
  preferredContentWidth: number;
  readingMode: ReadingMode;
  sectionKind: SectionKind;
  fontScale: number;             // 新增：字号缩放（影响行长判断）
}

export interface LayoutCalculatorOutput extends ResponsiveLayout {
  // 基础布局（已有）
  forcedColCount: "1" | "2";
  pageWidthPx: number;
  pageGutterPx: number;
  columnGapPx: number;
  maxLineLengthPx: number;
  illustrationWidthPx: number;
  shouldUseSpread: boolean;
  
  // 新增：引擎级度量
  spreadWidthPx: number;         // 整个 spread 的像素宽度（含间距）
  effectiveContentWidth: number; // 实际内容区宽度（去除边距）
  windowBreakpoint: WindowBreakpoint;
  verticalPadding: { top: number; bottom: number };
}

export type WindowBreakpoint = "compact" | "medium" | "expanded" | "large";

export function computeLayout(input: LayoutCalculatorInput): LayoutCalculatorOutput;
```

## 验收标准

1. 现有 `readium-layout.test.ts` 中的测试继续通过。
2. 新增布局字段的值与 CSS 变量设置一致。

4. 窗口断点判断正确。
5. 不影响现有渲染效果。
