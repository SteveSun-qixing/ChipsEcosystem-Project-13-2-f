# 阶段 06 — 文档控制器重构

## 目标

将 `ReadiumDocumentController` 从 1301 行的巨型类重构为精简的组合协调器，内部委托 `PaginationEngine`、`ScrollEngine`、`ReadiumCssManager`、`LayoutCalculator` 等模块工作。

## 背景

当前 `ReadiumDocumentController` 承担了太多职责：
- ReadiumCSS 样式加载和注入
- CSS 变量设置
- 分页测量和翻页
- 滚动测量和边界检测
- 窗口缩放处理
- 位置锚点恢复
- fragment 定位

重构后的文档控制器只做 **协调和生命周期管理**，具体工作委托给各个引擎模块。

## 具体实现

### 公共 API

```typescript
export class DocumentController {
  constructor(document: Document, options: EngineOptions);
  
  // ─── 生命周期 ───
  async mount(): Promise<void>;
  destroy(): void;
  
  // ─── 配置更新 ───
  async update(options: EngineOptions): Promise<void>;
  
  // ─── 导航 ───
  turnPage(direction: PageDirection): NavigationResult;
  scrollBy(deltaPx: number): NavigationResult;
  goToSpread(spreadIndex: number): void;
  goToBoundary(boundary: ReadingBoundary): void;
  async goToFragment(fragment?: string): Promise<void>;
  
  // ─── 查询 ───
  isAtBoundary(direction: PageDirection): boolean;
  getProgress(): ReadingProgress;
  getReadingMode(): ReadingMode;
  
  // ─── 事件 ───
  on(event: EngineEventType, handler: (event: EngineEvent) => void): () => void;
}

export interface NavigationResult {
  moved: boolean;              // 是否发生了移动
  reachedBoundary: boolean;    // 是否到达了边界
  boundary?: ReadingBoundary;  // 到达了哪个边界
}
```

### 内部结构

```
DocumentController
├── ReadiumCssManager       → 样式管理
├── PaginationEngine        → 分页计算（仅 paginated 模式使用）
├── ScrollEngine            → 滚动计算（仅 scroll 模式使用）
├── LayoutCalculator        → 布局参数
├── PositionTracker         → 位置追踪
├── ResizeObserver           → 窗口缩放监听
└── EventEmitter             → 事件分发
```

### mount() 生命周期

```
mount()
  ├── readiumCssManager.injectStyles(document)
  ├── readiumCssManager.applySettings(document, options, layout)
  ├── positionTracker.initialize()
  ├── attachResizeObserver()
  ├── waitForLayoutPasses(document, 2)
  └── alignToViewport()
```

### 翻页流程（paginated 模式）

```
turnPage("next")
  ├── paginationEngine.measure(document, directionality)
  ├── resolveSpreadIndex(currentScrollOffset, spreadWidth)
  ├── computeTargetOffset(currentIndex, "next", metrics)
  │   └── 返回 { targetIndex, targetOffset }
  ├── writeScrollOffset(document, targetOffset, behavior)
  ├── positionTracker.update(targetIndex, metrics)
  └── emit("page-changed", { direction, spreadIndex, totalSpreads })
```

### 窗口缩放恢复流程

```
onResize()
  ├── positionTracker.captureAnchor()       // 保存当前 spread 索引和比例
  ├── readiumCssManager.applySettings(...)  // 重新计算布局
  ├── waitForLayoutPasses(document, 1)
  ├── positionTracker.restoreAnchor()       // 从 spread 索引恢复
  └── alignToViewport()
```

### 与旧 ReadiumDocumentController 的对比

| 维度 | 旧实现 | 新实现 |
|------|--------|--------|
| 行数 | 1301 行（含所有内联逻辑） | 约 200–250 行（纯协调） |
| 职责 | 样式 + 分页 + 滚动 + 布局 + 缩放 + 定位 | 仅协调和生命周期管理 |
| 翻页计算 | 内联在 move() 中 | 委托 PaginationEngine |
| 样式管理 | 内联在 ensureReadiumStyles() 中 | 委托 ReadiumCssManager |
| 位置恢复 | 基于 ratio 的全局进度 | 基于 spread 索引的精确锚点 |
| 可测试性 | 需要完整 DOM 环境 | 各模块可独立测试 |
| 事件系统 | 无 | 有正式的事件发射器 |

## 验收标准

1. `mount()` 后 ReadiumCSS 正确注入，分页/滚动正常工作。
2. `update()` 后偏好和主题变化即时生效，不重建章节 DOM。
3. `turnPage()` 在分页模式下无累积漂移。
4. `scrollBy()` 在滚动模式下流畅响应。
5. 窗口缩放后阅读位置正确恢复，不跳页。
6. `isAtBoundary()` 正确检测章节边界。
7. `goToFragment()` 正确定位锚点。
8. 事件系统正常工作。
9. `npm run build` 编译通过。
