# 阶段 08 — 翻页动画控制器

## 目标

实现分页模式下的平滑翻页动画，替代当前依赖浏览器 `scrollTo({ behavior: 'smooth' })` 的不可控动画。

## 背景

当前翻页动画完全依赖浏览器的 `scrollTo({ behavior: 'smooth' })` 实现，存在以下问题：

1. **不可控**：动画时长、缓动函数由浏览器决定，不同浏览器/平台表现不一致。
2. **可能不生效**：部分浏览器或 iframe 上下文中 smooth scrolling 可能不生效。
3. **无法打断**：一旦触发 smooth scroll，无法精确控制中间状态。
4. **视觉单调**：只有简单的滑动效果，缺少现代阅读器常见的翻页过渡。

### 设计目标

实现自研的翻页动画系统：
- 使用 `requestAnimationFrame` 手动控制动画帧。
- 可配置的缓动函数和动画时长。
- 支持动画打断（快速连续翻页时前一个动画立即完成）。
- 未来可扩展支持更丰富的翻页效果。

## 具体实现

### 公共 API

```typescript
export class AnimationController {
  constructor(options?: AnimationOptions);
  
  // ─── 执行翻页动画 ───
  animateScroll(params: {
    element: Element;
    from: number;
    to: number;
    axis: "x" | "y";
    isRtl?: boolean;
    onComplete?: () => void;
    onCancel?: () => void;
  }): void;
  
  // ─── 取消当前动画（立即跳到目标位置） ───
  cancel(): void;
  
  // ─── 是否正在动画中 ───
  isAnimating(): boolean;
  
  // ─── 销毁 ───
  destroy(): void;
}

export interface AnimationOptions {
  duration: number;            // 动画时长 ms，默认 280
  easing: EasingFunction;     // 缓动函数
  minDuration: number;         // 最小时长 ms，默认 120
  maxDuration: number;         // 最大时长 ms，默认 450
}

export type EasingFunction = (t: number) => number;
```

### 内置缓动函数

```typescript
export const EASING = {
  // 标准缓动：从快到慢
  easeOutCubic: (t: number) => 1 - Math.pow(1 - t, 3),
  
  // 平滑缓动：先加速后减速
  easeInOutCubic: (t: number) =>
    t < 0.5 ? 4 * t * t * t : 1 - Math.pow(-2 * t + 2, 3) / 2,
  
  // 弹性缓动（微弹）
  easeOutBack: (t: number) => {
    const c1 = 1.70158;
    const c3 = c1 + 1;
    return 1 + c3 * Math.pow(t - 1, 3) + c1 * Math.pow(t - 1, 2);
  },
  
  // 线性
  linear: (t: number) => t,
} as const;
```

### 动画循环实现

```typescript
function animateScroll(params: AnimateScrollParams): void {
  // 如果有正在进行的动画，先取消（跳到目标）
  this.cancel();
  
  const { element, from, to, axis, isRtl, onComplete, onCancel } = params;
  const distance = Math.abs(to - from);
  
  // 距离太小，直接跳转
  if (distance < 2) {
    writeScroll(element, to, axis, isRtl);
    onComplete?.();
    return;
  }
  
  // 根据距离动态计算时长
  const duration = clamp(
    this.options.duration * Math.min(1, distance / 1440),
    this.options.minDuration,
    this.options.maxDuration,
  );
  
  const startTime = performance.now();
  let rafId: number;
  let cancelled = false;
  
  const tick = (currentTime: number) => {
    if (cancelled) return;
    
    const elapsed = currentTime - startTime;
    const rawProgress = Math.min(1, elapsed / duration);
    const easedProgress = this.options.easing(rawProgress);
    
    const current = from + (to - from) * easedProgress;
    writeScroll(element, current, axis, isRtl);
    
    if (rawProgress < 1) {
      rafId = requestAnimationFrame(tick);
    } else {
      // 确保精确到达目标位置
      writeScroll(element, to, axis, isRtl);
      this.currentAnimation = null;
      onComplete?.();
    }
  };
  
  rafId = requestAnimationFrame(tick);
  
  this.currentAnimation = {
    rafId,
    targetOffset: to,
    cancel: () => {
      cancelled = true;
      cancelAnimationFrame(rafId);
      writeScroll(element, to, axis, isRtl);
      this.currentAnimation = null;
      onCancel?.();
    },
  };
}
```

### 快速连续翻页处理

当用户快速连续翻页时（如按住方向键），动画系统的行为：
1. 收到新的翻页请求时，`cancel()` 当前动画（立即跳到当前动画的目标位置）。
2. 从新的位置开始下一个动画。
3. 动画时长会根据距离动态缩短，保证快速翻页的响应性。

## 验收标准

1. 分页模式翻页有明显的平滑过渡动画。
2. 动画时长约 200–300ms，视觉上流畅不拖沓。
3. 快速连续翻页时响应灵敏，不卡顿。
4. 动画结束后滚动位置精确对齐到 spread 边界。
5. RTL 模式下动画方向正确。
6. `cancel()` 后立即跳到目标位置。
7. 不影响滚动模式（滚动模式不使用此控制器）。
