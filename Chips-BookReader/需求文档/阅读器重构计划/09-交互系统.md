# 阶段 09 — 交互系统

## 目标

将所有输入事件处理逻辑从 `ReaderShell.tsx` 中抽离，形成独立的、可测试的交互系统。

## 背景

当前所有交互逻辑（键盘、鼠标点击、滚轮、触摸手势）都内嵌在 `ReaderShell.tsx` 的 useEffect 回调中，具体包括：

1. **键盘处理**（第 355–413 行）：方向键、PageUp/PageDown、Space、Home/End、`[`、`]`。
2. **鼠标点击**（第 445–511 行）：阅读区左/右/中间点击区域判定、链接点击、外部链接。
3. **滚轮/触摸板**（第 513–558 行）：分页模式翻页、滚动模式越章、滚轮累加器、方向阈值。
4. **触摸手势**（第 560–630 行）：触摸滑动翻页/越章。

这些代码占 ReaderShell 总行数的约 25%，且与 UI 渲染逻辑强耦合。

## 具体实现

### 交互层类型

```typescript
// ─── 交互意图（所有输入最终都映射到意图） ───
export type InteractionIntent =
  | { type: "navigate"; direction: PageDirection }
  | { type: "navigate-boundary"; boundary: ReadingBoundary }
  | { type: "toggle-chrome" }
  | { type: "close-panel" }
  | { type: "open-link"; href: string; isExternal: boolean }
  | { type: "epub-link"; path: string; fragment?: string }
  | { type: "adjust-font"; delta: number }
  | { type: "adjust-width"; delta: number }
  | { type: "none" };

// ─── 交互回调 ───
export interface InteractionCallbacks {
  onNavigate: (direction: PageDirection) => void;
  onNavigateBoundary: (boundary: ReadingBoundary) => void;
  onToggleChrome: () => void;
  onClosePanel: () => void;
  onOpenLink: (href: string, isExternal: boolean) => void;
  onEpubLink: (path: string, fragment?: string) => void;
  onAdjustFont: (delta: number) => void;
  onAdjustWidth: (delta: number) => void;
}
```

### 滚轮处理器

```typescript
export class WheelHandler {
  constructor(
    private callbacks: InteractionCallbacks,
    private getReadingMode: () => ReadingMode,
    private getController: () => DocumentController | null,
  );
  
  attach(target: Document): () => void;
  
  // ─── 滚轮累加器 ───
  private accumulator: {
    delta: number;
    timestamp: number;
    direction: PageDirection | null;
  };
  
  // ─── 滚轮归一化（跨浏览器、跨设备） ───
  private normalizeWheelDelta(event: WheelEvent): { x: number; y: number };
  
  // ─── 消费累积的滚轮增量 ───
  private consumeAccumulated(threshold: number): PageDirection | null;
  
  // ─── 分页模式滚轮阈值 ───
  private getPaginatedThreshold(deltaMode: number): number;
  
  // ─── 滚动模式边界越章阈值 ───
  private getScrollBoundaryThreshold(deltaMode: number): number;
}
```

关键改进：
- 滚轮累加器增加了更短的超时窗口（280ms → 当前 320ms），减少残留意图。
- 分页模式阈值更低（一个鼠标刻度即可翻页），操作更灵敏。
- 滚动模式在章节边界自动检测并触发越章。

## 验收标准

1. 所有现有的键盘快捷键功能保留。
2. 鼠标点击区域判定与旧实现一致。
3. 滚轮翻页灵敏度与旧实现一致或更好。
4. 触摸滑动翻页/越章与旧实现一致。
5. 快速连续输入有正确的防抖处理。
6. 交互处理器可独立于 React 组件测试。
7. 支持 iframe 内和宿主窗口两个事件源。
8. 内存泄漏检查：`destroy()` 后无残留事件监听。
