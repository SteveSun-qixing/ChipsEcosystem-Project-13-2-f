# 阶段 11 — UI 组件拆分

## 目标

将 1156 行的 `ReaderShell.tsx` 拆分为多个职责单一的 React 组件。

## 背景

当前 `ReaderShell.tsx` 包含以下 UI 区块：
- 空态界面（第 759–770 行）
- 阅读视口 + iframe（第 772–856 行）
- 控制层 Chrome（第 788–849 行，含标题卡、导航按钮、坞栏）
- 导入面板（第 860–912 行）
- 目录面板（第 914–952 行）
- 偏好设置面板（第 954–1150 行）
- 反馈消息（第 1152 行）
- `NavigationTree` 子组件（第 100–130 行）
- `ControlButton` 子组件（第 131–158 行）

所有这些 UI 元素和约 600 行的事件处理/状态管理代码混在一个文件中。

## 各组件职责

### ReaderShell.tsx（精简后约 200 行）

纯组合容器，不包含任何业务逻辑：
- 从 props 接收所有状态和回调。
- 组合 EmptyState / ViewerStage / ReaderChrome / 各面板。
- 管理拖拽状态（drag & drop）。
- 管理面板显隐状态（activePanel）。
- 管理控件可见性（isChromeVisible）。

### EmptyState.tsx

空态界面：居中的导入提示按钮。

```typescript
interface EmptyStateProps {
  onOpenSource: () => void;
  t: TranslateFunction;
}
```
