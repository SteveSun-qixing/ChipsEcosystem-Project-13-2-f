# 阶段 12 — 阅读进度系统

## 目标

实现完整的阅读进度可视化系统，包括进度条、页码指示、章节位置和百分比显示。

## 背景

当前阅读器完全没有阅读进度的可视化反馈。用户无法知道当前在章节的哪个位置、全书的哪个位置、当前章节有多少页等信息。

优秀的阅读器（Apple Books、Kobo）通常提供以下进度信息：
1. **底部进度条**：显示章节内或全书进度的细线条。
2. **页码指示**：分页模式下显示"第 X / Y 页"。
3. **百分比**：全书阅读进度百分比。
4. **章节位置**：当前章节名称和在全书中的位置。

## 具体实现

### 进度条 UI

#### 视觉设计

- **位置**：底部，位于坞栏之上。
- **样式**：2px 高度的细线，使用主题 primary 色。
- **背景**：使用 `color-mix(in srgb, var(--chips-sys-color-on-surface) 8%, transparent)` 半透明底色。
- **交互**：点击可跳转到对应位置；拖拽可实时预览。
- **显示时机**：只在 Chrome 可见时显示，与 Chrome 同步淡入淡出。

#### 进度条结构

```
┌─────────────────────────────────────────────┐
│ ──────────────────█─────────────────────── │  进度条
│      第 3 章 · 12/42 页 · 28%              │  信息文字
└─────────────────────────────────────────────┘
```

#### 分页模式

- 进度条基于章节内 spread 位置。
- 显示页码："第 X / Y 页"。
- 点击进度条可跳转到对应 spread。

#### 滚动模式

- 进度条基于章节内滚动比例。
- 不显示页码，显示百分比。
- 点击进度条可滚动到对应位置。

### 进度数据流

```
引擎层 (DocumentController)
  ├── 翻页/滚动事件
  ├── 计算 PaginationMetrics / ScrollMetrics
  └── 调用 PositionTracker.computeProgress()
        │
        ▼
Hook 层 (useReaderProgress)
  ├── 监听引擎 progress-updated 事件
  ├── 节流处理（200ms）
  └── 输出 ReadingProgress 状态
        │
        ▼
UI 层 (ProgressBar)
  ├── 渲染进度条
  ├── 显示页码/百分比
  └── 处理点击/拖拽交互
```

### 进度计算增强（position-tracker.ts）

需要增强 `computeProgress` 以支持更精确的进度信息：

```typescript
interface ReadingProgress {
  // 章节信息
  sectionIndex: number;
  sectionCount: number;
  sectionTitle: string;
  
  // 章节内进度
  sectionFraction: number;           // 0–1
  currentPage: number;               // 分页模式下的页码
  totalPages: number;                // 分页模式下的总页数
  
  // 全书进度
  bookFraction: number;              // 0–1
  bookPercentage: number;            // 0–100，取整
  
  // 显示文案（已翻译）
  pageLabel: string;                 // "12 / 42"
  sectionLabel: string;              // "第 3 / 15 章"
  percentageLabel: string;           // "28%"
}
```

## 验收标准

1. 分页模式下进度条显示章节内页码进度。
2. 滚动模式下进度条显示章节内滚动进度。
3. 百分比显示全书进度。
4. 页码显示正确（与实际翻页数一致）。
5. 点击进度条可跳转到对应位置。
6. 进度条与 Chrome 同步显隐。
7. 进度更新节流，不导致频繁渲染。
8. 文案全部走 i18n。
