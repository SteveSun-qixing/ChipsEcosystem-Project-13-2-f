# 阶段 13 — 书签系统

## 目标

实现书签功能，允许用户标记阅读位置并快速跳转回溯。

## 背景

书签是阅读器的核心功能之一，目前完全缺失。

### 功能设计

1. **添加书签**：在当前阅读位置添加书签，记录章节、页码/滚动位置。
2. **书签列表**：在书签面板中查看所有书签，按章节分组。
3. **跳转书签**：点击书签列表中的项目跳转到对应位置。
4. **删除书签**：从列表中移除书签。
5. **书签指示器**：在控制层显示当前位置是否有书签。

### 存储策略

书签数据目前存储在内存中（按 `bookSourceId` 隔离）。持久化存储需要 Host 的配置/存储服务支持——如果 Host 尚未提供此能力，需要提交工单。

## 具体实现

### 书签数据结构

```typescript
export interface Bookmark {
  id: string;                       // 唯一 ID
  sectionIndex: number;             // 章节索引
  sectionTitle: string;             // 章节标题
  spreadIndex: number;              // 分页模式下的 spread 索引
  scrollFraction: number;           // 滚动模式下的滚动比例
  readingMode: ReadingMode;         // 创建时的阅读模式
  createdAt: number;                // 创建时间戳
  label?: string;                   // 用户自定义标签（可选）
  excerpt?: string;                 // 上下文摘录（可选，未来扩展）
}
```

### 控制层书签按钮

在坞栏中添加书签按钮：
- 图标：当前位置无书签时为 `bookmark_border`，有书签时为 `bookmark`。
- 点击行为：切换当前位置的书签（添加/删除）。
- 长按或右键：打开书签面板。

### 书签存储 Hook

```typescript
export function useBookmarks(params: UseBookmarksParams): UseBookmarksReturn {
  // 内存存储，按 bookSourceId 分组
  const [bookmarksMap, setBookmarksMap] = useState<Map<string, Bookmark[]>>(new Map());
  
  const bookmarks = useMemo(() => {
    if (!params.bookSourceId) return [];
    return bookmarksMap.get(params.bookSourceId) ?? [];
  }, [bookmarksMap, params.bookSourceId]);
  
  // ... addBookmark, removeBookmark, hasBookmarkAtPosition
}
```
