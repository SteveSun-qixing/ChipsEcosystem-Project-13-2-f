# 阶段一：基线冻结与前置依赖

## 1. 阶段目标

- 归档旧版文档与旧版产品口径；
- 冻结新的页面范围；
- 冻结服务器独立 Web 主题 / 图标 / 多语言体系边界；
- 确认本轮不新增后端接口的大方向。

## 2. 交付物

- 新版需求手册；
- 新版技术手册；
- 新版开发计划；
- 前置工单 `工单073`。

## 3. 关键事项

1. 确认社区前端只保留个人主页与工作区上传页；
2. 确认旧版首页、发现、房间、工作台不再进入新方案；
3. 明确当前旧版主题 / 图标 / 词典实现不能继续作为最终方案；
4. 明确社区服务器与 Host 为独立底层系统，前端不得直接依赖 Host 运行时。

## 4. 阻塞项

- `工单073` 未闭环前，代码实施阶段不能把旧版硬编码主题 / 图标 / 多语言实现继续扩展下去。

## 5. 阶段验收

- 新文档全部完成；
- 旧文档全部归档；
- 新范围与旧范围边界清晰；
- 前置工单已登记并可追踪。
