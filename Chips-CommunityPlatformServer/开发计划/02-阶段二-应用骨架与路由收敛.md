# 阶段二：应用骨架与路由收敛

## 1. 阶段目标

- 清理旧版壳层；
- 把应用路由收敛到欢迎页、简介页、认证页、个人页、工作区与后台中转页；
- 去掉旧版总站式工作台导航；
- 建立管理员账号自动切换后台的入口规则。

## 2. 实施内容

1. 重写 `App.tsx` 顶层路由；
2. 用 `SiteHeader` 替代旧版 `GlobalNav`；
3. 建立欢迎页、简介页与 `/admin` 中转页；
4. 处理普通用户与管理员的登录分流逻辑；
5. 建立 `/workspace` 受保护路由；
6. 把旧版页面从活动路由树中移除。

## 3. 代码影响面

- `packages/web/src/App.tsx`
- `packages/web/src/main.tsx`
- `packages/web/src/pages/*`
- `packages/web/src/components/*`
- `packages/admin/src/*`

## 4. 阶段验收

- 活动路由收敛到目标集合；
- 普通用户登录默认进入本人主页；
- 管理员登录默认进入后台；
- 未登录访问工作区会正确跳转；
- 旧版工作台、房间页不再可达。
