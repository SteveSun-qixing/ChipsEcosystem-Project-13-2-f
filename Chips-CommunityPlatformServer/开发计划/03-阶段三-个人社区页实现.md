# 阶段三：个人社区页实现

## 1. 阶段目标

- 实现新的 `@username` 个人页；
- 完成头图区和统一作品网格；
- 接通本人专属“+”入口。

## 2. 实施内容

1. 并行拉取用户信息、卡片列表、箱子列表；
2. 转换为统一作品模型并按时间混排；
3. 实现头图区视觉层；
4. 实现作品网格与封面卡片；
5. 完成 loading / empty / error / owner 四类状态。
6. 将卡片封面卡面改为独立封面 HTML 直出样式，移除内层窗口化视觉。

## 3. 关键注意项

- 不再把房间作为公开浏览主结构；
- 不为个人页添加搜索、筛选、统计面板；
- “+”只在本人页显示。
- 公开个人页只展示 `public + ready` 的卡片，避免处理失败或未完成卡片污染公开页面。

## 4. 阶段验收

- 访客看到公开作品混排；
- 本人看到相同公开作品混排和“+”入口；
- 页面视觉上已接近最终产品气质。
