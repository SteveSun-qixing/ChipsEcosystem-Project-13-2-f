# 阶段六：联调验证与发布归档

## 1. 阶段目标

- 以正式脚本和人工验收完成收口；
- 对齐代码、文档、验证记录；
- 归档无效残留。

## 2. 验证清单

1. `pnpm --dir Chips-CommunityPlatformServer lint`
2. `pnpm --dir Chips-CommunityPlatformServer test`
3. `pnpm --dir Chips-CommunityPlatformServer build`
4. 登录跳转验收；
5. 个人页 owner / visitor 双场景验收；
6. 卡片 / 箱子上传验收；
7. 主题、多语言、空态、错误态验收。

## 3. 文档收口

- 需求、技术、计划与最终实现一致；
- 未采用的旧口径继续留在归档目录；
- 若实施中再发现生态级依赖缺口，先登记工单，再决定是否继续推进。

## 4. 阶段验收

- 正式脚本通过；
- 页面行为满足需求手册；
- 文档、验证记录和实际代码一致；
- 可进入下一轮真实代码实施或用户验收。
