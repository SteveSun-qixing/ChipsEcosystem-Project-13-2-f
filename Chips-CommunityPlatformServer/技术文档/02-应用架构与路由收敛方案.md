# 应用架构与路由收敛方案

## 1. 目标结构

前端应用保留一个轻量公开站点外壳，并与独立后台应用做入口协同：

1. 欢迎页；
2. 社区简介页；
3. 认证入口层；
4. 个人社区页；
5. 工作区上传页；
6. `/admin` 中转页。

不再保留旧版总站式工作台导航容器，也不再设置全局顶部菜单栏；公开站点改为页面独立结构，只在页底保留轻量相关链接区。

## 2. 路由方案

### 2.1 公开站点顶层路由

- `/`：欢迎页；管理员登录态下自动切换 `/admin`；
- `/about`：社区简介页；
- `/login`：登录页；
- `/register`：注册页；
- `/@:username`：个人社区页；
- `/workspace`：上传工作区；
- `/admin`：后台中转页。

### 2.2 内容直达路由

- `/cards/:cardId`
- `/boxes/:boxId`

它们保留为内容打开链路，但不再参与社区壳层布局。

补充约束：

- `cards/:cardId` 在跳转到独立内容 HTML 前不再显示带标题与文案的中转页；
- 若详情查询与跳转足够快，则不显示任何中间态；
- 仅当等待时间超过短阈值时，才显示基于原站背景的纯转圈加载层。
- 内容直达路由不渲染站点页脚等公共壳层文字，只保留最小化加载背景。

## 3. 组件拆分建议

### 3.1 页面层

- `pages/HomePage`
- `pages/AboutPage`
- `pages/ProfilePage`
- `pages/WorkspacePage`
- `pages/LoginPage`
- `pages/RegisterPage`
- `pages/AdminRedirectPage`

### 3.2 视图组件

- `components/SiteHeader`
- `components/ProfileHero`
- `components/WorkGrid`
- `components/WorkTile`
- `components/UploadDropzone`
- `components/UploadQueue`
- `components/UploadQueueItem`

组件数量控制在必要最小集，不回到旧版“面板化前台工作台”方向；后台组件继续留在独立 `packages/admin` 中。

## 4. 旧页面移除策略

以下旧页面从活动路由中移除，不做兼容双跑：

- `DashboardPage`
- `RoomPage`
- 旧版 `CardUploadPage`
- `GlobalNav`

## 5. 导航规则

- 公开站点头部只暴露欢迎、简介和普通创作者必要入口；
- 页面跳转主要通过页面内部按钮完成；
- 页面底部只保留小型相关链接，不承载主导航职责；
- 登录成功后，普通用户直接进入本人主页；
- 登录成功后，管理员用户直接进入 `/admin`；
- 本人主页的“+”是进入工作区的唯一主入口；
- 工作区页面只保留“返回我的主页”这类最短路径；
- 后台入口不通过显性按钮暴露，而通过账号角色自动切换。

## 6. 应用状态最小集合

顶层只保留四类全局状态：

1. 当前登录用户；
2. 当前主题运行时；
3. 当前语言运行时；
4. 路由守卫状态。

其他状态全部下沉到页面内部，不再建立工作台式全局聚合态。
