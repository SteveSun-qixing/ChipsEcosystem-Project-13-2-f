# 工作区上传页技术方案

## 1. 页面目标

工作区页只围绕上传任务建模，不再承担工作台聚合职责。

## 2. 输入方式

必须同时支持两种输入路径：

1. 拖拽文件进入页面；
2. 点击拖拽区调起文件选择器。

## 3. 文件识别

前端根据文件扩展名直接判定类型：

- `.card` -> 卡片上传
- `.box` -> 箱子上传

不再保留旧版手动模式切换。

## 4. 上传默认参数

为保持页面极简，前端统一采用：

- `roomId: undefined`
- `visibility: 'public'`

也就是默认上传到用户根目录并公开发布。

## 5. 队列模型

推荐统一使用：

```ts
type UploadQueueItem = {
  localId: string;
  file: File;
  type: 'card' | 'box';
  status: 'queued' | 'uploading' | 'processing' | 'success' | 'error';
  progress: number;
  remoteId?: string;
  resultHref?: string;
  errorMessage?: string;
};
```

## 6. 卡片上传流程

1. 调用 `POST /api/v1/upload/card`；
2. 利用现有上传回调显示百分比；
3. 若富文本节点采用 `content_source=file`，服务端先把作者态 Markdown 资源吸收到 content YAML，再继续发布链路；
4. 服务端上传并替换卡片资源，生成新的 `.card`；
5. 服务端输出内容 HTML，并直接发布转换模块的原生导出结果；
6. 服务端单独输出封面 HTML；
7. 轮询 `GET /api/v1/cards/:cardId/status`；
8. `ready` 时进入成功态；
9. `error` 时进入失败态。

## 7. 箱子上传流程

1. 调用 `POST /api/v1/upload/box`；
2. 上传成功即进入成功态；
3. 为结果卡片写入 `resultHref`。

## 8. 并发策略

- 推荐上传串行、状态展示并行；
- 先保证稳定和反馈清晰，再考虑更高并发；
- 本轮不为了“批量高吞吐”增加复杂上传调度器。

## 9. 页面反馈

### 9.1 空闲态

- 显示大拖拽区；
- 给出支持格式与拖拽提示。

### 9.2 有队列时

- 拖拽区仍保留；
- 下方显示最近上传队列；
- 成功项允许一键返回个人页或打开内容。
