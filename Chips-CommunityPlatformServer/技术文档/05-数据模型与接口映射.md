# 数据模型与接口映射

## 1. 现有接口复用结论

本轮前端重构**原则上不新增后端接口**，优先复用当前已存在能力。

## 2. 个人页接口映射

### 2.1 用户信息

- 接口：`GET /api/v1/users/:username`
- 用途：头像、昵称、简介、用户名。

### 2.2 卡片列表

- 接口：`GET /api/v1/users/:username/cards`
- 用途：获取该用户公开且已完成处理的卡片。

### 2.3 箱子列表

- 接口：`GET /api/v1/users/:username/boxes`
- 用途：获取该用户公开箱子。

## 3. 本人状态接口映射

- 接口：`GET /api/v1/users/me`
- 用途：判断当前访问者是否为本人，并驱动“+”入口显隐。

## 4. 工作区接口映射

### 4.1 卡片上传

- 接口：`POST /api/v1/upload/card`
- 入参：`file` + 默认 `visibility=public`

### 4.2 卡片状态

- 接口：`GET /api/v1/cards/:cardId/status`
- 用途：处理链路轮询。
- 访问口径：与 `GET /api/v1/cards/:cardId` 一致，按卡片可见性决定是否可读，不再要求必须登录。

### 4.3 箱子上传

- 接口：`POST /api/v1/upload/box`
- 入参：`file` + 默认 `visibility=public`

## 5. 不再作为前端主消费接口的能力

以下能力暂时保留在后端，但不再作为社区前端 V2 的主界面入口：

- `GET /api/v1/users/:username/space`
- `GET /api/v1/users/:username/rooms`
- `GET /api/v1/rooms/:roomId`
- `GET /api/v1/rooms/:roomId/contents`
- `GET /api/v1/discover/cards`
- `GET /api/v1/discover/boxes`
- `GET /api/v1/search`

## 6. 页面内部统一模型

### 6.1 作品模型

```ts
type CommunityWorkItem = {
  id: string;
  type: 'card' | 'box';
  title: string;
  coverUrl: string | null;
  href: string;
  createdAt: string;
};
```

补充说明：

- `card.coverUrl` 在卡片场景中表示独立封面 HTML 入口；
- `box.coverUrl` 仍表示箱子封面资源地址；
- `card.htmlUrl` 表示卡片内容 HTML 入口，社区前台通过 `/cards/:cardId` 创建 `com.chips.card-viewer` Web 插件会话，并由原版 `CardViewer` 以托管文档方式承载它；
- 卡片内部图片点击后，不再停留在静态 HTML 中，而是通过正式 `resource.open` 路由进入新的 `/host/plugins/:sessionId` 插件宿主页，由原版 `PhotoViewer` 接管查看。

### 6.2 上传模型

```ts
type UploadQueueItem = {
  localId: string;
  type: 'card' | 'box';
  status: 'queued' | 'uploading' | 'processing' | 'success' | 'error';
  progress: number;
  resultHref?: string;
  errorMessage?: string;
};
```

## 7. 前端需要显式隐藏的旧能力

- 房间选择；
- 房间管理；
- 作品删除；
- 资料编辑；
- 发现搜索；
- 平台级导航入口。

这些能力不是后端删除，而是本轮前端明确不暴露。

## 8. 社区前台新增 Host Runtime 接口映射

本轮为正式 Web 插件宿主新增以下社区内部接口：

- `POST /api/v1/host/plugin-sessions`
  - 创建 Web 应用插件会话。
- `GET /api/v1/host/plugin-sessions/:sessionId`
  - 读取插件会话视图。
- `DELETE /api/v1/host/plugin-sessions/:sessionId`
  - 关闭插件会话。
- `POST /api/v1/host/resource-open-plan`
  - 在服务端按正式 `resource-handler / file-handler` 能力解析 Web 资源打开目标。
- `GET /api/v1/host/plugin-sessions/:sessionId/entry`
  - 返回注入 bootstrap 后的插件 HTML 入口。
- `GET /api/v1/host/plugin-sessions/:sessionId/bootstrap.js`
  - 返回浏览器侧 `window.chips` bootstrap。
- `GET /api/v1/host/plugin-sessions/:sessionId/assets/*`
  - 返回该插件会话所需静态资源。

这些接口只服务社区服务器自己的 Web 插件宿主链路，不作为生态公共 API 主发布位置。
