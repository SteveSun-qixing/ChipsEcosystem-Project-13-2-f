# 数据模型与接口映射

## 1. 现有接口复用结论

本轮前端重构**原则上不新增后端接口**，优先复用当前已存在能力。

## 2. 个人页接口映射

### 2.1 用户信息

- 接口：`GET /api/v1/users/:username`
- 用途：头像、昵称、简介、用户名。

### 2.2 卡片列表

- 接口：`GET /api/v1/users/:username/cards`
- 用途：获取该用户公开且已完成处理的卡片。

### 2.3 箱子列表

- 接口：`GET /api/v1/users/:username/boxes`
- 用途：获取该用户公开箱子。

## 3. 本人状态接口映射

- 接口：`GET /api/v1/users/me`
- 用途：判断当前访问者是否为本人，并驱动“+”入口显隐。

## 4. 工作区接口映射

### 4.1 卡片上传

- 接口：`POST /api/v1/upload/card`
- 入参：`file` + 默认 `visibility=public`

### 4.2 卡片状态

- 接口：`GET /api/v1/cards/:cardId/status`
- 用途：处理链路轮询。

### 4.3 箱子上传

- 接口：`POST /api/v1/upload/box`
- 入参：`file` + 默认 `visibility=public`

## 5. 不再作为前端主消费接口的能力

以下能力暂时保留在后端，但不再作为社区前端 V2 的主界面入口：

- `GET /api/v1/users/:username/space`
- `GET /api/v1/users/:username/rooms`
- `GET /api/v1/rooms/:roomId`
- `GET /api/v1/rooms/:roomId/contents`
- `GET /api/v1/discover/cards`
- `GET /api/v1/discover/boxes`
- `GET /api/v1/search`

## 6. 页面内部统一模型

### 6.1 作品模型

```ts
type CommunityWorkItem = {
  id: string;
  type: 'card' | 'box';
  title: string;
  coverUrl: string | null;
  href: string;
  createdAt: string;
};
```

补充说明：

- `card.coverUrl` 在卡片场景中表示独立封面 HTML 入口；
- `box.coverUrl` 仍表示箱子封面资源地址；
- `card.htmlUrl` 表示卡片内容 HTML 入口，且该入口已经去除导出舞台壳，直接用于 Web 查看。

### 6.2 上传模型

```ts
type UploadQueueItem = {
  localId: string;
  type: 'card' | 'box';
  status: 'queued' | 'uploading' | 'processing' | 'success' | 'error';
  progress: number;
  resultHref?: string;
  errorMessage?: string;
};
```

## 7. 前端需要显式隐藏的旧能力

- 房间选择；
- 房间管理；
- 作品删除；
- 资料编辑；
- 发现搜索；
- 平台级导航入口。

这些能力不是后端删除，而是本轮前端明确不暴露。
