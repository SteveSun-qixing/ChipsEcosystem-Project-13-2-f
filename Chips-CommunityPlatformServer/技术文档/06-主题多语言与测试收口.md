# 主题多语言与测试收口

## 1. 前置依赖

本轮前端重构不得继续把以下旧版实现直接当作最终方案：

- `packages/web/src/contexts/AppPreferencesContext.tsx`
- `packages/web/src/styles/tokens.css`

相关原因已登记到：

- `项目日志与笔记/工单073-社区平台前端旧版主题词典与多页面壳层未升级到服务器独立Web体系/任务说明.md`

## 2. 正式接线原则

### 2.1 主题

- 社区服务器必须建设**自己的 Web 主题系统**；
- 主题系统可以复用 Host 的设计理念与部分代码来源，但部署、资源和运行时均独立；
- 页面样式统一消费服务器侧主题 token；
- 主题切换后必须形成服务器前端自己的刷新链路；
- 不允许继续把旧版页面里的临时主题枚举当作正式架构。

### 2.2 多语言

- 社区服务器必须建设**自己的 Web 多语言系统**；
- 多语言系统可以沿用 Host 的命名哲学，但词典组织、加载方式和缓存策略要面向 Web 服务单独优化；
- 所有文案必须通过统一翻译入口获取；
- 语言切换后必须触发页面重渲染；
- 不允许再把硬编码词典作为长期正式口径。

### 2.3 图标

- 社区服务器必须建设**自己的 Web 图标系统**；
- 图标与主题、多语言一样，与 Host 没有直接运行时依赖；
- 图标资源、映射和尺寸策略要按 Web 前端的性能与可读性单独优化；
- 页面中不得长期保留散落的临时字符图标方案。

## 3. 微动效实现原则

- 不新增动画库；
- 优先使用 CSS keyframes + data attributes + 轻量随机种子；
- 漂动幅度控制在小范围内；
- 必须支持 `prefers-reduced-motion` 降级。

## 4. 测试与验证收口

由于当前前端包没有独立测试基线，本轮至少以以下正式脚本和人工验收收口：

1. `pnpm --dir Chips-CommunityPlatformServer build`
2. `pnpm --dir Chips-CommunityPlatformServer lint`
3. `pnpm --dir Chips-CommunityPlatformServer test`
4. 个人页人工验收：
   - 访客访问他人主页；
   - 用户访问本人主页；
   - 本人打开资料设置弹层并更新名字、头像、简介；
   - 作品混排与点击跳转。
5. 工作区人工验收：
   - 拖入 `.card`
   - 拖入 `.box`
   - 成功、失败、处理中三类状态反馈。

## 5. 发布前检查

- 旧版总站式页面不再出现在活动路由；
- 个人页与工作区页均完成服务器侧主题、图标、多语言、空态、错误态收口；
- 个人页透明头图区、磨砂背景和彩色光晕动效已完成主题化收口；
- 视觉动效已通过桌面端与移动端人工检查；
- 文档、实现、验证记录一致。
