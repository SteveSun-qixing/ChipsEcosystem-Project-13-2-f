## 4. 推荐的人工验收链路

### 4.1 认证链路

1. 注册新用户
2. 登录
3. 刷新页面，确认通过 refresh cookie 恢复登录态
4. 退出登录

### 4.2 卡片上传链路

1. 上传 `.card`
2. 记录返回的 `cardId`
3. 轮询 `/api/v1/cards/:cardId/status`
4. 确认状态从 `pending -> processing -> ready`
5. 打开 `/cards/:cardId`
6. 确认浏览器直接跳转到转换后的 HTML 页面
7. 检查图片、音视频、代码块等资源是否正常加载

### 4.3 箱子上传链路

1. 上传 `.box`
2. 打开箱子详情页
3. 检查 `layoutPlugin`、`boxFileId`、引用列表是否正确
4. 若箱子中的 `card_id` 已存在社区公开卡片，确认出现社区跳转入口

### 4.4 权限链路

1. 创建公开房间
2. 在公开房间中放入私有卡片 / 私有箱子
3. 用另一账号或访客访问房间
4. 确认：
   - 看不到私有内容
   - 房间计数也不泄露私有内容数量

## 5. 常见问题排查

### 5.1 启动即退出

优先检查：

- `deploy/.env` 是否完整
- `BASE_URL`、`S3_ENDPOINT` 是否为合法 URL
- `JWT_SECRET` 是否满足长度要求

### 5.2 数据库迁移失败

优先检查：

- `DATABASE_URL` 是否可连通
- PostgreSQL 是否已启动
- 当前账号是否具备迁移权限

### 5.3 Redis 相关鉴权异常

若出现登录失败计数或 token 黑名单异常，优先检查：

- `REDIS_URL`
- Redis 实例是否可访问
- 重启后是否清空了开发环境 Redis 状态

### 5.4 卡片状态停在 `error`

优先检查：

- 服务端日志中的流水线报错
- `.card` 文件结构是否完整
- Host 工作区插件是否安装成功
- 文件转换插件是否可正常执行

### 5.5 卡片图片不显示

优先检查：

1. 资源文件是否已上传到 `chips-card-resources`
2. 卡片内容中的资源 URL 是否已被替换为 CDN URL
3. `/cdn/*` 反向代理是否配置正确
4. 目标对象是否可匿名读

### 5.6 卡片页面效果与编辑引擎不一致

优先检查：

1. 当前是否真的走了正式文件转换插件
2. Host 是否安装并启用了正确主题
3. 是否错误引用了已归档的本地 HTML 渲染旧实现

### 5.7 房间页数量不对

优先区分：

- 当前查看者是否为房间所有者
- 卡片是否为 `ready`
- 内容是否为 `public`

因为当前 `cardCount` / `boxCount` 已按“当前请求者可见内容”返回，而不是绝对总量。

## 7. 当前已知边界

当前第一版仍然缺少：

- 更大范围的集成测试
- 端到端浏览器自动化测试
- Host 插件安装状态可视化面板
- Docker 真机构建结果在本 CLI 会话中的直接复核能力

因此当前最佳实践是：

1. 先跑现有自动化测试与构建
2. 再在真实部署环境中做上传、转换、查看链路验收
