# 薯片社区平台服务器需求文档索引

## 文档定位

- 本目录保存 `Chips-CommunityPlatformServer` 自身的产品需求与交付边界。
- 只写社区服务器项目内部的产品定义、交互规则、交付范围和非功能要求。
- 与整个生态复用相关的正式公共契约，统一写入 `生态共用技术文档/`。

## 使用规则

- 阅读顺序建议为：`01` 总览 → 角色权限 → 页面地图 → 核心流程 → 具体领域规则。
- 需求文档必须以当前代码与当前正式链路为准，不得保留已废弃的设想口径。
- 当需求与生态公共契约发生交叉时：
  - 项目内部产品取舍、页面行为、交付范围写在这里；
  - 复用型 API / 文件 / 协议契约写到 `生态共用技术文档/`。
