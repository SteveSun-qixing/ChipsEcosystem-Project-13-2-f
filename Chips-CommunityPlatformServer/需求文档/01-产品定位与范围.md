# 产品定位与范围

## 1. 产品目标

- 社区前端不做旧式信息流门户，但需要保留一个轻量的欢迎入口与社区简介页；
- 社区整体仍以**个人主页**为中心，每个用户拥有一个公开页面；
- 页面体系由欢迎页、简介页、登录 / 注册、个人主页、工作区和管理员后台共同组成；
- 公开站点负责讲清楚社区与展示创作，后台治理通过独立管理端完成。

## 2. 本轮范围

### 2.1 必做

- 个人主页头图区；
- 统一作品网格；
- 本人可见的“+”入口；
- 欢迎页；
- 社区简介页；
- 登录页 / 注册页；
- 工作区上传页；
- 管理员后台入口分流规则；
- 登录 / 注册后的最短访问链路；
- 极简、高质感、轻微立体、细腻灵动的视觉表现。

### 2.2 明确不做

- 传统信息流首页；
- 房间列表页、房间详情页；
- 工作台聚合页；
- 作品管理后台式信息面板；
- 与当前目标无关的多余功能模块。

## 3. 内容组织原则

- 用户主页只展示**该用户全部公开卡片与箱子**；
- 展示单位统一称为“作品”；
- 前端不再把房间作为公开浏览主结构；
- 后端已有房间能力继续保留，但本轮前端不暴露房间组织界面。

## 4. 交互主线

1. 访客通过欢迎页与社区简介页理解社区定位；
2. 访客进入 `@username` 页面浏览公开作品；
3. 普通用户登录后进入自己的 `@username` 页面；
4. 管理员用户登录后自动切换到后台管理端；
5. 作者点击作品区右上角“+”进入工作区；
6. 作者在工作区拖入 `.card` / `.box` 文件；
7. 页面显示上传与处理进度；
8. 上传完成后返回个人主页查看新作品。

## 5. 范围边界

- 本手册约束社区公开前端 `packages/web` 与其和 `packages/admin` 的入口衔接规则；
- `packages/server` 的业务逻辑继续视为既有能力，不在本手册内重写业务契约；
- 卡片 / 箱子内容消费链路继续沿用现有内容直达路由，不把它们扩展成社区壳层页面体系。
