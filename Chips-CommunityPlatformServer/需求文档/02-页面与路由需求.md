# 页面与路由需求

## 1. 可见页面集合

社区前端 V2 保留以下用户可见页面：

1. 欢迎页；
2. 社区简介页；
3. 登录页；
4. 注册页；
5. 个人社区页；
6. 工作区上传页；
7. 管理员后台跳转页与独立后台应用。

## 2. 路由口径

### 2.1 社区壳层正式路由

- `/`
  - 未登录时渲染欢迎页；
  - 欢迎页需要提供登录入口与社区简介入口；
  - 已登录普通用户访问时，自动切到当前用户个人页；
  - 已登录管理员访问时，自动切到 `/admin`。
- `/about`
  - 社区简介页；
  - 用于交代社区定位、系统边界与使用路径。
- `/login`
  - 登录页；
  - 普通账号成功后跳转到当前用户个人页；
  - 管理员账号成功后跳转到 `/admin`。
- `/register`
  - 注册页；
  - 成功后跳转到当前用户个人页。
- `/@:username`
  - 个人社区页；
  - 公开展示该用户全部公开作品。
- `/workspace`
  - 工作区上传页；
  - 仅已登录用户可访问。
- `/admin`
  - 只承担前台到后台的中转职责；
  - 真实后台界面由独立管理端承载；
  - 不在公开站点上设置显性入口按钮，而是通过管理员账号登录后自动切换。

### 2.2 保留但不计入社区公共信息架构设计

- `/cards/:cardId`
- `/boxes/:boxId`

这两类内容直达路由继续承担内容打开职责，但不再参与社区前端的信息架构设计。

## 3. 必须移除的旧页面层

- 旧版 `DashboardPage`
- 旧版 `RoomPage`
- 旧版 `CardUploadPage` 的分模式上传中心心智
- 旧版后台式 `GlobalNav`
- 旧版发现入口、工作台入口、后台入口按钮
- 全局顶部菜单栏

## 4. 所有权规则

- 访问他人主页时：
  - 只显示公开作品；
  - 不显示“+”按钮；
  - 不显示任何后台式管理入口。
- 访问本人主页时：
  - 作品区标题栏右上角显示“+”按钮；
  - 点击后进入 `/workspace`。
- 管理员入口：
  - 不在欢迎页、简介页、个人主页头部放显性后台按钮；
  - 通过登录账号角色识别后自动进入后台。
- 页面结构：
  - 每个页面独立成立，不挂在统一顶部菜单下；
  - 跳转通过页面内部按钮与底部小链接完成；
  - 页面最下方保留轻量相关链接区。

## 5. 认证跳转规则

- 未登录访问 `/workspace` 时，重定向到 `/login`；
- 普通用户登录成功后，若存在来源页，优先返回来源页；
- 普通用户若无来源页，直接进入本人的 `@username` 页面；
- 管理员登录成功后，忽略普通前台来源页，直接进入 `/admin`；
- 不再以工作台作为默认登录落点。
