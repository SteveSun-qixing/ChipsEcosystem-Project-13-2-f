# 个人社区页需求

## 1. 页面结构

个人社区页由两部分组成：

1. 顶部横向头图区；
2. 下方作品网格区。

除这两部分外，不再添加总站式模块、栏目页或管理面板。

## 2. 头图区

### 2.1 展示内容

- 用户头像；
- 昵称；
- 用户名；
- 个人简介。
- 本人访问本人页面时，头图区右上角提供资料设置入口。

### 2.2 表现要求

- 头图区是横向大条幅；
- 以主题驱动的视觉层承载内容，不额外新增“个人封面上传”业务；
- 头图区必须具备轻微立体感和高级留白，而不是信息堆叠。
- 头图区整体不再套传统资料卡底板，主体背景保持透明，由页面背景氛围负责承载质感。

## 3. 作品区

### 3.1 数据范围

- 展示该用户全部公开卡片；
- 展示该用户全部公开箱子；
- 两类内容在同一网格中统一呈现。

### 3.2 排序规则

- 按创建时间倒序混排；
- 最新作品优先展示；
- 卡片与箱子不拆成两个独立分区。

### 3.3 作品卡面信息

- 封面为主；
- 标题为辅；
- 卡片封面必须直接展示独立封面本体，不做额外“窗口套窗口”壳层；
- 卡片下方只保留简洁名称，不展示日期、类型行等冗余条目；
- 不展示冗余后台字段。

## 4. 本人专属入口

- 仅本人访问自己的 `@username` 页面时，头图区右上角显示资料设置按钮；
- 设置按钮打开浮层卡片，用于编辑个人名字、上传个人头像、编辑个人简介；
- 仅本人访问自己的 `@username` 页面时，作品区标题栏右上角仍显示“+”；
- “+”继续作为进入工作区的主动入口；
- 两个入口都必须简洁克制，不能把个人页做成后台管理台。

## 5. 状态要求

### 5.1 加载态

- 头图区与作品网格都需要骨架或等价轻量加载态；
- 不允许页面大面积跳变。

### 5.2 空态

- 无公开作品时，仍保留完整头图区；
- 作品区展示简洁空态文案；
- 本人空态下仍显示“+”按钮。
- 本人空态下仍保留资料设置按钮。

### 5.3 错误态

- 用户不存在时给出明确的轻量错误态；
- 错误态不能退化为杂乱调试信息。

## 6. 点击行为

- 点击卡片作品，进入卡片内容直达链路；
- 点击箱子作品，进入箱子内容直达链路；
- 不额外插入中间管理页。

## 7. 卡片封面正式要求

- 卡片封面不是内容 HTML 的截图，也不是任意猜测的一张资源图；
- 卡片上传处理完成后，必须单独保存一份封面 HTML 产物；
- 卡片内容 HTML 与封面 HTML 必须是两份独立产物；
- 两者需要保留可追溯的互相关联关系，便于后续独立调用与复用。
