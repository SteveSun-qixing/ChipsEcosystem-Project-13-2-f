# 核心用户流程

## 1. 新用户入驻流程

1. 进入首页
2. 点击注册
3. 输入用户名、密码、确认密码
4. 注册成功后直接获得登录态
5. 跳转到工作台
6. 可继续创建房间或上传内容

## 2. 登录与会话恢复流程

1. 用户在登录页输入用户名与密码
2. 服务端返回 `accessToken`，并写入 refresh cookie
3. 前台把 `accessToken` 保存在内存中
4. 页面刷新后，前台会主动调用 `/api/v1/auth/refresh`
5. 若 refresh 成功，则恢复登录态并重新拉取当前用户资料

## 3. 创建房间流程

1. 用户进入工作台
2. 填写房间名称、描述、可见性
3. 提交创建
4. 服务端生成 slug 并返回房间 DTO
5. 工作台刷新房间列表

## 4. 上传卡片流程

1. 用户进入上传中心
2. 选择“上传卡片”
3. 选择 `.card` 文件
4. 可选选择房间或根目录
5. 选择公开 / 私有
6. 提交上传
7. 前台收到 `cardId + pending`
8. 前台轮询状态接口
9. 服务端异步完成：
   - 解包
   - 资源上 CDN
   - URL 替换
   - 重打包新 `.card`
   - 调用 Host 正式转换插件转 HTML
   - 上传 HTML 目录
10. 状态变为 `ready`
11. 用户点击结果入口进入 `/cards/:cardId`
12. 页面直接跳转到转换后的 HTML 页面

## 5. 上传箱子流程

1. 用户进入上传中心
2. 选择“上传箱子”
3. 选择 `.box` 文件
4. 可选选择房间或根目录
5. 选择公开 / 私有
6. 提交上传
7. 服务端同步解析 `.box` 并入库
8. 前台直接获得箱子详情入口
9. 用户进入箱子详情页查看结构摘要与引用卡片

## 6. 浏览公开内容流程

1. 访客进入首页
2. 浏览最新公开卡片 / 箱子
3. 或通过搜索查找用户、卡片、箱子
4. 进入目标用户空间
5. 进入公开房间
6. 查看其中可见卡片 / 箱子
7. 点击卡片进入 HTML 页面
8. 点击箱子进入箱子详情页

## 7. 查看私有内容流程

1. 用户登录
2. 进入自己的空间或房间
3. 服务端识别其为所有者
4. 返回完整内容列表与完整计数
5. 用户可查看自己的私有卡片 / 箱子

## 8. 删除内容流程

### 8.1 删除卡片

1. 用户在工作台点击删除卡片
2. 服务端校验所有权
3. 清理卡片资源与 HTML 目录
4. 删除数据库记录
5. 工作台刷新

### 8.2 删除箱子

1. 用户在工作台点击删除箱子
2. 服务端校验所有权
3. 删除数据库记录
4. 工作台刷新

### 8.3 删除房间

1. 用户在工作台点击删除房间
2. 服务端先删除房间内卡片与箱子
3. 再删除房间本身
4. 工作台刷新

## 9. 管理后台流程

1. 管理员访问 `/admin/`
2. 输入管理员账号登录
3. 后台验证当前用户角色为 `admin`
4. 加载平台统计、用户列表、内容列表
5. 管理员可：
   - 搜索用户
   - 启用 / 禁用用户
   - 搜索卡片 / 箱子
   - 强制删除内容

## 10. 异常流程

### 10.1 卡片处理失败

1. 服务端把状态写为 `error`
2. 保存 `errorMessage`
3. 前台上传结果区显示失败状态
4. 用户回到工作台继续处理或删除该卡片

### 10.2 登录态失效

1. API 返回 401
2. 前端尝试 refresh
3. 若 refresh 失败，则清空内存 access token
4. 用户需重新登录

### 10.3 访问私有内容

1. 非所有者访问私有房间 / 卡片 / 箱子
2. 服务端按不可见资源处理
3. 前端表现为“未找到目标内容”
