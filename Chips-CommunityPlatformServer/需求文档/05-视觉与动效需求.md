# 视觉与动效需求

## 1. 总体气质

- 极简；
- 清爽；
- 干净；
- 优雅；
- 有轻微立体感；
- 有果味；
- 细节丝滑灵动。

## 2. 视觉层次

- 页面必须大量留白；
- 层级靠材质、阴影、模糊、透明度和节制的高光建立；
- 禁止堆叠过多边框、按钮、标签和面板。
- 个人页背景需要形成整页磨砂玻璃质感，彩色氛围光晕位于磨砂层之下缓慢漂动。

## 3. 作品封面表现

- 卡片与箱子封面都要有悬浮感；
- 封面本身是主角，文案不应压过封面；
- 每个封面允许轻微随机、小范围漂动。

## 4. 动效要求

### 4.1 常驻微动效

- 页面背景允许存在极淡、低饱和的七彩光晕缓慢漂动；
- 各色光晕的轨迹、节奏和大小要有差异，避免机械同步；
- 作品卡片默认存在极轻的漂浮动画；
- 漂动幅度必须小，不得造成眩晕；
- 不同作品的节奏和相位要有细微差别，避免机械同步。

### 4.2 悬停反馈

- 悬停时可轻微抬升、变亮、加深阴影；
- 反馈必须细腻，不可突然弹跳。

### 4.3 页面转场

- 页面切换、加载态消失、队列状态变化都需要平滑过渡；
- 不允许生硬闪切。

## 5. 头图区要求

- 头图区要像一条有质感的横向视觉带；
- 头像、昵称、简介的排布必须克制；
- 不能把头图区做成传统后台资料卡。
- 头图区本体保持透明，不额外加整块实体卡片底板；
- 本人可见的设置按钮要融入头图区上缘，像精致控件，不像后台操作条。

## 6. 主题、图标与语言要求

- 所有视觉样式必须走**社区服务器自身的 Web 主题系统**；
- 所有图标必须走**社区服务器自身的 Web 图标系统**；
- 所有文案必须走**社区服务器自身的 Web 多语言系统**；
- 三套系统与用户设备上的 Host 生态同源同理念，但实现、资源和运行时彼此独立；
- 不允许继续把旧版页面里的硬编码主题变量、内嵌词典和散落图标方案当作最终方案。

## 7. 降噪原则

- 不出现统计卡、发现入口、搜索入口、后台快捷操作；
- 不出现与主任务无关的分栏；
- 不出现为“丰富页面”而额外添加的视觉装饰模块。
