# 卡片与箱子产品规则

## 1. 卡片产品规则

### 1.1 输入

当前卡片上传输入为标准 `.card` 文件。

上传时可附带：

- `roomId?`
- `visibility?`

若指定 `roomId`，必须是当前用户自己的房间。

### 1.2 上传受理结果

卡片上传采用异步处理。

接口受理成功后：

- 立即创建卡片记录
- 返回 `cardId`
- 返回 `status = pending`

前端或客户端可通过 `GET /api/v1/cards/:cardId/status` 轮询处理进度。

### 1.3 正式处理链路

卡片处理的唯一正式链路是：

1. 解包 `.card`
2. 将卡片内本地资源上传到 CDN
3. 把卡片内资源 URL 全部替换为 CDN URL
4. 生成新的卡片目录内容
5. 删除旧的本地资源文件
6. 重打包为新的 `.card`
7. 删除原始上传包
8. 通过 Host 已安装的正式文件转换插件把新 `.card` 转为 HTML 目录
9. 将 HTML 目录上传到 CDN
10. 回写卡片记录为 `ready`

当前平台已经明确不允许保留第二条“服务端自行拼 HTML”的正式链路。

### 1.4 查看规则

卡片查看时，平台不再在转换结果外层再包一层查看器 UI。

当前 Web 前端行为是：

1. 打开 `/cards/:cardId`
2. 先读取卡片详情
3. 如果卡片 `status = ready` 且存在 `htmlUrl`
4. 直接跳转到转换后的 HTML 页面

因此用户最终看到的就是转换插件输出的 HTML 页面本身。

### 1.5 可见性规则

卡片可见性：

- `public`
- `private`

规则：

1. 私有卡片只对所有者可见
2. 公开房间中的私有卡片不会因为房间公开而自动公开
3. 房间公开页只会展示当前请求者可见的卡片

### 1.6 卡片管理动作

当前第一版卡片支持：

- 上传
- 查询详情
- 查询状态
- 修改可见性
- 移动到房间 / 根目录
- 删除

## 2. 箱子产品规则

### 2.1 输入

箱子上传输入为标准 `.box` 文件。

上传时可附带：

- `roomId?`
- `visibility?`

### 2.2 处理方式

箱子上传当前是同步入库，不走卡片那样的异步转换流水线。

系统会在服务端完成：

1. 校验 ZIP 魔数
2. 解包 `.box`
3. 读取 `.box/metadata.yaml`
4. 读取 `.box/structure.yaml`
5. 写入箱子记录

### 2.3 展示规则

箱子详情页当前展示：

- 标题
- 可见性
- 创建时间
- 文件大小
- 所有者
- 布局插件标识
- 箱子文件内 ID
- 引用卡片列表

当前第一版不做箱子在线布局渲染，只展示结构化信息与卡片引用。

### 2.4 社区映射规则

箱子中的卡片引用会尝试与社区内卡片做映射。

映射条件：

- `card_id` 相同
- 社区卡片 `status = ready`
- 社区卡片 `visibility = public`

命中后，箱子详情页会补充：

- `communityCardId`
- `communityHtmlUrl`

未命中时保留原始引用 URL。

### 2.5 可见性规则

箱子可见性：

- `public`
- `private`

规则：

1. 私有箱子只对所有者可见
2. 公开房间中的私有箱子不会自动公开
3. 房间公开页只会展示当前请求者可见的箱子

### 2.6 箱子管理动作

当前第一版箱子支持：

- 上传
- 查询详情
- 修改可见性
- 移动到房间 / 根目录
- 删除

## 3. 卡片与箱子的关键差异

| 维度 | 卡片 | 箱子 |
|---|---|---|
| 上传后是否异步处理 | 是 | 否 |
| 是否会重写内部资源 URL | 是 | 否 |
| 是否会重新打包新文件 | 是 | 否 |
| 是否会调用正式转换插件 | 是 | 否 |
| 是否生成 HTML 页面 | 是 | 否 |
| 是否有处理状态 | 是 | 否 |
| 是否会映射社区内其他内容 | 否 | 是 |

## 4. 当前范围外

当前第一版还不包含：

- 卡片版本历史
- 社区内卡片回下载为原生 `.card`
- 箱子在线布局渲染
- 箱子内卡片在线编辑
- 箱子封面自动生成
- 卡片 / 箱子评论、点赞、收藏
