# 非功能要求与验收

## 1. 性能

- 个人页首屏必须优先展示头图区与首批作品；
- 作品封面加载不能造成明显卡顿；
- 微动效不能明显拖慢滚动与悬停响应。

## 2. 可访问性

- 键盘可访问；
- 焦点态可见；
- 图片 / iframe / 装饰元素的语义必须清晰；
- 拖拽区域要有非拖拽的点击选文件回退路径。

## 3. 响应式

- 桌面端优先；
- 平板端保持头图区完整性；
- 移动端仍能自然阅读头图区和网格，不出现挤压失控。

## 4. 主题、图标与多语言

- 页面必须遵循社区服务器自身的 Web 主题、图标与多语言规范；
- 这三套系统与 Host 可保持设计理念一致，但不存在直接运行时依赖；
- 主题切换与语言切换后，界面必须同步刷新；
- 不允许因为重构进度而继续把旧版本地样式和旧版词典视为最终方案。

## 5. 稳定性

- 上传页必须正确区分卡片与箱子；
- 卡片轮询状态不能无限无反馈；
- 错误态必须可理解、可恢复。

## 6. 验收标准

满足以下条件才算前端 V2 需求收口：

1. 社区公开入口收敛到个人主页；
2. 欢迎页、社区简介页、登录页、注册页、个人主页、工作区与后台入口分流全部可用；
3. 本人主页显示“+”，他人主页不显示；
4. 工作区可拖入 `.card/.box` 并显示全过程状态；
5. 管理员账号登录后会自动切到后台，公开站点无显性后台按钮残留；
6. 页面无旧版总站式首页、发现、房间、工作台、后台式全局导航残留；
7. 作品网格具有轻微悬浮和漂动质感；
8. 主题、多语言、加载态、错误态、空态全部完整。
