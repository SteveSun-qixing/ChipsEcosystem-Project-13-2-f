# Chips-ComponentLibrary AGENTS

## 角色

- 本仓是薯片生态的 L10 无头组件层；负责结构、状态机、可访问性和主题接口点，不负责业务视觉皮肤。

## 开发规则

- 不要在组件里硬编码业务色板、阴影、圆角或主题模式；视觉必须通过 token、contract 和主题系统注入。
- 组件契约、`data-scope/data-part/data-state`、ARIA 语义和键盘交互优先于视觉便捷实现。
- 公共组件标准、token 对接规则、可访问性基线和显示链路标准属于生态公共信息，必须同步 `生态共用技术文档/组件库`。
- 不要在组件库中沉淀某个应用私有业务逻辑；应用级组合应留在应用项目内。
- 如果发现生态缺少正式组件、契约或主题接口点，先登记工单，不要在应用层长期保留私有替代物。

## 验证

- 常规改动优先运行 `npm run verify`。
- 变更涉及契约、a11y、性能或发布门禁时，再补跑 `npm run quality:gate` 及相关专项脚本。
