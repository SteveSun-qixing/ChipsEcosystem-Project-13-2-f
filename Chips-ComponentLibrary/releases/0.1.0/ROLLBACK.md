# 0.1.0 回滚说明

## 触发条件

- 发布后出现质量门禁失败（契约/测试/性能）
- 生态接入场景出现阻断性兼容问题
- 卡片显示链路在查看器或编辑器场景出现不可恢复故障

## 回滚步骤

1. 回滚到上一稳定标签（建议使用 `develop` 最近一次发布标签）。
2. 恢复上一版本发布包与对应文档索引。
3. 重新执行：
   - `npm run verify`
   - `npm run quality:gate`
4. 在 `项目日志与笔记/问题工单.md` 新增回滚原因工单并跟踪修复。

## 数据与文件策略

- 禁止直接删除发布期文件。
- 需移除的文件迁移到 `归档/` 并记录原路径。

## 重新发布前置

- 回滚缺陷完成修复并通过全量门禁。
- 更新变更说明和兼容性说明。
