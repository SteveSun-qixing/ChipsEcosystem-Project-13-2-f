# 阶段一：架构与契约冻结

## 阶段目标

建立组件库的不可变开发基线，冻结架构边界、主题权重规则、组件契约和包职责，作为后续所有开发的唯一依据。

## 开发任务

### 1. 架构边界冻结

- 明确L10组件库只负责结构、状态、a11y与契约挂点。
- 明确组件库不得直连Host服务，不得承载业务状态。
- 固化依赖方向：`tokens -> adapters -> primitives -> components`。

### 2. 主题规则冻结

- 固化主题权重链（低 -> 高）：`global -> app -> box -> composite-card -> base-card -> component`。
- 固化查找规则：高权重优先，缺失逐层向上回退。
- 固化主题接口动作：`theme.list/apply/getCurrent/getAllCss/resolve/contract.get`。

### 3. 组件契约冻结

- 统一契约字段：`data-scope/data-part/data-state/aria-*`。
- 统一基础状态集合：`idle/hover/focus/active/disabled/loading/error`。
- 固化受控/非受控组件命名规范（`value + onValueChange` 等）。

### 4. 产物模板冻结

- 组件契约JSON模板。
- token key 命名模板。
- 组件文档模板（API、状态、a11y、测试矩阵）。

## 阶段产物

- 架构冻结说明文档。
- 契约模板与示例文件。
- 主题权重规则与解析流程图（文档化）。

## 退出条件

- 全部核心规则在文档中唯一化，无冲突口径。
- 后续阶段可直接引用，不再重复决策。
- 若发现规范冲突，先提工单再进入下一阶段。
