# 阶段十迁移映射与批量替换工具说明

## 1. 目标

为旧项目迁移提供可重复执行的组件与 token 替换能力，避免手工替换带来的一致性风险。

## 2. 迁移资产

- 组件映射：`migrations/legacy-component-map.json`
- token 映射：`migrations/legacy-token-map.json`
- 组件迁移脚本：`scripts/migrate-legacy-components.mjs`
- token 迁移脚本：`scripts/migrate-legacy-tokens.mjs`

## 3. 运行模式

迁移脚本支持两种模式：

- `dry-run`（默认）：仅生成报告，不改文件
- `apply`：执行替换，并自动备份原文件

备份目录：`<target>/归档/迁移备份/<timestamp>/`。

## 4. 命令示例

### 4.1 组件迁移

```bash
npm run migrate:components -- --target ../旧项目目录
npm run migrate:components -- --target ../旧项目目录 --apply
```

### 4.2 token 迁移

```bash
npm run migrate:tokens -- --target ../旧项目目录
npm run migrate:tokens -- --target ../旧项目目录 --apply
```

可选参数：

- `--extensions .ts,.tsx,.jsx,.js,.css`
- `--include-archive`（默认不扫描 `归档/`）

## 5. 报告规范

报告目录：`reports/migration/`

- `component-migration-latest.json`
- `token-migration-latest.json`

模板：

- `reports/migration/component-migration-template.json`
- `reports/migration/token-migration-template.json`

报告字段包含：扫描文件数、命中文件数、替换总数、规则统计、文件明细。

## 6. 迁移验收清单

- [ ] 组件替换报告中无未识别核心基础组件
- [ ] token 替换报告中旧命名空间替换完成
- [ ] `data-scope/data-part/data-state` 契约仍完整
- [ ] `npm run verify` 与 `npm run quality:gate` 全通过
- [ ] 主题切换、语言切换、配置切换回归通过

## 7. 风险控制

- 默认先执行 `dry-run`，审查报告后再 `--apply`。
- `--apply` 模式必须保留归档备份，不允许覆盖后不可恢复。
- 如映射规则不足，先更新映射文件并追加测试后再迁移。
