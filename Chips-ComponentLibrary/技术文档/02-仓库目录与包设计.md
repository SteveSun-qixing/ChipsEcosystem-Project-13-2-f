# 仓库目录与包设计

## 1. Monorepo 结构

```text
Chips-ComponentLibrary/
  packages/
    tokens/                 # token 源文件、Style Dictionary 配置、构建产物
    theme-contracts/        # 组件主题契约定义与校验器
    primitives/             # Ark UI 原语封装
    components/             # 业务语义组件
    card-runtime/           # CardCoverFrame 与 CompositeCardWindow 等iframe高级组件
    hooks/                  # 组件层 hooks（不直连 Host）
    a11y/                   # 可访问性工具、语义检查
    testing/                # 测试工具、测试辅助、契约测试 runner
    adapters/
      tamagui-core/         # Tamagui Core token/runtime 适配
  scripts/                  # 构建、验证、发布脚本
  技术文档/
```

根 `workspace` 覆盖规则：

- `packages/*`
- `packages/adapters/*`

## 2. 包职责与依赖方向

- `tokens`：只负责 token 定义、构建、导出，不依赖 UI 包
- `theme-contracts`：定义和校验契约，不依赖组件实现
- `primitives`：仅封装 Ark UI 原语，可依赖 `tokens` 类型
- `components`：组合 `primitives`，实现语义组件，可依赖 `hooks` 和 `a11y`
- `card-runtime`：封装卡片显示 iframe 高级组件，依赖 `components/hooks` 与 SDK 显示接口契约
- `adapters/tamagui-core`：将 token 导出为 Tamagui Core 可消费结构
- `testing`：独立测试工具，不被业务包反向依赖

允许方向：`tokens -> adapters -> primitives -> components`。
禁止方向：`components -> tokens 源定义`、`components -> Host SDK 具体实现`。

## 3. 命名与文件规范

- 包名：`@chips/<package-name>`
- 文件命名：kebab-case
- 组件导出：PascalCase
- 内部纯类型模块：`*.types.ts`
- 对外发布声明文件：`*.d.ts`
- 契约文件：`*.contract.json`
- 测试文件：`*.test.mjs`（当前基线）/ `*.test.ts`（后续 TS 化阶段）

## 4. 变更边界控制

- 组件 API 变更必须同步更新契约文档和升级说明
- token key 变更必须提供映射表和变更脚本
- 任何破坏性变更必须走 major 版本策略
- 禁止跨包直接复制逻辑，公共能力必须上提到独立包
- 聚合包如遇跨子包同名导出冲突，必须在聚合层显式命名并补充测试，不允许依赖 `export *` 的歧义行为

## 5. 文档分层存储规则

- 生态可公开标准：存放到 `生态共用技术文档/组件库`
- 仓库内部实现细节：存放到本仓 `技术文档`
- 任务执行记录：写入 `项目日志与笔记`（按工单规则）
