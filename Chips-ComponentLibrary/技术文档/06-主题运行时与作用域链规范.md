# 主题运行时与作用域链规范

## 1. 作用域链定义

组件库遵循统一主题作用域链：

`global -> app -> box -> composite-card -> base-card -> component`

权重从低到高：越靠右权重越高，最小组件（`component`）权重最高。

## 2. 解析算法

```text
resolvedToken = component || base-card || composite-card || box || app || global || fallback
```

查找规则：先按高权重层查找（component），若当前层未定义则逐层向上回退到更低权重层（base-card -> composite-card -> box -> app -> global）。
每个 token key 单独解析，不做整包强覆盖。

## 3. CSS 变量注入策略

- 全局主题注入在 `:root`
- 应用级注入在 `[data-chips-app]`
- 箱子级注入在 `[data-chips-box]`
- 复合卡片级注入在 `[data-chips-composite-card]`
- 基础卡片级注入在 `[data-chips-base-card]`
- 组件级注入在 `[data-chips-component]`

组件层只消费变量，不负责变量生成。

## 4. 运行时事件

主题变更统一发布 `theme.changed`：

```ts
interface ThemeChangedEvent {
  previousThemeId: string;
  themeId: string;
  timestamp: number;
}
```

组件库只订阅事件并最小化刷新，不自行管理主题状态源。

## 5. 回退与容错

- token 缺失：记录告警并回退默认主题
- 主题包解析失败：回退系统默认主题并输出错误码 `THEME_LOAD_FAILED`
- 主题契约失败：阻断应用主题，错误码 `THEME_CONTRACT_INVALID`

## 6. 性能策略

- 主题切换优先变量更新，避免重建组件树
- 大范围主题变更采用分片注入
- 主题 CSS 缓存按 `themeId + version` 建立

目标：主题切换可见完成耗时 `<= 150ms`。

## 7. 当前实现（2026-03-04）

### 7.1 作用域解析器

- 实现位置：`packages/adapters/tamagui-core/src/index.js`
- 统一链路：`global -> app -> box -> composite-card -> base-card -> component`（低到高）
- 查找顺序：`component -> base-card -> composite-card -> box -> app -> global -> fallback`
- 缺失行为：抛出 `THEME_TOKEN_MISSING`

### 7.2 运行时 Provider

- 实现位置：`packages/hooks/src/index.js`
- `ChipsThemeProvider` 维护：
  - `themeId`
  - `version`
  - `cacheKey`
  - `lastChangedAt`
- 订阅事件：默认 `theme.changed`
- 事件源兼容：
  - `on/off`
  - `addEventListener/removeEventListener`
  - `subscribe`

### 7.3 变量级刷新

- 通过 `applyThemeVariables(target, variables)` 将 token 写入 CSS 变量。
- 组件结构不依赖重建；主题切换以变量更新为主。
- 大范围更新通过 `applyThemeVariablesInBatches(target, variables, options)` 分片注入。
- 分片注入支持：
  - `chunkSize`
  - `onChunkApplied`
  - `onDiagnostic`
  - `signal` 中断控制

### 7.4 诊断与错误码

- token 缺失错误码：`THEME_TOKEN_MISSING`
- 分片注入中断错误码：`THEME_VARIABLE_APPLY_ABORTED`
- 诊断事件：
  - `THEME_TOKEN_FALLBACK_APPLIED`
  - `THEME_TOKEN_MISSING`
  - `THEME_VARIABLE_CHUNK_APPLIED`
