# 组件契约与状态模型规范

## 1. 必备契约字段

每个组件必须公开以下契约挂点：

- `data-scope`：组件作用域（如 `button`）
- `data-part`：组件部件（如 `root`、`label`、`icon`）
- `data-state`：组件状态
- `aria-*`：可访问性语义

## 2. 标准状态集合

交互组件统一状态：

`idle | hover | focus | active | disabled | loading | error`

扩展状态必须在组件文档声明，不允许隐式新增。

## 3. 状态优先级

`disabled > loading > error > active > focus > hover > idle`

当多状态冲突时按优先级裁决。

## 4. API 规范

- 所有可控状态组件必须支持受控与非受控模式
- 受控属性命名：`value/open/checked` + `onValueChange/onOpenChange/onCheckedChange`
- 禁止同时暴露语义重复属性（如 `isOpen` 和 `open`）

## 5. 可访问性基线

- 组件文档必须列出 role 和 aria 关系
- 键盘交互路径必须覆盖 Tab / Shift+Tab / Enter / Space / Escape
- 错误提示区域必须支持 `aria-live`
- 图标按钮必须满足最小热区 44px

## 6. i18n 与配置约束

- 用户可见文案必须通过 i18n key 提供
- 行为阈值（如延迟、分页阈值）必须支持配置化
- 默认配置与运行时覆盖必须可追踪

## 7. 契约文件示例

```json
{
  "component": "button",
  "scope": "button",
  "parts": ["root", "label", "icon", "spinner"],
  "states": ["idle", "hover", "focus", "active", "disabled", "loading", "error"],
  "tokens": [
    "comp.button.height.md",
    "comp.button.radius",
    "comp.button.color.primary.bg",
    "comp.button.color.primary.text"
  ]
}
```

## 8. iframe 高级组件补充

针对 `CardCoverFrame`、`CompositeCardWindow` 这类 iframe 高级组件，除标准契约外必须补充：

- iframe 安全参数声明（sandbox、allow）
- frame 生命周期事件（ready/error）
- 降级状态说明（如 `degraded`）
- 跨 iframe 通信边界说明（禁止越权直连）
- 消息来源过滤说明（source window + origin 白名单）

## 9. Primitive 层落地补充

当前 Primitive 层标准 `scope`：

- 布局：`box`、`inline`、`stack`、`grid`
- 文本：`text`、`label`、`helper-text`

`HelperText` 默认语义：

- `role="status"`
- `aria-live` 按 `tone` 推导（`error -> assertive`，其他 -> `polite`）
