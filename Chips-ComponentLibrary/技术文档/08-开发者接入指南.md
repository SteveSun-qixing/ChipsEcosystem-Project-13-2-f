# 开发者接入指南

## 1. 适用对象

- 应用插件开发者（app）
- 卡片插件开发者（card）
- 布局插件开发者（layout）

## 2. 接入前提

- 插件工程已接入 React
- 已具备 `window.chips.*` 能力访问（通过 SDK 或 Bridge）
- 已加载至少一个可用主题包

## 3. 基础接入步骤

1. 引入组件库 Provider 与主题上下文
2. 在页面根节点挂载主题作用域标识（如 `data-chips-app`）
3. 使用组件库组件替换自建基础控件
4. 使用标准 hooks 接入主题/i18n/config
5. 完成契约测试与 a11y 验证

## 4. 示例

```tsx
import { ChipsThemeProvider, ChipsButton, ChipsInput, useToken } from '@chips/component-library';

function SettingsContent() {
  const surface = useToken('chips.sys.color.surface');

  return (
    <section data-chips-app="chips.settings" style={{ background: `var(--chips-sys-color-surface, ${surface})` }}>
      <ChipsInput placeholderKey="i18n.settings.name.placeholder" />
      <ChipsButton variant="primary">保存</ChipsButton>
    </section>
  );
}

export function SettingsPage() {
  return (
    <ChipsThemeProvider themeId="chips-official.default-theme" version="1.0.0">
      <SettingsContent />
    </ChipsThemeProvider>
  );
}
```

## 4.1 主题事件接入示例

```tsx
import { ChipsThemeProvider } from '@chips/component-library';

export function App({ resolver }) {
  return (
    <ChipsThemeProvider
      resolver={resolver}
      themeId="chips-official.default-theme"
      version="1.0.0"
      eventSource={window.chips}
      eventName="theme.changed"
    >
      <MainPage />
    </ChipsThemeProvider>
  );
}
```

## 5. 常见误区

- 误区：在业务层写组件颜色、圆角、阴影
  - 正解：全部通过主题 token 处理
- 误区：直接依赖 Ark UI 并绕开组件库
  - 正解：统一使用组件库封装层，避免契约漂移
- 误区：组件内写死中文文案
  - 正解：通过 i18n key 或外部传入文案
- 误区：在组件内写死 `+/-/x` 等控制字符
  - 正解：使用 `iconContent` / `expandIconContent` / `collapseIconContent` / `closeIconContent` 这类图标槽位

## 6. 主题权重补充

- 统一权重链（低 -> 高）：`global -> app -> box -> composite-card -> base-card -> component`
- 解析顺序：从 `component` 开始，缺失逐层向上回退到 `global`

## 7. 卡片显示组件接入补充

- 封面场景统一使用 `CardCoverFrame`，由 SDK 提供封面 iframe 地址。
- 复合卡片展示统一使用 `CompositeCardWindow`，只传入卡片文件（`mode` 仅 `view | preview`）。
- 复合卡片消息事件建议传入 `allowedOrigins`，按 origin 白名单过滤 iframe 消息来源。
- 调用方不重复实现基础卡片分发、模板编译和 iframe 拼接。

## 8. 接入验收清单

- [ ] 组件无视觉硬编码
- [ ] 文案可国际化
- [ ] data-scope/data-part/data-state 完整
- [ ] 键盘路径可用
- [ ] 主题切换无结构错乱
- [ ] 关键交互测试通过
- [ ] CardCoverFrame 与 CompositeCardWindow 链路测试通过（如涉及卡片显示）
