# 测试策略与质量门禁

## 1. 测试分层

1. 单元测试：状态机、hooks、工具函数
2. 组件测试：交互、键盘路径、状态投影、错误回调
3. 契约测试：组件契约、theme contract、token schema
4. 集成测试：组件 + 主题 + i18n + 配置联动
5. 性能测试：渲染耗时、主题切换耗时、长任务占比
6. 鲁棒性测试：异常注入、回退策略、边界输入

## 2. 最低门槛

- 每个组件至少 5 个测试用例
- 关键服务适配模块至少 8 个测试用例
- 契约测试通过率 100%
- 关键路径（正常/异常/边界）必须覆盖

## 3. CI 阻断规则

以下任一失败则禁止合入：

- lint 失败
- type-check 失败
- 单元或契约测试失败
- token/theme 契约校验失败
- a11y 基线校验失败
- 性能预算超阈值

当前阶段已落地并强制执行的门禁入口：

- `npm run verify`：token 校验、构建、主题契约校验、全量测试
- `npm run quality:gate`：`lint/typecheck/test:types/test/test:contracts/test:a11y/test:perf/build` 聚合阻断

`quality:gate` 规则：任一步骤失败后，后续步骤标记 `skipped` 并直接返回失败。

## 4. 性能预算

- 组件首次可交互（页面级）<= 2000ms
- 主题切换可见完成 <= 150ms
- 单次渲染提交 p95 <= 32ms
- 长任务占比 <= 5%

## 5. 发布前检查

- `npm run lint`
- `npm run typecheck`
- `npm run test:types`
- `npm run validate:tokens`
- `npm run build:tokens`
- `npm run validate:contracts`
- `npm test`
- `npm run test:contracts`
- `npm run test:a11y`
- `npm run test:perf`
- `npm run quality:gate`
- `npm run verify`

全部通过方可进入发布流程。

## 6. 结构化报告

- 质量门禁报告：`reports/quality-gate/quality-gate-latest.json`
- 性能基线报告：`reports/perf/perf-stage9-latest.json`
- 报告模板：`reports/quality-gate/report-template.json`

门禁报告字段：

- `generatedAt`
- `totalDurationMs`
- `status`
- `steps[].name/command/status/exitCode/durationMs`

## 7. Card Runtime 组件专项门禁

- iframe 沙箱策略测试必须通过
- 基础卡片节点失败时复合窗口降级测试必须通过
- SDK 显示接口一致性测试（查看器/编辑器）必须通过
- 聚合包类型冒烟测试必须覆盖 `@chips/component-library` 与 `@chips/card-runtime` 两条消费路径
