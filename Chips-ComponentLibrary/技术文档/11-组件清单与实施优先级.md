# 组件清单与实施优先级

## 1. 分级策略

- P0：生态关键路径组件，必须优先落地
- P1：高频业务组件，P0 稳定后实施
- P2：增强组件，按业务窗口实施

- Checkbox（已落地）
- Radio（已落地）
- Switch（已落地）
- Select（已落地）
- Dialog（已落地）
- Popover（已落地）
- Tabs（已落地）
- Menu（已落地）
- Tooltip（已落地）

### Data & Form

### System UX

## 5. 单组件完成定义（DoD）

每个组件交付必须满足：

- [ ] API 文档完整
- [ ] 组件契约文件完整
- [ ] 主题 token 映射完整
- [ ] iframe安全策略通过（仅iframe组件）
- [ ] a11y 测试通过
- [ ] 单元测试 >= 5
- [ ] 集成测试覆盖关键路径
- [ ] 示例页面可运行
