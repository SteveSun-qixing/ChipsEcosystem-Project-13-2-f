# 卡片封面组件规范

## 1. 组件定位

`CardCoverFrame` 是薯片生态卡片封面标准组件，本质是一个封装好的 iframe 窗口：

- iframe 内显示卡片封面 HTML 文件
- iframe 下方显示卡片名称
- 作为卡片列表、资源管理器、选择器等场景的统一封面展示组件

## 2. 职责边界

组件负责：

- 展示封面 iframe
- 展示卡片名称
- 提供统一加载/失败状态
- 透出标准点击和错误事件

组件不负责：

- 解析 `.card` 文件
- 生成封面 HTML
- 业务导航逻辑

上游由 SDK 提供封面资源地址和元数据。

## 3. 接口草案

```ts
export interface CardCoverFrameProps {
  cardId: string;
  cardName: string;
  coverUrl: string;
  ratio?: '1:1' | '4:3' | '16:9';
  loading?: boolean;
  disabled?: boolean;
  sandbox?: string;
  onOpenCard?: (cardId: string) => void;
  onFrameReady?: (cardId: string) => void;
  onFrameError?: (error: StandardError) => void;
}
```

说明：

- `sandbox` 会经过组件安全策略归一化，默认至少包含 `allow-scripts allow-same-origin`。
- 禁止提升到危险权限：`allow-top-navigation`、`allow-top-navigation-by-user-activation`、`allow-popups-to-escape-sandbox`。

## 4. 结构契约

- `data-scope="card-cover-frame"`
- `data-part="root|frame-container|iframe|name|status"`
- `data-state="idle|loading|ready|error|disabled|empty"`

## 5. 行为规范

1. `loading=true` 时显示加载态，不触发打开事件。
2. iframe 加载成功触发 `onFrameReady`。
3. iframe 加载失败触发 `onFrameError`，并进入 `error` 状态。
4. 点击封面或名称触发 `onOpenCard(cardId)`。
5. `disabled=true` 时不可交互。

## 6. 安全规范

- iframe 必须启用 sandbox（最小权限原则）。
- 禁止在组件层透传任意不可信 URL。
- 禁止直接注入 HTML 字符串，统一使用 SDK 生成的可控地址。

## 7. 主题与 token

- 组件容器、标题、状态文本全部走 token。
- 组件必须支持 `component` 级主题覆盖。
- 主题权重链：`global -> app -> box -> composite-card -> base-card -> component`。

## 8. 测试验收

- 加载成功/失败路径测试
- disabled 交互阻断测试
- 标题展示与溢出处理测试
- 主题切换视觉一致性测试
- a11y 测试（键盘、读屏标签、焦点顺序）
