# 卡片封面组件规范（含统一文档 iframe 基组件）

## 1. 组件定位

`EmbeddedDocumentFrame` 是薯片生态统一的受控文档 iframe 基组件。  
`CardCoverFrame` 是其卡片封面专用包装组件。

当前正式职责拆分如下：

- `EmbeddedDocumentFrame`：
  - 承载受控 `src` 或受控 `srcDoc`
  - 提供统一加载/失败状态
  - 提供统一激活、加载完成和错误事件
- `CardCoverFrame`：
  - 固定消费卡片正式封面链路返回的 `coverUrl`
  - 作为卡片列表、资源管理器、选择器等场景的统一卡片封面展示组件

## 2. 职责边界

组件负责：

- 展示受控文档 iframe
- 提供统一加载/失败状态
- 透出标准激活和错误事件

组件不负责：

- 解析 `.card` 文件
- 解析 `.box` 文件
- 生成封面 HTML 或布局背景 HTML
- 业务导航逻辑

上游由 SDK 或当前应用受控逻辑提供资源地址和元数据。

## 3. 接口草案

```ts
export interface EmbeddedDocumentFrameProps {
  surfaceId?: string;
  title?: string;
  src?: string;
  srcDoc?: string;
  ratio?: string;
  loading?: boolean;
  disabled?: boolean;
  sandbox?: string;
  scope?: string;
  onActivate?: (surfaceId?: string) => void;
  onFrameReady?: (surfaceId?: string) => void;
  onFrameError?: (error: StandardError) => void;
}

export interface CardCoverFrameProps {
  cardId?: string;
  title?: string;
  coverUrl?: string;
  ratio?: string;
  loading?: boolean;
  disabled?: boolean;
  sandbox?: string;
  onOpenCard?: (cardId?: string) => void;
  onFrameReady?: (cardId?: string) => void;
  onFrameError?: (error: StandardError) => void;
}
```

说明：

- `EmbeddedDocumentFrame` 默认 `data-scope="embedded-document-frame"`，`CardCoverFrame` 固定使用 `data-scope="card-cover-frame"`。
- `sandbox` 会经过组件安全策略归一化，当前正式最小权限策略是 `allow-scripts`。
- 禁止提升到危险权限：`allow-top-navigation`、`allow-top-navigation-by-user-activation`、`allow-popups-to-escape-sandbox`。

## 4. 结构契约

- `data-scope="embedded-document-frame"`
- `data-scope="card-cover-frame"`
- `data-part="root|frame-container|iframe|status"`
- `data-state="idle|loading|ready|error|disabled|empty"`

## 5. 行为规范

1. `loading=true` 时显示加载态，不触发打开事件。
2. iframe 加载成功触发 `onFrameReady`。
3. iframe 加载失败触发 `onFrameError`，并进入 `error` 状态。
4. `EmbeddedDocumentFrame` 在提供 `onActivate` 时进入可交互状态；`CardCoverFrame` 点击后触发 `onOpenCard(cardId)`。
5. `disabled=true` 时不可交互。
6. 正式卡片封面链路只允许传入 `coverUrl`；箱子封面、箱子条目封面与应用内受控 iframe 区域优先使用 `EmbeddedDocumentFrame`。

## 6. 安全规范

- iframe 必须启用 sandbox（最小权限原则）。
- 禁止在组件层透传任意不可信 URL。
- `CardCoverFrame` 禁止直接注入 HTML 字符串，统一使用 SDK 生成的可控 `coverUrl`。
- `EmbeddedDocumentFrame` 仅允许消费当前应用内部生成或 Host 受控转换得到的 `srcDoc`。

## 7. 主题与 token

- 组件容器与状态文本全部走 token。
- 组件必须支持 `component` 级主题覆盖。
- 主题权重链：`global -> app -> box -> composite-card -> base-card -> component`。

## 8. 测试验收

- 加载成功/失败路径测试
- disabled 交互阻断测试
- 激活事件与空态测试
- 主题切换视觉一致性测试
- a11y 测试（键盘、读屏标签、焦点顺序）
