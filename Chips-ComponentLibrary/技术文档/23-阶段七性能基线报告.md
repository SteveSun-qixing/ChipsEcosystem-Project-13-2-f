# 阶段七性能基线报告

## 1. 目标与范围

本报告用于完成阶段七产物中的“性能测试基线报告”交付，覆盖 Data-Form 与 Workbench 核心算法路径：

- `computeVirtualWindow`
- `applyDataGridSort`
- `flattenTreeNodes`
- `filterCommandPaletteItems`
- `resolveDockPanelStateMap`
- `toggleInspectorSection`
- `clampSplitRatio`

## 2. 执行环境

- 日期：2026-03-05
- 机器：Apple Silicon（arm64）
- 系统：Darwin 24.6.0
- Node.js：v22.18.0
- npm：10.9.3
- 命令：`npm run bench:stage7`
- 脚本：`scripts/benchmark-stage7.mjs`

## 3. 测试数据规模与方法

- 每个用例预热 `50` 次，正式采样 `500` 次。
- 数据规模：
  - VirtualList：`itemCount=10000`
  - DataGrid 排序：`5000` 行
  - Tree 扁平化：`120 x 12 x 3` 分层节点
  - CommandPalette 过滤：`6000` 条命令
  - DockPanel 状态映射：`320` 面板
  - Inspector section 切换：`120` section
  - Split ratio 计算：`50000` 次裁剪调用
- 输出指标：`averageMs`、`p95Ms`、`maxMs`。

## 4. 基线结果

| 用例 | average (ms) | p95 (ms) | max (ms) | 结论 |
|---|---:|---:|---:|---|
| `computeVirtualWindow(10k)` | 0.0022 | 0.0042 | 0.1882 | 通过 |
| `applyDataGridSort(5k)` | 1.4662 | 6.6474 | 108.7477 | 通过 |
| `flattenTreeNodes(120x12x3)` | 0.7442 | 1.2538 | 66.6254 | 通过 |
| `filterCommandPaletteItems(6k)` | 10.2503 | 30.7025 | 251.9616 | 通过（接近阈值） |
| `resolveDockPanelStateMap(320)` | 0.1561 | 0.1818 | 15.3513 | 通过 |
| `toggleInspectorSection(120)` | 0.1252 | 0.1107 | 16.2211 | 通过 |
| `clampSplitRatio(50k calls)` | 0.5211 | 0.9232 | 20.0221 | 通过 |

## 5. 结果解读

- 以 `p95 <= 32ms` 为阶段门禁，全部用例通过。
- `filterCommandPaletteItems` 为阶段七最重路径（`p95 30.7025ms`），仍满足门禁，但应作为阶段八优化关注点。
- `max` 指标存在偶发尖峰（GC/调度抖动），当前不作为阻断依据，以 `p95` 作为主要发布门禁。

## 6. 后续优化建议（非阻断）

- 命令面板过滤可增加前缀索引与关键字缓存，降低高频输入场景波动。
- DataGrid 排序可在数据未变更时复用排序快照，减少重复排序开销。
