# 阶段八跨层集成与回退策略

## 1. 目标

建立 System-UX 组件在主题、i18n、config、观测链路上的统一集成口径，保证在生态各应用中行为一致、失败可回退、问题可追踪。

## 2. 主题集成

- 所有 Stage8 组件只输出结构契约，不写视觉值。
- 视觉全部通过 `chips.comp.<scope>.*` token 注入。
- 主题解析遵循权重链：
  - 低 -> 高：`global -> app -> box -> composite-card -> base-card -> component`
  - 查找策略：高权重优先，缺失逐层向上回退。

## 3. i18n 集成

- 组件文案通过 `resolveI18nText` 解析：
  - 优先读 `i18n` 适配器结果
  - key 缺失时回退默认模板文案
- 缺失 key 诊断事件：
  - `SYSTEM_UX_I18N_KEY_FALLBACK`

## 4. config 集成

- 组件行为阈值通过 `resolveConfigValue` 读取：
  - 支持函数型、`get(key)`、对象路径
- 典型配置键：
  - `systemUx.loadingBoundary.delayMs`
  - `systemUx.loadingBoundary.skeletonLines`
  - `systemUx.notification.maxVisible`
  - `systemUx.notification.defaultDurationMs`
  - `systemUx.toast.maxStack`
  - `systemUx.toast.defaultDurationMs`
- 配置缺失回退事件：
  - `SYSTEM_UX_CONFIG_FALLBACK`

## 5. 错误与观测

- 错误对象统一由 `toStandardError` 归一。
- 观测记录统一由 `createObservationRecord` 输出，字段固定：
  - `traceId`
  - `component`
  - `action`
  - `errorCode`
  - `durationMs`

## 6. 回归用例清单

- ErrorBoundary：子树抛错 -> 捕获 -> 重试恢复。
- LoadingBoundary：`loading=true` 且延迟后显示骨架，`loading=false` 恢复内容。
- Notification：队列上限、超时移除、手动关闭、动作触发。
- Toast：堆栈上限、超时关闭、手动关闭、投放位置。
- EmptyState：i18n key 命中/缺失双路径。
- Skeleton：多行渲染、状态语义与可访问播报。
