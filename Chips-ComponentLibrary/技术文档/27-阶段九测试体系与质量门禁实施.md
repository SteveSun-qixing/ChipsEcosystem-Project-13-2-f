# 阶段九测试体系与质量门禁实施

## 1. 目标

在阶段九建立可持续执行的统一质量门禁，保证组件库在“开发-验证-交付”链路上具备稳定阻断能力。

## 2. 门禁链路

统一门禁命令：`npm run quality:gate`

执行顺序与阻断规则：

1. `lint`（`scripts/run-lint.mjs`）
2. `typecheck`（`scripts/run-typecheck.mjs`）
3. `test`（`npm run test`）
4. `test:contracts`（契约专项）
5. `test:a11y`（`scripts/run-a11y-tests.mjs`）
6. `test:perf`（`scripts/run-perf-tests.mjs`）
7. `build`（`npm run build`）

任一步失败后，后续步骤标记为 `skipped`，门禁直接返回失败并阻断合入。

## 3. 测试框架增强

`@chips/testing` 已新增 Stage9 通用能力：

- `createComponentFixture`：统一组件契约 fixture 生成
- `assertAriaRole`：ARIA 语义断言
- `assertStatePriority`：状态优先级断言
- `createThemeFallbackFixture`：主题回退场景构造
- `resolveFallbackScopeValue`：作用域链回退断言
- `injectFault`：故障注入描述模型

对应验证文件：`packages/testing/tests/index.test.mjs`。

## 4. Stage9 故障注入覆盖

`packages/components/tests/robustness-stage9.test.mjs` 已覆盖：

- token 缺失：`THEME_TOKEN_MISSING`
- 主题作用域异常：按高权重优先失败后向上回退
- config 源异常：`SYSTEM_UX_CONFIG_SOURCE_ERROR` + 默认值回退
- config 解析异常：`SYSTEM_UX_CONFIG_PARSER_ERROR` + 默认值回退
- i18n 适配器异常：`SYSTEM_UX_I18N_ADAPTER_ERROR` + fallback 文案
- 事件源异常：订阅失败场景安全处理

## 5. 结构化报告规范

质量门禁报告目录：`reports/quality-gate/`

- 模板：`reports/quality-gate/report-template.json`
- 最新结果：`reports/quality-gate/quality-gate-latest.json`
- 历史结果：`quality-gate-<timestamp>.json`

字段规范：

- 顶层：`generatedAt`、`totalDurationMs`、`status`、`steps[]`
- 步骤：`name`、`command`、`status`、`exitCode`、`durationMs`
