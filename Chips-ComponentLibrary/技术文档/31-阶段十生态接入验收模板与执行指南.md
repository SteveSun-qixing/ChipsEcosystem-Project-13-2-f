# 阶段十生态接入验收模板与执行指南

## 1. 目标

统一设置面板、编辑器、查看器三类典型插件场景的接入验收口径，保证跨应用调用一致性。

## 2. 验收模板位置

- 模板：`reports/ecosystem/ecosystem-acceptance-template.json`
- 校验脚本：`scripts/validate-ecosystem-acceptance.mjs`
- 校验命令：`npm run validate:ecosystem-acceptance`

## 3. 场景与检查项

### 3.1 设置面板插件（settings-panel-plugin）

- 主题切换联动
- 语言切换联动
- 配置变更联动
- 质量门禁记录

### 3.2 卡片编辑器插件（card-editor-plugin）

- 主题切换联动
- 语言切换联动
- 配置变更联动
- SDK 显示适配接口可用性

### 3.3 卡片查看器插件（card-viewer-plugin）

- 主题切换联动
- 语言切换联动
- 配置变更联动
- SDK 显示适配接口可用性

## 4. 验收状态定义

统一状态：`pending` / `passed` / `failed` / `blocked`。

- `passed`：全部检查项通过
- `failed`：存在失败项
- `blocked`：受跨项目依赖阻断
- `pending`：尚未执行

1. 协作方完成阻断工单并回传版本信息。

3. 运行 `npm run validate:ecosystem-acceptance` 校验报告结构。
4. 将结果并入发布包与阶段十收口记录。
