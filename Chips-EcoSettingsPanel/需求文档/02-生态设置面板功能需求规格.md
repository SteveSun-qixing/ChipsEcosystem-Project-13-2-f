# 生态设置面板功能需求规格

## 1. 信息架构

首版左侧菜单固定提供以下一级入口：

| 菜单 ID | 菜单名称 | 目标 |
| --- | --- | --- |
| `themes` | 主题管理 | 管理已安装主题包，安装新主题，切换当前主题，卸载主题 |
| `languages` | 多语言 | 查看已安装语言并切换当前语言 |
| `app-plugins` | 应用插件 | 管理应用插件安装、启用、停用、卸载 |
| `component-gallery` | 组件展示 | 使用当前主题预览薯片组件库组件 |

菜单系统必须采用“注册表驱动”模式实现，后续增加新菜单时只新增配置和页面模块，不允许把菜单逻辑散落在多个组件中。

## 2. 主题管理

### FR-THEME-001 列表展示

- 页面左上显示标题“主题管理”。
- 页面右上显示“安装主题”主操作按钮。
- 中央区域显示已安装主题列表；列表项至少包含：主题名称、主题 ID、版本、安装状态、当前生效状态。
- 当前正在使用的主题必须有明确标识。
- 首版正式布局使用列表，不使用主题条目卡片墙布局。

### FR-THEME-002 安装主题

- 点击“安装主题”后，调用正式文件选择能力选择主题包。
- 列表区域必须支持拖拽主题包文件完成安装。
- 仅接受 Host 正式支持的主题安装输入（如 `.cpk`、插件目录、`manifest.yaml` 所在目录或清单文件路径）。
- 安装成功后立即刷新列表；安装失败时展示错误信息和失败原因。

### FR-THEME-003 切换主题

- 每个可用主题项提供“切换”动作。
- 每个主题项提供“详情”动作，点击后以正式弹窗显示完整主题信息。
- 切换成功后必须立即刷新当前主题标记，并驱动页面主题同步更新。
- 如果目标主题已经是当前主题，应禁用或弱化该动作，避免重复提交。

### FR-THEME-004 卸载主题

- 每个主题项提供“卸载”动作。
- 当前主题卸载前必须给出明确阻断提示或先要求切换到其他主题。
- 卸载成功后刷新列表与当前主题状态。

## 3. 多语言

### FR-I18N-001 语言列表

- 页面展示当前已安装语言列表。
- 每个语言项至少包含语言名称、语言代码、当前状态。
- 当前语言必须高亮显示。
- 首版正式布局使用列表，并为每个语言项提供“详情”弹窗入口。

### FR-I18N-002 切换语言

- 用户点击目标语言后立即切换语言。
- 切换后页面文案应通过正式 i18n 机制刷新，不允许整页重载。
- 页面必须监听 `language.changed` 事件，确保与 Host 运行时保持一致。

## 4. 应用插件

### FR-APPPLUGIN-001 列表展示

- 页面展示所有已安装 `type: app` 插件的列表。
- 列表项至少包含：名称、插件 ID、版本、启用状态、安装时间。
- 系统自身设置面板插件若出现在列表中，首版正式规则为“不允许在自身界面中停用或卸载”，并在界面和文档中明确标识该限制。
- 首版正式布局使用列表，不使用插件条目卡片墙布局；每个插件项提供“详情”弹窗入口。

### FR-APPPLUGIN-002 安装插件

- 支持按钮选择文件安装。
- 支持拖拽插件包到列表区域安装。
- 安装完成后刷新运行时列表。

### FR-APPPLUGIN-003 启用与停用

- 已安装但未启用的插件提供“启用”动作。
- 已启用插件提供“停用”动作。
- 操作成功后必须即时刷新列表状态。

### FR-APPPLUGIN-004 卸载

- 插件项提供“卸载”动作。
- 如果存在生态约束不允许卸载，必须在交互中给出明确说明，而不是静默失败。

### FR-APPPLUGIN-005 快捷方式治理

- 插件项提供应用快捷方式状态展示，至少包含“已创建 / 未创建”与目标位置（Windows 桌面 / macOS 启动台）。
- 插件项提供“创建快捷方式”“移除快捷方式”“显示位置”操作。
- 快捷方式治理必须通过 Host 正式接口完成，不允许前端直接创建 `.lnk`、`.app` 或其他平台文件。
- 快捷方式创建后，用户应可在 Host 静默待机时直接从快捷方式启动对应应用插件。

## 5. 组件展示

### FR-GALLERY-001 真实组件预览

- 页面以 Bento 布局展示薯片组件库当前正式导出的组件。
- 所有展示必须直接使用 `@chips/component-library` 中的真实组件，不允许复制样式或自造近似组件。
- 每个展示卡片需尽量覆盖默认、悬停、激活、禁用、错误等关键状态中的至少一种。

### FR-GALLERY-002 组件分组

页面至少按以下分组展示：

- P0 交互基础组件：`ChipsButton`、`ChipsInput`、`ChipsCheckbox`、`ChipsRadioGroup`、`ChipsSwitch`、`ChipsSelect`、`ChipsDialog`、`ChipsPopover`、`ChipsTabs`、`ChipsMenu`、`ChipsTooltip`
- P0 数据表单组件：`ChipsFormField`、`ChipsFormGroup`、`ChipsVirtualList`
- Stage 7 高级数据组件：`ChipsDataGrid`、`ChipsTree`、`ChipsDateTime`、`ChipsCommandPalette`
- Stage 7 工作台组件：`ChipsSplitPane`、`ChipsDockPanel`、`ChipsInspector`、`ChipsPanelHeader`、`ChipsCardShell`、`ChipsToolWindow`
- Stage 8 系统体验组件：`ChipsErrorBoundary`、`ChipsLoadingBoundary`、`ChipsNotification`、`ChipsToast`、`ChipsEmptyState`、`ChipsSkeleton`

### FR-GALLERY-003 主题验证

- 切换主题后，组件展示页必须自动刷新并呈现真实主题效果。
- 该页面作为主题验收基准之一，后续新增正式组件时必须同步纳入展示清单。

## 6. 通用交互要求

- 左侧菜单切换不能造成整个应用刷新或丢失当前运行时状态。
- 窄屏场景必须提供正式菜单切换控件；首版允许采用顶部切换器，不允许仅依赖被压缩的桌面侧边栏。
- 所有 destructive 操作必须有明确确认或风险提示。
- 异步处理中需呈现加载态，防止重复提交。
- 空列表、错误态、权限不足、文件格式错误等场景必须有正式 UI 与可读提示。
