# 任务002总览与决议

## 1. 任务目标

任务002《重构编辑引擎渲染模块》的目标是同时重构编辑引擎的“编辑态显示链路”和“编辑链路”：

- 显示侧不再直接消费复合卡片整体 iframe；
- 改为按基础卡片粒度使用各自渲染器，生成单卡 iframe，再由编辑引擎内部拼装为复合卡片窗口；
- 编辑侧改为编辑引擎内部自管的编辑器运行时。

本任务最终要达到的目标是：

- `Host + SDK` 保持“显示能力提供方”的定位，不直接修改其代码；
- 编辑引擎内部以“基础卡片 iframe 组合层”作为正式编辑态显示入口；
- 编辑链路改为由编辑引擎内部的各基础卡片编辑器负责；
- 编辑引擎新增基础卡片渲染装配层、编辑器注册中心、编辑器会话层、编辑器宿主层和预览同步层；
- 所有卡片数据仍以编辑引擎当前解包态 `.card/` 目录为唯一持久化来源；
- 不做兼容设计，不保留双轨正式路径，不使用临时编辑器兜底。

## 2. 已确认的任务级架构决议

2026-03-14，用户已明确本任务边界：

1. `Host + SDK` 的接口和组件改为显示能力提供方，当前不直接修改其代码。
2. 每个基础卡片直接使用自己的渲染器，渲染完成后交付单卡 iframe。
3. 编辑引擎负责把各基础卡片 iframe 拼装成复合卡片窗口。
4. 编辑链路由各种编辑器自己负责。
5. 当前只修改 `Chips-EditingEngine`，并在编辑引擎内部补齐需要的新模块。

这条决议是本任务的正式执行边界。

## 4. 设计基线

本任务设计遵循以下基线：

- React 组件架构采用组合模式和上下文分层，避免新的布尔状态膨胀；
- 编辑器宿主、会话层、预览层、窗口层职责拆开，避免相互穿透；
- 保持主题系统和多语言系统在编辑引擎内部统一消费，不新建第二套皮肤或文案机制；
- 编辑器草稿态与卡片持久态分层，保证输入流畅性和自动保存稳定性；
- 所有实现都以最终正式交付为目标，不保留占位实现。
