# 任务边界与非目标

## 1. 实施边界

本任务只在 `Chips-EditingEngine` 仓内落地，不修改以下仓库的正式代码：

- `Chips-Host`
- `Chips-SDK`
- 基础卡片插件仓
- 组件库仓

允许同步更新的内容仅限：

- 编辑引擎任务目录文档；
- 必要的问题工单记录；
- 如实现阶段确有需要，再由用户单独授权是否回写更上层生态文档。

## 2. 正式范围

本任务正式范围包括：

1. 编辑引擎本地编辑器运行时设计与实现；
2. 编辑器注册中心、编辑器会话层、编辑器宿主层；
3. 编辑面板与卡片数据层的适配；
4. 预览刷新链路与编辑活动协同；
5. 窗口状态、选中态、拖拽和画布交互适配；
6. 测试、文档与验收闭环。

## 3. 明确不做的事

本任务明确不做：

1. 不修改 `Host + SDK` 的复合卡片显示链路。
2. 不把编辑态显示重新塞回 `client.card.compositeWindow.render(...)` 作为正式主路径。
3. 不把编辑链路重新塞回 `SDK editorPanel`。
4. 不保留“本地基础卡片组合显示 + 复合 iframe 显示”双正式路径。
5. 不保留“本地编辑器 + SDK 编辑器”双正式路径。
6. 不为了兼容旧路线而继续保留 `PluginHost` iframe 编辑器宿主作为正式方案。
7. 不做大版本兼容设计，不做过渡适配层。

## 4. 非目标

以下内容不属于本任务目标：

- 新增 Host 渲染动作；
- 新增 SDK 编辑接口；
- 新增 Host 复合卡片整体 iframe 适配层；
- 设计跨仓统一的编辑器插件协议；
- 改造查看器；
- 改造生态主题运行时或多语言运行时的底层实现。

## 5. 交付判断标准

只有当以下条件全部成立时，任务才算完成：

1. 编辑引擎本地编辑器链路可以独立承担正式编辑职责；
2. 编辑引擎可按基础卡片粒度完成 iframe 渲染与复合拼装；
3. 编辑数据、预览刷新、自动保存、窗口交互和测试门禁全部闭环；
4. 代码中不保留占位编辑器、临时兜底和未完成路径。
