# 开发总计划

## 阶段一：现状收口与契约冻结

目标：

- 把当前编辑引擎“基础卡片显示装配层”和“编辑链路”边界彻底分开；
- 冻结任务内正式边界：显示走“基础卡片渲染器 -> 单卡 iframe -> 编辑引擎拼装”，编辑走本地运行时；
- 明确当前支持的基础卡片类型和首批必须交付的正式编辑器。

输出物：

- 任务文档全集；
- 编辑器运行时对外接口草案；
- 首批卡片类型支持矩阵。

## 阶段二：基础卡片显示装配层与编辑器运行时骨架

目标：

- 新增基础卡片渲染注册中心；
- 新增单卡 iframe 渲染宿主；
- 新增复合卡片拼装层；
- 新增编辑器注册中心；
- 新增编辑器会话与草稿状态层；
- 新增本地编辑器宿主组件；
- 让 `CardWindow` 摆脱 `client.card.compositeWindow.render(...)` 依赖；
- 让编辑面板摆脱 `SDK editorPanel` 依赖。

输出物：

- `basecard-runtime` 目录骨架；
- `editor-runtime` 目录骨架；
- 本地 `CompositePreviewHost`；
- 本地 `EditorHost`；
- `CardWindow / EditPanel` 接入运行时；
- 与 `CardContext / CardService` 的基础连通。

## 阶段三：首批正式编辑器落地

目标：

- 把当前真实需要支持的基础卡片类型改造成正式本地编辑器；
- 完成字段编辑、校验、草稿同步和配置提交；
- 禁止继续使用通用占位编辑器作为正式入口。

优先顺序：

1. `RichTextCard / base.richtext`
2. 当前编辑引擎卡箱库中默认可创建且会进入主流程的其他基础卡片类型

输出物：

- 各卡片类型独立编辑器模块；
- 卡片类型到编辑器描述符的注册映射；
- 输入校验与默认值策略。

## 阶段四：预览同步与窗口协同

目标：

- 让 `CardWindow` 使用编辑引擎内部的基础卡片 iframe 拼装层；
- 本地编辑器修改数据后，通过 `CardService + EventEmitter` 驱动预览刷新；
- 优化编辑中高频更新下的刷新节流、窗口尺寸和选中同步。

输出物：

- 编辑活动与预览刷新协同策略；
- 窗口状态与编辑态交互规约；
- 复合卡片预览稳定性验证。

## 阶段五：全模块适配

目标：

- 适配文件管理器、卡箱库、无限画布、设置面板、卡片创建流程；
- 统一新卡片初始化和已有卡片编辑入口；
- 清理旧宿主逻辑和不再使用的耦合代码。

输出物：

- 模块改造清单全部闭环；
- 冗余依赖与旧路径移除；
- 文档和测试同步完成。

## 阶段六：质量门禁与验收

目标：

- 完成单元测试、集成测试、关键交互回归；
- 通过 lint、TypeScript、test、validate；
- 输出最终验收清单。

验收门禁：

- `npm run lint`
- `npx tsc --noEmit`
- `npm test`
- `npm run validate`

如果门禁因既有工程基线问题失败，且失败范围超出本任务改动边界，则需单独登记工单并说明是否纳入本任务收口。
