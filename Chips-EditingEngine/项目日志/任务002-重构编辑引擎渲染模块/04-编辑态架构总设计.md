# 编辑态架构总设计

## 1. 总体原则

当前已经落地的编辑引擎采用“双运行时、单数据源”架构：

- 显示链路：按基础卡片粒度调用各自渲染器，生成单卡 iframe，再由编辑引擎完成复合拼装；
- 编辑链路：由编辑引擎本地 `EditorHost` 直接托管基础卡片编辑器导出；
- 数据源：仍以 `CardService` 中的解包态卡片数据为唯一真实来源。

## 2. 当前实际分层

### A. Basecard Runtime Layer

职责：

- 维护基础卡片统一描述符契约；
- 负责单卡 iframe 渲染；
- 负责复合卡片节点装配、排序、选中和高度汇总。

当前实际文件：

- `src/basecard-runtime/contracts.ts`
- `src/basecard-runtime/registry.ts`
- `src/basecard-runtime/registrations/richtext.ts`
- `src/basecard-runtime/theme-css.ts`
- `src/basecard-runtime/frame-host.tsx`
- `src/basecard-runtime/CompositeCardAssembler.tsx`

说明：

- `registry.ts` 是当前唯一正式注册中心；
- `base.richtext` 是当前首个正式接入类型，但它只是描述符注册项，不是主流程特判；
- `CompositeCardAssembler` 当前不再额外包裹边框、圆角和阴影，基础卡片之间保持无感边缘。

### B. Editing Domain Layer

职责：

- 管理解包态 `.card/` 数据；
- 管理当前选中卡片与基础卡片；
- 负责自动保存、持久化队列和事件广播。

核心对象：

- `CardService`
- `CardContext`
- `globalEventEmitter`

说明：

- `CardService.createCard/openCard/addBasicCard/updateBasicCard` 统一通过描述符做 `cardType` 与 `data` 归一；
- 默认布局参数当前固定为 `padding: 16`、`gap: 12`；
- `updateBasicCard` 后会执行 `notify()`、进入持久化队列，并发出 `card:basic-card-updated`。

### C. Editor Runtime Layer

职责：

- 以 `cardId + baseCardId` 维护编辑会话；
- 保存草稿、校验状态、错误状态和提交节流；
- 直接把基础卡片编辑器导出挂载到编辑面板容器。

当前实际文件：

- `src/editor-runtime/contracts.ts`
- `src/editor-runtime/session-store.ts`
- `src/editor-runtime/context.tsx`
- `src/editor-runtime/EditorHost.tsx`

说明：

- 当前没有单独的 `EditorRegistry`；
- 查看态与编辑态都复用同一份 `BasecardDescriptor`；
- `EditorRuntimeProvider` 已在 `src/context/index.tsx` 中接入全局应用 Provider 树。

### D. Shell / UI Layer

职责：

- 托管窗口、画布和工具面板；
- 把显示运行时和编辑运行时连接起来；
- 处理无限画布的平移、缩放与窗口态切换。

核心组件：

- `CardWindow`
- `EditPanel`
- `PluginHost`
- `AppProviders`

说明：

- `CardWindow` 现在直接消费 `CompositeCardAssembler`；
- `PluginHost` 现在只是 `EditorHost` 的薄包装；
- `App.tsx` 创建初始基础卡片时会先做 `cardType` 归一与默认配置生成。

## 3. 数据责任划分

### CardService

负责：

- 打开和维护 `.card/` 解包态数据；
- 在内存模型中维护 `CompositeCard.structure.basicCards[*].data`；
- 自动保存到磁盘；
- 发出卡片状态、基础卡片更新和持久化事件。

### BasecardDescriptor

负责：

- 创建基础卡片默认配置；
- 归一化配置；
- 校验配置；
- 暴露 `renderView/renderEditor` 两类宿主入口。

### EditorSessionStore

负责：

- 保存当前基础卡片的本地草稿；
- 处理高频输入、防抖提交、校验状态；
- 在切换节点或卸载时 flush 最后草稿。

### CardWindow

负责：

- 消费编辑引擎内部基础卡片装配显示链路；
- 汇总预览高度、加载态和错误态；
- 管理窗口态、选中态与画布交互代理。

## 4. 当前关键实现约束

- 不再保留 `SDK compositeWindow` 作为编辑态复合卡片正式渲染入口；
- 不再保留 `SDK editorPanel` 作为编辑面板正式主入口；
- 不在主流程里为富文本或其他单一卡片类型保留硬编码分支；
- 不引入通用占位编辑器作为正式兜底；
- 不在运行时扫描插件目录，所有正式接入都由构建期静态注册决定。

## 5. 性能与稳定性原则

- 输入流和提交流分离，避免每次敲字都触发整窗重建；
- 单卡 iframe 只在真实渲染输入变化时重建；
- `BasecardFrameHost` 通过稳定回调引用避免父层重渲染导致的反复 loading；
- 编辑面板只更新当前会话，不把高频输入扩散到全局窗口树；
- 所有文件写入都收口在 `CardService`，不在渲染阶段做 IO。
