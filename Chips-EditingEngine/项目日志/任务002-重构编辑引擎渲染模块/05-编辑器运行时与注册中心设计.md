# 编辑器运行时与注册中心设计

## 1. 目标

把当前“复合 iframe 显示 + SDK editorPanel iframe 编辑”的模式，替换为：

- 显示：`BasecardFrameHost -> 基础卡片渲染器 -> 单卡 iframe`
- 编辑：`EditorHost -> 本地编辑器描述符 -> 本地编辑器组件`

## 2. 建议新增目录

建议在 `src/` 下新增：

- `basecard-runtime/contracts.ts`
- `basecard-runtime/registry.ts`
- `basecard-runtime/frame-host.tsx`
- `basecard-runtime/CompositeCardAssembler.tsx`
- `editor-runtime/contracts.ts`
- `editor-runtime/registry.ts`
- `editor-runtime/context.tsx`
- `editor-runtime/session-store.ts`
- `editor-runtime/EditorHost.tsx`
- `editors/shared/`
- `editors/rich-text/`
- 其他正式支持卡片类型的编辑器目录

## 3. 核心契约

建议定义以下核心接口：

```ts
export interface EditorDescriptor {
  cardType: string;
  displayName: string;
  createInitialDraft(source: Record<string, unknown>): Record<string, unknown>;
  normalizeDraft(draft: Record<string, unknown>): Record<string, unknown>;
  validateDraft(draft: Record<string, unknown>): EditorValidationResult;
  component: React.ComponentType<EditorComponentProps>;
}

export interface EditorComponentProps {
  cardId: string;
  baseCardId: string;
  cardType: string;
  sourceConfig: Record<string, unknown>;
  draftConfig: Record<string, unknown>;
  setDraftConfig: (next: Record<string, unknown>) => void;
  validation: EditorValidationResult;
  commit: () => void;
}

export interface EditorValidationResult {
  valid: boolean;
  errors: Record<string, string>;
}
```

## 4. 注册中心职责

`EditorRegistry` 负责：

- 维护 `cardType -> EditorDescriptor` 映射；
- 暴露查询、注册、断言能力；
- 保证编辑器解析逻辑统一，避免散落在 `EditPanel`、`PluginHost`、`CardContext` 中。

当前阶段建议使用静态注册，不做动态插件发现，不引入额外兼容层。

## 5. 会话层职责

`EditorSessionStore` 负责：

- 以 `cardId + baseCardId` 为 key 维护当前编辑会话；
- 保持源配置、草稿、dirty、valid、errors、lastCommittedAt；
- 控制提交节流与销毁时 flush。

会话层是唯一知道“草稿如何同步回卡片模型”的地方，编辑器组件本身不直接调用 `CardService`。

## 6. 宿主层职责

`EditorHost` 负责：

- 接收当前 `activeCardId + selectedBaseCardId`；
- 从 `CardContext` 获取源配置；
- 从 `EditorRegistry` 解析描述符；
- 从 `EditorSessionStore` 取得会话；
- 渲染对应编辑器组件；
- 在切换基础卡片时回收旧会话或提交最后草稿。

## 7. 旧模块处理原则

- `CardWindow.tsx` 不再把 `client.card.compositeWindow.render(...)` 作为正式主渲染入口；
- 复合卡片显示改为由编辑引擎内部拼装多个基础卡片 iframe；
- `PluginHost.tsx` 不再承担正式编辑器 iframe 宿主职责；
- 若保留该文件名，内部实现也必须彻底切成本地运行时；
- `DefaultEditor` 不再作为正式路径兜底，只能保留为开发诊断工具，或在实现阶段直接移除。
