# 编辑器运行时与注册中心设计

## 1. 已落地目标

当前已经把原来的“复合 iframe 显示 + SDK editorPanel iframe 编辑”模式，替换为：

- 显示：`BasecardDescriptor.renderView -> BasecardFrameHost -> 单卡 iframe`
- 编辑：`BasecardDescriptor.renderEditor -> EditorHost -> 编辑面板容器`

## 2. 当前实际目录

`src/` 下当前正式结构如下：

- `basecard-runtime/contracts.ts`
- `basecard-runtime/registry.ts`
- `basecard-runtime/registrations/richtext.ts`
- `basecard-runtime/frame-host.tsx`
- `basecard-runtime/CompositeCardAssembler.tsx`
- `basecard-runtime/theme-css.ts`
- `editor-runtime/contracts.ts`
- `editor-runtime/context.tsx`
- `editor-runtime/session-store.ts`
- `editor-runtime/EditorHost.tsx`

补充：

- 当前没有单独的 `editor-runtime/registry.ts`；
- 当前没有单独的 `editors/*` 目录；
- 查看态与编辑态统一由 `BasecardDescriptor` 驱动。

## 3. 核心契约

当前正式核心契约是 `BasecardDescriptor`：

```ts
export interface BasecardDescriptor {
  pluginId: string;
  cardType: string;
  displayName: string;
  aliases?: string[];
  commitDebounceMs?: number;
  createInitialConfig(baseCardId: string): Record<string, unknown>;
  normalizeConfig(input: Record<string, unknown>, baseCardId: string): Record<string, unknown>;
  validateConfig(config: Record<string, unknown>): {
    valid: boolean;
    errors: Record<string, string>;
  };
  renderView(ctx: {
    container: HTMLElement;
    config: Record<string, unknown>;
    themeCssText?: string;
  }): (() => void) | void;
  renderEditor?(ctx: {
    container: HTMLElement;
    initialConfig: Record<string, unknown>;
    onChange(next: Record<string, unknown>): void;
  }): (() => void) | void;
}
```

当前正式会话快照契约是 `EditorSessionSnapshot`：

```ts
export interface EditorSessionSnapshot {
  key: string;
  cardId: string;
  baseCardId: string;
  cardType: string;
  sourceConfig: Record<string, unknown>;
  draftConfig: Record<string, unknown>;
  validation: {
    valid: boolean;
    errors: Record<string, string>;
  };
  dirty: boolean;
  commitDebounceMs: number;
  mountRevision: number;
  revision: number;
  isCommitting: boolean;
  errorMessage: string | null;
}
```

## 4. 当前注册中心职责

当前唯一正式注册中心是 `basecard-runtime/registry.ts`，负责：

- 维护 `cardType -> BasecardDescriptor` 映射；
- 维护 `alias -> cardType` 映射；
- 暴露 `normalizeBasecardType/getBasecardDescriptor/createInitialBasecardConfig/normalizeBasecardConfig/validateBasecardConfig`；
- 保证 `App.tsx`、`CardService`、`CardWindow`、`EditorHost` 统一使用同一套描述符。

当前阶段特点：

- 静态注册；
- 不做运行时插件目录扫描；
- 仅已注册 `richtextBasecardDescriptor`；
- 富文本不是主流程特判，而是注册中心中的一个正常项。

## 5. 会话层职责

`EditorSessionStore` 当前负责：

- 以 `cardId + baseCardId` 为 key 维护编辑会话；
- 保存 `sourceConfig/draftConfig/validation/dirty/errorMessage`；
- 在源配置变化且会话未脏时，同步草稿并增加 `mountRevision`；
- 在草稿变化时更新 `revision` 与校验状态；
- 在 `commit()` 时执行归一化、调用外部提交回调并刷新源配置；
- 在卡片切换或卸载时清理会话。

当前正式默认值：

- 若描述符未声明 `commitDebounceMs`，会话层默认使用 `260ms`；
- `commit()` 的唯一正式落点是 `CardService.updateBasicCard(...)`。

## 6. 宿主层职责

`EditorHost` 当前负责：

- 根据 `cardType` 解析 `BasecardDescriptor`；
- 通过 `EditorRuntimeProvider` 获取会话存储；
- 确保当前会话存在并订阅其快照；
- 把 `renderEditor` 直接挂载到 `.plugin-host__container`；
- 接收插件 `onChange`，更新草稿；
- 在草稿合法且已变脏时，按节流时间自动提交；
- 在卸载时 flush 最后一版草稿。

当前错误处理方式：

- 描述符缺失、编辑器导出缺失或挂载失败时，只在编辑面板内显示错误；
- 通过 `renderRevision` 支持本地重试；
- 不影响左侧预览链路。

## 7. 已完成的旧模块适配

- `CardWindow.tsx` 已不再把 `client.card.compositeWindow.render(...)` 作为正式主渲染入口；
- `PluginHost.tsx` 已不再承担正式编辑器 iframe 宿主职责；
- `CardService` / `App.tsx` 已统一通过描述符注册中心处理 `cardType`、默认值和数据归一；
- `CardContext.addBasicCard` 已与真实返回值对齐；
- `EditorRuntimeProvider` 已接入全局 Provider 树。

## 8. 当前正式结论

- 本地显示运行时和本地编辑运行时已经全部落地；
- 基础卡片描述符成为当前唯一正式入口；
- 后续新增基础卡片类型时，应先补描述符注册，再接入渲染与编辑导出；
- 不应再恢复独立的 `EditorRegistry` 或 SDK iframe 编辑器主路径。
