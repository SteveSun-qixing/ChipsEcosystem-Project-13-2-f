# 窗口协同与交互设计

## 1. CardWindow 当前角色

`CardWindow` 当前仍然是：

- 复合卡片拼装容器；
- 节点选中入口；
- 预览刷新调度器；
- 无限画布交互协同层。

本任务没有把编辑表单塞回 `CardWindow` 内部。

## 2. EditPanel 当前角色

`EditPanel` 当前改为：

- 纯编辑态容器；
- 当前选中基础卡片的本地编辑入口；
- 显示加载、错误和空态信息；
- 不再承担 SDK iframe 编辑器装载职责。

## 3. 预览与编辑协同

当前正式行为：

1. `CardWindow` 内部按顺序渲染各基础卡片 iframe；
2. 每个基础卡片通过 `BasecardFrameHost` 保持独立失败隔离；
3. 用户在单卡 iframe 中点选基础卡片；
4. 右侧 `EditPanel` 打开对应编辑器；
5. 编辑器修改草稿；
6. 数据提交后，`CardWindow` 刷新对应基础卡片 iframe 与复合高度。

显示和编辑的职责边界当前已经固定：

- 拼装显示层只负责“渲染、拼装、选中、交互代理”；
- 编辑面板只负责“改”和“显示编辑态错误”。

## 4. 窗口状态规则

### 正常态

- `CardWindow` 使用内部复合拼装链路展示复合卡片；
- 当前 `windowState === 'normal'` 时，交互策略为 `delegate`；
- `EditPanel` 展示当前编辑器。

### 收起态

- 卡片窗口继续保持已有的 16:9 收起规则；
- 当前 `windowState !== 'normal'` 时，交互策略切回 `native`；
- 不因本地编辑器运行时重构而改变。

### 封面态

- 继续沿用当前封面逻辑；
- 不把编辑器入口塞进封面窗口。

## 5. 无限画布协同

当前正式交互链路如下：

- `BasecardFrameHost` 在 `delegate` 模式下接管单卡 iframe 内的滚轮交互；
- 普通滚轮被归一为桌面平移输入；
- `ctrl/meta + wheel` 被归一为桌面缩放输入；
- `CardWindow` 通过 `panByInput/zoomByFactorAtPoint` 驱动画布；
- 交互序列期间会调用 `markInteractionSequence`，抑制误触发桌面缩放。

本任务需要保证且当前已满足：

- 编辑面板的本地输入不会误触发画布级交互；
- 编辑器高频输入不会导致卡片窗口被频繁重建；
- 节点选中与编辑器切换在窗口层保持一致。

## 6. 交互细节约束

- 不在本地编辑器内部再维护一套“基础卡片列表”作为正式入口；
- 正式入口仍然是点击复合卡片预览节点；
- 编辑器切换时应保留上一节点未 flush 的草稿；
- 当卡片切换或窗口关闭时，必须先完成会话清理；
- 基础卡片装配层不再额外添加边框、圆角和阴影，窗口只保留结构编排职责。
