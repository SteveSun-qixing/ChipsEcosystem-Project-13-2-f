# 测试与验收计划

## 1. 单元测试范围

### 编辑器运行时

- 注册中心可正确解析 `cardType`
- 会话层能创建、复用、销毁会话
- 草稿修改与提交节流正确
- 错误和校验状态可正确传播

### 编辑面板

- 选中不同基础卡片时能切换对应编辑器
- 无选中节点时显示空态
- 会话提交后调用正确的 `updateBasicCard`

### 卡片窗口

- 能正确按顺序渲染多个基础卡片 iframe
- 节点选中事件继续驱动编辑面板切换
- 编辑活动与复合拼装刷新节流策略保持稳定

## 2. 集成测试范围

- 从卡箱库创建卡片后，右侧能打开对应正式编辑器
- 编辑器修改内容后，解包态 `.card/content/*.yaml` 正常更新
- 复合卡片窗口在合理节流后正确刷新对应基础卡片 iframe 与整体布局
- 切换窗口状态、切换卡片、关闭窗口时草稿不丢失

## 3. 回归测试重点

- 无限画布缩放和平移不被编辑器输入干扰
- 基础卡片 iframe 拼装后的窗口交互、选中与布局不退化
- 封面态、收起态、普通态不回归
- 文件管理器打开卡片、创建卡片、重命名卡片不回归

## 4. 质量门禁

实施完成后至少通过：

- `npm run lint`
- `npx tsc --noEmit`
- `npm test`
- `npm run validate`

## 5. 发布前人工验收清单

1. 打开任意复合卡片，点击基础卡片后右侧出现对应正式编辑器。
2. 编辑器连续输入时界面流畅，无明显卡顿。
3. 停止输入后预览按预期刷新，不出现整窗抖动。
4. 编辑内容能正确持久化到解包态卡片目录。
5. 编辑引擎中的复合卡片显示已由基础卡片 iframe 拼装完成，不再依赖复合整体 iframe 作为正式编辑态入口。
6. 不再存在以 iframe 编辑器作为正式编辑入口的路径。
7. 所有当前正式支持的基础卡片类型都有真实编辑器，不依赖通用占位实现。
