## 2. 里程碑定义

## M1（WP-01 ~ WP-02）

- PAL 与 Kernel 主链路跑通。
- 主机可稳定启动并输出结构化启动日志。

## M2（WP-03 ~ WP-05）

- 16 服务域契约齐套。
- 对外动作面与生态共用文档一致。

## M3（WP-06 ~ WP-07）

- 插件调用链路端到端闭环。
- 权限、安全、错误归一通过联调验收。

## M4（WP-08）

- 全量门禁通过。
- 可灰度、可回滚、可发布。

---

## 3. 每任务包统一交付模板

- 设计：模块设计说明 + 接口契约更新。
- 实现：功能代码 + 单元测试 + 契约测试。
- 验证：集成测试报告 + 性能对比报告。
- 交付：变更说明、风险点、回滚说明。

---

## 4. Git 执行规范

- 任务开发统一使用任务分支，命名采用 `codex/<task>`。
- 每个任务包至少对应一个可回滚的原子提交序列。
- 合并前必须完成：`lint + typecheck + test + contract + perf`。
- 任务完成后合并目标分支为 `develop`，并保留审查记录。

---

## 5. 阻断规则

以下情况触发“任务阻断”，不得继续推进：

- 发现生态底层缺陷未建工单。
- 关键契约与生态共用文档不一致。
- 关键质量门禁连续失败且原因未闭环。
