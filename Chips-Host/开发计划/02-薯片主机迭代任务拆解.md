## 2. 里程碑定义

## M1（WP-01 ~ WP-02）

- PAL 与 Kernel 主链路跑通。
- 主机可稳定启动并输出结构化启动日志。

## M2（WP-03 ~ WP-05）

- 16 服务域契约齐套。
- 对外动作面与生态共用文档一致。

## M3（WP-06 ~ WP-07）

- 插件调用链路端到端闭环。
- 权限、安全、错误归一通过联调验收。

## M4（WP-08）

- 全量门禁通过。
- 可灰度、可回滚、可发布。

## M5（WP-L8-01 ~ WP-L8-06）

- L8 声明式 UI 库完整交付（原语、节点模型、组合模式、事件工具）。
- `runtime-effect` 渲染阶段阻断规则已由测试锁定。
- 文档、测试、导出入口三者一致，满足并入 `develop` 条件。

## M6（WP-L9-01 ~ WP-L9-07）

- L9 统一渲染引擎完整交付（流水线、适配器、一致性工具、错误边界、调度器）。
- 同一声明树跨目标渲染语义一致并具备自动化检测能力。
- 大树更新与可见区域增量提交满足性能门禁并完成 Host 主链路接入。

## M7（WP-L11-01 ~ WP-L11-05）

- L11 主题运行时完整交付：
  - 支持五层 token 解析（`ref/sys/comp/motion/layout`）与 ThemeSnapshot 构建。
  - 主题作用域链与覆盖规则按生态规范生效，支持 `component → card → workspace → app → global` 解析语义。
  - `theme.resolve` 提供完整解析算法并与 L9/L10 链路闭环。
  - 主题契约门禁（token/组件接口点/动效/颜色对比度）与主题管理工具可用。
- Host 端主题相关接口与生态共用文档保持一致，质量门禁全部通过，满足合入 `develop` 条件。

## 3. 每任务包统一交付模板

- 设计：模块设计说明 + 接口契约更新。
- 实现：功能代码 + 单元测试 + 契约测试。
- 验证：集成测试报告 + 性能对比报告。
- 交付：变更说明、风险点、回滚说明。

---

## 4. Git 执行规范

- 任务开发统一使用任务分支，命名采用 `codex/<task>`。
- 每个任务包至少对应一个可回滚的原子提交序列。
- 合并前必须完成：`lint + typecheck + test + contract + perf`。
- 任务完成后合并目标分支为 `develop`，并保留审查记录。

---

## 5. 阻断规则

以下情况触发“任务阻断”，不得继续推进：

- 发现生态底层缺陷未建工单。
- 关键契约与生态共用文档不一致。
- 关键质量门禁连续失败且原因未闭环。
