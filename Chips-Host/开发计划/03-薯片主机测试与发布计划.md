# 薯片主机测试与发布计划

> 文档状态：测试发布执行稿
> 目标：确保开发交付满足稳定性、兼容性、可回滚要求。

---

## 1. 测试分层计划

| 测试层级 | 覆盖对象 | 目标 |
|---|---|---|
| 契约测试 | 路由动作、schema、权限、错误码 | 保证契约不漂移 |
| 单元测试 | PAL、Kernel、Service、Runtime 模块 | 保证模块逻辑正确 |
| 集成测试 | Router-Services、Bridge-Kernel、Runtime-Bridge | 保证跨模块协作 |
| E2E 测试 | 插件调用全链路 | 保证用户可感知路径稳定 |
| 性能测试 | 路由/Bridge/服务延迟与吞吐 | 保证性能预算 |
| 鲁棒性测试 | 超时、断连、熔断、回滚 | 保证故障可恢复 |

## 1.1 L9 扩展测试矩阵（Phase 6）

| 测试类目 | 覆盖对象 | 关键断言 |
|---|---|---|
| 渲染流水线单测 | Normalize/Validate/Resolve/Compute/Commit/Dispatch | 6 阶段全部可执行、错误定位准确 |
| 适配器一致性测试 | App Root/Card Iframe/Module Slot/Offscreen | 同树语义哈希一致、非兼容能力硬失败 |
| 故障注入测试 | 节点级/区域级 Error Boundary | 单节点失败不拖垮整页、降级渲染可用 |
| 调度性能测试 | render queue + 增量提交 | 渲染提交 p95 <= 32ms、长任务占比 <= 5% |

---

## 2. 测试环境与数据

- 环境：Windows/macOS/Linux 三平台测试矩阵。
- 数据：标准 card/box 样例、异常样例、边界样例。
- 工具：自动化测试流水线 + 诊断日志导出。

---

## 3. 质量门禁

### 3.1 必过门禁

- TypeScript 严格模式通过。
- lint 与格式检查通过。
- 契约测试通过率 100%。
- 关键链路集成测试全绿。
- 性能指标不超预算阈值。
- L9 阶段补充门禁：`npm run test:perf:l9` 通过（render-commit p95 <= 32ms）。

### 3.2 阻断级失败

- 崩溃类问题。
- 数据损坏或幂等失效。
- 权限绕过或敏感能力越权。

---

## 4. 发布策略

## 4.1 发布前检查

- 变更范围与需求文档一致。
- 生态共用接口文档已同步。
- 风险清单与回滚脚本已验证。

## 4.2 灰度发布

- 先灰度到小流量环境，观察关键指标。
- 指标稳定后逐步扩大覆盖范围。

## 4.3 回滚策略

- 保留上个稳定版本可快速切回。
- 插件安装与服务配置变更支持回滚。
- 回滚后自动执行健康检查并输出报告。

---

## 5. 发布后验证

- 验证主机启动、插件加载、桥接调用、主题与多语言核心路径。
- 核查错误码分布与性能趋势是否异常。
- 输出发布复盘：问题、原因、修复、预防措施。
