# 阶段五：L8 声明式 UI 分阶段开发计划

> 文档状态：已完成  
> 目标：将 L8 任务拆解为可验收、可回滚、可并行评审的多个子阶段，确保高质量交付

---

## 1. 总体目标与约束

- 完整交付声明式 UI 库、语义原语实现、组合模式规范文档、事件绑定工具。
- 严格遵循生态手册：L8 不承载视觉值，不直连 Host 服务，只经 UI Hooks / Runtime Client。
- 严格执行副作用分级：`ui-effect`、`runtime-effect`、`telemetry-effect`。
- 强制门禁：渲染阶段禁止直接触发 `runtime-effect`。

---

## 2. 分阶段执行计划

### Stage 5.1：契约冻结与技术基线

**目标**

- 冻结 L8 的接口边界、目录边界、测试边界。

**开发内容**

- 对齐 `生态设计原稿` 与 `生态共用技术文档` 中 L8 条款。
- 确定 `declarative-ui` 目录结构、导出策略、错误码前缀。
- 确认与 Runtime Client/UI Hooks 的依赖关系白名单。

**交付物**

- L8 技术基线清单（字段、接口、约束）。
- 实施任务拆解（WP-L8-01~06）。

**验收标准**

- 架构评审通过。
- 无跨层依赖争议项。

---

### Stage 5.2：声明树节点模型与语义原语

**目标**

- 先落地“可稳定复用的数据结构”，再落地原语工厂。

**开发内容**

- 实现 `UINode` / `UINodeInput` 标准模型，覆盖：
  - `id/type/props/state/bindings/events/themeScope/children`
- 实现 `createUINode`、`cloneUINode`、`appendChildren`、`bindNodeEvent`。
- 实现原语工厂：
  - `View`、`Stack`、`Grid`、`Form`、`List`

**交付物**

- `src/renderer/declarative-ui/types.ts`
- `src/renderer/declarative-ui/node-model.ts`
- `src/renderer/declarative-ui/primitives.ts`

**验收标准**

- 原语输出统一节点模型。
- 非法节点输入（空 id/type）可被阻断。

---

### Stage 5.3：组合模式规范与 Compound 内核

**目标**

- 用显式组合替代 boolean 模式扩张。

**开发内容**

- 实现 `createCompoundComponent` 组合工厂。
- 实现 slot 契约校验：`required/type/multiple`。
- 实现 `guardAgainstBooleanModeProps`，默认阻断 `is/has/show/with*` 布尔模式属性。
- 支持显式 slot 组装与约束错误提示。

**交付物**

- `src/renderer/declarative-ui/composition.ts`

**验收标准**

- 组合组件可声明化创建。
- slot 缺失/类型不匹配会被明确阻断。
- boolean 模式扩张可被测试拦截。

---

### Stage 5.4：事件绑定与副作用分级工具

**目标**

- 建立可治理的事件执行链与 effect 分发链。

**开发内容**

- 实现 `EventBindingRegistry`：
  - handler 注册/解绑
  - 语义事件 -> handlerId 映射分发
- 实现 `EffectDispatcher`：
  - 分级执行 `ui-effect/runtime-effect/telemetry-effect`
  - 渲染阶段拦截 `runtime-effect`
  - 支持 render 阶段调度、commit 阶段 flush

**交付物**

- `src/renderer/declarative-ui/events.ts`
- `src/renderer/declarative-ui/effects.ts`

**验收标准**

- `ui-effect` / `telemetry-effect` 正常执行。
- `runtime-effect` 在 `render` 阶段被拒绝并返回标准错误。
- `runtime-effect` 可在 `commit` 阶段按调度执行。

---

### Stage 5.5：文档规范与导出收口

**目标**

**开发内容**

**交付物**

- `技术文档/10-声明式UI组合模式规范.md`
- `src/renderer/declarative-ui/index.ts`
- `src/renderer/index.ts`

**验收标准**

- 文档条款与代码实现一致。
- 对外导出路径稳定、无循环依赖。

---

### Stage 5.6：测试门禁与发布准备

**目标**

- 通过自动化门禁确保“非临时实现、可长期维护”。

**开发内容**

- 编写 L8 单测，覆盖：
  - 节点模型

  - 副作用分级规则

**交付物**

- 阶段五验收结果记录

**验收标准**

- 构建通过、测试全绿。
- 无 TODO、无临时代码、无越层依赖。

## 3. 阶段依赖关系

- Stage 5.3 依赖 Stage 5.2 节点模型稳定。
- Stage 5.4 依赖 Stage 5.2/5.3 的节点与组合约束。
- Stage 5.5 与 Stage 5.4 并行收口，但必须晚于接口冻结。
- Stage 5.6 在全部功能与文档完成后统一执行。

---

## 4. 风险与阻断规则

- 若发现生态契约缺失、组件库能力断层、SDK 接口阻断，先登记 `项目日志与笔记/问题工单.md` 并暂停推进。
- 若出现跨层调用（L8 直连 Host 服务）立即阻断并回退到 Hooks/RuntimeClient 链路。
- 若出现 boolean 模式扩张或 render 触发 runtime-effect，视为阻断级缺陷。
