# L9统一渲染层开发计划

> 文档状态：新增阶段执行基线稿  
> 适用范围：`Chips-Host` L9 Unified Rendering（含与 L8/L11 的接口协作）  
> 目标：交付统一渲染引擎、目标适配器、渲染一致性测试工具、增量更新调度器。

---

## 1. 阶段目标与边界

- L9 归属 Host 运行时主实现，按生态口径在 `Chips-Host` 内完成。
- 本计划覆盖：渲染流水线、适配器、一致性约束、失败隔离、调度与增量更新。
- 本计划不改动 L10 组件库实现，不引入未审批第三方依赖。
- 所有实现必须满足“零临时实现、零占位实现、可测试、可回滚”的交付标准。

## 3. 详细阶段计划

## 3.1 R0 基线冻结与契约建模

### 目标

- 确认 L9 对外能力边界、输入声明树模型、输出提交结构。
- 明确与 `card.render`、主题服务、Runtime Client 的接口协同点。

### 任务清单

1. 定义统一声明树输入模型（`id/type/props/state/bindings/events/themeScope/children`）。
2. 冻结 6 阶段流水线接口：Normalize、Validate、Resolve、Compute、Commit、Dispatch。
3. 定义 4 类目标适配器契约：`App Root`、`Card Iframe`、`Module Slot`、`Offscreen Render`。
4. 定义一致性哈希规则与错误码清单（`RENDER_*` 前缀）。
5. 补齐测试目录结构：unit/contract/integration/perf/fault-injection。

### 交付物

- L9 接口契约文档与类型定义。
- 渲染错误码与诊断字段规范。
- 阶段实施任务分解表。

### 验收标准

- 契约字段完整且与生态共用文档口径一致。
- 测试样例覆盖正常路径、异常路径、边界路径。
- 未发现跨层违规依赖（仅允许 L9 依赖 L8/L11 接口）。

---

## 3.2 R1 渲染流水线内核

### 目标

- 实现统一渲染 6 阶段主流水线并打通最小可运行链路。

### 任务清单

1. Node Normalize：默认属性合并、节点 ID 归一化、复合节点展开。
2. Contract Validate：节点类型校验、属性契约校验、非兼容能力硬失败。
3. Theme Resolve：token 解析、作用域链合并、样式映射生成。
4. Layout Compute：布局计算、cpx→px 换算、渲染树构建。
5. Render Commit：提交抽象层，输出目标无关的提交载荷。
6. Effect Dispatch：`ui-effect/runtime-effect/telemetry-effect` 分级分发，禁止渲染阶段执行 runtime-effect。

### 交付物

- 统一渲染引擎核心模块。
- 流水线阶段耗时采样与日志字段。
- 单元测试与契约测试。

### 验收标准

- 6 阶段链路全部可执行。
- 关键错误码可定位（节点 ID、阶段、原因）。
- 相关单元测试通过率 100%。

---

## 3.3 R2 目标适配器与一致性约束

### 目标

- 为四类渲染目标提供统一适配器，并确保语义一致性。

### 任务清单

1. 实现 `App Root` 适配器（主窗口提交）。
2. 实现 `Card Iframe` 适配器（iframe 隔离渲染）。
3. 实现 `Module Slot` 适配器（模块插槽挂载）。
4. 实现 `Offscreen Render` 适配器（离屏预览/导出）。
5. 构建一致性测试工具：同一声明树跨适配器生成语义哈希并比对。
6. 建立非兼容能力 fail-fast 机制，阻断 silent fallback。

### 交付物

- 目标适配器实现。
- 渲染一致性测试工具与报告模板。

### 验收标准

- 四类目标适配器均可完成提交。
- 一致性测试工具可检测并报告语义漂移。
- 非兼容能力在 Validate 阶段失败，错误信息可审计。

---

## 3.4 R3 失败隔离与降级机制

### 目标

- 保证单节点失败不拖垮整页渲染，具备区域级隔离与降级展示。

### 任务清单

1. 节点级 Error Boundary：节点失败后替换为节点降级内容。
2. 区域级 Error Boundary：区域异常隔离，区域外继续渲染。
3. 失败信息写入诊断通道：`nodeId/stage/code/message`。
4. 故障注入测试：组件抛错、主题缺失、布局异常、适配器提交失败。

### 交付物

- 失败隔离模块。
- 故障注入测试套件。
- 降级渲染策略说明。

### 验收标准

- 单节点失败不影响兄弟节点提交。
- 区域失败不影响全局渲染结果。
- 故障注入场景通过并输出可读诊断。

---

## 3.5 R4 调度与增量更新

### 目标

- 实现可中断渲染队列与可见区域增量提交，控制大树更新成本。

### 任务清单

1. 维护 `render queue`，将大树更新拆分为批次。
2. 支持可中断调度（按帧预算让出执行）。
3. `List/Grid/Table` 默认启用可见区域增量提交。
4. 实现主题切换“样式先行、结构后行”策略。
5. 建立性能基线：单次提交 p95、长任务占比、滚动帧率。

### 交付物

- 增量更新调度器。
- 渲染队列监控指标与性能报告。

### 验收标准

- 大树更新可分批执行且支持中断恢复。
- 可见区域提交策略生效，非可见节点延后提交。
- 关键指标满足门禁阈值（渲染提交 p95 <= 32ms、长任务占比 <= 5%）。

---

## 3.6 R5 Host 集成与质量收口

### 目标

- 将 L9 能力接入 Host 现有 `card.render` 链路，并完成发布前门禁。

### 任务清单

1. 将 `card.render` 从模板拼接链路切换为统一渲染引擎入口。
2. 保持 `plugin/bridge/runtime-client` 调用语义兼容。
3. 补齐 route-manifest、schema、对外接口基线文档变更。
4. 执行回归：unit + contract + integration + e2e + perf + fault-injection。
5. 输出阶段验收报告与回滚方案。

### 交付物

- L9 集成版本代码。
- 全量测试报告与性能基线对比。
- 发布说明与回滚说明。

### 验收标准

- `lint + typecheck + test + contract + perf` 全部通过。
- 对外动作与错误语义无未声明变更。
- 变更可回滚且回滚脚本验证通过。

## 5. 阻断与工单规则

出现以下任一情况，立即按流程写入 `项目日志与笔记/问题工单.md` 并暂停实现推进：

- 生态契约缺失或跨层接口无法闭环。
- 渲染所需关键组件/服务在生态内不可用。
- Host/Kernel/公共基础层打通失败。
- 需要新增技术选型外第三方依赖。
