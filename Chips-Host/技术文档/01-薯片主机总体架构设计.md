# 薯片主机总体架构设计

> 文档状态：内部技术基线稿
> 边界说明：仅描述主机内部实现与调用链，不重复外部契约定义。

---

## 1. 主机进程拓扑

| 进程/层 | 位置 | 职责 |
|---|---|---|
| Main Process | `src/main` | 启动编排、服务注册、窗口治理、全局异常处理 |
| Preload | `src/preload` | `contextBridge` 注入、最小暴露 API、IPC 包装 |
| Renderer Runtime | `src/renderer` | 插件容器端 Runtime Client、UI Hooks、L8 Declarative UI 宿主 |
| Shared Contract | `src/shared` | 协议类型、错误模型、日志字段、常量 |

## 2. 包结构与层级映射

| 目录 | 层级映射 | 内部职责 |
|---|---|---|
| `packages/pal` | L1 | OS 能力适配、平台差异收敛 |
| `packages/kernel` | L2 | 路由、注册表、事件总线、生命周期 |
| `packages/common-basics` | L3 | 通用基建工具、公共中间件、基础校验器 |
| `packages/card-service` | L3 | 卡片 parse/render/validate 服务实现 |

| `src/renderer/declarative-ui` | L8 | 语义原语、节点模型、组合模式、事件/副作用模型 |

## 3. 启动链路（Boot Pipeline）

## 3.1 启动阶段定义

1. `boot:prepare`：加载环境配置、初始化日志系统。
2. `boot:pal`：初始化 PAL 适配器并完成平台自检。
3. `boot:kernel`：启动路由器、事件总线、生命周期管理器。
4. `boot:services`：按依赖顺序注册并启动服务域。
5. `boot:runtime`：初始化插件运行时、权限快照、配额策略。
6. `boot:bridge`：注册 IPC 通道并注入 `window.chips.*`。
7. `boot:ready`：开放入口窗口与命令行控制面。

## 3.2 启动失败策略

- 任一关键阶段失败（PAL、Kernel、核心服务）必须阻断进入 `running`。
- 失败后输出结构化诊断信息，并进入可恢复停止态。

---

## 4. 核心调用链

## 4.1 插件调用主机链路

`Renderer Plugin -> Runtime Client -> Bridge Transport -> Kernel Router -> Service Handler -> Kernel Router -> Bridge Transport -> Runtime Client -> Plugin`

### 链路约束

- 统一生成并透传 `requestId`。
- Kernel 必须在路由层完成 schema + 权限 + 超时检查。
- Service 仅处理业务语义，不重复实现路由策略。

## 4.2 事件广播链路

`Service Event -> Kernel EventBus -> Bridge Event Channel -> Renderer Listener`

### 事件约束

- 事件名仅允许点语义。
- 事件元信息必须包含 `id/name/source/timestamp`。

---

## 5. 内部数据与状态模型

## 5.1 生命周期状态机

`unloaded -> loading -> ready -> running -> stopping -> stopped/error`

## 5.2 运行态状态面

- `hostState`：主机运行状态与版本信息。
- `serviceStateMap`：服务级健康状态与最近错误。
- `pluginSessionMap`：插件会话、权限、资源配额、窗口绑定。
- `routeMetrics`：动作级延迟、超时率、错误率。

---

## 6. 内部错误处理架构

- 统一错误对象：`{ code, message, details?, retryable? }`。
- 层内错误不直接上抛原生异常；必须映射到标准错误码。
- Runtime Client 负责对 Bridge/Service 错误再归一并提供重试语义。

---

## 7. 代码组织与隔离策略

- `src/main` 不允许依赖 `src/renderer`。
- `packages/kernel` 不允许依赖具体服务实现目录。
- `packages/*-service` 之间禁止直接调用，统一经 Kernel 路由。
- `src/preload` 仅做桥接封装，不承载业务逻辑。
