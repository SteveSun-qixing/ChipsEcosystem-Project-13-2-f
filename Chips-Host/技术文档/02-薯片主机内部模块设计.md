> 文档状态：内部实现设计稿
> 适用范围：`Chips-Host` 仓内模块拆分、依赖策略、实现模板。

## 1. 模块分层设计

## 1.1 L1 PAL 模块

| 模块 | 目录建议 | 说明 |
|---|---|---|
| WindowAdapter | `packages/pal/src/window` | BrowserWindow 与平台窗口差异封装 |
| FileAdapter | `packages/pal/src/file` | 路径、读写、监听、权限检测 |
| SystemAdapter | `packages/pal/src/system` | 对话框、剪贴板、托盘、电源、快捷键 |
| IPCAdapter | `packages/pal/src/ipc` | 主渲染通信基础设施 |

## 1.2 L2 Kernel 模块

| 模块 | 目录建议 | 说明 |
|---|---|---|
| Router | `packages/kernel/src/router` | 路由注册、匹配、超时、熔断 |
| Registry | `packages/kernel/src/registry` | 服务注册与依赖图 |
| EventBus | `packages/kernel/src/event` | 发布订阅、过滤、回收 |
| Lifecycle | `packages/kernel/src/lifecycle` | 状态机推进与阻断策略 |
| ModuleLoader | `packages/kernel/src/module-loader` | 模块格式识别、依赖版本校验、隔离策略 |

## 1.3 L3 Services 模块

- 每个服务域遵循统一结构：

```text
service-domain/
  types.ts
  constants.ts
  schemas/
  service.ts
  __tests__/
```

- 推荐在 `packages/common-basics` 提供共享能力：
  - schema 校验器
  - 错误工厂
  - 审计日志中间件
  - 幂等去重组件

## 1.4 L4 Runtime 模块

| 模块 | 职责 |
|---|---|
| PluginLoader | 解析插件清单、加载入口、执行安全流水线 |
| SessionManager | 插件会话生命周期、权限快照、nonce 管理 |
| QuotaManager | CPU/内存/消息频率软配额与降级策略 |
| IsolationManager | BrowserWindow/iframe 隔离与清理 |

## 1.5 L5/L6 Bridge 模块

| 模块 | 职责 |
|---|---|
| BridgeTransport | IPC 通道注册、请求转发、事件回推 |
| ActionAliasMapper | 历史动作映射与兼容吸收 |
| RuntimeClientCore | 参数规范化、超时控制、重试、错误归一 |
| HookFacade | 面向 UI 的能力代理封装 |

## 2. 关键序列设计

## 2.1 服务调用序列（内部）

1. Runtime Client 组装请求：动作、payload、requestId、caller。
2. BridgeTransport 通过 `chips:invoke` 发送主进程。
3. Kernel Router 做 schema/权限/超时检查。
4. Service Handler 执行业务动作。
5. Router 回传响应并写入链路日志。
6. Runtime Client 归一错误或返回数据。

## 2.2 插件安装序列（内部）

1. 插件包校验（格式、签名、manifest）。
2. 生成安装事务上下文（可回滚）。
3. 写入插件注册信息与权限快照。
4. 执行 `plugin-init` 握手。
5. 成功则激活，失败则按事务回滚。

## 2.3 模块动态加载序列（内部）

1. 模块声明 `manifest(id/version/format/dependencies/isolation)`。
2. ModuleLoader 校验版本格式与依赖是否满足。
3. 根据 `format` 执行加载：
   - `service-registration`：注册服务并落地生命周期。
   - `route-bundle`：注册路由并写入模块路由索引。
4. 若 `isolation=isolated`，强制校验路由前缀必须以 `moduleId.` 开头。
5. 记录加载清单与路由列表，供控制平面查询。

## 3. 数据与配置设计

## 3.1 主机内部配置域

| 配置域 | 说明 | 读写策略 |
|---|---|---|
| runtime | 超时、重试、熔断、配额 | 启动读取，运行可热更新 |
| workspace | 工作区路径、插件目录、缓存目录 | 变更需校验与审计 |
| security | 权限策略、危险动作审计开关 | 仅管理员修改 |
| observability | 日志级别、指标采样率、诊断策略 | 支持动态调整 |

## 3.2 内部存储原则

- 业务内容数据（card/box）不进入主机私有数据库。
- 主机仅保存运行时元数据、索引、缓存、状态快照。
- 敏感凭证由 `credential` 服务托管，不允许散落存储。

## 4. 内部测试设计

| 目标模块 | 测试类型 | 关键断言 |
|---|---|---|
| PAL | 单元 + 跨平台回归 | 平台差异被统一吸收 |
| Kernel Router | 契约 + 压测 | 路由正确、超时生效、熔断可恢复 |
| Services | 单元 + 契约 + 集成 | 动作面完整、权限边界正确 |
| Runtime | 集成 + 故障注入 | 握手失败阻断、配额降级生效 |
| Bridge/Client | E2E + 错误映射测试 | 错误归一一致、重试策略正确 |

---

## 5. 内部代码审查清单

- 是否出现跨层依赖违规。
- 是否出现服务间直接调用。
- 是否有未声明 schema 的动作。
- 是否有未审计的危险动作。
- 是否出现硬编码错误对象或日志字段漂移。
