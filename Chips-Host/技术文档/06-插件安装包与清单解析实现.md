# 插件安装包与清单解析实现

## 1. 目标

- 补齐插件安装链路，使主机支持从 `.cpk` 包安装插件。
- 支持 `manifest.yaml` / `manifest.yml` / `manifest.json` / `*.plugin.json` 清单解析。
- 安装后统一落库存储到主机工作目录，避免直接依赖源路径。

## 2. 实现位置

- `src/runtime/plugin-runtime.ts`
- `src/shared/yaml-lite.ts`
- `packages/card-service/src/card-service.ts`（共用 YAML 解析器接入）
- `packages/box-service/src/box-service.ts`（共用 YAML 解析器接入）

## 3. 安装流程

### 3.1 输入来源

`plugin.install` 当前支持三类输入路径（字段名仍为 `manifestPath`，保持兼容）：

1. 目录路径：自动扫描目录中的清单文件
2. `.cpk` 文件：先解包，再扫描清单
3. 清单文件路径：直接读取清单并构建安装包

### 3.2 清单解析

- JSON 清单：`JSON.parse`
- YAML 清单：`parseYamlLite`
- 必填字段校验：`id/version/type/name/permissions`
- `type` 限定：`app/card/layout/module/theme`
- `entry` 声明存在时，必须能解析到真实入口文件，否则拒绝安装

### 3.3 落库存储

- 统一安装目录：`<workspace>/plugins/<pluginId>`
- 安装记录持久化：`plugin-runtime.json`
- 持久化字段新增：`installPath`
- 卸载时同时删除安装目录

## 4. 兼容策略

- 兼容旧记录：历史 `plugin-runtime.json` 若无 `installPath`，回退为 `dirname(manifestPath)`。
- 兼容旧调用：路由输入字段仍使用 `manifestPath`，调用方无需改造。
- 查询返回：`plugin.query` 的 `manifestPath` 从“源路径”切换为“安装后清单路径”。
