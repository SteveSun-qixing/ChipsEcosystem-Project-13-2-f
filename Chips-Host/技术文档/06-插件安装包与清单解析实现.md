# 插件安装包与清单解析实现

## 1. 目标

- 补齐插件安装链路，使主机支持从 `.cpk` 包安装插件。
- 支持 `manifest.yaml` / `manifest.yml` / `manifest.json` / `*.plugin.json` 清单解析。
- 安装后统一落库存储到主机工作目录，避免直接依赖源路径。

## 2. 实现位置

- `src/runtime/plugin-runtime.ts`
- `src/shared/yaml-lite.ts`
- `packages/card-service/src/card-service.ts`（共用 YAML 解析器接入）
- `packages/box-service/src/box-service.ts`（共用 YAML 解析器接入）

## 3. 安装流程

### 3.1 输入来源

`plugin.install` 当前统一使用 `manifestPath` 作为输入字段名，并支持三类输入路径：

1. 目录路径：自动扫描目录中的清单文件
2. `.cpk` 文件：先解包，再扫描清单
3. 清单文件路径：直接读取清单并构建安装包

当前 CLI 对应命令为 `chips plugin install <path>`，不再使用 `chips host plugin install` 二级写法。

`.cpk` 包格式约束：

- ZIP 必须使用 Store 模式（不压缩）；
- 包根目录必须存在 manifest；
- 构建产物应位于包内 `dist/`；
- 包内不得嵌套旧 `.cpk`、重复 manifest 或无关发布元数据。

### 3.2 清单解析

- JSON 清单：`JSON.parse`
- YAML 清单：`parseYamlLite`
- 必填字段校验：`id/version/type/name/permissions`
- `type` 限定：`app/card/layout/module/theme`
- `entry` 声明存在时，必须能解析到真实入口文件，否则拒绝安装

### 3.3 落库存储

- 统一安装目录：`<workspace>/plugins/<pluginId>`
- 安装记录持久化：`plugin-runtime.json`
- 持久化字段新增：`installPath`
- 卸载时同时删除安装目录

## 4. 当前存储与查询约束

- 运行时记录以安装后清单路径作为 `manifestPath` 的持久化值；
- 历史记录若缺少 `installPath`，Host 会在读取阶段根据清单路径补齐安装目录推断；
- `plugin.query` 返回的是安装后记录，而不是源工程路径快照。

## 6. 主题插件运行时补充（2026-03-10）

主题包在安装包层面仍属于普通插件，但运行时语义增加了以下约束：

- 主题包安装完成后不会自动进入可用主题集合，必须显式执行 `plugin enable`；
- Host 主题服务只装载当前工作区中已启用的主题插件；
- 主题插件发生安装、启用、禁用、卸载时，Host 会立即刷新主题运行时缓存与当前主题状态；
- 主题验证的正式命令链路为：`plugin install -> plugin enable -> theme apply -> theme validate`。
