# 文件关联入口与服务按需初始化

## 1. 目标

- 为主机提供统一的文件关联打开入口，支持 `.card` / `.box` 文件直达主机能力链路。
- 将 card/box/zip 三类重服务改为按需实例化，减少主机启动期初始化开销。

## 2. 实现位置

- 文件关联入口：
  - `src/main/core/file-association.ts`
  - `src/main/cli/index.ts`（`chips host open <file>`）
- 服务按需初始化：
  - `src/main/core/host-application.ts`
  - `src/main/services/register-host-services.ts`

## 3. 文件关联入口链路

### 3.1 CLI 命令

- 新增命令：`chips host open <target-file>`
- 支持行为：
  - `.card`：执行 `card.render` 预解析并创建窗口记录
  - `.box`：执行 `box.inspect` 静态预检查后，通过 `file-handler:.box` 启动正式查看器
  - 其他扩展名：转发 `platform.shellOpenPath`

### 3.2 插件优先策略

- 若存在启用中的应用插件且能力声明包含 `file-handler:<ext>`，优先走插件链路：
  - `plugin.init`
  - `plugin.handshake.complete`
  - `window.open`
- `.card` 在缺少应用插件时允许回退到主机默认窗口打开策略；
- `.box` 在缺少应用插件时必须直接返回可诊断错误，禁止静默创建空窗口。

## 4. 服务按需初始化

### 4.1 背景

原先主机启动时会立即创建 `CardService/BoxService/StoreZipService` 实例。

### 4.2 调整

- `HostApplication` 注入服务提供器（getter），首次使用时才实例化：
  - `getCardService()`
  - `getBoxService()`
  - `getZipService()`
- `register-host-services` 内部改为调用 getter 获取实例，避免启动期创建。
