# 模块加载与协议兼容闭环

## 1. 目标

- 补齐 Kernel 侧模块加载器能力，覆盖模块格式识别、依赖版本约束、隔离策略。
- 补齐 Protocol 实现层，覆盖历史字段兼容、动作别名升级、序列化/反序列化闭环。

## 2. Kernel 模块加载器

实现文件：`packages/kernel/src/module-loader.ts`

### 2.1 模块格式

- `service-registration`：按服务注册模型加载。
- `route-bundle`：直接加载路由集合。

### 2.2 依赖与版本校验

- 依赖项声明：`{ id, version? }`。
- 支持版本表达式：
  - 精确匹配：`1.2.3`
  - 主版本范围：`^1.0.0`
  - 次版本范围：`~1.2.0`
- 依赖缺失返回：`MODULE_DEPENDENCY_MISSING`。
- 版本不满足返回：`MODULE_DEPENDENCY_VERSION_MISMATCH`。

### 2.3 隔离策略

- 当 `manifest.isolation = isolated` 时，路由必须满足 `route.key` 以 `moduleId.` 开头。
- 违反约束返回：`MODULE_ISOLATION_ROUTE_VIOLATION`。

### 2.4 Kernel 集成点

- `packages/kernel/src/kernel.ts`
  - `loadModule(module)`
  - `listModules()`
- 通过 hooks 统一复用现有 `registerRoute/registerService` 链路。

## 3. Protocol 编解码兼容层

实现文件：`packages/protocol/src/codec.ts`

### 3.1 兼容输入字段

- Request：`version -> protocolVersion`、`id -> messageId`、`serviceName/action -> service`、`data -> payload`。
- Response：`id -> messageId`、`correlationId -> requestId`。
- Event：`id -> eventId`、`type/name -> eventType`。

### 3.2 兼容策略

- `0.x` 与 `1.0/1.0.0` 自动归一到 `1.0.0`。
- 旧动作别名自动升级为 `platform.*` 标准动作。
- 时间戳支持字符串与数字输入，统一为 ISO 字符串。

### 3.3 序列化能力

- `serializeRequestMessage/serializeResponseMessage/serializeEventMessage`
- `parseSerializedRequestMessage/parseSerializedResponseMessage/parseSerializedEventMessage`
