# 声明式 UI 组合模式规范（L8）

> 文档状态：执行稿  
> 适用范围：`Chips-Host/src/renderer/declarative-ui/*` 与上层页面编排层

---

## 1. 目标

- 为 L8 声明式 UI 提供统一的组合模式约束。
- 采用 Vercel 组合模式，优先 Compound Components，避免 boolean 模式扩张。
- 固化事件绑定与副作用分级模型，禁止渲染阶段直接触发 `runtime-effect`。

---

## 2. 核心语义原语

L8 对外只暴露语义原语，不暴露视觉实现细节：

- `View`
- `Stack`
- `Grid`
- `Form`
- `List`

所有原语统一输出 `UINode`：

```ts
interface UINode {
  id: string;
  type: 'View' | 'Stack' | 'Grid' | 'Form' | 'List' | string;
  props?: Record<string, unknown>;
  state?: Record<string, unknown>;
  bindings?: Record<string, string>;
  events?: Record<string, string>;
  themeScope?: string;
  children?: UINode[];
}
```

---

## 3. Compound Components 规范

### 3.1 允许模式

- 组件以“根节点 + 显式 slot 子节点”组合。
- slot 在定义时声明 `required/type/multiple`。
- Provider（页面域）是唯一状态实现知情者，L8 只消费已注入状态。

### 3.2 禁止模式

- 禁止通过多个 boolean 属性驱动多模式渲染（例如 `isCompact/isInline/showFooter` 组合）。
- 禁止在 slot 未声明时注入临时子节点。
- 禁止在 L8 层写主题视觉值。

### 3.3 工具约束

- 使用 `createCompoundComponent` 创建复合组件契约。
- 默认启用 `guardAgainstBooleanModeProps`，阻断 boolean 模式扩张。
- 业务变体通过显式组件/slot 组合表达，而不是 boolean 开关。

---

## 4. 事件绑定与副作用分级

### 4.1 事件绑定

- `events` 仅声明语义事件 -> 处理器 ID 映射（如 `onSubmit -> submit.handler`）。
- 处理器通过 `EventBindingRegistry.registerHandler` 注册。
- 事件分发统一走 `EventBindingRegistry.dispatch`。

### 4.2 副作用分级

- `ui-effect`：滚动、聚焦、过渡。
- `runtime-effect`：调用 Runtime Client 或 Host 能力。
- `telemetry-effect`：埋点与日志上报。

### 4.3 渲染阶段约束

- `EffectDispatcher.dispatch` 在 `phase=render` 且 effect 为 `runtime-effect` 时直接拒绝：
  - 错误码：`DECLARATIVE_UI_RUNTIME_EFFECT_FORBIDDEN_IN_RENDER`
- 如需延迟执行，必须显式调用 `schedule()` 并在 `commit` 阶段 `flushScheduledRuntimeEffects()`。

---

## 5. 质量门禁

- 单测覆盖：语义原语、节点模型、组合约束、事件绑定、副作用分发。
- 关键约束测试：
  - boolean 模式 props 拦截
  - required slot 校验
  - `runtime-effect` 渲染阶段阻断
  - `ui-effect/telemetry-effect` 正常分发
