# L8 声明式 UI 实现说明

> 文档状态：已实现  
> 适用范围：Host 内置运行时 L8（`src/renderer/declarative-ui/*`）

---

## 1. 实现范围

本次实现覆盖架构手册中 L8 的四类交付要求：

- 语义原语：`View`、`Stack`、`Grid`、`Form`、`List`
- 声明树节点模型：`id/type/props/state/bindings/events/themeScope/children`
- 组合模式规范：Compound Components + slot 契约 + boolean 模式守卫
- 事件与副作用模型：`ui-effect/runtime-effect/telemetry-effect` 分级与渲染阶段约束

## 3. 关键约束落实

### 3.1 节点模型约束

- `createUINode` 对 `id/type` 执行非空校验，非法输入直接拒绝。
- `cloneUINode` 深层递归克隆，避免外部状态共享污染。

### 3.2 组合模式约束

- `createCompoundComponent` 使用显式 slot 定义组件结构。
- slot 支持 `required/type/multiple` 校验。
- `guardAgainstBooleanModeProps` 默认阻断 `is/has/show/with*` 布尔模式扩张。

### 3.3 事件与副作用约束

- 事件声明仅保存 handlerId，实际执行器由 `EventBindingRegistry` 管理。
- `EffectDispatcher` 对 `runtime-effect + render` 组合强制报错：
  - `DECLARATIVE_UI_RUNTIME_EFFECT_FORBIDDEN_IN_RENDER`
- 若需要渲染后执行，必须走 `schedule()` + `flushScheduledRuntimeEffects()`。
