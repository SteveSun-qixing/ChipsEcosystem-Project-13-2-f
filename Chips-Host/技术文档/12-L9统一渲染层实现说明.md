# L9 统一渲染层实现说明

> 文档状态：已完成（R0-R5 已验收）  
> 适用范围：`Chips-Host/packages/unified-rendering/*` 与 `packages/card-service/*`

---

## 1. 目标与范围

本实现对应生态架构中 L9 Unified Rendering，覆盖以下交付物：

- 统一渲染引擎（6 阶段流水线）
- 目标适配器（App Root / Card Iframe / Module Slot / Offscreen Render）
- 渲染一致性测试工具
- 增量更新调度器

不在本次范围：L10 组件库实现、外部 SDK 新接口变更。

## 3. 六阶段流水线实现映射

### 3.1 Node Normalize

- 将输入声明树转换为 `NormalizedNode`。
- 自动补齐默认属性（如 `List/Grid/Table` 的增量参数）。
- 统一节点 ID，避免匿名节点导致定位困难。

### 3.2 Contract Validate

- 校验节点类型、视觉硬编码禁用项、能力依赖。
- 非兼容能力（adapter capability 缺失）在该阶段直接失败。
- 渲染阶段 `runtime-effect` 触发被判定为违规并失败。

### 3.3 Theme Resolve

- 解析 `token.*` 属性。
- 支持 `global + scope` 的 token 覆盖。
- token 缺失时抛出 `RENDER_THEME_TOKEN_NOT_FOUND`。

### 3.4 Layout Compute

- 计算每个节点 `x/y/width/height`。
- 支持 `cpx -> px` 换算（以 `1024cpx` 为基准）。
- `List/Grid/Table` 默认计算可见区 `visibleRange`。

### 3.5 Render Commit

- 通过目标适配器将准备好的渲染树提交为目标输出。
- 输出带语义标记的 HTML 与提交元数据。

### 3.6 Effect Dispatch

- 执行 `ui-effect` / `telemetry-effect`。
- `runtime-effect` 不在渲染阶段直接执行，进入 deferred 队列。

## 4. 目标适配器

| 适配器 | 目标 | 典型能力 |
|---|---|---|
| `app-root` | 主应用根容器 | `dom/window/runtime-effect/telemetry-effect` |
| `card-iframe` | 卡片隔离渲染 | `dom/iframe/telemetry-effect` |
| `module-slot` | 模块插槽渲染 | `dom/slot/module/telemetry-effect` |
| `offscreen-render` | 离屏渲染/导出 | `offscreen/snapshot/telemetry-effect` |

---

## 5. 失败隔离策略

- 节点级：`errorBoundary.level=node`，单节点失败降级为 fallback 节点。
- 区域级：`errorBoundary.level=region`，区域内子树失败被隔离，不影响外部。
- 每次隔离都会写入诊断项：`nodeId/stage/code/message/details`。

---

## 6. 调度与增量更新

- `render-queue` 先按树拍平，再按批次拆分（默认 batch size 可配置）。
- Scheduler 支持 `shouldYield(batchIndex)` 中断，返回 `interrupted=true`。
- `List/Grid/Table` 在可见区范围内优先提交，降低大树提交成本。

## 7. 与 CardService 集成

`packages/card-service/src/card-service.ts` 现承担复合卡片显示链路的 Host 侧编排职责：

- 解析 `.card` 包中的 `metadata.yaml`、`structure.yaml` 与 `content/*.yaml`。
- 按结构顺序解析基础卡片节点，并根据节点类型匹配已启用的基础卡片插件。
- 对基础卡片配置做 Host 侧标准化，再调用插件 `renderBasecardView` 生成节点窗口内容。
- 将每个基础卡片节点编译为独立 iframe，并拼接为最终复合卡片 HTML 文档。
- 复合卡片文档只保留基础卡片节点栈，不再额外挂载标题、节点计数或节点外框等展示壳层。
- 基础卡片子 iframe 需要在首次装载、内容变化和视口宽度变化时持续回传节点高度，保证图片等自适应内容在收窄显示区域时同步回流。
- 通过 `chips.composite:ready` / `chips.composite:node-error` / `chips.composite:fatal-error` 向外层窗口报告状态。
- `CardService.render(cardFile, options)` 仍返回 `target/semanticHash/diagnostics/consistency`，供 SDK 与上层应用统一消费。
