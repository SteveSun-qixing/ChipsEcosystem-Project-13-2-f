# 卡片转HTML模块需求规格说明

## 1. 模块定位

`Chips-CardtoHTML-Plugin` 是整个转换链路的底层原子模块，负责把卡片转换为：

- 目录态 HTML 中间产物
- 压缩态 HTML 最终交付物

## 2. 功能目标

- 复用正式卡片显示链路生成 HTML
- 收集并复制资源
- 重写离线资源路径
- 生成转换清单
- 输出目录或压缩包

## 3. 功能需求

- `FR-01` 支持 `.card` 文件与目录态卡片作为输入
- `FR-02` 支持 `directory` 与 `zip` 两种输出模式
- `FR-03` HTML 页面视觉效果必须与 Host `card.render` 一致
- `FR-04` 资源文件必须在导出结果中可离线访问
- `FR-05` 生成 `conversion-manifest.json`
- `FR-06` 失败时返回结构化警告和错误

## 4. 非功能需求

- `NFR-01` 不复制一套私有基础卡片渲染器
- `NFR-02` 对资源型基础卡片必须保持路径正确
- `NFR-03` 输出目录结构必须稳定
- `NFR-04` 视觉一致性必须可回归验证

## 5. 验收口径

- HTML 导出产物能在离线环境打开
- 目录态输出可被 PDF / 图片模块直接消费
- 与 Host 正式卡片显示结果在结构和视觉上保持一致
