# 阶段三：交互与质量门禁收口

## 目标

1. 完成 README、需求文档、技术文档、开发计划；
2. 补齐纯函数与元数据解析单元测试；
3. 执行 `lint/test/build/validate`；
4. 根据验证结果回写文档与实现。

## 已验证内容

1. 启动恢复、歌词解析、音频元数据解析具备单元测试覆盖；
2. Manifest 已完整声明音频处理能力与权限；
3. 正式脚本通过：
   - `npm run lint`
   - `npm test`
   - `npm run build`
   - `npm run validate`

## 收口标准

1. 工程内无视频播放器模板残留；
2. 文档与当前 `manifest.yaml`、测试和运行时实现一致；
3. 首版音乐播放器达到可交付状态。
