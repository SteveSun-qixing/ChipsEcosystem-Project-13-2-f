# 音乐播放器总体架构设计

## 1. 项目角色

`Chips-MusicPlayer` 是薯片生态中的应用插件音乐播放器。它不承载自研解码器、媒体库索引或转码服务，只负责：

1. 恢复要播放的音频目标；
2. 为当前音频目标构建正式播放会话；
3. 发现并解析歌词、封面与嵌入元数据；
4. 承载左封面右歌词界面；
5. 调用 Host 正式文件、资源、平台、主题与多语言能力。

## 2. 分层结构

当前实现分为四层：

### 2.1 App Shell

- 文件：`src/App.tsx`
- 职责：
  - 初始化 `chips-sdk` 客户端；
  - 读取主题、语言与宿主信息；
  - 恢复启动上下文；
  - 归一音频源；
  - 协调打开、保存、拖放、伴生文件发现与嵌入元数据解析。

### 2.2 Runtime Hooks

- 文件：
  - `src/hooks/useChipsClient.ts`
  - `src/hooks/useChipsBridge.ts`
  - `src/hooks/useMusicPlayerController.ts`
- 职责：
  - 封装 SDK 与 Bridge 接入；
  - 管理 `<audio>` 元素运行时状态；
  - 收口播放、暂停、进度、音量、静音、循环与快捷键行为。

### 2.3 View Stage

- 文件：`src/components/MusicPlayerStage.tsx`
- 职责：
  - 只负责展示与事件绑定；
  - 不直接知道 Host 路由细节；
  - 组合封面区、时间轴、控制 dock、歌词区与反馈浮层。

### 2.4 Pure Utils

- 文件：
  - `src/utils/music-player.ts`
  - `src/utils/launch-resource.ts`
  - `src/utils/lyrics.ts`
  - `src/utils/lyrics-layout.ts`
  - `src/utils/color.ts`
  - `src/utils/audio-metadata.ts`
- 职责：
  - 保持纯函数；
  - 处理文件名、格式支持、启动参数归一、歌词解析、歌词布局计算、按钮取色、时间格式化与 MP3 / FLAC 元数据读取。

## 3. 依赖边界

项目只依赖：

1. React
2. `@chips/component-library` 的主题运行时壳层与 `ChipsIcon`
3. `chips-sdk`
4. 原生 HTML5 `<audio>`
5. 浏览器内建 `TextDecoder`、`Blob`、`URL.createObjectURL`

项目不依赖：

1. Electron 私有对象
2. Host 内部源码模块
3. 第三方音频播放器 npm 运行时
4. 第三方标签解析 npm 包

## 4. 参考项目适配策略

本项目参考 `MusicPlayer-main` 的部分思路包括：

1. 左封面右歌词的双栏布局；
2. 轻量控制区与大号歌词视觉层级；
3. 同时处理音频、歌词、封面这组三元资源。

正式实现不直接导入参考仓库脚本，而是按薯片生态规范重写为：

1. `useMusicPlayerController(...)`
2. `MusicPlayerStage`
3. `parseLyricsText(...)`
4. `getLyricsLayoutPosition(...)`
5. `parseEmbeddedAudioMetadata(...)`

这样可以保证背景、歌词和主视觉仍贴近参考项目，但正式契约、多语言系统、Host 接线与资源处理链路全部服从薯片生态标准。
