# 启动恢复与音频资源解析设计

## 1. 正式输入来源

音乐播放器当前接受四类正式输入：

1. `launchParams.resourceOpen.filePath`
2. `launchParams.targetPath`
3. `launchParams.resourceOpen.resourceId`
4. 文件选择器 / 拖放得到的一组本地文件路径

其中第四类允许同时出现：

1. 一首音频；
2. 一张封面；
3. 一个歌词文件。

## 2. 启动目标归一

实现位置：

- `src/utils/launch-resource.ts`
- `src/App.tsx`

归一顺序：

1. `resourceOpen.filePath`
2. `targetPath`
3. `resourceOpen.resourceId`

输出结构：

```ts
interface LaunchAudioTarget {
  sourceId: string;
  filePath?: string;
  fileName?: string;
  mimeType?: string;
  title?: string;
}
```

## 3. 播放 URI 解析

实现位置：

- `src/App.tsx`

解析策略：

1. 若目标本身已是 `file://`、`http://`、`https://`、`blob:` 或 `data:`，直接作为可播放 URI；
2. 否则调用正式 `client.resource.resolve(...)`；
3. 解析完成后才进入音乐播放会话。

这样做的原因：

1. 本地文件路径必须通过正式资源解析转为可消费 URI；
2. 远端 URL 不应被错误当作本地路径二次包装；
3. 上层应用无需关心 Host 路由内部资源格式，只需要交给音乐播放器恢复会话。

## 4. 本地伴生资源发现

实现位置：

- `src/App.tsx`
- `src/utils/music-player.ts`
- `src/utils/binary.ts`

当音频存在稳定本地路径时，播放器会先查看用户显式选择的补充文件；若没有，再到同目录发现伴生资源。

封面候选名：

1. 与音频同名的 `.jpg/.jpeg/.png/.webp/.avif`
2. `cover.*`
3. `folder.*`
4. `front.*`

歌词候选名：

1. 与音频同名的 `.lrc`
2. `lyrics.lrc`
3. 与音频同名的 `.txt`

伴生歌词读取补充：

1. 伴生歌词统一通过 `client.file.read(path, { encoding: "binary" })` 读取；
2. Host 当前 `file.read` 返回形态为 `{ content }`，应用侧需要先解包 `content`；
3. 若解包后的结果是 `Uint8Array`，则直接进入 `decodeTextWithFallback(...)`；
4. 若解包后的结果是字符串，则先按生态现有链路还原为 `base64` 二进制内容，再做编码探测；
5. 这样可以避免 GBK / Big5 / Shift_JIS 等歌词文件被错误按 UTF-8 二次编码，导致乱码或解析失败。

## 5. 嵌入元数据解析

实现位置：

- `src/utils/audio-metadata.ts`

当前正式支持：

1. MP3 ID3v2：
   - `TIT2`
   - `TPE1`
   - `TALB`
   - `USLT`
   - `SYLT`
   - `APIC`
2. FLAC：
   - Vorbis Comment 中的 `TITLE / ARTIST / ALBUM / LYRICS`
   - `PICTURE` block

优先级策略：

1. 标题：`launch title > embedded title > 歌词元数据 title > 文件名`
2. 歌词：`显式歌词文件 > 自动发现歌词文件 > 嵌入歌词`
3. 封面：`显式或自动发现封面 > 嵌入封面覆盖 > 默认封面`

## 6. 支持格式判断

实现位置：

- `src/utils/music-player.ts`

判断规则：

1. `mimeType` 以 `audio/` 开头时优先判为支持；
2. 否则按本地路径或资源 ID 后缀匹配扩展名白名单；
3. 未命中白名单则拒绝打开并给出反馈。

## 7. 保存链路

实现位置：

- `src/App.tsx`

保存策略：

1. 本地文件路径存在时，使用 `platform.saveFile + file.copy`；
2. 远端或无稳定本地路径时，回退为下载链接；
3. 覆盖同路径时显式拒绝。
