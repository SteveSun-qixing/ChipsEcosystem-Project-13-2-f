# 播放会话与歌词交互控制设计

## 1. 控制器职责

实现位置：

- `src/hooks/useMusicPlayerController.ts`

控制器统一管理以下状态：

1. `isReady`
2. `isPlaying`
3. `isBuffering`
4. `isMuted`
5. `volume`
6. `duration`
7. `currentTime`
8. `bufferedUntil`
9. `loopMode`
10. `errorKey`

## 2. 状态来源

状态只来自两类输入：

1. 原生 `<audio>` 事件：
   - `loadedmetadata`
   - `canplay`
   - `playing`
   - `pause`
   - `waiting`
   - `ended`
   - `timeupdate`
   - `progress`
   - `volumechange`
   - `error`
2. 应用层传入的 `sessionKey`

## 3. 控制动作

控制器对外暴露：

1. `togglePlayback`
2. `seekBy`
3. `seekTo`
4. `setVolumeLevel`
5. `toggleMute`
6. `toggleLoopMode`
7. `handleKeyDown`

## 4. 快捷键模型

首版快捷键固定为：

1. 空格：播放 / 暂停
2. 左右方向键：前后 10 秒
3. 上下方向键：音量加减
4. `M`：静音切换
5. `L`：单曲循环切换

补充限制：

1. 当焦点在 `input/textarea/select` 上时不抢快捷键；
2. 快捷键逻辑只作用于当前播放器表面容器。

## 5. 歌词高亮模型

实现位置：

- `src/utils/lyrics.ts`
- `src/utils/lyrics-layout.ts`
- `src/components/MusicPlayerStage.tsx`

当前策略：

1. 歌词来源优先级固定为：`伴生歌词文件 > 嵌入时间轴歌词 > 嵌入纯文本歌词 > 空态`
2. 先把 LRC 解析成 `{ timeMs, text }[]`
3. 播放时基于当前 `currentTime * 1000` 二分查找当前高亮索引
4. 视图层按参考项目 `GetLyricsLayout / UpdateLyricsLayout` 的同构逻辑，基于真实行高和 `viewportHeight / 3.5` 计算每一行位移
5. 当前激活行不再整句直接跳白，而是根据“当前句开始时间 -> 下一句开始时间”的区间计算亮色遮罩进度；
6. 已播过的句子保持完整亮色遮罩，当前句按播放进度渐进填充，未播句保持暗态；
7. 相邻歌词仍按与高亮行的距离叠加 `blur(...)`，保留参考项目的空间层次
8. 亮色遮罩不再只依赖 `<audio>` 的 `timeupdate` 频率，而是播放时直接读取 `audio.currentTime` 做帧级更新，暂停 / 拖动时再做一次静态校准；
9. 逐行位移更新使用分段延迟，保留参考项目的滚动节奏
10. 歌词容器尺寸变化时通过 `ResizeObserver` 触发布局重算，避免右侧区域展开动画或窗口尺寸变化后歌词堆叠错位

若当前歌词不是时间轴歌词，则切换为静态文本模式，不做自动滚动。

## 6. 会话切换

当 `sessionKey` 变化时，控制器会：

1. 停止当前播放；
2. 重置进度、缓冲、错误与就绪状态；
3. 保留当前音量与循环偏好；
4. 等待新的 `<audio>` 元素建立元数据与播放状态。

这样可以保证切歌时不会把上一首的瞬时状态带进下一首。
