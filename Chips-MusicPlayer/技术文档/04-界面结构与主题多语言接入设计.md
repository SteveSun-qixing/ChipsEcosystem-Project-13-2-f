# 界面结构与主题多语言接入设计

## 1. Stage 结构

实现位置：

- `src/components/MusicPlayerStage.tsx`
- `src/global.css`

页面由四层组成：

1. `background`
   - 直接移植参考项目的多层 radial-gradient 色团背景；
   - 颜色来自当前封面主色提取结果。
2. `canvas`
   - 按参考项目的四象限切片旋转逻辑绘制封面氛围层；
   - 通过模糊、饱和度和亮度滤镜形成动态流动背景。
3. `main > leftcontent`
   - 直接沿用参考项目的 `svgcontainer / time / processbar / conbox` 布局骨架；
   - 在原骨架下补入薯片生态要求的元数据行、补充控制与音量条。
4. `main > rightcontent`
   - 承载 `lyricscontainer / lyrics`；
   - 时间轴歌词使用参考项目同构布局算法；
   - 纯文本歌词和空态歌词保持同一视觉语系下的静态回退。
5. `main--mobile-pager`
   - 当窗口进入“窄且高”的移动端比例时启用；
   - 左右两个主区域转为横向 `scroll-snap` 双页面；
   - 第一页保留封面与控制，第二页承载歌词，仅在顶部显示两个小圆点作为页码提示与快速切换入口；
   - 页面切换同时支持触摸屏滑动、触摸板横向滚动和鼠标按住拖拽。

## 2. 视觉策略

当前视觉方向：

1. 背景、歌词遮罩和左右收展动画按参考项目前端直接还原；
2. 参考仓库的字体资源已复制到 `assets/fonts/`，用于尽量贴近原始观感；
3. 所有控制按钮统一使用 `ChipsIcon` 与正式 `aria-label`，只保留参考项目的布局、大小和动效节奏；
4. 按钮颜色不再固定，而是从当前封面主色提取结果中优先选取“高饱和且足够明亮”的强调色；取色正式流程为“先降采样，再对封面色貌做高斯模糊，借此压平黑白极值和局部刺眼细节，再去掉剩余极值并做显著色评分与聚类”，最后再把结果提亮提饱和，生成更亮、更荧光的控制色板；
5. 播放 / 辅助控制按钮不再显示可见的圆形底色，只保留点击热区、图标本身的颜色变化和辉光；`ChipsIcon` 颜色通过显式 `--chips-icon-color` 注入，避免落回主题默认黑色；
6. 所有可点击按钮在悬浮或聚焦停留一小段时间后显示名称气泡；
7. 没有音频时沿用参考项目 `body.no-files` 动画，让右侧歌词区收起、左侧封面区居中；
8. 纯文本歌词与空歌词场景不新造一套界面，而是在原歌词视觉体系内做回退。
9. 小屏不再存在封面区和歌词区的上下堆叠；非移动端比例继续保持左右结构，仅按窗口尺寸缩放内容；
10. 移动端比例下切换成横向滑页，避免在小屏中同时压缩封面和歌词。

## 3. 多语言接入

实现位置：

- `i18n/zh-CN.json`
- `i18n/en-US.json`
- `src/i18n/messages.ts`

规则：

1. 所有项目界面文本都通过 `formatMessage(...)` 获取；
2. `language.changed` 事件触发后直接切换当前 locale；
3. 提示文案、按钮 `aria-label`、歌词空态和错误态全部走词典；
4. 运行时媒体错误通过 `errorKey -> i18n key` 映射展示，而不是直接输出浏览器默认错误。
5. 移动端小圆点页码也通过词典提供可访问名称，避免无标签按钮。

## 4. 主题接入

实现位置：

- `src/App.tsx`
- `src/global.css`

规则：

1. 首屏优先读取文档注入的主题快照；
2. 再通过 `client.theme.getCurrent()` 对齐当前正式主题；
3. `ChipsThemeProvider` 统一监听 `theme.changed`；
4. 音乐播放器首版为还原参考项目视觉，主体背景与歌词层级使用固定浅色设计和封面主色驱动，不再对其做额外主题变体；
5. 运行时会从封面主色里抽取“最亮颜色”，再生成 `base / strong / rim / glow / foreground` 控制色板，驱动按钮和范围控件；
6. 正式主题链路当前主要用于宿主挂载、主题快照同步与未来拓展，不改变本次直接移植的背景和歌词设计。
