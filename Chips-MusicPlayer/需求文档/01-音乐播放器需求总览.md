# 音乐播放器需求总览

## 1. 文档目标

冻结 `Chips-MusicPlayer` 首版的产品边界、正式使用场景、生态接入方式与交付目标，作为实现与验收依据。

## 2. 产品定位

`Chips-MusicPlayer` 是运行在薯片 Host 内部的 `type: app` 应用插件，负责承接“打开一首音频并进入沉浸式播放界面”这一单一职责。

首版只做以下事情：

1. 打开音频；
2. 播放音频；
3. 展示封面与歌词；
4. 保存音频副本；
5. 作为 Host 统一资源路由中的正式音频处理器。

本项目不是：

1. 音乐编辑器；
2. 媒体资源管理器；
3. 播放列表与媒体库中心；
4. 流媒体聚合平台；
5. 自研解码或转码服务。

## 3. 正式接入场景

### 3.1 Host 资源路由启动

- 上游应用调用正式 `client.resource.open(...)`；
- Host 通过 `resource-handler:view:audio/*` 自动命中音乐播放器；
- 启动上下文通过 `launchParams.resourceOpen` / `targetPath` 注入音频资源；
- 音乐播放器恢复到对应播放会话。

### 3.2 系统文件关联打开

- Host 通过 `file-handler:.mp3/.flac/...` 接收音频文件；
- 用户双击受支持格式时，Host 拉起音乐播放器；
- 音乐播放器读取 `targetPath` 恢复会话。

### 3.3 应用内手动打开

- 用户在播放器内部使用正式文件选择对话框选择本地音频；
- 用户可以同时选择歌词与封面文件；
- 应用通过正式 SDK / Bridge 接线打开并播放。

### 3.4 拖放打开

- 用户把音频文件或“音频 + 歌词 + 封面”组合拖到播放器窗口；
- 播放器通过正式 `platform.getPathForFile(...)` 恢复本地路径；
- 应用切换到新的播放会话。

## 4. 首版总体目标

1. 与图片查看器、视频播放器保持一致的正式应用插件架构；
2. 与 Host 资源路由、文件关联、主题、多语言和 Bridge / SDK 链路保持一致；
3. 满足本地音频与被路由音频的单曲播放需求；
4. 在不引入第三方运行时包的前提下，补齐歌词、封面与嵌入元数据体验。

## 5. 参考约束

- 界面构图直接参考开源项目 `MusicPlayer-main`；
- 但正式实现必须完全服从薯片生态的插件边界、主题系统、多语言系统与 Host 运行时接线；
- 参考仓库内测试音频不进入本仓交付物。
