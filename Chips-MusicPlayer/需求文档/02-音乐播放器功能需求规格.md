# 音乐播放器功能需求规格

## 1. 音频格式支持

首版正式支持：

- `.mp3`
- `.flac`
- `.wav`
- `.ogg`
- `.oga`
- `.m4a`
- `.aac`
- `.opus`
- `.webm`

说明：

- 首版只接入浏览器 / Host 当前运行时中可直接由 HTML5 `<audio>` 消费的基础格式；
- 不在本仓内新增自定义解码链路、流媒体引擎或转码兜底逻辑。

## 2. 音频打开

应用必须支持：

1. 读取 `launchParams.resourceOpen`；
2. 读取 `launchParams.resourceOpen.payload` 中的正式音乐基础卡片播放上下文；
3. 读取 `launchParams.targetPath`；
4. 使用正式文件选择器选择一组本地文件；
5. 通过拖放打开一组本地文件；
6. 对缺失路径、不支持格式、资源解析失败给出明确反馈。

## 3. 播放能力

应用必须提供以下基础播放能力：

1. 播放；
2. 暂停；
3. 时间轴拖动；
4. 前进 10 秒；
5. 后退 10 秒；
6. 静音；
7. 音量调节；
8. 单曲循环开关。

补充要求：

1. 必须有空态、加载态、成功态、错误态；
2. 封面、时间行、进度条与控制 dock 的骨架必须稳定；
3. 播放区不能因为歌词数量、封面比例或元数据缺失而抖动布局。

## 4. 封面与歌词

应用必须支持：

1. 同目录伴生封面自动发现；
2. 同目录伴生歌词自动发现；
3. 常见嵌入歌曲信息、封面与歌词读取：
   - MP3 `ID3v2 / ID3v1`
   - FLAC `Vorbis Comment / PICTURE`
   - MP4 / M4A / AAC `moov.udta.meta.ilst`
   - Ogg Vorbis / Opus `comment packet / OpusTags`
4. 时间轴歌词滚动高亮；
5. 纯文本歌词静态展示；
6. 没有歌词时显示明确空态。

补充要求：

1. 当 Host 只提供正式资源 ID 而未提供本地文件路径时，播放器仍必须继续尝试读取嵌入元数据；
2. 当歌词标签内容本身已经是 LRC 时间轴文本时，播放器必须按时间轴歌词处理，而不是退化为纯文本。
3. 当上游通过 `resourceOpen.payload` 透传音乐基础卡片上下文时，播放器必须优先恢复卡片配置中的封面、歌词和标题，而不是只把它当作单一音频文件处理。

## 5. 播放器界面结构

首版正式界面必须满足：

1. 左侧是大封面区域；
2. 右侧是歌词区域；
3. 封面下方依次展示时间行、元数据行、进度条和控制 dock；
4. 控制按钮统一走薯片生态标准图标链路，并保持参考项目的布局和尺寸节奏；
5. 控制按钮视觉采用“图标直接变色”的方式，不显示独立圆形按钮底；
6. 背景要以当前封面主色生成色团氛围层，并叠加画布切片模糊扩散；
7. 无文件时仍保持完整骨架和打开提示，并沿用参考项目的左右区域收展动画；
8. 当窗口进入移动端纵向比例时，界面切换为两个横向滑动页面：
   - 第一个页面显示封面、元数据、进度和控制按钮；
   - 第二个页面显示歌词；
   - 页面顶部只允许出现两个小圆点作为状态提示与切换入口，不增加明显的标签菜单条；
   - 页面切换需要同时支持触摸屏滑动、触摸板横向滚动和鼠标拖拽；
9. 不再提供封面区和歌词区的竖向上下排列布局。

## 6. 键盘交互

首版键盘语义固定为：

1. `Space`：播放 / 暂停；
2. `ArrowLeft`：后退 10 秒；
3. `ArrowRight`：前进 10 秒；
4. `ArrowUp`：增大音量；
5. `ArrowDown`：减小音量；
6. `M`：切换静音；
7. `L`：切换单曲循环。

## 7. 保存音频

“保存音频”在首版中的正式语义是“将当前音频源保存为一个副本”。

要求如下：

1. 对本地文件源，使用正式保存对话框选择目标路径，并通过正式 `file.copy` 完成复制；
2. 对远端 URL 或不具备稳定本地路径的资源，回退为浏览器 / 宿主下载；
3. 目标路径与原始路径相同时，必须给出明确提示，不执行重复写入；
4. 成功、失败、取消都要有可见状态反馈。

## 8. Manifest 与生态能力要求

应用必须正式声明：

1. `resource-handler:view:audio/*`
2. 常见音频扩展名的 `file-handler`
3. `resource.read`
4. `file.read`
5. `file.write`
6. `platform.read`
7. `theme.read`
8. `i18n.read`

## 9. 明确不在首版范围

以下能力不进入当前交付：

1. 播放列表、随机播放、媒体库；
2. 均衡器、音效器、频谱可视化；
3. 歌词编辑与时间轴校对；
4. 外挂网络歌词搜索；
5. 音频剪辑、转码、格式转换；
6. 账户登录与云同步。
