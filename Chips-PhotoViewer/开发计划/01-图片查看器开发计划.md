# 图片查看器开发计划

## 1. 计划目标

本计划用于记录 `Chips-PhotoViewer` 首版的实施顺序、当前完成情况与后续扩展边界。

## 2. 阶段状态

### 阶段一：项目基线冻结

状态：已完成

已交付：

1. 使用官方应用插件脚手架初始化项目；
2. 清理脚手架身份残留；
3. 冻结需求文档、技术文档、开发计划；
4. 建立图片查看器相关生态工单与项目记录。

### 阶段二：图片打开与查看基线

状态：已完成

已交付：

1. `launchParams.targetPath` 启动恢复；
2. 应用内手动选择图片文件；
3. 单图查看态；
4. 空态、加载态、错误态；
5. 路径、分辨率、缩放比例、文件大小信息区。

### 阶段三：缩放与保存

状态：已完成

已交付：

1. 放大；
2. 缩小；
3. 适应窗口；
4. 实际大小；
5. 保存图片副本。

### 阶段四：Host 接入联调

状态：已完成

已交付：

1. Host 通用 `file-handler:<ext>` 文件关联分发；
2. Host 启动入口对普通扩展名文件的接收；
3. 安装器根据内置 app 插件能力条件注册图片类型；
4. 图片查看器作为可选内置插件接入 Host 分发链路；
5. “没安装插件就不接图片”的发行级行为收口。

### 阶段五：后续生态扩展

状态：后续规划

后续仅在共享契约冻结后再进入：

1. 非文件路径图片资源转交；
2. 多处理器优先级治理；
3. 动态卸载 / 回收系统文件关联治理。

## 3. 当前交付清单

截至 2026-04-02，首版已交付：

1. 图片查看器正式应用插件；
2. Host 泛化文件关联接线；
3. 安装器能力驱动的图片文件关联注册；
4. 项目文档、Host 文档、工单记录同步更新；
5. 与改动范围匹配的单元测试、集成测试、契约测试和正式脚本验证。

## 4. 验证计划与结果

本轮已实际执行并通过：

1. `Chips-PhotoViewer`
   - `npm run lint`
   - `npm test`
   - `npm run build`
   - `npm run validate`
2. `Chips-Host`
   - `npm run build`
   - `npm test`
   - `npm run test:contract`

## 5. 收口结论

图片查看器首版已经完成当前产品范围内的正式交付。

后续如需继续扩展，优先顺序应为：

1. 共享资源转交契约；
2. 默认处理器治理；
3. 安装器 / 卸载器回收策略。
