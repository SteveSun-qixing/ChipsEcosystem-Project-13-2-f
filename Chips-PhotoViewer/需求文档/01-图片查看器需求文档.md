# 图片查看器需求文档

## 1. 文档目标

本文档用于冻结 `Chips-PhotoViewer` 首版的正式产品范围、可交付能力、生态接入边界与验收标准。

## 2. 产品定位

`Chips-PhotoViewer` 是运行在薯片 Host 内部的 `type: app` 应用插件，首版只负责三件事：

1. 查看图片；
2. 缩放图片；
3. 保存图片副本。

本项目不是图片编辑器、媒体资源管理器、相册应用，也不在仓库内私建第二套文件分发协议。

## 3. 正式支持的使用场景

### 3.1 显式 `targetPath` 启动

- 其他应用或 Host 已明确知道目标图片的本地文件路径；
- 通过正式启动上下文 `launchParams.targetPath` 拉起图片查看器；
- 图片查看器恢复到目标图片的查看状态。

### 3.2 应用内手动打开

- 用户在图片查看器中主动选择本地图片文件；
- 应用通过正式文件对话框打开图片。

### 3.3 系统文件关联打开

- Host 安装包若携带内置 `Chips-PhotoViewer`，安装器会根据插件 manifest 中的 `file-handler:<ext>` 能力，为对应图片类型写入系统文件关联；
- 用户双击这些图片文件时，Host 会把文件关联输入路由到图片查看器；
- 若 Host 发行包没有携带图片查看器插件，则不会注册这些图片类型，Host 也不会接住这些图片文件。

## 4. 首版功能需求

### 4.1 图片打开

首版支持以下格式：

- `.png`
- `.jpg`
- `.jpeg`
- `.webp`
- `.gif`
- `.bmp`
- `.svg`
- `.avif`

应用必须支持：

1. 从启动上下文读取图片路径；
2. 通过正式文件对话框选择图片；
3. 对不存在路径、不可读路径、非支持格式给出明确错误反馈。

### 4.2 图片查看

应用必须提供单图查看态，并满足：

1. 主区域始终只承载当前一张图片；
2. 提供空态、加载态、成功态、错误态；
3. 图片显示窗口必须稳定撑满整个应用窗口，不能随着图片缩放而改变自身骨架尺寸；
4. 首次打开图片时默认按长边 80% 规则缩放：
   横图按应用窗口宽度的 80%，竖图按应用窗口高度的 80%，保证首屏尺寸稳定且不过度撑满窗口；
5. 底部操作控制使用浮动 dock，并固定在窗口视口底部，不因图片尺寸变化而位移；
6. 默认不显示图片信息弹窗或信息面板；
7. 不提供多图列表、历史栈、缩略图侧栏、幻灯片等额外能力。

### 4.3 图片缩放

应用必须提供以下查看控制：

1. 放大；
2. 缩小；
3. 适应窗口；
4. 实际大小。

并且要适配常见输入方式：

1. 鼠标滚轮缩放；
2. 鼠标拖动画面平移；
3. 触摸板双指滚动平移；
4. 触摸板捏合缩放；
5. 触摸屏单指拖动与双指捏合。

缩放行为只影响查看态，不引入旋转、裁剪、镜像、滤镜等编辑动作。

交互细则补充：

1. 触控板普通双指滚动优先保持平移语义；
2. 鼠标滚轮与触控板捏合都可以触发缩放；
3. 不检查当前缩放倍率或图片是否超出视口，任何时候都允许直接拖动画面。

### 4.4 图片保存

“保存图片”在首版中的正式语义为“把当前图片复制到用户选择的新位置”。

要求如下：

1. 输出路径由 Host 正式保存对话框选择；
2. 默认保留原始文件内容，不做转码；
3. 成功、失败、取消都需要有明确状态反馈；
4. 当目标路径与当前图片路径相同，应用要给出清晰提示，不执行无意义写入。

### 4.5 生态一致性

应用必须接入以下正式链路：

1. `chips-sdk` / `window.chips.*`；
2. 主题系统；
3. 多语言系统；
4. Host 启动上下文；
5. Host 文件服务与对话框服务。

## 5. 明确不在首版范围

以下能力不进入本轮交付：

1. 图片编辑、裁剪、旋转、批注、滤镜；
2. 相册、收藏夹、时间线、媒体库管理；
3. 批量打开、批量导出、目录浏览；
4. 非文件路径图片资源的跨应用票据化转交；
5. 多图片查看器冲突时的用户偏好治理界面。

## 6. 生态边界与结论

### 6.1 本轮已正式闭环的部分

当前已经闭环：

1. `targetPath` 显式打开；
2. 应用内手动打开；
3. 基于 Host 文件关联输入的图片查看器自动接收；
4. 安装器按内置插件能力条件注册图片文件类型；
5. “没有安装图片查看器插件时，Host 不接图片”。

### 6.2 暂未扩展的部分

以下内容仍属于后续生态扩展，而非首版阻塞：

1. 其他应用只有临时资源、没有稳定文件路径时的图片转交；
2. 多处理器并存时的默认处理器优先级治理；
3. 安装后动态卸载或禁用图片查看器时的系统文件关联回收治理。

## 7. 验收基线

首版验收至少包括：

1. 通过 `launchParams.targetPath` 能稳定打开受支持图片；
2. 通过正式文件对话框能稳定打开受支持图片；
3. 放大、缩小、适应窗口、实际大小行为正常；
4. 能通过正式保存对话框把图片复制到用户指定位置；
5. 主题和语言切换不会破坏应用壳层；
6. `npm run lint`、`npm test`、`npm run build`、`npm run validate` 全部通过。
