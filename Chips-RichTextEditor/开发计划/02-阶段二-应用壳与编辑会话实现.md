# 阶段二：应用壳与编辑会话实现

## 阶段目标

完成 Typora 风格单文档壳、富文本基础卡片编辑装配、复合卡片会话治理与正式保存链路。

## 当前状态

状态：进行中

当前已完成的基线准备：

1. 脚手架项目已建立；
2. `manifest.yaml`、图标、i18n、日志与基础 helper 已初始化；
3. 富文本基础卡片配置、复合卡片文件构建、运行时目录辅助模块已开始落地；
4. 脚手架示例残留已开始归档清理。

## 本阶段开发任务

### 1. 应用壳实现

1. 新建文稿；
2. 打开文稿；
3. 保存与另存为；
4. 自动保存；
5. 未保存改动确认；
6. 启动参数恢复；
7. 右上角菜单、信息弹窗与状态反馈。

### 2. 编辑器装配

1. 通过 `client.card.editorPanel.render(...)` 装配 `base.richtext`；
2. 稳定接入 `onReady / onChange / onError`；
3. 保证普通状态刷新不重建正文编辑器。

### 3. 会话与持久化

1. 打开时解包 `.card` 到隐藏运行时目录；
2. 保存时重建目录态卡片并重新打包；
3. 管理 Markdown 文件模式下的待导入资源与待删除资源；
4. 重建 `structure.yaml manifest.resources`。

## 退出条件

1. 可新建、打开、保存、另存为；
2. 合法 `.card` 可稳定二次打开；
3. 长正文和短正文两种保存模式都能闭环；
4. 壳层交互与正文编辑器装配稳定。
