# 阶段三：测试、文档同步与交付收口

## 阶段目标

在功能完成后补齐验证、同步文档并完成首版交付收口。

## 开发任务

### 1. 测试补齐

1. 增补文件模型和持久化相关单元测试；
2. 清理脚手架示例测试并替换为项目真实测试；
3. 进行一轮 Host 开发态手动联调。

### 2. 文档同步

1. 根据最终代码回写 README；
2. 复核需求文档、技术手册、开发计划是否与最终实现一致；
3. 如发现公共契约变更，再评估是否需要同步到生态共用技术文档。

### 3. 正式验证

执行并通过：

1. `npm run lint`
2. `npm test`
3. `npm run build`
4. `npm run validate`

## 退出条件

1. 文档与代码口径一致；
2. 正式验证脚本通过；
3. 无阻断级缺陷；
4. 交付内容符合首版需求文档与技术手册定义。
