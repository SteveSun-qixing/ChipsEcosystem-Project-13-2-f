# 富文本编辑器系统概览与依赖边界

## 1. 项目身份

- 插件类型：`type: app`
- 插件 ID：`com.chips.rich-text-editor`
- 运行入口：`dist/index.html`
- 技术栈：React + `chips-sdk` + `@chips/component-library`

## 2. manifest 冻结口径

当前项目已冻结以下权限：

1. `card.read`
2. `card.write`
3. `file.read`
4. `file.write`
5. `platform.read`
6. `theme.read`
7. `i18n.read`

这些权限对应的正式能力分别用于：

- `.card` 解包 / 打包；
- 运行时工作目录读写；
- 文件对话框与确认弹窗；
- 主题与多语言状态读取。

## 3. 应用内部模块划分

首版实现按以下层级组织：

### 3.1 应用壳层

职责：

1. 极简界面编排、右上角菜单与信息弹窗；
2. 新建、打开、保存、另存为；
3. 自动保存与未保存确认；
4. 主题、多语言、状态反馈；
5. 启动参数恢复。

### 3.2 富文本编辑器宿主层

职责：

1. 调用 `client.card.editorPanel.render(...)` 装配 `base.richtext` 编辑器；
2. 订阅 `onReady / onChange / onError`；
3. 为富文本基础卡片提供资源解析、导入、删除桥。

### 3.3 复合卡片会话层

职责：

1. 维护 `cardId / baseCardId / filePath / dirty / runtimePaths`；
2. 维护待写入资源与待删除资源；
3. 在保存时把当前会话收敛为正式复合 `.card` 文件。

### 3.4 文件持久化层

职责：

1. 调用 `card.unpack` 打开现有 `.card`；
2. 在隐藏运行时工作目录中重建目录态卡片；
3. 写入 `.card/metadata.yaml`、`.card/structure.yaml`、`.card/cover.html` 与 `content/<baseCardId>.yaml`；
4. 重建 `manifest.resources`；
5. 调用 `card.pack` 输出正式 `.card` 文件。

## 4. 正式依赖边界

### 4.1 允许依赖

- `chips-sdk`
- `@chips/component-library`
- `yaml`

### 4.2 禁止依赖

1. 直接 import `Chips-BaseCardPlugin/richtext-BCP` 源码作为正式运行时；
2. 直接 import Host 内部服务实现；
3. 在应用内私建第二套 `.card` 打包器或解包器。

## 5. 当前生态限制

首版不在 manifest 中声明通用 `.card` 文件处理能力，原因如下：

1. 现有 Host 对同扩展名 app 处理器的选择策略尚未形成完整治理；
2. 当前工作区已有 `Chips-CardViewer` 可接收 `.card`；
3. 若本应用也声明 `.card`，会造成普通 `.card` 文件路由歧义。

因此，首版只支持：

1. 应用内手动打开；
2. 显式 `launchParams.targetPath` 恢复；
3. 手动保存 / 另存为输出。
