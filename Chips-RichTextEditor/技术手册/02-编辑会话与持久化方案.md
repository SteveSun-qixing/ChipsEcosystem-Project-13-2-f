# 富文本编辑器编辑会话与持久化方案

## 1. 会话模型

每个编辑窗口只维护一个文稿会话。会话的核心字段冻结为：

1. `cardId`
2. `baseCardId`
3. `title`
4. `filePath`
5. `runtimePaths`
6. `dirty`
7. `pendingImports`
8. `pendingDeletions`
9. `editorSessionKey`

其中：

- `cardId` 是复合卡片 ID；
- `baseCardId` 是唯一富文本基础卡片节点 ID；
- `runtimePaths` 是当前文稿的隐藏运行时工作目录；
- `editorSessionKey` 用于保证只有在新建 / 打开新文稿时才重建编辑器 iframe。

## 2. 运行时工作目录

当文稿已经绑定正式路径后，应用在目标文件同级目录下建立隐藏运行时目录：

```text
<target-parent>/.chips-rich-text-editor-runtime/<sessionId>/card
```

说明：

1. `card/` 内部结构与正式 `.card` 解包态目录一致；
2. 打开现有文稿时，先把 `.card` 解包到该目录；
3. 保存时以该目录态卡片为基础重建正式输出；
4. 该目录只属于本应用内部运行时，不作为生态公共契约暴露。

## 3. 打开流程

### 3.1 输入

- `targetFilePath`
- 当前新建的 `sessionId`

### 3.2 步骤

1. 计算隐藏运行时目录；
2. 清理同会话旧运行时目录；
3. 调用 `client.card.unpack(targetFilePath, runtimePaths.workDir)`；
4. 读取：
   - `.card/metadata.yaml`
   - `.card/structure.yaml`
   - `content/<baseCardId>.yaml`
5. 校验是否为“单富文本基础卡片复合卡片”；
6. 生成应用层会话状态。

## 4. 保存流程

### 4.1 目标

把当前内存会话收敛为一个正式可复开的复合 `.card` 文件。

### 4.2 步骤

1. 根据目标输出路径计算会话运行时目录；
2. 在运行时目录下创建临时 `card-stage-*` 目录；
3. 写入复合卡片元数据与正文配置；
4. 把当前富文本基础卡片引用到的资源写入临时目录；
5. 扫描卡片根目录真实资源，重建 `structure.yaml manifest.resources`；
6. 调用 `client.card.pack(stageDir, outputPath)`；
7. 保存成功后，用最新的 `stageDir` 替换当前会话正式 `workDir`；
8. 清空已提交的待导入 / 待删除资源状态。

## 5. 正文资源模型

富文本基础卡片的正文存在两种正式保存模式：

### 5.1 内联模式

- `content_source: inline`
- Markdown 正文直接写入 `content/<baseCardId>.yaml`

### 5.2 文件模式

- `content_source: file`
- `content_file` 保存相对于卡片根目录的 `.md` 路径
- 正文文件实体保存在卡片根目录中，例如：
  - `richtext-xxxx.md`

## 6. 资源桥策略

应用壳向 `editorPanel` 提供本地资源桥，用于处理：

1. `resolveResourceUrl`
2. `importResource`
3. `deleteResource`
4. `releaseResourceUrl`

具体策略：

- 已保存到工作目录的资源，优先解析为稳定 `file://` URL；
- 尚未落盘但已进入 `pendingImports` 的资源，临时解析为 `blob:` URL；
- 删除动作先登记为待删除意图，最终通过保存阶段的重建目录收口；
- 正式 YAML 中永远只记录相对资源路径，不写回 `blob:`、`data:` 或绝对路径。

## 7. 封面与标题

复合卡片封面由应用壳生成默认 `cover.html`。首版不引入独立封面编辑器，规则如下：

1. 已存在的正式标题沿用当前 `metadata.name`；
2. 新建草稿首次保存前若仍停留在默认草稿标题，则用保存文件名推导正式标题；
3. 保存时用当前正式标题重新生成默认封面 HTML；
4. 空标题在落盘前回退为“未命名文稿”或文件名推导值。
