# 富文本编辑器界面壳、主题与多语言接入

## 1. 界面结构

首版界面固定由三类正式元素组成：

1. Electron 原生窗口标题栏；
2. 页面内右上角单入口菜单按钮；
3. 中央连续正文编辑区；
4. 按需出现的信息弹窗与底部轻提示。

常驻页面不保留额外文件信息条、底部状态栏或自绘标题栏。正文承载区内部只放正式富文本基础卡片编辑器 iframe，不再额外套纸张式卡片壳。

## 2. 交互编排

### 2.1 菜单

- 菜单按钮固定在页面右上角，作为常驻唯一操作入口；
- 菜单分为“文件”和“信息”两组；
- 文件组包含：新建、打开、保存、另存为；
- 信息组包含：打开当前文稿信息弹窗。

### 2.2 信息弹窗

信息弹窗展示三组正式信息：

1. 当前卡片：标题、文件路径、`cardId`、`baseCardId`、创建与修改时间；
2. 富文本：正文格式、存储方式、正文资源路径、正文语言、正文主题、资源计数；
3. 运行时：保存状态、编辑器状态、自动保存状态、当前主题、上次保存时间。

### 2.3 轻提示

- 打开中、保存中、保存成功、保存失败等状态使用底部轻提示反馈；
- 成功与普通提示自动消失，错误提示保留直到后续状态覆盖；
- 状态提示刷新不得触发编辑器 iframe 重建。

## 3. 视觉方向

界面视觉目标调整为“极简正文优先 + 宿主题跟随 + 弱 chrome 干扰”。

实现要求：

1. 应用背景直接使用宿主题 surface token；
2. 中央编辑区不再额外绘制圆角卡片、描边和纸张阴影；
3. 菜单、弹窗和轻提示只在需要时出现；
4. 所有颜色、边框和阴影层级都通过 `--chips-*` token 与 `color-mix(...)` 推导，不私建独立视觉体系。

## 4. 窗口外观基线

窗口原生外观通过 [manifest.yaml](/Users/sevenstars/Documents/ChipsCard/Develop/Project-13-2-f/Chips-RichTextEditor/manifest.yaml) 的 `ui.window.chrome` 声明，当前正式配置为：

1. `frame: true`
2. `transparent: false`
3. `backgroundColor: "#ffffff"`
4. `titleBarStyle: default`

这表示窗口由 Host 按 Electron 原生标题栏创建，不再启用 `hidden` / `titleBarOverlay` 方案。

## 5. 主题接入

### 4.1 Provider

- 应用根层使用 `ChipsThemeProvider`；
- 主题事件源来自 `window.chips`；
- 监听事件名为 `theme.changed`。

### 4.2 主题快照

应用初始主题状态同时参考：

1. 文档上已注入的主题属性；
2. `client.theme.getCurrent()` 返回的正式主题信息。

### 4.3 壳层样式策略

- `src/global.css` 只消费宿主题 token，不写死亮色纸张风格；
- 菜单和信息弹窗使用 `surface`、`surface-container-low`、`on-surface`、`comp-dialog`、`comp-menu` 等正式 token；
- 富文本基础卡片编辑器输入区外层壳已移除圆角、描边和阴影，视觉边界由应用整体布局承担。

## 6. 多语言接入

### 5.1 文案来源

- 所有用户可见文案统一来自 `i18n/zh-CN.json` 与 `i18n/en-US.json`；
- 应用内通过本地 `formatMessage(...)` 工具读取。

### 5.2 语言同步

语言状态需同时参考：

1. `document.documentElement.lang`
2. `client.i18n.getCurrent()`
3. Bridge `language.changed` 事件

## 7. 交互稳定性要求

1. 普通输入、菜单开合、弹窗开合、状态提示刷新不得导致编辑器 iframe 重建；
2. 只有在新建文稿、打开其他文稿、明确重置会话时才允许重新挂载编辑器；
3. 资源桥和编辑器事件回调必须保持稳定身份，避免重复装配；
4. 主题切换后允许壳层与正文区同步刷新，但不得出现持续 loading 闪烁。
