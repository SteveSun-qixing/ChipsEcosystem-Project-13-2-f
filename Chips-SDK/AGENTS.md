# Chips-SDK AGENTS

## 角色

- 本仓是 SDK 与开发者 CLI 工具仓；职责是提供类型化封装、CLI、脚手架接入和测试辅助，不承载 Host 运行时主实现。

## 开发规则

- 新能力优先在 `src/api`、类型定义、契约清单和测试中成体系落地，不要在调用方散落私有封装。
- 不要把本应属于 Host 的运行时逻辑迁入 SDK；SDK 只做正式接口的封装、校验和调用辅助。
- CLI、模板和工作区行为变更时，必须同步更新测试、相关文档和必要的脚手架联动逻辑。
- 全生态正式 API、Bridge 口径和公共使用方式应以 `生态共用技术文档/` 为主发布位置；本仓内部文档只写实现细节和封装策略。
- 若 Host、SDK、共享文档三者口径不一致，先登记工单，再决定如何收口，不要让 SDK 单方面漂移。

## 验证

- 优先运行 `npm test`。
