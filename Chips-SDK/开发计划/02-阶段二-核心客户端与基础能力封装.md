# 阶段二：核心客户端与基础能力封装

> 阶段目标：实现 Core Client Layer 与文件/卡片/箱子/资源等基础能力封装，建立可用的 SDK 核心。

---

## 1. 阶段范围

- 实现 `src/core/` 中的：
  - 环境探测；
  - Bridge 适配器（至少插件环境）；
  - 客户端工厂与统一 `invoke`；
  - 错误归一与重试策略。
- 实现 `src/api-wrapper/` 中的基础能力域：
  - `file`, `card`, `box`, `resource`。
- 为以上模块补齐单元测试与基础契约测试。

---

## 2. 任务拆解

### 2.1 Core Client 实现

- 实现：
  - `src/core/environment.ts`
  - `src/core/bridge-adapter.ts`
  - `src/core/client.ts`
  - `src/core/errors.ts`
- 建立最小可用客户端：
  - 在插件环境中通过 `window.chips.invoke` 调用 Host；
  - 支持基础错误归一与可配置的重试策略。

### 2.2 基础能力域封装

- 实现 `file` 封装：
  - `read/write/list/stat` 映射到 Host `file.*` 动作；
  - 覆盖基本参数校验与错误路径。
- 实现 `card` 封装：
  - `parse/validate/render` 映射到 Host `card.*` 动作；
  - `render` 返回类型与渲染诊断信息对齐 Host 对外接口。
- 实现 `box` 与 `resource` 封装：
  - 按《文件与资源能力封装设计》执行。

---

## 3. 质量要求

- 所有新增模块配套单元测试：
  - 每模块 ≥ 8 个用例；
  - 覆盖正常、错误与边界路径。
- 引入基础契约测试：
  - 对比 `route-manifest` 中的动作清单；
  - 确保封装调用的动作均在清单中存在。

---

## 4. 阶段出口条件

- 在插件环境下，使用真实 Host 或模拟 Host：
  - 能够通过 `chips-sdk` 完成文件读写与卡片解析/验证/渲染；
  - 错误对象符合标准格式。
- 所有新增测试通过，性能基线脚本确认封装层额外开销在阈值内。
