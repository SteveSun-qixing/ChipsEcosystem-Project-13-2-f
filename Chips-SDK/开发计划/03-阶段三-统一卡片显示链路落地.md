# 阶段三：统一卡片显示链路落地

> 阶段目标：在 Core Client 与基础能力封装稳定的前提下，实现统一卡片显示链路，并与组件库完成联调。

---

## 1. 阶段前置条件

- 阶段二已完成并通过质量门禁：
  - Core Client 稳定可用；
  - `card.render` 封装已实现并通过测试。
- 与 `工单001-SDK-UNIFIED-CARD-DISPLAY-API` 对齐最新口径，确认无新的阻塞项。

---

## 2. 任务拆解

### 2.1 显示接口实现

- 在 `src/api-wrapper/card/display.ts` 中实现：
  - `card.coverFrame.render`；
  - `card.compositeWindow.render`；
  - 事件订阅工具。
- 确保：
  - `mode` 非法值在 SDK 层直接阻断并返回 `INVALID_ARGUMENT` 错误；
  - 返回的 iframe 与 origin 信息满足组件库安全要求。

### 2.2 事件协议实现

- 实现对以下事件的监听与分发：
  - `chips.composite:ready`
  - `chips.composite:node-error`
  - `chips.composite:fatal-error`
- 集成 origin 白名单校验逻辑，忽略非白名单消息。

### 2.3 与组件库联调

- 在组件库仓的 demo 或测试场景中，切换适配器为真实 SDK 调用：
  - 验证封面与复合窗口展示效果；
  - 验证错误与降级行为；
  - 验证 origin 与安全策略。

---

## 3. 质量与验收

- 单元测试：
  - 覆盖正常显示、非法参数、事件收发与 origin 校验。
- 集成测试：
  - 与组件库 demo 联调通过。
  - 与 Host 真实渲染链路至少完成一次完整 E2E 演练。
- 阶段出口：
  - 统一卡片显示链路对组件库与应用可用，工单001状态更新为“已闭环”或进入下一阶段跟踪状态。
