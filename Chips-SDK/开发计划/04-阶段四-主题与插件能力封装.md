# 阶段四：主题与插件能力封装

> 阶段目标：完成主题、多语言、配置与插件信息等能力封装，使 SDK 满足大部分插件与应用场景。

---

## 1. 阶段范围

- 实现 `theme/config/i18n` 能力封装。
- 实现 `plugin/module/window/platform` 查询与管理能力封装（含插件安装/启停/卸载）。
- 补齐类型定义与基础测试。

---

## 2. 任务拆解

### 2.1 主题与配置能力

- 实现 `client.theme.*` API，对齐生态共用文档与 Host 服务域设计。
- 实现 `client.config.*` 与 `client.i18n.*`：
  - 保持语义简单，不在 SDK 层增加额外合并或默认值逻辑。

### 2.2 插件与窗口能力

- 实现 `client.plugin.*` 信息查询与管理 API：
  - `getSelf/list/get/getCardPlugin/getLayoutPlugin` 面向插件与应用业务使用场景；
  - `install/enable/disable/uninstall/query` 映射 Host `plugin.*` 服务动作，用于 Host 侧插件管理。
- 实现基础 `client.window.*` 与 `client.module.*` 封装：
  - 明确高风险能力的权限与使用场景。

---

## 3. 验收标准

- 所有新能力均有单元测试覆盖。
- 通过契约测试确认动作名与 Host 对外接口一致。
- 与至少一个插件或应用场景进行联调验证（可通过 demo 或实验性代码完成）。
