# 阶段五：工具与契约集成

> 阶段目标：实现契约校验与调试工具，提升 SDK 在长期演进中的可靠性与可维护性。

---

## 1. 阶段范围

- 集成 `route-manifest` 与其他契约快照。
- 实现契约漂移检测与测试脚本。
- 实现基础调试工具与性能基线采样工具。

---

## 2. 任务拆解

### 2.1 契约集成

- 将 Host 导出的 `route-manifest.json` 引入 `src/contracts/`。
- 实现 `tooling/route-manifest.ts`：
  - `assertKnownAction` 等工具函数。
- 编写契约漂移测试：
  - 检查 SDK 封装调用的动作与清单一致。

### 2.2 调试与性能工具

- 实现 `tooling/diagnostics.ts` 与 `tooling/perf-benchmark.ts`：
  - 提供开发期调试信息；
  - 提供封装层性能基线采样。

---

## 3. 验收标准

- 契约漂移测试可在 CI 中稳定运行。
- 性能基线脚本可输出可读报告并集成到 CI。
