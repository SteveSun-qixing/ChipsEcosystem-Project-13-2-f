# 阶段六：测试体系与发布验收

> 阶段目标：完善测试体系，形成发布前质量门禁与验收流程，使 SDK 达到可对外发布标准。

---

## 1. 阶段范围

- 完善单元测试、契约测试与集成测试覆盖。
- 建立发布前检查清单与流程。
- 协同更新生态共用文档（如《SDK 使用指南》）。

---

## 2. 任务拆解

### 2.1 测试覆盖补齐

- 根据《测试策略与质量门禁设计》补齐各能力域测试：
  - 核心客户端；
  - 文件与资源；
  - 统一卡片显示链路；
  - 主题与插件能力。

### 2.2 发布流程定义

- 定义发布前检查清单：
  - 测试结果；
  - 契约漂移； 
  - 性能基线；
  - 文档同步。
- 按仓库规范定义标签、发布包内容与变更日志结构。

### 2.3 生态文档协同

- 将 SDK 实际实现口径同步到：
  - 《生态共用技术文档/插件开发/08-SDK使用指南.md》；
  - 其他需要引用 SDK 能力的生态共用文档。

---

## 3. 阶段出口条件

- 所有门禁均通过，无未处理的高危缺陷。
- SDK 可在至少一个真实集成场景中稳定运行（示例项目或内部应用）。
- 生态共用文档已更新并通过评审。
