# 测试策略与质量门禁设计

> 文档状态：设计基线稿  
> 目标：定义 `Chips-SDK` 的测试层级与质量门禁，确保交付质量

---

## 1. 测试层级

### 1.1 单元测试

- 覆盖对象：
  - Core Client 逻辑（环境探测、错误归一、重试）。
  - 各能力域封装的参数校验与结果映射。
  - 工具函数（契约校验、性能采样等）。

### 1.2 契约测试

- 覆盖对象：
  - SDK 封装的动作名与 Host `route-manifest` 的一致性。
  - 错误码映射是否保持原样（除包装外不更改语义）。

### 1.3 集成测试

- 覆盖对象：
  - 使用模拟 Host/Bridge 环境的 end-to-end 调用。
  - 在具备真实 Host 环境时，执行少量烟雾测试。

---

## 2. 测试用例规划

### 2.1 Core Client

- 环境探测：
  - `plugin` 环境（存在 `window.chips`）。
  - 浏览器环境（无 `window.chips`）。
  - Node 环境（无 `window`）。
- 错误归一：
  - 标准错误对象透传。
  - 原生 `Error` 与字符串包装。
- 重试策略：
  - 可重试错误的重试次数与退避间隔。

### 2.2 能力域封装

- `file/card/box/resource`：
  - 正常路径：基础操作成功。
  - 错误路径：文件不存在、格式错误等。
- 统一卡片显示链路：
  - 合法/非法 `mode`。
  - 事件协议触发与 origin 过滤。
- `theme/config/i18n/plugin`：
  - 各 API 功能行为。
  - 权限不足错误处理。

---

## 3. 质量门禁

### 3.1 本地门禁

- `npm test`：单元与集成测试全部通过。
- `npm run lint`：无 lint 错误。
- 性能基线脚本：封装层额外开销在阈值之内。

### 3.2 CI 门禁

- 在 CI 中执行：
  - 单元测试（覆盖率门槛）。
  - 契约漂移检测。
  - 性能基线采样（关键路径）。

### 3.3 发布前检查

- 所有测试通过。
- 契约漂移工单（如有）已闭环。
- 统一卡片显示链路 E2E 场景已通过。

---

## 4. 小结

本文件定义了 `Chips-SDK` 的测试层级、用例规划与质量门禁要求。实现时应优先建立测试基线，再逐步实现功能，以避免后补测试与质量滑坡。
