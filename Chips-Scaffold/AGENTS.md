# Chips-Scaffold AGENTS

## 角色

- 本目录包含官方脚手架工程；职责是生成符合生态正式规范的完整工程，而不是提供临时样板。

## 通用规则

- 生成产物必须默认可运行、可测试、可校验，并提供正式 README 与必要说明；默认不创建 `需求文档/`、`技术文档/`、`技术手册/`、`开发计划/` 目录；不要输出 TODO、空实现或占位工程。
- 脚手架只能消费已经冻结的生态契约与正式公共规范；若接口尚未收口，先登记工单，不要把不确定口径写进模板。
- 模板中的占位符必须在生成后完全回填，不得把 `{{ ... }}`、`app-standard` 之类模板身份泄露到正式工程。
- 跨生态的正式模板规范、目录约定、Manifest 约定和公共接线规则，应同步到 `生态共用技术文档/`。
- 修改脚手架时，不要依赖工作区私有 hacks、隐藏依赖或手工补基线。

## 验证

- 按对应子脚手架的 `build/test/test:templates/test:e2e` 与工作区校验脚本做验证。
