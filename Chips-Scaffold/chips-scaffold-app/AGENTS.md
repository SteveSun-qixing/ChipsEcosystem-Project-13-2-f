# chips-scaffold-app AGENTS

## 角色

- 生成应用插件正式模板；必须覆盖 React、组件库、主题、多语言、Bridge / SDK 接线和项目文档骨架。

## 开发规则

- 生成的应用工程必须是正式可开发基线，不允许遗留模板占位符、示例身份和未收口的脚手架文本。
- 应用模板的公共规范以应用插件开发指南、Manifest 规范、Bridge API 规范和共用技术文档为准。
- 若某项模板能力需要新的生态公共规范，先更新共享文档或登记工单，再修改模板。
- 模板文件、参数渲染器、CLI 集成和生成工程验证必须同步演进。

## 验证

- 优先运行 `build`、`test`、`test:templates`、`test:e2e`。
