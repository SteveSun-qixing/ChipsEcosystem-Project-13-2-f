# 阶段三：模板扩展与生态对接

> 阶段目标：在标准模板基础上扩展基础卡片插件模板族，并完成与 `chips dev` 的集成，使基础卡片插件脚手架能够在完整生态工作流中使用。

---

## 1. 阶段范围

- 在 `chips-scaffold-basecard` 仓库中：
  - 设计并实现必要的模板变体（如媒体类、文本类基础卡片模板等）；
  - 完成 `listBasecardTemplates` 实现与模板选择逻辑；
  - 与 `chips dev create --type card` 命令完成集成；
  - 根据需要补充文档与示例。

---

## 2. 关键任务

### 2.1 模板变体设计与实现

- 根据生态设计原稿与卡片业务规范，选择 1–2 个具有代表性的基础卡片类型，设计对应模板变体，例如：
  - 文本型基础卡片模板（以富文本/Markdown 为主内容）；
  - 媒体型基础卡片模板（以图片/视频为主内容）。
- 为每个模板变体：
  - 创建独立模板目录与元数据文件；
  - 根据类型特点调整配置 Schema、渲染视图与编辑表单示例；
  - 在需求与技术文档中记录模板用途与差异点。

### 2.2 模板列表与选择机制

- 在 `src/core/` 中实现：
  - 扫描 `templates/` 目录并读取所有模板元数据；
  - 暴露 `listBasecardTemplates` 接口，返回模板概要信息列表；
- 在 `createBasecardProject` 中支持 `templateId` 参数：
  - 校验模板是否存在且可用；
  - 未指定模板时选择默认模板（如 `card-standard`）。

### 2.3 与 chips dev 的集成

- 与 `chips dev` 仓库协作：
  - 增加 `--type card` 支持与基础卡片模板选择逻辑；
  - 在交互式流程中调用 `listBasecardTemplates` 展示模板列表；
  - 将用户选择与参数采集结果传递给 `createBasecardProject`。
- 在文档中补充：
  - `chips dev create --type card` 的使用示例；
  - 不同模板适用场景说明。

### 2.4 示例与文档补充

- 为每个模板变体补充：
  - README 中的具体说明与截图占位；
  - 简短的“从创建到第一次渲染/编辑”的步骤说明。
- 在本仓技术文档中更新：
  - 模板清单与元数据示例；
  - 与 CLI 集成的流程图或文字描述。

---

## 3. 验收标准

- 模板变体：
  - 至少存在 1 个除 `card-standard` 之外的模板变体；
  - 每个模板变体都通过模板一致性测试与生成工程 E2E 测试（可按 CI 策略裁剪）。
- CLI 集成：
  - 在本地环境中，`chips dev create my-basecard --type card` 能够：
    - 展示模板列表；
    - 创建工程并通过基础构建与测试；
  - 使用不同模板 ID 创建工程行为正确。
- 文档：
  - 本仓技术文档中的模板与工作流描述与实现一致；
  - 如涉及生态共用文档（例如新增“官方基础卡片模板清单”），已在对应目录中更新。

---

## 4. 输出物

- 一组可用的基础卡片模板变体与对应元数据；
- 完整的 `listBasecardTemplates` / `createBasecardProject` 接口与 `chips dev` 集成逻辑；
- 更新后的技术文档与使用示例。

本阶段完成后，可进入阶段四，聚焦测试体系与质量门禁建设。
