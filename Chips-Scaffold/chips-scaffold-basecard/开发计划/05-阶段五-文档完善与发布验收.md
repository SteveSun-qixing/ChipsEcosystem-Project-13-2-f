# 阶段五：文档完善与发布验收

> 阶段目标：完善基础卡片插件脚手架相关文档与对外说明，完成首次可用版本的发布验收。

---

## 1. 阶段范围

- 对齐并完善：
  - 本仓需求文档、技术文档与开发计划；
  - 与基础卡片插件脚手架相关的生态共用文档；
  - 生成工程 README 与示例说明。
- 完成首次发布前的验收与版本标记。

---

## 2. 关键任务

### 2.1 文档梳理与更新

- 检查本仓所有文档：
  - 删除或更正过时描述；
  - 确保目录结构、模板清单、参数模型、工作流与实际实现一致；
- 如有需要沉淀为生态正式口径的信息（例如“基础卡片插件工程结构推荐”），在 `生态共用技术文档` 中新增或更新对应章节。

### 2.2 使用说明与示例补充

- 为生成工程提供：
  - 详细的 `README.md`（由模板渲染）；
  - 至少一个“从创建到第一次渲染/编辑”的完整操作示例；
- 在本仓中添加面向生态内部的简要使用说明（例如如何在 `chips dev` 中启用基础卡片模板）。

### 2.3 发布前验收

- 执行发布前检查清单（见《技术文档/05-测试策略与质量门禁设计》）：
  - 本仓质量门禁通过；
  - 模板生成工程在目标平台上通过完整工作流测试；
  - 文档与实现已对齐；
  - 与脚手架相关的开放问题已在工单中记录，且不会阻塞当前版本。
- 规划版本号并在仓库中打标签。

---

## 3. 验收标准

- 文档：
  - 本仓文档无明显缺失或明显陈旧内容；
  - 生态共用文档中已包含基础卡片插件脚手架相关的正式口径（如有需要）。
- 工程：
  - 所有质量门禁脚本通过；
  - 从“创建基础卡片插件工程”到“在 Host 中完成一次示例渲染/编辑”的链路在受控环境中跑通。

---

## 4. 输出物

- 一套对内对外一致、可持续维护的文档体系；
- 标记为首个可用版本的 `chips-scaffold-basecard` 仓库版本。

本阶段完成后，基础卡片插件脚手架进入常规演进阶段，后续改动需遵守既有文档与质量门禁约束，并通过问题工单管理重大变更。
