# 布局插件脚手架总体架构设计

## 1. 仓库角色

`chips-scaffold-boxlayout` 是布局插件正式模板仓。  
它只参与开发期工程初始化，不参与任何运行时装载。

它的职责是：

1. 为 `type: layout` 插件提供标准工程模板；
2. 为 `chipsdev create` 提供布局插件初始化能力；
3. 为下游布局插件仓提供稳定、统一的工程主干。

## 2. 与其他仓库的关系

| 仓库 | 关系 |
|---|---|
| `Chips-Host` | 模板必须遵守 Host 已冻结的箱子运行时契约 |
| `Chips-SDK` | 模板可内建正式 SDK 类型与调用示例 |
| `Chips-ComponentLibrary` | 模板默认接入组件库和主题系统 |
| `Chips-BoxLayoutPlugin` | 首个下游正式消费方 |
| `chipsdev` | 通过 `create` 命令消费本仓模板 |

## 3. 内部分层

仓库内部建议分为三层：

1. **模板资产层**：`templates/`
2. **脚手架核心层**：`src/core/`
3. **CLI 集成层**：`src/cli/`

依赖方向固定为：

- `templates` <- `src/core` <- `src/cli`

不允许模板反向依赖脚手架实现。

## 4. 架构原则

- 模板优先，逻辑最小化；
- 契约优先，不私建公共规则；
- 与其他官方脚手架仓保持同构；
- 所有生成工程都必须是正式可开发基线。
