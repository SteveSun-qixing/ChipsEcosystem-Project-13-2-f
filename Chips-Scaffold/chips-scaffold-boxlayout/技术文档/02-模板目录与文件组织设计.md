# 模板目录与文件组织设计

## 1. 脚手架仓目录

建议脚手架仓至少包含：

```text
chips-scaffold-boxlayout/
├─ package.json
├─ tsconfig.json
├─ src/
│  ├─ cli/
│  └─ core/
├─ templates/
│  └─ boxlayout-standard/
├─ tests/
├─ scripts/
├─ 需求文档/
├─ 技术文档/
└─ 开发计划/
```

## 2. 生成工程目录

标准布局插件模板生成工程至少包含：

```text
<project>/
├─ manifest.yaml
├─ package.json
├─ tsconfig.json
├─ src/
│  ├─ index.ts
│  ├─ view/
│  ├─ editor/
│  ├─ schema/
│  └─ shared/
├─ tests/
├─ assets/
├─ i18n/
├─ contracts/
└─ README.md
```

## 3. 组织原则

- 模板必须一次性输出正式主干；
- `dist/` 不属于初始化模板默认目录；
- `template.json` 只作为脚手架模板元数据存在于脚手架仓内部，不能输出到生成工程；
- 不默认创建 `需求文档/`、`技术文档/`、`开发计划/` 到生成工程；
- README 必须写清 build/test/validate 入口。
