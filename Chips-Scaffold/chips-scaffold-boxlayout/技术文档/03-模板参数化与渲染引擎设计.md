# 模板参数化与渲染引擎设计

## 1. 参数模型

布局插件脚手架至少需要以下参数：

- `projectName`
- `pluginId`
- `displayName`
- `layoutType`
- `description`

## 2. 占位符原则

- 占位符命名统一、可诊断；
- 生成后不得残留占位符文本；
- 不允许把模板身份泄露到正式工程。

## 3. 渲染流程

建议流程：

1. 读取模板元数据；
2. 校验参数；
3. 渲染文件名与文件内容；
4. 写入目标目录；
5. 返回生成摘要；
6. 对生成工程执行基础检查。

## 4. 参数校验要求

- `pluginId` 必须符合插件 ID 规范；
- `layoutType` 必须符合正式布局标识规则；
- `displayName` 不得为空；
- 目标目录冲突必须可诊断。
