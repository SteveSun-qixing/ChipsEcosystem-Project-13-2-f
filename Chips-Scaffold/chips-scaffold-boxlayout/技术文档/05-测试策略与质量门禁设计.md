# 测试策略与质量门禁设计

## 1. 测试分层

布局插件脚手架至少需要：

1. 脚手架核心单元测试；
2. 模板一致性测试；
3. 生成工程端到端测试。

## 2. 门禁脚本

脚手架仓建议固定脚本：

- `build`
- `test`
- `test:templates`
- `test:e2e`

## 3. 模板校验重点

- 模板目录完整；
- 关键文件存在；
- 占位符已正确替换；
- Manifest、布局导出、测试脚本和 README 保持一致。

## 4. E2E 验收重点

- 生成工程可安装依赖；
- 生成工程可构建；
- 生成工程可测试；
- 生成工程可作为 `Chips-BoxLayoutPlugin` 起始工程。
