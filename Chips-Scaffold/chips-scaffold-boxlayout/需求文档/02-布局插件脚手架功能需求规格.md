# 布局插件脚手架功能需求规格

## 1. 功能目标

`chips-scaffold-boxlayout` 必须为布局插件项目生成完整、正式、可继续开发的工程基线，而不是演示模板。

## 2. 模板工程需求

### FR-BOXLAYOUT-TEMPLATE-001 标准模板骨架

脚手架必须至少提供一个标准布局插件模板，生成工程至少包含：

- `manifest.yaml`
- `package.json`
- `src/index.ts`
- `src/view/`
- `src/editor/`
- `src/schema/`
- `src/shared/`
- `tests/`
- `assets/`
- `i18n/`
- `contracts/`
- `README.md`

### FR-BOXLAYOUT-TEMPLATE-002 正式导出契约基线

模板必须默认生成 `layoutDefinition` 导出骨架，并覆盖：

- `pluginId`
- `layoutType`
- `displayName`
- `createDefaultConfig`
- `normalizeConfig`
- `validateConfig`
- `getInitialQuery`
- `renderView`
- `renderEditor`

### FR-BOXLAYOUT-TEMPLATE-003 Manifest 基线

模板生成的 `manifest.yaml` 必须默认满足：

- `type: layout`
- `entry` 指向正式构建产物
- `layout.layoutType`
- `layout.displayName`
- `ui.layout`
- 默认权限、主题、多语言与资源入口说明

### FR-BOXLAYOUT-TEMPLATE-004 查看态模板

模板必须默认提供：

- 摘要态页面渲染入口；
- Host runtime 消费示例；
- 空态、错误态、加载态；
- 主题与多语言接线；
- cleanup 闭环示例。

### FR-BOXLAYOUT-TEMPLATE-005 编辑态模板

模板必须默认提供：

- 配置编辑入口；
- `onChange` 输出配置快照；
- 资源导入/删除回调接线示例；
- 布局预览挂载示例。

## 3. 参数化与 CLI 接入需求

### FR-BOXLAYOUT-CLI-001 创建入口

`chipsdev create` 必须能选择或指定布局插件模板。

### FR-BOXLAYOUT-CLI-002 核心参数

模板创建至少支持以下参数：

- `projectName`
- `pluginId`
- `displayName`
- `layoutType`
- `description`

### FR-BOXLAYOUT-CLI-003 占位符完全回填

生成工程中不得残留模板占位符、模板名泄露、示例身份或未回填路径。

## 4. 测试与验证需求

### FR-BOXLAYOUT-TEST-001 脚手架自测

脚手架仓自身必须具备：

- `build`
- `test`
- `test:templates`
- `test:e2e`

### FR-BOXLAYOUT-TEST-002 生成工程门禁

模板生成的工程必须默认支持：

- `build`
- `test`
- `lint`
- `validate`

### FR-BOXLAYOUT-TEST-003 生成工程可作为首个布局插件仓起点

生成结果必须足以直接用作 `Chips-BoxLayoutPlugin` 的工程基线，不需要再手工补主干结构。

## 5. 协作约束需求

### FR-BOXLAYOUT-COLLAB-001 与共享文档一致

模板中的公共字段和正式接口必须与共享文档一致，不得私自新增第二套约定。

### FR-BOXLAYOUT-COLLAB-002 与下游箱子大任务对齐

脚手架模板必须服务于箱子全链路任务，保证阶段五网格布局插件工程能够直接复用。
