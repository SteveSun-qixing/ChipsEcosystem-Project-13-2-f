# 阶段一：架构与需求冻结

## 1. 阶段目标

- 阅读生态设计原稿、生态共用技术文档、Host、SDK、CLI 与现有模板实现。
- 冻结模块脚手架应遵守的新正式契约。
- 明确模板必须从旧页面挂载模型切换到无界面 capability 模型。

## 2. 产出

- 确定正式口径为 `module.listProviders/resolve/invoke/job.*`。
- 确定模块调用模型为 `capability + method`。
- 确定模板正式 Manifest 来源为 `module.provides/module.consumes`。
- 确定旧 UI 模板必须归档，不得继续复制。

## 3. 阶段出口条件

- 需求、技术、计划文档统一到新口径。
- 可以开始正式实施模板与 CLI 重构。
