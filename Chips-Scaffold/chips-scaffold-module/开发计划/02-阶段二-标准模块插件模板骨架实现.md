# 阶段二：标准模块插件模板骨架实现

## 1. 阶段目标

- 建立脚手架核心能力。
- 落地 `module-standard` 模板。
- 让模板默认生成真实可运行的无界面 capability 模块示例。

## 2. 主要工作项

- 实现模板引擎、错误模型、文件系统工具与 CLI API。
- 重写 `manifest.yaml.tpl` 为 `manifest.module` 结构。
- 增加 `contracts/*.schema.json.tpl`。
- 重写 `src/index.ts.tpl` 为默认导出模块定义对象。
- 增加同步方法和异步 job 方法测试。
- 更新 README 和模板元数据。
- 把旧 UI 模板移入 `归档/`，并让渲染器跳过归档目录。

## 3. 阶段完成标准

- `npm run build` 与模板单测通过。
- 模板生成工程结构与新契约一致。
- 生成工程不再包含旧 UI 模块模板残留。
