# 阶段三：模板扩展与生态对接

## 1. 阶段目标

- 将模块脚手架接入 `chipsdev create module`；
- 对齐 Host/SDK/Bridge/CLI 文档中的模块正式口径；
- 验证生成工程在生态工作区内的真实创建链路。

## 2. 主要工作项

- 在 `Chips-SDK/cli/index.js` 中增加 `module` 创建分支；
- 更新 `chipsdev` 帮助、错误提示与测试脚本；
- 更新模块相关生态文档：
  - Bridge API
  - Host 服务域设计
  - 模块插件开发指南
  - Manifest 规范
  - CLI 手册
  - SDK 需求/技术文档
- 增加 `run-cli-create-module-workspace-tests.cjs`。

## 3. 阶段完成标准

- `chipsdev create module` 可在临时工作区内成功生成工程；
- 生成工程可完成安装、构建、测试、校验与打包；
- 文档与实现口径一致。
