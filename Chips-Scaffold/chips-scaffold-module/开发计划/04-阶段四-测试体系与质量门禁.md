# 阶段四：测试体系与质量门禁

## 1. 阶段目标

- 建立脚手架自身的测试与门禁；
- 建立 CLI 侧工作区集成测试；
- 让模板、CLI、工作区、打包链路形成长期可回归验证的体系。

## 2. 主要工作项

- `template-engine` 单测；
- `check-templates` 模板完整性检查；
- `run-generated-e2e` 最小生成自检；
- `run-cli-create-module-workspace-tests` 工作区集成测试；
- 必要时补充回归测试，覆盖生成工程中的真实问题。

## 3. 阶段完成标准

- 本仓测试全部通过；
- CLI 侧新增模块创建测试通过；
- 质量门禁足以阻断模板缺文件、CLI 失联或生成工程不可运行等问题。
