# 模板目录与文件组织设计

## 1. 当前模板目录

```text
templates/module-standard/
├─ template.json
├─ .eslintrc.cjs.tpl
├─ .gitignore.tpl
├─ README.md.tpl
├─ chips.config.mjs.tpl
├─ manifest.yaml.tpl
├─ package.json.tpl
├─ tsconfig.json.tpl
├─ vitest.config.mts.tpl
├─ contracts/
│  ├─ run.input.schema.json.tpl
│  ├─ run.output.schema.json.tpl
│  ├─ runAsync.input.schema.json.tpl
│  └─ runAsync.output.schema.json.tpl
├─ src/
│  └─ index.ts.tpl
├─ tests/
│  └─ unit/
│     └─ module-definition.test.ts.tpl
```

初始化生成工程仅复制上述模板资产；`需求文档/`、`技术文档/`（或“技术手册”）与 `开发计划/` 目录不属于模块模板的默认输出范围。

## 2. 生成工程结构

```text
module-standard-project/
├─ .eslintrc.cjs
├─ .gitignore
├─ README.md
├─ chips.config.mjs
├─ manifest.yaml
├─ package.json
├─ tsconfig.json
├─ vitest.config.mts
├─ contracts/
│  ├─ run.input.schema.json
│  ├─ run.output.schema.json
│  ├─ runAsync.input.schema.json
│  └─ runAsync.output.schema.json
├─ src/
│  └─ index.ts
└─ tests/
   └─ unit/
      └─ module-definition.test.ts
```

## 3. 组织原则

- 方法实现和 schema 契约必须一一对应。
- 所有公共入口统一收敛到 `src/index.ts`。
- 模板仅包含 `README`、源代码、测试、清单与 contract 等目录，不会自动创建 `需求文档/`、`技术文档/` / `开发计划/` 这类文档根目录；需要时由业务仓自行补充。
