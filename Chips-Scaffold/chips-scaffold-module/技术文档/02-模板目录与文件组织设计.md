# 模板目录与文件组织设计

## 1. 当前模板目录

```text
templates/module-standard/
├─ template.json
├─ .eslintrc.cjs.tpl
├─ .gitignore.tpl
├─ README.md.tpl
├─ chips.config.mjs.tpl
├─ manifest.yaml.tpl
├─ package.json.tpl
├─ tsconfig.json.tpl
├─ vitest.config.mts.tpl
├─ contracts/
│  ├─ run.input.schema.json.tpl
│  ├─ run.output.schema.json.tpl
│  ├─ runAsync.input.schema.json.tpl
│  └─ runAsync.output.schema.json.tpl
├─ src/
│  └─ index.ts.tpl
├─ tests/
│  └─ unit/
│     └─ module-definition.test.ts.tpl
└─ 归档/
```

## 2. 生成工程结构

生成后的工程遵循以下组织：

- 根目录放清单、构建配置、README；
- `contracts/` 放正式 schema 契约；
- `src/index.ts` 放默认导出模块定义；
- `tests/unit/` 放最小可运行单测。

## 3. 组织原则

- 方法实现和 schema 契约必须一一对应。
- 所有公共入口统一收敛到 `src/index.ts`。
- 历史模板只保存在 `归档/`，不进入新工程。
- 模板 README 必须与实际生成结构保持同步。
