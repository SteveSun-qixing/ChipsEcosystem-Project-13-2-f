# 模板参数化与渲染引擎设计

## 1. 参数模型

当前模板渲染参数模型如下：

```ts
interface CreateModuleProjectOptions {
  projectName: string;
  targetDir: string;
  templateId: string;
  pluginId: string;
  moduleCapability: string;
  displayName: string;
  version: string;
  authorName?: string;
  authorEmail?: string;
}
```

## 2. 占位符规则

模板使用 `{{ VARIABLE_NAME }}` 形式进行文本替换。当前正式支持：

- `PROJECT_NAME`
- `TARGET_DIR`
- `TEMPLATE_ID`
- `PLUGIN_ID`
- `MODULE_CAPABILITY`
- `DISPLAY_NAME`
- `VERSION`
- `AUTHOR_NAME`
- `AUTHOR_EMAIL`

## 3. 渲染流程

1. 校验必填参数；
2. 校验目标目录是否为空；
3. 定位模板目录；
4. 加载 `template.json`；
5. 遍历模板文件树；
6. 文本文件执行占位符替换；
7. 非文本文件直接复制；
8. 返回生成结果摘要。

## 4. 保护策略

- 目标目录非空直接报错；
- 模板目录不存在或为空直接报错；
- `pluginId` 与 `moduleCapability` 缺失直接报错；
- 不做静默回退，不自动切换模板。

## 5. 文本与非文本识别

模板引擎当前把以下类型视为文本：

- `ts/tsx/js/jsx`
- `json`
- `yaml/yml`
- `md`
- `mjs/cjs`
- `css/html/txt/svg`
- `.tpl`

其余文件按二进制复制处理。

## 6. 设计取舍

- 采用轻量字符串替换而不是复杂模板 DSL，降低维护成本；
- 不在脚手架层做业务语义推断，只处理确定性参数渲染；
- 模块能力的生成由 CLI 负责推导，模板引擎只负责消费最终值。
