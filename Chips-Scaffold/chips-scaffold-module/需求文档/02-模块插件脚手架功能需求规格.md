# 模块插件脚手架功能需求规格

> 文档状态：已按模块插件重构实现更新

## 1. 模板管理

- 脚手架必须支持枚举可用模板。
- 每个模板必须声明 `id/name/description/version`。
- 模板可声明是否集成 SDK，以及是否仍依赖 React、主题系统、多语言系统。
- 模板缺少 `template.json` 或元数据非法时，必须阻断创建。

## 2. 工程创建

- 创建参数至少包含 `projectName/targetDir/templateId/pluginId/moduleCapability/displayName/version`。
- 目标目录已存在且非空时必须拒绝覆盖。
- 文本模板文件按占位符渲染，非文本文件按原样复制。
- 创建结果必须返回工程目录、模板 ID 和生成文件数。

## 3. 标准模板 `module-standard`

- 必须生成：
  - `.eslintrc.cjs`
  - `.gitignore`
  - `README.md`
  - `chips.config.mjs`
  - `manifest.yaml`
  - `package.json`
  - `tsconfig.json`
  - `vitest.config.mts`
  - `contracts/run.input.schema.json`
  - `contracts/run.output.schema.json`
  - `contracts/runAsync.input.schema.json`
  - `contracts/runAsync.output.schema.json`
  - `src/index.ts`
  - `tests/unit/module-definition.test.ts`
- 模板生成的 `manifest.yaml` 必须默认符合 `type: module`、`entry: dist/index.mjs`、`module.apiVersion/runtime/activation/provides/consumes`。
- 模板必须默认提供一个 capability。
- 模板必须默认提供一个同步方法示例和一个异步 job 方法示例。
- 模板必须让方法实现和 schema 文件一一对应。
- 模板不得再生成 React 运行时、页面挂载逻辑、主题/i18n UI 示例。

## 4. CLI 集成

- `chipsdev create module <targetDir>` 必须可用。
- CLI 必须自动推导 `projectName/pluginId/moduleCapability`。
- CLI 必须默认选择 `module-standard`。
- 当目标位于生态工作区中时，CLI 必须自动注册 workspace 并写入 `volta.extends`。
- 模板或脚手架无效时，CLI 必须输出可诊断错误。

## 5. 生成工程验证

- 默认生成工程必须通过：
  - `npm run lint`
  - `npm run build`
  - `npm test`
  - `npm run validate`
  - `chipsdev package`
- 生成出的 `.cpk` 必须能够被 Host 安装、启用并通过 `module.invoke` 正常调用。

## 6. 文档要求

- 模板 README 必须说明工程定位、目录结构、capability/method/schema 用法和正式约束。
- 若模板目录结构、CLI 行为或验证链路发生变化，必须同步更新本仓文档。
