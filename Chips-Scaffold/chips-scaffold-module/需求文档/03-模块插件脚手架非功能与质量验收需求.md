# 模块插件脚手架非功能与质量验收需求

> 文档状态：已按模块插件重构实现更新

## 1. 稳定性

- 相同输入必须生成相同文件内容和目录结构。
- `chipsdev create module` 不能依赖隐式手工步骤。
- 生成工程必须在无界面运行时场景下可直接被 Host 加载。

## 2. 一致性

- 文档、模板、CLI、Host、SDK 必须使用同一套模块正式口径。
- 不能同时保留 `module.mount/unmount/query/list` 与 `module.listProviders/resolve/invoke/job.*` 双口径。
- 模板 README、Manifest、schema、测试和打包结果必须一致。

## 3. 可维护性

- 模板变量命名必须稳定清晰。
- 模板引擎应尽量简单，复杂逻辑放在生成工程内部。
- `归档/` 中的历史模板不得再被复制到新工程。

## 4. 代码质量

- 不允许 TODO、FIXME、占位实现、空测试。
- 不允许引入未经批准的新第三方库。
- 生成工程默认代码必须能通过 TypeScript、Vitest、ESLint。

## 5. 验收标准

满足以下条件视为通过：

1. `chips-scaffold-module` 自身 `build/test/test:templates/test:e2e` 全部通过。
2. `chipsdev create module` 能在工作区中创建真实模块工程。
3. 生成工程能通过 `lint/build/test/validate/package`。
4. 生成模块包能被 Host 安装、启用并通过 `module.invoke` 调用。
5. 本仓文档已同步到新口径。
