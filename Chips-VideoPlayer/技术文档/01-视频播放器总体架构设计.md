# 视频播放器总体架构设计

## 1. 项目角色

`Chips-VideoPlayer` 是薯片生态中的应用插件视频查看器。它不承载视频解码内核或转码服务，只负责：

1. 恢复要播放的视频目标；
2. 为当前视频目标构建正式播放会话；
3. 承载控制条、播放区域和状态反馈；
4. 调用 Host 正式文件与平台能力。

## 2. 分层结构

当前实现分为四层：

### 2.1 App Shell

- 文件：`src/App.tsx`
- 职责：
  - 初始化 `chips-sdk` 客户端；
  - 读取主题、语言与宿主信息；
  - 恢复启动上下文；
  - 归一视频源；
  - 协调打开、保存和拖放行为。

### 2.2 Runtime Hooks

- 文件：
  - `src/hooks/useChipsClient.ts`
  - `src/hooks/useChipsBridge.ts`
  - `src/hooks/useVideoPlayerController.ts`
- 职责：
  - 封装 SDK 与 Bridge 接入；
  - 管理 `<video>` 元素运行时状态；
  - 收口播放、暂停、时间轴、音量、全屏、画中画和热键行为。

### 2.3 View Stage

- 文件：`src/components/VideoPlayerStage.tsx`
- 职责：
  - 只负责展示与事件绑定；
  - 不直接知道 Host 路由细节；
  - 使用 Controller 暴露的状态和动作组装界面。

### 2.4 Pure Utils

- 文件：
  - `src/utils/video-player.ts`
  - `src/utils/launch-resource.ts`
- 职责：
  - 保持纯函数；
  - 处理文件名、格式支持、时间格式化、启动参数归一与媒体错误映射。

## 3. 依赖边界

项目只依赖：

1. React
2. `@chips/component-library` 的主题运行时壳层
3. `chips-sdk`
4. 原生 HTML5 `<video>`

项目不依赖：

1. Electron 私有对象
2. Host 内部源码模块
3. 第三方播放器运行时 npm 包

## 4. DPlayer 参考策略

本项目参考 DPlayer 的部分思路包括：

1. 控制条分组；
2. 热键语义；
3. 播放控制器与界面模板解耦。

正式实现不直接导入 DPlayer 运行时，而是把这些经验重写为：

1. `useVideoPlayerController(...)`
2. `VideoPlayerStage`
3. 本地 CSS 主题化控制条

这样可以避免把第三方运行时外溢为生态正式接口。
