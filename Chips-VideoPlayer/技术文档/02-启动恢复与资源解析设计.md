# 启动恢复与资源解析设计

## 1. 正式输入来源

视频播放器当前只接受四类正式输入：

1. `launchParams.resourceOpen.filePath`
2. `launchParams.targetPath`
3. `launchParams.resourceOpen.resourceId`
4. 文件选择器 / 拖放得到的本地文件路径

## 2. 归一策略

实现位置：

- `src/utils/launch-resource.ts`
- `src/App.tsx`

归一顺序：

1. `resourceOpen.filePath`
2. `targetPath`
3. `resourceOpen.resourceId`

输出结构：

```ts
interface LaunchVideoTarget {
  sourceId: string;
  filePath?: string;
  fileName?: string;
  mimeType?: string;
  title?: string;
}
```

## 3. 播放 URI 解析

实现位置：

- `src/App.tsx`

解析策略：

1. 若目标本身已是 `file://`、`http://`、`https://`、`blob:` 或 `data:`，直接作为可播放 URI；
2. 否则调用正式 `client.resource.resolve(...)`；
3. 解析完成后才进入视频播放会话。

这样做的原因：

1. 本地文件路径必须通过正式资源解析转为可消费 URI；
2. 远端 URL 不应被错误当作本地路径再二次包装；
3. 上层应用无需关心 Host 路由内部资源格式，只需要交给视频播放器恢复会话。

## 4. 支持格式判断

实现位置：

- `src/utils/video-player.ts`

判断规则：

1. `mimeType` 以 `video/` 开头时优先判为支持；
2. 否则按本地路径或资源 ID 后缀匹配扩展名白名单；
3. 未命中白名单则拒绝打开并给出反馈。

## 5. 保存链路

实现位置：

- `src/App.tsx`

保存策略：

1. 本地文件路径存在时，使用 `platform.saveFile + file.copy`；
2. 远端或无稳定本地路径时，回退为下载链接；
3. 覆盖同路径时显式拒绝。
