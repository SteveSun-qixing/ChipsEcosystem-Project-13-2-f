# 播放会话与交互控制设计

## 1. 控制器职责

实现位置：

- `src/hooks/useVideoPlayerController.ts`

控制器统一管理以下状态：

1. `isReady`
2. `isPlaying`
3. `isBuffering`
4. `isMuted`
5. `volume`
6. `playbackRate`
7. `duration`
8. `currentTime`
9. `bufferedUntil`
10. `isFullscreen`
11. `isPictureInPicture`
12. `videoSize`
13. `errorKey`

## 2. 状态来源

状态只来自两类输入：

1. 原生 `<video>` 事件：
   - `loadedmetadata`
   - `canplay`
   - `playing`
   - `pause`
   - `waiting`
   - `ended`
   - `timeupdate`
   - `progress`
   - `volumechange`
   - `error`
2. 文档级事件：
   - `fullscreenchange`
   - `enterpictureinpicture`
   - `leavepictureinpicture`

## 3. 控制动作

控制器对外暴露：

1. `togglePlayback`
2. `seekBy`
3. `seekTo`
4. `setVolumeLevel`
5. `toggleMute`
6. `setPlaybackRate`
7. `toggleFullscreen`
8. `togglePictureInPicture`
9. `handleKeyDown`

## 4. 热键模型

当前热键直接参考 DPlayer 中的基础语义，但实现完全重写为 React Hook：

1. 空格：播放 / 暂停
2. 左右方向键：前后 5 秒
3. 上下方向键：音量加减
4. `F`：全屏
5. `M`：静音

并补充了两条正式限制：

1. 当焦点在 `input/textarea/select` 上时不抢快捷键；
2. 快捷键逻辑只作用于当前播放器表面容器。

## 5. 会话切换

当 `sessionKey` 变化时，控制器会：

1. 清空就绪、播放、缓冲、进度与错误状态；
2. 重置倍速；
3. 退出当前画中画状态标记；
4. 等待新的视频元素重新建立状态。

这样可以保证切换视频时不会把上一段媒体的瞬时状态带进下一段会话。
