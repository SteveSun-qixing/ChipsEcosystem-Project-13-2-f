# 界面结构与主题多语言接入设计

## 1. Stage 结构

实现位置：

- `src/components/VideoPlayerStage.tsx`
- `src/global.css`

页面由三层组成：

1. `video-player-viewport`
   - 全窗口视频画布；
   - 有文件时只承载 `<video>` 本体与状态遮罩；
   - 无文件时只显示居中的打开提示。
2. `video-player-dock`
   - 底部悬浮小型控制条；
   - 上层是细时间轴，下层是纯图标工具条；
   - 所有按钮通过 tooltip 显示名称，不在工具条内展示文字标签；
   - 在连续播放且用户无交互时自动淡出，交互恢复后再次显示。
3. `video-player-side-panel`
   - 点击“更多”后从右侧浮出；
   - 固定包含“视频信息 / 字幕 / 音频”三个标签页；
   - 当前只消费 HTML5 `<video>` 已暴露的正式轨道信息，不自行伪造媒体能力；
   - 在常规桌面窗口下使用固定宽高，内容溢出后在面板内部滚动。

## 2. 视觉策略

当前视觉方向：

1. 以黑色播放背景为唯一主视觉，保证视频内容占满窗口时不被界面噪声干扰；
2. 视频使用 `object-fit: contain`，确保任何比例的视频都不变形；
3. 控件、更多面板、提示反馈都采用低对比半透明深色浮层；
4. 整体不做装饰性风格化处理，只保留播放器必需层级。

## 3. 多语言接入

实现位置：

- `i18n/zh-CN.json`
- `i18n/en-US.json`
- `src/i18n/messages.ts`

规则：

1. 所有本项目界面文本都通过 `formatMessage(...)` 获取；
2. `language.changed` 事件触发后直接切换当前 locale；
3. 工具栏 tooltip、更多面板标签页、轨道空态和媒体错误都必须走多语言键；
4. 运行时媒体错误通过 `errorKey -> i18n key` 映射展示，而不是直接输出硬编码英语。

## 4. 主题接入

实现位置：

- `src/App.tsx`
- `src/global.css`

规则：

1. 首屏优先读取文档注入的主题快照；
2. 再通过 `client.theme.getCurrent()` 对齐当前正式主题；
3. `ChipsThemeProvider` 统一监听 `theme.changed`；
4. 播放区为了保持视频内容稳定，采用项目内收敛的深色 CSS 变量；主题系统负责生命周期接入与后续主题事件同步，不在当前版本让主题色干扰播放背景。
