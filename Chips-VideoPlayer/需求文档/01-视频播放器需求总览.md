# 视频播放器需求总览

## 1. 文档目标

冻结 `Chips-VideoPlayer` 首版的产品边界、正式使用场景、生态接入方式与交付目标，作为实现与验收依据。

## 2. 产品定位

`Chips-VideoPlayer` 是运行在薯片 Host 内部的 `type: app` 应用插件，负责承接“打开一个视频并进行基础播放控制”这一单一职责。

首版只做以下事情：

1. 打开视频；
2. 播放视频；
3. 保存视频副本；
4. 作为 Host 统一资源路由中的正式视频处理器。

本项目不是：

1. 视频编辑器；
2. 媒体资源管理器；
3. 多媒体转码器；
4. 字幕治理中心；
5. 流媒体聚合平台。

## 3. 正式接入场景

### 3.1 Host 资源路由启动

- 上游应用调用正式 `client.resource.open(...)`；
- Host 通过 `resource-handler:view:video/*` 自动命中视频播放器；
- 启动上下文通过 `launchParams.resourceOpen` / `targetPath` 注入视频资源；
- 视频播放器恢复到对应播放会话。

### 3.2 系统文件关联打开

- Host 通过 `file-handler:.mp4/.webm/...` 接收视频文件；
- 用户双击受支持格式时，Host 拉起视频播放器；
- 视频播放器读取 `targetPath` 恢复会话。

### 3.3 应用内手动打开

- 用户在播放器内部使用正式文件选择对话框选择本地视频；
- 应用通过正式 SDK / Bridge 接线打开并播放。

### 3.4 拖放打开

- 用户把视频文件拖到播放器窗口；
- 播放器通过正式 `platform.getPathForFile(...)` 恢复本地路径；
- 应用切换到新的视频播放会话。

## 4. 首版总体目标

1. 与图片查看器保持一致的正式应用插件架构；
2. 与 Host 资源路由、文件关联、主题、多语言和 Bridge / SDK 链路保持一致；
3. 满足本地视频与被路由视频的基础播放需求；
4. 不引入生态外临时协议、临时脚本或绕过 Host 的私有调用。

## 5. 参考约束

- 播放交互与控制条分组可参考开源项目 `DPlayer` 的设计经验；
- 但正式实现必须完全服从薯片生态的插件边界、主题系统、多语言系统与 Host 运行时接线。
