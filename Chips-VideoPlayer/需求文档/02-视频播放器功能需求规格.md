# 视频播放器功能需求规格

## 1. 视频格式支持

首版正式支持：

- `.mp4`
- `.webm`
- `.mov`
- `.m4v`
- `.ogv`
- `.ogg`

说明：

- 首版只接入浏览器 / Host 当前运行时中可直接由 HTML5 `<video>` 消费的基础格式；
- 不在本仓内新增自定义解码链路、流媒体引擎或转码兜底逻辑。

## 2. 视频打开

应用必须支持：

1. 读取 `launchParams.resourceOpen`；
2. 读取 `launchParams.targetPath`；
3. 使用正式文件选择器选择本地视频；
4. 通过拖放打开本地视频；
5. 对缺失路径、不支持格式、资源解析失败给出明确反馈。

## 3. 视频播放

应用必须提供以下基础播放能力：

1. 播放；
2. 暂停；
3. 时间轴拖动；
4. 前进 5 秒；
5. 后退 5 秒；
6. 静音；
7. 音量调节；
8. 倍速切换；
9. 全屏；
10. 画中画。

补充要求：

1. 当宿主不支持全屏或画中画时，界面应体现不可用状态；
2. 播放区骨架必须稳定，不因为视频尺寸变化而引起窗口 chrome 位移；
3. 视频必须占满窗口主画布，背景保持纯黑，视频不得被拉伸变形；
4. 必须有空态、加载态、成功态、错误态。

## 4. 播放器界面结构

首版正式界面必须满足：

1. 底部只有一个小型悬浮工具条；
2. 工具条按钮统一用图标呈现，按钮名称通过鼠标悬浮气泡展示；
3. 无文件时，窗口中央只保留一行带图标的打开提示；
4. 点击“更多”按钮后，右侧出现悬浮更多面板；
5. 更多面板固定包含“视频信息、字幕切换、音频切换”三个标签页；
6. 播放进行中且用户无交互时，工具栏应自动隐藏；鼠标移动、聚焦或交互后再次显示。

## 5. 键盘交互

首版键盘语义固定为：

1. `Space`：播放 / 暂停；
2. `ArrowLeft`：后退 5 秒；
3. `ArrowRight`：前进 5 秒；
4. `ArrowUp`：增大音量；
5. `ArrowDown`：减小音量；
6. `F`：切换全屏；
7. `M`：切换静音。

## 6. 保存视频

“保存视频”在首版中的正式语义是“将当前视频源保存为一个副本”。

要求如下：

1. 对本地文件源，使用正式保存对话框选择目标路径，并通过正式 `file.copy` 完成复制；
2. 对远端 URL 或不具备稳定本地路径的资源，回退为浏览器 / 宿主下载；
3. 目标路径与原始路径相同时，必须给出明确提示，不执行重复写入；
4. 成功、失败、取消都要有可见状态反馈。

## 7. 更多面板能力要求

更多面板中的三个标签页要求如下：

1. 视频信息页必须展示至少标题、来源类型、格式、总时长、当前位置、分辨率和资源路径；
2. 字幕页只展示当前视频运行时已经暴露的可切换字幕轨道，并支持关闭字幕；
3. 音频页只展示当前视频运行时已经暴露的可切换音轨，不自行生成虚假轨道；
4. 更多面板尺寸在有足够窗口空间时应保持固定，仅在小窗口下按可用空间收缩，并通过内部滚动承载溢出内容。

## 8. Manifest 与生态能力要求

应用必须正式声明：

1. `resource-handler:view:video/*`
2. `file-handler:.mp4/.webm/.mov/.m4v/.ogv/.ogg`
3. `resource.read`
4. `file.write`
5. `platform.read`
6. `theme.read`
7. `i18n.read`

## 9. 明确不在首版范围

以下能力不进入当前交付：

1. 视频剪辑、裁剪、旋转、滤镜；
2. 弹幕系统；
3. 外挂字幕加载与字幕编辑；
4. 清晰度切换与多码率治理；
5. 播放列表、历史记录、媒体库；
6. 自研流媒体协议或转码模块。
