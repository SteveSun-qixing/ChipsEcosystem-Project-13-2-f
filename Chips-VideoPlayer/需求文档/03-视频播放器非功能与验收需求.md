# 视频播放器非功能与验收需求

## 1. 架构与工程要求

1. 应用必须基于官方应用脚手架初始化；
2. 外层仍然是标准 `type: app` 应用插件；
3. 系统能力调用只允许通过 `chips-sdk` / `window.chips.*`；
4. 所有界面文案必须接入多语言词典；
5. 所有视觉样式必须走主题系统与 CSS token。

## 2. 质量要求

1. 不得保留模板身份泄露、示例占位、TODO 或临时兼容分支；
2. 运行时必须对启动参数、资源解析失败、媒体解码失败给出稳定反馈；
3. 控制条与播放区域在缩放窗口时应保持布局稳定；
4. 播放器窗口在任何尺寸下都应保持黑底全窗视频、不变形、底部工具条不遮挡主要播放信息；
5. 文档必须与真实 `manifest.yaml`、测试和正式脚本一致。

## 3. 文档要求

本项目必须至少维护以下文档体系：

1. `需求文档/`
2. `技术文档/`
3. `开发计划/`
4. `README.md`

## 4. 验收门禁

首版验收至少包括：

1. `resource.open` 能正确拉起视频播放器；
2. 系统文件关联能把受支持的视频扩展名交给视频播放器；
3. 应用内手动打开、拖放打开、本地副本保存可正常工作；
4. 播放、暂停、时间轴、音量、倍速、全屏、画中画行为符合预期；
5. 图标工具条、tooltip 和右侧更多面板在有文件与无文件场景下都保持稳定；
6. `npm run lint`
7. `npm test`
8. `npm run build`
9. `npm run validate`

## 5. 风险与边界

当前可接受但必须显式标注的边界：

1. 实际可播放的编解码能力受当前 Host / 浏览器运行时限制；
2. 远端资源的“保存”是下载语义，不保证与本地复制完全等价；
3. 开源 DPlayer 仅作为内部实现参考，不改变本项目对外契约。
