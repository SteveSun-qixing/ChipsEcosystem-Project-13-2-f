# ThemePack AGENTS

## 角色

- 本目录按子目录维护多个正式主题包工程；主题包负责提供 token、CSS 和契约实现，不负责 Host / SDK 运行时逻辑。

## 通用规则

- 每个主题包必须遵守“一包一外观”，以独立 `themeId` 交付，不在单包内混合 light / dark 双模式。
- 主题包只能消费既有 Theme Runtime、组件库契约和 token 标准；不要在主题工程中发明新的跨生态协议。
- 公共主题规范、token 对接标准和接口契约应沉淀在 `生态共用技术文档/`；单主题仓只记录如何消费既有契约。
- 如果发现组件契约、Theme Runtime 或打包链路缺口，先登记工单，再决定主题仓的后续工作。

## 验证

- 优先运行各主题工程的构建、校验和测试脚本。
