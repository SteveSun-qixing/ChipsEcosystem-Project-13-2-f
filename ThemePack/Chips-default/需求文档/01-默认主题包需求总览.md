# 默认主题包需求总览（Chips-default）

> 文档状态：需求基线稿  
> 对齐基线：生态设计原稿 + 生态共用技术文档（主题系统 + 组件库）  
> 项目路径：`ThemePack/Chips-default`

---

## 1. 项目背景与目标

薯片生态采用“一切皆插件”的扩展模型，所有 UI 组件的视觉样式由主题包负责。为了给生态提供一个稳定、长期可用的基础视觉基线，需要交付一套官方默认主题包：

- 作为 Host 安装完成后的默认主题；
- 作为组件库与应用开发的参考视觉样例；
- 作为第三方主题的继承与对比基线。

本项目 `ThemePack/Chips-default` 的目标是实现：

- **官方默认主题包**：`themeId: chips-official.default-theme`；
- 风格定位：简约、干净、优雅，无多余装饰与花哨动效；
- 与 `@chips/tokens` 和组件库主题契约完全对齐，覆盖所有当前已交付的 `chips.comp.*` 组件。

### 1.1 北极星目标

- 提供一套可持续 5–10 年演进的默认浅色主题：
  - 颜色体系中性、耐看，可作为多种业务皮肤的基础；
  - 间距与密度适中，兼顾桌面与笔记本常见分辨率；
  - 动效统一且克制，避免炫技式动画。
- 任何在薯片主机中运行的官方应用与组件库，在不指定主题的前提下自动使用本主题，并获得：
  - 清晰的层级关系与主次对比；
  - 一致的交互反馈与对焦样式；
  - 高可读性的文本与控件状态。

## 2. 范围与边界

### 2.1 In Scope（本项目范围内）

- 默认主题包本身的实现：
  - 五层 token：`ref/sys/comp/motion/layout`；
  - 与组件库对接的 `chips.comp.*` 组件级 token；
  - 基础 CSS：全局样式、常用交互组件、容器组件、反馈组件；
  - 主题契约与本地验证脚本。
- 与 Host / 组件库 / SDK 的集成行为说明：
  - 如何在 Host 中安装、应用与解析该主题；
  - 与组件库 `ChipsThemeProvider` 和 token resolver 的协作方式。
- 本项目自身的开发计划、测试策略与发布流程。

### 2.2 Out of Scope（本项目不做）

- 不在本主题包中：
  - 重新设计 token 结构或新增跨生态通用 token；
  - 修改 Host / 组件库 / SDK 的任何实现与对外接口；
  - 引入多模式主题（如 light/dark 合并在一个包中）。
- 不承担以下职责：
  - 针对具体业务（例如特定应用的品牌风格）设计主题；
  - 处理迁移旧版本主题的兼容性策略（兼容性由生态整体规划）。

## 3. 约束条件与质量要求

### 3.1 架构与规范约束

- 必须遵守：
  - 主题系统设计文档中的 L11 Theme Runtime 角色与接口；
  - 组件库《Token 与主题对接标准》中的五层 token 与 `chips.comp.*` 命名；
  - 主题包开发指南中的目录结构与 Manifest 规范。
- 对外接口：
  - 只通过 `theme.*` 动作与 SDK `client.theme.*` 暴露能力；
  - 不在主题包内部发明新的主题管理接口。

### 3.2 工程与质量约束

- 禁止临时版、简化版、占位实现：
  - token 层必须完整覆盖组件库已声明的组件；
  - CSS 层至少为常用组件提供可用的基础视觉；
  - 不留 TODO、占位样式或“以后再实现”的注释。
- 禁止随意修改架构设计：
  - 不更改 Theme Runtime 的五层结构与作用域链；
  - 不绕过 Host 的主题服务直接注入全局变量。
- 测试与门禁：
  - 本主题包必须内建构建与验证脚本（`build` / `validate:theme` / `test`）；
  - 在发布前必须通过本仓与 Host 联调的质量门禁。

## 4. 干系人与协作边界

| 干系人 | 角色 | 关注点 |
|---|---|---|
| Host 团队 | 主题运行时与对外接口维护者 | 默认主题是否稳定可用、是否满足 Host 对外行为与性能要求 |
| 组件库团队 | 头less 组件与主题契约维护者 | token 覆盖是否完整、契约是否满足所有组件视觉需求 |
| SDK 团队 | 主题调用封装维护者 | 默认主题是否通过 SDK 接口工作正常 |
| 应用开发者 | 使用默认主题的第一方开发者 | 默认主题是否足够中性、易于继承与覆盖 |
| 第三方主题开发者 | 在默认主题基础上二次设计 | 默认主题是否提供合理的 token 与 CSS 基线 |

默认主题包开发阶段与 `开发计划/00-开发计划索引.md` 对应：
