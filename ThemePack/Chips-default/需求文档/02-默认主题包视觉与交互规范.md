# 默认主题包视觉与交互规范

> 文档状态：视觉基线设计稿（简约优雅黑白浅色风格）  
> 适用范围：`ThemePack/Chips-default` 默认主题包

---

## 1. 整体风格定位

- 关键词：**简约 / 干净 / 克制 / 易读**。
- 背景：浅色为主，少量中性灰分层。
- 主色：适度饱和的蓝色（参考 `chips.ref.color.blue-500`）。
- 对比度：满足基本可访问性建议（常规文本对比度优先考虑 4.5:1，标题 >= 3:1）。
- 动效：仅使用小幅度、短时长的渐变，不使用复杂 keyframes 动画。

---

## 2. 色彩体系

### 2.1 基础色板（ref）

- 基于 `@chips/tokens` 中的 `ref` 定义：
  - 白色：`chips.ref.color.gray-0` 作为主背景色；
  - 文本主色：`chips.ref.color.gray-1000`；
  - 主色：`chips.ref.color.blue-500` 用于主按钮、强调元素；
  - 错误色：`chips.ref.color.red-500` 用于错误高亮。

### 2.2 语义色彩（sys）

- `chips.sys.color.surface`：普通容器与卡片背景；
- `chips.sys.color.on-surface`：普通文本与图标颜色；
- `chips.sys.color.primary`：主按钮、选中状态、主强调元素；
- `chips.sys.color.error`：错误提示文本与图标。

默认主题中，所有组件视觉均通过 `sys` 与 `comp` token 组合实现，无直接依赖业务名。

---

## 3. 排版与密度

### 3.1 字体

- 字体族：
  - 英文/数字：Roboto Flex / system-ui；
  - 中文：Noto Sans SC；
  - 实际接入由组件库与主题系统共同完成，本主题包不直接硬编码具体字体文件路径。

### 3.2 字号与行高

- 小号文本：`chips.text.sm`（约 14px）；
- 正文：`chips.text.md`（约 16px）；
- 标题：`chips.text.lg` / `chips.text.xl`（20px / 24px 级别）。

### 3.3 密度

- 列表项高度：约 `40px`；
- 表单元素高度（按钮、输入框）：约 `40px`；
- 默认外边距与内边距依据 `chips.ref.space.*` 统一控制。

---

## 4. 组件视觉规范（概要）

以下为常用组件的视觉基线说明（具体 token 对应关系在技术文档中给出）。

### 4.1 按钮（button）

- 主按钮：
  - 背景：主色 `chips.sys.color.primary`；
  - 文本：浅色 `chips.sys.color.surface`；
  - 圆角：中等（`chips.ref.radius.md`）；
  - hover：轻微变暗或保持同色，增加轻微阴影或亮度；
  - active：略微压暗，保持可识别；
  - disabled：背景改为浅灰，文本为中灰。
- 次级按钮（可选）：使用透明背景 + 边框，对比主按钮更弱。

### 4.2 输入框（input）

- 背景：`chips.sys.color.surface`；
- 文本：`chips.sys.color.on-surface`；
- 边框：单像素浅灰；
- focus：使用主色描边，增加轻微外发光或阴影以提示焦点；
- 禁用：浅灰背景，文本和占位符色降低对比度。

### 4.3 选择控件（checkbox / radio / switch）

- 控件尺寸适中，统一圆角风格；
- 选中态使用主色填充，图标为浅色；
- 禁用态使用浅灰填充与中灰边框；
- 文案使用常规正文颜色。

### 4.4 容器组件（dialog / tool-window / tabs / menu 等）

- 背景：统一使用 `chips.sys.color.surface`；
- 标题区域：与背景颜色一致或略微加深；
- 边框：细线中灰；
- 选中/高亮态使用主色作为线条或背景块；
- focus：使用主色描边或阴影。

### 4.5 提示与反馈（notification / toast / empty-state / skeleton）

- Notification / Toast：
  - 背景为浅色，错误/警告信息使用主色或错误色边条；
  - 文本对比度清晰，不使用复杂的渐变背景。
- EmptyState：
  - 使用简单图形或图标与简短文案，颜色中性；
  - 不使用复杂插画。
- Skeleton：
  - 使用淡灰色占位块，轻微亮度变化即可，避免过度动画。

---

## 5. 动效规范

- 过渡时长：
  - 快速交互（按钮、输入框聚焦）：`chips.motion.duration.fast`（约 150ms）；
  - 一般交互（对话框出现/隐藏）：`chips.motion.duration.normal`（约 250ms）。
- 缓动曲线：
  - 默认使用 `chips.motion.easing.standard`；
- 禁止：
  - 闪烁、强烈缩放等明显干扰用户注意力的动效；
  - 超过 400ms 的长时间动画。

---

## 6. 交互细节

- hover：颜色变化温和，避免强烈闪烁；
- focus：统一使用主色轮廓/阴影，保证键盘可访问性；
- active：略微压暗或缩小，反馈按压感；
- disabled：通过降低对比度 + 禁止光标体现，不改变组件布局。

以上规范为视觉与交互的基线约束，具体 token 映射和 CSS 实现会在技术文档与代码中按此落地。
