# 暗夜主题包技术架构与集成说明

> 文档状态：技术基线设计稿  
> 适用范围：`ThemePack/Chips-theme-default-dark` 暗夜主题包工程

---

## 1. 在生态中的位置

- 暗夜主题包是一个标准的 `type: theme` 插件：
  - 通过 `manifest.yaml` 声明元信息与 `themeId` 等主题字段；
  - 通过 `.cpk` 打包后由 Host 插件运行时进行安装与管理；
  - 通过 `theme.*` 服务域动作进行加载、应用、解析与契约查询。
- Host 侧：
  - 在 `RuntimeState` 中维护 `themes: ThemeRecord[]` 与 `currentThemeId`；
  - 暗夜主题以独立主题记录形式注册：`chips-official.default-dark-theme`；
  - Theme Runtime 使用 `ThemeTokenLayers`（`ref/sys/comp/motion/layout`）解析 token 并输出扁平变量表。
- 组件库侧：
  - 通过 `chips.comp.*` token 消费组件级视觉数据；
  - 通过 Theme hooks 与 token resolver 将扁平 token 映射为 CSS 变量或框架内部 token。

本暗夜主题包只负责 **提供数据与 CSS**，不包含任何业务逻辑与运行时代码。

---

## 2. 数据模型与文件结构

### 2.1 ThemeTokenLayers 对应关系

暗夜主题包的 `dist/tokens.json` 使用以下结构：

```json
{
  "ref": { ... },
  "sys": { ... },
  "comp": { ... },
  "motion": { ... },
  "layout": { ... }
}
```

- `ref`：对齐 `@chips/tokens` 中的 `ref.json`；
- `sys`：对齐 `@chips/tokens` 中的 `sys.json`；
- `motion`：对齐 `@chips/tokens` 中的 `motion.json`；
- `layout`：对齐 `@chips/tokens` 中的 `layout.json`；
- `comp`：合并 `@chips/tokens` 中所有 `comp/*.json`（按钮、输入、对话框、Tabs 等）。

通过 Theme Runtime 的 `flattenLayer` 处理后，可以得到类似：

- `chips.ref.color.gray-0`
- `chips.sys.color.primary`
- `chips.comp.button.root.surface.idle`

等扁平 token 键名。

### 2.2 目录结构与职责

```text
ThemePack/Chips-theme-default-dark/
├─ manifest.yaml        # 插件与主题声明
├─ package.json         # NPM 配置与脚本
├─ tsconfig.json        # TypeScript 配置
├─ chips.config.mjs     # Chips Dev 配置（类型为 theme）
├─ tokens/              # 暗夜主题 token 源（对齐 @chips/tokens）
├─ src/                 # 构建与验证脚本
├─ styles/              # CSS 基础视觉实现
├─ contracts/           # 主题契约与最小功能集描述
├─ tests/               # 单元与验证测试
└─ 归档/                # 脚手架初始 token 模板归档（不参与构建）
```

## 3. 与 Host 的集成

### 3.1 Manifest 与 ThemeRecord

`manifest.yaml` 中的关键字段：

- `id`: 插件 ID，反向域名格式（如 `theme.theme.chips-official-default-dark-theme`）；
- `version`: 语义化版本；
- `type: theme`；
- `permissions: [theme.read]`（默认只需要读取主题能力）；
- `entry.tokens: dist/tokens.json`；
- `entry.themeCss: dist/theme.css`；
- `themeId: chips-official.default-dark-theme`；
- `displayName: 薯片官方 · 暗夜主题`。

Host 侧加载主题包后，会形成内部 `ThemeRecord`：

- `id`: `themeId`；
- `name`: `displayName`；
- `publisher`: 根据 manifest 或 Host 策略确定；
- `css`: 来自本主题包构建的 `dist/theme.css`；
- `tokens`: 来自本主题包构建的 `dist/tokens.json`。

### 3.2 Theme Runtime 流程

Host 的主题服务在执行 `theme.apply` / `theme.resolve` 时：

- 将当前主题链（默认只包含本主题）转换为 `ThemeTokenLayers`；
- 使用 Theme Runtime 的解析算法解析引用；
- 使用 `theme.contract.get` 与契约守卫对 token 完整性进行检查；
- 向组件库与 UI 层发出 `theme.changed` 事件。

本主题包需要确保：

- `tokens.json` 中的结构可以被 `asThemeTokenLayers` 正确解析；
- `chips.comp.*` token 集与组件库的契约定义一致；
- 如果未来 Host 扩展契约校验范围，本主题包不需要大的结构调整即可通过。

## 4. 与组件库的集成

### 4.1 token 与组件契约

组件库的主题契约在 `Chips-ComponentLibrary/packages/theme-contracts/contracts/components/*.contract.json` 中定义，每个组件对应需要覆盖的一组 `chips.comp.*` token 键名。

暗夜主题包通过：

- 对齐 `@chips/tokens` 中的 `comp/*.json`；
- 确保构建输出的 `comp` 层中包含所有契约所需 token；

从而保证在组件库中使用 `useComponentTokens("button")` 等接口时，得到完整的 token 映射。

### 4.2 CSS 层消费方式

组件库主要通过：

- 将 Theme Runtime 输出的扁平 token 写入 CSS 变量（`--chips-*`）；
- 在适配图层（如 `ChipsThemeProvider` + Tamagui adapter）中将 token 进一步映射至组件内部。

本主题包在 `styles/` 中实现：

- 基于 CSS 变量的基础样式定义；
- 使用简洁类名和 data-attribute 挂点；
- 不直接依赖组件内部实现细节，只依赖公共契约中暴露的类、属性或挂点（若已定义）。

## 5. 本工程中的脚本与命令

### 5.1 构建脚本

- `npm run build:tokens`：
  - `src/build-tokens.ts`；
  - 读取 `tokens/ref.json/sys.json/motion.json/layout.json` 与 `tokens/comp/*.json`；
  - 输出 `dist/tokens.json`。
- `npm run build:css`：
  - `src/build-css.ts`；
  - 读取 `styles/base.css` 与 `styles/components/*.css`、`styles/motions.css`；
  - 输出 `dist/theme.css`。
- `npm run build`：
  - 顺序执行 `build:tokens` 与 `build:css`。

### 5.2 验证脚本与测试

- `npm run validate:theme`：
  - 执行 `src/validate-theme.ts`；
  - 静态检查 `dist/tokens.json` 的结构与部分关键 `chips.comp.*` token 存在性。
- `npm test`：
  - 使用 Vitest 运行 `tests/*.spec.ts`；
  - 对构建完成后的 token 结构与契约进行基本验证。

---

## 6. 小结

暗夜主题包在技术上遵循“数据与 CSS 分离”的原则：

- 数据层：完全对齐 @chips/tokens 与主题契约；
- CSS 层：以简约风格消费 token，确保视觉统一与可维护性；
- 集成层：通过 Host 的主题服务与组件库的 token resolver 完成闭环，不修改任何运行时代码结构。
