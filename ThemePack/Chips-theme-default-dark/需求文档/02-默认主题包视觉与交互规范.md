# 暗夜主题包视觉与交互规范

> 文档状态：视觉基线设计稿（简约沉静暗夜风格）
> 适用范围：`ThemePack/Chips-theme-default-dark` 暗夜主题包

---

## 1. 整体风格定位

- 关键词：**沉静 / 克制 / 专注 / 清晰**。
- 背景：深灰蓝黑色为主，使用低对比层次区分画布、容器、浮层与选中态。
- 主色：冷调蓝色，用于主操作、焦点、高亮反馈。
- 对比度：正文优先满足 4.5:1，关键按钮与错误提示维持明确可识别度。
- 动效：短时长、低幅度，不使用炫技型动画。

---

## 2. 色彩体系

### 2.1 基础色板（ref）

- 画布基底：`chips.ref.color.gray-950`，用于应用整体背景。
- 主容器：`chips.ref.color.gray-900` / `gray-800`，用于卡片、表单、面板。
- 主文本：`chips.ref.color.gray-0`。
- 次级文本：`chips.ref.color.gray-100` / `gray-200`。
- 主色：`chips.ref.color.blue-600`，hover 与 active 分别使用邻近明暗层级。
- 错误色：`chips.ref.color.red-500`。

### 2.2 语义色彩（sys）

- `chips.sys.color.canvas`：页面级底色。
- `chips.sys.color.surface`：基础容器表面。
- `chips.sys.color.surface-raised`：高于基础面的一层容器。
- `chips.sys.color.surface-elevated`：浮层与聚焦态面。
- `chips.sys.color.surface-accent`：选中、激活、焦点附近的低饱和强调底色。
- `chips.sys.color.on-surface`：主要文字与图标。
- `chips.sys.color.on-surface-muted` / `on-surface-soft`：说明、辅助、占位内容。
- `chips.sys.color.border-subtle` / `border-strong`：分隔线与边框。
- `chips.sys.color.primary` / `primary-hover` / `primary-active`：主操作色。
- `chips.sys.color.focus-ring`：键盘焦点轮廓。
- `chips.sys.color.error`：错误文本、边框与状态提示。

暗夜主题中不依赖业务命名，所有视觉表现均通过 `sys` 与 `comp` token 组合完成。

---

## 3. 排版与密度

### 3.1 字体

- 英文/数字：`Roboto Flex` 优先，回退 `system-ui`。
- 中文：`Noto Sans SC` 优先。
- 主题 CSS 只声明字体回退链，不直接绑定特定字体文件加载方式。

### 3.2 字号与行高

- 小号文本：约 `13px - 14px`，用于辅助文案与提示。
- 正文：约 `14px - 16px`。
- 面板标题：约 `16px`，卡片标题可提升至 `20px - 24px`。

### 3.3 密度

- 延续生态默认舒适密度，控件高度保持约 `40px`。
- 间距沿用 `chips.ref.space.2/3/4`，避免在暗色界面中出现拥挤或过度松散。

---

## 4. 组件视觉规范（概要）

### 4.1 按钮（button）

- 主按钮使用冷调蓝色底，hover 更明亮，active 更稳重。
- 文本保持高对比浅色，disabled 改为低对比深色容器。
- 圆角中等，不添加发光描边或重阴影。

### 4.2 输入框（input）

- 背景采用比页面略高一层的深色容器。
- idle 为细致低对比分隔边框，focus 使用焦点色与轻微层级提升。
- 占位符弱化但保持可读，不出现过低对比的“消失式”占位文本。

### 4.3 选择控件（checkbox / radio / switch）

- 未选中态采用深色容器 + 细边框；
- 选中态使用主色，但面积控制克制，保持整体暗色秩序；
- 文案使用主文本色，禁用态整体降对比。

### 4.4 容器组件（dialog / tool-window / tabs / menu 等）

- 通过 `surface / surface-raised / surface-elevated` 三层建立结构层次。
- 弹层与菜单阴影偏暗、边框偏细，避免浅色发光感。
- 选中态优先使用低饱和蓝灰强调底，而不是整块高亮纯蓝。

### 4.5 提示与反馈（notification / toast / empty-state / skeleton）

- 反馈容器与内容区使用深色基底，说明文字使用次级文本色。
- 错误边框/状态只在关键区域使用红色，避免大面积警报式视觉噪声。
- Skeleton 使用深浅两个近邻层级做轻微脉动，不做明显闪烁。

---

## 5. 动效规范

- 快速交互：`150ms` 左右。
- 一般过渡：`220ms - 250ms`。
- 缓动保持标准曲线，强调平稳与低打扰。
- 禁止强烈缩放、闪烁、霓虹光效等会破坏专注度的表现。

---

## 6. 交互细节

- hover：仅做轻微亮度/底色变化。
- focus：统一使用清晰但不过分刺眼的焦点轮廓。
- active：强调反馈应短促、稳定，不产生明显抖动。
- disabled：通过降低对比度与交互光标来表达，不改变组件结构与尺寸。
