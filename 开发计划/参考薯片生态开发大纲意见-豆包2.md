# 薯片生态开发大纲

## 一、开发整体规划

薯片生态采用从底层到上层的分层开发策略，遵循 "底层厚、上层薄" 的设计理念，先构建基础架构和核心服务，再逐步开发插件系统和上层应用。开发过程严格遵循十二层架构设计，确保各层之间的解耦和标准化。

## 二、开发顺序与详细内容

### 第一阶段：底层基础架构开发

#### 1. 平台抽象层（L1: PAL）开发

**开发内容**：

- 适配 Windows、macOS、Linux 三大操作系统的底层能力

- 封装系统级 API，提供统一的平台能力接口

- 实现系统信息获取、屏幕信息查询、能力检测、全局快捷键等功能

- 开发平台兼容性测试套件，确保跨平台一致性

**交付物**：

- PAL 层核心代码库

- 平台兼容性测试报告

- 平台抽象层 API 文档

#### 2. Host 内核（L2: Host Kernel）开发

**开发内容**：

- 实现中心路由系统，支持`namespace.action`格式的路由命名

- 开发服务注册机制，支持`exact/prefix/wildcard`匹配模式

- 构建事件总线系统，支持订阅模式、过滤器、超时配置

- 实现生命周期管理，服务状态机：`unloaded -> loading -> ready -> running -> stopping -> stopped/error`

- 开发路由契约描述符，实现参数 schema 校验和权限判定

- 实现并发与熔断机制，路由级超时配置，高频失败路由软熔断

**交付物**：

- 内核核心代码库

- 路由系统测试工具

- 内核 API 文档

- 生命周期管理工具

#### 3. Host 服务域（L3: Host Services）开发

**开发内容**：

- 开发 16 个核心服务：

    - `file`服务：文件读写、stat、list 等基础文件操作

    - `resource`服务：资源解析、元数据读取、二进制资源获取

    - `config`服务：层级化配置管理，支持四级配置覆盖

    - `theme`服务：主题包注册、查询、分发，六级主题继承解析

    - `i18n`服务：多语言支持，语言包加载、文本翻译

    - `window`服务：窗口管理，打开、聚焦、 resize、状态设置

    - `plugin`服务：插件安装、启用、禁用、卸载、查询

    - `module`服务：模块挂载、卸载、查询、列表管理

    - `platform`服务：平台能力封装，系统信息、外部应用打开

    - `log`服务：统一日志记录，分级过滤、文件输出

    - `credential`服务：凭证管理，加密存储认证信息

    - `card`服务：卡片文件解析、渲染、验证

    - `box`服务：箱子文件打包、解压、检查

    - `zip`服务：ZIP 文件压缩、解压、列表查看

    - `serializer`服务：数据序列化，YAML、JSON 格式解析

    - `control-plane`服务：治理聚合，健康检查、指标监控、诊断

- 为每个服务实现标准接口，包括输入 schema、输出 schema、权限、超时、错误码

- 实现服务间通信的路由机制，确保所有服务调用都通过内核路由

**交付物**：

- 16 个核心服务的代码实现

- 服务接口文档

- 服务测试套件

- 服务治理工具

### 第二阶段：插件运行时与桥接层开发

#### 4. Plugin Runtime（L4: 插件运行时）开发

**开发内容**：

- 实现插件加载机制，支持五种插件类型（app、card、layout、module、theme）

- 开发插件隔离策略，包括 App 级隔离（独立 BrowserWindow）、Card 渲染隔离（iframe）、权限隔离

- 实现插件会话管理，每个插件会话包含`pluginId`、`pluginType`、`permissions`、`locale`、`themeId`、`launchParams`、`sessionNonce`

- 开发插件资源配额管理，定义 CPU / 内存 / 消息频率软配额，超配额时触发降级

- 实现插件运行时安全流水线：Manifest 校验、签名与来源校验、启动握手、权限快照加载、沙箱注入、运行时审计、会话结束清理

**交付物**：

- 插件运行时核心代码

- 插件隔离机制实现

- 插件安全校验工具

- 插件运行时监控工具

#### 5. Bridge Transport（L5: 桥接传输层）开发

**开发内容**：

- 实现`window.chips.*`标准接口，包括`invoke`、`on`、`once`、`emit`方法

- 开发 IPC 主通道`chips:invoke`和子能力通道（`chips:window:*`、`chips:dialog:*`、`chips:plugin:*`等）

- 实现 Electron 的 contextBridge 机制，确保插件只能通过标准接口访问系统能力

**交付物**：

- 桥接传输层核心代码

- IPC 通道实现

- 桥接接口文档

#### 6. Runtime Client（L6: 运行时客户端）开发

**开发内容**：

- 实现参数规范化、协议映射、错误归一、超时控制、重试与回退功能

- 开发候选动作回退能力，用于历史差异吸收

- 实现错误归一映射，将`bridge`层、`service`层、`runtime`层错误统一为标准错误对象

**交付物**：

- Runtime Client 核心代码

- 错误处理工具

- 重试策略实现

- Runtime Client 文档

#### 7. UI Hooks（L7: UI 钩子）开发

**开发内容**：

- 实现`useTheme`、`useI18n`、`useFile`、`useWindow`、`usePlugin`、`useConfig`等 UI Hooks

- 确保 Hook 仅调用 Runtime Client，不直接连接 Transport 层

- 开发面向页面的能力调用接口，简化上层应用的系统能力调用

**交付物**：

- UI Hooks 库

- Hooks 使用文档

- Hooks 测试套件

### 第三阶段：前端框架与渲染层开发

#### 8. Declarative UI（L8: 声明式 UI 层）开发

**开发内容**：

- 实现核心语义原语：`View`、`Stack`、`Grid`、`Form`、`List`

- 开发声明树节点模型，支持`id`、`type`、`props`、`state`、`bindings`、`events`、`themeScope`、`children`等属性

- 实现组合模式规范，采用 Vercel 组合模式，优先使用 Compound Components，避免 boolean 模式扩张

- 开发副作用与事件绑定模型，区分`ui-effect`、`runtime-effect`、`telemetry-effect`，禁止在渲染阶段直接触发`runtime-effect`

**交付物**：

- 声明式 UI 库

- 语义原语实现

- 组合模式规范文档

- 事件绑定工具

#### 9. Unified Rendering（L9: 统一渲染层）开发

**开发内容**：

- 实现渲染流水线：Node Normalize、Contract Validate、Theme Resolve、Layout Compute、Render Commit、Effect Dispatch

- 开发目标适配器，支持`App Root`、`Card Iframe`、`Module Slot`、`Offscreen Render`

- 实现渲染一致性约束，确保同一声明树在不同目标适配器中语义一致

- 开发失败隔离机制，实现节点级 / 区域级 Error Boundary，单节点失败不影响整页渲染

- 实现调度与增量更新，维护`render queue`，将大树更新拆分为可中断批次，`List/Grid/Table`默认启用可见区域增量提交

**交付物**：

- 统一渲染引擎

- 目标适配器实现

- 渲染一致性测试工具

- 增量更新调度器

#### 10. Component Headless（L10: 无头组件库）开发

**开发内容**：

- 基于 Zag.js 和 Ark UI 构建无头组件库

- 实现必备基础层组件：Button、Input、Select、Checkbox、Radio、Switch、Dialog、Popover、Tabs、Menu、Tooltip

- 实现必备数据层组件：DataGrid、Tree、VirtualList、FormField、FormGroup、DateTime、CommandPalette

- 实现必备工作台层组件：SplitPane、DockPanel、Inspector、PanelHeader、CardShell、ToolWindow

- 实现必备系统层组件：ErrorBoundary、LoadingBoundary、Notification、Toast、EmptyState、Skeleton

- 确保组件只负责状态机、结构、a11y，不包含任何视觉硬编码

- 为每个组件声明可组合子部件清单、主题接口点清单、可访问性语义清单、单测最小覆盖矩阵

**交付物**：

- 无头组件库

- 组件接口文档

- 组件测试套件

- 组件可访问性报告

#### 11. Theme Runtime（L11: 主题运行时）开发

**开发内容**：

- 实现 token 五层解析：`ref`（原始值）、`sys`（语义值）、`comp`（组件值）、`motion`（动效值）、`layout`（布局密度值）

- 实现主题作用域链：`component → card → workspace → app → global`


- 开发主题覆盖规则：作用域优先级、选择器特异性、声明顺序

- 实现主题解析算法：加载作用域链主题快照、按 token 层级合并、执行冲突裁决、生成变量表与组件映射表、注入作用域样式并触发`theme.changed`

- 实现主题契约门禁：token 完整性校验、组件接口点校验、动效参数安全校验、颜色对比度校验

**交付物**：

- 主题运行时核心代码

- 主题解析引擎

- 主题契约校验工具

- 主题管理工具

## 三、开发质量保障

### 1. 测试策略

- 为每个模块开发单元测试，确保核心功能的正确性

- 开发集成测试，确保各模块之间的交互正常

- 开发端到端测试，模拟用户场景测试完整功能
