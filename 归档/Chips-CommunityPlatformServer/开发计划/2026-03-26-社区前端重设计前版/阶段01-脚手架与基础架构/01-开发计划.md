# 阶段一开发计划：项目脚手架与基础架构

**阶段编号**：Phase 1  
**前置阶段**：无  
**预计工时**：2-3 天  
**目标**：搭建完整可运行的骨架，所有基础设施服务启动正常，端到端健康检查可通。

---

## 交付标准

- `docker-compose up` 后，`GET /api/v1/health` 返回 200
- PostgreSQL 数据库 schema 已初始化（所有表存在）
- MinIO bucket 已自动创建
- Redis 可连接
- TypeScript 编译无报错
- ESLint 无报错

---

## 任务清单

### 1. Monorepo 初始化

- [ ] 创建根 `package.json`（设置 `pnpm-workspace.yaml`）
- [ ] 创建三个子包：`packages/server`、`packages/web`、`packages/admin`
- [ ] 配置根 `tsconfig.base.json`（`strict: true`, `target: ES2022`, `module: NodeNext`）
- [ ] 各子包继承根 `tsconfig.json`
- [ ] 配置 ESLint（`@typescript-eslint/recommended`，无额外规则集）
- [ ] 配置 `.gitignore`、`.npmrc`、`.env.example`

### 2. Fastify 服务搭建（`packages/server`）

- [ ] 安装依赖：`fastify`、`@fastify/cors`、`@fastify/helmet`
- [ ] 创建 `src/app.ts`：Fastify 实例工厂函数（注册插件，不含路由）
- [ ] 创建 `src/server.ts`：入口，读取 `PORT/HOST` 环境变量，启动服务
- [ ] 创建健康检查路由：`GET /api/v1/health` → `{ status: "ok", timestamp: "..." }`
- [ ] 配置 Pino 日志（开发模式 pretty，生产模式 JSON）
- [ ] 配置 Zod 环境变量校验（`src/config/env.ts`）

### 3. 数据库配置（PostgreSQL + Drizzle ORM）

- [ ] 安装依赖：`drizzle-orm`、`drizzle-kit`、`pg`、`@types/pg`
- [ ] 创建 `src/db/client.ts`：连接池配置（使用 `DATABASE_URL`）
- [ ] 创建 `src/db/schema/users.ts`：users 表 Drizzle schema
- [ ] 创建 `src/db/schema/rooms.ts`：rooms 表
- [ ] 创建 `src/db/schema/cards.ts`：cards 表
- [ ] 创建 `src/db/schema/boxes.ts`：boxes 表
- [ ] 创建 `src/db/schema/index.ts`：统一导出
- [ ] 配置 `drizzle.config.ts`
- [ ] 生成并验证 migration 文件（`drizzle-kit generate`）
- [ ] 创建 `src/db/migrate.ts`：启动时自动执行迁移

### 4. Redis 配置

- [ ] 安装 `ioredis`
- [ ] 创建 `src/cache/redis.ts`：Redis 连接单例，含连接错误处理

### 5. MinIO / S3 配置

- [ ] 安装 `@aws-sdk/client-s3`
- [ ] 创建 `src/storage/minio.ts`：S3 客户端实例
- [ ] 创建 `src/storage/buckets.ts`：定义所有 bucket 名称常量
- [ ] 创建 `src/storage/init.ts`：启动时检查并创建所有必要 bucket（若不存在）

### 6. Docker Compose 编排

- [ ] 创建 `deploy/docker-compose.yml`：
  - `postgres`：PostgreSQL 16，含数据卷
  - `redis`：Redis 7，含数据卷
  - `minio`：MinIO，含数据卷，设置初始 access key
  - `server`：Fastify API，挂载 `.env`，依赖 postgres/redis/minio
- [ ] 创建 `deploy/.env.example`：所有必要环境变量及注释
- [ ] 创建 `deploy/docker-compose.dev.yml`：开发环境覆盖（端口暴露便于调试）
- [ ] 编写 `Dockerfile`（`packages/server`）：多阶段构建，最终镜像基于 `node:20-alpine`

### 7. Nginx 基础配置

- [ ] 创建 `deploy/nginx/nginx.conf`：
  - 代理 `/api/*` 到 Fastify
  - 代理 `/cdn/*` 到 MinIO（资源访问）
  - 预留前端静态文件服务位置
- [ ] 在 docker-compose 中添加 `nginx` 服务

### 8. 开发工具脚本

- [ ] 根 `package.json` scripts：
  - `dev`：并发启动 server + web（开发模式）
  - `build`：各子包构建
  - `db:migrate`：执行数据库迁移
  - `db:generate`：生成 migration
  - `lint`：全工作区 lint
  - `test`：全工作区测试

## 验证方式

```bash
# 1. 启动所有服务
cd deploy
docker-compose up -d

# 2. 等待服务就绪，检查健康接口
curl http://localhost/api/v1/health
# 期望: {"status":"ok","timestamp":"..."}

# 3. 检查数据库表
docker-compose exec postgres psql -U chips -d chips_community -c "\dt"
# 期望: 看到 users, rooms, cards, boxes 四张表

# 4. 检查 MinIO bucket
# 访问 http://localhost:9001，登录 MinIO Console，确认 bucket 已创建
```
