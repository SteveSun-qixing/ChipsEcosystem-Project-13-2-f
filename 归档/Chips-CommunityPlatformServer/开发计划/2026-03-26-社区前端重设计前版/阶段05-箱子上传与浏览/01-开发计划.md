# 阶段五：箱子上传与浏览

**阶段编号**：Phase 5  
**前置阶段**：Phase 4（卡片流水线完成）  
**预计工时**：2 天  
**目标**：支持箱子文件上传，可以通过 URL 浏览箱子中的卡片列表。

---

## 交付标准

- 用户上传有效的 `.box` 文件后，服务端解析并入库
- 通过 `GET /boxes/{boxId}` 可获取箱子摘要和卡片列表
- 前端可上传箱子并在箱子详情页查看卡片缩略列表

---

## 关键任务

### 服务端

#### 箱子解包模块（`src/pipeline/box-unpack.ts`）

- [ ] 校验文件魔数（ZIP）
- [ ] 解压到临时目录，路径安全检查
- [ ] 解析 `.box/metadata.yaml`（箱子 ID、名称、布局插件、标签等）
- [ ] 解析 `.box/structure.yaml`（卡片引用列表，含 URL、摘要）
- [ ] 返回 `BoxUnpackResult`：包含 metadata + structure

#### TypeScript 类型定义（`src/types/box.ts`）

```typescript
interface BoxMetadata {
  chip_standards_version: string;
  id: string;
  name: string;
  layout_plugin?: string;
  tags?: string[][];
  created_at: string;
  updated_at: string;
}

interface BoxCardRef {
  url: string;
  card_id?: string;
  title?: string;
  cover_url?: string;
  sort_index?: number;
  enabled?: boolean;
}

interface BoxStructure {
  cards: BoxCardRef[];
}
```

#### BoxService（`src/services/box.service.ts`）

- [ ] `createBox(userId, roomId, boxFilePath)` → 解包、入库（`status` 字段箱子无需流水线，直接 `ready`）
- [ ] `getBoxById(boxId, requesterId?)` → 权限校验，返回箱子 + 关联卡片摘要
  - 对于 structure.yaml 中的每条卡片引用 URL，尝试在本社区 `cards` 表中通过 `card_file_id` 匹配
  - 匹配到则补充 `community_card_id`、`html_url`、社区内封面 URL 等信息
- [ ] `deleteBox(boxId, userId)` → 删除数据库记录（MinIO 无额外资源，直接删除）
- [ ] `listUserBoxes(userId, requesterId?)` → 分页列表

#### 上传路由

- [ ] `POST /api/v1/upload/box`（需认证）
  - multipart 字段：`file`（.box）+ `roomId?` + `visibility?`
  - 校验扩展名 `.box`，大小 ≤ 100MB
  - 调用 BoxService.createBox
  - 返回 201：`{ data: { boxId } }`

#### 查询路由

- [ ] `GET /api/v1/boxes/:boxId` → 箱子详情 + 卡片引用列表
- [ ] `PATCH /api/v1/boxes/:boxId` → 更新 roomId / visibility
- [ ] `DELETE /api/v1/boxes/:boxId` → 删除
- [ ] `GET /api/v1/users/me/boxes` → 我的箱子列表

#### `GET /api/v1/rooms/:roomId/contents` 更新

- [ ] 同时返回房间内的箱子列表（结合已有卡片列表）

### 前端

- [ ] 箱子上传区：支持 `.box` 文件拖拽上传，选择目标房间
- [ ] `BoxDetailPage`（`/boxes/:boxId`）：
  - 箱子标题、布局插件名、创建时间
  - 卡片引用列表网格（封面 + 标题，若在社区内有对应卡片则封面/标题可点击跳转）
  - 对于社区外 URL（非本站卡片），显示外部链接标识
- [ ] Dashboard 新增"我的箱子"标签页
