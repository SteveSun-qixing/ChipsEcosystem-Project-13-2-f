# 阶段七：管理后台

**阶段编号**：Phase 7  
**前置阶段**：Phase 6（公开浏览完成）  
**预计工时**：2-3 天  
**目标**：平台管理员可以通过 Web UI 管理用户、内容和查看运营数据。

---

## 交付标准

- 管理员可登录独立后台（`/admin`）
- 可查看平台总览数据
- 可管理用户（禁用 / 启用）
- 可删除违规内容
- 管理员账号在容器首次启动时通过环境变量自动创建

---

## 关键任务

### 服务端

#### AdminService（`src/services/admin.service.ts`）

- [ ] `getPlatformStats()` → 统计用户数、卡片数、箱子数、存储用量（MinIO bucket size）
- [ ] `listUsers(params)` → 分页，支持关键词搜索
- [ ] `setUserActive(userId, isActive)` → 禁用 / 启用账号
- [ ] `forceDeleteContent(type, id)` → 强制删除卡片 / 箱子（含 CDN 资源）

#### 管理路由（`src/routes/admin.ts`）

- [ ] 所有 admin 路由需 `role === 'admin'` 校验（`requireAdmin` preHandler）
- [ ] `GET /admin/api/v1/stats`
- [ ] `GET /admin/api/v1/users?page=&q=`
- [ ] `PATCH /admin/api/v1/users/:userId`（is_active 字段）
- [ ] `GET /admin/api/v1/content?page=&type=card,box&q=`
- [ ] `DELETE /admin/api/v1/content/:type/:id`
- [ ] `GET /admin/api/v1/storage` → 各用户存储用量 Top 20

#### 管理员账号初始化（`src/db/init-admin.ts`）

- [ ] 应用启动时检查 `ADMIN_USERNAME` / `ADMIN_PASSWORD` 环境变量
- [ ] 若 admin 账号不存在，自动创建（role=admin）
- [ ] 若已存在，跳过（不覆盖密码）
