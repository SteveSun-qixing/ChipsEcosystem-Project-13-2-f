# 环境配置与依赖清单

## 1. 配置入口

- 主配置文件：`Chips-CommunityPlatformServer/deploy/.env`
- 示例配置：`Chips-CommunityPlatformServer/deploy/.env.example`
- 运行时解析：`Chips-CommunityPlatformServer/packages/server/src/config/env.ts`
- Docker 编排入口：`Chips-CommunityPlatformServer/deploy/docker-compose.yml`
- 当前已核对：`.env` 与 `.env.example` 都包含 Host 动态插件配置段

## 2. 环境变量总表

### 2.1 服务基础

| 变量 | 当前用途 | 当前默认口径 |
|---|---|---|
| `NODE_ENV` | 控制日志、CORS、调试开关 | `development` |
| `PORT` | Fastify 容器内监听端口 | `3000` |
| `HOST` | Fastify 监听地址 | `0.0.0.0` |
| `BASE_URL` | 平台对外根地址，用于 API 文档、对象 URL、前后端同源口径 | `http://localhost` |

### 2.2 数据层

| 变量 | 当前用途 |
|---|---|
| `DATABASE_URL` | PostgreSQL 连接串 |
| `REDIS_URL` | Redis 连接串 |

### 2.3 对象存储

| 变量 | 当前用途 | 说明 |
|---|---|---|
| `S3_ENDPOINT` | 服务端写入 MinIO / S3 的内网地址 | Docker 内默认 `http://minio:9000` |
| `S3_PUBLIC_URL` | 可选的公开资源根地址 | 留空时回退到 `${BASE_URL}/cdn` |
| `S3_ACCESS_KEY` | 对象存储访问 key | 与 MinIO root user 对齐 |
| `S3_SECRET_KEY` | 对象存储 secret | 与 MinIO root password 对齐 |
| `S3_REGION` | S3 region | 默认 `us-east-1` |
| `S3_FORCE_PATH_STYLE` | 是否使用 path-style URL | MinIO 场景下应为 `true` |

### 2.4 认证

| 变量 | 当前用途 |
|---|---|
| `JWT_SECRET` | JWT 签发密钥 |
| `JWT_ACCESS_EXPIRES_IN` | access token 时长 |
| `JWT_REFRESH_EXPIRES_IN` | refresh token 时长 |

### 2.5 上传限制

| 变量 | 当前用途 |
|---|---|
| `MAX_CARD_SIZE_MB` | `.card` 上传限制 |
| `MAX_BOX_SIZE_MB` | `.box` 上传限制 |
| `MAX_AVATAR_SIZE_MB` | 头像上传限制 |

### 2.6 管理员引导

| 变量 | 当前用途 |
|---|---|
| `ADMIN_USERNAME` | 首次启动时自动创建管理员账号 |
| `ADMIN_PASSWORD` | 对应管理员初始密码 |

### 2.7 文档与调试

| 变量 | 当前用途 |
|---|---|
| `ENABLE_SWAGGER` | 是否开启 `/api/docs` |

### 2.8 Host 动态插件配置

| 变量 | 当前用途 |
|---|---|
| `HOST_CARD_PLUGIN_PATHS` | 覆盖默认基础卡片插件来源 |
| `HOST_THEME_PLUGIN_PATHS` | 覆盖默认主题插件来源 |
| `HOST_MODULE_PLUGIN_PATHS` | 覆盖默认转换模块来源 |
| `HOST_ACTIVE_THEME_ID` | Host 启动后应用的主题 ID |

## 3. 当前对外地址口径

### 3.1 默认 Docker Compose 访问入口

- 平台统一入口：`http://localhost`
- API：`http://localhost/api/...`
- Web 前台：`http://localhost/`
- Admin 后台：`http://localhost/admin/`
- CDN 代理：`http://localhost/cdn/...`
- MinIO Console：`http://localhost:9001`

### 3.2 对象 URL 生成规则

- 若设置了 `S3_PUBLIC_URL`：对象 URL 以该值为根前缀；
- 若未设置：对象 URL 统一由 `BASE_URL + /cdn/{bucket}/{key}` 生成；
- 当前项目默认推荐后者，以保证卡片资源、HTML 产物、头像和封面都走主站反向代理口径。

## 4. 运行依赖

### 4.1 必需服务

- PostgreSQL
- Redis
- MinIO 或其他 S3 兼容对象存储
- Nginx 反向代理

### 4.2 Node 运行时

- Node.js 20+

### 4.3 包管理

- `pnpm >= 9`
- 当前本地验证若系统 `pnpm` 版本不足，可使用 `npm_config_cache=/tmp/ccps-npm-cache npx pnpm@9 ...`

## 5. 当前关键依赖

- Fastify
- Drizzle ORM
- `@aws-sdk/client-s3`
- `ioredis`
- `bcryptjs`
- `js-yaml`
- `unzipper`
- `chips-host`
- `chips.module.chips.htmltoimage.plugin`

## 6. Host 依赖约束

- 服务器当前依赖生态内：
  - 基础卡片插件
  - 默认主题插件
  - 文件转换模块插件
- 这些插件由 `HostIntegration` 在独立工作区 `.chips-server-host/` 中动态安装与启用。
- Docker 镜像构建时必须把相关生态目录一并带入构建上下文，否则 `packages/server` 中的本地 `file:` 依赖与默认插件路径都无法成立。
