# 箱子解析与社区映射

## 1. 文档范围

本文只描述当前 `.box` 上传、解析、入库与社区卡片映射实现。

对应实现：

- `packages/server/src/services/box.service.ts`
- `packages/server/src/routes/upload.ts`
- `packages/server/src/routes/boxes.ts`
- `packages/server/src/types/box.ts`

## 2. 上传入口

箱子上传入口为：

- `POST /api/v1/upload/box`

上传约束：

- 仅接受 `.box`
- 受 `MAX_BOX_SIZE_MB` 限制
- 支持附带 `roomId` 与 `visibility`
- 若指定 `roomId`，必须先通过房间所有权校验

## 3. 解包与校验

### 3.1 ZIP 校验

服务端先检查 ZIP 魔数，拒绝非合法 ZIP 伪装文件。

### 3.2 解压安全

解压过程会：

- 解压到系统临时目录
- 拒绝路径穿越
- 仅把文件写入临时目录树内

### 3.3 必需目录与文件

当前要求 `.box` 包中至少存在：

- `.box/`
- `.box/metadata.yaml`

`structure.yaml` 允许缺省；缺省时会回退为：

```yaml
cards: []
```

## 4. 入库字段来源

### 4.1 `metadata.yaml`

用于填充：

- `boxFileId`
- `title`
- `layoutPlugin`
- `metadata`

### 4.2 `structure.yaml`

用于填充：

- `structure`

## 5. 数据库存储策略

箱子当前不会拆分多张关联子表，而是直接把解析结果保存为 JSON：

- `metadata`
- `structure`

这保证第一版实现足够简单，同时保留原始结构信息。

## 6. 详情查询增强

`GET /api/v1/boxes/:boxId` 在返回前会调用：

- `BoxService.enrichCardRefs(...)`

对箱子内的卡片引用做社区内匹配。

## 7. 社区映射规则

### 7.1 触发条件

当箱子结构项中存在 `card_id` 时，服务端会尝试查找社区卡片。

### 7.2 匹配条件

当前匹配规则为：

1. `cards.cardFileId === ref.card_id`
2. `cards.status === 'ready'`
3. `cards.visibility === 'public'`

### 7.3 匹配结果

命中后会回写到返回 DTO：

- `communityCardId`
- `communityHtmlUrl`

未命中则不补这两个字段。

## 8. 前台展示方式

当前 Web 箱子详情页不会运行布局插件，也不会渲染整个箱子页面。

前台只展示：

- 箱子基础信息
- 布局插件标识
- 箱子文件内 ID
- 箱内引用列表
- 若已命中社区卡片，则提供跳转社区卡片与 HTML 链接

## 9. 删除与清理

### 9.1 上传阶段清理

箱子创建完成后会在 `finally` 中删除：

- 解包临时目录
- 上传时写入的临时 `.box`

### 9.2 删除阶段

当前箱子删除只删除数据库记录。

由于第一版没有为箱子额外上传对象资源，所以没有额外对象存储清理动作。

## 10. 当前边界

当前第一版明确不包含：

- 箱子 HTML 导出
- 箱子布局运行时
- 箱子封面生成
- 箱内卡片引用回写社区关系表
- 箱子内卡片在线编辑
