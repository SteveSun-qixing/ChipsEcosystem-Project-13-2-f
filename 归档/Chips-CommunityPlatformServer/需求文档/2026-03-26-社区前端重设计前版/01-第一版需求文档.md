# 第一版需求文档

## 1. 产品定位

薯片社区平台服务器第一版是一个面向 `.card` 与 `.box` 内容的轻量社区发布平台。

第一版目标不是做完整社交网络，而是先把最核心的三件事做完整：

1. 用户注册登录并拥有个人空间
2. 上传卡片 / 箱子并获得稳定链接
3. 通过房间组织内容并提供基础管理后台

## 2. 第一版必须完成的能力

### 2.1 账户能力

- 用户注册
- 用户登录
- refresh cookie 恢复会话
- 用户退出登录
- 修改个人资料
- 上传头像
- 修改密码

### 2.2 空间与房间能力

- 用户空间页
- 房间创建
- 房间公开 / 私有
- 房间详情页
- 房间封面上传
- 房间删除
- 根目录内容展示

### 2.3 卡片能力

- 上传 `.card`
- 卡片异步处理状态
- 卡片资源 CDN 化
- 卡片内资源 URL 替换
- 重打包新的 `.card`
- 通过 Host 已安装的正式转换插件输出 HTML
- 卡片详情页直接跳转 HTML 页面
- 卡片移动到房间 / 根目录
- 卡片公开 / 私有
- 卡片删除

### 2.4 箱子能力

- 上传 `.box`
- 箱子结构解析
- 箱子详情页
- 箱子引用卡片社区映射
- 箱子移动到房间 / 根目录
- 箱子公开 / 私有
- 箱子删除

### 2.5 公共浏览能力

- 首页最新公开卡片 / 箱子
- 用户空间浏览
- 房间浏览
- 卡片访问
- 箱子访问
- 基础搜索

### 2.6 后台能力

- 管理员登录
- 平台统计
- 用户列表与启停
- 内容列表与删除

## 3. 用户角色

- 访客：浏览公开内容
- 注册用户：管理自己的空间与内容
- 管理员：治理平台用户与内容

## 4. 第一版核心原则

### 4.1 卡片查看原则

平台不再自己包一层卡片查看 UI。

用户查看卡片时，最终看到的是转换插件输出的 HTML 页面本身。

### 4.2 转换原则

平台不自己维护第二条 HTML 转换正式链路。

正式实现必须通过：

- Host
- 已安装卡片插件
- 已安装主题插件
- 已安装文件转换插件

共同完成最终转换。

### 4.3 权限原则

- 房间是内容组织单元
- 内容仍保留自己的公开 / 私有属性
- 公开房间不会放大私有内容可见范围
- 房间计数只反映当前请求者可见内容

## 5. 当前第一版交付物

- `packages/server` API 服务
- `packages/web` 前台站点
- `packages/admin` 后台站点
- `deploy` 一键部署目录
- 项目内部技术文档
- 对外共享 API 契约文档

## 6. 当前第一版明确不做的内容

- 评论、点赞、关注、私信
- 房间成员与协作权限
- 箱子在线布局运行时
- 社区内卡片回下载为原生 `.card`
- 实时通知系统
- 复杂推荐系统

## 7. 第一版验收底线

第一版交付必须至少满足：

1. 用户可注册并登录
2. 用户可创建公开 / 私有房间
3. 用户可上传 `.card`，并在处理完成后直接查看 HTML 页面
4. 用户可上传 `.box`，并浏览箱子详情与引用列表
5. 公开访问者只能看到公开内容与公开计数
6. 管理员可进入后台并治理用户与内容
7. 部署目录可完成一键安装所需的正式编排
