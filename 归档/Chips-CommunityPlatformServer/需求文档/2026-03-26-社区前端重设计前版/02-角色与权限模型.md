# 角色与权限模型

## 1. 角色定义

当前第一版社区服务器只有三种实际访问态：

1. 访客
2. 注册用户
3. 管理员

其中数据库持久化角色只有：

- `user`
- `admin`

“访客”是未登录请求态，不写入数据库。

## 2. 访客权限

访客可以：

- 查看公开用户资料
- 查看公开空间
- 查看公开房间
- 查看公开卡片
- 查看公开箱子
- 浏览发现页与搜索结果

访客不能：

- 上传卡片 / 箱子
- 创建房间
- 修改资料
- 查看任何私有资源
- 使用后台接口

## 3. 注册用户权限

注册用户在具备访客全部公开读取能力之外，还可以：

- 访问自己的私有空间
- 访问自己的私有房间
- 查看自己的私有卡片与私有箱子
- 创建、修改、删除自己的房间
- 上传、移动、删除自己的卡片与箱子
- 上传自己的头像
- 修改自己的资料与密码

注册用户不能：

- 访问他人的私有资源
- 启用后台治理能力
- 把内容挂到他人的房间中

## 4. 管理员权限

管理员在注册用户能力基础上，额外拥有：

- 访问后台统计接口
- 查看平台用户列表
- 查看平台内容列表
- 禁用 / 启用用户
- 强制删除任意卡片 / 箱子

当前第一版里，管理员能力边界仅在后台治理域生效。

当前没有单独定义：

- 管理员代登录
- 管理员进入私有前台空间页面
- 管理员细粒度 RBAC

## 5. 资源权限模型

### 5.1 用户资料

- 公开资料：所有人可见
- 当前用户资料、密码、头像：仅本人可管理

### 5.2 房间

房间是内容组织与第一层可见性边界。

房间可见性：

- `public`
- `private`

规则：

1. 所有者始终可见
2. 公开房间对所有请求者可见
3. 私有房间对非所有者不可见，并按 404 处理

### 5.3 卡片与箱子

卡片、箱子各自都有独立可见性：

- `public`
- `private`

规则：

1. 私有内容只对所有者可见
2. “已登录但不是所有者”不会放宽私有内容可见性
3. 房间公开不代表房间内私有内容自动公开

## 6. 房间内容列表规则

`GET /api/v1/rooms/:roomId/contents` 的正式口径是：

1. 先判断房间本身是否可访问
2. 再按“是否房间所有者”过滤房间内内容

因此：

- 房间所有者：可看到房间内全部卡片 / 箱子
- 非所有者：
  - 只能看到公开卡片 / 公开箱子
  - 其中卡片还必须已经完成转换并处于 `ready`

## 7. 房间计数规则

房间对外暴露的 `cardCount` / `boxCount` 必须反映“当前请求者可见内容数量”，而不是原始总量。

因此：

- 所有者看到完整计数
- 非所有者：
  - 卡片计数只统计 `public + ready`
  - 箱子计数只统计 `public`

公开空间页与公开房间页都遵守这一规则。

## 8. 管理动作矩阵

| 动作 | 访客 | 注册用户 | 管理员 |
|---|---|---|---|
| 注册 / 登录 | 是 | 是 | 是 |
| 查看公开空间 / 房间 / 卡片 / 箱子 | 是 | 是 | 是 |
| 查看自己的私有空间与内容 | 否 | 是 | 是 |
| 创建房间 | 否 | 是 | 是 |
| 上传卡片 | 否 | 是 | 是 |
| 上传箱子 | 否 | 是 | 是 |
| 修改自己的资料 / 头像 / 密码 | 否 | 是 | 是 |
| 删除自己的房间 / 卡片 / 箱子 | 否 | 是 | 是 |
| 访问后台统计 | 否 | 否 | 是 |
| 禁用用户 | 否 | 否 | 是 |
| 强制删除任意内容 | 否 | 否 | 是 |

## 9. 当前范围外

当前权限模型还没有：

- 房间成员
- 协作者
- 团队 / 组织
- 分享链接 ACL
- 标签级权限
- 草稿协同
- 评论、点赞、关注、举报

因此当前正式第一版仍是“单所有者 + 公开/私有二元权限 + 管理后台治理”。
