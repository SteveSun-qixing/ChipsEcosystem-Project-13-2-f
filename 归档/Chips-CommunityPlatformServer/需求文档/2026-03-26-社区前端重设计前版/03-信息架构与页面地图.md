# 信息架构与页面地图

## 1. 前台页面地图

### 1.1 公共页面

- `/`：首页
- `/login`：登录页
- `/register`：注册页
- `/cards/:cardId`：卡片详情中转页
- `/boxes/:boxId`：箱子详情页
- `/@:username`：用户空间页
- `/@:username/rooms/:roomSlug`：房间页

### 1.2 登录后页面

- `/dashboard`：用户工作台
- `/upload`：上传中心

## 2. 后台页面地图

当前后台对外入口只有一个：

- `/admin/`

它是单页应用，内部包含三个功能板块：

1. 平台总览
2. 用户管理
3. 内容治理

## 3. 首页信息结构

首页当前由三部分组成：

1. Hero 区
   - 品牌介绍
   - 基础价值说明
   - 主要 CTA
2. 搜索区
   - 统一搜索用户 / 卡片 / 箱子
3. 公开内容区
   - 最新公开卡片
   - 最新公开箱子

## 4. 用户空间信息结构

空间页当前包含：

1. 用户资料头部
2. 房间网格
3. 根目录箱子区
4. 根目录卡片区

若当前访问者为本人，还会展示：

- 进入工作台按钮
- 进入上传页按钮

## 5. 房间页信息结构

房间页当前包含：

1. 房间头部信息
   - 名称
   - 描述
   - 创建时间
   - 当前请求者可见的卡片 / 箱子计数
2. 卡片网格区
3. 箱子网格区
4. 作者空间跳转入口

## 6. 卡片访问结构

卡片访问当前分两段：

1. 先访问平台路由 `/cards/:cardId`
2. 平台读取卡片详情并在就绪后直接跳转 `htmlUrl`

因此卡片的最终阅读页不在平台 React 页面树中，而在转换插件输出的 HTML 页面中。

## 7. 箱子详情结构

箱子详情页当前包含：

1. 箱子头部摘要
2. 侧边元信息区
3. 引用卡片列表区

每个引用项可提供：

- 跳转社区卡片入口
- 跳转原始源链接入口

## 8. 工作台结构

工作台当前为一个长页面，按区块组织：

1. 工作台头部
2. 统计卡片
3. 个人资料编辑区
4. 新建房间区
5. 房间管理表格
6. 卡片管理表格
7. 箱子管理表格

## 9. 后台信息结构

后台当前结构为：

1. 顶部栏
   - 品牌
   - 语言
   - 主题
   - 退出登录
2. 登录态区
   - 未登录：管理员登录表单
   - 非管理员：禁止访问提示
   - 管理员：后台仪表板
3. 仪表板内容
   - 统计指标区
   - 用户管理区
   - 内容治理区

## 10. 当前导航原则

- 前台导航聚焦“浏览 + 上传 + 工作台”
- 后台导航聚焦“治理”
- 卡片最终内容页面不再纳入前台站点导航体系，而是直接进入 HTML 页面
