# 房间与空间产品规则

## 1. 个人空间定位

个人空间是用户在社区服务器中的公开入口页，用来承载：

- 用户公开资料
- 该用户创建的房间列表
- 该用户根目录下的卡片
- 该用户根目录下的箱子

当前空间页对应接口：

- `GET /api/v1/users/:username/space`

当前前端页面：

- `packages/web/src/pages/SpacePage.tsx`

## 2. 个人空间组成

### 2.1 用户信息区

展示：

- 头像
- 显示名
- 用户名
- 简介
- 注册时间

若当前访问者就是所有者，还会额外显示：

- 进入管理台入口
- 进入上传页入口

### 2.2 房间列表区

空间页会按网格列出用户房间。

每个房间卡片当前展示：

- 封面图（若有）
- 房间名称
- 卡片数 / 箱子数

### 2.3 根目录内容区

根目录表示 `roomId = null` 的内容。

空间页会分别展示：

- 根目录箱子
- 根目录卡片

## 3. 房间定义

房间是用户个人空间中的一级内容组织单元。

当前第一版中，房间只承担：

- 分组组织内容
- 管理可见性
- 提供独立链接
- 提供独立封面与描述

当前第一版不承担：

- 自定义布局容器
- 富交互页面编排
- 协作权限面板
- 多级层级结构

## 4. 房间字段

当前房间正式字段包括：

- `id`
- `userId`
- `name`
- `slug`
- `description`
- `coverUrl`
- `visibility`
- `cardCount`
- `boxCount`
- `createdAt`
- `updatedAt`

## 5. slug 规则

房间 slug 当前由房间名称自动生成，遵守：

1. 同一用户名下唯一
2. 若名称冲突，则自动追加序号
3. 重命名房间时，slug 会随名称变化重新生成

房间当前前台访问路径为：

- `@username/rooms/:roomSlug`

## 6. 房间可见性

房间可见性只有两种：

- `public`
- `private`

规则：

1. 公开房间可被所有人访问
2. 私有房间只对所有者可见
3. 非所有者访问私有房间时，系统按“资源不存在”处理

## 7. 房间内容规则

### 7.1 内容类型

房间内当前只允许两类内容：

- 卡片
- 箱子

### 7.2 房间内容列表

房间详情页展示方式当前非常简单：

- 卡片区
- 箱子区

不做复杂混排，不做二次 UI 壳，不做房间级布局插件运行时。

### 7.3 公开房间中的私有内容

即使房间本身是公开房间，房间内内容仍保留各自独立可见性。

正式规则：

- 房间所有者可以看到私有卡片 / 私有箱子
- 非所有者只能看到公开内容
- 已登录但不是所有者，不会额外获得私有内容可见性

## 8. 房间计数规则

房间 DTO 中的 `cardCount` / `boxCount` 不是绝对总量，而是“当前请求者可见计数”。

因此：

- 所有者看到完整数量
- 非所有者：
  - 卡片数只统计 `public + ready`
  - 箱子数只统计 `public`

这条规则同时适用于：

- 房间列表
- 房间详情
- 空间页内的房间卡片

## 9. 房间管理动作

当前第一版房间管理支持：

- 创建房间
- 修改房间名称
- 修改描述
- 修改公开 / 私有状态
- 上传封面
- 删除房间

删除房间时采用“业务级联删除”规则：

1. 先删除房间内卡片
2. 再删除房间内箱子
3. 最后删除房间本身

当前不保留软删除态。

## 10. 空间与房间当前范围外

当前第一版不包含：

- 空间首页自定义装修
- 房间排序配置
- 房间置顶
- 多级子房间
- 房间成员
- 协作编辑
- 房间邀请码
- 房间权限 ACL 细分
