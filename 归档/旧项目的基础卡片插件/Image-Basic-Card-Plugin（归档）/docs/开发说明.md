# 图片基础卡片插件 - 开发说明

## 架构说明

本插件属于薯片生态 monorepo 工作区的一部分，遵循薯片生态统一的开发规范和标准。

### 核心原则

1. **中心路由架构**：所有模块间通信通过薯片内核路由，不直接调用
2. **零硬编码文本**：所有界面文本通过 i18n 系统管理
3. **零硬编码样式**：所有样式通过 CSS 变量和主题系统注入
4. **配置驱动**：所有可变参数通过配置文件管理
5. **前后端分离**：前端只负责展示和交互，后端逻辑由内核处理

### 依赖关系

```
@chips/image-card-plugin
├── @chips/foundation (peer) - 公共基础层
├── @chips/sdk (peer) - 开发工具包
└── vue (dependency) - 前端框架
```

### 数据流

```
卡片文件(content/{ID}.yaml) → 渲染器读取配置 → 解析图片路径 → 挂载Vue组件 → 生成iframe
```

## 配置说明

### layout_type 自动判断

当图片列表只有 0 或 1 张图片时，无论配置中设置了什么 layout_type，渲染器都会自动使用 `single` 模式。这确保了单张图片时不会出现不必要的布局选项。

### 网格溢出机制

- `2x2` 模式最多显示 4 张图片
- `3x3` 模式最多显示 9 张图片，如果超出，最后一格显示 "+N"（N为剩余图片数量）
- `3-column-infinite` 模式无限制，显示所有图片
