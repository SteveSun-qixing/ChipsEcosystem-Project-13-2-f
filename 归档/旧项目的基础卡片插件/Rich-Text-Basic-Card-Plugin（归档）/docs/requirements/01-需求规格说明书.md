## 1. 文档概述

本文档定义富文本基础卡片插件的完整需求规格，作为技术设计和开发实现的依据。

### 1.2 范围

本文档覆盖：
- 功能需求
- 数据模型设计
- 接口规范
- 用户交互设计
- 非功能需求

### 1.3 术语定义

| 术语 | 定义 |
|-----|------|
| 基础卡片 | 薯片生态中最小的不可拆分内容单元 |
| 复合卡片 | 包含一个或多个基础卡片的可存储单元 |
| 渲染组件 | 负责在查看模式下显示卡片内容的前端组件 |
| 编辑组件 | 负责在编辑模式下编辑卡片内容的前端组件 |
| 富文本 | 包含格式化信息的文本，支持字体、颜色、大小等样式 |

## 2. 产品概述

### 2.1 产品定位

富文本基础卡片是薯片生态26种基础卡片类型之一，属于"文本类"卡片。用于显示和编辑带有格式的文本内容，是最常用的基础卡片类型之一。

### 2.2 目标用户

- 需要创建格式化文档的普通用户
- 需要在卡片中嵌入富文本内容的内容创作者
- 需要集成富文本功能的第三方开发者

### 2.3 核心价值

- **所见即所得**：用户在编辑时可以直接看到最终效果
- **轻量简单**：提供日常所需的基础格式化功能，不过度复杂
- **响应式设计**：在不同尺寸容器中自适应显示
- **主题支持**：完全支持薯片主题系统

## 3. 功能需求

### 3.1 渲染组件功能

#### 3.1.1 文本格式渲染

| 功能 | 说明 | 优先级 |
|-----|------|--------|
| 加粗 | 渲染`<strong>`或`<b>`标签的加粗文本 | P0 |
| 斜体 | 渲染`<em>`或`<i>`标签的斜体文本 | P0 |
| 下划线 | 渲染`<u>`标签的下划线文本 | P0 |
| 删除线 | 渲染`<s>`或`<del>`标签的删除线文本 | P1 |
| 上标 | 渲染`<sup>`标签的上标文本 | P2 |
| 下标 | 渲染`<sub>`标签的下标文本 | P2 |

#### 3.1.2 段落格式渲染

| 功能 | 说明 | 优先级 |
|-----|------|--------|
| 标题 | 渲染H1-H6六级标题 | P0 |
| 段落 | 渲染普通段落文本 | P0 |
| 有序列表 | 渲染`<ol><li>`有序列表 | P0 |
| 无序列表 | 渲染`<ul><li>`无序列表 | P0 |
| 引用块 | 渲染`<blockquote>`引用内容 | P1 |
| 水平线 | 渲染`<hr>`分隔线 | P1 |

#### 3.1.3 样式渲染

| 功能 | 说明 | 优先级 |
|-----|------|--------|
| 文字颜色 | 渲染文字前景色 | P0 |
| 背景色 | 渲染文字背景高亮色 | P1 |
| 字体大小 | 渲染不同字号的文本 | P0 |
| 对齐方式 | 渲染左对齐/居中/右对齐/两端对齐 | P0 |

#### 3.1.4 其他渲染

| 功能 | 说明 | 优先级 |
|-----|------|--------|
| 超链接 | 渲染可点击的超链接 | P0 |
| 图片 | 渲染内嵌图片 | P1 |
| 代码行 | 渲染行内代码`<code>` | P1 |

#### 3.1.5 响应式布局

- 内容宽度自适应容器宽度
- 图片自动缩放不超出容器
- 长链接自动换行
- 在移动端保持可读性

#### 3.2.2 编辑区功能

- **所见即所得编辑**：编辑区直接显示格式化效果
- **光标定位**：支持鼠标点击和键盘导航定位
- **文本选择**：支持鼠标拖选和Shift+方向键选择
- **复制粘贴**：支持粘贴纯文本和带格式文本
- **拖放**：支持拖入图片文件
- **自动保存**：内容变更后自动保存到配置

#### 3.2.3 辅助功能

- **占位符**：空内容时显示提示文字
- **字数统计**：显示当前字数
- **清除格式**：一键清除选中内容的所有格式

### 3.3 生命周期管理

#### 3.3.1 初始化流程

```
1. 接收配置数据（CardConfig）
2. 验证配置格式
3. 加载内容（从文件或内联）
4. 初始化内部状态
5. 准备渲染资源
```

#### 3.3.2 渲染流程

```
1. 接收容器元素（HTMLElement）
2. 解析HTML内容
3. 应用主题样式
4. 渲染到容器
5. 绑定交互事件
```

#### 3.3.3 更新流程

```
1. 接收配置变更
2. 差异对比
3. 增量更新DOM
4. 触发变更事件
```

#### 3.3.4 销毁流程

```
1. 解绑所有事件
2. 清理定时器
3. 释放DOM引用
4. 清空内部状态
```

## 4. 数据模型

### 4.1 配置文件结构

富文本卡片的配置保存在复合卡片的`content/{ID}.yaml`文件中：

```yaml
# RichTextCard配置文件结构
card_type: "RichTextCard"

# 主题设置（可选，为空使用上级主题）
theme: ""

# 布局参数（可选）
layout:
  height_mode: "auto"     # "auto" | "fixed"
  fixed_height: null      # 固定高度像素值

# 内容来源（必需）
content_source: "inline"  # "file" | "inline"

# 文件路径（content_source为file时使用）
content_file: null

# 内联内容（content_source为inline时使用）
content_text: "<p>Hello World</p>"

# 是否显示工具栏（编辑模式）
toolbar: false

# 是否只读
read_only: true
```

### 4.2 字段说明

| 字段 | 类型 | 必需 | 默认值 | 说明 |
|-----|------|------|--------|------|
| card_type | string | 是 | - | 固定值"RichTextCard" |
| theme | string | 否 | "" | 主题包标识 |
| layout.height_mode | string | 否 | "auto" | 高度模式 |
| layout.fixed_height | number | 否 | null | 固定高度值 |
| content_source | string | 是 | - | "file"或"inline" |
| content_file | string | 条件 | null | 富文本文件路径 |
| content_text | string | 条件 | "" | 内联HTML内容 |
| toolbar | boolean | 否 | false | 是否显示工具栏 |
| read_only | boolean | 否 | true | 是否只读 |

### 4.3 内容格式规范

#### 4.3.1 支持的HTML标签

**文本格式**：
- `<strong>`, `<b>` - 加粗
- `<em>`, `<i>` - 斜体
- `<u>` - 下划线
- `<s>`, `<del>` - 删除线
- `<sup>` - 上标
- `<sub>` - 下标
- `<code>` - 行内代码

**段落结构**：
- `<p>` - 段落
- `<h1>`-`<h6>` - 标题
- `<ul>`, `<ol>`, `<li>` - 列表
- `<blockquote>` - 引用
- `<hr>` - 分隔线
- `<br>` - 换行

**链接和媒体**：
- `<a href="">` - 超链接
- `<img src="">` - 图片

**样式属性**：
- `style="color: #xxx"` - 文字颜色
- `style="background-color: #xxx"` - 背景色
- `style="font-size: xxpx"` - 字号
- `style="text-align: xxx"` - 对齐

#### 4.3.2 不支持的标签

为保持轻量，以下标签不支持：
- `<table>` - 表格（使用专门的表格卡片）
- `<video>`, `<audio>` - 媒体（使用专门的媒体卡片）
- `<script>`, `<style>` - 脚本和样式块
- `<iframe>` - 内嵌框架
- `<form>` - 表单元素

#### 4.3.3 安全处理

- 所有HTML内容必须经过XSS过滤
- 移除危险属性（onclick, onerror等）
- 链接href只允许http/https/mailto协议
- 图片src只允许相对路径或http/https

interface RichTextEditor {

## 6. 用户交互设计

### 6.1 渲染模式交互

#### 6.1.1 链接点击
- 点击链接在新标签页打开
- 显示链接预览提示

#### 6.1.2 图片交互
- 点击图片可放大查看（调用系统图片查看器）
- 右键菜单：复制图片、保存图片

#### 6.1.3 文本选择
- 允许选择和复制文本
- 右键菜单：复制、搜索

### 6.2 编辑模式交互

#### 6.2.1 工具栏
- 工具栏固定在编辑区顶部
- 工具按钮显示当前选中文本的格式状态
- 点击工具按钮切换格式
- 悬浮显示工具提示和快捷键

#### 6.2.2 编辑区
- 点击编辑区获得焦点
- 双击选中单词
- 三击选中段落
- 回车创建新段落
- Shift+回车插入换行

#### 6.2.3 链接编辑
- 选中文本后点击链接工具弹出编辑框
- 编辑框包含URL输入和文本输入
- 支持取消链接

#### 6.2.4 图片插入
- 点击图片工具弹出对话框
- 支持从本地上传或输入URL
- 上传后图片保存到卡片内部

## 9. 非功能需求

### 9.1 性能要求

| 指标 | 要求 |
|-----|------|
| 初始化时间 | < 100ms |
| 渲染时间（1000字） | < 50ms |
| 输入延迟 | < 16ms (60fps) |
| 内存占用 | < 50MB |

### 9.2 兼容性要求

| 环境 | 要求 |
|-----|------|
| 浏览器 | Chrome 90+, Firefox 88+, Safari 14+, Edge 90+ |
| Electron | 22+ |
| Node.js | 18+ |

### 9.3 安全要求

- HTML内容必须经过XSS过滤
- 不执行任何脚本代码
- 链接只允许安全协议
- 图片上传需要类型和大小限制

### 9.4 可访问性要求

- 支持键盘导航
- 提供ARIA属性
- 工具栏按钮有明确标签
- 颜色对比度符合WCAG 2.1 AA标准

### 9.5 测试要求

- 单元测试覆盖率 ≥ 80%
- 所有公开API有测试用例
- 关键用户流程有集成测试

## 10. 约束和假设

### 10.1 约束

1. 必须遵循薯片生态的中心路由架构
2. 所有通信必须通过薯片内核
3. 不能硬编码任何文本和样式
4. 必须支持主题系统

### 10.2 假设

## 附录A：用户故事

### A.1 查看富文本内容

**作为**普通用户  
**我想要**查看格式化的富文本内容  
**以便**阅读带有标题、列表、链接的文档

**验收标准**：
- 正确渲染所有支持的HTML格式
- 链接可点击跳转
- 图片正确显示
- 响应式适配不同宽度

### A.2 编辑富文本内容

**作为**内容创作者  
**我想要**使用工具栏编辑富文本  
**以便**快速创建格式化的文档

**验收标准**：
- 工具栏显示所有格式化选项
- 点击按钮可以应用/取消格式
- 快捷键正常工作
- 编辑实时显示效果

### A.3 插入图片

**作为**用户  
**我想要**在富文本中插入图片  
**以便**丰富文档内容

**验收标准**：
- 可以通过工具栏打开图片对话框
- 支持上传本地图片
- 支持输入图片URL
- 图片正确显示在文档中
