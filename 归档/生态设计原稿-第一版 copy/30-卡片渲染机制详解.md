# 卡片渲染机制详解

## 渲染流程概述

薯片生态的所有卡片都使用网页技术渲染。卡片文件保存内容数据，基础卡片插件保存前端代码和布局逻辑，渲染器负责连接两者并生成完整的网页。

## 基础卡片的渲染

每个基础卡片插件包含完整的前端代码。以视频卡片为例，插件中包含视频播放器的 HTML 结构、CSS 样式（通过主题包注入）、JavaScript 交互逻辑。这些前端代码定义了视频卡片如何显示、如何响应用户操作。

卡片文件的 content/ 目录中保存基础卡片的内容配置。视频卡片的配置文件记录视频路径、封面路径、字幕列表、播放参数等数据。

渲染器的作用是连接前端代码和内容数据。渲染器读取基础卡片的配置文件，提取内容数据，调用对应插件的前端代码，将内容数据填充到前端模板中，生成完整的 HTML 网页。

每个基础卡片渲染为一个独立的 iframe 窗口。iframe 内部是渲染器生成的完整网页，包含该卡片的所有内容和交互逻辑。iframe 的背景设置为透明，与整体卡片窗口融为一体。

## 复合卡片的渲染

复合卡片包含多个基础卡片，渲染时需要将多个 iframe 组合在一起。

渲染器读取 structure.yaml，获取基础卡片列表和顺序。按顺序渲染每个基础卡片，每个生成一个 iframe 窗口。将所有 iframe 窗口垂直排列，组合成完整的卡片窗口。

复合卡片本身也是一个大的网页窗口。窗口顶部是菜单栏，显示卡片名称和操作按钮。窗口主体是一列 iframe，每个 iframe 显示一个基础卡片。如果用户设置了卡片级别的主题包，主题包作为整个大窗口的背景网页。

最终交付的是一个完整的 iframe 窗口，包含菜单栏、所有基础卡片的 iframe、主题背景。这个大 iframe 可以嵌入到任何产品中，比如嵌入到网页、嵌入到其他桌面应用、嵌入到编辑引擎的桌面上。

## 箱子的渲染

箱子的每种布局插件包含完整的前端代码。网格布局插件包含网格排列的 HTML、CSS、JavaScript，瀑布流布局插件包含瀑布流算法的完整实现。

箱子文件的 .box/content.yaml 保存布局配置数据。配置数据包括列数、间距、排序规则等参数。

渲染器读取箱子的配置，提取布局类型和配置数据，加载对应的布局插件，调用插件的前端代码，将配置数据和卡片列表传入，插件生成完整的展示网页。

箱子渲染的结果也是一个 iframe 窗口，内部是布局插件生成的网页，按照布局规则展示所有卡片。用户点击某个卡片时，调用卡片渲染器渲染该卡片。

## 主题的应用

主题包本质是样式代码的集合，包含所有组件类型的样式定义。主题包以网页技术实现，包含 CSS 样式文件、可选的 JavaScript 交互增强、可选的装饰性图片资源。

基础卡片插件的前端代码不包含样式，只定义结构和交互逻辑。样式通过 CSS 类名和变量暴露接口点，主题包通过这些接口点注入样式。

渲染器应用主题时，读取组件的主题配置（如果为空则使用上级主题），加载对应的主题包，提取该组件类型的样式代码，注入到组件的前端代码中，生成包含样式的完整网页。

主题配置采用层级覆盖：组件主题 > 卡片主题 > 应用主题 > 全局主题。渲染器从最高层级开始查找，找到第一个非空的主题配置就使用，如果所有层级都为空，使用系统默认主题。

## 渲染器在基础层的作用

公共基础层的 CardRenderer 模块包含完整的渲染逻辑。这个模块可以被任何产品调用，不只是 Chips-Viewer。

Chips-Editor 渲染卡片时调用 CardRenderer。Chips-Viewer 渲染卡片时调用 CardRenderer。第三方开发者通过 SDK 也调用 CardRenderer。所有地方使用相同的渲染引擎，确保卡片显示的一致性。

渲染器通过薯片内核的路由系统调用其他模块。需要解析 ZIP 文件时，通过路由调用 ZIPProcessor 模块。需要加载图片时，通过路由调用 ImageViewer 模块。需要播放视频时，通过路由调用 VideoPlayer 模块。渲染器本身不直接实现这些功能，而是组织和调度。

渲染器的输出是标准化的。无论是渲染基础卡片、复合卡片还是箱子，最终都输出一个 iframe 窗口或 DOM 元素。调用方获得这个输出后，可以嵌入到自己的界面中，不需要关心渲染的内部细节。

## 插件的前端代码

基础卡片插件是完整的插件包，包含渲染组件和编辑组件两部分。

渲染组件是插件的前端代码，定义了这种卡片如何显示。视频卡片插件的渲染组件包含视频播放器的界面结构、控制条的布局、字幕的显示方式、播放逻辑等完整实现。代码使用 HTML、CSS（通过主题注入）、JavaScript 编写。

渲染组件暴露数据接口。渲染器调用插件时，传入内容数据（从卡片文件的配置中读取），插件的前端代码接收数据，填充到界面中，生成完整网页。比如视频路径、封面、字幕等数据传入视频卡片插件，插件生成包含这些内容的视频播放器网页。

编辑组件是插件的编辑界面，在编辑引擎中使用。编辑组件提供界面让用户设置卡片的参数，修改后保存到配置文件。渲染组件读取这些配置进行显示。

插件的前端代码可以调用公共基础层的模块。视频卡片插件可以调用 VideoPlayer 模块获得底层的视频播放能力，在此基础上实现视频卡片特定的功能。插件不需要从零实现视频解码和播放，只需要组织界面和交互逻辑。

箱子布局插件也是同样的机制。布局插件包含完整的前端代码，定义了这种布局如何排列和展示卡片。网格布局插件包含网格排列的 HTML 结构、响应式布局的 CSS、拖拽排序的 JavaScript。渲染器调用插件时，传入卡片列表和布局配置，插件生成展示网页。
