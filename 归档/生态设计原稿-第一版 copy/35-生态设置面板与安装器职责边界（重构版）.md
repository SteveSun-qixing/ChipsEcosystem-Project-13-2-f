# 生态设置面板与安装器职责边界（重构版）

薯片生态是"单一 Host 底座 + 多插件应用"的生态系统。底层常驻静默运行，上层插件为用户提供可见功能。安装器与设置面板都服务于底层，但职责必须分离：

- 安装器负责“把生态装上去并可启动”
- 设置面板负责“装好之后的日常管理与调优”

如果边界不清，会产生重复实现、责任冲突和稳定性风险。

## 一、安装器的职责

安装器是独立程序，在薯片主机可能尚未安装时也能运行。

它负责：

1. 环境检查（依赖版本、空间、端口）
2. 资源解压与文件落位
3. 系统集成（快捷方式、注册项、桌面入口）
4. 安装失败回滚
5. 安装后启动引导与首轮健康校验

它不负责长期设置管理界面。

## 二、设置面板的职责

设置面板是薯片主机内部的应用插件，是"后安装管理面板"。

它负责：

1. 生态进程管理（状态、健康、日志、报告）
2. 语言管理（切换、词典更新）
3. 主题管理（切换、安装、删除）
4. 插件管理（安装、启停、卸载、更新）
5. 基础卡片与箱子套装管理（更新、修复）
6. 文件系统管理（工作区、交换策略）

它不负责首次安装、安装事务回滚、安装包下载编排。

## 三、统一能力来源

设置面板与 CLI 必须使用同一底层控制面：

- GUI 入口：`window.chips.invoke(...)`
- CLI 入口：`chips ...` 命令
- 执行核心：`kernel.router.invoke`

任何 GUI 能做的设置操作，都必须可被 CLI 等价执行。

## 四、信息架构建议

设置面板首版采用“左侧菜单 + 右侧设置区”结构：

1. 进程管理
2. 语言管理
3. 主题管理
4. 插件管理
5. 基础卡片与箱子管理
6. 文件系统管理

未来新增设置域采用注册式扩展，不改主框架。

## 五、设计原则

1. 薄 UI：界面只负责展示与交互，不承载底层业务。
2. 厚控制面：核心能力统一沉到底层服务。
3. 可观测：关键操作可追踪、可审计、可导出。
4. 最小权限：每条路由有明确权限边界。

## 六、生效声明

本设计稿用于修正原"设置面板包含安装器职责"的旧叙述。

自本稿发布后，涉及安装流程的规范以安装器文档为准；涉及后安装管理的规范以设置面板文档为准。
