# 薯片生态设计文档索引

## 文档分类原则

1. **绝对原子化**：一个概念只存在于一个文档，禁止复制粘贴
2. **零冗余**：禁止解释设计原因、商业价值、优劣对比
3. **动态可扩展**：所有数量均为"无限可添加"
4. **架构纯度**：严格遵循薯片生态核心架构原则

---

## 核心概念与架构

### 01 - 生态概述
整个薯片生态的核心理念、核心组件（卡片、箱子、社区）、软件产品（编辑引擎、查看器）、插件系统、底层架构。

### 02 - 架构原则
**核心架构原则**：
- 中心路由原则：所有服务间通信必须经过内核路由器
- 桥接隔离原则：插件只能通过 Bridge API 访问系统能力
- 底层厚上层薄：底层尽可能包含所有能力
- 平台无关性：内核和服务层不直接调用平台特定 API
- 一切皆插件：每个插件独立仓库，统一 .cpk 打包格式
- 无头组件库：组件只有逻辑和结构，零样式

### 03 - 层次结构
**内容层次**：基础卡片 → 复合卡片 → 箱子 → 房间 → 空间 → 社区（无限可添加层次）

**软件层次**：Electron 宿主 → PAL → 内核 → 公共基础层 → Bridge API → 插件应用

### 04 - 词条与概念
统一术语使用规范，核心词汇定义。
