# 插件系统架构

## 一切皆插件

薯片生态采用"一切皆插件"的设计理念。每个功能都是独立模块，层层嵌套组合。

基础卡片是插件、箱子布局是插件、编辑引擎布局是插件、查看工具是插件、主题包是插件，甚至文件管理器和程序坞也是插件。

---

## 插件类型

插件分为五种类型：

### 应用插件
完整的应用，如编辑引擎、查看器、资源管理器，拥有独立 BrowserWindow 窗口。

### 卡片插件
定义基础卡片类型的渲染和编辑组件。每种基础卡片都是一个独立的卡片插件，包含渲染组件和编辑组件。

### 布局插件
定义箱子的展示方式，如网格布局、瀑布流布局、列表布局等。布局插件决定箱子中卡片的排列方式和交互行为。

### 模块插件
共享功能模块，如视频播放器、音频播放器、图片查看器、Markdown 渲染器等。这些模块可被多个其他插件加载使用，类似操作系统的共享库。

### 主题插件
定义视觉样式的 CSS 样式包。主题插件包含所有组件类型的样式定义，通过 CSS 变量系统控制视觉表现。

---

## 插件打包格式

所有插件使用统一的 `.cpk` 格式打包，本质是 ZIP 压缩包。

每个插件包包含：
- `manifest.yaml`：插件清单，定义插件 ID、类型、版本、依赖等元数据
- `package.json`：npm 包配置
- 插件代码和资源文件

---

## 插件清单

每个插件的 `manifest.yaml` 必须包含：

```yaml
id: string          # 唯一标识符
name: string        # 显示名称
type: string        # 插件类型 (app|card|layout|module|theme)
version: string     # 语义化版本号
description: string # 描述
author: string      # 作者
dependencies:       # 依赖列表
  - pluginId: versionRange
entry: string       # 入口文件路径
```

## 热插拔机制

插件支持热插拔，可以不重启软件即可安装、更新和卸载。

### 安装流程
1. 用户安装插件（从市场下载或直接导入）
2. 系统验证插件清单和签名
3. 解压插件到插件目录
4. 更新插件注册表
5. 插件立即可用

### 更新流程
1. 用户更新插件
2. 系统备份旧版本
3. 替换新版本文件
4. 通知依赖该插件的其他插件
5. 如有必要，提示用户重启相关功能

### 卸载流程
1. 用户卸载插件
2. 检查是否有其他插件依赖该插件
3. 如有依赖，提示用户并阻止卸载或级联卸载
4. 移除插件文件
5. 更新插件注册表

---

## 插件隔离

每个插件运行在独立的渲染进程中，一个插件崩溃不会影响其他插件和主进程。

插件之间完全解耦，通过内核路由通信。插件可以声明依赖其他插件，内核在加载时会自动处理依赖关系。

---

## 插件注册表

内核维护一个插件注册表，记录所有已安装的插件信息。

注册表包含：
- 插件 ID 和版本
- 插件类型
- 入口文件路径
- 依赖关系
- 启用状态
- 安装位置

---

## 模块化架构

每个功能都是独立模块，层层嵌套组合：

- **原子模块**：不可再分的最小功能单元（按钮组件、输入框组件等）
- **功能模块**：由多个原子模块组合而成（图片卡片编辑组件等）
- **插件**：完整的业务功能单元（基础卡片插件等）
- **软件产品**：多个插件组合而成（编辑引擎等）

模块之间通过内核路由通信，完全解耦。
