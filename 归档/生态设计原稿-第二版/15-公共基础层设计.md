# 公共基础层设计

## 架构定位

公共基础层位于薯片内核之上、插件桥接层之下，是整个生态的基础设施。它承载生态中所有插件和应用共用的通用能力，体现"底层厚、上层薄"的设计理念。

公共基础层分为两部分：

**A. 核心服务（运行在主进程中，数据/系统型）**：
- 文件服务：文件读写、压缩解压、格式转换
- 卡片服务：.card 文件的解析和操作
- 箱子服务：.box 文件的解析和操作
- 资源管理：统一资源访问层
- ZIP 处理：零压缩率 ZIP 打包解包
- 配置管理：层级化配置系统
- 主题管理：主题包的注册和分发
- 多语言：国际化支持
- 日志系统：统一的日志记录
- 标签数据库：标签映射管理
- 凭证管理：加密存储认证信息
- 数据序列化：YAML、JSON 解析
- 平台服务：PAL 能力封装
- 模块加载：共享功能模块管理

**B. 共享功能模块（作为"模块插件"独立安装，UI 型）**：
- 视频播放器模块
- 音频播放器模块
- 图片查看模块
- 3D 渲染器模块
- 富文本编辑器模块
- Markdown 渲染器模块
- 代码高亮器模块
- 文本编辑器模块

薯片内核只提供最核心的路由、注册、事件总线、生命周期管理等能力。公共基础层承载所有通用功能，让生态内的插件和软件保持轻量化，避免重复造轮子，提高代码模块复用率。

---

## 层级关系

从底层到顶层的完整架构是六层：Electron 宿主 → 平台抽象层（PAL）→ 薯片内核 → 基础服务层 + 共享功能模块 → 插件桥接层（Bridge API）→ 插件应用。

---

## 核心服务

### 卡片渲染服务
卡片服务负责解析和操作 .card 文件，包括：
- 打开（解包）
- 保存（打包）
- 读取元数据和结构
- 管理基础卡片配置

### 文件处理服务
- ZIP 处理器：创建和解析卡片和箱子文件（零压缩率 ZIP）
- 文件识别器：根据文件扩展名和内容判断文件类型

### 资源管理服务
提供统一的资源访问层，屏蔽本地文件、网络资源、缓存的差异。上层插件通过 Bridge API 请求资源，资源管理服务自动处理获取、缓存和分发。

### 网络能力
- HTTP 请求
- 下载管理
- 网络协议适配器（WebDAV 等）
- 缓存管理器

### 配置管理
层级化的配置系统，支持多级配置覆盖（系统默认 → 用户全局 → 应用级 → 插件级），配置变化时自动通知相关模块。

### 主题管理
- 主题包注册
- 主题查询
- 六级主题继承解析

### 数据序列化
YAML、JSON 等格式的解析和生成。所有配置文件的读写都通过这个服务。

### 日志系统
统一的日志记录能力，所有服务和插件的日志都通过这个系统输出。

### 多语言系统
语言包的加载、文本翻译、语言切换等功能。

## 共享功能模块

共享功能模块作为独立的模块插件安装，各自是独立的代码仓库，可被多个上层插件共享使用。

### 通用媒体模块
- 视频播放器模块：视频解码和播放
- 音频播放器模块：音频播放和波形显示
- 图片查看模块：图片显示、缩放、旋转

### 文本处理模块
- 富文本编辑器模块
- Markdown 渲染器模块
- 代码高亮器模块
- 文本编辑器模块

这些共享功能模块不创建自己的窗口，由上层插件在自己的渲染进程中按需加载使用。

---

## 无头组件库

界面组件库基于 Zag.js 和 Ark UI 构建，提供通用的界面元素。按钮、输入框、下拉菜单、滑块、弹窗等基础控件，这些控件只实现功能逻辑、结构和布局，**完全不包含任何视觉样式**。所有样式由主题包通过 CSS 变量和类名注入。

---

## 模块组织原则

### 高度解耦
每个核心服务是独立的功能单元，通过内核路由器对外提供服务。服务之间不能直接依赖，必须通过薯片内核的路由系统通信。

### 接口一致性
所有服务使用统一的接口规范：
- 参数传递方式统一
- 返回值格式统一
- 错误处理方式统一

上层插件通过 Bridge API 以一致的方式调用不同服务：`window.chips.invoke(namespace, action, params)`。

### 职责单一
模块的职责单一明确。视频播放器只负责视频播放，不包含视频编辑功能。图片查看器只负责图片显示，不包含图片编辑功能。

### 按需加载
模块支持按需加载，不是所有服务在启动时都加载，而是当某个功能需要时才加载对应的服务或模块插件。

## 协议设计

### 模块调用协议
上层插件通过 Bridge API 发送请求到主进程，主进程的内核路由器将请求路由到对应的服务处理器。

### 资源访问协议
本地文件、网络资源、嵌入资源等都通过统一的资源标识符访问，资源管理服务自动处理资源的获取和缓存。

### 事件通知协议
服务通过内核事件总线发布事件，插件通过 Bridge API 的事件订阅接口（`window.chips.on(eventName, callback)`）接收事件通知。

---

## 轻量化原则

公共基础层保持轻量化，只包含必需的功能：
- 不包含任何业务逻辑
- 不包含具体的卡片实现
- 不包含具体的布局实现
- 不包含身份认证、用户管理、权限系统等功能
- 不包含数据同步、云端备份等功能

只保留所有插件和软件都需要的通用能力。

---

## 开源优先原则

公共基础层的开发遵循开源优先原则。在设计每个功能模块时，优先评估是否存在成熟的开源方案。

凡是有开源方案的，采用提取和拆分开源项目模块的方式集成，仅对生态高度定制的部分进行全新设计。

界面组件库采用 Zag.js（MIT 协议）作为组件状态机引擎，Ark UI（MIT 协议）作为 Vue 3 框架绑定层。
