# SDK 与协议

## SDK 能力封装

开发工具包（SDK）封装绝大部分底层逻辑和通用能力。第三方开发者只需要编写极少的代码，主要是简单的界面调用，即可实现复杂功能。

SDK 提供的封装包括：
- 文件操作封装
- 渲染功能封装
- 媒体播放封装
- 界面组件封装
- 数据处理封装
- 网络请求封装

比如开发者想创建一个视频播放器软件，不需要自己实现视频解码、播放控制、字幕渲染等功能。通过 SDK 调用公共基础层的 VideoPlayer 模块，几行代码就能实现完整的视频播放器。开发者只需要设计界面和用户体验部分。

SDK 基于 Bridge API 构建，提供类型安全的开发接口。

## 薯片协议

生态定义了一套薯片协议，规范各方面的标准：

### 文件格式协议

- .card 文件结构
- .box 文件结构
- metadata.yaml、structure.yaml 等配置文件的格式
- 资源引用的路径规范

### 接口标准协议

- 前端接口标准：定义前端如何调用后端功能
- 后端接口标准：定义后端模块提供哪些功能
- 模块调用协议
- 资源访问协议
- 事件通知协议

### 通信协议

- 请求响应格式
- 路由调用标准
- 错误处理规范

所有协议文档保存在"生态共用"文件夹中，所有项目都遵循这些协议，确保整个生态的一致性和互操作性。
