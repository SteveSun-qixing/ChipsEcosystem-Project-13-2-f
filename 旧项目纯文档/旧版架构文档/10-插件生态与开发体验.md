## 一、独立仓库策略

### 1.1 核心原则

> **每个插件、每个模块都是独立的代码仓库。完全解耦，完全独立。**

这意味着：
- 基础卡片插件（26 种）→ 26 个独立仓库
- 布局插件（8+ 种）→ 8+ 个独立仓库
- 应用插件（编辑器、查看器等）→ 每个一个独立仓库
- 主题包 → 每个一个独立仓库
- 公共基础层中的可替换模块（图片查看器、视频播放器等）→ 每个一个独立仓库
- 文件转换插件 → 每个一个独立仓库
- 工具插件 → 每个一个独立仓库

### 1.3 版本管理

每个仓库独立版本号，遵循语义化版本。插件在 manifest.yaml 中声明对主机和 SDK 版本的兼容范围。
页面型插件（`app/card/layout/module`）还必须声明 `ui.layout` 契约字段；`theme` 插件豁免 `ui.layout`：

```yaml
# manifest.yaml
compatibility:
  host: ">=1.0.0 <2.0.0"
  sdk: ">=1.0.0"
```

## 二、统一插件包格式（.cpk）

### 2.1 Chips Plugin Package

所有插件使用统一的打包格式——`.cpk`（Chips Plugin Package）：

```
my-plugin.cpk （本质是 ZIP 文件）
├── manifest.yaml                   # 插件清单（必需）
├── index.html                      # 应用入口（应用插件必需）
├── renderer/                       # 渲染组件（卡片/布局插件）
│   └── index.html
├── editor/                         # 编辑组件（卡片/布局插件）
│   └── index.html
├── assets/                         # 静态资源
│   ├── js/
│   │   ├── main.[hash].js
│   │   └── vendor.[hash].js
│   ├── css/
│   │   └── style.[hash].css
│   └── images/
├── tokens/                         # 主题 Token（主题插件）
├── components/                     # 组件样式（主题插件）
├── icons/                          # 图标（主题插件）
├── locales/                        # 多语言文件
│   ├── zh-CN.yaml
│   └── en-US.yaml
├── README.md                       # 插件说明
├── LICENSE                         # 许可证
├── CHANGELOG.md                    # 更新日志
└── preview.png                     # 预览图（应用市场展示）
```

### 2.2 打包命令

```bash
# 使用 Chips Develop CLI 打包
chipsd pack

# 输出：my-plugin-1.0.0.cpk
```

### 2.3 安装方式

```bash
# 命令行安装
chips install ./my-plugin-1.0.0.cpk    # 注意：install 属于用户 CLI（chips），不是开发 CLI

# 从薯片应用市场安装
chips install chips-official.video-card  # 同上，属于用户 CLI

# 拖拽 .cpk 文件到薯片主机窗口
# → 自动安装

# 在设置面板中浏览和安装
```

## 三、端到端开发工具链

### 3.1 Chips Develop CLI（开发命令行工具）

> **注意**：Chips Develop CLI（`chipsd`）专用于插件和应用开发。薯片生态还有一个面向最终用户的 Chips CLI（`chips`，未来开发），用于日常操作如创建卡片、创建箱子、使用应用等。两者分开，本文仅涉及开发 CLI。

```bash
# 安装开发 CLI
npm install -g @chips/develop-cli

# 创建新插件项目
chipsd create my-plugin
  ? 插件类型：(应用 / 卡片 / 布局 / 主题)
  ? 前端框架：(React / React / Svelte / 原生 JS)
  ? 插件名称：
  ? 发行商名称：
  ? 描述：

# 开发服务器
chipsd dev
# → 启动 Vite 开发服务器
# → 自动注入 Mock Bridge API（浏览器中开发）
# → 支持热模块替换

# 在薯片主机中实机调试
chipsd dev --host
# → 通知正在运行的薯片主机加载开发中的插件
# → 实时看到在真实环境中的效果

# 构建
chipsd build
# → Vite 构建，输出到 dist/

# 打包为 .cpk
chipsd pack
# → 将 dist/ + manifest.yaml 打包为 .cpk

# 发布到薯片应用市场
chipsd publish

# 验证插件
chipsd validate
# → 检查 manifest.yaml、文件结构、依赖兼容性

# CLI 仓库门禁执行口径：在 chips-develop-cli 目录内执行
(cd /Users/sevenstars/Documents/ChipsCard/Develop/Project-12/chips-develop-cli && pnpm test)
(cd /Users/sevenstars/Documents/ChipsCard/Develop/Project-12/chips-develop-cli && pnpm build)
```

### 3.3 Mock Bridge API

开发时不需要运行薯片主机，CLI 提供 Mock Bridge API 让插件可以在浏览器中独立开发：

```typescript
// 开发模式下自动注入
// chips dev 会在页面中注入 window.chips 的 Mock 实现

window.chips = {
  invoke: async (ns, action, params) => {
    // 根据 namespace.action 返回 Mock 数据
    if (ns === 'card' && action === 'getMetadata') {
      return mockCardMetadata;
    }
    // ...
  },
  on: (event, callback) => {
    // Mock 事件系统
    return () => {};
  },
  // ...
};
```

开发者可以在 `chips.mock.config.yaml` 中配置 Mock 数据：

```yaml
mocks:
  card.getMetadata:
    response:
      id: "a1B2c3D4e5"
      name: "测试卡片"
      created: "2026-01-01T00:00:00Z"

  card.getStructure:
    response:
      structure:
        - { id: "xY9Zw8Vt7U", type: "VideoCard" }
        - { id: "pQ3rS4tU5v", type: "RichTextCard" }

  theme.getCurrent:
    response:
      themeId: "chips-official.default"
```

## 四、填表式开发（低代码能力）

### 4.1 卡片插件声明式开发

对于简单的卡片插件，开发者不需要写代码，只需要填写声明文件：

```yaml
# card-declaration.yaml
# 声明式卡片定义——零代码开发

card:
  type: "BookmarkCard"
  name: "书签卡片"
  description: "保存网页书签和笔记"
  icon: "bookmark"

# 数据模型（自动生成配置文件结构）
fields:
  - name: url
    type: string
    label: "网址"
    required: true
    validation: url
    placeholder: "https://example.com"

  - name: title
    type: string
    label: "标题"
    maxLength: 200

  - name: description
    type: text
    label: "描述"
    maxLength: 1000

  - name: favicon
    type: image
    label: "图标"
    source: auto    # 自动从 URL 获取

  - name: tags
    type: tags
    label: "标签"

  - name: rating
    type: number
    label: "评分"
    min: 0
    max: 5
    step: 0.5

# 渲染布局（自动生成渲染组件）
renderer:
  layout: vertical
  sections:
    - type: image
      field: favicon
      size: 32x32
      position: inline-start
    - type: heading
      field: title
      level: 3
    - type: link
      field: url
      display: truncate
    - type: paragraph
      field: description
    - type: tags
      field: tags
    - type: rating
      field: rating
      style: stars

# 编辑器布局（自动生成编辑组件）
editor:
  layout: form
  fields: auto    # 自动根据 fields 生成表单
```

Chips Develop CLI 根据这个声明文件自动生成：
1. 配置文件的数据模型（TypeScript 类型 + Zod 验证）
2. 渲染组件（HTML + CSS 接口点）
3. 编辑组件（表单界面）
4. manifest.yaml
5. 完整的 .cpk 包

### 4.2 应用插件组装器

对于简单的应用插件（如"只是用卡片文件保存数据的独立工具"），提供可视化组装能力：

```yaml
# app-assembly.yaml
app:
  name: "我的待办清单"
  icon: "check-circle"

# 使用的卡片类型
cards:
  - type: "ListCard"     # 列表卡片（已有的基础卡片）

# 窗口布局
layout:
  type: single-card       # 单卡片窗口
  toolbar: true
  sidebar: false

# 功能配置
features:
  fileOpen: true          # 支持打开文件
  fileSave: true          # 支持保存文件
  fileNew: true           # 支持新建
  export: [html, pdf]     # 支持导出格式
```

这等同于"软件制作器"的声明式版本，让不会编程的人也能创建简单的薯片应用。

## 五、SDK 的开发体验

### 5.1 SDK 功能分层

```typescript
import { ChipsSDK } from '@chips/sdk';

// 初始化 SDK（自动连接 Bridge API）
const sdk = ChipsSDK.create();

// ===== 基础 API =====
// 文件操作
const content = await sdk.file.read('/path/to/file');
await sdk.file.write('/path/to/file', data);

// 卡片操作
const card = await sdk.card.open('/path/to/card.card');
const metadata = card.metadata;
const structure = card.structure;
const baseCards = card.baseCards;

// ===== 渲染辅助 =====
// 在容器中渲染一个完整的复合卡片
const renderer = sdk.createCardRenderer(containerElement);
await renderer.loadCard('/path/to/card.card');

// 渲染单个基础卡片
const baseCardRenderer = sdk.createBaseCardRenderer(iframeContainer);
await baseCardRenderer.render('VideoCard', configData);

// ===== 主题集成 =====
// 自动加载和应用主题
const theme = sdk.useTheme();
// theme.css → 当前主题的 CSS
// theme.onChange(callback) → 监听变化

// ===== React Composables（可选） =====
import { useCard, useTheme, useI18n } from '@chips/sdk/vue';

// 在 Vue 组件中使用
const card = useCard('/path/to/card.card');  // 响应式卡片数据
const theme = useTheme();                     // 响应式主题
const { t } = useI18n();                      // 翻译函数

// ===== React Hooks（可选） =====
import { useCard, useTheme, useI18n } from '@chips/sdk/react';
```

### 5.2 开发者最简路径

**场景：创建一个简单的待办应用**

```bash
# 1. 创建项目（30 秒）
chipsd create my-todo --template=single-card-app --framework=vue

# 2. 配置使用的卡片类型（编辑 manifest.yaml）
# 只需指定使用 ListCard

# 3. 启动开发（即时）
chipsd dev

# 4. 打包发布（1 分钟）
chipsd build && chipsd pack && chipsd publish
```

开发者不需要处理：
- Electron 配置（主机负责）
- 窗口管理（主机负责）
- 文件系统操作（通过 SDK）
- 主题系统（自动继承）
- 多语言（通过 SDK）
- 打包分发（CLI 负责）

### 5.3 进阶开发路径

**场景：开发一个全新的基础卡片类型**

```bash

chipsd create my-chart-card --template=interactive-card --framework=vue

# 2. 定义数据模型（src/types/config.ts）
# 3. 实现渲染组件（src/renderer/）
# 4. 实现编辑组件（src/editor/）
# 5. 开发调试
chipsd dev

# 6. 打包发布
chipsd build && chipsd pack && chipsd publish
```

## 六、插件市场

### 6.1 插件注册表（Registry）

类似 npm registry，薯片生态有自己的插件注册表：

```
chips-registry.example.com
  ├── /packages                    # 所有已发布的插件
  ├── /packages/{id}/versions      # 版本历史
  ├── /packages/{id}/latest        # 最新版本
  ├── /search?q=...&type=...       # 搜索
  └── /download/{id}/{version}     # 下载 .cpk
```

### 6.2 发布流程

```bash
# 登录开发者账号
chipsd login

# 验证插件
chipsd validate
# → 检查 manifest.yaml 完整性
# → 检查必需文件
# → 检查版本号格式
# → 检查许可证声明

# 发布
chipsd publish
# → 构建 .cpk
# → 上传到注册表
# → 自动审核（格式检查、安全扫描）
# → 审核通过后上架

# 发布更新
# 修改 manifest.yaml 中的版本号
chipsd publish
```

在设置面板的"插件管理"中：

## 八、第三方开发者体验总结

```
开发者想做一个薯片应用：

最简路径（零代码）：
  填写 YAML 声明文件 → chipsd pack → 发布
  适用：简单的数据展示或表单类应用

标准路径（少量代码）：
  chipsd create → 使用模板 → 修改 UI → chipsd publish
  适用：大部分标准应用

自由路径（完全自定义）：
  自建项目 → 引入 SDK → 自由开发 → chipsd pack → 发布
  适用：复杂的创新应用

共同点：
  - 所有路径都只需要 Web 前端技术
  - 不需要处理 Electron、打包、窗口管理、文件系统
  - 不需要实现主题系统（自动继承）
  - 不需要实现多语言（通过 SDK）
  - 统一的 .cpk 打包和分发
  - 统一的 chips.* Bridge API
```
