**版本**: 1.0.0  
**更新时间**: 2026-01-30  
**状态**: 正式版

## 1. 箱子概念定位

### 1.1 箱子与卡片的关系

**并行关系,非包含关系**:
- 箱子(Box)和卡片(Card)是**同等地位**的并行概念
- 箱子**不是**卡片的一种类型
- 箱子是卡片的**容器**,用于组织和展示多个卡片

### 1.2 核心特性

- **容器属性**: 可以包含多个卡片文件
- **布局系统**: 通过布局插件定义卡片展示方式
- **灵活存储**: 支持卡片内部存储或外部引用
- **独立配置**: 拥有自己的元数据和结构定义

---

## 2. 文件系统表现

### 2.1 物理结构

箱子在文件系统中表现为**文件夹 + 同名配置文件**:

```
My-Collection/              # 外部文件夹
├── My-Collection.box       # 箱子配置文件(ZIP压缩包)
├── card1.card              # 卡片文件(可选)
├── card2.card              # 卡片文件(可选)
└── card3.card              # 卡片文件(可选)
```

### 2.2 文件命名关系

- **箱子文件**: 文件名必须与父文件夹同名
- **示例**: 
  - 文件夹: `Travel-2026/`
  - 箱子文件: `Travel-2026/Travel-2026.box`

**注意**: 系统不需要强制验证母文件夹与箱子文件名是否一致,但建议保持一致以便识别。

### 2.3 双击行为

- 用户双击`.box`文件 → 系统用箱子查看器打开
- 修改扩展名为`.zip` → 可用任何解压工具查看内部结构

## 3. 箱子文件结构

### 3.1 .box文件内部结构

```
My-Collection.box (ZIP压缩包)
│
├── .box/                    # 核心配置目录(隐藏文件夹)
│   ├── metadata.yaml        # 箱子元数据
│   ├── structure.yaml       # 卡片列表
│   └── content.yaml         # 布局配置
│
├── card1.card               # 内部卡片(可选)
├── card2.card               # 内部卡片(可选)
└── ...
```

### 3.2 必需文件

箱子文件必须包含:
1. `.box/metadata.yaml` - 箱子元数据
2. `.box/structure.yaml` - 卡片列表
3. `.box/content.yaml` - 布局配置

### 3.3 可选文件

- 卡片文件(`.card`) - 根据箱子类型决定

## 4. 三种箱子类型

### 4.1 全填充箱子

**定义**: 所有卡片都保存在箱子文件内部

```
Collection.box (ZIP)
├── .box/
│   ├── metadata.yaml
│   ├── structure.yaml
│   └── content.yaml
├── card1.card      ✓ 在内部
├── card2.card      ✓ 在内部
└── card3.card      ✓ 在内部
```

**特点**:
- 完全自包含,不依赖外部文件
- 适合分享和备份
- 接收方无需额外下载

### 4.2 空壳箱子

**定义**: 所有卡片都保存在外部

```
Collection.box (ZIP)
└── .box/
    ├── metadata.yaml
    ├── structure.yaml (引用外部卡片)
    └── content.yaml

structure.yaml内容:
cards:
  - id: "card001"
    location: "external"
    path: "/Users/name/Cards/card1.card"
  - id: "card002"
    location: "external"
    path: "https://example.com/card2.card"
```

**特点**:
- 文件体积小
- 依赖外部资源
- 适合大量卡片的索引

### 4.3 半空壳箱子

**定义**: 部分卡片在内部,部分在外部

```
Collection.box (ZIP)
├── .box/
├── card1.card      ✓ 在内部
└── card2.card      ✓ 在内部

structure.yaml:
cards:
  - id: "card001"
    location: "internal"
    path: "card1.card"
  - id: "card002"
    location: "internal"
    path: "card2.card"
  - id: "card003"
    location: "external"
    path: "/external/card3.card"
```

**特点**:
- 平衡自包含性和灵活性
- 重要卡片内置,大文件外置
- 实际使用中最常见

### 4.4 类型判断

系统自动判断箱子类型:

```javascript
// 伪代码
function determineBoxType(structure) {
  const internal = structure.cards.filter(c => c.location === 'internal').length;
  const external = structure.cards.filter(c => c.location === 'external').length;
  
  if (external === 0) return 'FULL';      // 全填充
  if (internal === 0) return 'EMPTY';     // 空壳
  return 'HYBRID';                         // 半空壳
}
```

## 5. 元数据文件 (metadata.yaml)

### 5.1 完整Schema定义

```yaml
$schema: "http://json-schema.org/draft-07/schema#"
title: "Box Metadata Schema"
type: object
required:
  - chip_standards_version
  - box_id
  - name
  - created_at
  - modified_at
  - layout

properties:
  # === 基础信息 ===
  chip_standards_version:
    type: string
    pattern: '^\d+\.\d+\.\d+$'
    description: "薯片标准版本号"
    example: "1.0.0"
    
  box_id:
    type: string
    pattern: '^[0-9a-zA-Z]{10}$'
    description: "箱子唯一ID,10位62进制编号"
    example: "b1C2d3E4f5"
    
  name:
    type: string
    maxLength: 500
    description: "箱子名称,最长500字符"
    example: "我的旅行收藏"
    
  created_at:
    type: string
    format: date-time
    description: "创建时间,ISO 8601格式,UTC时区"
    example: "2026-01-15T08:00:00.000Z"
    
  modified_at:
    type: string
    format: date-time
    description: "最后修改时间"
    example: "2026-01-30T10:30:00.000Z"
    
  # === 布局设置 ===
  layout:
    type: string
    description: "当前使用的布局插件,格式: '发行商:布局名称'"
    example: "薯片官方:瀑布流布局"
    
  # === 标签系统 ===
  tags:
    type: array
    description: "箱子标签"
    items:
      oneOf:
        - type: string
        - type: array
          items:
            type: string
          minItems: 2
    examples:
      - ["收藏"]
      - ["类型", "旅行"]
      - ["时间", "2026年"]
      
  # === 可选属性 ===
  description:
    type: string
    description: "箱子描述"
    
  visibility:
    type: string
    enum: ["public", "private", "unlisted"]
    default: "private"
    description: "可见性设置"
    
  thumbnail:
    type: string
    description: "缩略图路径"
    
  # === 箱子类型(系统自动判断) ===
  box_type:
    type: string
    enum: ["full", "empty", "hybrid"]
    description: "箱子类型(系统生成)"
    
  # === 文件信息 ===
  file_info:
    type: object
    properties:
      total_size:
        type: integer
        description: "箱子总大小(字节)"
      card_count:
        type: integer
        description: "卡片总数"
      internal_card_count:
        type: integer
        description: "内部卡片数"
      external_card_count:
        type: integer
        description: "外部卡片数"
      checksum:
        type: string
        description: "完整性校验值,SHA256"
```

### 5.2 完整示例

```yaml
# metadata.yaml 示例

# 基础信息
chip_standards_version: "1.0.0"
box_id: "b1C2d3E4f5"
name: "日本旅行 2026"
created_at: "2026-01-15T08:00:00.000Z"
modified_at: "2026-01-30T10:30:00.000Z"

# 布局设置

# 标签
tags:
  - "旅行"
  - ["地点", "日本"]
  - ["时间", "2026年", "一月"]
  - ["类型", "个人收藏"]

# 可选属性
description: "2026年1月日本旅行的所有卡片收藏"
visibility: "public"
thumbnail: "box-cover.jpg"

# 系统信息
box_type: "hybrid"

file_info:
  total_size: 157286400

  internal_card_count: 15

```

## 6. 结构文件 (structure.yaml)

### 6.1 完整Schema定义

```yaml
$schema: "http://json-schema.org/draft-07/schema#"
title: "Box Structure Schema"
type: object
required:
  - cards

properties:
  cards:
    type: array
    description: "卡片列表"
    items:
      type: object
      required:
        - id
        - location
        - path
      properties:
        id:
          type: string
          pattern: '^[0-9a-zA-Z]{10}$'
          description: "卡片ID(不是文件名)"
          
        location:
          type: string
          enum: ["internal", "external"]
          description: "卡片位置"
          
        path:
          type: string
          description: "卡片路径"
          examples:
            - "card1.card"                    # 内部相对路径
            - "/Users/name/Cards/card.card"   # 外部绝对路径
            - "https://example.com/card.card" # 网络地址
            - "webdav://server/card.card"     # WebDAV地址
            
        filename:
          type: string
          description: "原始文件名(用于显示)"
          
        added_at:
          type: string
          format: date-time
          description: "添加到箱子的时间"
          
        metadata_cache:
          type: object
          description: "卡片元数据缓存(可选,提升性能)"
          properties:
            name:
              type: string
            thumbnail:
              type: string
            size:
              type: integer
            card_type:
              type: string
```

### 6.2 完整示例

```yaml
# structure.yaml 示例

cards:
  # 内部卡片
  - id: "a1B2c3D4e5"
    location: "internal"
    path: "tokyo-tower.card"
    filename: "东京塔夜景.card"
    added_at: "2026-01-16T10:00:00.000Z"
    metadata_cache:
      name: "东京塔夜景"
      thumbnail: "tokyo-tower.card/.card/cardcover/thumb.jpg"
      size: 5242880
      
  - id: "f6G7h8I9j0"
    location: "internal"
    path: "travel-diary.card"
    filename: "旅行日记.card"
    added_at: "2026-01-17T14:30:00.000Z"
    metadata_cache:
      name: "旅行日记"
      size: 2097152
      
  # 外部卡片(文件系统)
  - id: "k1L2m3N4o5"
    location: "external"
    path: "/Users/username/Documents/Cards/full-video.card"
    filename: "完整旅行视频.card"
    added_at: "2026-01-18T16:00:00.000Z"
    metadata_cache:
      name: "完整旅行视频"
      size: 104857600
      
  # 外部卡片(网络)
  - id: "p6Q7r8S9t0"
    location: "external"
    path: "https://cdn.example.com/shared/card123.card"
    filename: "共享卡片.card"
    added_at: "2026-01-19T09:15:00.000Z"
    metadata_cache:
      name: "景点介绍"
      size: 1048576
      
  # 外部卡片(WebDAV)
  - id: "u1V2w3X4y5"
    location: "external"
    path: "webdav://myserver.com/cards/backup.card"
    filename: "备份卡片.card"
    added_at: "2026-01-20T11:00:00.000Z"
```

### 6.3 元数据缓存说明

**目的**: 避免每次都读取外部卡片文件

**更新时机**:
- 添加卡片时生成
- 卡片修改时更新
- 定期刷新(可配置)

**可选性**: 缓存不是必需的,缺失时实时读取

## 7. 布局配置文件 (content.yaml)

### 7.1 配置结构

```yaml
$schema: "http://json-schema.org/draft-07/schema#"
title: "Box Content Schema"
type: object
required:
  - layout_configs

properties:
  layout_configs:
    type: object
    description: "多个布局的配置,key为布局标识"
    additionalProperties:
      type: object
      description: "具体布局的配置,结构由布局插件定义"
      
  active_layout:
    type: string
    description: "当前激活的布局(与metadata.yaml中的layout字段对应)"
```

### 7.2 布局配置示例

```yaml
# content.yaml 示例

active_layout: "薯片官方:瀑布流布局"

layout_configs:
  # 瀑布流布局配置
  "薯片官方:瀑布流布局":
    columns: 3
    gap: 16
    auto_load: true
    card_width: 300
    
  # 网格布局配置(备用)
  "薯片官方:网格布局":
    columns: 4
    rows: "auto"
    gap_horizontal: 20
    gap_vertical: 20
    card_aspect_ratio: "16:9"
    
  # 列表布局配置(备用)
  "薯片官方:列表布局":
    direction: "vertical"
    show_thumbnail: true
    show_summary: true
    item_height: 120
    
  # 无限桌面布局配置
  "薯片官方:无限桌面布局":
    canvas_width: 5000
    canvas_height: 3000
    background_color: "#f5f5f5"
    card_positions:
      - card_id: "a1B2c3D4e5"
        x: 100
        y: 100
        width: 400
        height: 300
      - card_id: "f6G7h8I9j0"
        x: 550
        y: 150
        width: 350
        height: 400
```

### 7.3 布局切换

用户切换布局时:
1. 更新`metadata.yaml`中的`layout`字段
2. 更新`content.yaml`中的`active_layout`
3. 加载对应的布局配置
4. 卡片数据不变,仅展示方式改变

## 8. 卡片导入方式

### 8.1 三种导入方式

#### 方式1: 复制到箱子

```
操作: 用户拖拽卡片到箱子,选择"复制"
结果:
  - 卡片文件复制到箱子根目录
  - structure.yaml添加记录,location: "internal"
  - 原文件保持不变
```

#### 方式2: 移动到箱子

```
操作: 用户拖拽卡片到箱子,选择"移动"
结果:
  - 卡片文件移动到箱子根目录
  - structure.yaml添加记录,location: "internal"
  - 原文件位置的文件被删除
```

#### 方式3: 登记地址

```
操作: 用户拖拽卡片到箱子,选择"仅登记地址"
结果:
  - 卡片文件保持原位置
  - structure.yaml添加记录,location: "external",path: "原路径"
  - 不复制文件
```

## 9. 外部卡片扫描

### 9.1 默认扫描路径

箱子默认扫描`.box`文件的同级文件夹:

```
My-Collection/
├── My-Collection.box       # 箱子文件
├── card1.card              # 自动扫描
├── card2.card              # 自动扫描
└── subfolder/
    └── card3.card          # 不扫描子文件夹
```

### 9.2 自定义扫描路径

用户可以在箱子设置中添加扫描路径:

```yaml
# 箱子内部配置(不在文件中,在查看器本地数据库)
scan_paths:
  - "/Users/username/Documents/Cards"
  - "https://cdn.example.com/shared-cards"
  - "webdav://myserver.com/cards"
  - "~/Dropbox/ChipsCards"
```

### 9.3 扫描策略

```javascript
// 伪代码
function findExternalCard(cardPath) {
  // 1. 尝试直接访问路径
  if (exists(cardPath)) return cardPath;
  
  // 2. 在默认位置查找(同级文件夹)
  const defaultPath = boxFolder + '/' + basename(cardPath);
  if (exists(defaultPath)) return defaultPath;
  
  // 3. 在自定义扫描路径中查找
  for (const scanPath of scanPaths) {
    const fullPath = scanPath + '/' + basename(cardPath);
    if (exists(fullPath)) return fullPath;
  }
  
  // 4. 未找到
  return null;
}
```

## 10. 网络存储支持

### 10.1 支持的网络协议

- **HTTPS**: 普通网络地址
- **WebDAV**: WebDAV服务器
- **FTP/SFTP**: FTP服务器(可选)

# 本地秘钥管理器(查看器本地数据库)

      Authorization: "Bearer token123"

## 11. 布局系统

### 11.1 布局插件标识

格式: `发行商:布局名称`

示例:
- `薯片官方:瀑布流布局`
- `薯片官方:网格布局`
- `薯片官方:列表布局`
- `第三方开发者:画廊布局`

### 11.2 标准布局类型

参见《28-箱子布局类型设计》笔记:

1. **列表布局**: 垂直或水平排列
2. **网格布局**: 固定列数网格
3. **瀑布流布局**: 瀑布流算法
4. **无限桌面布局**: 自由拖放
5. **时间线布局**: 按时间排序
6. **书架布局**: 书架视觉效果
7. **个人主页布局**: 个人展示页
8. **朋友圈布局**: 社交动态流

### 11.3 布局配置结构

每种布局插件定义自己的配置Schema:

```yaml
# 示例: 瀑布流布局配置Schema
$schema: "http://json-schema.org/draft-07/schema#"
title: "Waterfall Layout Config"
type: object
properties:
  columns:
    type: integer
    minimum: 1
    maximum: 10
    default: 3
    description: "列数"
    
  gap:
    type: integer
    minimum: 0
    default: 16
    description: "间距(像素)"
    
  auto_load:
    type: boolean
    default: true
    description: "滚动到底部是否自动加载"
    
  card_width:
    type: integer
    description: "卡片宽度(像素),不设置则自动计算"
```

### 11.4 布局插件接口

布局插件必须实现的接口:

```typescript
// TypeScript接口定义
interface BoxLayoutPlugin {
  // 插件元数据
  metadata: {
    id: string;              // "publisher:layout-name"
    name: string;            // 显示名称
    version: string;         // 版本号
    description: string;     // 描述
    icon: string;            // 图标
  };
  
  // 配置Schema
  configSchema: JSONSchema;
  
  // 默认配置
  defaultConfig: object;
  
  // 渲染函数
  render(cards: Card[], config: object, container: HTMLElement): void;
  
  // 配置编辑器(可选)
  renderConfigEditor?(config: object, onChange: Function): HTMLElement;
  
  // 卸载函数
  destroy(): void;
}
```

## 12. 版本兼容

### 12.1 版本号

```yaml
# metadata.yaml
chip_standards_version: "1.0.0"
```

遵循语义化版本规范,兼容策略同卡片文件格式。

thumbnail_cache:
