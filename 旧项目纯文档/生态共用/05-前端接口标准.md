## 1. 概述

### 1.1 前端接口的定位

前端接口标准定义了**前端组件**必须实现的接口规范,确保:

- 不同开发者的前端实现可以互换
- 前端组件可以对接标准化的后端模块
- 同一功能可以有多种前端实现并存

### 1.2 前后端分离原则

```
┌──────────────┐         ┌──────────────┐
│  前端组件 A   │         │  前端组件 B   │
│  (React实现)  │         │  (Vue实现)    │
└───────┬──────┘         └───────┬──────┘
        │                        │
        └────────┬───────────────┘
                 │
        ┌────────▼──────────┐
        │   Chips Core      │
        │   (中心路由)       │
        └────────┬──────────┘
                 │
        ┌────────▼──────────┐
        │    后端模块        │
        └───────────────────┘
```

**关键原则**:
- 前端只负责界面展示和用户交互
- 前端不包含业务逻辑
- 前端通过 Bridge API（`window.chips.*`）与后端通信，Bridge 通过 IPC 转发到主进程内核路由器
- 前端组件之间不直接通信
- 组件库采用无头设计（基于 Zag.js + Ark UI），只有逻辑和结构，零样式，视觉由主题包注入

### 1.3 适用范围

本标准适用于:
- 基础卡片的编辑组件和渲染组件
- 箱子布局组件
- 编辑引擎布局组件
- 工具窗口组件
- 主题系统

## 2. 组件生命周期接口

### 2.1 生命周期方法

所有前端组件必须实现以下生命周期方法:

```typescript
interface ComponentLifecycle {
  // 初始化
  initialize(config: ComponentConfig): Promise<void>;
  
  // 挂载到DOM
  mount(container: HTMLElement): Promise<void>;
  
  // 更新配置
  update(config: Partial<ComponentConfig>): Promise<void>;
  
  // 卸载
  unmount(): Promise<void>;
  
  // 销毁
  destroy(): Promise<void>;
}
```

### 2.2 生命周期流程

```
initialize() → mount() → [update()]* → unmount() → destroy()
```

### 2.3 详细说明

#### initialize()

**职责**: 初始化组件内部状态,不涉及DOM操作

```typescript
async initialize(config: ComponentConfig): Promise<void> {
  this.config = config;
  this.state = this.createInitialState();
  await this.loadDependencies();
}
```

#### mount()

**职责**: 将组件挂载到指定的DOM容器

```typescript
async mount(container: HTMLElement): Promise<void> {
  this.container = container;
  this.render();
  this.attachEventListeners();
  await this.onMounted();
}
```

#### update()

**职责**: 响应配置变更,更新组件显示

```typescript
async update(config: Partial<ComponentConfig>): Promise<void> {
  this.config = { ...this.config, ...config };
  this.render();
}
```

#### unmount()

**职责**: 从DOM卸载,保留内部状态

```typescript
async unmount(): Promise<void> {
  this.removeEventListeners();
  this.container?.remove();
  this.container = null;
}
```

#### destroy()

**职责**: 完全销毁组件,释放所有资源

```typescript
async destroy(): Promise<void> {
  await this.unmount();
  this.state = null;
  this.config = null;
  // 清理其他资源
}
```

```typescript
class VideoCardRenderer implements BaseCardRenderer {
  metadata = {
    id: "chipshub:video-card-renderer",
    name: "视频卡片渲染器",
    version: "1.0.0",
    cardType: "VideoCard",
    icon: "video-icon.svg",
    description: "标准视频卡片渲染组件"
  };
  
  configSchema = {
    type: "object",
    properties: {
      video_file: { type: "string" },
      subtitles: { type: "array" },
      autoplay: { type: "boolean" }
    }
  };
  
  private player: HTMLVideoElement;
  private config: VideoCardConfig;
  
  async initialize(config: VideoCardConfig): Promise<void> {
    this.config = config;
    await this.preloadVideo();
  }
  
  async mount(container: HTMLElement): Promise<void> {
    this.container = container;
    await this.render();
  }
  
  async render(): Promise<void> {
    // 创建视频播放器
    this.player = document.createElement('video');
    this.player.src = await this.resolveVideoPath(this.config.video_file);
    this.player.controls = this.config.controls !== false;
    
    // 添加字幕
    for (const subtitle of this.config.subtitles || []) {
      const track = document.createElement('track');
      track.src = await this.resolveSubtitlePath(subtitle.file);
      track.srclang = subtitle.language;
      track.label = subtitle.label;
      this.player.appendChild(track);
    }
    
    this.container.appendChild(this.player);
  }
  
  async update(config: Partial<VideoCardConfig>): Promise<void> {
    this.config = { ...this.config, ...config };
    await this.render();
  }
  
  async unmount(): Promise<void> {
    this.player?.pause();
    this.player?.remove();
  }
  
  async destroy(): Promise<void> {
    await this.unmount();
    this.player = null;
    this.config = null;
  }
  
  getState(): VideoState {
    return {
      currentTime: this.player?.currentTime || 0,
      paused: this.player?.paused || true,
      volume: this.player?.volume || 1
    };
  }
  
  setState(state: Partial<VideoState>): void {
    if (state.currentTime !== undefined) {
      this.player.currentTime = state.currentTime;
    }
    if (state.paused === false) {
      this.player.play();
    }
  }

    // 通过 Bridge API 的资源服务解析路径
    const resource = await window.chips.invoke("resource", "fetch", {
      uri: `chips://card/${this.config.card_id}/${path}`
    });
    return resource.url;
  }
  
  private async resolveSubtitlePath(path: string): Promise<string> {
    // 类似视频路径解析
    return path;
  }
  
  private async preloadVideo(): Promise<void> {

  }
}
```

### 4.4 示例:图片卡片编辑组件

```typescript
class ImageCardEditor implements BaseCardEditor {
  metadata = {
    id: "chipshub:image-card-editor",
    name: "图片卡片编辑器",
    version: "1.0.0",
    cardType: "ImageCard",
    icon: "image-icon.svg",
    description: "标准图片卡片编辑组件"
  };
  
  private config: ImageCardConfig;
  private changeCallback: ((config: ImageCardConfig) => void) | null;
  
  async render(
    config: ImageCardConfig,
    container: HTMLElement,
    options: EditorOptions
  ): Promise<void> {
    this.config = config;
    this.container = container;
    
    // 创建编辑界面
    const editor = document.createElement('div');
    editor.className = 'image-card-editor';
    
    // 文件上传区
    const uploader = this.createUploader();
    editor.appendChild(uploader);
    
    // 标题输入
    const titleInput = this.createTitleInput();
    editor.appendChild(titleInput);
    
    // 说明输入
    const captionInput = this.createCaptionInput();
    editor.appendChild(captionInput);
    
    // 适配模式选择
    const fitModeSelector = this.createFitModeSelector();
    editor.appendChild(fitModeSelector);
    
    // 预览
    if (options.preview !== false) {
      const preview = this.createPreview();
      editor.appendChild(preview);
    }
    
    container.appendChild(editor);
  }
  
  getConfig(): ImageCardConfig {
    return { ...this.config };
  }
  
  setConfig(config: Partial<ImageCardConfig>): void {
    this.config = { ...this.config, ...config };
    this.updateUI();
    this.notifyChange();
  }
  
  validate(): ValidationResult {
    const errors: ValidationError[] = [];
    
    if (!this.config.image_file) {
      errors.push({
        field: 'image_file',
        message: '请上传图片',
        code: 'REQUIRED'
      });
    }
    
    return {
      valid: errors.length === 0,
      errors: errors.length > 0 ? errors : undefined
    };
  }
  
  onChange(callback: (config: ImageCardConfig) => void): void {
    this.changeCallback = callback;
  }
  
  private createUploader(): HTMLElement {
    const uploader = document.createElement('div');
    uploader.className = 'file-uploader';
    
    const input = document.createElement('input');
    input.type = 'file';
    input.accept = 'image/*';
    input.onchange = async (e) => {
      const file = (e.target as HTMLInputElement).files?.[0];
      if (file) {
        await this.handleImageUpload(file);
      }
    };
    
    uploader.appendChild(input);
    return uploader;
  }
  
  private createTitleInput(): HTMLElement {
    const container = document.createElement('div');
    const label = document.createElement('label');
    label.textContent = '标题:';
    const input = document.createElement('input');
    input.type = 'text';
    input.value = this.config.title || '';
    input.oninput = () => {
      this.setConfig({ title: input.value });
    };
    container.appendChild(label);
    container.appendChild(input);
    return container;
  }
  
  private createCaptionInput(): HTMLElement {
    // 类似标题输入
    return document.createElement('div');
  }
  
  private createFitModeSelector(): HTMLElement {
    const container = document.createElement('div');
    const label = document.createElement('label');
    label.textContent = '适配模式:';
    const select = document.createElement('select');
    
    const modes = ['contain', 'cover', 'fill', 'none'];
    modes.forEach(mode => {
      const option = document.createElement('option');
      option.value = mode;
      option.textContent = mode;
      option.selected = this.config.fit_mode === mode;
      select.appendChild(option);
    });
    
    select.onchange = () => {
      this.setConfig({ fit_mode: select.value as any });
    };
    
    container.appendChild(label);
    container.appendChild(select);
    return container;
  }
  
  private createPreview(): HTMLElement {
    const preview = document.createElement('div');
    preview.className = 'preview';
    
    if (this.config.image_file) {
      const img = document.createElement('img');
      img.src = this.config.image_file;
      preview.appendChild(img);
    }
    
    return preview;
  }
  
  private async handleImageUpload(file: File): Promise<void> {
    // 上传图片到卡片
    await window.chips.invoke("resource", "write", {
      uri: `chips://card/${this.config.card_id}/${file.name}`,
      data: await file.arrayBuffer()
    });
    
    this.setConfig({ image_file: file.name });
  }
  
  private updateUI(): void {
    // 更新UI显示
  }
  
  private notifyChange(): void {
    if (this.changeCallback) {
      this.changeCallback(this.config);
    }
  }
}
```

### 5.3 示例:瀑布流布局组件

```typescript
class WaterfallLayoutComponent implements BoxLayoutComponent {
  metadata = {
    id: "chipshub:waterfall-layout",
    name: "瀑布流布局",
    version: "1.0.0",
    icon: "waterfall-icon.svg",
    description: "瀑布流卡片布局"
  };
  
  configSchema = {
    type: "object",
    properties: {
      columns: { type: "integer", minimum: 1, maximum: 10, default: 3 },
      gap: { type: "integer", minimum: 0, default: 16 },
      auto_load: { type: "boolean", default: true },
      card_width: { type: "integer", default: 300 }
    }
  };
  
  defaultConfig = {
    columns: 3,
    gap: 16,
    auto_load: true,
    card_width: 300
  };
  
  private cards: CardItem[] = [];
  private config: WaterfallConfig;
  private columns: HTMLElement[] = [];
  
  async render(
    cards: CardItem[],
    config: WaterfallConfig,
    container: HTMLElement
  ): Promise<void> {
    this.cards = cards;
    this.config = config;
    this.container = container;
    
    // 创建列
    this.createColumns();
    
    // 分配卡片到列
    this.distributeCards();
    
    // 自动加载
    if (this.config.auto_load) {
      this.setupAutoLoad();
    }
  }
  
  addCard(card: CardItem, position?: number): void {
    if (position !== undefined) {
      this.cards.splice(position, 0, card);
    } else {
      this.cards.push(card);
    }
    this.distributeCards();
  }
  
  removeCard(cardId: string): void {
    this.cards = this.cards.filter(c => c.id !== cardId);
    this.distributeCards();
  }
  
  reorder(cardIds: string[]): void {
    const orderedCards: CardItem[] = [];
    cardIds.forEach(id => {
      const card = this.cards.find(c => c.id === id);
      if (card) orderedCards.push(card);
    });
    this.cards = orderedCards;
    this.distributeCards();
  }
  
  renderConfigEditor(
    config: WaterfallConfig,
    onChange: (config: WaterfallConfig) => void
  ): HTMLElement {
    const editor = document.createElement('div');
    
    // 列数选择
    const columnsInput = this.createNumberInput(
      '列数',
      config.columns,
      (value) => onChange({ ...config, columns: value })
    );
    editor.appendChild(columnsInput);
    
    // 间距选择
    const gapInput = this.createNumberInput(
      '间距',
      config.gap,
      (value) => onChange({ ...config, gap: value })
    );
    editor.appendChild(gapInput);
    
    return editor;
  }
  
  private createColumns(): void {
    this.container.innerHTML = '';
    this.columns = [];
    
    const columnsContainer = document.createElement('div');
    columnsContainer.style.display = 'flex';
    columnsContainer.style.gap = `${this.config.gap}px`;
    
    for (let i = 0; i < this.config.columns; i++) {
      const column = document.createElement('div');
      column.style.flex = '1';
      column.style.display = 'flex';
      column.style.flexDirection = 'column';
      column.style.gap = `${this.config.gap}px`;
      columnsContainer.appendChild(column);
      this.columns.push(column);
    }
    
    this.container.appendChild(columnsContainer);
  }
  
  private distributeCards(): void {
    // 清空所有列
    this.columns.forEach(col => col.innerHTML = '');
    
    // 计算每列的高度
    const columnHeights = new Array(this.columns.length).fill(0);
    
    // 将卡片分配到最短的列
    this.cards.forEach(card => {
      const shortestColumnIndex = columnHeights.indexOf(Math.min(...columnHeights));
      const cardElement = this.createCardElement(card);
      this.columns[shortestColumnIndex].appendChild(cardElement);
      
      // 更新列高度(简化,实际需要测量)
      columnHeights[shortestColumnIndex] += 300;
    });
  }
  
  private createCardElement(card: CardItem): HTMLElement {
    const element = document.createElement('div');
    element.className = 'card-item';
    element.dataset.cardId = card.id;
    element.style.width = `${this.config.card_width}px`;
    
    // 缩略图
    if (card.metadata.thumbnail) {
      const img = document.createElement('img');
      img.src = card.metadata.thumbnail;
      img.style.width = '100%';
      element.appendChild(img);
    }
    
    // 标题
    const title = document.createElement('h3');
    title.textContent = card.metadata.name;
    element.appendChild(title);
    
    // 点击事件
    element.onclick = () => this.handleCardClick(card);
    
    return element;
  }
  
  private setupAutoLoad(): void {
    const observer = new IntersectionObserver(
      (entries) => {
        entries.forEach(entry => {
          if (entry.isIntersecting) {
            this.loadMoreCards();
          }
        });
      },
      { threshold: 0.1 }
    );
    
    // 观察最后一个卡片
    const lastCard = this.container.querySelector('.card-item:last-child');
    if (lastCard) {
      observer.observe(lastCard);
    }
  }
  
  private async loadMoreCards(): Promise<void> {
    // 通过 Bridge API 加载更多卡片
  }
  
  private handleCardClick(card: CardItem): void {
    // 通过 Bridge API 打开卡片
    window.chips.invoke("card", "open", {
      card_id: card.id
    });
  }
  
  private createNumberInput(
    label: string,
    value: number,
    onChange: (value: number) => void
  ): HTMLElement {
    const container = document.createElement('div');
    const labelElement = document.createElement('label');
    labelElement.textContent = label + ':';
    const input = document.createElement('input');
    input.type = 'number';
    input.value = String(value);
    input.oninput = () => onChange(Number(input.value));
    container.appendChild(labelElement);
    container.appendChild(input);
    return container;
  }
}
```

## 6. 主题系统接口

### 6.1 主题插件契约（Manifest 口径）

```typescript
interface ThemeManifestContract {
  schemaVersion: '1.0.0';
  type: 'theme';
  id: string; // 例如: chips-official.default-theme
  name: string;
  publisher: string;
  version: string;
  entry: {
    tokens: string; // 包根相对路径
    themeCss: string; // 包根相对路径
  };
  theme: {
    tokensVersion: string;
    inherits?: string;
  };
  compatibility: {
    host: string;
    sdk?: string;
  };
}
```

> 主题入口必须使用包根相对路径，`entry.tokens` / `entry.themeCss` 禁止 `dist/` 前缀。

### 6.1.1 阶段06模板契约边界（补充）

- 五模板统一要求：`schemaVersion: "1.0.0"`、`compatibility.host` 必填。
- 页面型模板（`app/card/layout/module`）强制声明 `ui.layout.owner/unit/baseWidth/contract/minFunctionalSet`。
- `theme` 模板豁免 `ui.layout`，仅要求 `entry.tokens`、`entry.themeCss`、`theme` 字段。
- 公共调用链统一为 `UI Hooks -> Runtime Client -> Bridge Transport`。

### 6.2 主题标识与展示命名

```typescript
interface ThemeIdentity {
  themeId: string;      // 技术主键（唯一）
  displayName: string;  // 展示名（可本地化）
}
```

- 系统主键统一使用 `themeId`，默认主题固定为 `chips-official.default-theme`。
- “发行商：主题名”仅作为展示名格式，不作为配置主键或路由参数。
- 主题包采用“一包一语义”口径；主题清单中不再使用白天/夜间标签字段。

### 6.3 主题运行时接口

统一动作命名：`theme.list`、`theme.apply`、`theme.getCurrent`、`theme.getAllCss`、`theme.resolve`、`theme.contract.get`。

```typescript
interface ThemeRuntimeAPI {
  // 列出可用主题
  list(params?: { publisher?: string }): Promise<{
    themes: Array<{ id: string; name: string; version?: string }>;
  }>;

  // 查询当前主题
  getCurrent(params?: { appId?: string; pluginId?: string }): Promise<{ themeId: string }>;

  // 切换主题
  apply(params: { id: string }): Promise<{ applied: boolean; themeId: string }>;

  // 获取主题 CSS（分层对象，按 Host 规范返回）
  getAllCss(params?: { appId?: string; pluginId?: string }): Promise<{
    css: Record<string, string>;
  }>;

  // 解析多层级主题链并返回合并后的 Token
  resolve(params: {
    chain: {
      component?: string;
      card?: string;
      workspace?: string;
      app?: string;
      global?: string;
    };
  }): Promise<{
    resolvedThemeId: string;
    mergedTokens: Record<string, string>;
  }>;

  // 查询主题接口契约
  contractGet(params?: { component?: string }): Promise<{ contract: Record<string, unknown> }>;
}
```

### 6.3.1 历史动作禁用约束（阶段05任务03后生效）

- 历史动作 `theme.getCSS`、`theme.setCurrent` 已废止，不再提供 SDK 自动映射。
- 页面层和业务层禁止手写主题动作兼容分支，调用方必须直接使用统一动作。
- 运行时若收到历史动作，应按契约阻断并返回标准错误。
- `theme.get/install/uninstall` 为 Host 内部治理动作，不作为插件 Bridge 外部主路径。

### 6.3.2 主题全链路事件流（2026-02-27 实现完成）

`theme.apply` 调用后的完整事件传播链路：

```
theme.apply({ id }) → Host theme-service 写配置 + 清缓存
                    → emitThemeChanged(themeId) 发布 theme.changed 事件（propagate: true）
                    → event-forwarder（通配订阅 '*'）自动转发到所有渲染进程
                    → SDK useTheme().onChanged 回调触发
                    → viewer/editor 壳层 onThemeChanged 回调
                    → 壳层调用 theme.getAllCss（分层对象）+ theme.resolve（tokens）并按 tokens->components->animations->icons 拼接 CSS 字符串
                    → broadcastThemeChange(css, tokens, themeId) 到 card iframe
                    → card iframe 收到 theme-change 消息 → 更新本地主题样式
```

Card Runtime 初始化时，壳层通过 bridge 调用 `theme.getAllCss` 和 `i18n.getCurrent` 获取真实数据注入 init 载荷，card iframe 启动即拥有正确的主题和语言环境。

### 6.4 继承与覆盖规则

1. 查找优先级：组件 -> 卡片 -> 工作区 -> 应用 -> 全局
2. Token 合并顺序：全局 -> 应用 -> 工作区 -> 卡片 -> 组件
3. 主题系统采用 CSS 资源注入模型，不使用对象化主题包模型作为运行时主模型

## 8. 多语言接口

### 8.1 多语言系统概述

**薯片生态采用系统统一管理的多语言方案**。所有前端组件都使用系统提供的多语言服务，核心特点：

- **零硬编码**：代码中不允许出现任何明文字符串
- **系统接管**：所有文本显示由系统统一管理
- **编码替换**：开发时使用key，打包时替换为系统编码
- **配置表管理**：所有文本必须在配置表中定义

> **完整规范**: 详见 [多语言系统规范](../生态共用/11-多语言系统规范.md)

### 8.2 多语言接口

```typescript
interface I18nSupport {
  // 获取可用语言列表
  list(): Promise<{ locales: string[] }>;

  // 获取当前语言
  getCurrent(): Promise<{ locale: string }>;

  // 设置语言
  setCurrent(params: { locale: string; persist?: boolean }): Promise<{ changed: boolean }>;

  // 翻译文本（使用词汇编码）
  translate(params: {
    key: string;
    locale?: string;
    vars?: Record<string, string | number>;
    fallback?: string;
  }): Promise<{ text: string; locale: string }>;

  // 批量翻译（使用词汇编码）
  translateBatch(params: {
    keys: string[];
    locale?: string;
    vars?: Record<string, Record<string, string | number>>;
  }): Promise<{ items: Array<{ key: string; text: string }> }>;

  // 注册插件词汇表
  registerVocabulary(params: {
    pluginId: string;
    entries: Record<string, Record<string, string>>;
  }): Promise<{ registered: number }>;
}
```

### 8.2.1 历史动作禁用约束（阶段05任务03后生效）

- 历史动作 `i18n.getCurrentLanguage`、`i18n.setLanguage` 已废止，不再提供 SDK 自动映射。
- 调用方必须直接使用 `i18n.getCurrent` 与 `i18n.setCurrent`。

多语言事件口径固定：
- 事件名：`language.changed`
- 事件载荷：`{ language: string }`

### 8.3 开发阶段用法

**开发时使用key**：

```typescript
// 开发阶段代码
import { t } from '@chips/i18n';

// 使用人类可读的key
const playText = t('video.play');
const uploadHint = t('image.upload_hint');
```

**开发时词汇表** (`dev_i18n.yaml`)：

```yaml
# 开发阶段词汇定义
video:
  play: "播放"
  pause: "暂停"
  
image:
  upload: "上传图片"
  upload_hint: "点击或拖拽图片到此处"
```

### 8.4 打包后代码

打包时，构建工具自动将key替换为系统词汇编码：

```typescript
// 打包后代码（自动替换）
const playText = t('i18n.card.400015');
const uploadHint = t('i18n.card.400032');
```

### 8.5 词汇编码分类

前端组件使用的词汇编码范围：

| 编码前缀 | 编码范围 | 说明 |
|---------|---------|------|
| `i18n.core.` | 000001-099999 | 核心系统词汇 |
| `i18n.ui.` | 100000-199999 | UI组件通用词汇 |
| `i18n.card.` | 400000-499999 | 卡片类型词汇 |
| `i18n.plugin.` | 600000-899999 | 插件自定义词汇 |

### 8.6 最佳实践

1. **优先使用系统词汇**：常用词汇（如"确定"、"取消"、"保存"等）使用系统已有编码
2. **避免重复定义**：查阅系统词汇表，避免重复定义相同含义的词汇
3. **明确的key命名**：开发阶段使用清晰、有意义的key名称
4. **支持变量插值**：文本中可以使用变量占位符

```typescript
// 支持变量插值
t('ui.file_count', { count: 10 })  // "共 10 个文件"
```
