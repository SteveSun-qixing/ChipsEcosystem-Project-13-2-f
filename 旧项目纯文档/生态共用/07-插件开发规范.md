# 插件开发规范

> 文档状态：阶段01冻结同步稿（2026-02-24）
> 统一口径：12层 vNext、Bridge 三层（`UI Hooks -> Runtime Client -> Bridge Transport`）、主题动作（`theme.list/theme.apply/theme.getCurrent/theme.getAllCss/theme.resolve/theme.contract.get`）、`cpx` + `baseWidth: 1024`。
> 冲突优先级：若与历史描述冲突，以 `生态架构设计/生态重构开发计划/18-阶段18-主题系统组件库前端框架开发/执行计划/阶段01-全量技术文档与标准冻结/` 目录下冻结文档为准。


**版本**: 1.2.0
**更新时间**: 2026-02-26
**状态**: 正式版

## 1. 基础原则

- 中心路由：系统能力统一通过 `window.chips.invoke(namespace, action, params)`。
- Bridge 隔离：插件不得直接使用 Node.js/Electron API。
- 调用链固定：`UI Hooks -> Runtime Client -> Bridge Transport`。
- 零硬编码文本：用户可见文本必须接入 i18n。
- 零样式组件：结构与样式分离，由主题注入视觉表现。

## 2. 插件类型

`manifest.yaml` 中 `type` 只允许：

- `app`
- `card`
- `layout`
- `module`
- `theme`

## 3. 插件包结构

```text
my-plugin/
├── manifest.yaml
├── README.md
├── LICENSE
├── assets/
├── src/
├── locales/
└── dist/
```

## 4. manifest.yaml 规范（基线）

### 4.1 顶层字段

```yaml
schemaVersion: "1.0.0"
id: "publisher.plugin-name"
name: "i18n.plugin.600001"
version: "1.0.0"
type: "app"
publisher: "publisher"
description: "i18n.plugin.600002"
icon: "assets/icon.png"
entry:
  main: "dist/index.html"
permissions:
  - file.read
  - file.write
compatibility:
  host: ">=1.0.0"
  sdk: ">=1.0.0"
  platform:
    - win32
    - darwin
    - linux
dependencies:
  plugins:
    - id: "chips-official.default-theme"
      version: ">=1.0.0"
locales:
  default: "zh-CN"
  supported:
    - zh-CN
    - en-US
  path: "locales"
```

### 4.2 页面型模板布局契约（阶段06）

- 页面型插件（`app/card/layout/module`）必须声明：
  - `ui.layout.owner=page`
  - `ui.layout.unit=cpx`
  - `ui.layout.baseWidth=1024`
  - `ui.layout.contract`
  - `ui.layout.minFunctionalSet`
- `theme` 插件豁免 `ui.layout`，仅受 `entry.tokens`、`entry.themeCss` 与 `manifest.theme` 字段约束。

### 4.3 禁止字段

以下旧字段不再使用：

- `protocol_version`
- `min_core_version`
- `resource.write`（作为示例权限）

## 5. 按类型入口要求

- `app`: `entry.main`
- `card`: `entry.renderer` + `entry.editor`
- `layout`: `entry.renderer` + 可选 `entry.editor`
- `module`: `entry.module` 或 `entry.iframe`（可同时提供）
- `theme`: `entry.tokens` + `entry.themeCss` + `theme`

## 6. 权限声明

采用 `namespace.action` 格式，按最小权限声明：

```yaml
permissions:
  - file.read
  - file.write
  - card.read
  - card.write
  - plugin.read
  - dialog.open
  - dialog.save
  - shell.openExternal
```

说明：

- `chips.invoke` 的最终权限判断由 Host 路由元数据执行。
- 高风险权限（如 `shell.openExternal`）应谨慎申请。

## 7. 开发红线

- 禁止在插件代码中 `require('fs')` / `require('electron')`。
- 禁止绕过 Bridge 直接访问系统文件与窗口。
- 禁止在组件中写死用户可见文本与视觉样式。

## 8. 验收要求

- `manifest.yaml` 字段与类型通过校验。
- 权限声明与实际调用一致。
- 运行时仅通过 `window.chips.*` 调用系统能力。
- 文档与示例不引用未导出 SDK API。
