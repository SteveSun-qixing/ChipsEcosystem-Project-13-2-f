# 多语言系统规范

**版本**: 1.2.0  
**最后更新**: 2026-02-26

---

## 1. 系统概述

### 1.1 设计理念

薯片生态采用**系统统一管理 + 分层继承覆盖**的多语言方案，所有文本内容都通过统一多语言系统管理和显示。核心原则：

- **零硬编码**：代码中不允许出现任何明文字符串
- **系统接管**：所有文本显示由系统统一管理
- **编码替换**：开发时使用词汇key，打包时替换为系统编码
- **插件共享**：插件使用系统词汇表，自动支持多语言
- **逐级继承**：词汇查找优先级固定为“插件级 > 应用级 > 生态级”

### 1.2 架构设计

```
┌───────────────────────────────────────────────────────────┐
│                        用户界面                            │
└──────────────────┬────────────────────────────────────────┘
                   │ 显示文本
┌──────────────────▼────────────────────────────────────────┐
│                   多语言解析器                             │
│   1) 插件级词典 → 2) 应用级词典 → 3) 生态级词典          │
│   命中后执行变量插值和格式化                              │
└──────────────────┬────────────────────────────────────────┘
                   │ 读取编码对应文本
┌──────────────────▼────────────────────────────────────────┐
│                  分层词汇存储                             │
│   plugin locale packs / app locale packs / system packs  │
└───────────────────────────────────────────────────────────┘
                   ▲
                   │ 打包时替换
┌──────────────────┴────────────────────────────────────────┐
│               开发者词汇表（开发时）                      │
│   key → 文本（开发阶段临时使用）                          │
└───────────────────────────────────────────────────────────┘
```

### 1.3 Host 路由冻结口径（阶段05）

- 对外动作固定：`i18n.list/getCurrent/setCurrent/translate/translateBatch/registerVocabulary`
- 旧动作 `getCurrentLanguage/setLanguage` 不再作为外部主路径
- 写动作统一权限：`i18n.write`；读动作统一权限：`i18n.read`

## 2. 词汇编码系统

### 2.1 编码规范

所有文本使用唯一的词汇编码标识：

**格式**: `i18n.{分类}.{序号}`

**示例**:
```
i18n.core.001    - 核心系统词汇
i18n.ui.001      - UI组件词汇
i18n.editor.001  - 编辑器词汇
i18n.card.001    - 卡片相关词汇
i18n.plugin.001  - 插件专用词汇（由插件注册）
```

### 2.2 编码分类

| 分类前缀 | 编码范围 | 说明 |
|---------|---------|------|
| `i18n.core.` | 000001-099999 | 薯片内核词汇 |
| `i18n.ui.` | 100000-199999 | UI组件通用词汇 |
| `i18n.editor.` | 200000-299999 | 编辑引擎词汇 |
| `i18n.viewer.` | 300000-399999 | 查看器词汇 |
| `i18n.card.` | 400000-499999 | 卡片类型词汇 |
| `i18n.box.` | 500000-599999 | 箱子布局词汇 |
| `i18n.plugin.` | 600000-899999 | 插件自定义词汇 |
| `i18n.user.` | 900000-999999 | 用户自定义词汇 |

### 2.3 编码分配

- **系统预留**：000001-599999 由系统预留和分配
- **插件注册**：600000-899999 插件可注册自己的词汇编码
- **用户自定义**：900000-999999 用户可添加自定义词汇

### 3.1 词汇表结构

```yaml
supported_languages:
  - "zh-CN"
  - "zh-TW"
  - "en-US"
  - "ja-JP"
  - "ko-KR"

vocabulary:
  # 核心系统词汇
  i18n.core.000001:
    zh-CN: "确定"
    zh-TW: "確定"
    en-US: "OK"
    ja-JP: "OK"
    ko-KR: "확인"
    
  i18n.core.000002:
    zh-CN: "取消"
    zh-TW: "取消"
    en-US: "Cancel"
    ja-JP: "キャンセル"
    ko-KR: "취소"
    
  i18n.core.000003:
    zh-CN: "保存"
    zh-TW: "儲存"
    en-US: "Save"
    ja-JP: "保存"
    ko-KR: "저장"
    
  # UI组件词汇
  i18n.ui.100001:
    zh-CN: "加载中..."
    zh-TW: "載入中..."
    en-US: "Loading..."
    ja-JP: "読み込み中..."
    ko-KR: "로딩 중..."
    
  # 支持变量插值
  i18n.ui.100002:
    zh-CN: "共 {count} 个文件"
    zh-TW: "共 {count} 個檔案"
    en-US: "{count} files"
    ja-JP: "{count} ファイル"
```

### 3.2 词汇表文件

系统词汇表分布在多个文件中：


打包时合并为 `system_i18n.yaml`。

## 4. 开发阶段词汇管理

### 4.1 开发者词汇表

开发时，开发者使用人类可读的key：

```yaml
# dev_i18n.yaml (开发阶段)
common:
  ok: "确定"
  cancel: "取消"
  save: "保存"
  delete: "删除"
  
ui:
  loading: "加载中..."
  file_count: "共 {count} 个文件"
  
editor:
  create_card: "创建卡片"
  add_content: "添加内容"
```

### 4.2 代码中使用

开发者在代码中使用 `t()` 函数：

```vue
<!-- Vue 3 示例 -->
<script setup lang="ts">
import { useI18n } from '@chips/sdk';

const { t } = useI18n();
</script>

<template>
  <div>
    <button>{{ t('common.ok') }}</button>
    <button>{{ t('common.cancel') }}</button>
    <p>{{ t('ui.file_count', { count: 10 }) }}</p>
  </div>
</template>
```

```typescript
// TypeScript 工具层示例
import { t } from '@chips/sdk/i18n';

export function buildSuccessMessage(count: number): string {
  return t('ui.file_count', { count });
}
```

### 4.3 开发时翻译

开发阶段，系统提供临时翻译：

```javascript
// 开发模式：直接读取 dev_i18n.yaml
t('common.ok')  // → "确定"

// 多语言测试
setCurrent('en-US');
t('common.ok')  // → "OK" (如果dev_i18n.yaml中有英文)
```

## 5. 打包时编码替换

### 5.1 替换流程

打包时，构建工具自动进行编码替换：

```
1. 扫描代码中的 t('key') 调用
2. 在系统词汇表中查找匹配的编码
3. 替换 key 为 编码
4. 未匹配的key，生成新的插件编码
```

**替换示例**:

```javascript
// 开发代码
<button>{t('common.ok')}</button>

// 打包后
<button>{t('i18n.core.000001')}</button>
```

### 5.2 匹配规则

系统维护 **key → 编码** 的映射表：

```yaml
# key_mapping.yaml (系统维护)
mapping:
  "common.ok": "i18n.core.000001"
  "common.cancel": "i18n.core.000002"
  "common.save": "i18n.core.000003"
  "ui.loading": "i18n.ui.100001"
  "ui.file_count": "i18n.ui.100002"
```

### 5.3 未匹配处理

如果开发者使用了系统词汇表中不存在的key：

**选项1: 自动生成插件编码**
```javascript
// 开发代码
t('my_plugin.custom_text')

// 打包后自动分配编码
t('i18n.plugin.600001')

// 同时生成插件词汇表
// plugin_i18n.yaml
i18n.plugin.600001:
  zh-CN: "自定义文本"  # 从 dev_i18n.yaml 提取
```

**选项2: 构建警告**
```
Warning: Key 'my_plugin.custom_text' not found in system vocabulary.
Generated code: i18n.plugin.600001
Please submit to system vocabulary if this is common text.
```

## 6. 运行时多语言系统

### 6.1 语言切换

用户可以切换界面语言：

```javascript
// 系统API
import { setCurrent, getCurrent } from '@chips/i18n';

// 切换到英文
setCurrent('en-US');

// 获取当前语言
const lang = getCurrent(); // 'en-US'

// 自动刷新所有界面文本
```

### 6.1.1 语言切换全链路事件流

语言切换后，系统通过事件链路通知所有渲染进程和卡片 iframe：

```
i18n.setCurrent(locale)
  → Host i18n-service 更新当前语言
    → emitLanguageChanged()              // 发布 language.changed 事件（propagate: true）
      → 内核事件总线
        → event-forwarder（通配符 '*' 订阅）
          → 所有渲染进程窗口（IPC）
            → SDK onLanguageChanged 回调
              → viewer/editor 更新本地 locale 状态
                → card-runtime protocol 广播 language-change 消息
                  → 每个卡片 iframe 接收并切换语言
```

关键点：
1. `language.changed` 事件通过 `propagate: true` 传播到所有渲染进程
2. 卡片 iframe 通过 `language-change` postMessage 接收语言变更，无需自行订阅 Bridge 事件
3. 词汇表数据可通过 `language-change` 消息的 `vocabulary` 字段一并下发

### 6.2 文本显示引擎

```javascript
// 伪代码
function t(code, vars = {}) {
  // 1. 获取当前语言
  const lang = getCurrent();

  // 2. 按优先级查找：插件级 > 应用级 > 生态级
  let text = pluginVocabulary[code]?.[lang];
  if (!text) {
    text = appVocabulary[code]?.[lang];
  }
  if (!text) {
    text = systemVocabulary[code]?.[lang];
  }

  // 3. 如果都没有，返回编码本身（开发模式）
  if (!text) {
    console.warn(`Missing translation for: ${code}`);
    return code;
  }

  // 4. 处理变量插值
  return interpolate(text, vars);
}

function interpolate(text, vars) {
  // 替换 {count} → vars.count
  return text.replace(/\{(\w+)\}/g, (match, key) => {
    return vars[key] !== undefined ? vars[key] : match;
  });
}
```

### 6.3 词汇表加载

```javascript
// 系统初始化时加载
async function initI18n() {
  // 1. 加载系统词汇表
  const systemVocab = await loadVocabulary('system_i18n.yaml');

  // 2. 加载当前应用词汇表
  const appVocab = await loadVocabulary('app_i18n.yaml');

  // 3. 加载所有插件词汇表
  const pluginVocabMap = {};
  const plugins = getInstalledPlugins();
  for (const plugin of plugins) {
    const pluginVocab = await loadVocabulary(
      `${plugin.path}/i18n.yaml`
    );
    pluginVocabMap[plugin.id] = pluginVocab;
  }

  // 4. 初始化显示引擎
  initDisplayEngine({
    system: systemVocab,
    app: appVocab,
    plugin: pluginVocabMap
  });
}
```

## 7. 分层词汇支持

### 7.1 应用级词汇表

应用插件可以定义 `app_i18n.yaml`，承载“应用内通用但未进入生态词典”的词汇：

```yaml
scope: app
app_id: chips-official.editor
vocabulary:
  i18n.app.700001:
    zh-CN: "工作区"
    en-US: "Workspace"
```

### 7.2 插件级词汇表

插件可以定义自己的 `i18n.yaml`，用于覆盖或补充词汇：

```yaml
scope: plugin
plugin_id: my-awesome-plugin
vocabulary:
  i18n.plugin.600001:
    zh-CN: "我的自定义按钮"
    en-US: "My Custom Button"
  i18n.plugin.600002:
    zh-CN: "插件设置"
    en-US: "Plugin Settings"
```

### 7.3 插件开发

```typescript
import { t } from '@chips/sdk/i18n';

const buttonLabel = t('i18n.plugin.600001');
const title = t('i18n.plugin.600002');
```

### 7.4 覆盖规则

同一编码在多层存在时，按以下顺序取值：

1. 插件级词汇  
2. 应用级词汇  
3. 生态级词汇

### 7.5 编码申请

插件开发者可以申请系统词汇编码：

1. 插件首次打包时自动生成临时编码（600000段）
2. 如果是通用词汇，可提交到系统词汇表
3. 系统审核后分配正式编码（100000-500000段）
4. 下次更新时使用正式编码

## 8. 信息配置表

### 8.1 零硬编码原则

代码中不允许出现任何明文字符串，所有文本必须通过配置表管理：

**❌ 错误示例**:
```javascript
// 硬编码 - 禁止
<button>确定</button>
<p>加载中...</p>
alert("保存成功！");
```

**✅ 正确示例**:
```javascript
// 使用编码
<button>{t('i18n.core.000001')}</button>
<p>{t('i18n.ui.100001')}</p>
alert(t('i18n.ui.100050'));
```

### 8.2 配置表类型

系统提供多种配置表：

**1. 文本配置表** (`i18n.yaml`)
- 所有用户可见的文本
- 按钮、标签、提示信息等

**2. 消息配置表** (`messages.yaml`)
```yaml
# 错误消息、提示消息
messages:
  error:
    file_not_found: "i18n.core.010001"
    network_error: "i18n.core.010002"
  success:
    save_success: "i18n.core.011001"
```

**3. 日志配置表** (`logs.yaml`)
```yaml
# 日志消息（开发者可见）
logs:
  info:
    module_loaded: "i18n.log.001"
  error:
    init_failed: "i18n.log.002"
```

### 8.3 配置表管理

```javascript
// 配置表管理器
class ConfigManager {
  // 加载所有配置表
  loadConfigs() {
    this.i18n = loadYaml('i18n.yaml');
    this.messages = loadYaml('messages.yaml');
    this.logs = loadYaml('logs.yaml');
  }
  
  // 获取配置值
  get(category, key) {
    return this[category][key];
  }
}

// 使用
const config = new ConfigManager();
const errorCode = config.get('messages', 'error.file_not_found');
console.log(t(errorCode)); // "文件未找到"
```

## 10. 最佳实践

### 10.1 命名规范

**开发阶段的key命名**:
```yaml
# 良好的命名
common.ok              # 明确的含义
ui.button.save         # 层次清晰
editor.toolbar.undo    # 具体的上下文

# 不好的命名
btn1                   # 无意义
text                   # 太泛化
ok_button_text         # 冗余
```

### 10.2 避免重复

优先使用系统已有词汇，避免重复定义：

```yaml
# ❌ 不要重复定义相同含义的词汇
my_plugin:
  confirm: "确定"      # 系统已有 common.ok
  
# ✅ 使用系统词汇
my_plugin:
  confirm: "@common.ok"  # 引用系统词汇
```

### 10.3 上下文相关

某些词汇在不同上下文有不同含义：

```yaml
# 使用完整的key表示上下文
editor.save: "保存卡片"
viewer.save: "保存查看设置"
toolbar.save: "保存"

# 而不是
save: "保存"  # 太泛化
```

### 10.4 变量使用

```yaml
# 清晰的变量名
file_count: "共 {count} 个文件"
user_greeting: "你好，{username}!"

# 避免模糊的变量
message: "{a} {b} {c}"  # 不清楚含义
```

## 11. 性能优化

### 11.1 懒加载

按需加载词汇表：

```javascript
// 只加载当前语言
const vocab = await loadVocabulary(currentLang);

// 插件词汇表在插件加载时才加载
```

### 11.2 缓存策略

```javascript
// 内存缓存
const cache = new Map();

function t(code) {
  const key = `${code}_${currentLang}`;
  if (cache.has(key)) {
    return cache.get(key);
  }
  
  const text = lookupVocabulary(code);
  cache.set(key, text);
  return text;
}
```

### 11.3 预编译

打包时预编译常用语言：

```javascript
// 预编译中文和英文
const precompiled = {
  'zh-CN': compileVocabulary('zh-CN'),
  'en-US': compileVocabulary('en-US'),
};

// 运行时直接使用，无需解析YAML
```

## 13. 安全考虑

### 13.1 注入防护

防止恶意注入：

```javascript
// ❌ 不安全
const text = t('i18n.ui.100001', { 
  html: '<script>alert("xss")</script>' 
});

// ✅ 自动转义
function t(code, vars) {
  const text = lookup(code);
  // 转义所有变量
  const safeVars = Object.fromEntries(
    Object.entries(vars).map(([k, v]) => [k, escape(v)])
  );
  return interpolate(text, safeVars);
}
```

### 13.2 编码验证

只接受合法的编码格式：

```javascript
const VALID_CODE = /^i18n\.[a-z]+\.\d{6}$/;

function t(code) {
  if (!VALID_CODE.test(code)) {
    throw new Error(`Invalid i18n code: ${code}`);
  }
  // ...
}
```

## 14. 总结

薯片生态的多语言系统特点：

✅ **零硬编码** - 所有文本通过编码管理

这套系统确保了整个生态的多语言支持统一、高效、易维护。
