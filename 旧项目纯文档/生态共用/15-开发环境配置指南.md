# 开发环境配置指南

---

## 一、环境准备

### 1.1 必需工具

| 工具 | 版本要求 | 用途 |
|------|---------|------|
| Node.js | ≥ 18.x | 主机和插件的运行环境 |
| pnpm | ≥ 8.x | 包管理器 |
| Git | 最新 | 版本控制 |

> **注意**：内核使用 TypeScript 实现，不再需要 Rust 工具链。

### 1.2 安装依赖

```bash
# 克隆仓库（含子模块）
git clone --recurse-submodules <仓库URL>
cd Project-12

# 安装所有工作区依赖
pnpm install

# 构建薯片主机（首次需要）
cd Chips-Host && pnpm build
```

## 二、工作目录配置

### 2.1 开发工作目录

生态内的开发测试工作目录：

```
ProductFinishedProductTestingSpace/
├── TestWorkspace/           ← 生态内工作文件夹（开发测试用）
│   ├── <cardId>/            ← 测试卡片
│   └── ...
└── ExternalEnvironment/     ← 与外界交互的文件夹（导入导出用）
    ├── *.card               ← 导入的卡片文件
    └── html-export-*/       ← 导出的 HTML 文件
```

### 2.2 配置工作目录

在开发时，需要配置工作目录路径：

```typescript
// SDK 初始化时配置
const sdk = new ChipsSDK({
  workspacePath: './ProductFinishedProductTestingSpace/TestWorkspace',
  externalPath: './ProductFinishedProductTestingSpace/ExternalEnvironment'
});
```

## 三、开发模式配置

### 3.1 Foundation 开发模式

```bash
# 启动 Foundation 开发服务
cd Chips-Foundation
pnpm run dev
```

Foundation 开发模式会：
- 启动 Electron 开发环境
- 启用热重载
- 开启调试工具

### 3.2 Editor/Viewer 开发模式

```bash
# 先确保 Foundation 已构建
cd Chips-Foundation && pnpm run build

# 启动 Editor 开发服务
cd Chips-Editor && pnpm run dev

# 或启动 Viewer 开发服务
cd Chips-Viewer && pnpm run dev
```

### 3.3 使用启动脚本

```bash
# 启动 Foundation
./脚本库/start-foundation.sh

# 启动 Editor
./脚本库/start-editor.sh

# 启动 Viewer
./脚本库/start-viewer.sh
```

## 四、配置文件

### 4.1 根目录配置

```ini
# .npmrc - pnpm 配置
shamefully-hoist=true
auto-install-peers=true
strict-peer-dependencies=false
```

### 4.2 TypeScript 配置

各模块的 `tsconfig.json` 继承共享配置：

```json
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "ESNext",
    "moduleResolution": "bundler",
    "strict": true,
    "declaration": true,
    "outDir": "./dist"
  }
}
```

### 4.3 模块配置示例

```json
// package.json
{
  "name": "@chips/my-module",
  "version": "1.0.0",
  "type": "module",
  "main": "./dist/index.cjs",
  "module": "./dist/index.js",
  "types": "./dist/index.d.ts",
  "scripts": {
    "dev": "tsup --watch",
    "build": "tsup",
    "test": "vitest"
  },
  "peerDependencies": {
    "@chips/foundation": "^1.0.0"
  }
}
```

## 五、插件开发配置

### 5.1 创建插件项目

```bash
# 在 BasicCardPlugin 或其他插件目录下
mkdir MyPlugin && cd MyPlugin
pnpm init
```

### 5.2 插件配置

```json
// package.json
{
  "name": "@chips/my-plugin",
  "chips": {
    "type": "plugin",
    "category": "card"
  },
  "peerDependencies": {
    "@chips/sdk": "^1.0.0"
  }
}
```

### 5.3 插件入口

```typescript
// src/index.ts
import type { PluginRegistration } from '@chips/sdk';

export const plugin: PluginRegistration = {
  id: 'my-plugin',
  metadata: {
    name: 'My Plugin',
    version: '1.0.0'
  },
  activate: (ctx) => {
    // 插件激活逻辑
  },
  deactivate: () => {
    // 插件停用逻辑
  }
};

export default plugin;
```

## 六、调试配置

### 6.1 VS Code 配置

```json
// .vscode/launch.json
{
  "configurations": [
    {
      "name": "Debug Electron Main",
      "type": "node",
      "request": "launch",
      "cwd": "${workspaceFolder}/Chips-Foundation",
      "runtimeExecutable": "${workspaceFolder}/Chips-Foundation/node_modules/.bin/electron",
      "args": [".", "--inspect=5858"]
    }
  ]
}
```

### 6.2 开启日志

```typescript
// 开发环境开启详细日志
const sdk = new ChipsSDK({
  logger: {
    level: 'debug',
    transports: ['console']
  }
});
```

cd ProductFinishedProductTestingSpace

## 八、环境变量

| 变量名 | 说明 | 默认值 |
|--------|------|--------|
| `CHIPS_ENV` | 运行环境 | `development` |
| `CHIPS_LOG_LEVEL` | 日志级别 | `info` |
| `CHIPS_WORKSPACE` | 工作目录 | `./TestWorkspace` |
| `CHIPS_CORE_PORT` | 内核通信端口 | `9527` |
