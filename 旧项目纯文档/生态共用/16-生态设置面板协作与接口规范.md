# 生态设置面板协作与接口规范

> 说明（2026-02-19）：本文件保留历史协作描述；涉及“后安装控制面 API/CLI 一致性”与“安装器/设置面板职责边界”时，以 `26-后安装控制面API与CLI一致性规范.md` 与 `27-安装器与设置面板职责分层规范.md` 为准。

**版本**: 1.0.2  
**更新时间**: 2026-02-26  
**状态**: 草案

---

## 1. 概述

本规范定义生态设置面板（Chips Ecosystem Settings）在薯片生态中的协作边界、对接接口和联动机制。设置面板现行主路径使用 React + TypeScript 开发，作为薯片主机的标准应用插件运行，通过 Bridge API（`window.chips.*`）与底层通信。历史 Vue 实现已归档到 `chips-app-settings/src/archive/legacy-vue/`，不参与主路径构建与测试。

生态设置面板的定位是“设备侧底层环境管理入口”，负责安装、治理和系统级配置，不承担应用层内容编辑职责。

---

## 2. 协作矩阵

| 协作对象 | 关系定位 | 必须遵循 |
|---|---|---|
| 薯片内核 | 中心路由与系统状态入口 | 所有操作通过 Bridge API 走内核路由 |
| 基础服务层 | 通用基础能力提供方 | 通过 Bridge API 调用标准服务，不直连内部模块 |
| Chips-SDK | 设置面板主要开发接口层 | 统一使用 SDK 封装的 Bridge API 能力 |
| 薯片组件库（无头组件） | UI 与交互能力来源 | 仅用于界面，不承载业务规则，样式由主题注入 |
| 编辑引擎 / 查看器等应用插件 | 被治理的应用层插件 | 通过事件与状态接口联动，不直接耦合 |
| 插件与套装仓库 | 安装与更新对象 | 通过清单协议管理生命周期 |

---

## 3. 接口调用规范

### 3.1 通信原则

1. 禁止直连底层服务，必须通过 Bridge API（`window.chips.invoke(namespace, action, params)`）或 SDK 封装调用。
2. 所有请求通过统一的 Bridge → IPC → 内核路由器路径转发。
3. 所有请求必须具备超时控制和标准错误码返回。
4. settings 主路径实现调用链固定为：`UI -> ecosystem-settings-service -> runtime-gateway -> SDK Runtime Client/Hooks`，不允许页面域绕过 service 层直连 transport。

### 3.2 推荐服务清单

| 服务名 | 说明 | 权限 |
|---|---|---|
| `runtime.health` | 运行时健康检查 | `runtime.read` |
| `runtime.status` | 运行时状态读取 | `runtime.read` |
| `plugin.list` | 插件列表查询 | `plugin.read` |
| `plugin.install` | 插件安装 | `plugin.install` |
| `config.get` | 配置读取 | `config.read` |
| `config.set` | 配置写入 | `config.write` |
| `log.query` | 日志查询 | `log.read` |
| `event.subscribe` | 事件订阅 | `event.subscribe` |

> 兼容说明：历史 `system.status` 可作为兼容别名保留，但新开发与联调以 `runtime.*` 为准。

### 3.3 扩展服务命名约束

生态设置面板新增服务应使用专有命名空间：

- `ecosystem.bootstrap.*`
- `ecosystem.runtime.*`
- `ecosystem.workspace.*`
- `ecosystem.diagnostic.*`

## 4. 与其他软件的协作机制

### 4.1 启停联动

- 生态设置面板对底层环境执行启动、停止、重启时，必须发布标准事件。
- Editor、Viewer 等应用通过事件总线感知状态变化并更新自身状态。

### 4.2 配置联动

- 全局语言、主题、文件策略变更后，必须通过事件广播通知生态应用。
- 应用收到事件后应执行“拉取最新配置 -> 校验 -> 局部刷新”流程。

### 4.3 插件联动

- 插件安装、禁用、升级完成后，必须写入统一状态并广播变更事件。
- 依赖该插件的应用不可直接猜测状态，必须通过标准接口查询。

---

## 5. 禁止事项

1. 禁止生态设置面板直接访问其他应用私有存储。
2. 禁止生态设置面板调用 Foundation 未公开内部 API。
3. 禁止绕过权限模型执行高风险操作（卸载核心组件、修改关键路径）。
4. 禁止无日志记录的关键配置变更。

---

## 6. 版本与演进

1. 生态设置面板相关服务协议变更时，必须更新本规范版本号。
2. 若新增跨模块字段，必须在变更记录中注明兼容影响。
3. 本规范正式版发布前，需与 Core / SDK / Foundation 三方联合评审。
