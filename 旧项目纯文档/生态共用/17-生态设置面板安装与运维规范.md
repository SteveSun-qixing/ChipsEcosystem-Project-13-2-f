# 生态设置面板安装与运维规范

> 说明（2026-02-19）：本文件中的安装流程相关描述已由阶段 8 安装器文档接管；设置面板职责边界以 `27-安装器与设置面板职责分层规范.md` 为准。

**版本**: 1.0.1  
**更新时间**: 2026-02-26  
**状态**: 草案

---

## 1. 目标

本规范定义生态设置面板（Chips Ecosystem Settings）的插件管理流程、运行维护流程和故障恢复要求，确保用户设备上的薯片生态底层环境可维护、可运行、可诊断、可恢复。设置面板是薯片主机的标准应用插件（现行主路径为 React Web 应用，历史 Vue 实现已归档到 `chips-app-settings/src/archive/legacy-vue/`）。

---

## 2. 安装治理范围

生态设置面板负责以下对象的安装和管理：

1. 薯片主机（Chips Host）更新
2. 基础卡片必装套装（卡片插件）
3. 箱子布局必装套装（布局插件）
4. 共享功能模块必装套装（模块插件）
5. 默认主题包（主题插件）
6. 其他必要基础插件

> **注意**：薯片内核和基础服务层编译在主机中，随主机更新。SDK 和组件库是开发依赖，不需要在用户设备上单独安装。

---

## 3. 安装顺序规范

### 3.1 标准顺序

安装顺序必须遵循：

`Core -> Foundation -> SDK -> ComponentLibrary -> 必装套装 -> 可选插件`

### 3.2 顺序约束

1. 上一层未安装成功，不得继续下一层安装。
2. 任一步骤失败，必须停止后续安装并进入恢复流程。
3. 每一步骤结束后必须记录任务快照和结果状态。

---

## 4. 下载源与导入包规范

### 4.1 下载源

下载源至少应包含以下信息：

- `source_id`
- `base_url`
- `supported_platforms`
- `supported_arch`
- `checksums`
- `signature`

### 4.2 手动导入

手动导入包必须执行：

1. 完整性校验（哈希）
2. 签名校验
3. 平台与架构匹配校验

未通过任一校验不得进入安装执行阶段。

---

## 5. 任务状态机规范

安装与维护任务统一使用状态机：

`PENDING -> PRECHECK -> RUNNING -> VERIFYING -> SUCCEEDED | FAILED | ROLLED_BACK`

状态切换必须记录以下字段：

- `task_id`
- `stage`
- `timestamp`
- `operator`
- `error_code`（如有）

---

## 6. 健康检查规范

### 6.1 必检项

安装完成后必须执行以下检查：

1. Core 进程可用性
2. 路由调用可用性
3. Foundation 关键模块可用性
4. SDK 基础调用可用性
5. 必装套装完整性
6. 工作区目录读写性

### 6.2 结果处理

- 全部通过：进入设置面板正常模式。
- 部分失败：进入告警模式并给出修复建议。
- 关键失败：进入修复模式，阻断高风险操作。

---

## 7. 运行维护规范

### 7.1 进程管理

生态设置面板必须支持：

- 启动、停止、重启
- 运行状态查看
- 异常状态告警

### 7.2 日志与报告

1. 关键操作必须记录结构化日志。
2. 错误必须带统一错误码。
3. 支持导出诊断报告（状态、日志、任务轨迹、建议动作）。

---

## 8. 恢复与回滚规范

1. 安装失败必须支持“从失败点重试”。
2. 高风险升级必须支持回滚到上一个稳定快照。
3. 回滚后必须自动执行一次健康检查。

---

## 9. 与生态应用的联动要求

1. 运行状态变化必须发布标准事件。
2. 语言/主题/插件关键设置变更必须通知 Editor、Viewer 等应用刷新状态。
3. 文件交换策略变更后必须同步到相关文件处理模块。

---

## 10. 安全要求

1. 高权限安装与卸载操作必须进行权限确认。
2. 核心组件不得被无条件卸载。
3. 日志与报告导出默认脱敏敏感信息。
