## 2. 开发工具全景

| 层级 | 工具/系统 | 作用 | 备注 |
|---|---|---|---|
| 运行时 | Chips Host | 生态唯一宿主，加载所有插件 | Electron + TypeScript |
| 开发 CLI | `@chips/develop-cli`（`chipsd`） | 插件脚手架、调试、构建、打包、校验、发布 | 面向开发者 |
| SDK | `@chips/sdk` | Bridge API 类型安全封装 | 插件侧标准调用入口 |
| 组件库 | `@chips/component-library`（Zag.js + Ark UI） | 无头组件能力 | 零样式，主题注入视觉 |
| 主题构建 | Style Dictionary | Token 构建与主题产物生成 | 主题插件标准管线 |
| 测试 | vitest | 单元测试与模块测试 | 必须覆盖正常/异常/边界 |
| 包管理 | pnpm | 依赖与脚本管理 | 生态统一包管理器 |
| 工作区管理 | git worktree | 多任务并行开发与分支隔离 | 禁止混改 |
| 联调验证 | 生态测试器（ecosystem tester） | Host 联调与全链路验证 | 阶段五关键工具 |

## 3. `chipsd` 命令基线

`chipsd` 是插件开发主入口。标准主流程命令如下：

```bash
# 安装
pnpm add -g @chips/develop-cli

# 创建项目
chipsd create my-plugin

# 开发调试（Mock Bridge）
chipsd dev

# 联调调试（连接 Host）
chipsd dev --host

# 构建与打包
chipsd build
chipsd pack

# 合规校验（manifest/结构/规则）
chipsd validate

# 发布链路
chipsd login
chipsd publish
```

多语言相关命令（按需）：

```bash
chipsd i18n extract
chipsd i18n check
chipsd i18n generate --lang en-US
chipsd i18n sync
```

阶段06模板与 CLI 门禁命令口径：

```bash
# 五模板校验示例
(cd chips-template-app && pnpm exec chipsd validate)
(cd chips-template-theme && pnpm exec chipsd validate)

# CLI 仓库门禁（不使用 workspace filter）
```

## 4. 标准开发方式（必须遵守）

### 4.1 分支与工作区

1. 在目标子仓库 `Develop` 分支基础上创建任务分支  
2. 使用 `git worktree` 创建独立工作区  
3. 任务完成后回合并到 `Develop`

### 4.2 开发与实现

1. 优先使用 `chipsd create` 标准模板  
2. 插件侧只通过 `window.chips.*` / SDK 调用能力  
3. 服务间调用必须走内核路由，禁止主进程服务直接 import 互调  
4. 文本走多语言系统，禁止硬编码  
5. 组件保持零样式，视觉由主题注入  
6. TypeScript strict，禁止无说明的 `any`

### 4.3 验证与交付

1. `vitest` 全量通过  
2. `chipsd validate` 通过  
3. `chipsd build && chipsd pack` 产出 `.cpk`  
4. 在 Host 联调场景验证关键链路（启动、Bridge、权限、事件、安装/卸载）  
5. 提交变更清单、测试结果清单、风险说明

## 5. 两种调试方式

### A. 独立调试（Mock Bridge）

- 命令：`chipsd dev`
- 适用：UI 开发、基础交互、快速迭代
- 特点：无需 Host，速度快

### B. 宿主联调（真实 Bridge）

- 命令：`chipsd dev --host`（或等价的 Host 接入调试流程）
- 适用：权限、路由、事件总线、插件生命周期、安装链路
- 特点：结果最接近真实交付环境

> 发布前必须完成宿主联调，不可只依赖 Mock 模式。

## 6. 常见失败点（重点）

### 6.1 Manifest 校验失败

高频问题：

- `schemaVersion` 非 `1.0.0`
- `permissions` 格式不合法
- 缺失 `compatibility` 字段
- `ui.layout` 缺失（页面型模板）；`theme` 模板豁免

处理原则：先修清单，再重跑 `chipsd validate`。

### 6.2 Bridge 不可用

高频原因：

- 仅在独立模式启动（未接入 Host）
- 插件误用 Node/Electron 原生 API
- 预加载/桥接注入链路未生效

处理原则：切换 Host 联调并核查 `window.chips.*` 注入状态。

## 7. 交付质量门禁（发布前）

- 架构门禁：中心路由、Bridge 隔离、单一 Host 模式  
- 规范门禁：多语言、零样式、命名规范、错误格式统一  
- 测试门禁：服务/组件测试覆盖达标且全通过  
- 打包门禁：`.cpk` 可安装、可启动、可卸载  
- 文档门禁：变更说明、接口变更、迁移说明同步完成
