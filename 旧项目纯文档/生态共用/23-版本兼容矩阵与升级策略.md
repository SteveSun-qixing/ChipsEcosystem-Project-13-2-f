# 版本兼容矩阵与升级策略

## 1. 组件兼容矩阵（阶段 5.16）

| 组件 | 最低版本 | 说明 |
| --- | --- | --- |
| Chips Host | `>=1.0.0` | 必须支持 `/dev/health` 与 `/dev/router/invoke` |
| chips-develop-cli | `>=1.1.0` | 提供 `chipsd ecosystem` 命令组 |
| chips-app-ecosystem-tester | `>=1.0.0` | 保留 `pnpm test:ecosystem` 兼容入口 |

## 2. 升级策略

1. 先升级 Host，再升级 CLI，最后升级测试器兼容脚本。
2. 每次升级后执行 `chipsd ecosystem test --profile smoke`。
3. 大版本升级必须执行 `chipsd ecosystem test --profile full` 并归档 JSON/JUnit 报告。

## 3. 质量门禁策略

默认门禁：

- 阻断套件：`P01,P02,P03,P06`
- 允许波动：`P10,P12`

配置文件：`chipsd.ecosystem-gate.json`

```json
{
  "blockingSuites": ["P01", "P02", "P03", "P06"],
  "allowedFailures": ["P10", "P12"]
}
```

## 4. changed profile 映射

`chipsd ecosystem test --profile changed` 使用 Git diff + mapping 规则自动筛选必跑套件。

- `Chips-Host/**` -> `P01,P02,P03,P04,P05,P06,P09,P11,P12`
- `Chips-SDK/**` -> `P03,P08,P11`
- `chips-theme-default/**` -> `P04,P07`
- `chips-app-ecosystem-tester/**` -> `P01-P12`

## 5. 发布前检查清单

1. `chipsd ecosystem test --profile smoke`
2. `chipsd ecosystem test --profile changed`
3.（发布版本）`chipsd ecosystem test --profile full`
4. 报告归档：`report.json` + `report.junit.xml`
