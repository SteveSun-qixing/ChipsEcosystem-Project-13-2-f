# 快速入门

## 1.1 5 分钟创建你的第一个薯片插件

本节将带你快速创建一个简单的薯片插件，让你对插件开发流程有一个直观的认识。

### 环境准备

在开始之前，请确保你的开发环境满足以下要求：

- **Node.js**: 18.0.0 或更高版本
- **pnpm**: 8.0.0 或更高版本（推荐使用 pnpm 作为包管理器）
- **Chips Develop CLI**: 薯片插件开发命令行工具

安装 Chips Develop CLI：

```bash
# 全局安装 Chips Develop CLI
pnpm add -g @chips/develop-cli

# 验证安装
chipsd --version
```

### 创建插件项目

使用 `chipsd create` 命令创建一个新的插件项目：

```bash
# 创建一个简单的应用插件
chipsd create my-first-plugin

# 交互式选择插件类型
? 选择插件类型:
  ❯ app-blank        空白应用插件
    app-single-card  单卡片应用
    card-simple      简单卡片插件
    theme-full       完整主题包
    ...
```

选择 `app-blank` 创建一个空白应用插件。

### 项目结构

创建完成后，你会得到以下项目结构：

```
my-first-plugin/
├── manifest.yaml          # 插件清单（必需）
├── package.json           # npm 包配置
├── tsconfig.json          # TypeScript 配置
├── vite.config.ts         # Vite 构建配置
├── src/
│   ├── main.ts            # 应用入口
│   ├── App.vue            # 主组件
│   └── types.ts           # 类型定义
├── locales/               # 多语言文件
│   └── vocabulary.yaml    # 插件词汇表
├── assets/                # 静态资源
│   └── icon.png           # 插件图标
└── dist/                  # 构建输出（自动生成）
```

### 启动开发服务器

```bash
# 进入项目目录
cd my-first-plugin

# 安装依赖
pnpm install

# 启动开发服务器
chipsd dev
```

开发服务器启动后，会自动注入 Mock Bridge API，让你可以在没有薯片主机的情况下进行开发调试。

### 编写第一个功能

打开 `src/App.vue`，编写一个简单的功能：

```vue
<template>
  <div class="chips-app">
    <h1>{{ t('app.title') }}</h1>
    <button
      class="chips-button chips-button--primary"
      @click="handleClick"
    >
      {{ t('app.button') }}
    </button>
    <p v-if="message">{{ message }}</p>
  </div>
</template>

<script setup lang="ts">
import { ref } from 'vue';
import { useI18n } from '@chips/sdk';

const { t } = useI18n();
const message = ref('');

async function handleClick() {
  // 通过 Bridge API 调用系统服务
  const result = await window.chips.invoke('platform', 'getSystemInfo', {});
  message.value = `${t('app.system')}: ${result.os} ${result.version}`;
}
</script>
```

### 构建和打包

```bash
# 构建项目
chipsd build

# 打包为 .cpk 文件
chipsd pack
```

打包完成后，你会在项目根目录得到一个 `.cpk` 文件，这就是可以安装到薯片主机的插件包。

### 验证插件

```bash
# 验证插件包的完整性
chipsd validate ./my-first-plugin-1.0.0.cpk
```

恭喜！你已经成功创建了第一个薯片插件。

## 1.2 核心概念

在深入学习之前，你需要了解薯片生态的几个核心概念：

### 中心路由原则

薯片生态采用中心路由架构，所有服务间通信必须经过内核路由器：

```typescript
// ✅ 正确：通过 Bridge API 调用服务
const result = await window.chips.invoke('file', 'read', { path: '/path/to/file' });

// ❌ 错误：直接使用 Node.js API（插件中禁止）
const fs = require('fs');
const data = fs.readFileSync('/path/to/file');
```

### Bridge 隔离原则

插件运行在渲染进程中，只能通过 `window.chips.*` Bridge API 访问系统能力：

- 插件不能直接访问 Node.js API
- 插件不能直接访问 Electron API
- 插件不能直接操作文件系统
- 所有系统调用都通过 Bridge API 进行

### 零硬编码文本

所有面向用户的文本必须使用多语言系统：

```typescript
// ✅ 正确：使用多语言系统
const label = t('common.save');

// ❌ 错误：硬编码文本
const label = '保存';
```

### 零样式组件

组件库中的组件不包含任何视觉样式，样式由主题包提供：

```vue
<!-- ✅ 正确：使用 CSS 类名，样式由主题注入 -->
<button class="chips-button chips-button--primary">
  {{ t('common.save') }}
</button>

<!-- ❌ 错误：内联样式 -->
<button style="background: blue; color: white;">
  保存
</button>
```
