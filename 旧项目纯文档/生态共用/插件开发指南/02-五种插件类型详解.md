# 五种插件类型详解

薯片生态支持五种插件类型，每种类型有不同的用途和运行方式。

##  插件类型概览

| 类型 | type 值 | 运行容器 | 典型用途 |
|------|---------|----------|----------|
| 应用插件 | `app` | BrowserWindow | 编辑器、查看器、设置面板 |
| 卡片插件 | `card` | iframe | 基础卡片渲染与编辑 |
| 布局插件 | `layout` | iframe | 箱子布局渲染 |
| 模块插件 | `module` | iframe / ES Module | 视频播放器、Markdown 渲染器 |
| 主题插件 | `theme` | 无运行容器 | Token 与样式注入 |

##  应用插件（App）

应用插件是拥有独立窗口的完整应用，如编辑器、查看器、设置面板等。

### 特点

- 拥有独立的 BrowserWindow 窗口
- 可以控制窗口大小、位置、标题等
- 可以创建菜单、工具栏等
- 生命周期由薯片主机管理

### 项目结构

```
my-app-plugin/
├── manifest.yaml
├── package.json
├── src/
│   ├── main.ts              # 应用入口
│   ├── App.vue              # 主组件
│   ├── components/          # 组件目录
│   ├── composables/         # 组合式函数
│   └── types.ts             # 类型定义
├── locales/
│   └── vocabulary.yaml
├── assets/
│   └── icon.png
└── dist/
    └── index.html           # 构建输出
```

### manifest.yaml 示例

```yaml
schemaVersion: "1.0.0"
id: "my-publisher.my-app"
name: "i18n.plugin.600001"
version: "1.0.0"
type: "app"
publisher: "my-publisher"
description: "i18n.plugin.600002"
icon: "assets/icon.png"

entry:
  main: "dist/index.html"

window:
  width: 1200
  height: 800
  minWidth: 800
  minHeight: 600
  resizable: true
  frame: true

permissions:
  - "file.read"
  - "file.write"
  - "card.read"
  - "card.write"

compatibility:
  host: ">=1.0.0"
  platform:
    - "win32"
    - "darwin"
    - "linux"

locales:
  default: "zh-CN"
  supported:
    - "zh-CN"
    - "en-US"
  path: "locales"
```

### 完整示例

```vue
<!-- src/App.vue -->
<template>
  <div class="chips-app">
    <header class="chips-app__header">
      <h1>{{ t('app.title') }}</h1>
      <div class="chips-app__actions">
        <button
          class="chips-button chips-button--primary"
          @click="handleSave"
        >
          {{ t('common.save') }}
        </button>
      </div>
    </header>

    <main class="chips-app__content">
      <div v-if="loading" class="chips-loading">
        {{ t('common.loading') }}
      </div>
      <div v-else>
        <!-- 应用内容 -->
      </div>
    </main>
  </div>
</template>

<script setup lang="ts">
import { ref, onMounted } from 'vue';
import { useI18n, usePlugin } from '@chips/sdk';

const { t } = useI18n();
const { pluginInfo } = usePlugin();
const loading = ref(true);

onMounted(async () => {
  // 初始化应用
  try {
    // 获取插件信息
    const info = await window.chips.plugin.getSelf();
    console.log('Plugin loaded:', info.id);

    // 设置窗口标题
    await window.chips.window.setTitle(t('app.title'));
  } finally {
    loading.value = false;
  }
});

async function handleSave() {
  try {
    // 保存逻辑
    await window.chips.invoke('file', 'write', {
      path: '/workspace/data.json',
      content: JSON.stringify({ data: 'example' }),
      encoding: 'utf8'
    });

    // 显示成功提示
    await window.chips.dialog.showMessageBox({
      type: 'info',
      title: t('common.success'),
      message: t('message.save_success')
    });
  } catch (error) {
    // 显示错误提示
    await window.chips.dialog.showMessageBox({
      type: 'error',
      title: t('common.error'),
      message: t('message.save_failed')
    });
  }
}
</script>
```

##  卡片插件（Card）

卡片插件定义一种基础卡片类型的渲染和编辑组件，如视频卡片、图片卡片、富文本卡片等。

### 特点

- 以 iframe 方式加载在宿主应用中
- 包含渲染组件和编辑组件两个入口
- 通过 postMessage 与宿主通信
- 接收卡片配置数据进行渲染

### 项目结构

```
my-card-plugin/
├── manifest.yaml
├── package.json
├── src/
│   ├── renderer/            # 渲染组件
│   │   ├── index.ts
│   │   └── Renderer.vue
│   ├── editor/              # 编辑组件
│   │   ├── index.ts
│   │   └── Editor.vue
│   ├── bridge/              # iframe 通信层
│   │   ├── iframe-bridge.ts
│   │   └── message-types.ts
│   └── types.ts
├── locales/
│   └── vocabulary.yaml
└── dist/
    ├── renderer/
    │   └── index.html
    └── editor/
        └── index.html
```

### manifest.yaml 示例

```yaml
schemaVersion: "1.0.0"
id: "my-publisher.video-card"
name: "i18n.plugin.600010"
version: "1.0.0"
type: "card"
publisher: "my-publisher"

entry:
  renderer: "dist/renderer/index.html"
  editor: "dist/editor/index.html"

capabilities:
  cardType: "VideoCard"
  supportedFormats:
    - "mp4"
    - "webm"
    - "mov"

permissions:
  - "resource.fetch"

compatibility:
  host: ">=1.0.0"
```

### 渲染组件示例

```vue
<!-- src/renderer/Renderer.vue -->
<template>
  <div class="video-card-renderer">
    <video
      ref="videoRef"
      class="video-card__player"
      :src="videoUrl"
      :poster="config.cover_image"
      :controls="config.controls"
      :autoplay="config.autoplay"
      @timeupdate="handleTimeUpdate"
    />
  </div>
</template>

<script setup lang="ts">
import { ref, computed, onMounted, onUnmounted } from 'vue';
import { useCardBridge } from '../bridge/iframe-bridge';

interface VideoCardConfig {
  video_file: string;
  cover_image?: string;
  autoplay?: boolean;
  controls?: boolean;
}

const { config, resourceUrl, sendMessage, onMessage } = useCardBridge<VideoCardConfig>();
const videoRef = ref<HTMLVideoElement>();

// 解析视频 URL
const videoUrl = computed(() => {
  return resourceUrl(config.value.video_file);
});

// 处理播放进度更新
function handleTimeUpdate() {
  if (videoRef.value) {
    sendMessage('state-update', {
      currentTime: videoRef.value.currentTime,
      duration: videoRef.value.duration
    });
  }
}

// 监听宿主消息
onMounted(() => {
  onMessage('seek', (data: { time: number }) => {
    if (videoRef.value) {
      videoRef.value.currentTime = data.time;
    }
  });

  onMessage('play', () => {
    videoRef.value?.play();
  });

  onMessage('pause', () => {
    videoRef.value?.pause();
  });
});
</script>
```

### 编辑组件示例

```vue
<!-- src/editor/Editor.vue -->
<template>

        {{ t('editor.autoplay') }}
      </label>
    </div>

    <div class="editor-section">
      <label class="chips-checkbox">
        <input
          type="checkbox"
          v-model="config.controls"
          @change="updateConfig"
        />
        {{ t('editor.show_controls') }}
      </label>
    </div>
  </div>
</template>

<script setup lang="ts">
import { reactive } from 'vue';
import { useCardBridge } from '../bridge/iframe-bridge';
import { useI18n } from '@chips/sdk';

const { t } = useI18n();
const { config, updateConfig, uploadResource } = useCardBridge();

async function handleFileSelect(event: Event) {
  const file = (event.target as HTMLInputElement).files?.[0];
  if (file) {
    const resourcePath = await uploadResource(file);
    config.video_file = resourcePath;
    updateConfig();
  }
}

async function handleCoverSelect(event: Event) {
  const file = (event.target as HTMLInputElement).files?.[0];
  if (file) {
    const resourcePath = await uploadResource(file);
    config.cover_image = resourcePath;
    updateConfig();
  }
}
</script>
```

##  布局插件（Layout）

布局插件定义箱子的展示方式，如瀑布流布局、网格布局、列表布局等。

### 特点

- 以 iframe 方式加载
- 负责渲染箱子中的卡片列表
- 可以有可选的编辑组件用于配置布局参数
- 接收卡片列表数据进行布局渲染

### manifest.yaml 示例

```yaml
schemaVersion: "1.0.0"
id: "my-publisher.waterfall-layout"
name: "i18n.plugin.600020"
version: "1.0.0"
type: "layout"
publisher: "my-publisher"

entry:
  renderer: "dist/renderer/index.html"
  editor: "dist/editor/index.html"

capabilities:
  layoutType: "waterfall"
  supportedFeatures:
    - "infinite-scroll"
    - "drag-reorder"

compatibility:
  host: ">=1.0.0"
```

### 布局渲染组件示例

```vue
<!-- src/renderer/Renderer.vue -->
<template>
  <div class="waterfall-layout" ref="containerRef">
    <div
      v-for="column in columns"
      :key="column.id"
      class="waterfall-column"
      :style="{ gap: `${config.gap}px` }"
    >
      <div
        v-for="card in column.cards"
        :key="card.id"
        class="waterfall-item"
        @click="handleCardClick(card)"
      >
        <img
          v-if="card.thumbnail"
          :src="card.thumbnail"
          :alt="card.name"
          class="waterfall-item__thumbnail"
        />
        <div class="waterfall-item__info">
          <h3 class="waterfall-item__title">{{ card.name }}</h3>
          <p class="waterfall-item__meta">{{ card.type }}</p>
        </div>
      </div>
    </div>
  </div>
</template>

<script setup lang="ts">
import { ref, computed, onMounted, watch } from 'vue';
import { useLayoutBridge } from '../bridge/iframe-bridge';

interface WaterfallConfig {
  columns: number;
  gap: number;
  cardWidth: number;
}

interface CardItem {
  id: string;
  name: string;
  type: string;
  thumbnail?: string;
}

const { config, cards, sendMessage } = useLayoutBridge<WaterfallConfig, CardItem>();
const containerRef = ref<HTMLElement>();

// 瀑布流算法：将卡片分配到各列
const columns = computed(() => {
  const cols: Array<{ id: number; cards: CardItem[]; height: number }> = [];

  for (let i = 0; i < config.value.columns; i++) {
    cols.push({ id: i, cards: [], height: 0 });
  }

  cards.value.forEach(card => {
    // 找到最短的列
    const shortestCol = cols.reduce((min, col) =>
      col.height < min.height ? col : min
    );
    shortestCol.cards.push(card);
    // 估算高度（实际应该根据图片尺寸计算）
    shortestCol.height += 200;
  });

  return cols;
});

function handleCardClick(card: CardItem) {
  sendMessage('card-click', { cardId: card.id });
}
</script>
```

##  模块插件（Module）

模块插件是共享功能模块，可被多个插件加载使用，如视频播放器、Markdown 渲染器等。

### 特点

- 可以以 iframe 或 ES Module 方式加载
- 提供可复用的功能组件
- 被其他插件动态加载使用
- 支持版本管理和缓存

### 加载模式

- **iframe 模式**：隔离性高，适合复杂 UI 模块
- **ES Module 模式**：性能更高，适合纯渲染/工具模块

### manifest.yaml 示例

```yaml
schemaVersion: "1.0.0"
id: "my-publisher.video-player-module"
name: "i18n.plugin.600030"
version: "1.0.0"
type: "module"
publisher: "my-publisher"

entry:
  module: "dist/index.js"
  iframe: "dist/index.html"

exports:
  - name: "createVideoPlayer"
    description: "创建视频播放器实例"
  - name: "VideoPlayerComponent"
    description: "视频播放器 Vue 组件"

capabilities:
  features:
    - "media.video"
    - "subtitle"

compatibility:
  host: ">=1.0.0"
```

### 模块实现示例

```typescript
// src/index.ts
import { defineComponent, h, ref } from 'vue';

// 导出创建函数
export function createVideoPlayer(options: VideoPlayerOptions): VideoPlayer {
  return new VideoPlayerImpl(options);
}

// 导出 Vue 组件
export const VideoPlayerComponent = defineComponent({
  name: 'VideoPlayer',
  props: {
    src: { type: String, required: true },
    poster: String,
    autoplay: Boolean,
    controls: { type: Boolean, default: true }
  },
  emits: ['play', 'pause', 'ended', 'timeupdate'],
  setup(props, { emit }) {
    const videoRef = ref<HTMLVideoElement>();

    return () => h('video', {
      ref: videoRef,
      src: props.src,
      poster: props.poster,
      autoplay: props.autoplay,
      controls: props.controls,
      onPlay: () => emit('play'),
      onPause: () => emit('pause'),
      onEnded: () => emit('ended'),
      onTimeupdate: (e: Event) => {
        const video = e.target as HTMLVideoElement;
        emit('timeupdate', { currentTime: video.currentTime });
      }
    });
  }
});

// 类型定义
interface VideoPlayerOptions {
  container: HTMLElement;
  src: string;
  poster?: string;
}

interface VideoPlayer {
  play(): void;
  pause(): void;
  seek(time: number): void;
  destroy(): void;
}
```

### 使用模块

```typescript
// 在其他插件中使用模块
const module = await window.chips.invoke('module', 'load', {
  moduleId: 'my-publisher.video-player-module',
  prefer: 'esm'
});

// 动态导入模块
const { createVideoPlayer, VideoPlayerComponent } = await import(module.moduleUrl);

// 使用创建函数
const player = createVideoPlayer({
  container: document.getElementById('player-container'),
  src: videoUrl
});

// 或在 Vue 中使用组件
app.component('VideoPlayer', VideoPlayerComponent);
```

##  主题插件（Theme）

主题插件定义视觉样式，包括 CSS Token、组件样式、图标和动画。

### 特点

- 不是运行时插件，而是资源注入
- 包含完整的设计系统实现
- 支持多层级主题继承
- 使用 Style Dictionary 构建 Token

### 项目结构

```
my-theme/
├── manifest.yaml
├── tokens/                  # Token 定义（Style Dictionary 格式）
│   ├── global.json          # 全局 Token
│   ├── semantic.json        # 语义化 Token
│   └── components/          # 组件级 Token
│       ├── button.json
│       ├── input.json
│       └── ...
├── styles/                  # 组件样式
│   ├── button.css
│   ├── input.css
│   └── ...
├── icons/                   # 图标资源
│   ├── icon-font.woff2
│   └── icons.css
├── animations/              # 动画定义
│   └── transitions.css
├── sd.config.js             # Style Dictionary 配置
└── dist/                    # 构建输出
    ├── tokens/
    │   └── variables.css
    └── theme.css
```

### manifest.yaml 示例

```yaml
schemaVersion: "1.0.0"
id: "my-publisher.business-theme"
name: "i18n.plugin.600040"
version: "1.0.0"
type: "theme"
publisher: "my-publisher"

entry:
  tokens: "tokens/global.css"
  themeCss: "theme.css"

theme:
  tokensVersion: "1.0.0"
  inherits: "chips-official.default-theme"

compatibility:
  host: ">=1.0.0"
```

> 主题入口必须使用包根相对路径，`entry.tokens` 与 `entry.themeCss` 禁止 `dist/` 前缀；主题包不再使用白天/夜间标签字段。
