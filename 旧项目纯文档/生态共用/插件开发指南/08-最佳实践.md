# 最佳实践

##  代码规范

### 使用 TypeScript

始终使用 TypeScript 进行开发，确保类型安全：

```typescript
// ✅ 正确：明确的类型定义
interface CardConfig {
  title: string;
  content: string;
  tags?: string[];
}

async function saveCard(config: CardConfig): Promise<void> {
  await window.chips.invoke('card', 'update', {
    cardId: currentCardId,
    config
  });
}

// ❌ 错误：使用 any 类型
async function saveCard(config: any): Promise<any> {
  // ...
}
```

### 遵循命名规范

```typescript
// 组件命名：PascalCase
export const MyCardRenderer = defineComponent({ ... });

// 函数命名：camelCase
function handleCardClick() { ... }

// 常量命名：UPPER_SNAKE_CASE
const MAX_FILE_SIZE = 10 * 1024 * 1024;

// CSS 类名：BEM 命名法
// .chips-card__header--active
```

##  多语言规范

### 零硬编码文本

所有面向用户的文本必须使用多语言系统：

```vue
<template>
  <!-- ✅ 正确 -->
  <button>{{ t('common.save') }}</button>
  <p>{{ t('message.file_saved', { filename: file.name }) }}</p>

  <!-- ❌ 错误 -->
  <button>保存</button>
  <p>文件 {{ file.name }} 已保存</p>
```

##  样式规范

### 使用 CSS 类名

不要使用内联样式，始终使用 CSS 类名：

```vue
<template>
  <!-- ✅ 正确 -->
  <div class="my-card">
    <h1 class="my-card__title">{{ title }}</h1>
    <p class="my-card__content">{{ content }}</p>

  <div style="padding: 16px; background: white;">
    <h1 style="font-size: 24px; color: #333;">{{ title }}</h1>
```

##  性能优化

### 避免不必要的 API 调用

```typescript
// ✅ 正确：批量获取数据
const cards = await window.chips.invoke('card', 'batchRead', {
  cardIds: ['card-1', 'card-2', 'card-3']
});

// ❌ 错误：多次单独调用
const card1 = await window.chips.invoke('card', 'read', { cardId: 'card-1' });

const card3 = await window.chips.invoke('card', 'read', { cardId: 'card-3' });
```


##  错误处理

### 统一错误处理

```typescript
import { createLogger } from '@chips/sdk';

const logger = createLogger('my-plugin');

async function safeOperation<T>(
  operation: () => Promise<T>,
  fallback: T
): Promise<T> {
  try {
    return await operation();
  } catch (error) {
    const bridgeError = error as ChipsBridgeError;
    logger.error('操作失败', { code: bridgeError.code, message: bridgeError.message });

    // 显示用户友好的错误信息
    await window.chips.dialog.showMessageBox({
      type: 'error',
      title: t('common.error'),
      message: t(`error.${bridgeError.code}`, { default: bridgeError.message })
    });

    return fallback;
  }
}
```

##  安全实践

### 最小权限原则

只申请必需的权限：

```yaml
# ✅ 正确：只申请需要的权限
permissions:
  - file.read
  - card.read

# ❌ 错误：申请过多权限
permissions:
  - file.read
  - file.write
  - file.delete
  - card.read
  - card.write
  - card.delete
  - shell.openExternal
```

### 输入验证

```typescript
function validateInput(input: string): boolean {
  // 检查长度
  if (input.length > 1000) {
    return false;
  }

  // 检查危险字符
  const dangerousPatterns = [/<script/i, /javascript:/i, /on\w+=/i];
  for (const pattern of dangerousPatterns) {
    if (pattern.test(input)) {
      return false;
    }
  }

  return true;
}
```

##  开发工作流

### 推荐的开发流程

1. 使用 `chipsd create` 创建项目
2. 配置 `manifest.yaml`
3. 编写多语言词汇表
4. 开发功能组件
5. 编写测试用例
6. 本地调试 (`chipsd dev`)
7. 构建和打包 (`chipsd build && chipsd pack`)
8. 验证插件 (`chipsd validate`)
9. 本地安装测试 (`chipsd install`)
10. 发布到插件市场 (`chipsd publish`)
