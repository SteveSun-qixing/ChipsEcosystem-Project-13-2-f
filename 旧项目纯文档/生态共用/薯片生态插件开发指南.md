## 1. 快速入门

### 1.1 5 分钟创建你的第一个薯片插件

本节将带你快速创建一个简单的薯片插件，让你对插件开发流程有一个直观的认识。

#### 环境准备

在开始之前，请确保你的开发环境满足以下要求：

- **Node.js**: 18.0.0 或更高版本
- **pnpm**: 8.0.0 或更高版本（推荐使用 pnpm 作为包管理器）
- **Chips Develop CLI**: 薯片插件开发命令行工具

安装 Chips Develop CLI：

```bash
# 全局安装 Chips Develop CLI
pnpm add -g @chips/develop-cli

# 验证安装
chipsd --version
```

#### 创建插件项目

使用 `chipsd create` 命令创建一个新的插件项目：

```bash
# 创建一个简单的应用插件
chipsd create my-first-plugin

# 交互式选择插件类型
? 选择插件类型:
  ❯ app-blank        空白应用插件
    app-single-card  单卡片应用
    card-simple      简单卡片插件
    theme-full       完整主题包
    ...
```

选择 `app-blank` 创建一个空白应用插件。

#### 项目结构

创建完成后，你会得到以下项目结构：

```
my-first-plugin/
├── manifest.yaml          # 插件清单（必需）
├── package.json           # npm 包配置
├── tsconfig.json          # TypeScript 配置
├── vite.config.ts         # Vite 构建配置
├── src/
│   ├── main.ts            # 应用入口
│   ├── App.vue            # 主组件
│   └── types.ts           # 类型定义
├── locales/               # 多语言文件
│   └── vocabulary.yaml    # 插件词汇表
├── assets/                # 静态资源
│   └── icon.png           # 插件图标
└── dist/                  # 构建输出（自动生成）
```

#### 启动开发服务器

```bash
# 进入项目目录
cd my-first-plugin

# 安装依赖
pnpm install

# 启动开发服务器
chipsd dev
```

开发服务器启动后，会自动注入 Mock Bridge API，让你可以在没有薯片主机的情况下进行开发调试。

#### 编写第一个功能

打开 `src/App.vue`，编写一个简单的功能：

```vue
<template>
  <div class="chips-app">
    <h1>{{ t('app.title') }}</h1>
    <button
      class="chips-button chips-button--primary"
      @click="handleClick"
    >
      {{ t('app.button') }}
    </button>
    <p v-if="message">{{ message }}</p>
  </div>
</template>

<script setup lang="ts">
import { ref } from 'vue';
import { useI18n } from '@chips/sdk';

const { t } = useI18n();
const message = ref('');

async function handleClick() {
  // 通过 Bridge API 调用系统服务
  const result = await window.chips.invoke('platform', 'getSystemInfo', {});
  message.value = `${t('app.system')}: ${result.os} ${result.version}`;
}
</script>
```

#### 构建和打包

```bash
# 构建项目
chipsd build

# 打包为 .cpk 文件
chipsd pack
```

打包完成后，你会在项目根目录得到一个 `.cpk` 文件，这就是可以安装到薯片主机的插件包。

#### 验证插件

```bash
# 验证插件包的完整性
chipsd validate ./my-first-plugin-1.0.0.cpk
```

恭喜！你已经成功创建了第一个薯片插件。

### 1.2 核心概念

在深入学习之前，你需要了解薯片生态的几个核心概念：

#### 中心路由原则

薯片生态采用中心路由架构，所有服务间通信必须经过内核路由器：

```typescript
// ✅ 正确：通过 Bridge API 调用服务
const result = await window.chips.invoke('file', 'read', { path: '/path/to/file' });

// ❌ 错误：直接使用 Node.js API（插件中禁止）
const fs = require('fs');
const data = fs.readFileSync('/path/to/file');
```

#### Bridge 隔离原则

插件运行在渲染进程中，只能通过 `window.chips.*` Bridge API 访问系统能力：

- 插件不能直接访问 Node.js API
- 插件不能直接访问 Electron API
- 插件不能直接操作文件系统
- 所有系统调用都通过 Bridge API 进行

#### 零硬编码文本

所有面向用户的文本必须使用多语言系统：

```typescript
// ✅ 正确：使用多语言系统
const label = t('common.save');

// ❌ 错误：硬编码文本
const label = '保存';
```

#### 零样式组件

组件库中的组件不包含任何视觉样式，样式由主题包提供：

```vue
<!-- ✅ 正确：使用 CSS 类名，样式由主题注入 -->
<button class="chips-button chips-button--primary">
  {{ t('common.save') }}
</button>

<!-- ❌ 错误：内联样式 -->
<button style="background: blue; color: white;">
  保存
</button>
```

## 2. 五种插件类型详解

薯片生态支持五种插件类型，每种类型有不同的用途和运行方式。

### 2.1 插件类型概览

| 类型 | type 值 | 运行容器 | 典型用途 |
|------|---------|----------|----------|
| 应用插件 | `app` | BrowserWindow | 编辑器、查看器、设置面板 |
| 卡片插件 | `card` | iframe | 基础卡片渲染与编辑 |
| 布局插件 | `layout` | iframe | 箱子布局渲染 |
| 模块插件 | `module` | iframe / ES Module | 视频播放器、Markdown 渲染器 |
| 主题插件 | `theme` | 无运行容器 | Token 与样式注入 |

### 2.2 应用插件（App）

应用插件是拥有独立窗口的完整应用，如编辑器、查看器、设置面板等。

#### 特点

- 拥有独立的 BrowserWindow 窗口
- 可以控制窗口大小、位置、标题等
- 可以创建菜单、工具栏等
- 生命周期由薯片主机管理

#### 项目结构

```
my-app-plugin/
├── manifest.yaml
├── package.json
├── src/
│   ├── main.ts              # 应用入口
│   ├── App.vue              # 主组件
│   ├── components/          # 组件目录
│   ├── composables/         # 组合式函数
│   └── types.ts             # 类型定义
├── locales/
│   └── vocabulary.yaml
├── assets/
│   └── icon.png
└── dist/
    └── index.html           # 构建输出
```

#### manifest.yaml 示例

```yaml
schemaVersion: "1.0.0"
id: "my-publisher.my-app"
name: "i18n.plugin.600001"
version: "1.0.0"
type: "app"
publisher: "my-publisher"
description: "i18n.plugin.600002"
icon: "assets/icon.png"

entry:
  main: "dist/index.html"

window:
  width: 1200
  height: 800
  minWidth: 800
  minHeight: 600
  resizable: true
  frame: true

permissions:
  - "file.read"
  - "file.write"
  - "card.read"
  - "card.write"

compatibility:
  host: ">=1.0.0"
  platform:
    - "win32"
    - "darwin"
    - "linux"

locales:
  default: "zh-CN"
  supported:
    - "zh-CN"
    - "en-US"
  path: "locales"
```

#### 完整示例

```vue
<!-- src/App.vue -->
<template>
  <div class="chips-app">
    <header class="chips-app__header">
      <h1>{{ t('app.title') }}</h1>
      <div class="chips-app__actions">
        <button
          class="chips-button chips-button--primary"
          @click="handleSave"
        >
          {{ t('common.save') }}
        </button>
      </div>
    </header>

    <main class="chips-app__content">
      <div v-if="loading" class="chips-loading">
        {{ t('common.loading') }}
      </div>
      <div v-else>
        <!-- 应用内容 -->
      </div>
    </main>
  </div>
</template>

<script setup lang="ts">
import { ref, onMounted } from 'vue';
import { useI18n, usePlugin } from '@chips/sdk';

const { t } = useI18n();
const { pluginInfo } = usePlugin();
const loading = ref(true);

onMounted(async () => {
  // 初始化应用
  try {
    // 获取插件信息
    const info = await window.chips.plugin.getSelf();
    console.log('Plugin loaded:', info.id);

    // 设置窗口标题
    await window.chips.window.setTitle(t('app.title'));
  } finally {
    loading.value = false;
  }
});

async function handleSave() {
  try {
    // 保存逻辑
    await window.chips.invoke('file', 'write', {
      path: '/workspace/data.json',
      content: JSON.stringify({ data: 'example' }),
      encoding: 'utf8'
    });

    // 显示成功提示
    await window.chips.dialog.showMessageBox({
      type: 'info',
      title: t('common.success'),
      message: t('message.save_success')
    });
  } catch (error) {
    // 显示错误提示
    await window.chips.dialog.showMessageBox({
      type: 'error',
      title: t('common.error'),
      message: t('message.save_failed')
    });
  }
}
</script>
```

### 2.3 卡片插件（Card）

卡片插件定义一种基础卡片类型的渲染和编辑组件，如视频卡片、图片卡片、富文本卡片等。

#### 特点

- 以 iframe 方式加载在宿主应用中
- 包含渲染组件和编辑组件两个入口
- 通过 postMessage 与宿主通信
- 接收卡片配置数据进行渲染

#### 项目结构

```
my-card-plugin/
├── manifest.yaml
├── package.json
├── src/
│   ├── renderer/            # 渲染组件
│   │   ├── index.ts
│   │   └── Renderer.vue
│   ├── editor/              # 编辑组件
│   │   ├── index.ts
│   │   └── Editor.vue
│   ├── bridge/              # iframe 通信层
│   │   ├── iframe-bridge.ts
│   │   └── message-types.ts
│   └── types.ts
├── locales/
│   └── vocabulary.yaml
└── dist/
    ├── renderer/
    │   └── index.html
    └── editor/
        └── index.html
```

#### manifest.yaml 示例

```yaml
schemaVersion: "1.0.0"
id: "my-publisher.video-card"
name: "i18n.plugin.600010"
version: "1.0.0"
type: "card"
publisher: "my-publisher"

entry:
  renderer: "dist/renderer/index.html"
  editor: "dist/editor/index.html"

capabilities:
  cardType: "VideoCard"
  supportedFormats:
    - "mp4"
    - "webm"
    - "mov"

permissions:
  - "resource.fetch"

compatibility:
  host: ">=1.0.0"
```

#### 渲染组件示例

```vue
<!-- src/renderer/Renderer.vue -->
<template>
  <div class="video-card-renderer">
    <video
      ref="videoRef"
      class="video-card__player"
      :src="videoUrl"
      :poster="config.cover_image"
      :controls="config.controls"
      :autoplay="config.autoplay"
      @timeupdate="handleTimeUpdate"
    />
  </div>
</template>

<script setup lang="ts">
import { ref, computed, onMounted, onUnmounted } from 'vue';
import { useCardBridge } from '../bridge/iframe-bridge';

interface VideoCardConfig {
  video_file: string;
  cover_image?: string;
  autoplay?: boolean;
  controls?: boolean;
}

const { config, resourceUrl, sendMessage, onMessage } = useCardBridge<VideoCardConfig>();
const videoRef = ref<HTMLVideoElement>();

// 解析视频 URL
const videoUrl = computed(() => {
  return resourceUrl(config.value.video_file);
});

// 处理播放进度更新
function handleTimeUpdate() {
  if (videoRef.value) {
    sendMessage('state-update', {
      currentTime: videoRef.value.currentTime,
      duration: videoRef.value.duration
    });
  }
}

// 监听宿主消息
onMounted(() => {
  onMessage('seek', (data: { time: number }) => {
    if (videoRef.value) {
      videoRef.value.currentTime = data.time;
    }
  });

  onMessage('play', () => {
    videoRef.value?.play();
  });

  onMessage('pause', () => {
    videoRef.value?.pause();
  });
});
</script>
```

#### 编辑组件示例

```vue
<!-- src/editor/Editor.vue -->
<template>
  <div class="video-card-editor">
    <div class="editor-section">
      <label class="chips-label">{{ t('editor.video_file') }}</label>

    <div class="editor-section">

        <input
          type="checkbox"
          v-model="config.controls"
          @change="updateConfig"
        />
        {{ t('editor.show_controls') }}
      </label>
    </div>
  </div>
</template>

<script setup lang="ts">
import { reactive } from 'vue';
import { useCardBridge } from '../bridge/iframe-bridge';
import { useI18n } from '@chips/sdk';
```

### 2.4 布局插件（Layout）

布局插件定义箱子的展示方式，如瀑布流布局、网格布局、列表布局等。

#### 特点

- 以 iframe 方式加载
- 负责渲染箱子中的卡片列表
- 可以有可选的编辑组件用于配置布局参数
- 接收卡片列表数据进行布局渲染

#### manifest.yaml 示例

```yaml

id: "my-publisher.waterfall-layout"
name: "i18n.plugin.600020"
version: "1.0.0"
type: "layout"
publisher: "my-publisher"

entry:
  renderer: "dist/renderer/index.html"
  editor: "dist/editor/index.html"

capabilities:
  layoutType: "waterfall"
  supportedFeatures:
    - "infinite-scroll"
    - "drag-reorder"

compatibility:
  host: ">=1.0.0"
```

#### 布局渲染组件示例

```vue
<!-- src/renderer/Renderer.vue -->
<template>
  <div class="waterfall-layout" ref="containerRef">
    <div
      v-for="column in columns"
      :key="column.id"
      class="waterfall-column"
      :style="{ gap: `${config.gap}px` }"
    >
      <div
        v-for="card in column.cards"
        :key="card.id"
        class="waterfall-item"
        @click="handleCardClick(card)"
      >
        <img
          v-if="card.thumbnail"
          :src="card.thumbnail"
          :alt="card.name"
          class="waterfall-item__thumbnail"
        />
        <div class="waterfall-item__info">
          <h3 class="waterfall-item__title">{{ card.name }}</h3>
          <p class="waterfall-item__meta">{{ card.type }}</p>
        </div>
      </div>
    </div>
  </div>
</template>

<script setup lang="ts">
import { ref, computed, onMounted, watch } from 'vue';
import { useLayoutBridge } from '../bridge/iframe-bridge';

interface WaterfallConfig {
  columns: number;
  gap: number;
  cardWidth: number;
}

interface CardItem {
  id: string;
  name: string;
  type: string;
  thumbnail?: string;
}

const { config, cards, sendMessage } = useLayoutBridge<WaterfallConfig, CardItem>();
const containerRef = ref<HTMLElement>();

// 瀑布流算法：将卡片分配到各列
const columns = computed(() => {
  const cols: Array<{ id: number; cards: CardItem[]; height: number }> = [];

  for (let i = 0; i < config.value.columns; i++) {
    cols.push({ id: i, cards: [], height: 0 });
  }

  cards.value.forEach(card => {
    // 找到最短的列
    const shortestCol = cols.reduce((min, col) =>
      col.height < min.height ? col : min
    );
    shortestCol.cards.push(card);
    // 估算高度（实际应该根据图片尺寸计算）
    shortestCol.height += 200;
  });

  return cols;
});

function handleCardClick(card: CardItem) {
  sendMessage('card-click', { cardId: card.id });
}
</script>
```

### 2.5 模块插件（Module）

模块插件是共享功能模块，可被多个插件加载使用，如视频播放器、Markdown 渲染器等。

#### 特点

- 可以以 iframe 或 ES Module 方式加载
- 提供可复用的功能组件
- 被其他插件动态加载使用
- 支持版本管理和缓存

#### 加载模式

- **iframe 模式**：隔离性高，适合复杂 UI 模块
- **ES Module 模式**：性能更高，适合纯渲染/工具模块

#### manifest.yaml 示例

```yaml
schemaVersion: "1.0.0"
id: "my-publisher.video-player-module"
name: "i18n.plugin.600030"
version: "1.0.0"
type: "module"
publisher: "my-publisher"

entry:
  module: "dist/index.js"
  iframe: "dist/index.html"

exports:
  - name: "createVideoPlayer"
    description: "创建视频播放器实例"
  - name: "VideoPlayerComponent"
    description: "视频播放器 Vue 组件"

capabilities:
  features:
    - "media.video"
    - "subtitle"

compatibility:
  host: ">=1.0.0"
```

#### 模块实现示例

```typescript
// src/index.ts
import { defineComponent, h, ref } from 'vue';

// 导出创建函数
export function createVideoPlayer(options: VideoPlayerOptions): VideoPlayer {
  return new VideoPlayerImpl(options);
}

// 导出 Vue 组件
export const VideoPlayerComponent = defineComponent({
  name: 'VideoPlayer',
  props: {
    src: { type: String, required: true },
    poster: String,
    autoplay: Boolean,
    controls: { type: Boolean, default: true }
  },
  emits: ['play', 'pause', 'ended', 'timeupdate'],
  setup(props, { emit }) {
    const videoRef = ref<HTMLVideoElement>();

    return () => h('video', {
      ref: videoRef,
      src: props.src,
      poster: props.poster,
      autoplay: props.autoplay,
      controls: props.controls,
      onPlay: () => emit('play'),
      onPause: () => emit('pause'),
      onEnded: () => emit('ended'),
      onTimeupdate: (e: Event) => {
        const video = e.target as HTMLVideoElement;
        emit('timeupdate', { currentTime: video.currentTime });
      }
    });
  }
});

// 类型定义
interface VideoPlayerOptions {
  container: HTMLElement;
  src: string;
  poster?: string;
}

interface VideoPlayer {
  play(): void;
  pause(): void;
  seek(time: number): void;
  destroy(): void;
}
```

#### 使用模块

```typescript
// 在其他插件中使用模块
const module = await window.chips.invoke('module', 'load', {
  moduleId: 'my-publisher.video-player-module',
  prefer: 'esm'
});

// 动态导入模块
const { createVideoPlayer, VideoPlayerComponent } = await import(module.moduleUrl);

// 使用创建函数
const player = createVideoPlayer({
  container: document.getElementById('player-container'),
  src: videoUrl
});

// 或在 Vue 中使用组件
app.component('VideoPlayer', VideoPlayerComponent);
```

### 2.6 主题插件（Theme）

主题插件定义视觉样式，包括 CSS Token、组件样式、图标和动画。

#### 特点

- 不是运行时插件，而是资源注入
- 包含完整的设计系统实现

## 3. Bridge API 参考手册

Bridge API 是插件访问系统能力的唯一入口。所有系统调用都通过 `window.chips.*` 进行。

### 3.1 核心调用方法

#### `chips.invoke(namespace, action, params?)`

通用服务调用方法，用于调用所有注册到内核路由器的服务。

```typescript
// 签名
chips.invoke(namespace: string, action: string, params?: unknown): Promise<unknown>

// 示例：读取文件
const content = await window.chips.invoke('file', 'read', {
  path: '/workspace/data.json',
  encoding: 'utf8'
});

// 示例：获取卡片元数据
const metadata = await window.chips.invoke('card', 'getMetadata', {
  cardId: 'card-123'
});
```

参数约束：
- `namespace`：服务命名空间（如 `file`、`card`、`platform`）
- `action`：动作名称（如 `read`、`write`、`getMetadata`）
- `params`：可结构化克隆的对象，不允许函数、DOM 实例、循环引用

### 3.2 14 类基础服务

薯片生态提供 14 类基础服务，覆盖所有系统能力：

| 命名空间 | 说明 | 典型动作 |
|---------|------|---------|
| `file` | 文件操作 | read, write, delete, exists, stat, list |
| `card` | 卡片管理 | create, read, update, delete, getMetadata |
| `box` | 箱子管理 | create, read, update, delete, listCards |
| `resource` | 资源管理 | fetch, upload, getUrl, delete |
| `zip` | 压缩解压 | compress, decompress, list |
| `config` | 配置管理 | get, set, delete, list |
| `theme` | 主题服务 | getCurrent, list, apply, getTokens |
| `i18n` | 多语言 | translate, translateBatch, getCurrent, setCurrent |
| `credential` | 凭证管理 | get, set, delete, list |
| `log` | 日志服务 | debug, info, warn, error |
| `tag` | 标签管理 | create, list, attach, detach |
| `serializer` | 序列化 | serialize, deserialize |
| `platform` | 平台信息 | getSystemInfo, getAppInfo, getPaths |
| `module` | 模块加载 | load, unload, list |

### 3.3 常用服务 API 详解

#### 文件服务（file）

```typescript
// 读取文件
const content = await window.chips.invoke('file', 'read', {
  path: '/workspace/notes.txt',
  encoding: 'utf8'  // 可选：'utf8' | 'base64' | 'binary'
});

// 写入文件
await window.chips.invoke('file', 'write', {
  path: '/workspace/output.json',
  content: JSON.stringify(data),
  encoding: 'utf8'
});

// 检查文件是否存在
const exists = await window.chips.invoke('file', 'exists', {
  path: '/workspace/config.yaml'
});

// 获取文件信息
const stat = await window.chips.invoke('file', 'stat', {
  path: '/workspace/image.png'
});
// 返回: { size, mtime, ctime, isFile, isDirectory }

// 列出目录内容
const files = await window.chips.invoke('file', 'list', {
  path: '/workspace',
  recursive: false
});

// 删除文件（需要权限，会触发二次确认）
await window.chips.invoke('file', 'delete', {
  path: '/workspace/temp.txt'
});
```

#### 卡片服务（card）

```typescript
// 创建卡片
const card = await window.chips.invoke('card', 'create', {
  boxId: 'box-123',
  type: 'TextCard',
  config: {
    content: '卡片内容'
  }
});

// 读取卡片
const cardData = await window.chips.invoke('card', 'read', {
  cardId: 'card-456'
});

// 更新卡片

  cardId: 'card-456',
  config: {
    content: '更新后的内容'
  }
});

// 获取卡片元数据
const metadata = await window.chips.invoke('card', 'getMetadata', {
  cardId: 'card-456'
});
// 返回: { id, type, boxId, createdAt, updatedAt, tags }
```

#### 多语言服务（i18n）

```typescript
// 翻译单个键
const text = await window.chips.invoke('i18n', 'translate', {
  key: 'common.save',
  params: {}  // 可选的插值参数
});

// 批量翻译
const texts = await window.chips.invoke('i18n', 'translateBatch', {
  keys: ['common.save', 'common.cancel', 'common.delete']

```

#### 主题服务（theme）

```typescript
// 获取当前主题

// 返回: { themeId }

```

#### 平台服务（platform）

```typescript
// 获取系统信息
const sysInfo = await window.chips.invoke('platform', 'getSystemInfo', {});

// 获取应用信息

// 获取系统路径

```

### 3.4 事件订阅

#### `chips.on(event, callback)`

持续订阅事件，返回取消订阅函数。

```typescript
// 订阅主题变更事件
const unsubscribe = window.chips.on('theme.changed', (payload) => {
  console.log('主题已切换:', payload.themeId);
  // 重新加载样式
});

// 取消订阅
unsubscribe();
```

#### `chips.once(event, callback)`

单次订阅，触发后自动解绑。

```typescript
// 等待系统就绪
window.chips.once('system.ready', () => {
  console.log('系统已就绪');
  initializeApp();
});
```

#### 支持的系统事件

| 事件名 | 触发时机 | 载荷 |
|-------|---------|------|
| `system.ready` | 系统初始化完成 | `void` |
| `system.shuttingDown` | 系统即将关闭 | `void` |
| `theme.changed` | 主题切换 | `{ themeId: string }` |
| `language.changed` | 语言切换 | `{ language: string }` |
| `config.updated` | 配置更新 | `{ key: string, value: unknown }` |
| `plugin.installed` | 插件安装 | `{ pluginId: string }` |
| `plugin.uninstalled` | 插件卸载 | `{ pluginId: string }` |
| `card.opened` | 卡片打开 | `{ cardPath: string }` |
| `card.saved` | 卡片保存 | `{ cardPath: string }` |
| `file.changed` | 文件变更 | `{ path: string, type: 'create' \| 'modify' \| 'delete' }` |

#### 通配符订阅

```typescript
// 订阅所有卡片相关事件
window.chips.on('card.*', (payload) => {
  console.log('卡片事件:', payload);
});

// 订阅所有插件相关事件
window.chips.on('plugin.*', (payload) => {
  console.log('插件事件:', payload);
});
```

### 3.10 错误处理

所有 Bridge API 调用失败时会抛出标准错误对象：

```typescript
interface ChipsBridgeError {
  code: string;      // 错误码
  message: string;   // 错误信息
  details?: unknown; // 详细信息
}
```

常见错误码：

| 错误码 | 说明 |
|-------|------|
| `PERMISSION_DENIED` | 权限不足 |
| `VALIDATION_ERROR` | 参数校验失败 |
| `ROUTE_NOT_FOUND` | 服务或动作不存在 |
| `TIMEOUT` | 请求超时 |
| `USER_CANCELLED` | 用户取消操作 |
| `PLUGIN_NOT_FOUND` | 插件不存在 |
| `FILE_NOT_FOUND` | 文件不存在 |

错误处理示例：

```typescript
try {
  await window.chips.invoke('file', 'read', { path: '/nonexistent' });
} catch (error) {
  const bridgeError = error as ChipsBridgeError;

  switch (bridgeError.code) {
    case 'FILE_NOT_FOUND':
      console.log('文件不存在');
      break;
    case 'PERMISSION_DENIED':
      console.log('没有读取权限');
      break;
    default:
      console.error('未知错误:', bridgeError.message);
  }
}
```

### 3.11 权限声明

插件需要在 `manifest.yaml` 中声明所需权限：

```yaml
permissions:
  - file.read           # 读取文件
  - file.write          # 写入文件
  - card.read           # 读取卡片
  - card.write          # 写入卡片
  - window.create       # 创建新窗口
  - dialog.open         # 打开文件对话框
  - dialog.save         # 保存文件对话框
  - dialog.message      # 消息对话框
  - clipboard.read      # 读取剪贴板
  - clipboard.write     # 写入剪贴板
  - shell.openPath      # 打开文件
  - shell.openExternal  # 打开外部链接
  - plugin.read         # 查询插件信息
```

无需权限的 API：
- `chips.on()` / `chips.once()` - 事件订阅
- `chips.window.*` - 当前窗口控制（不含 openPlugin）
- `chips.plugin.getSelf()` - 获取自身信息
- `i18n.translate` / `i18n.translateBatch` - 翻译
- `theme.getCurrent` / `theme.getTokens` - 主题查询

```vue
<template>
  <div class="my-app">

    <div class="toolbar">

        @click="handleSave"

        class="chips-button chips-button--secondary"

<script setup lang="ts">
import { ref, onMounted } from 'vue';
import {
  useI18n,
  useFile,
  useDialog,
  useWindow,
  useEvent,
  createLogger
} from '@chips/sdk';

const { t } = useI18n();
const { read, write } = useFile();
const { openFile, saveFile, message } = useDialog();
const { setTitle } = useWindow();

const logger = createLogger('my-app');
const content = ref('');
const currentPath = ref<string | null>(null);

onMounted(async () => {
  await setTitle(t('app.title'));
});

// 监听主题变化
useEvent('theme.changed', () => {
  logger.info('主题已切换');
});

async function handleOpenFile() {
  const files = await openFile({
    title: t('dialog.open_file'),
    filters: [{ name: t('filter.text'), extensions: ['txt', 'md'] }]
  });

  if (files && files.length > 0) {
    currentPath.value = files[0];
    content.value = await read(files[0]);
    await setTitle(`${t('app.title')} - ${files[0]}`);
  }
}

async function handleSave() {
  let path = currentPath.value;

  if (!path) {
    path = await saveFile({
      title: t('dialog.save_file'),
      filters: [{ name: t('filter.text'), extensions: ['txt'] }]
    });
  }

  if (path) {
    await write(path, content.value);
    currentPath.value = path;
    await message({
      type: 'info',
      title: t('common.success'),
      content: t('message.save_success')
    });
  }
}
</script>
```

## 6. manifest.yaml 参考

`manifest.yaml` 是插件的清单文件，定义了插件的元信息、入口、权限和兼容性要求。

### 6.1 基础字段

```yaml
# 清单版本（必需）
schemaVersion: "1.0.0"

# 插件唯一标识（必需）
# 格式：{publisher}.{plugin-name}
id: "my-publisher.my-plugin"

# 插件名称（必需）
# 使用 i18n 键，实际文本在 locales 中定义
name: "i18n.plugin.600001"

# 插件版本（必需）
# 遵循语义化版本规范
version: "1.0.0"

# 插件类型（必需）
# 可选值：app | card | layout | module | theme
type: "app"

# 发布者标识（必需）
publisher: "my-publisher"

# 插件描述（可选）
description: "i18n.plugin.600002"

# 插件图标（可选）
icon: "assets/icon.png"
```

### 6.2 入口配置

不同类型的插件有不同的入口配置：

#### 应用插件（app）

```yaml
entry:
  main: "dist/index.html"
```

#### 卡片插件（card）

```yaml
entry:
  renderer: "dist/renderer/index.html"
  editor: "dist/editor/index.html"
```

#### 布局插件（layout）

```yaml
entry:
  renderer: "dist/renderer/index.html"
  editor: "dist/editor/index.html"  # 可选
```

#### 模块插件（module）

```yaml
entry:
  module: "dist/index.js"      # ES Module 入口
  iframe: "dist/index.html"    # iframe 入口（可选）
```

#### 主题插件（theme）

```yaml
entry:
  tokens: "tokens/global.css"
  themeCss: "theme.css"
```

> 主题入口要求：`entry.tokens` / `entry.themeCss` 必须使用包根相对路径，不能以 `dist/` 开头。

| `name` | string | 是 | i18n 键格式 |
| `version` | string | 是 | 语义化版本格式 |
| `type` | string | 是 | `app` \| `card` \| `layout` \| `module` \| `theme` |
| `publisher` | string | 是 | 仅允许小写字母、数字、连字符 |
| `entry` | object | 是 | 根据 type 不同有不同的必需字段 |
| `permissions` | array | 否 | 有效的权限标识列表 |
| `compatibility.host` | string | 是 | 语义化版本范围表达式 |

## 7. .cpk 打包和发布

`.cpk`（Chips Plugin Package）是薯片插件的标准打包格式，用于分发和安装插件。

### 7.1 .cpk 文件结构

`.cpk` 文件本质上是一个 ZIP 压缩包，包含以下内容：

```
my-plugin-1.0.0.cpk
├── manifest.yaml          # 插件清单（必需）
├── dist/                   # 构建产物
│   └── index.html
├── locales/                # 多语言文件
│   ├── zh-CN/
│   │   └── vocabulary.yaml
│   └── en-US/
│       └── vocabulary.yaml
├── assets/                 # 静态资源
│   └── icon.png
└── signature.json          # 签名文件（可选）
```

### 7.2 使用 CLI 打包

#### 构建项目

```bash
# 构建生产版本
chipsd build

# 构建并监听变化（开发模式）
chipsd build --watch
```

#### 打包为 .cpk

```bash
# 打包插件
chipsd pack

# 指定输出目录
chipsd pack --output ./releases

# 指定版本号（覆盖 manifest 中的版本）
chipsd pack --version 1.0.1
```

#### 验证插件包

```bash
# 验证 .cpk 文件
chipsd validate ./my-plugin-1.0.0.cpk

# 详细验证输出
chipsd validate ./my-plugin-1.0.0.cpk --verbose
```

验证内容包括：
- manifest.yaml 格式和必需字段
- 入口文件存在性
- 多语言文件完整性
- 权限声明有效性
- 文件大小限制

### 7.3 手动打包

如果需要手动打包，按以下步骤操作：

```bash
# 1. 构建项目
pnpm build

# 2. 创建临时目录
mkdir -p .cpk-temp

# 3. 复制必需文件
cp manifest.yaml .cpk-temp/
cp -r dist .cpk-temp/
cp -r locales .cpk-temp/
cp -r assets .cpk-temp/

# 4. 创建 ZIP 包
cd .cpk-temp
zip -r ../my-plugin-1.0.0.cpk .
cd ..

# 5. 清理临时目录
rm -rf .cpk-temp
```

### 7.4 签名（可选）

官方插件需要签名才能获得更高的信任级别：

```bash
# 生成签名
chipsd sign ./my-plugin-1.0.0.cpk --key ./private-key.pem

# 验证签名
chipsd verify ./my-plugin-1.0.0.cpk
```

签名后的 `.cpk` 文件会包含 `signature.json`：

```json
{
  "algorithm": "RSA-SHA256",
  "signature": "base64-encoded-signature",
  "publicKey": "base64-encoded-public-key",
  "timestamp": "2026-02-13T10:00:00Z"
}
```

### 7.7 版本管理

#### 版本号规范

遵循语义化版本（Semantic Versioning）：

- `MAJOR.MINOR.PATCH`
- `MAJOR`：不兼容的 API 变更
- `MINOR`：向后兼容的功能新增
- `PATCH`：向后兼容的问题修复

#### 版本更新流程

```bash
# 1. 更新 manifest.yaml 中的版本号
# 2. 更新 CHANGELOG.md
# 3. 构建和打包
chipsd build && chipsd pack

# 4. 验证
chipsd validate ./my-plugin-1.0.1.cpk

# 5. 发布
chipsd publish ./my-plugin-1.0.1.cpk --update
```

### 8.5 错误处理

#### 统一错误处理

```typescript
import { createLogger } from '@chips/sdk';

const logger = createLogger('my-plugin');

async function safeOperation<T>(
  operation: () => Promise<T>,
  fallback: T
): Promise<T> {
  try {
    return await operation();
  } catch (error) {
    const bridgeError = error as ChipsBridgeError;
    logger.error('操作失败', { code: bridgeError.code, message: bridgeError.message });

    // 显示用户友好的错误信息
    await window.chips.dialog.showMessageBox({
      type: 'error',
      title: t('common.error'),
      message: t(`error.${bridgeError.code}`, { default: bridgeError.message })
    });

    return fallback;
  }
}
```

```typescript
function validateInput(input: string): boolean {
  // 检查长度
  if (input.length > 1000) {
    return false;
  }

  // 检查危险字符
  const dangerousPatterns = [/<script/i, /javascript:/i, /on\w+=/i];

    if (pattern.test(input)) {
      return false;
    }
  }

  return true;
}
```
