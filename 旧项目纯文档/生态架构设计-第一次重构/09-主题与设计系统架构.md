## 一、设计哲学：自管理前端架构

薯片生态采用**自管理前端架构**，核心原则是：

> **组件库只负责功能逻辑、结构和布局，完全不包含样式。所有视觉表现由主题包提供。**

这意味着：
- 一个按钮组件只知道"我是一个按钮，我可以被点击"，不知道自己长什么样
- 是圆角还是直角、是渐变还是纯色、是有阴影还是扁平，全部由主题包决定
- 图标、背景、交互动画、组件形状都在主题包中
- 切换主题包 = 切换整个设计语言（Material Design 3 → Fluent UI → Glassmorphism）

## 二、多层级主题继承系统

### 2.1 层级架构

```
优先级从高到低：

组件级主题    ← 最高优先级，每个组件可以单独设置
  ↓ 如果未设置
基础卡片级主题  ← 每个基础卡片可以单独设置
  ↓ 如果未设置
复合卡片级主题  ← 每个复合卡片可以单独设置
  ↓ 如果未设置
箱子级主题    ← 箱子可以为其中的卡片设置统一主题
  ↓ 如果未设置
应用级主题    ← 当前应用的主题设置
  ↓ 如果未设置
全局主题      ← 整个薯片生态的默认主题（最低优先级）
```

### 2.2 CSS 变量级联实现

利用 CSS 自定义属性（Custom Properties）的天然级联特性实现多层级主题：

```css
/* 全局主题（注入到最顶层） */
:root {
  --chips-color-primary: #6200ee;
  --chips-color-surface: #ffffff;
  --chips-radius-sm: 4px;
  --chips-radius-md: 8px;
  --chips-shadow-md: 0 4px 6px rgba(0,0,0,0.1);
  --chips-font-family: 'Inter', sans-serif;
  --chips-transition-normal: 200ms ease;
  /* ... 数百个设计 Token */
}

/* 应用级主题覆盖 */
[data-chips-app="editor"] {
  --chips-color-primary: #1976d2;
  /* 只覆盖需要变化的 Token，其余继承全局 */
}

/* 卡片级主题覆盖 */
[data-chips-card="abc123"] {
  --chips-color-surface: #f5f5f5;
}

/* 组件级主题覆盖 */
[data-chips-component="btn-save"] {
  --chips-color-primary: #4caf50;
}
```

自动继承逻辑：
- CSS 自定义属性**天然向下级联**（父元素的变量值自动被子元素继承）
- 如果某个层级设置了变量，所有子层级自动使用该值
- 子层级可以覆盖父层级的值
- 不需要额外的 JavaScript 逻辑，纯 CSS 实现

### 2.3 主题引擎工作流程

```
1. 系统启动，加载全局主题 → 注入到 :root
2. 打开应用插件 → 检查应用是否有自己的主题 → 注入到应用容器
3. 渲染卡片 → 检查卡片 metadata.yaml 中的 theme 字段
   → 如果有值，加载该主题的对应 Token → 注入到卡片容器
   → 如果为空，自动继承应用级主题
4. 渲染基础卡片 → 检查基础卡片配置中的 theme 字段
   → 如果有值，注入到基础卡片 iframe
   → 如果为空，继承父级卡片主题
```

## 三、无头组件库架构

### 3.1 技术选型：Zag.js + Ark UI

经过对现有开源方案的评估，推荐使用以下技术栈：

**底层状态引擎：[Zag.js](https://zagjs.com/)（MIT 协议）**

Zag.js 由 Chakra UI 团队开发，提供与框架无关的 UI 组件状态机。每个组件的行为逻辑（焦点管理、键盘导航、ARIA 属性、状态转换）都封装在状态机中，不绑定任何 UI 框架。

为什么选 Zag.js：
- **MIT 协议**，完全开源
- **框架无关**：核心是纯 JavaScript 状态机，可以绑定到 Vue、React、Solid 等任何框架
- **零样式**：只提供行为和可访问性，完全不包含视觉样式
- **完整的组件覆盖**：按钮、输入框、对话框、下拉菜单、滑块、标签页、手风琴、日历、颜色选择器等 40+ 组件
- **内置无障碍支持**：WAI-ARIA 模式，键盘导航
- **活跃维护**：Chakra UI 团队持续开发

**React 绑定层：[Ark UI](https://ark-ui.com/)（MIT 协议）**

Ark UI 是 Zag.js 的上层框架绑定，提供 React、React、Solid 三个版本的组件。

为什么选 Ark UI：
- **MIT 协议**
- **基于 Zag.js**：直接使用 Zag.js 的状态机
- **同时支持 React、React、Solid**：满足多框架生态需求
- **Headless 设计**：组件不包含任何默认样式
- **组合式 API**：通过 slot 和 render props 暴露内部结构，支持完全自定义渲染
- **TypeScript 优先**：完整的类型定义

```
~/.chips/

│   ├── chips-official.default/          # 默认主题
│   ├── chips-official.material-3/       # Material Design 3
│   ├── chips-official.fluent/           # Fluent UI
│   ├── chips-official.glassmorphism/    # 毛玻璃
│   ├── community.custom-theme/          # 社区主题
│   └── ...
├── theme-registry.json                  # 主题注册表
└── theme-config.json                    # 用户主题配置
```

### 5.2 主题配置文件

```json
{
  "global": "chips-official.default",
  "apps": {
    "chips-official.editor": "chips-official.material-3",
    "chips-official.viewer": null
  },
  "cards": {
    "a1B2c3D4e5": "chips-official.glassmorphism"
  },
  "components": {}
}
```

- `global`：全局默认主题
- `apps`：应用级覆盖（null 表示继承全局）
- `cards`：卡片级覆盖
- `components`：组件级覆盖（极少使用）

### 5.3 主题加载流程

```
1. 应用插件启动
2. SDK 调用 chips.invoke('theme', 'resolve', { context: 'app', appId: '...' })
3. 主题服务查找层级：组件 → 卡片 → 应用 → 全局
4. 找到目标主题包 ID
5. 加载主题包的 CSS 文件（按需加载组件类型）
6. 返回 CSS 内容给插件
7. 插件将 CSS 注入到 DOM 中
8. CSS 变量立即生效，所有组件自动更新视觉
```

### 5.4 动态主题切换

```typescript
// 插件中切换主题（canonical 动作名）
await chips.invoke('theme', 'apply', { id: 'chips-official.fluent' });

// 监听主题变化
chips.on('theme.changed', async (event) => {
  // 重新加载 CSS
  const css = await chips.invoke('theme', 'getAllCss');
  updateStyleSheet(css);
});
```

### 5.5 主题全链路事件流（2026-02-27 实现完成）

主题切换后的事件传播链路：

```
1. 插件调用 theme.apply({ id }) → Host theme-service 写入配置
2. theme-service 调用 emitThemeChanged(themeId) → 内核事件总线发布 theme.changed
3. event-forwarder（通配订阅 '*'）自动将 theme.changed 转发到所有渲染进程窗口
4. 各插件 SDK useTheme().onChanged 回调触发
5. viewer 壳层 onThemeChanged 回调 → 通过 bridge 获取新 CSS → broadcastThemeChange 到所有 card iframe
6. card iframe 收到 theme-change 消息 → 更新本地主题样式
```

关键实现点：
- Host `theme-service.ts` 的 `handleApply` 在写配置后调用 `emitThemeChanged()`，发布 `theme.changed` 事件（`propagate: true`）
- `event-forwarder.ts` 通配订阅 `'*'`，自动将点语义事件转发到所有 renderer 窗口
- viewer `card-runtime-protocol-host.ts` 的 `handleReady` 通过 bridge 获取真实主题 CSS/tokens 和 Host locale 注入 init payload
- viewer `card-runtime-protocol-host.ts` 提供 `broadcastThemeChange()` / `broadcastLanguageChange()` 方法向所有已初始化 card iframe 广播变更
- SDK `CardRuntimeMessageType` 包含 `theme-change` / `language-change` 消息类型，与 `chips-template-card` 协议对齐

## 六、Style Dictionary 设计 Token 构建管线

### 6.1 Style Dictionary 的定位

[Style Dictionary](https://amzn.github.io/style-dictionary/)（Apache 2.0，Amazon 开源）是设计 Token 的构建工具。它将一套 Token 定义（JSON/YAML）转换为多种平台的输出格式，是主题包开发的核心工具链。

```
Token 定义文件（JSON/YAML）
  ↓ Style Dictionary 构建
  ├── CSS 自定义属性（Web 端主题包输出）
  ├── TypeScript 常量和类型定义（类型安全引用）
  ├── iOS Swift 常量（未来移动端）
  ├── Android XML 资源（未来移动端）
  └── JSON Token 集合（设计工具/Figma 交换）
```

### 6.2 Token 定义格式

主题包开发者使用 JSON 定义设计 Token，Style Dictionary 自动处理引用关系并输出 CSS：

```json
{
  "chips": {
    "color": {
      "primary": {
        "value": "#6200ee",
        "type": "color",
        "description": "主色调"
      },
      "on-primary": {
        "value": "#ffffff",
        "type": "color"
      },
      "surface": {
        "value": "#ffffff",
        "type": "color"
      }
    },
    "radius": {
      "sm": { "value": "4px", "type": "dimension" },
      "md": { "value": "8px", "type": "dimension" },
      "lg": { "value": "16px", "type": "dimension" }
    },
    "button": {
      "radius": {
        "value": "{chips.radius.md}",
        "type": "dimension",
        "description": "引用语义化 Token，自动解析"
      },
      "primary-bg": {
        "value": "{chips.color.primary}",
        "type": "color"
      }
    }
  }
}
```

Style Dictionary 自动解析 `{chips.radius.md}` 这样的引用，生成最终 CSS：

```css
:root {
  --chips-color-primary: #6200ee;
  --chips-color-on-primary: #ffffff;
  --chips-color-surface: #ffffff;
  --chips-radius-sm: 4px;
  --chips-radius-md: 8px;
  --chips-radius-lg: 16px;
  --chips-button-radius: 8px;        /* 已解析引用 */
  --chips-button-primary-bg: #6200ee; /* 已解析引用 */
}
```

### 6.3 主题包开发流程

```
1. 设计师在 Figma 中定义设计 Token 并导出
      ↓
2. Token 文件保存为 JSON（Style Dictionary 格式）
      ↓
3. 运行 Style Dictionary 构建：
   - 输出 tokens/global.css（全局 Token）
   - 输出 tokens/semantic.css（语义化 Token）
   - 输出 tokens/components/*.css（组件级 Token）
   - 输出 types/tokens.d.ts（TypeScript 类型定义）
      ↓
4. 手工编写组件样式文件（引用生成的 CSS 变量）
      ↓
5. 加入图标集、动画定义、装饰性资源
      ↓
6. 使用 Chips CLI 打包为 .cpk 主题包
      ↓
7. 发布到薯片插件市场
```

### 6.4 Chips Develop CLI 的主题构建集成

```bash
# 创建主题包项目
chipsd create my-theme --template=full-theme

# 项目结构
my-theme/
├── tokens/
│   ├── global.json          # Token 定义（Style Dictionary 格式）
│   ├── semantic.json
│   └── components/
│       ├── button.json
│       ├── input.json
│       └── ...
├── styles/                  # 手写的组件样式
│   ├── button.css
│   ├── input.css
│   └── ...
├── icons/                   # 图标资源
├── animations/              # 动画定义
├── sd.config.js             # Style Dictionary 配置
├── manifest.yaml
└── package.json

# 构建主题包（自动调用 Style Dictionary）
chipsd build
# → 生成 dist/tokens/*.css（从 JSON 构建）
# → 复制 styles/*.css
# → 复制 icons/, animations/
# → 生成 dist/manifest.yaml

# 打包发布
chipsd pack && chipsd publish
```

## 七、图标系统

### 6.1 图标与主题绑定

图标不是全局统一的，而是主题包的一部分。不同的主题包使用不同风格的图标：

| 主题包 | 推荐图标来源 | 风格 |
|--------|------------|------|
| Material Design 3 | Material Symbols | 圆润、填充/线框双模式 |
| Fluent UI | Fluent UI System Icons (MIT) | 清晰、几何 |
| Linear | 自定义 / Lucide Icons (ISC) | 极简线条 |
| Neobrutalism | 自定义 / Phosphor Icons (MIT) | 粗线条 |
| 默认 | Lucide Icons (ISC) | 通用简洁线条 |

### 6.2 图标引用方式

```css
/* 主题包中定义图标映射 */
:root {
  --chips-icon-close: url('./icons/close.svg');
  --chips-icon-menu: url('./icons/menu.svg');
  --chips-icon-search: url('./icons/search.svg');
  --chips-icon-file: url('./icons/file.svg');
  /* ... */
}

/* 或使用图标字体 */
:root {
  --chips-icon-font: 'Material Symbols Outlined';
}
```

组件通过语义化的 CSS 变量引用图标，切换主题时图标自动更换。
