# Chips 生态概述

Chips 是一个内容创作和管理生态系统，核心理念是让复杂内容像打开图片一样简单。用户双击 .card 文件就能自动打开查看，可以把卡片嵌入到网页或其他软件中，可以分享给他人直接使用。

## 核心组件

### 卡片文件

卡片是 Chips 的基本内容单元。一个卡片可以包含文字、图片、视频、音乐、代码、列表、打分组件等各种内容，也可以嵌套其他卡片。

卡片分为基础卡片和复合卡片两种。基础卡片是单一功能的原子组件，比如视频卡片、图片卡片、富文本卡片等，它们不能单独存在。复合卡片是可以保存为 .card 文件的完整卡片，包含一个或多个基础卡片。用户拖拽第一个基础卡片到编辑引擎时，系统自动创建复合卡片容器。复合卡片可以层层嵌套。

### 箱子文件

箱子是卡片的容器，在文件系统中表现为文件夹，包含多个卡片文件和一个配置文件。箱子通过布局插件定义卡片的展示方式，可以是网格布局、瀑布流布局、列表布局、看板布局等。同一个箱子可以切换不同的布局方式，卡片数据不变，只是展示形式改变。

### 社区服务器

社区服务器是服务器端软件，提供多用户社区服务。由存储服务器和社区平台两部分组成。用户可以在社区中创建房间，保存箱子和卡片，浏览其他用户的内容。

### SDK

Chips SDK 让第三方开发者可以将 Chips 能力集成到自己的软件中。开发者可以在博客系统中支持显示卡片文件，在知识库软件中导入 Chips 卡片，在课程平台中用卡片制作课件。

## 软件产品

### 本地客户端

卡片编辑引擎用于创建和编辑卡片内容。编辑引擎本身是插件化的，支持多种布局模式：无限画布布局、工作台布局、页面式布局、看板布局等。用户可以根据工作场景切换不同布局。

卡片查看器用于查看卡片和箱子。查看器包含各种文件查看工具（视频播放器、音乐播放器、图片查看器等），每个工具都是插件，可以替换和扩展。

### 开发工具

软件制作器是可视化组装工具。用户通过拖拽选择前端组件和后端模块，制作器自动生成路由配置，支持低代码开发，产物统一为 `.cpk` 插件包并由薯片主机加载运行。

## 插件系统

Chips 采用"一切皆插件"的设计。基础卡片是插件，箱子布局是插件，编辑引擎布局是插件，查看工具是插件，甚至文件管理器和程序坞也是插件。

插件支持热插拔，大部分插件可以不重启软件即可安装、更新和卸载。用户只需安装需要的插件，不需要的可以卸载。第三方开发者可以开发各种插件发布到应用市场。

每个插件都可以形成独立应用入口体验。比如待办能力可以做成独立入口的应用插件，视频能力可以做成播放器应用插件。它们都由薯片主机加载，数据使用卡片文件保存，不依赖数据库。

## 底层架构

### 薯片主机

薯片主机（Chips Host）是整个生态唯一的底层程序，基于 Electron 技术栈，长期在用户设备后台运行。所有上层软件都是运行在主机内部的 Web 插件应用。对于操作系统来说，始终只识别为一个软件。

### 薯片内核

薯片内核使用 TypeScript 实现，运行在 Electron 主进程中，是整个生态的神经中枢。内核提供中心路由系统、服务注册、事件总线、生命周期管理等核心能力。

所有服务间的通信都必须经过内核路由，服务之间不能直接通信。插件通过 Bridge API（`window.chips.*`）访问系统能力，Bridge API 将请求转发到主进程的内核路由器。内核还负责管理所有插件的生命周期，统一执行生态标准。

### 前后端分离

前端和后端完全解耦。前端只负责界面，后端只负责业务逻辑。它们通过 Bridge API 和内核路由通信，从不直接接触。同一个后端功能可以有多个不同的前端实现，不同开发者可以开发不同风格的界面。前端和后端通过标准接口对接，后端完全不知道前端如何实现。

### 运行模式

采用单一主机模式。薯片主机是唯一安装在系统中的 Electron 应用，所有上层软件以 `.cpk` 格式的插件安装到主机中。每个安装的应用插件在操作系统中创建独立的快捷方式，用户使用体验与普通软件一致。多个插件共享同一个 Chromium 引擎和主进程服务，系统资源占用远少于独立运行多个应用。

## 层次结构

### 内容层次

从小到大依次是：基础卡片 → 复合卡片 → 箱子 → 房间 → 空间 → 社区。

房间是用户在空间内的组织单位，可以嵌套。空间对应物理存储设备，系统自动识别。社区是网络上的公共平台，由社区服务器提供。

### 软件层次

从底层到顶层依次是六层：Electron 宿主 → 平台抽象层（PAL）→ 薯片内核 → 基础服务层 + 共享功能模块 → 插件桥接层（Bridge API）→ 插件应用。

基础服务由内核统一管理，按需加载。共享功能模块（视频播放器、图片查看器等）作为独立的模块插件安装。插件通过 Bridge API 调用底层能力，不直接访问底层依赖。插件分为五种类型：应用插件、卡片插件、布局插件、模块插件、主题插件。

## 跨平台

Chips 基于 Chromium 内核，可以跨平台运行。Web 端直接在浏览器使用，桌面端用 Electron 封装，移动端用 Capacitor 封装。同一套代码，多平台运行。
