# 查看器的插件化设计

查看器是调用公共基础层卡片渲染模块的产品，提供完整的卡片查看体验。当前实现口径下，查看器以应用插件形式运行，由 Chips Host 加载。卡片的核心渲染逻辑在公共基础层的 CardRenderer 模块中，查看器调用这个模块并提供扩展功能，如窗口管理、导航控制、布局切换等。这样设计让任何开发者都可以通过 SDK 调用卡片渲染模块，在自己的产品中实现卡片查看。

公共基础层提供所有基础卡片的渲染器。视频卡片渲染器调用 VideoPlayer 模块，音乐卡片渲染器调用 AudioPlayer 模块，图片卡片渲染器调用 ImageViewer 模块。所有这些渲染逻辑都在基础层实现，查看器只需要调用 CardRenderer 引擎即可。

查看器的扩展功能包括：窗口管理（最小化、全屏、分屏），导航控制（前进后退、快捷跳转），布局切换（简洁查看、沉浸式查看），收藏和历史记录。这些是查看器产品层面的功能，不是渲染层面的。

查看器的界面布局也是插件化的。默认布局是简洁查看模式，全屏显示卡片内容，底部有工具栏，侧边栏可隐藏显示卡片层级结构。未来可以有多窗口查看布局，支持分屏和标签页。还可以有沉浸式查看布局，全屏无工具栏，通过手势控制。

查看器和编辑引擎的关系有两种设计方案。一种是作为两个独立入口的应用插件，查看器轻量级只负责查看，编辑引擎功能完整负责编辑。另一种是作为同一应用插件的两种模式，编辑引擎包含查看模式，通过锁定按钮切换。目前倾向于后者，但保留前者作为可选方案。也就是说可以有一个轻量级的查看器应用插件，同时编辑引擎内也包含查看模式。

当卡片包含视频文件时，内核识别需要视频播放器，查找所有支持该格式的查看工具插件，使用用户设置的默认播放器。如果用户没有设置，使用系统默认播放器。内核调用插件的渲染接口，插件创建播放器界面并显示。如果安装了多个支持相同格式的插件，用户可以临时切换使用哪个。
