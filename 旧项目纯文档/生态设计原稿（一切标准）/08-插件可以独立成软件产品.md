# 插件可以独立成软件产品

这里的“独立成软件产品”指的是**独立应用入口与独立业务形态**，不是在操作系统里再安装一个新的底层程序。Chips 生态始终只有一个宿主软件（Chips Host），所有产品都以插件方式运行在主机内部。

传统软件常常“每个产品一套运行时 + 一套底层依赖 + 一套数据系统”。Chips 生态改为“共享底层 + 插件化业务层 + 文件化数据层”：产品能力来自插件，数据主要使用卡片或箱子文件保存。

以待办软件为例，传统的待办软件需要设计界面、实现业务逻辑、设计数据库结构、实现数据存储，然后打包成软件。而基于 Chips 生态开发待办软件，核心就是待办列表基础卡片的编辑组件插件。这个插件加上一个轻量级的窗口框架，数据保存为 .card 文件，就是一个完整的待办软件。

同样的，图片查看器产品可以由图片能力插件 + 应用壳插件构成；视频播放器产品可以由视频能力插件 + 应用壳插件构成。每个产品的核心能力都可拆为标准插件模块。

用卡片文件保存数据替代私有数据库，让数据成为标准文件格式并可复制分享。同一个待办卡片文件，可以在待办应用插件中打开，也可以在卡片编辑引擎中打开，还可以嵌入到其他复合卡片中。不同插件应用可以读取相同的数据文件。

一个插件可以被多个应用入口复用。待办列表插件既可以在待办应用插件中使用，也可以在编辑引擎中作为基础卡片使用，还可以在社区平台上使用。插件代码只需要开发一次，可以在多处运行。

每个“软件产品形态”本质上都是一个应用插件，运行在薯片主机（Chips Host）中。它通过 Bridge API 调用底层的窗口管理、文件操作、卡片读写等能力。应用插件只需要实现界面布局和业务编排，底层重能力由主机基础服务完成。核心卡片能力由对应卡片插件提供。

插件应用之间的互通性很强。用户在待办应用插件中创建的待办卡片，可以在编辑引擎中打开并嵌入到其他卡片中。在编辑引擎中创建的包含待办卡片的复合卡片，也可以在待办应用插件中编辑其中的待办部分。数据格式统一，产品入口可以自由互通。
