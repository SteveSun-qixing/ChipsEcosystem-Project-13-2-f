# 标签文件系统设计

薯片生态采用全新的标签文件系统，打破传统的树状目录结构。整个文件系统是一个大的资源池，通过标签进行层层筛选来组织和查找文件。

传统的文件系统使用固定的树状结构，一个文件只能存在于一个路径下。薯片的标签文件系统中，一个卡片可以拥有非常多的标签，这些标签从不同维度描述这个卡片。用户通过选择标签组合来筛选文件，不同的标签组合会呈现不同的文件组织视图。

标签不只是简单的分类词，可以是键值对结构。一个标签由分类方法和具体值组成。比如"月份：三月"、"创作者：小明"、"项目：网站改版"、"评分：五星"等。这种结构化标签让筛选更加精确和灵活。

文件路径在标签系统中的表现形式是标签的叠加筛选路径。用户在资源管理器中依次选择标签，每选择一个标签就是一次筛选，最终到达包含目标文件的结果集。比如用户先选择"月份：三月"，显示所有三月的文件。再选择"创作者：小明"，显示三月且创作者是小明的文件。再选择"项目：网站改版"，最终筛选出符合这三个标签的文件。

这个筛选路径可以看作是传统意义上的"文件路径"，但它不是固定的树状结构，而是动态的筛选结果。同一个文件可以通过不同的标签路径到达。用户可以通过"月份 → 创作者 → 项目"到达某个文件，也可以通过"创作者 → 项目 → 月份"到达同一个文件。

用户在某个筛选路径下创建新文件时，这个路径上的所有标签会自动添加到新文件中。比如用户在"三月 → 小明 → 网站改版"路径下创建新卡片，这个卡片自动获得"月份：三月"、"创作者：小明"、"项目：网站改版"三个标签。用户也可以手动修改或添加更多标签。

资源管理器中的"文件夹"视图是动态生成的，基于当前的筛选条件和下一级可用的标签。用户选择了"三月"标签后，系统检索所有三月的文件，分析这些文件还有哪些其他标签，然后按某个维度（比如创作者）分组显示。用户看到的是多个"文件夹"，每个代表一个创作者，但这些不是真实的文件夹，而是标签筛选的结果。

用户可以自定义筛选路径的顺序。在资源管理器的设置中，可以设置默认的标签展示顺序，比如"月份 → 项目 → 创作者"。也可以临时调整顺序，比如切换成"创作者 → 月份 → 项目"，文件的组织视图立即改变，但文件本身和标签都没有变化。

标签的层级关系是灵活的。同一个标签既可以作为第一级分类，也可以作为第二级或第三级分类。系统根据当前已选择的标签，智能推荐下一级可能的标签。比如已经选择了"月份：三月"，系统分析三月的文件，发现主要有"项目"、"创作者"、"类型"等标签维度，推荐给用户选择。

标签可以嵌套。一个标签可以有子标签。比如"地点"标签下可以有"亚洲"、"欧洲"等子标签，"亚洲"下又可以有"中国"、"日本"等子标签。用户选择"地点：亚洲：中国"，系统筛选出符合这个三级标签的文件。

用户可以保存常用的筛选路径作为快捷访问。比如经常查看"三月 → 网站改版 → 未完成"的文件，可以保存这个路径为"三月网站进度"。下次点击这个快捷方式，直接显示筛选结果。

搜索功能与标签系统结合。用户搜索关键词时，系统不仅搜索文件名和内容，还搜索标签。搜索结果可以按标签分组显示，用户可以进一步用标签筛选搜索结果。

标签系统支持标签的批量编辑。用户可以选中多个文件，批量添加标签、删除标签、修改标签值。可以创建标签规则，符合某些条件的文件自动添加特定标签。

视图模式让用户可以在不同的组织方式间切换。除了标签筛选视图，还可以有时间线视图（按创建时间排列）、最近使用视图、收藏视图等。所有视图都是同一份文件，只是展示方式不同。

## 文件系统映射

标签文件系统在软件内部建立用户文件系统的映射副本。用户授予软件访问权限后，软件读取指定范围内的文件信息，在内部建立映射关系。

映射采用简单的文件名匹配机制。软件记录每个文件的路径、文件名、修改时间等基本信息，不需要计算哈希值。文件变化时，通过文件名和修改时间判断是否是同一文件。

每个文件在软件中都有独立的标签记录。用户为文件添加的标签保存在软件的标签数据库中，不修改原文件。标签数据库记录文件路径和对应的标签列表，建立映射关系。

对于薯片生态的 .card 和 .box 文件，标签信息直接读取文件的 metadata.yaml，不需要额外记录。对于其他类型的文件，标签保存在软件的标签数据库中。

用户在软件中看到的文件结构是动态生成的虚拟视图。根据当前选择的标签，系统从标签数据库查询符合条件的文件，按某个标签维度分组显示。用户看到的"文件夹"是标签分组，不是真实的文件夹。

映射模式的优势在于不污染用户的文件系统。所有标签信息、索引数据、缓存文件都保存在软件自己的工作文件夹中。用户原本的文件系统保持干净，不会出现各种辅助文件散落在各处。用户卸载软件后，只需要删除软件的工作文件夹即可。

用户也可以选择直接创建模式。在这种模式下，薯片文件直接创建在用户文件系统的当前位置。标签信息保存在每个文件夹的隐藏配置文件中，或者对于薯片文件直接保存在 metadata.yaml 中。

两种模式可以共存。用户可以为不同的文件夹设置不同的模式。某些文件夹使用映射模式保持干净，某些文件夹使用直接创建模式便于与其他软件共享。
