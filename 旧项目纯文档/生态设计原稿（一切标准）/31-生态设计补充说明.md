# 生态设计补充说明

## 本地软件的简化设计

薯片生态的本地软件（编辑引擎、查看器、资源管理器等）不包含身份认证、用户管理、权限系统、秘钥管理等功能。这些是社区服务器的职责，本地软件保持简单。

本地软件只负责内容的创建、编辑、查看和组织。所有数据保存为文件，不需要用户注册登录。用户的文件就是用户的数据，完全本地化。

网络资源的认证信息（如 WebDAV 的用户名密码）保存在软件的本地配置中，由秘钥管理工具统一管理，但这不是用户身份系统，只是资源访问凭证的管理。

## SDK 的能力封装

开发工具包封装绝大部分底层逻辑和通用能力。第三方开发者只需要编写极少的代码，主要是简单的界面调用，即可实现复杂功能。

比如开发者想创建一个视频播放器软件，不需要自己实现视频解码、播放控制、字幕渲染等功能。通过 SDK 调用公共基础层的 VideoPlayer 模块，几行代码就能实现完整的视频播放器。开发者只需要设计界面和用户体验部分。

SDK 提供的封装包括：文件操作封装、渲染功能封装、媒体播放封装、界面组件封装、数据处理封装、网络请求封装。开发者通过简单的 API 调用就能使用这些能力，不需要了解底层实现细节。

## 薯片协议

生态定义了一套薯片协议，规范文件格式、接口标准、通信协议等方面。

文件格式协议定义了 .card 和 .box 文件的结构，metadata.yaml、structure.yaml 等配置文件的格式，资源引用的路径规范。

接口标准协议定义了前端接口标准、后端接口标准、模块调用协议、资源访问协议、事件通知协议。

通信协议规定了模块间通信的请求响应格式，路由调用的标准，错误处理的规范。

所有协议文档保存在"生态共用"文件夹中，所有项目都遵循这些协议，确保整个生态的一致性和互操作性。
