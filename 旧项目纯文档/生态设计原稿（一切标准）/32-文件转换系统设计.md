# 文件转换系统设计

## 系统定位

文件转换系统是薯片生态的核心基础设施之一，负责卡片文件与其他文件格式之间的双向转换。系统采用统一接口加插件化实现的架构，公共基础层提供转换接口，具体的转换逻辑由独立插件实现。

文件转换系统只负责文件格式之间的转换，不负责文件的创建或存储。卡片文件的制作由卡片打包器模块负责，文件的存储由文件系统模块负责。转换系统专注于格式转换这一单一职责。

## 架构设计

文件转换系统分为两个层次。公共基础层提供文件转换接口模块，这是一个统一的接口层，负责管理转换插件的注册、路由转换请求、查询转换能力。具体的转换逻辑由独立的转换插件实现，每种转换类型都是一个单独的插件项目。

文件转换接口模块本身不包含任何具体的转换实现。它只定义转换的标准接口，管理已注册的转换插件，根据请求的转换类型将请求路由到对应的插件。插件在启动时向接口模块注册自己，声明支持的源文件类型和目标文件类型。

应用层通过薯片 SDK 的转换 API 发起转换请求。SDK 封装了与微内核的通信细节，应用只需要指定源文件和目标格式，SDK 将请求通过微内核路由到文件转换接口模块，接口模块再路由到具体的转换插件执行转换。

## 转换插件

每种转换类型都是独立的插件，作为单独的项目开发和维护。所有卡片相关的转换插件统一放在 Chips-CardConversionPlugin 项目中，包含多个子插件。

CardtoHTMLPlugin 负责将卡片文件转换为完整的 HTML 网页。转换时读取卡片的元数据和结构信息，遍历所有基础卡片，获取对应的前端渲染代码。为每个基础卡片生成独立的 HTML 文件，然后生成主入口页面，使用 iframe 嵌套所有基础卡片，完整还原卡片的层级结构。资源文件复制到输出目录，主题样式提取并应用。最终输出一个可以离线浏览的完整网页目录。

CardtoImagePlugin 负责将卡片渲染为图片文件。使用内置的渲染引擎在内存中渲染卡片内容，然后截取渲染结果输出为图片格式。支持 PNG 和 JPG 两种格式，可以配置图片质量、分辨率和缩放比例。

CardtoPDFPlugin 负责将卡片转换为 PDF 文档。首先渲染卡片内容，然后转换为 PDF 格式。支持配置页面大小、方向、边距，可以选择是否包含封面页和目录。

未来可以扩展更多转换插件，比如将网页转换为卡片、将 Markdown 转换为卡片、将卡片导出为 Markdown 等。第三方开发者也可以开发自己的转换插件，只要遵循转换插件的接口标准即可注册到系统中使用。

## 插件注册

转换插件作为独立模块启动后，首先向微内核注册自己。文件转换接口模块通过微内核发现已注册的转换插件，调用插件的注册接口获取插件的能力信息，包括插件支持的源文件类型、目标文件类型、转换选项等。接口模块记录这些信息，建立转换类型到插件的映射关系。

应用层查询支持的转换类型时，接口模块返回所有已注册插件声明的转换能力。发起转换请求时，接口模块根据源类型和目标类型查找对应的插件，将请求转发给插件执行。

插件卸载时，向接口模块注销自己，接口模块移除该插件的转换能力记录。如果某种转换类型没有对应的插件，转换请求会返回不支持的错误。

## 转换流程

应用通过 SDK 发起转换请求，指定源文件路径或数据、目标格式、转换选项。SDK 将请求封装为标准格式，通过微内核路由到文件转换接口模块。

接口模块解析请求，根据源类型和目标类型查找对应的转换插件。如果找到插件，验证转换选项的有效性，然后调用插件的转换方法。插件执行具体的转换逻辑，可能需要读取源文件、解析内容、渲染处理、生成目标格式、写入输出。

转换完成后，插件返回转换结果，包括是否成功、输出路径或数据、转换统计信息。接口模块将结果通过微内核返回给 SDK，SDK 再返回给应用。整个过程中，应用可以通过任务 ID 查询转换进度，也可以取消正在进行的转换任务。

## HTML 导出结构

卡片转换为 HTML 时，输出一个完整的目录结构。根目录有一个 index.html 作为主入口文件，这个页面包含卡片的基本框架，使用 iframe 嵌套所有基础卡片。

cards 子目录存放每个基础卡片的独立 HTML 文件，文件名使用基础卡片的 ID。每个基础卡片的 HTML 包含该卡片的完整渲染内容，可以独立浏览。

assets 子目录存放所有资源文件，包括图片、视频、音频等媒体文件，以及样式文件。根目录的 theme.css 包含主题样式定义。

主入口页面的结构是一个容器元素，内部按顺序排列多个 iframe 元素，每个 iframe 加载一个基础卡片的 HTML 文件。这种 iframe 嵌套的方式完整还原了卡片的层级结构，每个基础卡片保持独立的样式和脚本作用域。

## 扩展机制

文件转换系统设计为高度可扩展。新增转换类型只需要开发新的转换插件，注册到接口模块即可，不需要修改接口模块或其他插件的代码。

同一种转换类型可以有多个插件实现，比如可以有官方的 CardtoHTMLPlugin，也可以有第三方开发的增强版本。用户可以选择使用哪个插件，或者由系统根据某种策略自动选择。

转换插件可以在应用市场发布和安装。用户根据需要安装转换插件，安装后自动注册到系统中，应用就可以使用新的转换能力。卸载插件后对应的转换能力自动移除。
