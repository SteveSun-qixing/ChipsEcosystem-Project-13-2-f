## 核心职责

生态设置面板负责四类核心工作：

第一，初始化安装。用户可选择下载源或手动导入安装包，系统自动探测设备环境并匹配安装方案，然后按依赖顺序安装必要的插件套装。

第二，运行治理。系统统一管理生态进程启动、停止、重启和健康巡检，并提供日志查询、错误筛选和报告导出。

第三，全局设置。集中管理语言系统、主题系统、插件系统、基础卡片与箱子套装、文件系统交换策略。

第四，生态协作桥接。作为生态内外文件交换和策略落地入口，统一约束链接复制策略、工作区目录、导入导出行为。

## 与生态层次的关系

生态设置面板是一个标准的应用插件，运行在薯片主机的 BrowserWindow 中。它通过 Bridge API 访问底层能力，与其他应用插件使用相同的通信方式。它对接的底层部分包括：

- 薯片内核：通过 Bridge API 调用，所有操作必须走中心路由。
- 基础服务层：通过 Bridge API 调用通用能力，不直接访问内部实现。
- 薯片 SDK：作为 Bridge API 的类型安全封装，简化开发。
- 薯片组件库（无头组件）：提供统一 UI 交互能力，样式由主题包注入。

这意味着生态设置面板必须成为"规则执行者"，而不是另一个耦合点。它不能破坏中心路由原则，也不能跨层直连其他服务模块。

## 首版功能结构

首版设置面板分为左右双栏，左侧是菜单域，右侧是设置域详情。首批设置域包括：

1. 生态进程管理（含高级日志与报告）
2. 语言管理（切换与词典更新）
3. 主题管理（安装、切换、删除）
4. 插件管理（安装、更新、停用）
5. 基础卡片与箱子套装管理（必装域，仅可整体更新和修复）
6. 文件系统管理（工作区与生态内外交换策略）

后续新增设置域采用可扩展注册方式，不改变主框架结构。

## 架构原则

生态设置面板在研究期就要遵守以下原则：

- 首版优先长期架构，不做临时方案。
- 不考虑历史兼容重构，优先保证结构正确。
- 所有关键流程可观测、可回滚、可追踪。
- 模块边界清晰，避免"巨石设置页"。

只有这样，生态设置面板才能支撑薯片生态长期稳定演进，而不是成为新的技术债入口。
