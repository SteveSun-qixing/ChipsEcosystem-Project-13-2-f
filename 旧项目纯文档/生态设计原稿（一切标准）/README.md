## 核心设计理念

- **中心路由**：所有模块通过薯片内核通信
- **单一主机**：用户设备只有一个 Chips Host，所有上层产品都以插件运行
- **一切皆插件**：基础卡片、布局、编辑引擎布局、查看工具都是插件
- **前后端分离**：前端可以自由更换，后端完全框架无关
- **文件优先**：用文件保存数据，不依赖数据库
- **极致模块化**：每个功能都是独立模块
- **原子级主题**：每个组件可以单独设置主题

## 层次结构

**内容层次**：基础卡片 → 复合卡片 → 箱子 → 房间 → 空间 → 社区

**软件层次**：Electron 宿主 → 平台抽象层（PAL）→ 薯片内核 → 基础服务层 + 共享功能模块 → 插件桥接层（Bridge API）→ 插件应用
