
## 版本管理

格式版本记录在chip_standards_version字段中，遵循语义化版本规范。当前版本为1.0.0。

版本兼容性规则：主版本变更表示不兼容的格式变化，次版本变更表示向后兼容的功能新增，修订号变更表示向后兼容的问题修正。

解析器需要检查版本号以决定如何处理文件内容。

## 打包解包API

卡片服务提供卡片打包器模块。

### 核心服务动作

根据架构手册第4.5节服务动作面基线，卡片服务必须实现以下动作：

| 动作名 | 输入 | 输出 | 幂等性 | 权限 | 典型错误码 |
|---|---|---|---|---|---|
| `pack` | cardDir, outputPath | cardFile | 否 | `card.write` | `CARD_PACK_FAILED` |
| `unpack` | cardFile, outputDir | dir | 否 | `card.read` | `CARD_UNPACK_FAILED` |
| `validate` | cardFile | validationResult | 是 | `card.read` | `CARD_SCHEMA_INVALID` |
| `readMetadata` | cardFile | metadata | 是 | `card.read` | `CARD_NOT_FOUND` |
| `parse` | cardFile/buffer | CardAST | 是 | `card.read` | `CARD_PARSE_FAILED` |
| `render` | cardDoc/AST, themeId?, locale? | RenderedView | 否 | `card.read` | `CARD_RENDER_FAILED` |

### API 详细说明

- **pack(cardDir, outputPath)**：将卡片文件夹打包为card文件，使用零压缩模式
- **unpack(cardFile, outputDir)**：解压card文件到指定目录
- **validate(cardFile)**：验证card文件结构完整性，包括schema校验和版本兼容性检查
- **readMetadata(cardFile)**：在不解包的情况下读取元数据信息
- **parse(cardFile)**：解析card文件为抽象语法树（CardAST），供编辑器和查看器使用
- **render(cardDoc, themeId?, locale?)**：使用指定主题和语言渲染卡片内容，返回可显示的视图对象

### 错误响应格式

所有卡片服务动作遵循统一错误格式：
```ts
{ code: string; message: string; details?: unknown; retryable?: boolean }
```
