## 2. 箱子的正式业务定位

箱子是卡片引用集合的消费与组织载体，不是卡片的变体，也不是卡片实体的打包容器。

箱子的正式业务职责包括：

1. 记录卡片引用；
2. 记录卡片摘要快照；
3. 保存布局配置；
4. 保存布局自有资源；
5. 为查看器、编辑器、接口服务提供统一消费入口。

箱子不承担以下职责：

1. 不保存被引用卡片实体；
2. 不负责直接扫描文件系统；
3. 不负责直接访问网络凭证；
4. 不由布局插件直接实现底层资源解析。

---

## 3. 正式职责边界

### 3.1 Host

Host 是箱子消费链路的唯一运行时承载，负责：

1. 解析 `.box` 文件；
2. 维护箱子查看会话；
3. 根据卡片 URL 动态定位和加载卡片信息；
4. 处理本地文件、网络访问、凭证、缓存和失败状态；
5. 向布局插件提供摘要数据和按需资源；
6. 统一回收会话、缓存和临时资源。

### 3.2 布局插件

布局插件是箱子的前端界面层，负责：

1. 展示箱子页面；
2. 按布局算法组织卡片；
3. 优先消费卡片封面、名称、标签、摘要等信息；
4. 在需要更多信息时通过正式契约向 Host 申请；
5. 渲染布局自有资源。

布局插件不负责：

1. 直接扫描磁盘；
2. 直接访问网络；
3. 直接读取凭证；
4. 直接打开卡片文件路径。

### 3.3 查看器 / 应用插件

查看器或其他应用插件负责承载布局插件输出，不负责自己重新实现一套箱子资源定位逻辑。

### 3.4 编辑引擎

编辑引擎在创作阶段负责维护：

1. `metadata.yaml`
2. `structure.yaml`
3. `content.yaml`
4. 箱子自有资源

正式写回路径应使用工作目录 + `file` 服务编辑，再通过 `box.pack` 生成最终 `.box` 文件。

## 4. 箱子业务对象

### 4.1 箱子文件

正式产物是单个 `.box` 文件，内部保存：

1. 元数据；
2. 卡片引用清单；
3. 卡片摘要；
4. 布局配置；
5. 布局资源。

### 4.2 卡片引用条目

每个箱子条目至少包含：

1. 稳定的 `entry_id`；
2. 目标卡片 `url`；
3. 摘要快照；
4. 启用状态；
5. 布局索引信息。

### 4.3 箱子资源

箱子自有资源是箱子文件的一部分，但不等于卡片资源。  
常见类型包括：

1. 布局背景图；
2. 占位图；
3. 装饰素材；
4. 箱子内缓存的摘要封面；
5. 布局私有配置附属文件。

## 5. 正式消费流程

### 5.1 打开箱子

1. 查看器请求 Host 打开 `.box`；
2. Host 校验包结构；
3. Host 读取 `metadata.yaml`、`structure.yaml`、`content.yaml`；
4. Host 创建箱子查看会话；
5. Host 返回箱子基础信息、首批摘要条目和布局配置；
6. 查看器加载对应布局插件；
7. 布局插件开始渲染摘要页面。

### 5.2 布局动态取数

1. 布局插件先使用摘要态数据渲染首屏；
2. 当用户滚动、悬停、切换视图或打开详情时，布局插件再按需向 Host 请求：
   - 更多条目；
   - 指定条目的详细信息；
   - 封面资源；
   - 预览资源；
   - 箱子自有资源；
3. Host 执行定位、加载、缓存和失败归一；
4. Host 返回结构化数据或资源句柄；
5. 布局插件完成增量更新。

### 5.3 切换布局

1. 查看器或布局设置面板切换 `layout_type`；
2. Host 读取对应布局配置；
3. 原箱子会话继续存在；
4. 新布局插件复用同一份箱子条目数据与运行时能力；
5. 布局自有资源按新布局配置重新申请。

### 5.4 关闭箱子

1. 查看器通知 Host 关闭箱子会话；
2. Host 回收会话上下文；
3. Host 释放临时资源、缓存引用和监听器。

## 6. 正式服务动作面

箱子服务分为两类动作：静态文件动作、运行时消费动作。

### 6.1 静态文件动作

| 动作名 | 输入 | 输出 | 幂等性 | 权限 | 典型错误码 |
|---|---|---|---|---|---|
| `pack` | `boxDir`, `outputPath` | `boxFile` | 否 | `box.write` | `BOX_PACK_FAILED` |
| `unpack` | `boxFile`, `outputDir` | `outputDir` | 否 | `box.write` | `BOX_UNPACK_FAILED` |
| `inspect` | `boxFile` | `inspection` | 是 | `box.read` | `BOX_FORMAT_INVALID` |
| `validate` | `boxFile` | `validationResult` | 是 | `box.read` | `BOX_SCHEMA_INVALID` |
| `readMetadata` | `boxFile` | `metadata` | 是 | `box.read` | `BOX_NOT_FOUND` |

### 6.2 运行时消费动作

| 动作名 | 输入 | 输出 | 幂等性 | 权限 | 典型错误码 |
|---|---|---|---|---|---|
| `openView` | `boxFile`, `layoutType?`, `initialQuery?` | `session` | 否 | `box.read` | `BOX_OPEN_FAILED` |
| `listEntries` | `sessionId`, `query` | `pagedEntries` | 是 | `box.read` | `BOX_VIEW_SESSION_NOT_FOUND` |
| `readEntryDetail` | `sessionId`, `entryIds`, `fields` | `entryDetails` | 是 | `box.read` | `BOX_ENTRY_NOT_FOUND` |
| `resolveEntryResource` | `sessionId`, `entryId`, `resourceRequest` | `resourceHandle` | 是 | `box.read` | `BOX_RESOURCE_NOT_FOUND` |
| `readBoxAsset` | `sessionId`, `assetPath` | `resourceHandle` | 是 | `box.read` | `BOX_ASSET_NOT_FOUND` |
| `prefetchEntries` | `sessionId`, `entryIds`, `targets` | `ack` | 否 | `box.read` | `BOX_PREFETCH_FAILED` |
| `closeView` | `sessionId` | `ack` | 是 | `box.read` | `BOX_VIEW_SESSION_NOT_FOUND` |

### 6.3 设计说明

- `inspect` 用于静态检查和工具链集成；
- `openView` 之后的所有取数都必须围绕 `sessionId`；
- `resolveEntryResource` 用于封面、预览图、卡片入口资源等动态获取；
- `readBoxAsset` 只读取箱子自有资源，不读取卡片文件内部资源；
- 运行时动作默认只向当前会话所属的查看器 / 布局插件开放。

## 7. 数据最小化原则

箱子业务链路必须遵守以下最小化原则：

1. 首屏只返回摘要态数据；
2. 不因箱子打开就强制解析全部卡片；
3. 不直接暴露凭证；
4. 不直接暴露文件系统绝对路径给布局插件；
5. 不把卡片内部私有资源整体透传给布局插件；
6. 只在需要时返回对应字段和资源。

---

## 8. 写入与打包原则

首版正式写入链路遵循：

1. 编辑器基于解包工作目录读写箱子文件；
2. 结构、配置和资源由 `file` 服务维护；
3. 正式导出时调用 `box.pack`；
4. 正式消费时调用 `box.openView`；
5. 不保留旧的“内部卡片 / 外部卡片 / 三种箱子类型”写法。

## 9. 典型错误码

| 错误码 | 含义 |
|---|---|
| `BOX_FORMAT_INVALID` | `.box` 结构不合法 |
| `BOX_SCHEMA_INVALID` | 元数据、结构或内容配置不符合规范 |
| `BOX_PACK_FAILED` | 打包失败 |
| `BOX_UNPACK_FAILED` | 解包失败 |
| `BOX_NOT_FOUND` | 箱子文件不存在 |
| `BOX_OPEN_FAILED` | 查看会话创建失败 |
| `BOX_VIEW_SESSION_NOT_FOUND` | 会话不存在或已关闭 |
| `BOX_ENTRY_NOT_FOUND` | 指定条目不存在 |
| `BOX_RESOURCE_NOT_FOUND` | 指定条目资源不存在 |
| `BOX_ASSET_NOT_FOUND` | 指定箱子资源不存在 |
| `BOX_URL_SCHEME_UNSUPPORTED` | URL 协议未被当前 Host 支持 |
| `BOX_RESOLVE_FAILED` | URL 定位或加载失败 |
| `BOX_LAYOUT_ACCESS_DENIED` | 当前布局插件无权访问该会话 |

## 10. 与其他服务域的关系

| 服务域 | 关系 |
|---|---|
| `file` | 维护编辑工作目录与导出路径 |
| `card` | 解析卡片元数据、封面与卡片运行时资源 |
| `resource` | 返回统一资源句柄与缓存入口 |
| `credential` | 为网络 URL 解析提供凭证能力 |
| `plugin` | 解析并加载对应布局插件 |
| `window` | 承载查看器与布局页面 |

---

## 11. 正式结论

首版正式箱子业务基线如下：

1. 箱子是引用容器，不是卡片实体容器；
2. Host 是唯一解析与调度中心；
3. 布局插件是前端页面，不是文件解析器；
4. 箱子消费默认摘要优先、按需加载；
5. 箱子自有资源与卡片资源严格分层；
6. 不保留旧的内部/外部三种箱子模型。
