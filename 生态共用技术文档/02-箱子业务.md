
## 打包解包API

箱子服务提供箱子打包器模块。

### 核心服务动作

根据架构手册第4.5节服务动作面基线，箱子服务必须实现以下动作：

| 动作名 | 输入 | 输出 | 幂等性 | 权限 | 典型错误码 |
|---|---|---|---|---|---|
| `pack` | boxDir, outputPath | boxFile | 否 | `box.pack` | `BOX_PACK_FAILED` |
| `unpack` | boxFile, outputDir | dir | 否 | `box.pack` | `BOX_UNPACK_FAILED` |
| `inspect` | boxFile | artifact/meta | 是 | `box.read` | `BOX_FORMAT_INVALID` |
| `validate` | boxFile | validationResult | 是 | `box.read` | `BOX_SCHEMA_INVALID` |
| `readMetadata` | boxFile | metadata | 是 | `box.read` | `BOX_NOT_FOUND` |

### API 详细说明

- **pack(boxDir, outputPath)**：将箱子文件夹打包为box文件，使用零压缩模式
- **unpack(boxFile, outputDir)**：解压box文件到指定目录
- **inspect(boxFile)**：检查箱子内部结构，返回卡片列表和资源统计信息（幂等操作）
- **validate(boxFile)**：验证box文件结构完整性，包括schema校验和版本兼容性检查
- **readMetadata(boxFile)**：在不解包的情况下读取元数据信息

### 错误响应格式

所有箱子服务动作遵循统一错误格式：
```ts
{ code: string; message: string; details?: unknown; retryable?: boolean }
`
