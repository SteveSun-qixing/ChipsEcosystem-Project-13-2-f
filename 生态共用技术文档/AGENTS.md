# 生态共用技术文档 AGENTS

## 角色

- 这里保存整个薯片生态通用、稳定、正式、可复用或对外公开的技术信息。
- 协议、契约、公共接口、主题系统、多语言规范、组件标准、文件格式、公共架构设计，都应优先在这里沉淀。

## 编写规则

- 修改前必须重新核对相关代码仓、`manifest.yaml`、测试和设计原稿，确认当前口径真实生效。
- 如果一个信息只在单一项目内部流转，不要写到这里，改写到对应项目内部技术文档。
- 不要把任务计划、工单过程、人工排查笔记、临时 workaround、短期分支策略写进这里。
- 不要把未来可能频繁变动的业务策略、局部实现细节或仅当前版本有效的临时参数固化为生态公共规范。
- 若公共契约发生变化，应同步核对并推动受影响项目与文档一起收口。

## 质量要求

- 文档必须明确哪些内容是正式对外口径，哪些内容引用自其他仓库的内部实现。
- 同一类公共规范只保留一个主发布位置，避免在多个目录重复定义互相冲突的版本。
