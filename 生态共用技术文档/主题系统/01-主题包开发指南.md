# 主题包开发指南

## 概述

主题包是定义生态视觉外观的插件类型。主题包通过 CSS 变量和类名定义界面的颜色、字体、圆角、阴影、动画等视觉属性。主题包可以完全改变软件的视觉风格而不影响功能逻辑。

## 文件结构

主题包的 CPK 文件包含以下组成部分：

```
theme-package/
├── manifest.yaml                   # 主题包配置文件
├── tokens/                         # 设计令牌目录
│   ├── global.css                  # 全局设计 Token（色板、字体、间距）
│   ├── semantic.css                # 语义化 Token（primary、secondary、error）
│   ├── typography.css              # 排版 Token（标题、正文、标注）
│   ├── motion.css                  # 动画 Token（曲线、时长）
│   ├── global.json                 # Token 定义源文件（Style Dictionary 格式）
│   ├── semantic.json
│   └── components/                 # 组件级 Token
│       ├── button.json
│       ├── input.json
│       └── ...
├── components/                      # 组件样式目录
│   ├── button.css                  # 按钮的完整视觉实现
│   ├── input.css                   # 输入框
│   ├── modal.css                   # 弹窗
│   ├── tabs.css                    # 标签页
│   ├── card-wrapper.css            # 卡片容器
│   ├── dropdown.css                # 下拉菜单
│   └── ...                         # 所有组件
├── animations/                     # 动画定义目录
│   ├── transitions.css             # 过渡动画
│   ├── keyframes.css               # 关键帧动画
│   └── interactions.js             # 交互动画（可选 JS 增强）
├── icons/                          # 图标资源目录
│   ├── icon-font.woff2             # 图标字体（可选）
│   ├── icons.css                   # 图标类名定义
│   └── svg/                        # SVG 图标
│       ├── close.svg
│       ├── menu.svg
│       └── ...
├── decorations/                    # 装饰资源目录
│   ├── backgrounds/                # 背景图片/纹理
│   ├── patterns/                   # 装饰图案
│   └── cursors/                    # 自定义光标（可选）
├── fonts/                         # 自带字体（可选）
│   ├── Inter-Regular.woff2
│   └── Inter-Bold.woff2
├── contracts/                      # 契约目录
│   └── theme-interface.contract.json  # 主题接口契约
└── preview/                        # 预览资源目录
    ├── preview.png                # 主题预览截图
    └── preview.html               # 预览页面（可选）
```

## manifest 配置

主题包的 manifest.yaml 有特定的配置要求：

```yaml
schemaVersion: "1.0.0"
id: theme.theme.chips-official-default-theme
name: 薯片官方 · 默认主题
type: theme
entry:
  tokens: dist/tokens.json
  themeCss: dist/theme.css
themeId: chips-official.default-theme  # 主题唯一标识符
displayName: 默认主题                  # 显示名称
version: 1.0.0                      # 语义化版本
isDefault: false                    # 是否为默认主题
parentTheme: ""                     # 可选，父主题实现继承
ui:
  layout:
    owner: page
    unit: cpx                      # 逻辑尺寸单位
    baseWidth: 1024                 # 基准宽度
    contract: ./contracts/theme-interface.contract.json
    minFunctionalSet: ./contracts/theme-min-functional-set.json
```

其中：

- `entry.tokens` 必须指向构建产物 `dist/tokens.json`；
- `entry.themeCss` 必须指向构建产物 `dist/theme.css`；
- Host Theme Runtime 只消费 `dist/` 中的正式产物，不直接读取源目录 `tokens/` 或 `styles/`。

### 主题包语义约束

> **重要约束**：一个主题包仅承载一种外观，不在包内区分白天/夜间、light/dark 或其他模式标签。外观切换通过切换 `themeId` 到另一个主题包完成。

## 设计令牌（Token 体系）

设计令牌是主题的基础构建块，采用分层架构设计。Token 定义使用 JSON 格式（Style Dictionary 标准），构建时自动生成 CSS 变量。

> **注意**：薯片生态采用**自管理前端架构**，组件库只负责功能逻辑、结构和布局，完全不包含样式。所有视觉表现由主题包通过 CSS 变量提供。

### Token 文件结构

```
tokens/
├── global.json       # 全局 Token（色板级别）
├── semantic.json     # 语义化 Token（设计意图）
├── typography.json   # 排版 Token
├── motion.json       # 动效 Token
└── components/      # 组件级 Token
    ├── button.json
    ├── input.json
    └── ...
```

### 运行时产物结构

主题工程的源文件需要经过构建，最终运行时只使用以下正式产物：

```text
dist/
├── tokens.json      # Theme Runtime 消费的 ref/sys/comp/motion/layout 五层结构
└── theme.css        # 注入插件窗口与卡片渲染链路的完整主题 CSS
```

### 1. global 层（全局 Token / 色板级）

最底层的视觉值，不含语义：

```json
{
  "chips": {
    "color": {
      "blue-50": "#e3f2fd",
      "blue-100": "#bbdefb",
      "blue-500": "#2196f3",
      "blue-900": "#0d47a1"
    },
    "space": {
      "1": "4px",
      "2": "8px",
      "3": "12px",
      "4": "16px"
    },
    "radius": {
      "none": "0",
      "sm": "4px",
      "md": "8px",
      "lg": "16px",
      "full": "9999px"
    }
  }
}
```

### 2. semantic 层（语义化 Token / 设计意图）

基于 global 层定义的语义化值：

```json
{
  "chips": {
    "color": {
      "primary": { "value": "{chips.color.blue-500}" },
      "on-primary": { "value": "#ffffff" },
      "secondary": { "value": "#9c27b0" },
      "surface": { "value": "#ffffff" },
      "on-surface": { "value": "#1c1b1f" },
      "error": { "value": "#b3261e" },
      "border": { "value": "rgba(0,0,0,0.12)" }
    },
    "shadow": {
      "sm": { "value": "0 1px 2px rgba(0,0,0,0.05)" },
      "md": { "value": "0 4px 6px rgba(0,0,0,0.1)" },
      "lg": { "value": "0 10px 25px rgba(0,0,0,0.15)" }
    }
  }
}
```

### 3. typography 层（排版 Token）

字体和字号参数：

```json
{
  "chips": {
    "font": {
      "family": { "value": "Inter, system-ui, sans-serif" },
      "heading": { "value": "Inter, system-ui, sans-serif" },
      "body": { "value": "Inter, system-ui, sans-serif" }
    },
    "text": {
      "xs": { "value": "12px" },
      "sm": { "value": "14px" },
      "md": { "value": "16px" },
      "lg": { "value": "20px" },
      "xl": { "value": "24px" }
    }
  }
}
```

### 4. motion 层（动效 Token）

过渡和动画参数：

```json
{
  "chips": {
    "duration": {
      "fast": { "value": "150ms" },
      "normal": { "value": "250ms" },
      "slow": { "value": "400ms" }
    },
    "easing": {
      "standard": { "value": "cubic-bezier(0.4, 0, 0.2, 1)" },
      "emphasized": { "value": "cubic-bezier(0.2, 0, 0, 1)" }
    }
  }
}
```

### 5. components 层（组件级 Token）

针对具体组件的样式值：

```json
{
  "chips": {
    "button": {
      "height-sm": { "value": "32px" },
      "height-md": { "value": "40px" },
      "height-lg": { "value": "48px" },
      "radius": { "value": "{chips.radius.md}" },
      "primary-bg": { "value": "{chips.color.primary}" },
      "primary-color": { "value": "{chips.color.on-primary}" }
    },
    "input": {
      "height": { "value": "40px" },
      "radius": { "value": "{chips.radius.md}" },
      "border-color": { "value": "{chips.color.border}" }
    }
  }
}
```

### Style Dictionary 构建

> 主题包开发推荐使用 [Style Dictionary](https://amzn.github.io/style-dictionary/)（Apache 2.0，Amazon 开源）作为 Token 构建工具。

```bash
# 安装 Style Dictionary
npm install -D style-dictionary

# 构建主题包
sd build
```

Style Dictionary 会自动解析 `{chips.color.primary}` 这样的引用，生成最终 CSS：

```css
:root {
  --chips-color-blue-500: #2196f3;
  --chips-color-primary: #2196f3;
  --chips-color-on-primary: #ffffff;
  --chips-radius-md: 8px;
  --chips-button-radius: 8px;
  --chips-button-primary-bg: #2196f3;
}
```

## 组件样式

组件样式定义各个 UI 组件的视觉表现，使用 CSS 变量引用 token。

> **设计理念**：薯片生态采用**无头组件库**架构。组件库（如 `@chips/component-library`）只负责功能逻辑、结构和布局，完全不包含样式。所有视觉表现由主题包通过 CSS 类名和 CSS 变量提供。

### 技术选型

薯片生态的组件库基于以下技术栈构建：

| 依赖 | 许可证 | 用途 |
|------|--------|------|
| [Zag.js](https://zagjs.com/) | MIT | 组件状态机引擎（框架无关） |
| [Ark UI](https://ark-ui.com/) | MIT | React/Vue/Solid 框架绑定 |
| [Style Dictionary](https://amzn.github.io/style-dictionary/) | Apache 2.0 | 设计 Token 构建管线 |

### 组件样式示例

主题包中的按钮样式定义：

```css
/* 按钮样式 */
.chips-button {
  /* 使用语义化 CSS 变量 */
  background-color: var(--chips-button-primary-bg);
  color: var(--chips-button-primary-color);
  border: none;
  border-radius: var(--chips-button-radius);
  padding: var(--chips-button-padding-y) var(--chips-button-padding-x);
  font-family: var(--chips-font-family);
  font-size: var(--chips-button-font-size);
  font-weight: var(--chips-button-font-weight);
  height: var(--chips-button-height-md);
  transition: background-color var(--chips-duration-fast) var(--chips-easing-standard);
}

.chips-button:hover {
  background-color: var(--chips-button-primary-hover-bg);
}

.chips-button:active {
  background-color: var(--chips-button-primary-active-bg);
}

/* 按钮变体 */
.chips-button--primary { }
.chips-button--secondary { }
.chips-button--ghost { }

/* 按钮尺寸 */
.chips-button--sm { height: var(--chips-button-height-sm); }
.chips-button--md { height: var(--chips-button-height-md); }
.chips-button--lg { height: var(--chips-button-height-lg); }
```

### CSS 变量命名规范

主题包中的 CSS 变量必须遵循以下命名规范：

```css
:root {
  /* 色板级变量 */
  --chips-palette-blue-500: #2196f3;
  --chips-palette-gray-100: #f5f5f5;

  /* 语义级变量 */
  --chips-color-primary: var(--chips-palette-blue-500);
  --chips-color-on-primary: #ffffff;
  --chips-color-surface: #ffffff;
  --chips-color-on-surface: #1c1b1f;

  /* 组件级变量 */
  --chips-button-height-sm: 32px;
  --chips-button-height-md: 40px;
  --chips-button-height-lg: 48px;
  --chips-button-radius: var(--chips-radius-md);
  --chips-button-primary-bg: var(--chips-color-primary);
  --chips-button-primary-color: var(--chips-color-on-primary);

  /* 动效变量 */
  --chips-duration-fast: 150ms;
  --chips-duration-normal: 250ms;
  --chips-easing-standard: cubic-bezier(0.4, 0, 0.2, 1);
}
```

### 契约挂点

组件样式必须包含以下数据属性以支持主题契约校验：

```css
/* 按钮契约挂点 */
.chips-button {
  data-scope: "button";
  data-part: "root";
  data-state: "idle";
}
```

## 动画效果

动画定义组件的过渡和动效，使用 `animations/` 目录存储：

```css
/* 过渡效果 */
.fade-enter-active,
.fade-leave-active {
  transition: opacity var(--chips-duration-normal) var(--chips-easing-standard);
}

.fade-enter-from,
.fade-leave-to {
  opacity: 0;
}

/* 动效值引用 */
.slide-up {
  animation: slideUp var(--chips-duration-normal) var(--chips-easing-standard);
}

@keyframes slideUp {
  from {
    transform: translateY(10px);
    opacity: 0;
  }
  to {
    transform: translateY(0);
    opacity: 1;
  }
}
```

## 图标资源

图标使用 SVG 格式，支持以下特性：

- 单色图标使用 `currentColor` 作为填充色，自动适配文字颜色
- 图标尺寸通过 CSS 类名控制，如 `icon-sm`、`icon-md`、`icon-lg`
- 图标列表需要覆盖常用的操作图标如添加、删除、编辑、设置等

## 主题继承

主题支持继承机制，子主题可以继承父主题的样式：

- 在 manifest 中声明 `parentTheme` 字段指定父主题
- 子主题只需覆盖需要修改的部分，未覆盖的部分自动继承父主题
- 继承可以实现主题系列如浅色主题、深色主题基于同一套色彩方案

## 作用域链

主题采用级联作用域链，利用 CSS 自定义属性的天然级联特性实现多层级主题覆盖。

### 层级架构

```
主题权重从低到高：

全局主题        ← 最低权重
应用级主题
箱子级主题
复合卡片级主题
基础卡片级主题
组件级主题      ← 最高权重（最小组件）
```

查找规则（逐层向上回退）：
`component -> base-card -> composite-card -> box -> app -> global`

### CSS 变量级联实现

利用 CSS 自定义属性的天然级联特性：

```css
/* 全局主题（注入到最顶层） */
:root {
  --chips-color-primary: #6200ee;
  --chips-color-surface: #ffffff;
  --chips-radius-md: 8px;
}

/* 应用级主题覆盖 */
[data-chips-app="editor"] {
  --chips-color-primary: #1976d2;
}

/* 箱子级主题覆盖 */
[data-chips-box="abc123"] {
  --chips-color-surface: #f0f0f0;
}

/* 复合卡片级主题覆盖 */
[data-chips-composite-card="card-shell-01"] {
  --chips-color-surface: #f5f5f5;
}

/* 基础卡片级主题覆盖 */
[data-chips-base-card="card-node-09"] {
  --chips-color-surface: #f5f5f5;
}

/* 组件级主题覆盖 */
[data-chips-component="btn-save"] {
  --chips-color-primary: #4caf50;
}
```

### 覆盖规则

1. **作用域优先级**：近者优先（component > base-card > composite-card > box > app > global）
2. **选择器特异性**：特异性越高越优先
3. **声明顺序**：同一作用域内，后声明的覆盖先声明的

## cpx 逻辑尺寸单位

生态统一使用 `cpx` 作为逻辑尺寸单位，页面基准宽度为 `1024cpx`。

### 换算公式

```
px = cpx * (containerWidthPx / 1024)
```

### 工程规范

- 所有布局常量以 `cpx` 声明，渲染前统一换算到 `px`
- 换算结果保留 3 位小数，提交到 DOM/CSS 前按策略裁剪：
  - 字体：四舍五入到 `0.5px`
  - 边框：最小 `1px` 可见阈值
  - 位移：允许亚像素渲染

## 主题契约门禁

每个主题包在发布前必须通过以下校验：

### 1. Token 完整性校验

- 检查 global/semantic/typography/motion/components 各层是否完整
- 检查是否存在冗余 token
- 检查 token 引用链是否完整（无循环引用）
- 验证 JSON 格式符合 Style Dictionary 标准

### 2. 组件接口点校验

- 检查组件是否包含 `data-scope` 和 `data-part` 属性
- 检查状态是否投影到 `data-state`

### 3. 动效参数安全校验

- 检查最大动画时长是否超过阈值（建议 500ms）
- 检查缓动曲线是否为白名单曲线

### 4. 颜色对比度校验

- 检查文本与背景的 WCAG 对比度
- 建议文本对比度 >= 4.5:1，大文本 >= 3:1

## 预览功能

主题包可以包含预览功能：

- `preview/` 目录存放主题预览图片
- 系统会在主题管理界面显示预览
- `preview.html` 可以包含预览页面代码，用于展示主题效果

## 发布流程

主题包开发完成后：

1. **打包**：使用 `chipsdev build` 与 `chipsdev package` 生成 `.cpk` 文件
2. **校验**：先执行 `chipsdev validate`，再使用 `chips theme validate` 完成 Host 侧契约门禁校验
3. **发布**：提交到应用市场供用户安装使用
4. **安装**：使用 `chips plugin install <cpk路径>` 安装后，主题会出现在主题管理界面的主题列表中

主题 `.cpk` 需要遵循与普通插件一致的安装包契约：包根目录保留 `manifest.yaml`，主题构建产物位于 `dist/`，并采用 ZIP Store 模式输出。建议每次发布前都执行一次 `chips plugin install <cpk路径>` 的本地安装联调。

## 最佳实践

主题开发应遵循以下最佳实践：

- 使用语义化的令牌名称如 `primary` 而非 `blue-500`
- 保持令牌的层级结构便于管理
- 确保对比度满足可访问性要求
- 提供足够的变体覆盖各种使用场景
- 测试在不同设备和浏览器上的显示效果
- 每个主题包仅承载一种外观，不在包内区分模式
- 使用 `cpx` 作为布局尺寸单位
- 组件样式必须包含契约挂点（`data-scope`、`data-part`、`data-state`）
