# 主题包开发指南

## 概述

主题包是定义生态视觉外观的插件类型。主题包通过 CSS 变量和类名定义界面的颜色、字体、圆角、阴影、动画等视觉属性。主题包可以完全改变软件的视觉风格而不影响功能逻辑。

## 文件结构

主题包的 CPK 文件包含以下组成部分：

```
theme-package/
├── manifest.yaml                 # 主题包配置文件
├── tokens/                       # 设计令牌目录
│   ├── global.css               # 全局 CSS 变量
│   ├── ref.json                 # 原始值令牌
│   ├── sys.json                 # 语义值令牌
│   ├── comp.json                # 组件值令牌
│   ├── motion.json              # 动效值令牌
│   └── layout.json              # 布局密度值令牌
├── components/                   # 组件样式目录
│   ├── button.css
│   ├── input.css
│   ├── card.css
│   └── ...
├── motions/                      # 动画目录
│   └── transitions.css
├── icons/                        # 图标资源目录
│   └── *.svg
├── contracts/                    # 契约目录
│   └── theme-interface.contract.json  # 主题接口契约
└── preview/                      # 预览资源目录
    └── preview.png
```

## manifest 配置

主题包的 manifest.yaml 有特定的配置要求：

```yaml
type: theme
themeId: chips-official:dark-theme  # 主题唯一标识符，格式：发行商:名称
displayName: 深色主题                # 显示名称
version: 1.0.0                      # 语义化版本
isDefault: false                    # 是否为默认主题
parentTheme: chips-official:base-theme  # 可选，父主题实现继承
ui:
  layout:
    owner: page
    unit: cpx                      # 逻辑尺寸单位
    baseWidth: 1024                 # 基准宽度
    contract: ./contracts/page-layout.contract.json
    minFunctionalSet: ./contracts/min-functional-set.json
```

### 主题包语义约束

> **重要约束**：一个主题包仅承载一种外观，不在包内区分白天/夜间、light/dark 或其他模式标签。外观切换通过切换 `themeId` 到另一个主题包完成。

## 设计令牌（Token 五层系统）

设计令牌是主题的基础构建块，采用五层架构：

### 1. ref 层（原始值）

最底层的视觉值，不含语义：

```json
{
  "color": {
    "blue-500": "#3b82f6",
    "gray-100": "#f3f4f6"
  },
  "spacing": {
    "unit": "4px"
  },
  "radius": {
    "sm": "4px",
    "md": "8px"
  }
}
```

### 2. sys 层（语义值）

基于 ref 层定义的语义化值：

```json
{
  "color": {
    "primary": "{color.blue-500}",
    "background": "{color.gray-100}",
    "text-primary": "#111827",
    "border": "#e5e7eb"
  },
  "typography": {
    "font-family": "system-ui, sans-serif",
    "font-size-base": "14px"
  }
}
```

### 3. comp 层（组件值）

针对具体组件的样式值：

```json
{
  "button": {
    "primary-bg": "{color.primary}",
    "primary-text": "#ffffff",
    "padding-x": "16px",
    "padding-y": "8px",
    "radius": "{radius.md}"
  },
  "input": {
    "border": "{color.border}",
    "focus-ring": "{color.primary}"
  }
}
```

### 4. motion 层（动效值）

过渡和动画参数：

```json
{
  "duration": {
    "fast": "150ms",
    "normal": "250ms",
    "slow": "350ms"
  },
  "easing": {
    "ease-out": "cubic-bezier(0, 0, 0.2, 1)",
    "ease-in-out": "cubic-bezier(0.4, 0, 0.2, 1)"
  }
}
```

### 5. layout 层（布局密度值）

间距和布局密度参数：

```json
{
  "density": {
    "compact": "4px",
    "default": "8px",
    "comfortable": "12px",
    "spacious": "16px"
  }
}
```

## 组件样式

组件样式定义各个 UI 组件的视觉表现，使用 CSS 变量引用 token：

```css
/* 按钮样式 */
.btn {
  background-color: var(--button-primary-bg);
  color: var(--button-primary-text);
  border: none;
  border-radius: var(--button-radius);
  padding: var(--button-padding-y) var(--button-padding-x);
  transition: background-color var(--motion-duration-fast) var(--motion-ease-out);
}

.btn:hover {
  background-color: var(--button-primary-hover-bg);
}
```

### 契约挂点

组件样式必须包含以下数据属性以支持主题契约校验：

```css
/* 按钮契约挂点 */
.btn {
  data-scope: "button";
  data-part: "root";
  data-state: "idle";
}
```

## 动画效果

动画定义组件的过渡和动效：

```css
/* 过渡效果 */
.fade-enter-active,
.fade-leave-active {
  transition: opacity var(--motion-duration-normal) var(--motion-ease-out);
}

.fade-enter-from,
.fade-leave-to {
  opacity: 0;
}

/* 动效值引用 */
.slide-up {
  animation: slideUp var(--motion-duration-normal) var(--motion-ease-out);
}

@keyframes slideUp {
  from {
    transform: translateY(10px);
    opacity: 0;
  }
  to {
    transform: translateY(0);
    opacity: 1;
  }
}
```

## 图标资源

图标使用 SVG 格式，支持以下特性：

- 单色图标使用 `currentColor` 作为填充色，自动适配文字颜色
- 图标尺寸通过 CSS 类名控制，如 `icon-sm`、`icon-md`、`icon-lg`
- 图标列表需要覆盖常用的操作图标如添加、删除、编辑、设置等

## 主题继承

主题支持继承机制，子主题可以继承父主题的样式：

- 在 manifest 中声明 `parentTheme` 字段指定父主题
- 子主题只需覆盖需要修改的部分，未覆盖的部分自动继承父主题
- 继承可以实现主题系列如浅色主题、深色主题基于同一套色彩方案

## 作用域链

主题采用级联作用域链，优先级从低到高：

```
global -> app -> workspace -> card -> component
```

### 覆盖规则

1. **作用域优先级**：近者优先（component > card > workspace > app > global）
2. **选择器特异性**：特异性越高越优先
3. **声明顺序**：同一作用域内，后声明的覆盖先声明的

## cpx 逻辑尺寸单位

生态统一使用 `cpx` 作为逻辑尺寸单位，页面基准宽度为 `1024cpx`。

### 换算公式

```
px = cpx * (containerWidthPx / 1024)
```

### 工程规范

- 所有布局常量以 `cpx` 声明，渲染前统一换算到 `px`
- 换算结果保留 3 位小数，提交到 DOM/CSS 前按策略裁剪：
  - 字体：四舍五入到 `0.5px`
  - 边框：最小 `1px` 可见阈值
  - 位移：允许亚像素渲染

## 主题契约门禁

每个主题包在发布前必须通过以下校验：

### 1. Token 完整性校验

- 检查 ref/sys/comp/motion/layout 五层是否完整
- 检查是否存在冗余 token
- 检查 token 引用链是否完整

### 2. 组件接口点校验

- 检查组件是否包含 `data-scope` 和 `data-part` 属性
- 检查状态是否投影到 `data-state`

### 3. 动效参数安全校验

- 检查最大动画时长是否超过阈值（建议 500ms）
- 检查缓动曲线是否为白名单曲线

### 4. 颜色对比度校验

- 检查文本与背景的 WCAG 对比度
- 建议文本对比度 >= 4.5:1，大文本 >= 3:1

## 预览功能

主题包可以包含预览功能：

- `preview/` 目录存放主题预览图片
- 系统会在主题管理界面显示预览
- `preview.html` 可以包含预览页面代码，用于展示主题效果

## 发布流程

主题包开发完成后：

1. **打包**：使用 `chips pack` 命令生成 cpk 文件
2. **校验**：主题包必须通过契约门禁校验
3. **发布**：提交到应用市场供用户安装使用
4. **安装**：主题会出现在主题管理界面的主题列表中

## 最佳实践

主题开发应遵循以下最佳实践：

- 使用语义化的令牌名称如 `primary` 而非 `blue-500`
- 保持令牌的层级结构便于管理
- 确保对比度满足可访问性要求
- 提供足够的变体覆盖各种使用场景
- 测试在不同设备和浏览器上的显示效果
- 每个主题包仅承载一种外观，不在包内区分模式
- 使用 `cpx` 作为布局尺寸单位
- 组件样式必须包含契约挂点（`data-scope`、`data-part`、`data-state`）
