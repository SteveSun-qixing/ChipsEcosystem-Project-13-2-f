# 主题接口规范

## 概述

主题接口规范定义主题系统与外部模块之间的交互方式。规范包括主题查询、加载、应用等核心接口。

> **设计理念**：薯片生态采用**自管理前端架构**，组件库只负责功能逻辑、结构和布局，完全不包含样式。所有视觉表现由主题包提供，主题系统负责管理这些视觉资源的生命周期。

## 固定公开接口

以下接口为生态统一的公共接口，所有实现必须遵循：

| 接口 | 说明 | 幂等 |
|---|---|---|
| `theme.list` | 获取所有已安装主题列表 | ✓ |
| `theme.apply` | 切换当前主题 | - |
| `theme.getCurrent` | 获取当前活动主题 | ✓ |
| `theme.getAllCss` | 获取当前主题的完整 CSS | ✓ |
| `theme.resolve` | 解析主题依赖链 | ✓ |
| `theme.contract.get` | 获取主题契约接口 | ✓ |

### theme.list

获取所有已安装主题列表。

```typescript
// 请求
{ publisher?: string }

// 响应
{
  themes: Array<{
    id: string;
    displayName: string;
    version: string;
    isDefault: boolean;
    publisher?: string;
  }>
}
```

### theme.apply

切换当前主题。

```typescript
// 请求
{ id: string }

// 响应
{ success: boolean; themeId: string }
```

### theme.getCurrent

获取当前活动主题。

```typescript
// 请求
{ appId?: string; pluginId?: string }

// 响应
{
  themeId: string;
  displayName: string;
  version: string;
}
```

### theme.getAllCss

获取当前主题的完整 CSS 代码。

```typescript
// 请求
{}

// 响应
{ css: string; themeId: string }
```

### theme.resolve

解析主题及其依赖链。

```typescript
// 请求
{ chain: string[] }

// 响应
{
  resolved: Array<{
    id: string;
    displayName: string;
    order: number;
  }>
}
```

### theme.contract.get

获取主题契约接口定义。

```typescript
// 请求
{ component?: string }

// 响应
{
  contracts: {
    [component: string]: {
      scope: string;
      parts: string[];
      states: string[];
      tokens: string[];
    }
  }
}
```

## 事件接口

### theme.changed

主题切换事件。当主题发生变化时触发。

```typescript
// 事件载荷
{
  previousThemeId: string;
  themeId: string;
  timestamp: number;
}

// 订阅方式
window.chips.on('theme.changed', (payload) => {
  console.log('主题已切换:', payload.themeId);
});
```

> **注意**：事件命名统一采用点语义 `theme.changed`，禁止使用冒号或其他分隔符。

## 作用域链

主题系统支持级联作用域，利用 CSS 自定义属性的天然级联特性实现多层级主题覆盖。

### 层级架构

```
优先级从高到低：

组件级主题    ← 最高优先级，每个组件可以单独设置
  ↓ 如果未设置
基础卡片级主题  ← 每个基础卡片可以单独设置
  ↓ 如果未设置
复合卡片级主题  ← 每个复合卡片可以单独设置
  ↓ 如果未设置
箱子级主题    ← 箱子可以为其中的卡片设置统一主题
  ↓ 如果未设置
应用级主题    ← 当前应用的主题设置
  ↓ 如果未设置
全局主题      ← 整个薯片生态的默认主题（最低优先级）
```

### 覆盖规则

1. **作用域优先级**：近者优先（component > card > box > app > global）
2. **选择器特异性**：特异性越高越优先
3. **声明顺序**：同一作用域内，后声明的覆盖先声明的

## 错误处理

所有主题操作遵循统一错误结构：

```typescript
interface ThemeError {
  code: string;
  message: string;
  details?: unknown;
  retryable?: boolean;
}
```

### 标准错误码

| 错误码 | 说明 | 可重试 |
|---|---|---|
| `THEME_NOT_FOUND` | 主题不存在 | false |
| `THEME_CONTRACT_INVALID` | 主题契约无效 | false |
| `THEME_DEPENDENCY_ERROR` | 依赖主题加载失败 | true |
| `THEME_PERMISSION_DENIED` | 权限不足 | false |
| `THEME_LOAD_FAILED` | 主题加载失败 | true |
| `THEME_CSS_PARSE_ERROR` | CSS 解析错误 | false |

### 错误示例

```typescript
// 主题不存在
{
  code: "THEME_NOT_FOUND",
  message: "主题 chips-official:dark-theme 不存在",
  details: { themeId: "chips-official:dark-theme" },
  retryable: false
}

// 权限不足
{
  code: "THEME_PERMISSION_DENIED",
  message: "当前用户无权限切换主题",
  details: { requiredPermission: "theme.write" },
  retryable: false
}
```

## 类型定义

```typescript
// 主题信息
interface ThemeInfo {
  id: string;
  displayName: string;
  version: string;
  isDefault: boolean;
  publisher?: string;
  parentTheme?: string;
}

// 主题配置
interface ThemeConfig {
  id: string;
  tokens: {
    global: Record<string, unknown>;
    semantic: Record<string, unknown>;
    typography: Record<string, unknown>;
    motion: Record<string, unknown>;
    components: Record<string, unknown>;
  };
  contracts: ThemeContract[];
}

// 解析结果
interface ThemeResolveResult {
  resolved: ThemeInfo[];
  order: number[];
}

// 主题契约
interface ThemeContract {
  component: string;
  scope: string;
  parts: string[];
  states: string[];
  tokens: string[];
}

// 统一错误
interface StandardError {
  code: string;
  message: string;
  details?: unknown;
  retryable?: boolean;
}
```

## 扩展接口（可选实现）

以下接口为扩展功能，非必须实现：

| 接口 | 说明 |
|---|---|
| `theme.preview` | 临时预览主题（不保存） |
| `theme.reset` | 重置为系统默认主题 |

> 扩展接口仅在需要时实现，不属于生态统一公共接口范畴。
