> 编写时间：2026-02-24  
> 适用范围：Chips 前端架构、组件库、主题包、插件模板

## 1. 目标与边界

- 统一薯片生态图标与字体基线，消除多源字体与图标混用。
- 采用 Google 官方开源资产，保证授权清晰、可追溯、可持续更新。
- 本文只定义新架构口径，不包含兼容旧口径和迁移策略。

## 3. 目录与资产规范

## 3.1.1 应用启动图标源目录
- `design-assets/Appicon/`：应用插件系统入口图标正式设计源目录。

  - `AppIcon.*`：`chips-scaffold-app` 默认应用模板源；

## 4. 图标系统架构设计

## 4.1 统一图标组件（强制）
- 生态内所有图标必须通过 `ChipsIcon`（或同职责统一组件）输出。
- 页面、业务组件禁止直接引用任意第三方图标组件。

## 4.2 语义输入模型
- 语义主键：`name`（来自 Material Symbols 命名）。
- 风格参数：`style = outlined | rounded | sharp`。
- 可变轴参数：`fill / wght / grad / opsz`。
- 大小参数使用 `cpx`，渲染阶段换算为 px。

## 4.3 主题映射（强制）
图标视觉参数必须来自主题 token，禁止组件内硬编码：
- `icon.color`
- `icon.weight`
- `icon.fill`
- `icon.grade`
- `icon.opticalSize`
- `icon.size`

## 4.4 可访问性
- 装饰图标：`aria-hidden=true`。
- 功能图标：必须有可读标签（`aria-label` 或外部 label 绑定）。
- 图标按钮必须满足最小热区 44px。

## 5. 字体系统架构设计

## 5.1 字体分层
- `font.ui`: Roboto Flex（UI 英文/数字/符号）。
- `font.text.zh`: Noto Sans SC（中文正文）。
- `font.mono`: Noto Sans Mono（代码、日志、终端）。

## 5.1.1 多语言覆盖（冻结）
- English (英语): `Roboto Flex` + `Noto Sans`
- 简体中文 (Chinese Simplified): `Noto Sans SC`
- 繁体中文 (Chinese Traditional): `Noto Sans TC`
- Español (西班牙语): `Roboto Flex` + `Noto Sans`
- Français (法语): `Roboto Flex` + `Noto Sans`
- Deutsch (德语): `Roboto Flex` + `Noto Sans`
- 日本語 (日语): `Noto Sans JP`
- 한국어 (韩语): `Noto Sans KR`
- Português (葡萄牙语): `Roboto Flex` + `Noto Sans`
- Português (Brasil) (巴西葡语): `Roboto Flex` + `Noto Sans`
- Русский (俄语): `Noto Sans`
- Italiano (意大利语): `Roboto Flex` + `Noto Sans`

## 5.2 字体回退链（建议默认）
- UI：`Roboto Flex -> Noto Sans SC -> system-ui`。
- 正文：`Noto Sans SC -> Roboto Flex -> system-ui`。
- 等宽：`Noto Sans Mono -> ui-monospace -> monospace`。

## 5.3 字体变量轴治理
- 允许页面通过主题 token 控制 `wght`、`wdth` 等轴。
- 禁止在业务页面直接写死 `font-variation-settings`。

---

## 6. 运行时加载与性能策略

- 优先加载 `woff2`（图标字体）与变量字体主文件。
- 字体声明由主题层统一注入，页面只消费语义 token。
- 关键界面可使用预加载策略，但禁止无差别全量预加载。
- 允许按插件类型分组加载字体资源，避免首屏无关资源阻塞。

---

## 7. 开发与门禁

## 7.1 开发约束
- 禁止硬编码字体家族名到业务组件。
- 禁止硬编码图标 SVG path 到业务页面。
- 禁止直接操作 icon 字体文件路径（必须通过统一资源注册表）。

## 7.2 测试门禁
- 图标：语义名到 glyph 映射可用性测试。
- 字体：三套基础字体加载可用性测试。
- 主题：图标与字体 token 覆盖测试。
- 可访问性：功能图标标签与按钮热区测试。

---

## 8. 与前端总架构的关系

- 图标与字体属于“主题系统 + 组件库”共享基础设施。
- 页面层只使用语义接口，不接触底层资源文件。
- Host/SDK 不直接耦合具体图标字体文件，仅负责资源分发与能力调用。

---

## 9. 版本与追溯

- 每次更新图标/字体必须同时更新对应目录内 `SOURCE.md`。
- `SOURCE.md` 必须包含：上游仓库、commit、下载时间、文件清单。
- 未提供追溯信息的资源变更一律不允许合入。

## 10. 正式实施补充（2026-03-25）

### 10.1 运行时 UI 图标主链路

- 运行时 UI 图标统一使用 `ChipsIcon`。
- 统一输入模型为 `IconDescriptor`：

```ts
interface IconDescriptor {
  name: string;
  style?: "outlined" | "rounded" | "sharp";
  fill?: 0 | 1;
  wght?: number;
  grad?: number;
  opsz?: number;
  decorative?: boolean;
  label?: string;
}
```

- 基础卡片描述符、布局描述符、编辑引擎窗口、设置治理页回退图标都必须消费这套模型。
- 不再保留 `icon: string`、emoji、字符图标作为正式运行时 UI 图标口径。

### 10.2 启动图标与运行时图标分层

- `manifest.ui.launcher.icon` 是操作系统入口图标文件路径；
- `PluginShortcutRecord.iconPath` 是 Host 返回的系统入口图标文件路径；
- 两者都不属于 `ChipsIcon` 运行时图标模型；
- 治理页只能把它们当作文件资源展示，缺失时再回退到 `ChipsIcon`。

### 10.2.1 启动图标源与工程输出同步规则

- `design-assets/Appicon/` 是应用启动图标的唯一正式设计源目录；
- 各应用工程纳入版本控制的 `assets/icons/app-icon.png/.ico/.icns` 必须由该目录对应源文件同步得出；
- `app-icon.svg` 可作为工程内补充展示资源保留，但不替代 `ico / icns / png` 系统入口链路；
- 不允许在项目内长期手工维护与 `design-assets/Appicon/` 漂移的另一套启动图标资产；
- 基础卡片和布局插件静态 SVG 不得继续混入该目录，历史遗留文件应转入 `归档/`。

### 10.3 主题包正式产物要求

- 主题包源码必须包含三套 Material Symbols variable font：
  - `MaterialSymbolsOutlined[FILL,GRAD,opsz,wght].woff2`
  - `MaterialSymbolsRounded[FILL,GRAD,opsz,wght].woff2`
  - `MaterialSymbolsSharp[FILL,GRAD,opsz,wght].woff2`
- 构建阶段必须复制到 `dist/icons/variablefont/`；
- `dist/theme.css` 必须注入对应 `@font-face`；
- 只把字体留在 `icons/variablefont/` 源目录、不进入 `dist/` 的做法视为未接入正式运行时链路。

### 10.4 token 对接要求

- 主题必须提供以下正式图标 token：
  - `chips.sys.icon.color`
  - `chips.sys.icon.size`
  - `chips.sys.icon.fill`
  - `chips.sys.icon.wght`
  - `chips.sys.icon.grad`
  - `chips.sys.icon.opsz`
- 主题校验脚本必须把这些 token 作为硬性门禁。
