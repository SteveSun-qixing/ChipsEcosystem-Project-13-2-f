# 文件转换能力契约

> 文档状态：首版冻结稿  
> 适用范围：`Chips-FileConversion-Plugin`、`Chips-CardtoHTML-Plugin`、`Chips-HtmltoPDF-Plugin`、`Chips-HtmltoImage-Plugin`

## 1. 定位

文件转换系统继续构建在模块插件正式契约之上：

- 对外统一入口是模块能力调用；
- Host 不新增 `conversion.*` 服务域；
- 编排模块和原子模块都通过 `module.provides/module.consumes` 声明能力；
- 调用方始终通过 `module.invoke` 访问转换能力。

## 2. 能力清单

### 2.1 对外统一入口

编排模块对外公开能力：

- capability：`converter.file.convert`
- method：`convert`
- mode：`job`

### 2.2 原子能力

当前首版冻结以下原子能力：

- `converter.card.to-html`
- `converter.html.to-pdf`
- `converter.html.to-image`

它们的正式方法名统一为 `convert`。

## 3. 支持矩阵

| 源类型 | 目标类型 | 正式流水线 |
|---|---|---|
| `card` | `html` | `converter.card.to-html` |
| `card` | `pdf` | `converter.card.to-html(directory)` -> `converter.html.to-pdf` |
| `card` | `image` | `converter.card.to-html(directory)` -> `converter.html.to-image` |
| `html` | `pdf` | `converter.html.to-pdf` |
| `html` | `image` | `converter.html.to-image` |

## 4. 编排模块输入契约

`converter.file.convert/convert` 输入对象冻结如下：

```ts
interface FileConvertRequest {
  source: {
    type: "card" | "html";
    path: string;
  };
  target: {
    type: "html" | "pdf" | "image";
  };
  output: {
    path: string;
    overwrite?: boolean;
  };
  options?: {
    html?: {
      packageMode?: "zip" | "directory";
      includeAssets?: boolean;
      includeManifest?: boolean;
    };
    pdf?: {
      pageSize?: "A4" | "A3" | "Letter" | "Legal";
      landscape?: boolean;
      printBackground?: boolean;
      marginMm?: {
        top?: number;
        right?: number;
        bottom?: number;
        left?: number;
      };
    };
    image?: {
      format?: "png" | "jpeg" | "webp";
      width?: number;
      height?: number;
      scaleFactor?: number;
      background?: "transparent" | "white" | "theme";
    };
    locale?: string;
    themeId?: string;
  };
}
```

约束：

- `card -> pdf/image` 时，编排模块必须强制使用 `html.packageMode = "directory"` 作为中间产物模式；
- `card -> pdf/image` 时，编排模块必须同时强制 `html.includeAssets = true` 与 `html.includeManifest = true`，保证中间 HTML 满足下游正式输入契约；
- 直接导出 HTML 时，默认 `packageMode = "zip"`；
- `output.path` 永远指向最终交付物路径，而不是中间目录路径。

## 6. Card -> HTML 契约

### 6.1 输入

```ts
interface CardToHtmlRequest {
  cardFile: string;
  output: {
    path: string;
    packageMode: "directory" | "zip";
    overwrite?: boolean;
  };
  options?: {
    includeAssets?: boolean;
    includeManifest?: boolean;
    locale?: string;
    themeId?: string;
  };
}
```

### 6.2 输出

```ts
interface CardToHtmlResult {
  packageMode: "directory" | "zip";
  outputPath: string;
  entryFile: "index.html";
  manifestFile?: "conversion-manifest.json";
  semanticHash: string;
  assetCount: number;
  warnings?: Array<{
    code: string;
    message: string;
    details?: unknown;
  }>;
}
```

### 6.3 目录结构

目录态输出结构如下：

```text
<output>/
├─ index.html
├─ <node-id>.html               # 复合卡片中每个基础卡片 iframe 的离线文档
├─ assets/
│  └─ content/                  # includeAssets=true 时存在
└─ conversion-manifest.json     # includeManifest=true 时存在
```

压缩态输出是上述目录结构的 ZIP 包。

补充约束：

- `card -> html` 上游若传入 `themeId/locale`，原子卡片转 HTML 模块必须把这两个字段透传到 Host `card.render` 的单次调用覆盖参数中。
- 当导出结果包含复合卡片 iframe 文档时，模块必须把每个 iframe 子文档外置为同目录 sibling HTML，并让 `index.html` 通过相对 `src` 引用；正式中间产物不得继续保留大体积 `srcdoc` 内嵌文档。
- `includeAssets=false` 只允许用于 `packageMode = "directory"` 的直接 HTML 导出；此时导出结果保留原始 `file://` 引用，不得作为 PDF / 图片模块的正式输入。
- `includeManifest=false` 只允许用于不再进入下游转换链路的直接 HTML 导出；此时结果对象中的 `manifestFile` 必须省略。
- `assets/theme/` 保留给未来可能出现的独立主题工件；当前首版不要求模块强制产出该目录。

## 7. HTML -> PDF 契约

Host 正式承载动作：

- `platform.renderHtmlToPdf`

说明：

- 该动作属于 Host 内部正式渲染导出能力；
- 它不改变对调用方公开的文件转换 capability；
- 它不等同于模块可直接访问 Electron API。

### 7.1 输入

```ts
interface HtmlToPdfRequest {
  htmlDir: string;
  entryFile?: string;
  outputFile: string;
  options?: {
    pageSize?: "A4" | "A3" | "Letter" | "Legal";
    landscape?: boolean;
    printBackground?: boolean;
    marginMm?: {
      top?: number;
      right?: number;
      bottom?: number;
      left?: number;
    };
  };
}
```

### 7.2 输出

```ts
interface HtmlToPdfResult {
  outputFile: string;
  pageCount?: number;
  warnings?: Array<{
    code: string;
    message: string;
    details?: unknown;
  }>;
}
```

## 8. HTML -> Image 契约

Host 正式承载动作：

- `platform.renderHtmlToImage`

说明：

- 该动作属于 Host 内部正式渲染导出能力；
- 它不改变对调用方公开的文件转换 capability；
- 它不等同于模块可直接访问 Electron API。

### 8.1 输入

```ts
interface HtmlToImageRequest {
  htmlDir: string;
  entryFile?: string;
  outputFile: string;
  options?: {
    format?: "png" | "jpeg" | "webp";
    width?: number;
    height?: number;
    scaleFactor?: number;
    background?: "transparent" | "white" | "theme";
  };
}
```

### 8.2 输出

```ts
interface HtmlToImageResult {
  outputFile: string;
  width?: number;
  height?: number;
  format: "png" | "jpeg" | "webp";
  warnings?: Array<{
    code: string;
    message: string;
    details?: unknown;
  }>;
}
```

补充约束：

- `entryFile` 必须位于 `htmlDir` 内；
- `format = "webp"` 属于正式保留输入，但若当前 Host 运行时不具备对应编码器，可返回 `PLATFORM_UNSUPPORTED`；
- `converter.html.to-image` 只消费目录态 HTML 中间产物；
- `converter.html.to-image` 不解析卡片文件，也不直接依赖基础卡片插件；
- `converter.html.to-image` 不得在模块内直接导入 `electron`；
- `converter.html.to-image` 不得在模块内直接引入 `puppeteer` 或其他独立浏览器运行时；
- 页面截图必须通过 Host 正式渲染导出能力完成；

## 9. 进度模型

所有转换 job 统一使用以下阶段语义：

- `prepare`
- `render-html`
- `rewrite-assets`
- `package-html`
- `render-pdf`
- `render-image`
- `cleanup`
- `completed`

进度事件载荷最少包含：

```ts
{
  stage: string;
  percent: number;
  message?: string;
}
```

## 10. 错误码基线

首版冻结以下错误前缀：

- `CONVERTER_INPUT_*`
- `CONVERTER_PIPELINE_*`
- `CONVERTER_HTML_*`
- `CONVERTER_PDF_*`
- `CONVERTER_IMAGE_*`
- `CONVERTER_OUTPUT_*`

典型错误：

- `CONVERTER_INPUT_UNSUPPORTED`
- `CONVERTER_PIPELINE_NOT_FOUND`
- `CONVERTER_HTML_RENDER_FAILED`
- `CONVERTER_HTML_ASSET_REWRITE_FAILED`
- `CONVERTER_OUTPUT_WRITE_FAILED`
- `CONVERTER_HTML_PACKAGE_FAILED`
- `CONVERTER_PDF_PRINT_FAILED`
- `CONVERTER_IMAGE_CAPTURE_FAILED`
- `CONVERTER_OUTPUT_EXISTS`

## 11. 实施边界

- 本文档冻结的是公共契约与输出口径；
- 仓库内部的目录组织、实现分层、测试分层和阶段计划，不在此重复定义；
- 若后续新增 `doc -> card`、`pdf -> card` 等能力，应新增 capability，不破坏本首版矩阵。
