# 箱子布局运行时契约

**版本**：1.0.0  
**更新时间**：2026-03-23  
**状态**：正式版

---

## 1. 文档定位

本文档定义 Host、查看器与布局插件在箱子消费阶段的正式运行时契约。

主旨只有一条：  
**布局插件负责前端页面，Host 负责数据与资源解析，双方通过会话化、按需、最小化的数据接口协作。**

---

## 2. 核心原则

1. **Host 唯一解析权**：URL 解析、凭证、缓存、失败状态归一都由 Host 负责；
2. **布局摘要优先**：布局插件默认先消费封面、名称、标签、摘要与状态；
3. **卡片信息中心唯一口径**：卡片详细信息统一来自 Host `card.readInfo` 模块，不再由箱子层散落拼装；
4. **Host 统一激活**：点击封面后是否打开新窗口、如何打开，统一由 Host `box.openEntry` 决定；
5. **按需取数**：不要求首屏返回全部卡片详细信息；
6. **会话隔离**：所有运行时读操作都必须绑定到 `sessionId`；
7. **最小暴露**：布局插件不直接拿到凭证和底层文件句柄；
8. **资源分层**：箱子自有资源与卡片资源严格分层。

---

## 3. 角色边界

| 角色 | 职责 | 禁止事项 |
|---|---|---|
| Host | 解析 `.box`、维护会话、解析 URL、返回数据/资源 | 把所有卡片资源全量无差别灌给布局插件 |
| 布局插件 | 渲染页面、组织布局、按需申请资源 | 自行扫描磁盘、直接访问网络、直接读取凭证 |
| 查看器/应用插件 | 承载布局插件输出、管理窗口和页面 | 复制一套箱子资源解析逻辑 |

## 4. 会话模型

### 4.1 `box.openView`

打开箱子必须先创建查看会话。

#### 请求

```ts
interface BoxOpenViewRequest {
  boxFile: string;
  layoutType?: string;
  initialQuery?: BoxEntryQuery;
}
```

#### 响应

```ts
interface BoxOpenViewResponse {
  sessionId: string;
  box: BoxSessionInfo;
  initialView: BoxEntryPage;
}
```

#### 会话信息

```ts
interface BoxSessionInfo {
  boxId: string;
  boxFile: string;
  name: string;
  activeLayoutType: string;
  availableLayouts: string[];
  tags?: Array<string | string[]>;
  coverAsset?: string;
  capabilities: {
    listEntries: true;
    readEntryDetail: true;
    renderEntryCover: true;
    resolveEntryResource: true;
    readBoxAsset: true;
    prefetchEntries: true;
    openEntry: true;
  };
}
```

### 4.2 会话关闭

```ts
interface BoxCloseViewRequest {
  sessionId: string;
}
```

```ts
interface BoxCloseViewResponse {
  ack: true;
}
```

## 5. 摘要态条目模型

布局插件默认消费条目摘要页。

```ts
interface BoxEntrySnapshot {
  entryId: string;
  url: string;
  enabled: boolean;
  snapshot: {
    cardId?: string;
    title?: string;
    summary?: string;
    tags?: Array<string | string[]>;
    cover?: {
      mode: 'asset' | 'runtime' | 'none';
      assetPath?: string;
      mimeType?: string;
      width?: number;
      height?: number;
    };
    lastKnownModifiedAt?: string;
    contentType?: string;
  };
  layoutHints?: {
    sortKey?: string | number;
    aspectRatio?: number;
    group?: string;
    priority?: number;
  };
}
```

### 5.1 分页结果

```ts
interface BoxEntryPage {
  items: BoxEntrySnapshot[];
  total: number;
  nextCursor?: string;
}
```

### 5.2 查询结构

```ts
interface BoxEntryQuery {
  cursor?: string;
  limit?: number;
  filter?: Record<string, unknown>;
  sort?: {
    key: string;
    direction: 'asc' | 'desc';
  };
}
```

## 6. 运行时动作面

### 6.1 `box.listEntries`

#### 请求

```ts
interface BoxListEntriesRequest {
  sessionId: string;
  query?: BoxEntryQuery;
}
```

#### 响应

```ts
interface BoxListEntriesResponse {
  page: BoxEntryPage;
}
```

### 6.2 `box.readEntryDetail`

当布局插件需要超出摘要态的数据时，使用 `readEntryDetail`。

#### 请求

```ts
interface BoxReadEntryDetailRequest {
  sessionId: string;
  entryIds: string[];
  fields: Array<
    | 'cardInfo'
    | 'coverDescriptor'
    | 'previewDescriptor'
    | 'runtimeProps'
    | 'status'
  >;
}
```

#### 响应

```ts
interface BoxReadEntryDetailResponse {
  items: Array<{
    entryId: string;
    detail: Record<string, unknown>;
  }>;
}
```

### 6.3 `box.renderEntryCover`

布局插件在需要显示条目正式封面时，应优先使用 `renderEntryCover`。

#### 请求

```ts
interface BoxRenderEntryCoverRequest {
  sessionId: string;
  entryId: string;
}
```

#### 响应

```ts
interface BoxRenderEntryCoverResponse {
  view: {
    title: string;
    coverUrl: string;
    mimeType: string;
    ratio?: string;
  };
}
```

说明：

- `title` 是封面显示链路返回的正式标题字段，供应用层、布局前端或封面组件外层 UI 使用；
- 直接显示条目封面时，推荐把 `coverUrl/ratio/title` 交给正式 `CardCoverFrame` 组件消费，而不是在布局里长期维护一套私有 iframe 壳层；
- 标题是否显示、显示在封面下方还是旁侧，由应用层和布局前端决定，不属于 `box.renderEntryCover` 自身职责。

### 6.4 `box.resolveEntryResource`

当布局插件需要真正的资源句柄时使用。

#### 请求

```ts
interface BoxResolveEntryResourceRequest {
  sessionId: string;
  entryId: string;
  resource: {
    kind: 'cover' | 'preview' | 'cardFile' | 'custom';
    key?: string;
    sizeHint?: {
      width?: number;
      height?: number;
    };
  };
}
```

#### 响应

```ts
interface BoxResolveEntryResourceResponse {
  resource: ResolvedRuntimeResource;
}
```

说明：

1. `renderEntryCover` 用于“直接显示封面”的正式链路；
2. `resolveEntryResource` 仍用于封面之外的预览、卡片入口资源和自定义资源；
3. 布局插件默认不应再把 `resolveEntryResource(kind='cover')` 当作首选封面接口。

### 6.5 `box.openEntry`

布局插件点击封面或条目卡片时，必须使用 `box.openEntry` 请求 Host 执行正式打开。

#### 请求

```ts
interface BoxOpenEntryRequest {
  sessionId: string;
  entryId: string;
}
```

#### 响应

```ts
interface BoxOpenEntryResponse {
  result: {
    mode: 'card-window' | 'external';
    windowId?: string;
    pluginId?: string;
    url?: string;
  };
}
```

### 6.6 `box.readBoxAsset`

读取箱子自有资源。

#### 请求

```ts
interface BoxReadBoxAssetRequest {
  sessionId: string;
  assetPath: string;
}
```

#### 响应

```ts
interface BoxReadBoxAssetResponse {
  resource: ResolvedRuntimeResource;
}
```

### 6.6 `box.prefetchEntries`

允许布局插件在滚动前做温和预取。

#### 请求

```ts
interface BoxPrefetchEntriesRequest {
  sessionId: string;
  entryIds: string[];
  targets: Array<'cover' | 'preview' | 'cardInfo'>;
}
```

#### 响应

```ts
interface BoxPrefetchEntriesResponse {
  ack: true;
}
```

## 7. 统一资源句柄

布局插件不直接拿到底层文件句柄，而是拿到统一资源句柄。

```ts
interface ResolvedRuntimeResource {
  resourceUrl: string;
  mimeType: string;
  cacheKey?: string;
  expiresAt?: string;
  width?: number;
  height?: number;
}
```

### 7.1 句柄要求

1. `resourceUrl` 必须可被当前布局会话安全消费；
2. 不得把凭证直接拼到 URL；
3. 资源可以是临时 URL、受控本地句柄或 Host 内部桥接地址；
4. 同一资源可复用 `cacheKey`。

---

## 8. 安全与权限

### 8.1 权限

布局插件消费箱子运行时数据需要 `box.read`。

### 8.2 会话所有权

- `sessionId` 只能由打开该箱子的查看器 / 布局插件会话使用；
- 不允许跨插件窃取其他会话数据；
- Host 必须校验调用方是否属于当前箱子会话。

### 8.3 凭证与网络

- 布局插件永远不直接访问凭证；
- 网络 URL 的认证、刷新、重试由 Host 统一处理；
- 失败时返回结构化错误，不把底层鉴权细节暴露给布局插件。

---

## 9. 事件模型

Host 可以向布局插件发出箱子会话相关事件。

### 9.1 推荐事件

| 事件名 | 用途 |
|---|---|
| `box.session.updated` | 条目状态、分页或摘要变化 |
| `box.session.entryStateChanged` | 指定条目资源状态变化 |
| `box.session.closed` | 会话关闭 |

### 9.2 原则

- 事件只推送变化事实；
- 大数据量仍通过 `box.*` 读接口重新拉取；
- 不以事件直接推送整箱完整数据。

---

## 10. 错误码

| 错误码 | 含义 |
|---|---|
| `BOX_VIEW_SESSION_NOT_FOUND` | 会话不存在 |
| `BOX_LAYOUT_ACCESS_DENIED` | 当前布局插件无权访问该会话 |
| `BOX_ENTRY_NOT_FOUND` | 条目不存在 |
| `BOX_OPEN_FAILED` | 条目激活失败 |
| `BOX_RESOURCE_NOT_FOUND` | 请求的卡片资源不存在 |
| `BOX_ASSET_NOT_FOUND` | 请求的箱子资源不存在 |
| `BOX_URL_SCHEME_UNSUPPORTED` | URL 协议未被 Host 支持 |
| `BOX_RESOLVE_FAILED` | 解析或加载失败 |

---

## 11. 设计结论

首版正式箱子运行时契约只认以下事实：

1. 布局插件是页面，不是解析器；
2. Host 是解析中心，不是被动文件搬运工；
3. 箱子消费默认是摘要页，不是全量展开页；
4. 数据与资源都必须按需申请；
5. 箱子自有资源与卡片资源严格隔离。
