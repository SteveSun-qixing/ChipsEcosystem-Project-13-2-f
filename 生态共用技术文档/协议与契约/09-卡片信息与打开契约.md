# 卡片信息与打开契约

**版本**：1.0.0  
**更新时间**：2026-03-24  
**状态**：正式版

---

## 1. 文档定位

本文档定义整个生态统一使用的卡片信息中心与卡片打开动作。

核心原则：

1. 卡片信息统一由 Host `card.readInfo` 提供；
2. 卡片新窗口打开统一由 Host `card.open` 提供；
3. 箱子、资源管理器、选择器、查看器和编辑器都不应各自散落实现第二套卡片信息读取或打开逻辑。

---

## 2. `card.readInfo`

### 2.1 请求

```ts
type CardInfoField = 'status' | 'metadata' | 'cover';

interface CardReadInfoRequest {
  cardFile: string;
  fields?: CardInfoField[];
}
```

### 2.2 响应

```ts
interface CardReadInfoResponse {
  info: {
    cardFile: string;
    info: {
      status?: {
        state: 'ready' | 'missing' | 'invalid';
        exists: boolean;
        valid: boolean;
        errors?: string[];
      };
      metadata?: {
        raw: Record<string, unknown>;
        chipStandardsVersion?: string;
        cardId?: string;
        name?: string;
        createdAt?: string;
        modifiedAt?: string;
        theme?: string;
        coverRatio?: string;
        tags?: Array<string | string[]>;
      };
      cover?: {
        title: string;
        resourceUrl: string;
        mimeType: 'text/html';
        ratio?: string;
      };
    };
  };
}
```

### 2.3 正式要求

1. `metadata.raw` 是卡片元数据的正式原始视图；
2. `metadata.*` 归一字段用于生态通用消费；
3. `cover.resourceUrl` 必须是 Host 受控资源地址；
4. `cover.mimeType` 当前正式固定为 `text/html`，对应 `.card/cover.html`；
5. `fields` 为空时，Host 默认返回全部稳定字段。

## 3. `card.open`

### 3.1 请求

```ts
interface CardOpenRequest {
  cardFile: string;
}
```

### 3.2 响应

```ts
interface CardOpenResponse {
  result: {
    mode: 'card-window';
    windowId?: string;
    pluginId?: string;
  };
}
```

### 3.3 正式要求

1. Host 必须优先复用正式卡片查看器 / 应用插件文件处理能力；
2. 若当前环境没有专门处理器，Host 可以使用自身宿主窗口承载正式查看态；
3. 调用方不得自行假设具体查看器实现、窗口标题和打开参数；
4. `card.open` 是生态正式卡片打开入口，箱子 `box.openEntry` 应在本地卡片场景下复用它。
