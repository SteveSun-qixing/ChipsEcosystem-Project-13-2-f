# 资源图像转换契约

> 文档状态：首版冻结稿
> 适用范围：Host `resource` 服务、`chips-sdk` 资源 API、生态内需要把 TIFF 资源归一为 PNG 文件的调用方

## 1. 定位

资源图像转换继续构建在 Host `resource` 服务之上：

- 不新增新的服务域；
- 不引入新的插件类型；
- 调用方通过正式 `resource.convertTiffToPng` 动作访问能力；
- 平台差异继续收敛到 PAL，调用方不感知底层平台细节。

## 2. 正式动作

- 动作名：`resource.convertTiffToPng`
- SDK 对应封装：`client.resource.convertTiffToPng(...)`

## 3. 输入契约

```ts
interface ResourceConvertTiffToPngRequest {
  resourceId: string;
  outputFile: string;
  overwrite?: boolean;
}
```

约束：

- `resourceId` 只允许本地文件路径或 `file://` 本地资源；
- 输入文件必须是合法 TIFF 文件；
- `outputFile` 必须指向 `.png` 文件路径；
- `overwrite` 默认视为 `false`。

## 4. 输出契约

```ts
interface ResourceConvertTiffToPngResult {
  outputFile: string;
  mimeType: "image/png";
  sourceMimeType: "image/tiff";
  width?: number;
  height?: number;
}
```

语义：

- `outputFile` 指向已经落盘完成的 PNG 文件；
- `width/height` 是 Host 在输出 PNG 上读取到的像素尺寸；
- 当前正式结果不返回二进制正文，调用方若需要继续读文件，应再走 `file.read` 或 `resource.readBinary`。

## 5. 权限与执行约束

- 所需权限：`resource.read`、`file.read`、`file.write`
- Host 路由超时：15 秒
- 输出目录由 Host 创建；若目标文件已存在且 `overwrite !== true`，必须直接失败

## 6. 错误码基线

当前首版冻结以下错误语义：

- `RESOURCE_TIFF_UNSUPPORTED_URI`：输入不是本地文件路径或 `file://` 本地资源
- `RESOURCE_TIFF_INVALID_SOURCE`：输入文件不是合法 TIFF
- `RESOURCE_TIFF_SOURCE_NOT_FOUND`：输入文件不存在
- `RESOURCE_TIFF_INVALID_OUTPUT`：输出路径非法或不是 `.png`
- `RESOURCE_TIFF_OUTPUT_EXISTS`：输出文件已存在且未允许覆盖
- `RESOURCE_TIFF_CONVERSION_UNSUPPORTED`：当前 Host 运行时不具备 TIFF 转 PNG 能力
- `RESOURCE_TIFF_CONVERSION_FAILED`：底层转换执行失败

## 7. 设计边界

- 该契约当前只冻结 `TIFF -> PNG` 单向转换，不在本轮扩展其他格式映射；
- 输出始终是磁盘文件，不定义内存态返回；
- 若未来需要更多图片格式归一化能力，应在保持 `resource` 服务语义统一的前提下扩展，不新增第二套路由体系。
