# 图像颜色提取模块能力契约

> 文档状态：首版正式稿
> 适用范围：生态内需要从图片中提取背景色与个性色的调用方

## 1. 定位

该能力由模块插件提供，用于从单张本地图片中提取两个代表色：

- 一个更适合纯色背景的背景色；
- 一个更适合作为强调 / 个性色的高亮色。

当前正式能力：

- `capability`: `image.color.pick`
- `method`: `pick`

## 2. 输入契约

```ts
interface ImageColorPickInput {
  imagePath: string;
  options?: {
    sampleSize?: number;
  };
}
```

约束：

- `imagePath` 只接受本地绝对路径或 `file://` 本地 URL；
- 当前不接受远程 URL；
- `sampleSize` 取值范围为 `48-160`，默认 `96`。

## 3. 输出契约

```ts
interface ImageColorPickOutput {
  backgroundColor: string;
  accentColor: string;
}
```

语义：

- `backgroundColor` 是更适合大面积铺底、浓郁且稳重，但不会被压得过暗的颜色；
- `accentColor` 是更适合按钮、强调点、个性化视觉，并会被提亮增艳的颜色；
- 两个字段均使用 `#rrggbb` 十六进制格式。

## 4. 权限

- `file.read`

## 5. 错误语义

当前首版冻结以下错误码：

- `COLOR_PICKER_INPUT_INVALID`
- `COLOR_PICKER_INPUT_NOT_FOUND`
- `COLOR_PICKER_IMAGE_EMPTY`
- `COLOR_PICKER_IMAGE_SAMPLE_FAILED`
- `COLOR_PICKER_IMAGE_SAMPLE_INVALID`
- `COLOR_PICKER_IMAGE_SAMPLE_UNSUPPORTED`
- `COLOR_PICKER_ANALYSIS_FAILED`

## 6. 调用建议

- 调用方应把该能力视为图片到颜色结果的正式入口；
- 调用方不应自己复制另一套主色 / 强调色提取算法；
- 若后续需要更多颜色、前景文字推荐色或对比度评级，应在本能力之上扩展新的正式契约，而不是修改已有字段语义。
