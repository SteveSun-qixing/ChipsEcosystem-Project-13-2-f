# 多语言规范

## 概述

多语言系统支持生态的国际化，让界面可以显示不同语言的文本。多语言系统采用键值对存储翻译文本，支持多种语言并行安装，运行时可以动态切换语言。

## i18n 服务接口（vNext 冻结口径）

i18n 服务提供以下固定动作接口：

| 动作 | 说明 | 幂等 |
|---|---|---|
| `i18n.getCurrent()` | 获取当前语言环境 | 是 |
| `i18n.setCurrent({ locale })` | 设置当前语言环境 | 否 |
| `i18n.translate({ key, params? })` | 翻译指定键 | 是 |
| `i18n.listLocales()` | 获取已安装语言列表 | 是 |

**迁移说明**：旧动作 `getCurrentLanguage`、`setLanguage`、`setLocale` 已归档，仅做内部兼容映射。

**语言切换事件**：切换语言时触发 `language.changed` 事件，页面订阅后只做重渲染，不重启应用。

## 语言包结构

语言包是一个JSON文件，键是翻译键，值是翻译文本。键采用点号分隔的层级结构，如card.create.button表示创建卡片按钮的文本。顶级键通常是功能模块名，如common表示通用文本，card表示卡片相关文本，box表示箱子相关文本，settings表示设置相关文本。

语言包文件命名采用语言代码加.json后缀，如en.json表示英语，zh-CN.json表示简体中文，ja.json表示日语。语言代码遵循BCP 47标准。

## 加载链（vNext 冻结口径）

翻译加载链按优先级从高到低：

1. **插件词典** - 当前加载插件的翻译文本
2. **应用词典** - 当前应用的翻译文本
3. **生态词典** - 薯片生态通用翻译文本

缺失 key 治理规则：
- 缺失 key 必须触发告警并回退默认文案模板
- CI 门禁输出缺失 key 列表
- 缺失率超过阈值（建议 5%）禁止发布

## 文本插值

翻译文本支持变量插值，在文本中使用占位符引用动态值。占位符使用花括号包裹，如Hello, {name}表示显示时name变量被替换为实际值。复数形式使用特殊语法，如{count, plural, =0 {No items} =1 {One item} other {# items}}根据数量显示不同文本。

## 性别占位符

某些语言需要区分性别，使用{gender, select, male {His} female {Her} other {Their}}根据性别显示不同文本。

## 语言切换

系统提供语言切换API，允许运行时动态改变界面语言。切换语言后，所有使用翻译函数的文本自动更新，无需重新加载页面。

## 翻译函数

插件前端使用翻译函数获取翻译文本。翻译函数接收键名和可选的参数对象，返回翻译后的文本。当翻译键不存在时，返回键名本身作为后备文本。

翻译函数作为全局函数暴露，开发者可以在JavaScript代码中直接调用。对于HTML模板，提供对应的指令或组件封装翻译功能。

## 语言检测

系统自动检测用户系统的首选语言，在首次运行时使用检测结果作为默认语言。用户可以在设置中手动选择语言，选择结果会被保存。

## 语言优先级

当某个翻译键在当前语言包中不存在时，系统使用备用语言链查找翻译。默认的备用语言链是当前语言的语言家族、然后是英语、最后是键名本身。

## RTL语言支持

系统支持从右到左书写的语言如阿拉伯语和希伯来语。当切换到RTL语言时，界面布局自动反转适应阅读习惯。RTL支持包括文字方向、图标方向、滚动方向等。

## 日期和时间格式

不同语言和地区使用不同的日期时间格式。系统使用Intl.DateTimeFormat API格式化日期时间，根据当前语言和地区自动应用正确的格式。

## 数字格式

不同语言使用不同的数字格式，包括小数点分隔符、千位分隔符、货币符号等。系统使用Intl.NumberFormat API格式化数字。

## 开发工作流

开发者在源代码中使用翻译键而非硬编码文本。翻译键命名遵循一致的约定，便于翻译人员理解上下文。翻译文件与源代码分离，翻译人员可以独立维护翻译文件。
