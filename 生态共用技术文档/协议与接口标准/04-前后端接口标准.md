# 生态对外服务 API 标准

> **文档状态**：本标准描述薯片生态与外部服务端（如社区服务器、云服务）之间的通信规范。
>
> **重要说明**：前端应用与 Host 内核的通信不适用本标准，应参考《Bridge API 规范》和《薯片协议规范》中的 Bridge IPC 架构。

## 概述

本文档定义薯片生态客户端与外部服务端之间的通信规范。**注意**：前端应用与后端 Host 内核之间通过 Bridge IPC（`window.chips.invoke`）通信，不使用 REST 风格，详见《Bridge API 规范》。

本标准适用于：
- 生态客户端与社区服务器的同步通信
- 生态客户端与云端服务的 API 调用
- 第三方服务集成场景

## 接口设计原则

接口遵循REST风格，使用标准的HTTP方法和URL结构。GET用于获取资源，POST用于创建资源，PUT用于更新资源，DELETE用于删除资源。URL使用名词表示资源，如/cards表示卡片资源，/boxes表示箱子资源。

接口使用JSON格式传输数据，请求Content-Type为application/json，响应返回JSON格式数据。每个响应都包含状态码和消息，成功请求返回200系列状态码，客户端错误返回400系列状态码，服务端错误返回500系列状态码。

接口版本通过URL路径或请求头指定，如/v1/cards或Accept: application/vnd.chips.v1+json。

## 认证授权

接口使用Bearer Token认证，客户端在请求头中携带授权令牌。Token通过登录接口获取，包含用户身份和权限信息。Token有过期时间，过期后需要重新获取。

授权使用权限模型，每个接口需要特定权限才能访问。用户权限存储在后端数据库，调用接口时验证权限是否满足。

## 卡片接口

创建卡片接口使用POST方法，URL为/cards。请求体包含卡片名称、标签、初始内容等。响应返回创建的卡片ID和完整信息。

读取卡片接口使用GET方法，URL为/cards/:id。响应返回卡片的完整信息，包括元数据、内容列表、结构定义等。

更新卡片接口使用PUT方法，URL为/cards/:id。请求体包含要更新的字段，支持部分更新。响应返回更新后的卡片信息。

删除卡片接口使用DELETE方法，URL为/cards/:id。响应返回操作结果，确认删除成功。

列出卡片接口使用GET方法，URL为/cards。查询参数支持分页、排序、筛选。响应返回卡片列表和分页信息。

## 箱子接口

箱子在对外服务中仍然是“URL 引用容器”而不是“卡片实体容器”。因此，箱子接口默认返回元数据、条目摘要和布局配置，不直接返回全部卡片展开内容。

### 箱子资源接口

- `POST /boxes`：创建箱子。请求体包含箱子名称、标签、`entries`、`activeLayoutType` 与 `layoutConfigs`。响应返回箱子标识、元数据和首批条目摘要。
- `GET /boxes/:id`：读取箱子。响应返回箱子元数据、当前布局类型、布局配置和条目摘要页；若条目很多，摘要页必须分页。
- `PUT /boxes/:id`：更新箱子。请求体可以更新 `metadata`、`entries` 与 `layoutConfigs`，但仍只记录卡片 URL 与摘要快照。
- `DELETE /boxes/:id`：删除箱子资源。
- `GET /boxes`：分页列出箱子。列表项返回箱子级摘要，如名称、标签、封面资源、条目总数和当前布局类型。

### 箱子消费会话接口

当外部服务需要支持“像 Host 一样的箱子消费模型”时，必须使用会话化接口，而不是把全部卡片数据一次性展开返回。

- `POST /boxes/:id/view-sessions`：打开箱子查看会话。响应返回 `sessionId`、箱子基础信息和首批条目摘要。
- `POST /boxes/:id/view-sessions/:sessionId/entries/query`：分页查询条目摘要，支持 `cursor`、`limit`、`filter`、`sort`。
- `POST /boxes/:id/view-sessions/:sessionId/entries/detail`：按条目读取补充字段，例如封面描述、卡片元数据、运行时属性与状态。
- `POST /boxes/:id/view-sessions/:sessionId/resources/resolve`：解析指定条目的封面、预览或卡片入口资源，返回受控资源句柄。
- `GET /boxes/:id/view-sessions/:sessionId/assets`：读取箱子自有资源，资源路径通过查询参数或路径变量传入。
- `DELETE /boxes/:id/view-sessions/:sessionId`：关闭会话。

正式约束：

- 箱子条目统一使用 URL，不再区分内部 / 外部；
- 箱子会话接口必须返回摘要优先的数据模型；
- 不在箱子响应中直接嵌入被引用卡片实体；
- 鉴权信息不得拼入条目 URL 或返回给前端消费者。

## 资源接口

上传资源接口使用POST方法，URL为/resources。请求体包含资源文件和元数据。响应返回资源的存储路径和访问URL。

下载资源接口使用GET方法，URL为/resources/:id。响应返回资源文件流。

删除资源接口使用DELETE方法，URL为/resources/:id。响应返回操作结果。

## 用户接口

注册接口使用POST方法，URL为/auth/register。请求体包含用户名、邮箱、密码。响应返回用户信息和认证Token。

登录接口使用POST方法，URL为/auth/login。请求体包含邮箱和密码。响应返回认证Token和用户信息。

获取用户信息接口使用GET方法，URL为/users/:id。响应返回用户公开信息。

## 错误响应格式

错误响应包含错误代码、错误消息、详细信息字段。错误代码是数字，如40001表示参数错误，40101表示未授权，50001表示服务器内部错误。错误消息是面向用户的描述，详细信息包含调试信息。

## 分页规范

分页使用limit和offset参数。limit指定每页数量，默认20，最大100。offset指定跳过的记录数。响应返回总记录数、当前页码、每页数量、总页数。

## 速率限制

接口实施速率限制，防止滥用。限制通过X-RateLimit-Limit和X-RateLimit-Remaining响应头告知客户端。当达到限制时返回429状态码。

## 缓存策略

接口支持ETag和Last-Modified缓存机制。客户端可以发送If-None-Match或If-Modified-Since请求头，服务端根据条件返回完整响应或304状态码。
