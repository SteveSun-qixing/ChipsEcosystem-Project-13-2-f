# 开发环境配置指南

## 环境要求

开发环境需要满足以下软件要求。Node.js版本需要18或更高版本。包管理器使用npm或yarn。代码编辑器推荐使用VSCode并安装生态提供的扩展包。Git版本控制工具。

硬件要求包括至少8GB内存以保证编译速度，至少20GB可用磁盘空间，操作系统支持Windows、macOS、Linux。

## 安装步骤

安装开发工具的步骤如下。首先克隆项目仓库到本地。然后安装依赖运行npm install。在生态工作区内创建或进入插件工程后，需要在项目 `package.json` 中声明本地 `chips-sdk` 开发依赖并再次执行 `npm install`，确保可以在工程目录直接调用 `chipsdev`。接着配置本地开发环境运行chipsdev init。最后启动开发服务器运行chipsdev server。

项目目录结构包括src目录存放源代码，dist目录存放编译输出，tests目录存放测试文件，docs目录存放文档，config目录存放配置文件。

## 开发工具

chipsdev 是开发者命令行工具，提供以下命令。chipsdev init 初始化开发环境。chipsdev server 启动开发服务器。chipsdev run 在开发模式下启动完整 Host 并运行应用插件。chipsdev build 编译项目。chipsdev test 运行测试。chipsdev lint 代码检查。

VSCode扩展提供以下功能。语法高亮和自动补全。代码片段和模板。调试支持和断点设置。任务运行器和终端集成。

## 调试配置

VSCode调试配置在launch.json文件中设置。调试类型包括Node.js调试用于主进程，Chrome调试用于渲染进程。调试配置可以设置断点、变量观察、调用堆栈等。

开发模式下启用热重载，代码修改后自动重新加载。开发者可以在浏览器开发者工具中查看日志和调试渲染进程。

## 代码规范

代码规范遵循生态制定的编码标准。ESLint配置在.eslintrc文件中。Prettier配置在.prettierrc文件中。编辑器自动格式化保存时触发格式化。

提交前运行lint检查和测试，确保代码质量。Git钩子会自动运行pre-commit检查。

## 测试配置

测试框架使用Vitest。测试文件放在tests目录中，文件命名以.test.ts或.spec.ts结尾。运行测试使用npm test或chipsdev test。

单元测试验证独立函数和模块。集成测试验证组件间协作。端到端测试验证完整用户场景。

## 性能优化

开发环境可以进行性能优化。启用增量编译只编译修改的文件。设置缓存减少重复构建。使用并行处理加速编译。

开发时可以使用生产构建配置测试实际性能。性能分析工具可以帮助定位瓶颈。

## 常见问题

常见问题包括依赖安装失败、端口被占用、权限问题等。解决方案包括清理node_modules重新安装、修改端口配置、以管理员权限运行等。

## 环境变量

环境变量在.env文件中配置。development开发环境配置，staging测试环境配置，production生产环境配置。敏感信息不应提交到版本控制。

## 持续集成

持续集成配置在.github/workflows目录中。自动化流水线包括代码检查、测试运行、构建发布等步骤。持续集成确保代码质量。

## 热重载

热重载功能在开发时自动启用。前端代码修改后页面自动刷新。主进程代码修改后应用自动重启。样式修改后页面局部更新无需刷新。
