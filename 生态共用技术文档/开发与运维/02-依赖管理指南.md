# 依赖管理指南

## 概述

依赖管理是项目开发的重要组成部分。生态使用npm作为包管理器，遵循Node.js生态的最佳实践。

## 依赖类型

项目依赖分为多种类型。dependencies是运行时依赖，项目运行所必需的包。devDependencies是开发依赖，仅在开发过程中使用的包。peerDependencies是对等依赖，插件声明其兼容的宿主环境。optionalDependencies是可选依赖，可能不存在的依赖。

## 版本规范

版本号遵循语义化版本规范。格式为主版本号.次版本号.修订号。^1.0.0表示兼容1.x.x版本。~1.0.0表示兼容1.0.x版本。*表示任意版本。

锁定版本使用package-lock.json文件，确保所有机器安装相同版本。提交lock文件到版本控制。

## 安装命令

安装依赖使用npm install。添加依赖使用npm install package-name。添加开发依赖使用npm install --save-dev package-name。

## 更新依赖

检查更新使用npm outdated命令，显示当前版本和最新版本。更新依赖使用npm update命令，更新到符合版本范围的最新版本。更新主版本使用npm install package-name@latest。

## 删除依赖

删除依赖使用npm uninstall package-name命令。删除开发依赖使用npm uninstall --save-dev package-name命令。删除后手动从package.json中移除对应条目。

## 镜像配置

国内访问npm仓库可能较慢，可以配置镜像。使用npm config set registry命令设置镜像地址。常见镜像包括淘宝npm镜像。

## 私有依赖

私有包使用npmrc文件配置认证信息。配置认证令牌访问私有仓库。发布包到私有仓库使用npm publish命令。

## 依赖安全

定期运行npm audit检查安全漏洞。修复漏洞使用npm audit fix命令。关注依赖的CVE报告。

## Monorepo

大型项目可能使用Monorepo结构。使用npm workspaces管理多个包。工作区根目录的package.json定义工作区。子包可以相互依赖。

## CI/CD

持续集成中需要缓存依赖目录加快构建。使用npm ci代替npm install确保安装lock文件中的精确版本。CI环境应该有干净的环境确保一致性。

## 最佳实践

遵循以下最佳实践：使用lock文件锁定版本，定期更新依赖保持安全，审查依赖许可证避免法律风险，移除未使用的依赖减少体积。
