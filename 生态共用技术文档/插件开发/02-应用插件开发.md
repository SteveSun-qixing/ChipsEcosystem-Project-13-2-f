# 应用插件开发

## 文档定位

本文档说明 `type: app` 插件的正式开发口径。

当前关键变化：

- 应用插件已升级为跨平台界面插件
- 正式容器语义从 `window` 升级为 `surface`
- Manifest 必须声明 `runtime.targets` 与 `ui.surface`

## 1. 应用插件的正式定义

应用插件不是“只能在桌面独立窗口里运行的 HTML 页面”，而是：

- 由 Host 托管
- 通过 `surface` 语义启动
- 可按宿主映射到不同界面容器的跨平台界面插件

当前映射：

- Desktop：`window`
- Web：预留 `route / tab / modal`
- Mobile：预留 `fullscreen / sheet`

## 2. 基本结构

应用插件工程至少包括：

- 工程根 `manifest.yaml`
- 前端入口文件
- 构建产物目录 `dist/`
- 静态资源

正式约束：

- `entry` 应指向构建产物，例如 `dist/index.html`
- 入口页必须声明正式 `Content-Security-Policy`
- 静态资源路径必须支持 Host 以本地入口方式加载

## 3. 必填 Manifest 重点字段

应用插件必须重点声明：

```yaml
type: app
entry: dist/index.html
runtime:
  targets:
    desktop:
      supported: true
    web:
      supported: false
    mobile:
      supported: false
    headless:
      supported: false
ui:
  surface:
    defaultKind: window
    preferredKinds:
      desktop: window
      web: route
      mobile: fullscreen
      headless: window
```

可选但常用字段：

- `ui.window.chrome`
- `ui.launcher`
- `capabilityFallbacks`
- `capabilities`

## 4. 启动链路

当前应用插件正式启动方式有两条：

1. `client.surface.open({ target: { type: "plugin", pluginId } })`
2. `client.plugin.launch(pluginId, launchParams?)`

推荐顺序：

- 新的跨平台调用优先使用 `surface.open(target=plugin)`
- 已有桌面兼容代码可以继续使用 `plugin.launch`

### 启动上下文

应用插件如果要知道“为什么被打开”，必须读取 launch context：

```ts
const launchContext = client.platform.getLaunchContext();
```

或：

```ts
const launchContext = window.chips.platform.getLaunchContext();
```

返回：

```ts
{
  pluginId?: string;
  sessionId?: string;
  launchParams: Record<string, unknown>;
}
```

常见触发来源：

- `app-shortcut`
- `file-association`
- `chipsdev.run`
- `resource.open`

## 5. 系统入口与快捷方式

应用插件可以通过 `ui.launcher` 声明系统入口元数据。

当前正式链路：

1. 治理应用调用 `plugin.createShortcut`
2. Host 读取 `ui.launcher.displayName` 与 `ui.launcher.icon`
3. Host 通过 PAL / Shell 创建平台入口
4. 用户触发入口后，Host 统一走应用插件正式启动链路

约束：

- 应用插件不得自行创建 `.lnk`、`.app` 或其他系统快捷方式文件

## 6. 资源处理器

如果应用插件要承担资源查看 / 处理职责，应声明：

```yaml
capabilities:
  - resource-handler:view:image/*
  - file-handler:.png
  - file-handler:.jpg
```

上游应用应统一调用：

```ts
await client.resource.open({
  intent: "view",
  resource: {
    resourceId,
    mimeType,
    title,
    fileName,
  },
});
```

应用侧应从 launch context 中读取：

- `launchParams.resourceOpen`
- 必要时读取 `launchParams.targetPath`

## 7. 前端栈与 UI 规则

- 官方前端栈统一采用 React
- 界面文案必须走多语言系统
- 视觉样式必须走主题系统与 token
- 不得依赖 Electron 私有 API

## 8. 能力调用规则

应用插件访问 Host 能力必须使用：

- `chips-sdk`
- `window.chips.*`

不得：

- 直接访问 Node.js API
- 直接操作原生窗口对象
- 私下约定跨应用启动协议绕过 Host

## 9. 当前开发建议

1. 新工程直接使用官方 Scaffold 生成。
2. 发布前至少执行：
   - `chipsdev validate`
   - `chipsdev package`
3. 开发运行优先使用 `chipsdev run`，由完整 Host 链路托管应用插件。
