# 图像颜色提取模块使用指南

> 适用范围：生态内需要从单张图片中提取背景色与个性色的调用方
> 主契约文档：`生态共用技术文档/协议与契约/12-图像颜色提取模块能力契约.md`

## 1. 能力概览

当前生态内的图像颜色提取能力由模块插件提供，正式调用口径如下：

- `capability`: `image.color.pick`
- `method`: `pick`
- `mode`: `sync`

该能力接收一张本地图片，返回两个十六进制颜色值：

- `backgroundColor`
  - 适合纯色背景、卡片底色、大面积铺底。
- `accentColor`
  - 适合按钮、强调点、个性化高亮元素。

## 2. 什么时候使用

适合以下场景：

- 音乐播放器根据专辑封面提取背景色与按钮色；
- 基础卡片根据封面或海报生成统一视觉主色；
- 作品卡片、资源卡片根据图片生成强调色；
- 其他需要“图片 -> 两个稳定代表色”能力的模块或应用。

不建议以下做法：

- 调用方自己复制另一套主色提取算法；
- 把该能力返回的两个颜色直接当成文本前景色；
- 传入远程 URL 或非本地图片路径。

## 3. 输入与输出

### 3.1 输入

```ts
interface ImageColorPickInput {
  imagePath: string;
  options?: {
    sampleSize?: number;
  };
}
```

约束：

- `imagePath` 只接受本地绝对路径或 `file://` 本地 URL；
- 当前不接受远程 URL；
- `sampleSize` 取值范围 `48-160`；
- 默认 `sampleSize = 96`。

### 3.2 输出

```ts
interface ImageColorPickOutput {
  backgroundColor: string;
  accentColor: string;
}
```

返回值格式：

- 两个字段都使用 `#rrggbb` 十六进制格式；
- `backgroundColor` 更适合铺底；
- `accentColor` 更适合强调和高亮。

## 4. SDK 调用示例

```ts
import { createClient } from "chips-sdk";

const client = createClient();

const result = await client.module.invoke({
  capability: "image.color.pick",
  method: "pick",
  input: {
    imagePath: "/absolute/path/to/cover.png",
  },
});

if (result.mode === "sync") {
  const { backgroundColor, accentColor } = result.output;
  console.log(backgroundColor, accentColor);
}
```

带可选参数的示例：

```ts
const result = await client.module.invoke({
  capability: "image.color.pick",
  method: "pick",
  input: {
    imagePath: "/absolute/path/to/poster.jpg",
    options: {
      sampleSize: 96,
    },
  },
});
```

## 5. 使用建议

- 常规场景直接使用默认 `sampleSize = 96` 即可。
- 若图片很大且更关注速度，可保持默认值，不必继续放大采样尺寸。
- 若希望颜色更稳定用于大面积铺底，优先使用 `backgroundColor`。
- 若希望按钮、进度条、图标更显眼，优先使用 `accentColor`。
- 若调用方还需要文字颜色，请额外做对比度判断，不要直接复用这两个返回色作为文字色。

## 6. 错误处理

当前调用方至少应识别以下错误码：

- `COLOR_PICKER_INPUT_INVALID`
- `COLOR_PICKER_INPUT_NOT_FOUND`
- `COLOR_PICKER_IMAGE_EMPTY`
- `COLOR_PICKER_IMAGE_SAMPLE_FAILED`
- `COLOR_PICKER_IMAGE_SAMPLE_INVALID`
- `COLOR_PICKER_IMAGE_SAMPLE_UNSUPPORTED`
- `COLOR_PICKER_ANALYSIS_FAILED`

建议处理方式：

- 路径错误、文件不存在时，回退到调用方默认颜色；
- 图片解码失败时，记录日志并使用兜底主题色；
- 不要把模块错误直接暴露为未处理异常导致页面中断。

## 7. 性能说明

- 该能力正式集成到长驻 Host 进程后，模块本体调用是快速返回的。
- 对同一张未变化图片的重复调用，模块会直接复用缓存结果。
- 开发阶段如果使用 `chipsdev module invoke` 观察到秒级耗时，通常包含构建、拉起 Electron Host 和安装开发插件的开销，不代表模块算法本身慢。

## 8. 联调命令

开发期可使用以下命令验证：

```bash
chipsdev module invoke \
  --capability image.color.pick \
  --method pick \
  --input '{"imagePath":"/absolute/path/to/cover.png"}'
```

## 9. 注意事项

- 当前能力只保证“返回两个稳定代表色”，不保证输出完整调色板。
- 若后续需要更多颜色、推荐文字色、对比度评级，应新增正式契约扩展，而不是改变当前字段语义。
- 当前能力是生态内正式入口，新的消费方应直接接入该能力，不要再私自演化分叉版本。
