# 卡片文件格式规范

**版本**: 1.0.0  
**更新时间**: 2026-04-18
**状态**: 正式版

---

## 1. 文件格式概述

### 1.1 文件本质

卡片文件(`.card`)的本质是一个ZIP压缩包,后缀名改为`.card`。

- **压缩算法**: ZIP存储模式(零压缩率)
- **编码**: UTF-8
- **文件扩展名**: `.card`
- **MIME类型**: `application/x-card`

### 1.2 零压缩率说明

使用ZIP存储模式而不压缩的原因:
- 最快的文件访问速度
- 简化解析流程
- 支持随机访问
- 文件体积影响可以忽略(多数资源已压缩)

### 1.3 文件识别

操作系统文件关联:
- 双击`.card`文件 → 自动用卡片查看器打开
- 修改扩展名为`.zip` → 可用任何解压工具打开查看内部结构

## 2. 目录结构

### 2.1 根目录结构

```
card-example.card (ZIP压缩包)
│
├── .card/                    # 核心配置目录(隐藏文件夹)
│   ├── metadata.yaml         # 卡片元数据
│   ├── structure.yaml        # 卡片结构定义
│   ├── cover.html            # 封面HTML
│   └── cardcover/            # 封面资源(可选)
│       ├── cover-image.jpg
│       └── cover-style.css
│
├── content/                  # 基础卡片配置目录
│   ├── a1B2c3D4e5.yaml      # 基础卡片配置文件
│   ├── f6G7h8I9j0.yaml      # 基础卡片配置文件
│   └── ...
│
├── video.mp4                 # 资源文件(根目录)
├── image.jpg                 # 资源文件(根目录)
├── subtitle-zh.srt          # 资源文件(根目录)
├── nested-card.card         # 嵌套的复合卡片(可选)
└── ...
```

### 2.2 文件组织原则

1. **配置集中**: 所有配置文件在`.card/`文件夹
2. **内容扁平**: 基础卡片配置在`content/`目录,不使用子文件夹
3. **资源根放**: 所有资源文件直接放在根目录,不分类
4. **隐藏配置**: `.card/`文件夹为隐藏文件夹(Unix/Linux标准)

补充正式约束:
- 基础卡片配置中记录的卡片内部资源路径,统一相对于卡片根目录;
- 官方资源导入与保存链路必须把内部资源写入卡片根目录,不得写入`content/`或其他临时目录;
- 运行时生成的`blob:`、`data:`、系统绝对路径不得写回正式内容配置。

## 3. 元数据文件 (metadata.yaml)

### 3.1 完整Schema定义

```yaml
# JSON Schema for metadata.yaml
$schema: "http://json-schema.org/draft-07/schema#"
title: "Card Metadata Schema"
type: object
required:
  - chip_standards_version
  - card_id
  - name
  - created_at
  - modified_at
  
properties:
  # === 基础信息 ===
  chip_standards_version:
    type: string
    pattern: '^\d+\.\d+\.\d+$'
    description: "薯片标准版本号"
    example: "1.0.0"
    
  card_id:
    type: string
    pattern: '^[0-9a-zA-Z]{10}$'
    description: "卡片唯一ID,10位62进制编号"
    example: "a1B2c3D4e5"
    
  name:
    type: string
    maxLength: 500
    description: "卡片名称,最长500字符"
    example: "我的学习笔记"
    
  created_at:
    type: string
    format: date-time
    description: "创建时间,ISO 8601格式,UTC时区"
    example: "2026-01-15T08:00:00.000Z"
    
  modified_at:
    type: string
    format: date-time
    description: "最后修改时间"
    example: "2026-01-30T10:30:00.000Z"
    
  # === 主题设置 ===
  theme:
    type: string
    description: "主题包标识,格式: '发行商:主题包名称'"
    default: ""
    example: "薯片官方:默认主题"

  cover_ratio:
    type: string
    pattern: '^\d+(\.\d+)?:\d+(\.\d+)?$'
    description: "封面比例,使用宽:高字符串保存,默认3:4"
    default: "3:4"
    example: "3:4"
    
  # === 标签系统 ===
  tags:
    type: array
    description: "标签数组,支持简单标签和结构化标签"
    items:
      oneOf:
        - type: string
          description: "简单标签"
        - type: array
          items:
            type: string
          minItems: 2
          description: "结构化标签,第一个元素为键,后续为值"
    examples:
      - ["动漫"]
      - ["时间戳", "15:20"]
      - ["地点", "日本", "东京"]
      - ["项目", "网站改版"]
      
  # === 可选扩展属性 ===
  visibility:
    type: string
    enum: ["public", "private", "unlisted"]
    default: "private"
    description: "可见性设置"
    
  downloadable:
    type: boolean
    default: true
    description: "是否允许下载"
    
  remixable:
    type: boolean
    default: true
    description: "是否允许二次创作"
    
  commentable:
    type: boolean
    default: true
    description: "是否允许评论"
    
  license:
    type: string
    description: "许可证类型"
    examples: ["CC-BY-4.0", "CC-BY-NC-4.0", "All Rights Reserved"]
    
  age_rating:
    type: string
    enum: ["all", "pg", "pg13", "r", "nc17"]
    description: "年龄分级"
    
  content_warning:
    type: array
    items:
      type: string
    description: "内容警告标签"
    examples: [["violence", "adult_content"]]
    
  # === 文件信息(系统自动生成) ===
  file_info:
    type: object
    properties:
      total_size:
        type: integer
        description: "卡片总大小(字节)"
      file_count:
        type: integer
        description: "文件总数"
      checksum:
        type: string
        description: "完整性校验值,SHA256"
        example: "e3b0c44298fc1c149afbf4c8996fb92427ae41e4649b934ca495991b7852b855"
      generated_at:
        type: string
        format: date-time
        description: "文件信息生成时间"
```

### 3.2 完整示例

```yaml
# metadata.yaml 示例

# 基础信息
chip_standards_version: "1.0.0"
card_id: "a1B2c3D4e5"
name: "日本旅行 2026"

modified_at: "2026-01-30T10:30:00.000Z"

# 主题设置
theme: "薯片官方:默认主题"
cover_ratio: "3:4"

# 标签系统
tags:
  - "旅行"
  - ["地点", "日本", "东京"]
  - ["时间", "2026年", "一月"]
  - ["类型", "个人记录"]
  - ["评分", "五星"]

# 可选属性
visibility: "public"
downloadable: true
remixable: true
commentable: true
license: "CC-BY-4.0"
age_rating: "all"
content_warning: []

# 文件信息(系统生成)
file_info:
  total_size: 52428800
  file_count: 15
  checksum: "a3f5c9b1e4d2f8a7b6c5d4e3f2a1b0c9d8e7f6a5b4c3d2e1f0"
  generated_at: "2026-01-30T10:30:00.000Z"
```

### 3.3 字段说明

**用户数据 vs 系统数据**:
- 用户使用数据(观看进度、浏览历史、收藏状态等)不保存在卡片文件中
- 这些数据保存在查看器/编辑器的本地数据库中
- 卡片文件只包含内容本身的信息

## 4. 结构文件 (structure.yaml)

### 4.1 完整Schema定义

```yaml
# JSON Schema for structure.yaml
$schema: "http://json-schema.org/draft-07/schema#"
title: "Card Structure Schema"
type: object
required:
  - structure
  - manifest

properties:
  # === 结构定义 ===
  structure:
    type: array
    description: "基础卡片列表,按渲染顺序排列"
    items:
      type: object
      required:
        - id
        - type
      properties:
        id:
          type: string
          pattern: '^[0-9a-zA-Z]{10}$'
          description: "基础卡片ID,对应content/目录下的配置文件"
        type:
          type: string
          description: "基础卡片类型,决定使用哪个渲染器"
          examples: ["VideoCard", "ImageCard", "RichTextCard", "MarkdownCard"]
          
  # === 清单索引 ===
  manifest:
    type: object
    description: "文件清单和元信息,用于快速索引和完整性验证"
    required:
      - card_count
      - resource_count
    properties:
      card_count:
        type: integer
        description: "基础卡片数量"
        
      resource_count:
        type: integer
        description: "资源文件数量"
        
      resources:
        type: array
        description: "资源文件清单"
        items:
          type: object
          required:
            - path
            - size
            - type
          properties:
            path:
              type: string
              description: "资源文件相对路径"
            size:
              type: integer
              description: "文件大小(字节)"
            type:
              type: string
              description: "文件MIME类型"
            duration:
              type: number
              description: "时长(秒),仅视频/音频文件"
            width:
              type: integer
              description: "宽度(像素),仅图片/视频文件"
            height:
              type: integer
              description: "高度(像素),仅图片/视频文件"
            language:
              type: string
              description: "语言代码,仅字幕文件"
            checksum:
              type: string
              description: "文件SHA256校验值"
```

### 4.2 完整示例

```yaml
# structure.yaml 示例

structure:
  - id: "a1B2c3D4e5"
    type: "RichTextCard"
    
  - id: "f6G7h8I9j0"
    type: "VideoCard"
    
  - id: "k1L2m3N4o5"
    type: "ImageCard"
    
  - id: "p6Q7r8S9t0"
    type: "MarkdownCard"

manifest:
  card_count: 4
  resource_count: 5
  
  resources:
    - path: "intro.rtf"
      size: 2048
      type: "application/rtf"
      
    - path: "travel-video.mp4"
      size: 52428800
      type: "video/mp4"
      duration: 180.5
      width: 1920
      height: 1080
      checksum: "b2c3d4e5f6a7b8c9d0e1f2a3b4c5d6e7f8a9b0c1d2e3f4a5b6c7d8e9f0a1"
      
    - path: "subtitle-zh.srt"
      size: 8192
      type: "application/x-subrip"
      language: "zh"
      
    - path: "photo.jpg"
      size: 1048576
      type: "image/jpeg"
      width: 3000
      height: 2000
      
    - path: "notes.md"
      size: 4096
      type: "text/markdown"
```

### 4.3 设计说明

**职责分离**:
- `structure.yaml`: 只负责结构关系和文件索引
- `content/*.yaml`: 负责具体内容配置
- `metadata.yaml`: 负责卡片自身元信息

## 5. 内容配置 (content/*.yaml)

### 5.1 通用字段

所有基础卡片配置文件都包含以下通用字段:

```yaml

  - card_type

    type: string
```


### 5.3 图片卡片配置

```yaml
# ImageCard配置Schema
$schema: "http://json-schema.org/draft-07/schema#"
title: "Image Card Config Schema"
type: object
required:
  - card_type
  - images
  - layout_type

properties:
  card_type:
    const: "ImageCard"

  theme:
    type: string
    description: "基础卡片主题覆盖标识,为空时沿用上层主题"

  images:
    type: array
    description: "图片列表,允许为空数组"
    items:
      type: object
      required:
        - id
        - source
      properties:
        id:
          type: string
          description: "图片项唯一ID"
        source:
          type: string
          enum: ["file", "url"]
        file_path:
          type: string
          description: "卡片内部文件路径,必须相对于卡片根目录"
        url:
          type: string
          description: "远程图片地址,仅允许 http/https 或 data:image/*"
        alt:
          type: string
        title:
          type: string

  layout_type:
    type: string
    enum: ["single", "grid", "long-scroll", "horizontal-scroll"]
    default: "single"

  layout_options:
    type: object
    properties:
      grid_mode:
        type: string
        enum: ["2x2", "3x3", "3-column-infinite"]
        default: "2x2"
      spacing_mode:
        type: string
        enum: ["none", "comfortable"]
        default: "comfortable"
      single_width_percent:
        type: number
        minimum: 10
        maximum: 100
        default: 100
      single_alignment:
        type: string
        enum: ["left", "center", "right"]
        default: "center"
```

**示例**:

```yaml
# content/k1L2m3N4o5.yaml
card_type: "ImageCard"
theme: ""

images:
  - id: "cover-1"
    source: "file"
    file_path: "photo.jpg"
    title: "东京塔夜景"
    alt: "东京塔夜景照片"
  - id: "cover-2"
    source: "url"
    url: "https://example.com/tokyo-night-2.jpg"
    title: "备用网络图"
    alt: "东京夜景补充图片"

layout_type: "grid"
layout_options:
  grid_mode: "2x2"
  spacing_mode: "comfortable"
  single_width_percent: 100
  single_alignment: "center"
```

补充说明:

- `images` 可以为空数组,此时图片基础卡片显示为空状态;
- `source = file` 时必须提供 `file_path`,且路径必须相对于卡片根目录;
- `source = url` 时必须提供合法的 `http/https` 或 `data:image/*` 地址;
- 正式保存链路会把导入文件写入卡片根目录,并在配置中仅记录相对路径,例如 `photo.jpg`;
- 当 `images.length <= 1` 时,运行时有效布局强制视为 `single`,即使配置中的 `layout_type` 为其他值;
- 当 `images.length > 1` 且传入 `layout_type = "single"` 时,归一化结果必须改写为 `grid`;
- `spacing_mode = "none"` 表示图片零间距且无圆角,`spacing_mode = "comfortable"` 表示图片保留适当间距并使用小圆角;
- `long-scroll` 为纵向完整展开布局,不允许再通过配置引入固定高度子滚动窗口。

### 5.4 网页卡片配置

```yaml
# WebPageCard配置Schema
$schema: "http://json-schema.org/draft-07/schema#"
title: "Web Page Card Config Schema"
type: object
required:
  - card_type
  - source_type
  - display_mode

    type: string
    description: "远程网页地址(source_type=url时使用)"

  bundle_root:
    type: string
    description: "网页目录在卡片根目录中的相对路径(source_type=bundle时使用)"

  entry_file:
    type: string
    description: "网页目录入口文件,正式必须为 index.html"
    default: "index.html"

  resource_paths:
    type: array
    description: "网页目录中所有已纳入卡片资源清单的文件路径"
    items:
      type: string

  display_mode:
    type: string
    enum: ["fixed", "free"]

  fixed_ratio:
    const: "7:16"

  max_height_ratio:
    const: 20
```

**示例1：远程网页**

```yaml
# content/w1E2b3P4g5.yaml
card_type: "WebPageCard"
source_type: "url"
source_url: "https://example.com/demo"
bundle_root: ""
entry_file: "index.html"
resource_paths: []
display_mode: "fixed"
fixed_ratio: "7:16"
max_height_ratio: 20
```

**示例2：卡片内网页目录**

```yaml
# content/w1E2b3P4g5.yaml
card_type: "WebPageCard"
source_type: "bundle"
source_url: ""
bundle_root: "webpage-bundle-a1b2c3"
entry_file: "index.html"
resource_paths:
  - "webpage-bundle-a1b2c3/index.html"
  - "webpage-bundle-a1b2c3/assets/app.js"
  - "webpage-bundle-a1b2c3/assets/style.css"
display_mode: "free"
fixed_ratio: "7:16"
max_height_ratio: 20
```

补充说明:

- 网页 ZIP 包导入后,正式产物不是压缩包本身,而是卡片根目录中的整个网页目录;
- 网页目录入口文件必须为 `index.html`;
- `bundle_root`、`entry_file`、`resource_paths` 都必须相对于卡片根目录记录;
- 网页卡片查看态正式支持两种显示模式:
  - `fixed`: 固定为宽:高 `7:16` 视口,并在该窗口内滚动网页;
  - `free`: 按页面实际高度展开,但最高不超过当前宽度的 `20` 倍;
- `fixed_ratio` 当前正式固定为 `7:16`,用于声明固定比例模式的标准比例。

### 5.5 富文本卡片配置

```yaml
# RichTextCard配置Schema
$schema: "http://json-schema.org/draft-07/schema#"
title: "Rich Text Card Config Schema"
type: object
required:
  - card_type
  - content_format
  - content_source

properties:
  card_type:
    const: "RichTextCard"

  content_format:
    const: "markdown"
    description: "富文本基础卡片正式只保存 Markdown 内容"
    
  content_source:
    type: string
    enum: ["file", "inline"]
    description: "Markdown 内容来源"
    
  content_file:
    type: string
    description: "卡片根目录相对 Markdown 文件路径(source为file时必需)"
    
  content_text:
    type: string
    description: "内联 Markdown 文本(source为inline时必需)"

  locale:
    type: string
    description: "内容语言"

  theme:
    type: string
    description: "基础卡片局部主题标识，未设置时为空字符串"
```

正式存储规则：

- 富文本基础卡片编辑器使用 Milkdown，但卡片文件中只保存 Markdown 数据，不保存编辑器内部 HTML；
- 当 Markdown 提取后的纯文本字符数 `<= 200` 时，必须写入 `content_text`；
- 当 Markdown 提取后的纯文本字符数 `> 200` 时，必须把正文写入卡片根目录 `.md` 文件，并在配置中通过 `content_file` 引用；
- `content_file` 必须是卡片根目录相对路径，不能写成绝对路径，也不能相对于 `content/` 目录；
- `inline` 与 `file` 两种模式互斥，正式配置中不得同时保存 `content_text` 与 `content_file`；
- 编辑器悬浮工具框属于运行时行为，不属于持久化配置字段。

**示例1：短内容内联保存**

```yaml
# content/a1B2c3D4e5.yaml
```

## 6. 封面文件 (cover.html)

### 6.1 封面HTML规范

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Card Cover</title>
  <style>
    /* 样式代码 */
    body {
      margin: 0;
      padding: 0;
      width: 100%;
      height: 100%;
      display: flex;
      align-items: center;
      justify-content: center;
    }
  </style>
</head>
<body>
  <!-- 封面内容 -->
  <div class="cover-container">
    <h1>卡片名称</h1>
  </div>
</body>
</html>
```

### 6.2 封面比例元数据

封面比例保存在 `metadata.yaml` 的 `cover_ratio` 字段中,格式为 `宽:高` 的正数比值字符串。

- 默认值: `3:4`
- 当前正式编辑器支持的预设比例:
  - `1:1`
  - `1:1.414`
  - `2:3`
  - `3:4`
  - `1:1.618`
  - `16:9`
  - `2:1`
  - `9:19.5`
  - `1.58:1`
- 横版通过交换宽高得到,例如 `3:4` 的横版保存为 `4:3`
- 运行时应按规范化后的字符串读取和回写 `cover_ratio`

### 6.3 正式显示链路

封面显示必须使用正式 Host + SDK 链路,不允许应用层自行拼接任意 HTML:

```text
card.renderCover({ cardFile })
  -> { view: { title, coverUrl, ratio? } }
  -> client.card.coverFrame.render({ cardFile })
  -> iframe(src = coverUrl, sandbox = "allow-scripts")
```

正式约束:

- `coverUrl` 必须指向受控的 `.card/cover.html` 入口
- `.card/cover.html` 中的相对路径相对于 `.card/` 目录解析,可继续引用 `.card/cardcover/*`
- 封面 iframe 内容完全由 `cover.html` 决定,Host/SDK 不自动注入主题系统、多语言系统或复合卡片壳层
- 应用层不得把封面内容改写为 `srcdoc` 或任意 HTML 字符串

### 6.4 封面创建方式

当前正式支持两种封面生成方式:

**方式1: 在线编辑 HTML 代码**
```
用户在编辑器中编写或粘贴 HTML
↓
系统直接保存到 .card/cover.html
```

**方式2: 上传图片并生成封面**
```
用户上传图片
↓
可按当前封面比例打开裁剪弹窗
↓
系统把生成图片保存到 .card/cardcover/
↓
系统生成引用该图片的 .card/cover.html
```

图片生成封面的正式 HTML 约定如下:

```html
<body
  data-chips-cover-mode="image"
  data-chips-cover-image-source="./cardcover/cover-image.ext"
>
  <img src="./cardcover/cover-image.ext" alt="" />
</body>
```

说明:

- `data-chips-cover-mode="image"` 用于标识该 `cover.html` 来自正式图片封面生成链路
- `data-chips-cover-image-source` 记录正式图片资源入口
- `cover-image.ext` 表示实际保存的图片扩展名；未裁剪时会尽量保留原始类型，裁剪输出当前正式实现固定为 PNG
- 生成的图片封面默认使用 `img { object-fit: cover; }`

### 6.5 正式支持边界

封面 HTML 在 iframe 中运行时,必须满足以下正式边界:

- 必须通过受控 `coverUrl` + `sandbox="allow-scripts"` 装载
- 正式支持的资源引用方式是访问 `.card/` 目录内的相对路径资源
- 封面文档不得依赖父窗口交互协议
- 封面文档不得把表单提交、弹窗或外部网络资源作为正式能力前提
- 应用层不得把封面文档当作同源 DOM 直接读取或挂接私有监听

### 6.6 默认封面

如果用户未设置封面,系统使用默认封面:

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Default Cover</title>
  <style>
    html, body { margin: 0; width: 100%; height: 100%; }
    body {
      display: grid;
      place-items: center;
      background: linear-gradient(135deg, #f5f7fb 0%, #eef3ff 100%);
      font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif;
      color: #111827;
      text-align: center;
      padding: 24px;
      box-sizing: border-box;
      overflow: hidden;
    }
    .card-name {
      font-size: 24px;
      font-weight: 600;

      word-wrap: break-word;
    }
  </style>
</head>
<body>
  <div class="card-name">{{CARD_NAME}}</div>
```

## 7. 资源组织模式

### 7.1 三种资源模式

#### 全填充卡片
```
card.card (ZIP)
├── .card/
├── content/
├── video.mp4          在内部
├── image.jpg          在内部
└── subtitle.srt       在内部
```

#### 空壳卡片
```
card.card (ZIP)
├── .card/
└── content/
    └── video-card.yaml
        video_file: "/external/path/video.mp4"    在外部
```

#### 半空壳卡片
```
card.card (ZIP)
├── .card/
├── content/
├── small-image.jpg    在内部
└── video-card.yaml
    video_file: "/external/large-video.mp4"    在外部
```

### 7.2 资源路径格式

**相对路径**(资源在卡片内部):
```yaml
video_file: "video.mp4"
file_path: "cover.png"
```

**绝对路径**(资源在文件系统):
```yaml
video_file: "/Users/username/Videos/video.mp4"
video_file: "C:\\Users\\username\\Videos\\video.mp4"  # Windows
```

**网络地址**(资源在网络):
```yaml
video_file: "https://example.com/videos/video.mp4"
video_file: "webdav://server/path/video.mp4"
```

### 7.3 嵌套卡片访问

嵌套卡片中的资源可以通过特殊路径访问:

```
card-parent.card
├── nested.card       # 嵌套的卡片
└── other-file.jpg

# 访问nested.card内的资源
path: "nested.card/internal-resource.jpg"

# 系统会自动识别.card文件为"目录"
```

### 7.4 资源缺失处理

当资源缺失时:

1. **卡片仍可打开**: 不会因为资源缺失而无法打开
2. **显示占位符**: 缺失的资源位置显示占位图和提示
3. **重新绑定**: 用户可以在设置中重新指定资源路径
4. **下载选项**: 如果资源有网络备份,提供下载选项

## 8. 文件命名规范

### 8.1 命名规则

1. **编码**: 使用UTF-8编码,支持中文文件名
2. **特殊字符**: 避免使用 `< > : " / \ | ? *`
3. **扩展名**: 使用小写(`.mp4`而非`.MP4`)
4. **空格**: 建议使用连字符`-`或下划线`_`代替空格
5. **长度**: 文件名建议不超过255字符

### 8.2 配置文件命名

```
content/目录:
  - 文件名格式: {10位62进制ID}.yaml
  - 示例: a1B2c3D4e5.yaml
  - 字符集: 0-9, a-z, A-Z
```

### 8.3 资源文件命名

**保留原名**: 用户导入的文件保留原文件名
```
用户导入 "my-video.mp4" → 保存为 "my-video.mp4"
```

**自动命名**: 编辑器创建的新内容使用卡片名称
```
卡片名: "旅行日记"
创建视频 → 保存为 "旅行日记-视频.mp4"
创建图片 → 保存为 "旅行日记-图片.jpg"
```

**冲突处理**: 文件名冲突时自动添加编号
```
photo.jpg        (第1个)
photo-2.jpg      (第2个)
photo-3.jpg      (第3个)
```

### 8.4 路径分隔符

**统一使用正斜杠**:
```yaml
# 正确
path: "folder/subfolder/file.jpg"

# 错误(Windows风格,不跨平台)
path: "folder\\subfolder\\file.jpg"
```

## 9. 版本兼容

### 9.1 版本号字段

```yaml
# metadata.yaml
chip_standards_version: "1.0.0"
```

### 9.2 版本演进策略

**主版本号变更** (1.x.x → 2.x.x):
- 不兼容的格式变更
- 查看器需要专门适配
- 提供格式转换工具

**次版本号变更** (1.0.x → 1.1.x):
- 向后兼容的功能新增
- 旧查看器可以忽略新字段
- 新查看器兼容旧格式

**修订号变更** (1.0.0 → 1.0.1):
- 向后兼容的问题修正
- 不影响文件格式
- 完全兼容

### 9.3 兼容性检查

查看器打开卡片时:

```javascript
// 伪代码
function checkCompatibility(cardVersion, viewerVersion) {
  const [cardMajor, cardMinor] = cardVersion.split('.');
  const [viewerMajor, viewerMinor] = viewerVersion.split('.');
  
  if (cardMajor !== viewerMajor) {
    return 'INCOMPATIBLE'; // 主版本不同,不兼容
  }
  
  if (parseInt(cardMinor) > parseInt(viewerMinor)) {
    return 'PARTIAL'; // 卡片版本更新,可能有未知字段
  }
  
  return 'COMPATIBLE'; // 完全兼容
}
```

## 10. 完整性验证

### 10.1 校验流程

打开卡片文件时:

1. **ZIP完整性**: 验证ZIP文件是否损坏
2. **必需文件**: 检查必需文件是否存在
3. **YAML格式**: 验证YAML文件格式正确性
4. **Schema验证**: 检查字段是否符合Schema定义
5. **资源清单**: 对比manifest中的资源是否存在
6. **校验值**: 可选,验证文件SHA256校验值

### 10.2 修复策略

**自动修复**:
- 缺少默认值的字段自动补充
- 错误的enum值修正为默认值

**提示修复**:
- 缺少必需文件时提示用户
- 资源缺失时提示重新绑定

**拒绝打开**:
- ZIP文件严重损坏
- metadata.yaml格式完全错误

## 11. 安全性

### 12.1 路径遍历防护

防止恶意路径访问父目录:

```
禁止的路径:
- ../../../etc/passwd
- ..\..\..\windows\system32

允许的路径:
- video.mp4
- folder/video.mp4
```

### 12.2 文件大小限制

建议的限制:

- 单个卡片文件: ≤ 2GB (ZIP格式限制)
- 单个资源文件: ≤ 500MB
- 配置文件: ≤ 1MB

### 12.3 内容安全

- HTML封面: 通过正式 `card.renderCover` / `coverFrame.render` 链路加载到受控 iframe
- JavaScript: 仅在封面 iframe 内部运行,不作为生态正式对外接口
- 外部资源: 不属于正式支持范围,兼容性不做保证

## 13. 性能优化

### 13.1 快速索引

优先读取轻量级文件:

```
1. 读取 metadata.yaml (< 1KB)
2. 读取 structure.yaml (< 10KB)
3. 按需读取 content/*.yaml
4. 按需加载资源文件
```

### 13.2 缓存策略

- 解析后的配置缓存在内存
- 封面图片缓存到本地
- 大文件不预加载

### 13.3 并发加载

- 多个资源文件并行加载
- 使用Worker线程解析配置
