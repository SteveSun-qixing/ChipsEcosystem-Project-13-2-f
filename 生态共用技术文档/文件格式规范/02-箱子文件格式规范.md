# 箱子文件格式规范

**版本**：1.0.0  
**更新时间**：2026-04-08  
**状态**：正式版

---

## 1. 文档定位

本规范定义 `.box` 文件的正式文件格式、目录结构、字段模型、验证规则和示例。

### 1.1 主发布边界

- 本文档是箱子文件格式的主发布位置；
- 箱子运行时取数接口以 `协议与契约/08-箱子布局运行时契约.md` 为准；
- 箱子业务职责与 Host 动作面以 `02-箱子业务.md` 为准。

### 1.2 首版冻结原则

- 不保留旧的三种箱子类型；
- 不保留内部 / 外部卡片二分字段；
- 不保留复制 / 移动 / 仅登记地址三种导入方式；
- 不做回退机制；
- 不做兼容性写法。

---

## 2. 概念与原则

### 2.1 箱子与卡片的关系

- 箱子（Box）与卡片（Card）是并行概念；
- 箱子不是卡片的一种类型；
- 箱子是卡片引用的组织与消费容器；
- 箱子不保存被引用卡片实体。

### 2.2 核心原则

1. **单文件正式形态**：正式产物是单个 `.box` 文件；
2. **URL 唯一定位模型**：箱子内部统一使用 URL 记录卡片引用；
3. **摘要优先**：箱子默认保存和输出卡片摘要，而不是卡片完整内容；
4. **Host 统一解析**：Host 负责 URL 解析、缓存、凭证和失败归一；
5. **布局资源独立**：箱子允许保存布局自身资源，但这些资源不等于卡片实体；
6. **按需加载**：运行时不要求一次性解析全部卡片。

---

## 3. 文件系统表现

### 3.1 正式文件形态

箱子的正式文件形态是单个 `.box` 文件。

示例：

```text
Travel-2026.box
```

`.box` 文件本质是 ZIP Store 包（零压缩 ZIP）。

### 3.2 不再采用的旧形态

以下形态都不是首版正式标准：

1. `文件夹 + 同名 .box`；
2. `.box` 内直接放置 `.card` 文件；
3. 用本地相对路径区分“内部卡片 / 外部卡片”；
4. 用箱子类型字段区分全填充 / 空壳 / 半空壳。

## 4. 包结构

### 4.1 标准目录结构

```text
Collection.box
├── .box/
│   ├── metadata.yaml
│   ├── structure.yaml
│   ├── content.yaml
│   ├── cover.html
│   └── boxcover/          # 可选，箱子封面附属资源
└── assets/                  # 可选，箱子自身资源
    ├── placeholders/
    ├── previews/
    └── layouts/
```

### 4.2 必需文件

箱子文件必须包含：

1. `.box/metadata.yaml`
2. `.box/structure.yaml`
3. `.box/content.yaml`
4. `.box/cover.html`

### 4.3 可选目录

`.box/boxcover/` 是可选目录，用于保存箱子封面编辑器生成的附属资源。

`assets/` 是可选目录，用于保存箱子自身资源，例如：

1. 布局背景图；
2. 占位图；
3. 箱子维护的摘要封面快照；
4. 布局私有配置附属文件。

### 4.4 禁止内容

`.box` 文件中禁止把被引用卡片实体作为正式内容写入，例如：

1. `*.card`
2. 卡片内容目录副本
3. 旧模型下的内部卡片缓存包

## 5. URL 模型

### 5.1 正式字段

`structure.yaml` 中每个箱子条目都必须包含 `url` 字段。

### 5.2 URL 基线

首版正式基线支持以下 URL：

1. `file://`（可指向 `.card` 或 `.box`）
2. `http://`
3. `https://`
4. `webdav://`（或由 Host 正式映射的 WebDAV URL 形式）

Host 后续可扩展更多 resolver scheme，但文件格式层始终只记录 URL，不再引入第二套本地路径字段。

### 5.3 约束

1. URL 必须是绝对定位；
2. 本地文件必须写为 `file://` URL；
3. 箱子条目不得使用“是否在内部”字段辅助解析；
4. 认证信息不得写入 URL。

## 6. `metadata.yaml`

### 6.1 用途

`metadata.yaml` 记录箱子本身的元信息。

### 6.2 必需字段

| 字段 | 类型 | 说明 |
|---|---|---|
| `chip_standards_version` | string | 薯片标准版本号 |
| `box_id` | string | 箱子唯一 ID，10 位 Base62 |
| `name` | string | 箱子名称 |
| `created_at` | string | 创建时间，ISO 8601 UTC |
| `modified_at` | string | 修改时间，ISO 8601 UTC |
| `active_layout_type` | string | 当前生效布局类型标识 |

### 6.3 可选字段

| 字段 | 类型 | 说明 |
|---|---|---|
| `description` | string | 箱子描述 |
| `tags` | array | 箱子标签 |
| `cover_ratio` | string | 箱子封面比例，正式封面由 `.box/cover.html` 渲染时使用 |
| `cover_asset` | string | 箱子封面资源路径，指向 `assets/` |
| `owner` | string | 逻辑所有者标识 |
| `source` | object | 来源信息 |

### 6.4 示例

```yaml
chip_standards_version: "1.0.0"
box_id: "b1C2d3E4f5"
name: "2026 旅行箱"
created_at: "2026-03-23T09:30:00.000Z"
modified_at: "2026-03-23T11:20:00.000Z"
active_layout_type: "chips.layout.grid"
cover_ratio: "3:4"
description: "用于浏览 2026 年旅行卡片集合"
tags:
  - "旅行"
  - ["年份", "2026"]
cover_asset: "assets/placeholders/box-cover.webp"
```

## 7. `structure.yaml`

### 7.1 用途

`structure.yaml` 记录箱子条目清单。  
这里记录的是**引用视图**，不是卡片实体。

### 7.2 标准结构

```yaml
entries:
  - entry_id: "e9K2m1P4q7"
    url: "file:///Users/name/Cards/day-01.card"
    enabled: true
    snapshot:
      document_id: "c7H1k2L9m3"
      title: "第一天"
      summary: "抵达后的路线记录"
      tags:
        - "旅行"
        - "东京"
      cover:
        mode: "asset"
        asset_path: "assets/previews/e9K2m1P4q7-cover.webp"
        mime_type: "image/webp"
        width: 640
        height: 360
      last_known_modified_at: "2026-03-22T08:00:00.000Z"
      content_type: "chips/card"
    layout_hints:
      sort_key: "2026-03-22"
      aspect_ratio: 1.7778
      group: "东京"
  - entry_id: "f4H8n2Q7r1"
    url: "file:///Users/name/Boxes/SubCollection.box"
    enabled: true
    snapshot:
      document_id: "b7X2m1K9p4"
      title: "东京分箱"
      summary: "箱子嵌套时正式显示该箱子的封面"
      cover:
        mode: "runtime"
      content_type: "chips/box"
```

### 7.3 必需字段

| 字段 | 类型 | 说明 |
|---|---|---|
| `entry_id` | string | 箱子条目唯一 ID，10 位 Base62 |
| `url` | string | 文档 URL，可指向卡片或箱子 |
| `enabled` | boolean | 是否参与布局展示 |

### 7.4 `snapshot` 字段

`snapshot` 是条目的摘要快照，推荐包含：

| 字段 | 类型 | 说明 |
|---|---|---|
| `document_id` | string | 上次已知文档 ID，可选，卡片或箱子都使用统一字段 |
| `title` | string | 标题 |
| `summary` | string | 摘要文本 |
| `tags` | array | 标签快照 |
| `cover` | object | 封面摘要 |
| `last_known_modified_at` | string | 上次已知修改时间 |
| `content_type` | string | 内容类型提示，当前正式值包括 `chips/card` 与 `chips/box` |

说明：

- 箱子支持层层嵌套；
- 当条目 `content_type = chips/box` 时，正式浏览态应显示被引用箱子的封面，而不是把它直接展开成另一页布局。

### 7.5 `cover` 字段

| 字段 | 类型 | 说明 |
|---|---|---|
| `mode` | enum | `asset` / `runtime` / `none` |
| `asset_path` | string | 当 `mode=asset` 时，指向 `assets/` |
| `mime_type` | string | 封面 MIME |
| `width` | number | 封面宽 |
| `height` | number | 封面高 |

说明：

- `asset` 表示箱子内已有封面快照；
- `runtime` 表示运行时由 Host 动态获取；
- `none` 表示无封面。
- 正式消费时，布局插件默认先使用 `snapshot` 作为排序和占位信息，实际显示优先通过 `box.renderEntryCover` 获取正式封面视图；
- 当条目引用的是箱子文件时，`box.renderEntryCover` 必须进一步复用目标箱子的 `.box/cover.html` 正式封面入口；
- 点击封面后的正式打开动作必须通过 `box.openEntry`。

### 7.6 `layout_hints` 字段

`layout_hints` 是布局可直接消费的索引信息，用于减少首屏额外取数。

推荐字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `sort_key` | string \| number | 排序键 |
| `aspect_ratio` | number | 摘要展示宽高比 |
| `group` | string | 分组键 |
| `priority` | number | 优先级 |

### 7.7 约束

1. `entry_id` 在同一箱子内必须唯一；
2. `url` 不得为空；
3. 不允许使用 `internal` / `external` / `location` 等旧字段；
4. 不允许使用相对路径表达卡片位置；
5. 快照字段可缺省，但 `entry_id`、`url`、`enabled` 必须存在。

## 8. `content.yaml`

### 8.1 用途

`content.yaml` 记录布局配置。

### 8.2 标准结构

```yaml
active_layout_type: "chips.layout.grid"
layout_configs:
  chips.layout.grid:
    schema_version: "1.0.0"
    props:
      column_count: 4
      gap: 16
      cover_ratio: "16:9"
      show_tags: true
      information_density: "comfortable"
    asset_refs:
      - "assets/layouts/grid/background.webp"
  chips.layout.board:
    schema_version: "1.0.0"
    props:
      columns:
        - "未开始"
        - "进行中"
        - "已完成"
```

### 8.3 顶层字段

| 字段 | 类型 | 说明 |
|---|---|---|
| `active_layout_type` | string | 当前生效布局类型 |
| `layout_configs` | object | 各布局类型的配置映射 |

### 8.4 布局配置字段

每个布局配置项建议包含：

| 字段 | 类型 | 说明 |
|---|---|---|
| `schema_version` | string | 当前布局配置 schema 版本 |
| `props` | object | 布局插件自定义参数 |
| `asset_refs` | array | 布局引用的箱子资源路径 |

### 8.5 原则

1. `props` 只描述布局配置，不保存布局代码；
2. 布局自定义参数允许自由扩展；
3. 布局资源必须引用箱子内 `assets/` 内容；
4. 不允许在 `content.yaml` 中写入凭证；
5. 不允许在 `content.yaml` 中保存被引用卡片实体数据。

---

## 9. `assets/` 资源规范

### 9.1 用途

`assets/` 存放箱子自身资源。

### 9.2 推荐子目录

| 目录 | 用途 |
|---|---|
| `assets/placeholders/` | 默认占位图与空态资源 |
| `assets/previews/` | 箱子维护的摘要封面快照 |
| `assets/layouts/` | 各布局使用的资源 |

### 9.3 资源引用规则

1. 资源路径使用包内相对路径；
2. 路径统一以 `assets/` 开头；
3. 不允许把卡片实体文件写入 `assets/`；
4. Host 在运行时通过箱子资源接口解析这些资源。

---

## 10. Host 消费原则

`.box` 文件格式必须为 Host 的按需消费模型服务。

### 10.1 首屏原则

Host 打开箱子后，优先使用：

1. `metadata.yaml`
2. `structure.yaml` 的摘要字段
3. `content.yaml`
4. `assets/` 中的布局资源

完成首屏渲染。

### 10.2 按需解析原则

Host 不要求一次性解析全部卡片，只在布局插件申请时动态获取：

1. 详细元数据；
2. 封面资源；
3. 预览资源；
4. 卡片入口资源；
5. 自定义运行时字段。

## 11. 校验规则

### 11.1 文件级校验

- `.box` 必须使用 ZIP Store；
- `.box/metadata.yaml` 必须存在；
- `.box/structure.yaml` 必须存在；
- `.box/content.yaml` 必须存在。

### 11.2 字段级校验

- `box_id` 必须为 10 位 Base62；
- `entry_id` 必须为 10 位 Base62；
- `url` 必须为绝对 URL；
- `active_layout_type` 必须存在；
- 时间字段必须为 ISO 8601 UTC。

### 11.3 架构级校验

- 不允许旧的 `box_type`；
- 不允许旧的 `internal` / `external`；
- 不允许被引用卡片实体进入包内；
- 不允许认证信息写入箱子文件。
