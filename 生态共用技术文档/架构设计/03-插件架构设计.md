# 插件架构设计

> **版本**：vNext 已落地口径  
> **层级**：L4 Plugin Runtime

## 架构归属声明（2026-04-08）

- 插件运行时由 Host 统一托管。
- 插件访问系统能力只能走 Host 正式链路。
- SDK 只提供开发工具与类型封装，不承载插件运行时主体。

## 1. 五种插件类型

| 类型 | 英文名 | 当前正式定位 |
|---|---|---|
| 应用插件 | `app` | 跨平台界面插件，由 Host 以 `surface` 语义启动 |
| 卡片插件 | `card` | 基础卡片渲染 / 编辑能力 |
| 布局插件 | `layout` | 箱子布局页面插件 |
| 模块插件 | `module` | Host 托管的无界面能力模块 |
| 主题插件 | `theme` | 主题与视觉包 |

## 2. 应用插件不再等同于“桌面独立窗口”

应用插件当前正式定义为“由 Host 托管、可映射到不同宿主界面容器的跨平台界面插件”。

当前对应关系：

- Desktop：映射为 `window`
- Web：预留映射为 `route / tab / modal`
- Mobile：预留映射为 `fullscreen / sheet`
- Headless：默认不承载交互式 app surface

因此：

- `app` 插件的公共语义是 `surface`
- `window` 只是 Desktop 的当前实现映射
- `plugin.launch` 是兼容入口；新的跨平台主链路是 `surface.open(target=plugin)`

## 3. 插件运行目标矩阵

所有插件都应声明 `runtime.targets`。

应用插件还必须声明：

- `ui.surface`

这使得 Host 可以在统一运行时中判断：

- 当前宿主是否支持该插件
- 插件希望以哪种 surface 形态运行

- 布局插件是箱子的前端页面，不是卡片文件定位器；
- 首屏默认以封面、名称、标签、摘要等摘要态信息渲染；
- 当需要更多信息时，再通过 Host 的 `box.*` 会话动作按需申请；
- Host `box-service` 统一负责查询已安装布局插件、加载 `layoutDefinition`、生成布局描述符、归一/校验布局配置，以及创建查看/编辑文档；
- 上层应用只能通过 Host `box.*` 动作与 SDK `client.box.* / client.document.window.*` 正式入口消费布局能力，不得自行扫描插件目录或直接 import 布局构建产物；
- 布局插件既可以消费插件包自身静态资源，也可以消费箱子文件中的 `assets/` 自有资源；
- 布局插件不得直接读取文件系统、直接访问网络或直接持有凭证。

## 4. 插件会话

Host 为插件维护正式会话，当前会话核心字段包括：

```ts
interface PluginSession {
  sessionId: string;
  pluginId: string;
  pluginType: "app" | "card" | "layout" | "module" | "theme";
  permissions: string[];
  locale: string;
  themeId: string;
  launchParams: Record<string, unknown>;
  sessionNonce: string;
  status: "handshaking" | "running" | "stopped";
}
```

应用插件通过 `surface.open(target=plugin)` 或 `plugin.launch` 启动时，Host 会：

1. 创建会话
2. 完成握手
3. 创建 surface
4. 向页面注入 launch context 与 Bridge

## 5. 各插件类型的正式边界

### 5.1 `app`

- 由 Host 管理生命周期与容器
- 不直接创建原生窗口
- 启动参数必须通过 launch context 获取

### 5.2 `card`

- 在 Host 通用渲染链路中运行在受控 iframe
- 需要输出可复用挂载函数

### 5.3 `layout`

- 消费 Host 提供的箱子会话和资源接口
- 不得直接访问底层文件系统或私有凭证

### 5.4 `module`

- 通过 capability + method 暴露能力
- 由 Host 统一做 provider 发现、校验、调度、审计

### 5.5 `theme`

- 负责 token、CSS、组件接口点与视觉资源

## 6. 隔离与权限

- 插件只能通过 Host 正式动作访问系统能力
- 模块之间相互调用必须继续走 Host 模块服务
- 应用插件不得绕过 Host 私自操作原生窗口、协议关联或系统快捷方式

## 7. 当前落地状态

当前已经落地的关键插件运行时能力：

- 应用插件运行目标矩阵
- `ui.surface`
- `capabilityFallbacks`
- `surface.open(target=plugin)` 正式插件启动链路
- Headless 宿主对无界面插件体系的复用

当前未在本轮交付的部分：

- Web Host Shell
- Mobile Host Shell

但相关公共契约已经冻结，后续实现不需要重新推翻插件模型。
