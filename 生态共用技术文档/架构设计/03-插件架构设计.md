# 插件架构设计

> **版本**：vNext 架构设计冻结稿  
> **层级**：L4 Plugin Runtime（插件运行时）  
> **适用阶段**：阶段18及后续重构实施阶段


## 架构归属声明（2026-03-06）

- Host 主责 L1-L9 运行时链路（含 Runtime Client、Declarative UI、Unified Rendering）并随 Host 发布。
- 组件库主责 L10 Component Headless，只负责状态机、结构与 a11y。
- SDK 仅作为开发者工具包提供类型、封装、脚手架与测试辅助，不承载运行时主实现。
- 用户设备仅安装 Host 即可运行；SDK 仅在开发与集成阶段按需使用。

## 插件系统概述

插件系统是薯片生态实现功能扩展的核心机制。整个生态系统采用"一切皆插件"的设计理念，所有功能模块都以插件形式存在。这种设计确保系统具有高度的灵活性和可扩展性，用户可以根据需求选择安装不同的插件，开发者可以独立开发和发布插件。

**设计目标**：
- 功能热插拔：无需重启应用即可添加、移除或更新插件
- 每个插件是独立的软件包，包含完整的功能实现和必要的元数据
- 插件之间通过标准化的接口进行通信，彼此隔离，互不干扰

## L4层级定位

| 层级 | 名称 | 核心职责 | 只允许依赖 | 禁止依赖 |
|---|---|---|---|---|
| L2 | Host Kernel | router/registry/event/lifecycle | PAL | 应用页面代码 |
| L3 | Host Services | file/resource/config/theme/i18n 等 | Kernel | 插件 UI |
| **L4** | **Plugin Runtime** | **插件加载、隔离、会话、权限** | **Kernel+Services** | **业务域实现细节** |

**依赖约束**：
- L4只允许依赖Kernel(L2)和Services(L3)
- L4禁止依赖业务域实现细节（L12）

## 插件类型系统

**五种插件类型**：

| 类型 | 英文名 | 说明 |
|------|--------|------|
| 应用插件 | `app` | 独立窗口应用插件（编辑引擎、查看器、设置面板、资源管理器等） |
| 卡片插件 | `card` | 基础卡片渲染/编辑插件（视频卡片、图片卡片、文本卡片等） |
| 布局插件 | `layout` | 布局插件（网格布局、瀑布流布局、看板布局等） |
| 模块插件 | `module` | 通用功能模块插件（播放器、渲染器等可复用功能） |
| 主题插件 | `theme` | 视觉主题插件（CSS样式主题包） |

### 应用插件（app）

应用插件是运行在独立窗口中的完整应用程序。每个应用插件可以创建自己的窗口，拥有完整的生命周期，可以与其他应用插件或系统进行交互。

典型例子：编辑器、查看器、资源管理器、设置面板等。

### 卡片插件（card）

卡片插件是内容展示的最小单元，每个卡片插件对应一种内容类型的渲染和编辑逻辑。卡片插件运行在隔离的iframe环境中，确保安全性。

典型例子：视频卡片、图片卡片、文本卡片、富文本卡片等。

### 布局插件（layout）

布局插件定义了卡片在容器中的排列方式。布局插件负责计算卡片位置和处理拖拽交互，支持响应式设计以适应不同屏幕尺寸。

典型例子：网格布局、瀑布流布局、看板布局、列表布局等。

### 模块插件（module）

模块插件是共享代码库的载体，提供可复用的功能模块供其他插件调用。模块插件不创建独立窗口，通过导出函数和类供其他插件使用。

典型例子：视频播放器、音频播放器、图片查看器、3D渲染器、富文本编辑器、Markdown渲染器、代码高亮器等。

### 主题插件（theme）

主题插件定义了应用的视觉外观，包括颜色、字体、间距、动画等设计token。主题插件采用CSS变量技术，可以动态切换无需重启应用。

## 插件生命周期

插件的生命周期包括五个阶段：

```
install -> enable -> running -> disable -> uninstall
```

### 安装阶段（install）

- 将插件文件解压到指定目录
- 解析manifest配置
- 验证插件完整性
- 检查依赖是否满足

### 激活阶段（enable）

- 加载插件代码
- 初始化插件状态
- 向系统注册插件提供的服务
- 分配sessionNonce

### 运行阶段（running）

- 插件正常工作
- 响应用户操作
- 处理业务逻辑
- 持续占用系统资源

### 停用阶段（disable）

- 保存当前状态
- 释放占用的资源
- 注销已注册的服务

### 卸载阶段（uninstall）

- 移除插件所有文件
- 清理配置文件和缓存数据
- 检查是否有依赖方

### 插件会话

每个插件会话包含：
```typescript
interface PluginSession {
  pluginId: string;          // 插件ID
  pluginType: 'app' | 'card' | 'layout' | 'module' | 'theme';  // 插件类型
  permissions: string[];     // 权限列表
  locale: string;           // 语言环境
  themeId: string;          // 主题ID
  launchParams: Record<string, unknown>;  // 启动参数
  sessionNonce: string;     // 会话唯一标识
}
```

**初始化握手**：
- 插件初始化必须走`plugin-init`握手
- 握手失败禁止进入running状态

## 插件隔离机制

### 进程级隔离

每个插件运行在独立的上下文中，实现了进程级或线程级的隔离。这种隔离确保一个插件的错误不会影响其他插件的正常运行，提高了系统的整体稳定性。

**隔离策略**：
- App级隔离：独立BrowserWindow
- Card渲染隔离：iframe优先，支持受控模式

### 权限隔离

插件的权限需要用户在安装时授权。权限模型采用最小权限原则，插件默认只拥有基本权限，需要额外权限时必须明确申请。

调用时按`sender + pluginId`做实时鉴权。

### 文件系统隔离

插件的文件系统访问受到严格限制：
- 插件只能读写自己的私有目录
- 无法访问其他插件的文件空间
- 需要通过系统API访问公共资源

### 网络访问隔离

- 插件可以配置允许访问的域名白名单
- 未授权的域名无法访问

### 通信隔离

- 插件之间的通信通过系统消息总线进行
- 发送方无法直接调用接收方的方法
- 只能发送消息，由接收方决定如何响应

## 资源配额

### 配额类型

| 资源类型 | 配额名称 | 说明 |
|----------|----------|------|
| CPU | cpuQuota | CPU时间配额 |
| 内存 | memoryQuota | 内存使用配额 |
| 消息频率 | messageQuota | 消息发送频率配额 |

### 降级策略

当插件超出配额时触发降级：
- **限频**：降低消息处理频率
- **暂停后台任务**：暂停非关键任务
- **告警**：通知用户和管理员

## 安全流水线

完整的插件安全流水线包含7个阶段：

1. **Manifest校验**
   - 类型验证
   - 权限声明检查
   - 入口文件验证
   - 契约版本校验

2. **签名与来源校验**
   - 官方渠道插件：验证官方签名
   - 第三方插件：验证开发者签名

3. **启动握手（plugin-init）**
   - 分配sessionNonce
   - 建立安全会话

4. **权限快照加载**
   - 加载最小权限集合
   - 注入权限上下文

5. **沙箱注入**
   - 只暴露window.chips.*接口
   - 禁用危险原生API

6. **运行时审计**
   - 越权调用检测
   - 超频检测
   - 异常事件记录

7. **会话结束清理**
   - 取消订阅
   - 清除缓存
   - 释放临时资源

## 插件依赖管理

### 依赖声明

插件可以声明对其他插件或系统服务的依赖：

```typescript
interface PluginManifest {
  id: string;
  version: string;
  dependencies: {
    [pluginId: string]: {
      version: string;  // 版本范围，如 ^1.0.0
      optional?: boolean;  // 是否可选
    };
  };
}
```

### 依赖解析

- **强依赖**：系统在激活插件前检查所有依赖是否满足
- **软依赖**：允许在依赖不可用时降级运行
- **循环依赖检测**：系统会阻止循环依赖

### 版本管理

- 插件采用语义化版本号规范
- 系统支持同一插件的多个版本同时存在
- 自动更新策略可配置

## 插件契约

### Manifest固定字段

```yaml
# 最小必须字段
id: string           # 唯一标识
version: string      # 语义化版本
type: app | card | layout | module | theme  # 插件类型
main: string         # 入口文件

# 扩展字段
permissions: string[]     # 权限列表
dependencies: object      # 依赖声明
ui:
  layout:
    owner: page
    unit: cpx
    baseWidth: 1024
    contract: ./contracts/page-layout.contract.json
    minFunctionalSet: ./contracts/min-functional-set.json
```

### 验收标准

### 分层验收
- L4职责边界清晰（插件加载、隔离、会话、权限）
- L4只依赖Kernel(L2)和Services(L3)
- L4不依赖业务域实现细节

### 安全验收
- Manifest校验通过
- 签名验证通过
- 握手协议完成
- 权限隔离生效

### 隔离验收
- 插件之间进程隔离
- 文件系统访问受限
- 网络访问受限
