# 渲染架构设计

> **版本**：vNext 架构设计冻结稿  
> **层级**：L8 Declarative UI、L9 Unified Rendering、L10 Component Headless  
> **适用阶段**：阶段18及后续重构实施阶段

## 渲染架构概述

渲染架构是薯片生态中负责界面展示的核心系统。它决定了内容如何呈现在用户面前，以及用户如何与内容进行交互。渲染架构采用分层设计，将渲染逻辑划分为多个独立的层次，每个层次专注于特定职责。

渲染架构的核心设计理念是性能与用户体验的平衡。系统需要在保证渲染效率的同时，提供流畅的交互体验和一致的视觉表现。

## 架构归属声明（2026-03-06）

- L8 Declarative UI 与 L9 Unified Rendering 均属于 Host 内置运行时层，由 Host 主责实现。
- L10 Component Headless 由组件库主责，组件仅提供状态机、结构、a11y 语义，不承载运行时渲染引擎。
- SDK 只提供开发封装与调用入口，不承载渲染流水线或 Runtime Client 主实现。

## 层级定位

| 层级 | 名称 | 核心职责 | 只允许依赖 | 禁止依赖 |
|---|---|---|---|---|
| L7 | UI Hooks | 面向页面的能力调用接口 | Runtime Client | Transport 细节 |
| **L8** | **Declarative UI** | **`View/Stack/Grid/Form/List` 语义原语** | **UI Hooks + Component Layer** | **Host 服务细节** |
| **L9** | **Unified Rendering** | **Normalize/Validate/Resolve/Commit** | **Declarative UI + Theme Runtime** | **Host 路由逻辑** |
| **L10** | **Component Headless** | **状态机、结构、a11y** | **Theme Runtime(接口)** | **视觉硬编码** |
| L11 | Theme Runtime | token 解析、作用域覆盖、变量注入 | Theme Package | 业务逻辑 |

## L8 Declarative UI（声明式UI层）

### 核心原语

声明式UI层提供语义化的UI原语，定义界面的结构而非具体实现。

**五大核心原语**：

| 原语 | 说明 | 典型用途 |
|------|------|----------|
| `View` | 基础容器 | 页面布局、区域划分 |
| `Stack` | 堆叠布局 | 垂直/水平列表 |
| `Grid` | 网格布局 | 复杂二维布局 |
| `Form` | 表单容器 | 数据输入 |
| `List` | 列表容器 | 重复项渲染 |

### 声明树节点模型

```typescript
interface UINode {
  id: string;                    // 节点唯一标识
  type: 'View' | 'Stack' | 'Grid' | 'Form' | 'List' | string;  // 节点类型
  props?: Record<string, unknown>;   // 属性（语义化）
  state?: Record<string, unknown>;   // 状态
  bindings?: Record<string, string>; // 绑定关系
  events?: Record<string, string>;   // 事件声明（语义化）
  themeScope?: string;              // 主题作用域
  children?: UINode[];              // 子节点
}
```

### 约束规则

**核心约束**：
- 声明层**不承载**视觉值（颜色、阴影、半径），仅声明语义
- 声明层**不可直接**发起系统调用，必须通过UI Hooks
- 声明层**只描述**UI结构，不涉及具体渲染实现

**生态公开实现细则**：
- 统一参考：`生态共用技术文档/架构设计/17-L8声明式UI实现与接口细则.md`
- 所有仓库对外可见的 L8 能力口径，以该细则为准。

### 组合模式规范（Vercel组合模式）

采用Vercel组合模式规范，避免组件API膨胀：

**原则**：
- 复杂组件优先使用**Compound Components（复合组件）**
- **避免boolean props爆炸**，改为显式子组件与slot
- **Provider**成为唯一状态实现知情者

**示例**：
```tsx
// ❌ 避免：boolean props 爆炸
<Button primary secondary large small loading disabled />
<Modal show showHeader showFooter showCloseButton />

// ✅ 采用：显式子组件
<Button>
  <Button.Primary>主要操作</Button.Primary>
  <Button.Secondary>次要操作</Button.Secondary>
</Button>

<Modal>
  <Modal.Header>标题</Modal.Header>
  <Modal.Body>内容</Modal.Body>
  <Modal.Footer>底部</Modal.Footer>
  <Modal.CloseButton />
</Modal>
```

### 副作用分级

声明式UI中的副作用需要明确分级管理：

| 副作用类型 | 说明 | 触发时机 |
|------------|------|----------|
| `ui-effect` | 滚动、聚焦、过渡 | UI交互时 |
| `runtime-effect` | 调用Host Runtime Client（可由 SDK 提供开发封装） | 业务逻辑执行时 |
| `telemetry-effect` | 日志与埋点 | 关键操作时 |

**强制规则**：
- **禁止在渲染阶段直接触发`runtime-effect`**
- `runtime-effect`必须在用户交互回调或useEffect中触发
- `telemetry-effect`应该异步执行，不阻塞主线程

## L9 Unified Rendering（统一渲染层）

### 渲染流水线

统一渲染层负责将声明树转换为最终像素，完整流水线包含6个阶段：

```
Node Normalize → Contract Validate → Theme Resolve → Layout Compute → Render Commit → Effect Dispatch
```

#### 1. Node Normalize（节点规范化）
- 解析输入的声明树
- 合并默认属性
- 展开复合组件
- 验证基础结构

#### 2. Contract Validate（契约校验）
- 验证节点类型合法性
- 验证属性符合契约
- 验证子节点关系
- **非兼容能力必须在此阶段失败，不允许silent fallback**

#### 3. Theme Resolve（主题解析）
- 解析token引用
- 合并主题变量
- 生成组件样式映射
- 应用作用域链

#### 4. Layout Compute（布局计算）
- 计算节点尺寸
- 计算节点位置
- 应用布局算法
- 生成渲染树

#### 5. Render Commit（渲染提交）
- 生成DOM/CSS
- 提交到渲染目标
- 触发视觉更新

#### 6. Effect Dispatch（副作用分发）
- 执行ui-effect（滚动、聚焦）
- runtime-effect进入延迟队列（仅在事件阶段或提交后由运行时触发）
- 执行telemetry-effect（埋点）

### 目标适配器

统一渲染层需要支持多种渲染目标：

| 适配器 | 说明 | 场景 |
|--------|------|------|
| `App Root` | 应用根容器 | 主应用窗口 |
| `Card Iframe` | 卡片iframe渲染 | 卡片内容隔离渲染 |
| `Module Slot` | 模块插槽 | 模块嵌入渲染 |
| `Offscreen Render` | 离屏渲染 | 预览、导出场景 |

### 渲染一致性约束

- 同一声明树在不同目标适配器中**语义一致**
- 非兼容能力必须在Validate阶段**失败**
- 不允许silent fallback导致行为不一致

### 失败隔离

- **节点级Error Boundary**：单个节点渲染失败不影响兄弟节点
- **区域级Error Boundary**：某个区域失败不影响整体页面
- **降级渲染**：失败时渲染降级版本而非空白

### 调度与增量更新

**渲染队列**：
- 维护`render queue`，将大树更新拆分为可中断批次
- 使用`requestIdleCallback`或`requestAnimationFrame`调度

**增量渲染**：
- `List/Grid/Table`默认启用可见区域增量提交
- 只渲染viewport内的内容

**主题切换策略**：
- 样式层更新优先（CSS变量更新）
- 结构层更新延后（DOM更新）
- 避免阻塞交互

### L9 声明树扩展字段（公开口径）

除 L8 基础字段外，L9 接收以下扩展字段用于统一渲染：

```typescript
interface L9NodeExtensions {
  effects?: Array<{
    id?: string;
    kind: 'ui-effect' | 'runtime-effect' | 'telemetry-effect';
    name: string;
    payload?: unknown;
    trigger?: 'render' | 'event' | 'commit';
  }>;
  contract?: {
    requires?: string[]; // 目标适配器能力声明
  };
  errorBoundary?: {
    level: 'node' | 'region';
    fallback?: {
      type?: string;
      props?: Record<string, unknown>;
    };
  };
}
```

约束：

- `runtime-effect` 在 `trigger=render` 时必须失败。
- `contract.requires` 与目标适配器能力不匹配时必须在 Validate 阶段失败。
- `errorBoundary` 只做失败隔离，不可绕过契约校验。

### L9 统一渲染入口（card.render）公开契约

统一渲染能力通过 Host `card.render` 动作对外提供：

```typescript
// request
{
  cardFile: string;
  options?: {
    target?: 'app-root' | 'card-iframe' | 'module-slot' | 'offscreen-render';
    viewport?: { width?: number; height?: number; scrollTop?: number; scrollLeft?: number };
    verifyConsistency?: boolean;
  };
}

// response
{
  view: {
    title: string;
    body: string;
    contentFiles: string[];
    target: string;
    semanticHash: string;
    diagnostics?: RenderNodeDiagnostic[];
    consistency?: RenderConsistencyResult;
  };
}
```

### L9 诊断结构与推荐错误码

```typescript
interface RenderNodeDiagnostic {
  nodeId: string;
  stage:
    | 'node-normalize'
    | 'contract-validate'
    | 'theme-resolve'
    | 'layout-compute'
    | 'render-commit'
    | 'effect-dispatch';
  code: string;
  message: string;
  details?: unknown;
}
```

推荐错误码前缀：

- `RENDER_CONTRACT_*`：契约校验失败
- `RENDER_THEME_*`：主题解析失败
- `RENDER_LAYOUT_*`：布局计算失败
- `RENDER_PIPELINE_*`：流水线阶段失败

### L9 性能门禁基线（公开）

渲染层默认门禁阈值：

- `render-commit p95 <= 32ms`
- `List/Grid/Table` 默认启用可见区增量提交
- 大树更新必须支持可中断批次（render queue）

建议在 CI 中执行 L9 性能门禁并保留基线报告，持续观察回归漂移。

## L10 Component Headless（无头组件层）

### 无头组件原则

无头组件只负责逻辑和结构，不包含任何视觉样式。

**核心原则**：
- **无头约束**：组件只负责结构和行为，视觉完全由主题注入
- **零视觉硬编码**：不允许组件内写死颜色、阴影、半径、动效值
- **零文案硬编码**：不允许组件内写死用户可见文案

### 组件状态机

所有交互组件统一使用状态机定义：

```typescript
type ComponentState = 
  | 'idle'      // 空闲
  | 'hover'     // 悬停
  | 'focus'     // 聚焦
  | 'active'    // 激活
  | 'disabled'  // 禁用
  | 'loading'   // 加载中
  | 'error';    // 错误
```

**状态投影**：
- 状态变化必须投影到`data-state`属性
- 示例：`<button data-state="idle" data-state="hover" />`

### 契约挂点

组件必须暴露以下契约挂点供主题注入：

```typescript
// 主题注入点
interface ComponentContract {
  // 作用域标识
  'data-scope': string;  // 如 'button', 'input'
  
  // 部件标识
  'data-part': string;   // 如 'icon', 'text', 'loader'
  
  // 状态标识
  'data-state': string;   // 如 'idle', 'loading'
  
  // 无障碍
  'aria-*': string;       // aria-label, aria-describedby等
}
```

## cpx 尺寸体系（页面级布局）

### 逻辑尺寸单位

生态统一逻辑尺寸单位：`cpx`

### 基准宽度

页面基准宽度：`1024cpx`

### 换算公式

```typescript
// cpx 转 px
const toPx = (cpx: number, containerWidthPx: number): number => 
  cpx * (containerWidthPx / 1024);

// px 转 cpx
const toCpx = (px: number, containerWidthPx: number): number =>
  px * (1024 / containerWidthPx);
```

### 换算精度规范

- 换算结果保留**3位小数**
- 提交到DOM/CSS前按策略裁剪：
  - **字体**：四舍五入到`0.5px`
  - **边框**：最小`1px`可见阈值
  - **位移**：允许亚像素渲染

### SDK 工具函数

```typescript
// SDK 提供统一函数，禁止应用侧重复实现
import { toPx, toCpx, getScale, getBreakpoint } from '@chips/layout-math';

// 获取当前缩放比例
const scale = getScale(containerWidthPx);

// 获取当前断点
const breakpoint = getBreakpoint(containerWidthPx);
```

### 页面自治原则

- **每页面定义自己的断点**、信息重排、交互降级策略
- **不做全局横竖模板**
- 平台层不提供全局横竖模板

### 布局契约

```yaml
ui:
  layout:
    owner: page              # 页面自治
    unit: cpx              # 逻辑单位
    baseWidth: 1024        # 基准宽度
    contract: ./contracts/page-layout.contract.json
    minFunctionalSet: ./contracts/min-functional-set.json
```

### 页面最小功能集

- 每页面必须声明关键任务路径在所有支持尺寸下可达
- 布局切换不能丢状态（表单输入、滚动位置、临时会话态）

## 渲染优化策略

### 性能指标

| 指标 | 阈值 | 说明 |
|------|------|------|
| TTI | <= 2000ms | 首屏可交互时间 |
| 渲染提交 | p95 <= 32ms | 单次渲染提交耗时 |
| 长任务占比 | <= 5% | JavaScript长任务占比 |
| 主题切换 | <= 150ms | 主题切换可见完成 |
| 内存峰值 | <= 基线1.5倍 | 峰值内存限制 |

### 优化手段

- 独立请求并行化（避免waterfall）
- 重组件按可见性分片渲染
- 大列表/树/表格强制虚拟化
- 主题切换使用异步注入与分片更新
