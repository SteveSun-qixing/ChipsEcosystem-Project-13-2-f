# 主题系统设计

> **版本**：vNext 架构设计冻结稿  
> **层级**：L11 Theme Runtime  
> **适用阶段**：阶段18及后续重构实施阶段


## 架构归属声明（2026-03-06）

- Host 主责 L1-L9 运行时链路（含 Runtime Client、Declarative UI、Unified Rendering）并随 Host 发布。
- 组件库主责 L10 Component Headless，只负责状态机、结构与 a11y。
- SDK 仅作为开发者工具包提供类型、封装、脚手架与测试辅助，不承载运行时主实现。
- 用户设备仅安装 Host 即可运行；SDK 仅在开发与集成阶段按需使用。

## 主题系统概述

主题系统是薯片生态实现视觉样式动态切换的核心机制。采用无头组件原则，组件只负责逻辑和结构，视觉完全由主题包注入。

## 运行时主题来源与工作区边界（2026-03-10 同步）

主题系统始终绑定到“当前工作区”，不存在跨工作区共享的当前主题。

- `chips` 默认操作用户工作区 `~/.chips-host`；
- `chipsdev` 默认操作开发工作区 `.chips-host-dev`；
- 主题服务只从当前工作区中“已启用”的主题插件构建可用主题集合；
- 安装但未启用的主题插件不会参与 `theme.list/getCurrent/resolve/apply`；
- 主题插件的安装、启用、禁用、卸载会立即触发主题运行时刷新。

## L11层级定位

| 层级 | 名称 | 核心职责 | 只允许依赖 | 禁止依赖 |
|---|---|---|---|---|
| L10 | Component Headless | 状态机、结构、a11y | Theme Runtime(接口) | 视觉硬编码 |
| **L11** | **Theme Runtime** | **token 解析、作用域覆盖、变量注入** | **Theme Package** | **业务逻辑** |
| L12 | App/Page Domain | 页面状态、业务流程、编排 | L7-L11 | L1-L5 细节 |

## token五层体系

主题系统采用五层token结构，从原始值到最终应用逐层解析：

### 1. ref（原始值层）
设计系统的基准值，是最底层的原始定义：
```json
{
  "color": {
    "blue": "#007AFF",
    "red": "#FF3B30",
    "green": "#34C759"
  },
  "spacing": {
    "xs": "4px",
    "sm": "8px",
    "md": "16px",
    "lg": "24px"
  },
  "font": {
    "size": {
      "xs": "12px",
      "sm": "14px",
      "md": "16px",
      "lg": "20px"
    },
    "weight": {
      "regular": 400,
      "medium": 500,
      "bold": 700
    }
  }
}
```

### 2. sys（语义值层）
基于使用场景的语义化定义：
```json
{
  "button": {
    "primary": "{color.blue}",
    "secondary": "{color.gray.500}",
    "danger": "{color.red}"
  },
  "text": {
    "body": "{font.size.md}",
    "heading": "{font.size.lg}",
    "caption": "{font.size.xs}"
  },
  "surface": {
    "background": "{color.white}",
    "elevated": "{color.gray.50}"
  }
}
```

### 3. comp（组件值层）
具体组件的样式值：
```json
{
  "Button": {
    "background": "{button.primary}",
    "color": "{color.white}",
    "borderRadius": "8px",
    "padding": "{spacing.sm} {spacing.md}"
  },
  "Input": {
    "border": "1px solid {color.gray.300}",
    "focusBorder": "{color.blue}",
    "borderRadius": "6px"
  },
  "Card": {
    "background": "{surface.background}",
    "shadow": "0 2px 8px rgba(0,0,0,0.1)"
  }
}
```

### 4. motion（动效值层）
动画和过渡效果：
```json
{
  "transition": {
    "fast": "150ms ease",
    "normal": "200ms ease",
    "slow": "300ms ease"
  },
  "animation": {
    "fadeIn": "fadeIn 200ms ease",
    "slideUp": "slideUp 300ms ease"
  }
}
```

### 5. layout（布局密度值层）
间距和布局相关：
```json
{
  "gap": {
    "compact": "4px",
    "normal": "8px",
    "relaxed": "16px"
  },
  "density": {
    "compact": "28px",
    "comfortable": "32px",
    "spacious": "40px"
  }
}
```

## 作用域链

主题变量的作用域遵循层级覆盖原则：

```
global -> app -> box -> composite-card -> base-card -> component
```

说明：主题权重从左到右递增，最小组件（`component`）权重最高。
解析时从高权重层开始查找，当前层缺失时逐层向上回退：
`component -> base-card -> composite-card -> box -> app -> global`。

### 作用域优先级
1. **component**（最高）：组件级别覆盖
2. **base-card**：基础卡片级覆盖
3. **composite-card**：复合卡片级覆盖
4. **box**：箱子级覆盖
5. **app**：应用级别覆盖
6. **global**（最低）：全局默认

### 覆盖规则
1. 作用域优先级（近者优先）
2. 选择器特异性
3. 声明顺序

## 公开接口（固定6个）

theme服务必须实现以下6个固定接口：

```typescript
interface ThemeService {
  /**
   * 获取主题列表
   * @param publisher - 可选的发行者过滤
   * @returns 主题元信息列表
   */
  list(publisher?: string): Promise<ThemeMeta[]>;

  /**
   * 应用主题
   * @param id - 主题ID
   */
  apply(id: string): Promise<void>;

  /**
   * 获取当前主题
   * @param appId - 可选的应用ID
   * @param pluginId - 可选的插件ID
   * @returns 当前主题状态
   */
  getCurrent(appId?: string, pluginId?: string): Promise<ThemeState>;

  /**
   * 获取所有CSS
   * @returns 完整的主题CSS字符串与主题标识
   */
  getAllCss(): Promise<{ css: string; themeId: string }>;

  /**
   * 解析主题链
   * @param chain - 作用域链
   * @returns 解析后的token
   */
  resolve(chain: string[]): Promise<ResolvedTokens>;

  /**
   * 获取主题契约
   * @param component - 可选的组件名
   * @returns 主题契约定义
   */
  contract.get(component?: string): Promise<ThemeContract>;
}

interface ResolvedTokens {
  /**
   * 解析后的主题链信息（按输入顺序排列）
   */
  resolved: Array<{
    id: string;
    displayName: string;
    order: number;
  }>;
  /**
   * 按作用域链与覆盖规则合并后的 token 视图
   */
  tokens: Record<string, unknown>;
}
```

## 主题解析算法（运行时）

完整的运行时解析流程：

```typescript
async function resolveTheme(chain: string[]): Promise<ResolvedTheme> {
  // 1. 加载作用域链主题快照
  const snapshots = await loadScopeChain(chain);
  
  // 2. 按token层级合并（ref -> sys -> comp/motion/layout）
  const merged = mergeByLevel(snapshots);
  
  // 3. 执行冲突裁决
  const resolved = resolveConflicts(merged);
  
  // 4. 生成变量表与组件映射表
  const variables = generateVariableMap(resolved);
  const componentMap = generateComponentMap(resolved);
  
  // 5. 注入作用域样式并触发 theme.changed 事件
  await injectStyles(variables, componentMap);
  emit('theme.changed', { themeId: getCurrentThemeId() });
  
  return { variables, componentMap };
}
```

## 主题契约门禁

每个主题包必须通过以下校验：

### 1. token完整性校验
- 检查所有必需token是否存在
- 检查是否有冗余token
- 检查token值格式

### 2. 组件接口点校验
- 验证 `data-scope` 声明
- 验证 `data-part` 声明
- 验证组件是否实现声明的接口点

### 3. 动效参数安全校验
- 最大时长限制（如3000ms）
- 曲线白名单验证
- 禁止危险动画值

### 4. 颜色对比度校验
- 文本可读性验证
- WCAG标准检查
- 警告不达标颜色组合

## 主题切换流程

```typescript
// 主题切换流程
async function switchTheme(themeId: string) {
  // 1. 验证主题存在
  const theme = await themeService.get(themeId);
  if (!theme) throw new Error('THEME_NOT_FOUND');
  
  // 2. 预加载主题资源
  await preloadTheme(theme);
  
  // 3. 解析主题token
  const resolved = await themeService.resolve(['global', 'app', themeId]);
  
  // 4. 生成CSS变量
  const css = generateCSSVariables(resolved);
  
  // 5. 异步注入样式（不阻塞交互）
  await injectStylesAsync(css);
  
  // 6. 更新当前主题状态
  await themeService.apply(themeId);
  
  // 7. 触发主题变更事件
  emit('theme.changed', { themeId, timestamp: Date.now() });
}
```

## 主题注入与显示链路（2026-03-10 同步）

主题切换的正式显示链路如下：

1. Host 主题服务产出当前主题的 `theme.getCurrent/getAllCss/resolve` 结果；
2. preload 将主题 CSS、变量和 `data-chips-theme-id`、`data-chips-theme-version` 注入应用文档；
3. 应用层主题提供器必须以 Host 当前主题作为初始状态，不允许写死默认主题；
4. 组件库主题运行时生成缓存键，供依赖主题快照的组件判断是否需要重建；
5. 复合卡片窗口在缓存键变化后重新渲染 iframe 内容；
6. 原生窗口背景在调用方未显式指定时，由 Host 从主题 token 自动推导；
7. 卡片渲染服务保留主题包自己的 `color-scheme` 和色彩模式声明，不得以运行时样式覆盖。
