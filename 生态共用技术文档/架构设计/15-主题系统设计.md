# 主题系统设计

> **版本**：vNext 架构设计冻结稿  
> **层级**：L11 Theme Runtime  
> **适用阶段**：阶段18及后续重构实施阶段

## 主题系统概述

主题系统是薯片生态实现视觉样式动态切换的核心机制。采用无头组件原则，组件只负责逻辑和结构，视觉完全由主题包注入。

## L11层级定位

| 层级 | 名称 | 核心职责 | 只允许依赖 | 禁止依赖 |
|---|---|---|---|---|
| L10 | Component Headless | 状态机、结构、a11y | Theme Runtime(接口) | 视觉硬编码 |
| **L11** | **Theme Runtime** | **token 解析、作用域覆盖、变量注入** | **Theme Package** | **业务逻辑** |
| L12 | App/Page Domain | 页面状态、业务流程、编排 | L7-L11 | L1-L5 细节 |

## token五层体系

主题系统采用五层token结构，从原始值到最终应用逐层解析：

### 1. ref（原始值层）
设计系统的基准值，是最底层的原始定义：
```json
{
  "color": {
    "blue": "#007AFF",
    "red": "#FF3B30",
    "green": "#34C759"
  },
  "spacing": {
    "xs": "4px",
    "sm": "8px",
    "md": "16px",
    "lg": "24px"
  },
  "font": {
    "size": {
      "xs": "12px",
      "sm": "14px",
      "md": "16px",
      "lg": "20px"
    },
    "weight": {
      "regular": 400,
      "medium": 500,
      "bold": 700
    }
  }
}
```

### 2. sys（语义值层）
基于使用场景的语义化定义：
```json
{
  "button": {
    "primary": "{color.blue}",
    "secondary": "{color.gray.500}",
    "danger": "{color.red}"
  },
  "text": {
    "body": "{font.size.md}",
    "heading": "{font.size.lg}",
    "caption": "{font.size.xs}"
  },
  "surface": {
    "background": "{color.white}",
    "elevated": "{color.gray.50}"
  }
}
```

### 3. comp（组件值层）
具体组件的样式值：
```json
{
  "Button": {
    "background": "{button.primary}",
    "color": "{color.white}",
    "borderRadius": "8px",
    "padding": "{spacing.sm} {spacing.md}"
  },
  "Input": {
    "border": "1px solid {color.gray.300}",
    "focusBorder": "{color.blue}",
    "borderRadius": "6px"
  },
  "Card": {
    "background": "{surface.background}",
    "shadow": "0 2px 8px rgba(0,0,0,0.1)"
  }
}
```

### 4. motion（动效值层）
动画和过渡效果：
```json
{
  "transition": {
    "fast": "150ms ease",
    "normal": "200ms ease",
    "slow": "300ms ease"
  },
  "animation": {
    "fadeIn": "fadeIn 200ms ease",
    "slideUp": "slideUp 300ms ease"
  }
}
```

### 5. layout（布局密度值层）
间距和布局相关：
```json
{
  "gap": {
    "compact": "4px",
    "normal": "8px",
    "relaxed": "16px"
  },
  "density": {
    "compact": "28px",
    "comfortable": "32px",
    "spacious": "40px"
  }
}
```

## 作用域链

主题变量的作用域遵循层级覆盖原则：

```
global -> app -> workspace -> card -> component
```

### 作用域优先级
1. **component**（最高）：组件级别覆盖
2. **card**：卡片级别覆盖
3. **workspace**：工作空间级别覆盖
4. **app**：应用级别覆盖
5. **global**（最低）：全局默认

### 覆盖规则
1. 作用域优先级（近者优先）
2. 选择器特异性
3. 声明顺序

## 公开接口（固定6个）

theme服务必须实现以下6个固定接口：

```typescript
interface ThemeService {
  /**
   * 获取主题列表
   * @param publisher - 可选的发行者过滤
   * @returns 主题元信息列表
   */
  list(publisher?: string): Promise<ThemeMeta[]>;

  /**
   * 应用主题
   * @param id - 主题ID
   */
  apply(id: string): Promise<void>;

  /**
   * 获取当前主题
   * @param appId - 可选的应用ID
   * @param pluginId - 可选的插件ID
   * @returns 当前主题状态
   */
  getCurrent(appId?: string, pluginId?: string): Promise<ThemeState>;

  /**
   * 获取所有CSS
   * @returns 完整的主题CSS字符串
   */
  getAllCss(): Promise<string>;

  /**
   * 解析主题链
   * @param chain - 作用域链
   * @returns 解析后的token
   */
  resolve(chain: string[]): Promise<ResolvedTokens>;

  /**
   * 获取主题契约
   * @param component - 可选的组件名
   * @returns 主题契约定义
   */
  contract.get(component?: string): Promise<ThemeContract>;
}
```

## 主题包标准结构

标准主题包必须包含以下结构：

```
theme-pkg/
├── manifest.yaml           # 主题清单
├── tokens/
│   ├── ref.json           # 原始值
│   ├── sys.json           # 语义值
│   ├── comp.json          # 组件值
│   ├── motion.json        # 动效值
│   └── layout.json        # 布局值
├── components/            # 组件样式
│   ├── Button.css
│   ├── Input.css
│   └── ...
├── motions/               # 动画定义
│   └── animations.css
├── icons/                 # 图标资源
│   └── ...
├── contracts/
│   └── theme-interface.contract.json  # 主题契约
└── preview/               # 预览资源
    └── preview.png
```

### manifest.yaml 示例
```yaml
id: chips-theme-default
name: 默认主题
version: 1.0.0
publisher: official
description: 薯片生态默认主题
preview: preview/preview.png
tokens:
  - tokens/ref.json
  - tokens/sys.json
  - tokens/comp.json
  - tokens/motion.json
  - tokens/layout.json
```

### 主题契约 contract.json 示例
```json
{
  "version": "1.0.0",
```

## 主题解析算法（运行时）

完整的运行时解析流程：

```typescript
async function resolveTheme(chain: string[]): Promise<ResolvedTheme> {
  // 1. 加载作用域链主题快照
  const snapshots = await loadScopeChain(chain);
  
  // 2. 按token层级合并（ref -> sys -> comp/motion/layout）
  const merged = mergeByLevel(snapshots);
  
  // 3. 执行冲突裁决
  const resolved = resolveConflicts(merged);
  
  // 4. 生成变量表与组件映射表
  const variables = generateVariableMap(resolved);
  const componentMap = generateComponentMap(resolved);
  
  // 5. 注入作用域样式并触发 theme.changed 事件
  await injectStyles(variables, componentMap);
  emit('theme.changed', { themeId: getCurrentThemeId() });
  
  return { variables, componentMap };
}
```

## 主题契约门禁

每个主题包必须通过以下校验：

### 1. token完整性校验
- 检查所有必需token是否存在
- 检查是否有冗余token
- 检查token值格式

### 2. 组件接口点校验
- 验证 `data-scope` 声明
- 验证 `data-part` 声明
- 验证组件是否实现声明的接口点

### 3. 动效参数安全校验
- 最大时长限制（如3000ms）
- 曲线白名单验证
- 禁止危险动画值

### 4. 颜色对比度校验
- 文本可读性验证
- WCAG标准检查
- 警告不达标颜色组合

## 主题切换流程

```typescript
// 主题切换流程
async function switchTheme(themeId: string) {
  // 1. 验证主题存在
  const theme = await themeService.get(themeId);
  if (!theme) throw new Error('THEME_NOT_FOUND');
  
  // 2. 预加载主题资源
  await preloadTheme(theme);
  
  // 3. 解析主题token
  const resolved = await themeService.resolve(['global', 'app', themeId]);
  
  // 4. 生成CSS变量
  const css = generateCSSVariables(resolved);
  
  // 5. 异步注入样式（不阻塞交互）
  await injectStylesAsync(css);
  
  // 6. 更新当前主题状态
  await themeService.apply(themeId);
  
  // 7. 触发主题变更事件
  emit('theme.changed', { themeId, timestamp: Date.now() });
}
```
