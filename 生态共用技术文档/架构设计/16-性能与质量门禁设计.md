# 性能优化与质量门禁设计

> **版本**：vNext 架构设计冻结稿  
> **适用阶段**：阶段18及后续重构实施阶段


## 架构归属声明（2026-03-06）

- Host 主责 L1-L9 运行时链路（含 Runtime Client、Declarative UI、Unified Rendering）并随 Host 发布。
- 组件库主责 L10 Component Headless，只负责状态机、结构与 a11y。
- SDK 仅作为开发者工具包提供类型、封装、脚手架与测试辅助，不承载运行时主实现。
- 用户设备仅安装 Host 即可运行；SDK 仅在开发与集成阶段按需使用。

## 性能预算与治理

### 预算基线总览

| 指标类别 | 指标名称 | 阈值 | 说明 |
|----------|----------|------|------|
| 首屏性能 | TTI | <= 2000ms | 首屏可交互时间 |
| 渲染性能 | 渲染提交耗时 | p95 <= 32ms | 单次渲染提交耗时 |
| 任务管理 | 长任务占比 | <= 5% | JavaScript长任务占比 |
| 主题性能 | 主题切换耗时 | <= 150ms | 主题切换可见完成 |
| 内存管理 | 峰值内存 | <= 基线1.5倍 | 应用级按能力等级配置 |

### 分层性能指标

#### L1-L3（底层）指标

| 指标 | 阈值 | 责任方 |
|------|------|--------|
| 路由延迟 p50 | <= 10ms | Host Kernel |
| 路由延迟 p95 | <= 50ms | Host Kernel |
| 服务启动时间 | <= 500ms | Host Services |
| 服务调用超时率 | <= 0.1% | Host Services |

#### L5-L7（Bridge层）指标

| 指标 | 阈值 | 责任方 |
|------|------|--------|
| Bridge调用延迟 | p95 <= 100ms | Runtime Client |
| 事件投递延迟 | <= 20ms | Bridge Transport |
| 重试成功率 | >= 95% | Runtime Client |

#### L8-L12（前端层）指标

| 指标 | 阈值 | 责任方 |
|------|------|--------|
| 首次内容绘制 FCP | <= 1500ms | App/Page |
| 首次可交互 FCP | <= 2000ms | App/Page |
| 主题切换 | <= 150ms | Theme Runtime |
| 列表滚动帧率 | >= 50fps | Unified Rendering |
| 渲染提交耗时 p95 | <= 32ms | Unified Rendering |

### L9 统一渲染性能基线数据格式

建议所有项目按统一 JSON 结构输出渲染基线，便于跨项目对比：

```ts
interface UnifiedRenderingPerfBaseline {
  generatedAt: string;
  runs: number;
  nodeCount: number;
  thresholds: {
    renderCommitP95Ms: number;
    longTaskRatioMax: number;
  };
  total: PerfStats;
  commit: PerfStats;
  layout: PerfStats;
  normalize: PerfStats;
}

interface PerfStats {
  p50: number;
  p95: number;
  avg: number;
  min: number;
  max: number;
}
```

### 优化策略

#### 请求优化
- 独立请求并行化（避免waterfall）
- 请求合并与批处理

#### 渲染优化
- 重组件按可见性分片渲染
- 大列表/树/表格强制虚拟化
- 主题切换使用异步注入与分片更新

#### 内存优化
- 组件懒加载
- 缓存策略（TTL + SWR）
- 内存泄漏检测

## 质量门禁

### 代码门禁

#### TypeScript门禁
- `strict` 模式全开
- 禁止 `any`（仅白名单例外并说明理由）
- 文件命名规范：
  - 组件/类：`PascalCase`
  - 文件/函数：`kebab-case`
  - 类型文件：`.types.ts`
  - 测试文件：`.test.ts`

#### 代码风格门禁
- ESLint + Prettier 强制校验
- 提交前必须通过 lint 检查

### 前端门禁

#### 零硬编码门禁
- **零硬编码用户文案**：所有UI文本必须通过i18n获取
- **零组件视觉硬编码**：所有样式必须通过主题token获取

#### 主题接口点门禁
- 组件必须声明主题接口点
- 主题接口点覆盖校验必须通过

### 契约门禁

#### 契约类型

| 契约类型 | 校验内容 | 工具 |
|----------|----------|------|
| theme 契约 | token完整性、组件接口点、动效安全 | theme-validator |
| layout 契约 | 布局结构、响应式断点 | layout-validator |
| bridge 契约 | 接口签名、参数schema | bridge-validator |
| component 契约 | 状态机、a11y、接口点 | component-validator |

#### manifest 门禁
- manifest schema 校验必须通过
- 版本号格式必须符合语义化版本

### 测试门禁

#### 单元测试覆盖要求

| 类型 | 最低要求 |
|------|----------|
| 组件 | >= 5 个测试用例 |
| 服务模块 | >= 8 个测试用例 |

#### 路径覆盖要求

| 路径类型 | 要求 |
|----------|------|
| 正常路径 | 100% 覆盖 |
| 异常路径 | 关键异常必须覆盖 |
| 边界路径 | 边界值必须覆盖 |

#### 契约测试
- 每服务动作必须有契约测试
- 契约测试验证输入输出schema

### 测试蓝图

| 测试类目 | 判定标准 |
|----------|----------|
| 契约测试 | 通过率 100%，无 schema 漂移 |
| 单元测试 | 组件/服务门槛达标且关键路径全绿 |
| 集成测试 | Bridge、Theme、Layout 链路无阻断 |
| E2E | 关键业务路径与基线行为一致 |
| 性能测试 | 不突破预算阈值 |
| 鲁棒性测试 | 故障注入后可恢复，无全局雪崩 |

## 可观测性与诊断

### 统一日志字段

```typescript
interface LogEntry {
  traceId: string;           // 链路追踪ID
  requestId: string;         // 请求ID
  pluginId?: string;         // 插件ID
  namespace?: string;         // 命名空间
  action?: string;           // 动作名
  durationMs?: number;       // 耗时
  result?: 'success' | 'error';  // 结果
  errorCode?: string;        // 错误码
}
```

### 链路追踪

- Transport 入站生成 `requestId`，贯穿 Runtime Client、Kernel、Service
- 事件链路记录 `source -> target`

### 诊断面板

| 监控项 | 内容 |
|--------|------|
| 路由性能 | 延迟分布、错误码 TOP |
| 权限审计 | 权限拒绝 TOP |
| 主题解析 | 主题解析失败率 |
| 插件状态 | 资源配额使用情况 |

## 鲁棒性与容灾

### 失败隔离

- 组件级 Error Boundary
- 页面局部失败可恢复，不影响全局窗口

### 回退策略

| 场景 | 回退方案 |
|------|----------|
| 主题失败 | 回退默认主题 |
| 布局契约非法 | 回退安全模板 |
| Bridge 超时 | 标准错误 + 可重试提示 |

### 熔断与恢复

- 同路由连续失败触发熔断
- 半开状态探测成功后自动恢复

### 数据安全

- 写操作支持幂等去重
- 崩溃恢复后可重放关键未完成操作（限白名单）
