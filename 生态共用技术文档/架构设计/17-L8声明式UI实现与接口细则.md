# L8 声明式 UI 实现与接口细则

> 文档状态：生态共用公开规范  
> 适用范围：所有接入 Host 渲染链路的应用插件、卡片插件、布局插件、SDK 调用封装  
> 修订日期：2026-03-06

---

## 1. 定位与边界

- L8（Declarative UI）是 Host 运行时能力，对外提供**语义层**，不提供视觉样式实现。
- L8 只允许依赖 L7（UI Hooks）与 L10（组件语义层），禁止直接访问 Host 服务细节。
- L8 对外关注点：
  - 语义原语
  - 声明树节点模型
  - 组合模式规则
  - 事件与副作用模型

---

## 2. 核心语义原语（公开）

统一语义原语：

- `View`
- `Stack`
- `Grid`
- `Form`
- `List`

语义原语必须只表达布局与交互语义，不得包含颜色、阴影、圆角、字体大小等视觉硬编码参数。

---

## 3. 声明树节点模型（公开契约）

```ts
interface UINode {
  id: string;
  type: 'View' | 'Stack' | 'Grid' | 'Form' | 'List' | string;
  props?: Record<string, unknown>;
  state?: Record<string, unknown>;
  bindings?: Record<string, string>;
  events?: Record<string, string>;
  themeScope?: string;
  children?: UINode[];
}
```

### 3.1 字段约束

- `id`：必填，不能为空。
- `type`：必填，不能为空；推荐优先使用标准语义原语。
- `props`：仅语义属性（布局、结构、行为标识），不得使用视觉值。
- `bindings`：声明数据绑定关系（`field -> source.path`）。
- `events`：仅声明事件语义与 handler 标识映射。
- `themeScope`：主题解析作用域，不得绕过主题运行时直接注入视觉参数。

---

## 4. 组合模式规范（Vercel 组合模式）

### 4.1 必须遵守

- 复杂节点优先 Compound Components。
- 通过显式 slot 组合结构，不通过 boolean 模式控制多变体。
- Provider 负责状态实现，L8 只消费注入状态与动作。

### 4.2 禁止行为

- 禁止 boolean 模式扩张（如 `isCompact/isInline/showFooter` 组合）。
- 禁止 slot 未声明即注入节点。
- 禁止在组合层引入视觉硬编码字段。

---

## 5. 事件与副作用模型（公开契约）

### 5.1 事件绑定

- `events` 仅声明语义事件（例如 `onSubmit`、`onOpenFile`）。
- 事件执行器在页面域注册，不在声明树中嵌入可执行逻辑。

### 5.2 副作用分级

- `ui-effect`：滚动、聚焦、过渡等 UI 层副作用。
- `runtime-effect`：调用 Runtime Client 的系统能力副作用。
- `telemetry-effect`：日志、指标、埋点副作用。

### 5.3 强制规则

- 禁止在渲染阶段直接触发 `runtime-effect`。
- `runtime-effect` 只能在事件回调或 commit 后调度执行。

---

## 6. 标准错误码（L8）

建议统一使用以下错误码（`DECLARATIVE_UI_*`）：

- `DECLARATIVE_UI_NODE_INVALID`
- `DECLARATIVE_UI_BOOLEAN_MODE_FORBIDDEN`
- `DECLARATIVE_UI_SLOT_UNDEFINED`
- `DECLARATIVE_UI_SLOT_REQUIRED`
- `DECLARATIVE_UI_SLOT_TYPE_MISMATCH`
- `DECLARATIVE_UI_SLOT_MULTIPLE_FORBIDDEN`
- `DECLARATIVE_UI_EVENT_HANDLER_NOT_FOUND`
- `DECLARATIVE_UI_RUNTIME_EFFECT_FORBIDDEN_IN_RENDER`
- `DECLARATIVE_UI_EFFECT_SCHEDULE_INVALID`

---

## 7. 质量门禁（生态共用）

- L8 单测必须覆盖：
  - 原语创建
  - 节点模型校验
  - 组合模式约束
  - 事件绑定与派发
  - 副作用分级
  - 渲染阶段 runtime-effect 禁止规则
- 任何接入仓必须保证：
  - `build` 通过
  - L8 相关测试通过
  - 文档与实现一致

---

## 8. 与其他层协同

- 与 L7：事件处理器内通过 Runtime Client 调用系统能力。
- 与 L9：L9 负责 Normalize/Validate/Resolve/Compute/Commit/Dispatch，L8 负责声明语义输入。
- 与 L11：L8 通过 `themeScope` 提供主题作用域线索，不直接处理 token 合并算法。

---

## 9. 实施建议

- 新项目接入时先落地节点契约与组合模式，再接入业务事件与效果分发。
- 任何生态共用能力变更，先更新本文件，再更新具体仓实现。
