# 组件库对外使用总览

## 1. 组件库定位

薯片组件库是生态统一的无头组件标准实现，提供：

- 稳定交互行为
- 标准化结构语义
- 统一主题接口点
- 可访问性基础能力

技术基线：React + Ark UI + Style Dictionary + Tamagui (Core)。

## 2. 使用边界

允许：

- 在应用插件、卡片插件、布局插件中直接使用组件库导出组件
- 按主题契约覆盖视觉表现
- 通过组合模式构建业务 UI

禁止：

- 绕过组件库直接复制 Ark UI 内部实现作为生态标准
- 在业务页面硬编码组件视觉值
- 改写组件契约字段语义

## 3. 最小接入步骤

1. 安装组件库与对应 peer 依赖
2. 在页面根节点接入组件库 Provider
3. 保证主题变量注入可用
4. 使用组件库基础组件替代自建基础控件
5. 完成契约测试与 a11y 检查

## 4. 对主题包的要求

- 主题包必须覆盖组件契约要求的 token key
- 必须支持 `data-scope/data-part/data-state` 选择器
- 不允许在主题中修改组件结构语义
- 必须遵循主题权重链（低 -> 高）：`global -> app -> box -> composite-card -> base-card -> component`
- token 查找必须按“高权重优先，缺失逐层向上回退”执行

## 6. 卡片显示链路共识

- 查看器、编辑器等应用通过 SDK 统一显示接口传入卡片文件。
- 复合窗口模式仅 `view | preview`，禁止自定义未约定 mode 值。
- Host 内置渲染运行时负责基础卡片分发、模板编译、iframe 拼接；SDK 仅提供开发封装。
- 复合卡片 iframe 事件应按窗口来源与 origin 白名单双重过滤。
- 应用层仅承担窗口承载与业务外壳，不重复实现渲染引擎。
