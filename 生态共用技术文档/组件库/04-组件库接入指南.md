# 组件库接入指南

## 1. 适用场景

- 新建插件项目接入薯片组件库
- 新建应用页面按统一契约直接使用组件库
- 首发主题包按 token 体系接入

## 2. 接入原则

1. 先接入契约，再接入视觉
2. 优先接入 P0 核心组件，再扩展高级组件
3. 禁止临时桥接代码进入主线
4. 新项目不包含历史迁移策略

## 3. 标准接入步骤

1. 建立任务分支（`codex/<task>`）
2. 接入组件库 provider 与主题运行时
3. 对齐 `data-scope/data-part/data-state`
4. 对齐主题权重与查找链（`global -> app -> box -> composite-card -> base-card -> component`，高权重优先回退）
5. 跑完契约测试、a11y、性能测试
6. 通过后合并到 `develop`

TypeScript 接入基线：

- 必须保持 `strict` 打开
- 必须直接使用组件库正式声明文件
- 禁止在应用侧添加本地声明兜底、`as any` 降级或关闭类型门禁
- 如果组件库缺少正式声明或导出冲突，必须先提交工单再处理

卡片显示场景必须额外接入：

- 封面显示统一使用 `CardCoverFrame`
- 复合卡片显示统一使用 `CompositeCardWindow`
- 渲染拼接统一由 Host 内置渲染运行时处理（可通过 SDK 显示接口调用）

## 4. 风险控制

- 对高频组件采用分批接入与灰度开关
- 保留回滚点（tag 或 release 分支）
- 每批次接入必须有回归测试记录

## 5. 阻断条件

遇到以下情况必须先提工单：

- 缺少生态标准组件导致无法接入
- SDK/Theme Runtime 接口不满足组件运行需求
- 主题契约缺失且无法补齐

工单路径：`项目日志与笔记/问题工单.md`。
