# 可访问性与质量基线

## 1. 可访问性基线

所有组件必须满足：

- 键盘可完整操作
- 焦点可见且顺序正确
- 语义 role 正确
- 读屏信息完整（label/description/error）
- 图标按钮最小热区 44px

## 2. 测试基线

- 单组件最少 5 个测试用例
- 契约测试 100% 通过
- 关键路径集成测试覆盖
- 回归测试包含主题切换与语言切换场景

## 3. 性能基线

- 主题切换可见完成 <= 150ms
- 渲染提交 p95 <= 32ms
- 长任务占比 <= 5%
- 组件库阶段性能力必须提供可复现基线脚本与报告（命令、数据规模、p95结果可追溯）

## 4. 发布基线

发布前必须通过：

- lint
- type-check
- unit test
- contract test
- a11y test
- perf test

任何一项失败即发布阻断。

阶段十发布补充门禁：

- `release:check`（发布包字段 + 门禁报告状态）
- 生态接入验收模板校验（`validate:ecosystem-acceptance`）

当前组件库仓库使用 `npm run verify` 作为聚合门禁入口，至少覆盖 token 校验、token 构建、主题契约校验、全量测试。
阶段九新增 `npm run quality:gate` 作为发布阻断入口，覆盖 `lint/typecheck/test/test:contracts/test:a11y/test:perf/build`。
最新基线结果：`219 token keys / 32 contracts / 151 tests` 全量通过，质量门禁报告 `passed`。

## 5. iframe 高级组件附加基线

- iframe 沙箱权限必须最小化
- 复合卡片节点失败时必须可降级，不可整窗崩溃
- SDK 显示接口在查看器/编辑器场景表现必须一致

## 6. 日志与可观测性

组件级错误至少包含：

- `traceId`
- `component`
- `action`
- `errorCode`
- `durationMs`

用于问题定位与质量审计。

## 7. A11y 工具基线

组件库应提供可复用基础工具，并在组件实现中统一复用：

- 键盘意图识别（activate/navigate/dismiss）
- ARIA 规则校验与断言
- 焦点元素收集与顺序移动工具

禁止在各组件内部重复实现不一致的键盘/焦点逻辑。
