# 卡片封面组件标准

## 1. 组件定义

`CardCoverFrame` 是生态统一卡片封面显示组件，输出为受控 iframe 窗口。

## 2. 使用场景

- 资源管理器卡片列表
- 卡片选择器
- 导入预览
- 编辑引擎侧边封面列表

## 3. 功能要求

- iframe 显示卡片封面 HTML
- 对外暴露封面标题与比例元信息，供应用层自行布局
- 加载、空态、失败态完整
- 对外事件：打开卡片、加载完成、加载失败
- 封面 iframe 必须来自正式 `coverUrl`，不接受任意 HTML 字符串
- 封面内容完全由 `.card/cover.html` 决定，不自动注入主题系统、多语言系统或复合卡片壳层

## 4. 正式链路

`CardCoverFrame` 的正式数据来源必须是：

1. Host `card.renderCover({ cardFile })`
2. SDK `client.card.coverFrame.render({ cardFile })`
3. 应用壳层挂载返回的 iframe，并自行决定是否显示标题、摘要和操作按钮

Host 返回的正式视图字段：

- `view.title`
- `view.coverUrl`
- `view.ratio?`

约束：

- `coverUrl` 必须指向受控的 `.card/cover.html`
- `CardCoverFrame` 只能消费 `coverUrl`，不得接收 `srcdoc` 或任意内联 HTML
- `.card/cover.html` 中的相对路径按 `.card/` 目录解析，可继续访问 `.card/cardcover/*`

## 5. 契约要求

- `data-scope="card-cover-frame"`
- `data-part="root|frame-container|iframe|status"`
- `data-state="idle|loading|ready|error|disabled|empty"`

## 6. 安全要求

- iframe 必须使用受限 sandbox
- sandbox 当前正式策略为 `allow-scripts`，不得加入 `allow-same-origin`，并禁止危险权限（top-navigation、popups-to-escape-sandbox）
- 封面资源 URL 必须来自受控 SDK 接口
- 禁止任意注入 HTML 字符串
- 应用层不得把封面 iframe 当作同源文档直接读取内部 DOM；封面事件与状态必须通过正式链路消费

## 7. 验收要求

- 封面可稳定加载
- `title/ratio` 元信息可被应用层稳定消费
- 失败可回退并给出标准错误
- 主题切换与语言切换不破坏布局
- 本地目录卡片与 `.card` 压缩包卡片都必须正确加载 `.card/cover.html` 及其相对资源
