# 底层内核和打包模式

薯片生态采用十二层架构。第一层是PAL（平台抽象层），负责适配不同操作系统（Windows、macOS、Linux）的差异，向上提供统一的平台能力接口。第二层是Host内核（薯片内核），使用TypeScript实现，运行在Electron主进程中，提供中心路由、服务注册、事件总线、生命周期管理等核心能力。第三层是Host服务域，包含file、resource、config、theme、i18n、window、plugin、module、platform、log、credential、card、box、zip、serializer、control-plane等服务。第四层是Plugin Runtime，负责插件加载、隔离、会话、权限管理。第五层是Bridge Transport，通过Electron的contextBridge机制向插件暴露 `window.chips.*` 标准接口，是插件访问系统能力的唯一通道。第六层是Runtime Client，负责协议编解码、错误归一、重试。第七层是UI Hooks，面向页面的能力调用接口。第八层是声明式UI层，提供View、Stack、Grid、Form、List等语义原语。第九层是统一渲染层，负责Normalize、Validate、Resolve、Commit。第十层是无头组件库，状态机、结构、a11y，无视觉硬编码。第十一层是Theme Runtime，token解析、作用域覆盖、变量注入。第十二层是App/Page Domain，页面状态、业务流程、编排。

架构归属声明（2026-03-06）：L1-L9 由 Host 内置运行时主责实现并随 Host 发布，L10 由组件库主责，SDK 仅为开发者工具包（封装、类型、脚手架），不承载 Runtime Client 或前端渲染主实现。

功能插件通过 Bridge API 调用底层服务。当插件需要使用视频处理能力时，通过正式模块服务按 capability 调用对应模块插件，例如发起 `module.invoke({ capability: 'media.video.process', method: 'run', input })`。插件只关注业务逻辑和界面编排，不需要实现视频解码、播放控制等基础功能。这就是"底层厚、上层薄"的设计理念——底层尽可能包含所有通用能力，上层应用只是调用底层的薄壳。

基础服务层的核心服务由薯片内核统一管理和路由。共享功能模块作为独立的模块插件安装在系统中，每个模块是独立的代码仓库，由 Host 托管为无界面能力提供者，可以被多个上层插件共享使用，类似操作系统的共享服务。

模块采用按需加载机制。薯片主机启动时只加载内核和最基本的核心服务（配置、日志、文件），当用户使用某个功能时，内核才动态加载对应的服务和模块。比如用户在待办软件中导入视频，此时才加载视频元数据提取、转码或预处理等对应模块能力。

薯片生态采用单一主机模式。用户设备上安装一个薯片主机（Chips Host），这是唯一的 Electron 应用程序，长期在后台运行。所有上层软件都是运行在主机内部的插件应用。每个插件以 `.cpk` 格式打包（本质是 ZIP 文件），安装时解压到插件目录，在操作系统中创建快捷方式。

用户双击快捷方式时，快捷方式指向薯片主机并带上启动参数指定加载哪个插件。主机从插件注册表中找到对应插件，创建 BrowserWindow 窗口，加载插件的 Web 入口页面。对用户来说，感觉就像启动了一个独立的软件，但实际上是主机加载了该插件。

如果用户同时打开多个薯片软件，它们都运行在同一个主机进程中。主机为每个插件创建独立的 BrowserWindow 渲染进程。多个插件共享同一个 Chromium 引擎和主进程中的基础服务，系统资源占用远少于独立运行多个 Electron 应用。每个插件的渲染进程互相隔离，一个插件崩溃不会影响其他插件和主进程。

插件的安装非常简单。用户可以从应用市场下载 `.cpk` 插件包，也可以直接拖拽到主机窗口安装。主机的插件管理器验证清单文件、检查兼容性、将插件解压到对应目录、更新注册表、在桌面创建快捷方式。卸载时反向操作即可。

插件分为五种类型。应用插件是完整的应用（如编辑引擎、查看器），拥有独立窗口。卡片插件定义基础卡片类型的渲染和编辑组件。布局插件定义箱子的展示方式。模块插件是 Host 托管的无界面能力模块，可被多个插件和 Host 自身统一调用。主题插件提供视觉样式。所有类型使用统一的 `.cpk` 打包格式和 `manifest.yaml` 清单文件。

每个插件都是独立的代码仓库，可以独立开发、构建、发布。第三方开发者通过 Chips SDK 和 Bridge API，使用标准的 Web 技术（HTML、CSS、JavaScript）开发插件，不需要了解 Electron 或底层实现细节。
