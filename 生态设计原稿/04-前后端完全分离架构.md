# 前后端完全分离架构

薯片生态的前端和后端完全解耦，可以自由组合不同的前端实现和后端模块。前端只负责界面展示和用户交互，后端只负责业务逻辑和数据处理。它们之间通过薯片内核进行通信，从不直接接触。

对于同一个后端功能，可以有多个不同的前端实现。比如待办功能的后端模块是统一的，但可以有列表风格的待办界面、看板风格的待办界面、日历风格的待办界面。不同的开发者可以用不同的技术、不同的设计理念开发前端。

这些不同的前端实现需要遵循统一的接口标准。薯片生态定义了前端接口规范，规定前端如何调用后端功能、如何接收数据、如何处理事件。所有前端组件都遵循这个规范，确保能对接后端。

后端模块也需要遵循统一的接口标准。后端接口规范定义了模块提供哪些功能、接收什么参数、返回什么数据。所有后端模块都遵循这个规范，确保能被任何前端调用。

前端组件调用后端时，通过标准接口发送请求到内核。内核验证请求格式和权限，然后路由到对应的后端模块。后端处理完成后，按标准格式返回结果给内核，内核转发给前端。

官方产品统一使用React框架开发。第三方开发者开发新的前端产品时，可以自由选择React、Svelte或任何其他技术。只要通过薯片内核提供的标准接口与后端通信即可。重点不是让官方组件库支持多种框架，而是通过标准接口支持不同开发者的不同前端产品。

软件制作器在组装软件时，分析前端组件和后端模块的接口，自动生成内核的路由配置。这个配置告诉内核如何将前端的请求路由到后端的具体模块和方法。

后端模块之间的协作也必须通过内核。模块不能直接调用其他模块，而是向内核发送请求，由内核路由。这确保了后端模块之间也是完全解耦的。

这种分离让前端和后端可以独立开发、独立更新、独立替换。开发者可以只开发前端复用已有后端，也可以只开发后端使用已有前端，还可以开发新的前后端组合创造新的产品。

## 架构归属声明（2026-03-06）

- Host 主责 L1-L9 运行时链路（含 Runtime Client、Declarative UI、Unified Rendering）并随 Host 发布。
- 组件库主责 L10 Component Headless，只负责状态机、结构与 a11y。
- SDK 仅作为开发者工具包提供类型、封装、脚手架与测试辅助，不承载运行时主实现。
- 用户设备仅安装 Host 即可运行；SDK 仅在开发与集成阶段按需使用。
