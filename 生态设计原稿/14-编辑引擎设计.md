# 编辑引擎设计

## 编辑引擎的定位

卡片编辑引擎是创建和编辑卡片的软件平台，是生态中最重要的生产力工具。编辑引擎本身是框架，实际的编辑处理功能由对应的插件提供。引擎负责接收用户操作，传达给相应插件进行数据处理，实时保存所有更改。用户操作完成后数据自动保存并刷新，不需要手动保存。

编辑引擎提供窗口管理、文件管理、拖拽处理、保存机制等基础能力。具体的编辑功能由插件实现。例如编辑面板窗口只提供容器和挂载点，实际的编辑界面由基础卡片的编辑组件插件负责设计。编辑面板不区分普通设置和高级设置，界面布局完全由插件自己决定。

## 布局模式

编辑引擎支持多种布局模式，每种模式是独立的插件。系统设计了多种基础布局，用户可以安装更多布局插件，在设置中选择默认布局，也可以临时切换布局。

无限画布布局采用两层界面设计。桌面层是底层，这是一个无限大的画布，背景显示网格图案。用户可以在桌面上自由摆放卡片和箱子，可以拖动桌面平移查看不同区域，可以放大缩小改变缩放比例。桌面没有边界限制，可以无限扩展。卡片和箱子窗口放置在桌面上，随桌面一起缩放和移动。

窗口层是顶层，像一块玻璃悬浮在桌面上方。窗口层中的元素固定在屏幕位置，不随桌面的缩放和平移而移动。窗口层包含工具窗口和程序坞。

程序坞在窗口层下方或侧边，显示所有工具组件的图标。工具组件最小化时变成图标保存在程序坞中，点击图标后展开成悬浮窗口显示在窗口层。

窗口层右下角有缩放比例滑块，用户除了用鼠标滚轮和触摸手势，还可以拖动滑块改变桌面缩放比例。滑块旁边显示当前的缩放百分比。

主要的工具组件窗口包括文件管理器窗口显示当前桌面上所有的卡片和箱子文件，用户可以创建新文件、打开已有文件、删除文件、重命名文件，点击文件后对应的卡片或箱子显示在无限桌面上。编辑面板窗口固定在窗口层右侧，显示当前选中基础卡片的编辑组件，用户选中某个基础卡片时引擎通过内核加载对应的编辑组件插件显示在编辑面板中。卡箱库窗口是网格列表显示各种基础卡片类型和箱子布局类型，用户从卡箱库拖动到桌面空白区域创建新的卡片或箱子，拖动到已有卡片添加到该卡片中。

在当前正式链路中：

- 画布中的复合卡片窗口由编辑引擎内部按基础卡片粒度装配；
- 每个基础卡片仍然独立渲染到自己的 iframe 中；
- 编辑引擎壳层负责汇总高度、节点选中和画布交互代理；
- 编辑面板由本地 `EditorHost` 直接挂载基础卡片编辑器导出；
- 编辑引擎通过“内置描述符 + Host 已安装并启用的基础卡片插件构建产物入口”动态建立运行时注册表；
- 当前正式已注册的基础卡片类型至少包括富文本与图片，两者都不在主流程里保留特殊分支。

工具窗口都是悬浮窗口，可以拖动改变位置，可以调整大小。点击收起按钮后窗口高度缩小到只显示标题栏，内容完全隐藏。点击最小化后窗口关闭，图标显示在程序坞。

工作台布局采用分区窗口组合方式，没有无限画布。左侧是树状文件管理器显示文件的层级结构，中间是预览区显示选中的卡片或箱子内容，右侧是编辑窗口显示编辑组件，底部可以有工具栏或其他辅助窗口。这些窗口可以拖动重新排列组合，可以调整大小，可以停靠到不同位置，可以创建标签页。

页面式布局类似笔记软件，左侧页面导航，中间全屏显示一个卡片。看板布局类似项目管理工具，横向多列结构，每列一个分类。网格布局以网格显示卡片封面，点击进入详情。时间轴布局按时间顺序排列卡片。

第三方开发者可以开发布局插件，用户从应用市场安装后，在布局切换菜单中会出现新选项。切换布局时，所有打开的卡片保持打开状态，数据自动保存，界面适配新布局。

虽然布局可以变化，但某些核心功能是所有布局都需要提供的：卡片管理创建、打开、保存、删除，基础卡片编辑添加、删除、调整顺序，文件管理浏览、导入导出、重命名，系统功能撤销重做、搜索、设置。这些功能由编辑引擎系统提供，布局插件调用。

## 复合卡片窗口设计

复合卡片显示为竖直长条状窗口。窗口顶部是菜单栏，左边显示卡片名称，右边排列操作按钮。

双击卡片名称进入编辑状态，点击其他地方失去焦点后自动保存。

最小化按钮直接关闭窗口，窗口从桌面消失，不在程序坞显示，也不显示为封面。

切换到封面按钮关闭当前窗口，在桌面上创建封面窗口。封面窗口显示卡片封面，封面通过网页框架显示，封面铺满窗口，下方有透明底文字显示卡片名称。封面窗口大小固定，比例可以在设置中调整。点击封面后，封面窗口关闭，展开卡片显示完整内容。

设置按钮打开设置窗口，用户可以修改卡片名称，选择使用的主题包，上传和编辑封面，调整封面比例，导出卡片等。

锁定按钮切换编辑模式和查看模式，编辑模式下点击基础卡片会打开编辑面板，查看模式下可以正常与卡片内容交互。

由于编辑态复合卡片是由编辑引擎在应用内拼装的，基础卡片选中态由编辑引擎外层壳层直接管理；用户在单卡 iframe 中点击基础卡片后，`BasecardFrameHost` 把 `baseCardId` 回传给编辑引擎，编辑引擎再将对应 `cardType/baseCardId/config` 交给本地 `EditorHost`。

编辑引擎右侧只保留正式编辑面板，不再在复合卡片窗口内部额外维护一条重复的“属性编辑列表”作为正式入口。

收起按钮改变窗口显示状态，卡片窗口收起后高度缩小到十六比九比例，内容可滚动查看，应用窗口收起后只显示标题栏。

复合卡片展开态必须保持“内容驱动高度”原则。各基础卡片 iframe 先由编辑引擎本地运行时测量单卡高度，再由 `CompositeCardAssembler` 汇总整张复合卡片当前总高度并驱动窗口更新。编辑引擎不再依赖 Host 复合 iframe 的 `resize` 事件完成这一步。

无限画布中的滚动/缩放行为必须遵循以下正式约束：

- 桌面背景上的鼠标滚轮、触摸板缩放与触摸屏捏合，用于桌面缩放；
- 工具窗口和非代理场景下的复合卡片窗口，保持各自局部页面的原生滚动；
- 展开态无限长复合卡片在 `interactionPolicy: 'delegate'` 下，把内部滚动解释为整桌面平移，把内部捏合解释为整桌面缩放；
- 当用户在代理中的复合卡片上持续滚动并滑出该卡片时，桌面缩放必须临时屏蔽；只有当前滚动序列中断后，桌面背景滚动才允许重新触发缩放。

关闭按钮将窗口从桌面移除，但文件在文件管理器中保留。

## 封面设计

封面保存在卡片文件的`.card/cover.html`文件中。封面是用户浏览卡片列表或箱子时看到的预览内容，可以完全自定义。

当前正式实现中，封面编辑器不是独立弹窗，而是嵌入在复合卡片设置页面中的一个独立工具区域。页面分为左右两栏：

- 左侧是单张封面实时预览区，只显示一个封面预览卡片；
- 右侧是编辑区，负责比例、方向、模式和资源操作。

当前正式编辑器支持九种比例预设：1:1、1:1.414、2:3、3:4、1:1.618、16:9、2:1、9:19.5、1.58:1，并支持横竖方向切换，默认是3:4竖版。最终比例保存在metadata.yaml的cover_ratio字段中。

当前正式支持两种编辑模式：

- 图片封面：用户上传图片后，编辑器生成引用该图片的`cover.html`；如果需要裁剪，进入裁剪弹窗后使用固定比例选框进行拖动和四角缩放，最终裁剪结果回写到`.card/cardcover`资源并生成新的`cover.html`。
- HTML封面：用户直接输入或粘贴HTML代码，内容原样保存为`.card/cover.html`。

当前正式封面工具的目标只有一个：无论通过哪种模式编辑，最后都必须落到`.card/cover.html`。如果用户没有设置封面，系统生成默认封面。当前默认封面是中性渐变背景与居中的卡片名称，而不是额外的灰色占位块。

封面运行时与主题、多语言系统解耦。封面显示内容完全由cover.html本身决定，Host和SDK不会向封面文档注入主题系统、多语言系统或复合卡片壳层。

无限画布中的卡片切换到封面状态时，编辑引擎不再使用本地占位视图，而是挂载正式的封面iframe。该iframe来自 `Host card.renderCover -> SDK client.card.coverFrame.render(...)` 链路，只显示一张封面卡片，并保持阴影、厚度和立体感。

## 卡片渲染

编辑引擎工作区中打开的是解包态卡片目录。编辑过程直接读写卡片根目录下的 `.card/metadata.yaml`、`.card/structure.yaml`、`content/*.yaml` 与 `.card/cover.html`，导出阶段再由专门打包流程生成标准 `.card` 压缩产物。

编辑态复合卡片的正式渲染方式如下：

1. `CardService` 读取并规范化每个基础卡片配置；
2. 编辑引擎根据 `cardType` 找到本地注册的 `BasecardDescriptor`；
3. `BasecardFrameHost` 为每个基础卡片创建一个独立 iframe；
4. `renderBasecardView` 挂载到该 iframe，并注入当前主题变量；
5. `CompositeCardAssembler` 按顺序拼装所有单卡 iframe；
6. 单卡高度、加载态和错误态被汇总回 `CardWindow`。

当前正式视觉要求：

- 基础卡片之间不再额外包裹边框、圆角和阴影；
- 装配层只负责结构编排，不负责附加视觉壳层；
- 每个基础卡片仍然保持自己的内容边界和失败隔离。

当前正式编辑方式如下：

1. 用户选中基础卡片；
2. `EditPanel -> PluginHost -> EditorHost` 打开对应编辑会话；
3. `renderBasecardEditor` 直接挂载到编辑面板容器；
4. 若编辑器需要内部资源，统一通过宿主提供的 `resolve/import/delete/releaseResourceUrl` 资源链路操作卡片根目录文件；
5. 编辑器输出草稿，由会话层统一防抖提交到 `CardService`；
6. `CardService` 在持久化时写入待导入资源、重建资源清单，并在确认无剩余引用后执行待删除资源；
7. 提交成功后，预览链路刷新对应基础卡片。

## 拖动操作

用户从卡箱库拖动基础卡片类型到桌面空白区域，拖动过程中卡片跟随鼠标，松开后在该位置创建新的复合卡片窗口，基础卡片自动添加进去。

拖动基础卡片到已有复合卡片时，根据鼠标位置判断插入位置。鼠标在某个基础卡片上方区域时，插入到该卡片上方。鼠标在下方区域时，插入到该卡片下方。拖动过程中显示插入位置的指示线，松开后卡片插入到指定位置。

拖动一个复合卡片到另一个复合卡片时，被拖动的复合卡片作为整体嵌套进目标卡片，保持完整结构。

## 文件拖拽支持

编辑引擎全局支持文件拖拽。用户从操作系统文件管理器拖动文件到引擎任意位置，系统都能接收。拖到桌面空白区域时，根据文件类型自动创建对应的卡片。拖到已有卡片窗口时，添加到该卡片中。拖到编辑面板窗口时，添加到当前正在编辑的基础卡片中。

系统通过内核的文件识别功能判断文件类型，然后路由到对应的处理模块。拖入视频文件创建视频卡片，拖入图片文件创建图片卡片，拖入卡片文件嵌套到当前卡片中。
