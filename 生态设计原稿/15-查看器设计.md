# 查看器设计

## 查看器的定位

卡片查看器用于查看卡片和箱子内容，是消费卡片内容的主要入口。查看器调用渲染模块展示各种类型的基础卡片内容，处理用户的浏览交互。

查看器本身是应用插件，提供查看功能框架。具体的卡片渲染由对应的卡片插件负责。查看器加载卡片文件，解析结构，根据基础卡片类型调用相应的渲染组件。渲染组件在iframe中运行，显示卡片内容。

当前正式查看态不自行拼装基础卡片 iframe，而是通过 Host 渲染链路与 SDK 显示接口消费统一复合卡片窗口。

## 渲染机制

查看器的核心是卡片渲染机制。查看器通过Bridge API请求 Host `card.render` 读取并渲染card文件内容，SDK 再通过 `client.card.compositeWindow.render(...)` 将结果交付为正式复合 iframe。

Host 根据每个基础卡片的类型加载对应的渲染组件。渲染组件接收配置数据，生成HTML内容，显示在独立iframe中。多个基础卡片的iframe按照顺序排列，形成完整的复合卡片视图。

查看器支持懒加载优化。当卡片包含大量基础卡片时，优先渲染可见区域的卡片。远离视口的卡片延迟加载，提高初始加载速度。

## 扩展功能

查看器内置各种文件查看工具，处理不同类型的内容。

视频播放器工具用于播放视频内容，支持多种视频格式，提供播放控制、进度条、音量调节等标准功能。播放器作为模块插件，可以被替换和扩展。

音乐播放器工具用于播放音频内容，显示专辑封面、播放进度、歌词等。播放器支持列表播放、随机播放、循环播放等模式。

图片基础卡片当前正式支持单图、网格、纵向完整展开和横向滚动四类展示方式。查看器负责承载这些内容，不额外定义一套脱离基础卡片插件的独立图片运行时。

文档查看器用于显示PDF、Office文档等内容，将文档渲染为可浏览的格式。文档查看器支持翻页、缩放、搜索等功能。

## 界面布局

查看器的界面设计围绕内容展示展开。

主区域显示卡片内容，根据窗口大小自动调整布局。内容区域可以缩放以适应不同屏幕尺寸。

工具栏提供常用操作按钮如返回、分享、导出等。工具栏的位置和样式可以自定义。

导航组件帮助用户在复合卡片的不同部分之间切换。如果复合卡片包含多个基础卡片，导航显示当前查看的部分，提供跳转功能。

侧边栏用于显示元数据和辅助信息如标签、创建时间、文件大小等。用户可以展开或收起侧边栏。

## 交互功能

查看器支持丰富的交互功能，提升浏览体验。

缩放功能允许用户调整内容的显示比例。可以通过滚轮、缩放按钮、手势等方式缩放。

滚动导航在内容超出窗口时提供滚动浏览。用户可以平滑滚动查看完整内容。

当查看器采用 Host 托管复合卡片窗口时，内部资源路径必须继续从卡片根目录开始记录。相对资源由 Host 统一解析并分发给单卡 iframe，查看器本身不拼接资源绝对路径。

书签功能让用户标记重要的位置，下次打开可以直接跳转到书签处。

历史记录跟踪用户的浏览历史，支持前进和后退操作。

## 查看模式

查看器支持多种查看模式适应不同场景。

全屏模式隐藏所有界面元素，只显示内容，适合沉浸式浏览。用户按Escape键或点击返回按钮退出全屏。

演示模式优化内容展示，适合分享和展示场景。演示模式可能调整字体大小、对比度等参数。

阅读模式简化界面，只保留核心阅读功能，隐藏工具栏和侧边栏。

打印模式优化打印输出，调整页面布局和样式确保打印效果。

## 分享功能

查看器提供内容分享能力。

导出功能将卡片保存为其他格式，如PDF、图片等。导出的文件可以在其他软件中打开。

分享链接生成卡片的分享链接，其他用户可以通过链接直接查看卡片内容。链接可以设置为公开或私有，控制访问权限。

嵌入代码生成可以在第三方网站嵌入卡片的HTML代码。嵌入的卡片保持交互能力，观看者可以直接在网页中浏览内容。

## 性能优化

查看器采用多种优化策略确保流畅体验。

缓存机制避免重复加载。相同卡片第二次打开时使用缓存内容。缓存可以配置大小和过期时间。

预加载在用户浏览时提前加载后续内容。当用户滚动到接近页面底部时，后台开始加载下一页内容。

资源压缩减少网络传输。对于网络资源，系统在传输前进行压缩，在本地解压缩使用。

渐进加载优先显示内容框架，后台逐步加载详细数据。用户在开始时就能看到内容轮廓，不需要等待全部加载完成。
