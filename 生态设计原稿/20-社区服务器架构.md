# 社区服务器架构


## 架构归属声明（2026-03-06）

- Host 主责 L1-L9 运行时链路（含 Runtime Client、Declarative UI、Unified Rendering）并随 Host 发布。
- 组件库主责 L10 Component Headless，只负责状态机、结构与 a11y。
- SDK 仅作为开发者工具包提供类型、封装、脚手架与测试辅助，不承载运行时主实现。
- 用户设备仅安装 Host 即可运行；SDK 仅在开发与集成阶段按需使用。

## 社区服务器的定位

社区服务器是服务器端软件，提供多用户社区服务。社区是网络上的公共平台，由社区服务器提供软件支持。用户可以在社区中创建房间，保存箱子和卡片，浏览其他用户的内容并与之互动。

社区服务器由存储服务器和社区平台两部分组成。存储服务器负责卡片和箱子的云端存储，社区平台提供用户管理、房间空间、内容分享等社区功能。两部分可以独立部署，也可以组合使用。

## 存储服务

存储服务负责卡片和箱子的云端存储。

文件存储提供卡片和箱子的持久化保存。用户上传的内容保存到云端存储，可以随时下载。存储服务支持多种存储后端，包括本地磁盘、网络存储、对象存储服务等。

文件版本管理保存文件的历史版本。用户可以查看文件的历史修改记录，可以恢复到之前的版本。版本历史帮助用户保护数据安全，防止误操作导致的内容丢失。

访问控制管理文件的访问权限。用户可以设置文件的公开或私有状态。私有文件只有授权用户可以访问。公开文件可以被所有社区用户浏览。

## 社区平台

社区平台提供用户交互和内容分享功能。

用户管理处理用户的注册和登录。用户可以创建个人主页，展示自己分享的内容。用户之间可以互相关注，形成社交关系。

房间空间是用户在社区中的组织单位。公开房间可以被其他用户浏览和访问。用户可以将本地箱子同步上传到社区房间，获得分享链接。

内容发现帮助用户找到感兴趣的内容。系统推荐根据用户的浏览历史推荐相关内容。搜索功能支持关键词检索。热门榜单展示最受欢迎的内容。

## 部署场景

社区服务器支持多种部署方式。

单机部署适合小规模使用场景。所有服务运行在一台服务器上，部署简单，维护成本低。适合个人或小团队自建使用。

分布式部署适合大规模使用场景。存储服务和社区平台可以分别部署到不同服务器。数据库可以独立部署提高性能。负载均衡分发请求到多个服务实例。

云部署支持主流云平台。可以使用云服务商提供的存储、计算、网络等服务。云部署弹性扩展，适应业务增长需求。

## 技术架构

社区平台采用微服务架构。

用户服务处理用户认证和基本信息管理。提供注册、登录、信息修改等接口。

内容服务处理内容的存储和检索。提供上传、下载、搜索等接口。

关系服务处理用户之间的关系。提供关注、粉丝、消息等接口。

通知服务处理实时通知推送。用户在社区中的活动可以实时通知到相关用户。

## 与本地生态的关系

社区服务器与本地软件协同工作。

本地内容可以同步上传到社区。用户选择要分享的箱子或卡片，一键同步到社区房间。同步保持本地和云端的内容一致。

社区内容可以下载到本地。用户浏览社区中的内容，可以下载到本地空间继续编辑。下载的内容保留原始的卡片格式。

数据隔离确保用户隐私。社区服务器不获取用户本地文件的完整内容，只同步用户主动分享的内容。
