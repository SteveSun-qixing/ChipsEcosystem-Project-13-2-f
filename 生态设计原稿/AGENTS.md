# 生态设计原稿 AGENTS

## 角色

- 这里保存产品设计、架构愿景、分层原则、能力边界和长期设计原稿。
- 这里是“设计意图”与“正式方向”的来源，不是开发日志或临时实现记录。

## 编写规则

- 只写架构原则、产品设计、能力边界、概念定义和正式设计路径，不写工单处理过程、临时兼容方案或阶段性任务报告。
- 修改前先核对相关代码和共用技术文档；如果发现设计与实现不一致，先登记工单，再决定是否更新原稿。
- 不要把项目内部的局部实现细节、仓库专有脚本或短期目录布局写成生态原稿结论。
- 若某设计已冻结并转化为生态正式标准，应在 `生态共用技术文档/` 同步形成对应公共文档。

## 质量要求

- 术语、分层和职责边界必须与其他原稿文档保持一致。
- 同一概念若已有正式定义，新增内容应引用现有定义，而不是创建第二套表述。
