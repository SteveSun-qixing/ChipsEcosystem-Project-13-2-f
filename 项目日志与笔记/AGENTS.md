# 项目日志与笔记 AGENTS

## 角色

- 这里保存工单、排查记录、研究笔记、阶段日志和过程性材料。
- 这里不是生态公共契约或正式对外技术标准的最终发布位置。

## 工单规则

- 新问题先追加到 `问题工单.md`，再创建 `工单NNN-描述/任务说明.md`。
- 工单说明至少写清：背景、证据、影响范围、复现步骤、期望行为、暂行处理、需要协作方和当前状态。
- 工单编号应保持递增；如果已有同主题未闭环工单，优先复用或明确补充关系。
- 当问题闭环后，要把处理结果和关键验证写回总工单或独立说明。

## 边界规则

- 不要把正式公共协议、共享接口标准、对外文档最终版长期只留在这里；需要沉淀为正式规范时，迁移或同步到 `生态共用技术文档/`。
- 不要直接删除历史记录；失效材料移入本目录下的 `归档/`。
