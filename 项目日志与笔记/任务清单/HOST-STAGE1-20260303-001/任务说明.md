# HOST-STAGE1-20260303-001 任务说明

## 1. 任务背景

本任务对应工单 `HOST-STAGE1-20260303-001`，用于闭环第一阶段项目一（薯片主机）在架构、能力与验收上的剩余缺口，目标是达到“完整可交付”的主机底座标准。

## 2. 问题分解

### 2.1 PAL 缺口

- 文件监控能力未落地（watch）。
- 剪贴板未覆盖图片/文件类型能力。
- 本地进程通信仅覆盖 Electron IPC，未覆盖命名管道/共享内存/Unix 域套接字抽象。

### 2.2 Kernel 缺口

- 模块加载器能力未形成独立闭环：
  - 多模块格式支持；
  - 隔离策略；
  - 依赖解析与版本约束。

### 2.3 Protocol 缺口

- 协议版本兼容策略缺失。
- 协议解析/路由/兼容/性能专项测试不足。

### 2.4 Bridge 缺口

- 默认全权限桥接上下文仍存在，不满足最小权限原则。
- `contextBridge.exposeInMainWorld` 主线接入未完全固化。

### 2.5 主进程骨架缺口

- 缺少全局异常托底（`uncaughtException` / `unhandledRejection`）闭环。
- Electron `app` 生命周期编排未形成完整可验收主线。

### 2.6 测试验收缺口

- 缺少跨平台一致性、压力、协议性能、故障注入专项验收。

## 3. 拟定交付清单

1. PAL 能力扩展与对应路由接入（含文档与测试）。
2. Kernel 模块加载器实现与契约测试。
3. Protocol 兼容策略与专项测试套件。
4. Bridge 权限模型收敛与 preload/contextBridge 固化。
5. 主进程入口与异常治理闭环。
6. 第一阶段验收矩阵测试报告（单元/集成/E2E/性能/故障）。

## 4. 风险与约束

- 该任务跨越 L1-L5 核心链路，属于高影响变更。
- 需严格遵循生态共用文档冻结口径，避免新增未约定接口。
- 禁止临时实现、占位实现，所有改动必须有测试与文档同步。

- 状态：已完成（闭环态）
