# 阶段八：验收收口与旧链路清退

## 1. 阶段目标

完成跨仓一致性验收、旧链路清退、文档归档和最终交付，确保模块插件在整个生态中的全链路已经稳定可用。

## 2. 入口条件

- 阶段七完成；
- Host / SDK / 脚手架 / 参考模块 / 调用方均已通过各自正式验证。

## 3. 任务拆解

### 3.1 全仓一致性复核

- 复核共享文档、项目级任务文档、代码实现和测试口径；
- 复核 Manifest、route manifest、schema、SDK 类型、脚手架模板；
- 复核调用方示例与参考模块示例。

### 3.2 旧链路清退

- 清退旧 `module.mount/unmount/query/list` 公开链路；
- 清退旧 React/slot 模块脚手架输出；
- 清退旧 README 和旧公共文档中的正向说明；
- 需要移出的旧文件按仓库规则移入 `归档/`，但不再参与正式运行和正式文档。

### 3.3 验收报告

- 输出最终任务完成情况；
- 列出剩余缺口和后续工单；
- 明确首版上线前的阻塞项是否全部关闭。

## 4. 最终验收清单

- 新模块可创建；
- 新模块可打包；
- 新模块可安装到 Host；
- Host 可注册 provider；
- 调用方可通过 SDK 调用；
- 模块可调用其他模块；
- 同步与异步 job 均可工作；
- 禁用、卸载、失败、取消和回收路径可验证；
- 公共文档与实现一致。

## 5. 验证门禁

- `Chips-Host`：`npm run build && npm test && npm run test:contract`
- `Chips-SDK`：`npm test`
- `chips-scaffold-module`：`npm run build && npm test && npm run test:templates && npm run test:e2e`
- 参考模块与接入方仓库的正式脚本全部通过

## 6. 阶段退出标准

- 模块插件全链路闭环成立；
- 旧公开模块链路不再作为正式方案存在；
- 可以进入后续常规迭代，而不是继续架构返工。
