# 大任务002：完成转换链路的开发

## 1. 任务目标

本大任务用于冻结并推进整个卡片转换链路的正式建设，首版范围固定为 4 个模块：

- `Chips-FileConversion-Plugin`
- `Chips-CardtoHTML-Plugin`
- `Chips-HtmltoPDF-Plugin`
- `Chips-HtmltoImage-Plugin`

整体目标不是只补单个导出按钮，而是完成一条从调用入口、模块编排、HTML 中间产物到 PDF / 图片输出的正式链路。

## 2. 当前已确认的正式基线

本轮开始前已按工作区要求核对以下现状：

- Host 已具备正式模块服务：`module.listProviders / module.resolve / module.invoke / module.job.*`
- Host 已具备正式卡片服务动作：`card.pack / card.unpack / card.readMetadata / card.parse / card.render / card.renderCover / card.renderEditor`
- `chips-scaffold-module` 已能生成 `type: module` 正式工程骨架
- `Chips-EditingEngine` 当前仅正式打通 `.card` 导出，HTML / PDF / 图片导出仍未接入

因此，本大任务现在可以先冻结“转换链路正式口径 + 各仓库文档与计划”，再进入具体实现。

## 4. 实施原则

- 对外只暴露统一文件转换入口模块；
- 原子转换模块只做单一映射，不做额外编排；
- `pdf` / `image` 必须复用 `card -> html` 的结果；
- HTML 显示效果必须与 Host 正式卡片渲染画面一致；
- 不引入新的插件类型，不引入临时协议，不保留占位实现作为正式链路。
