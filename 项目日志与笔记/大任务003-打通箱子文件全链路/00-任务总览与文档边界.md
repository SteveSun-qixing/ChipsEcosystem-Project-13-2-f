# 大任务003：打通箱子文件全链路

## 1. 任务目标

按已经冻结的生态设计原稿与生态共用技术文档，正式完成箱子文件全链路开发，使箱子在薯片生态中具备以下端到端能力：

1. Host 能以新口径解析与消费 `.box`；
2. SDK 能提供完整、类型化的箱子正式接口；
3. 布局插件仓能交付首个正式网格布局插件；
4. 查看器能正式打开并消费 `.box`；
5. 编辑引擎能正式创建、编辑、保存并预览 `.box`；
6. 整条链路在真实工作区、正式脚本和自动化验证下跑通。

## 2. 本任务不包含的范围

以下内容不在本大任务范围内：

1. 文件转换模块能力；
2. 向旧箱子模型兼容；
3. 回退机制、过渡字段、双写链路；
4. 临时样例实现、占位实现、单点手工演示脚本。

## 4. 已核对的真实代码基线

当前代码基线与正式目标之间存在以下明确差距：

### 4.1 Host

- `Chips-Host/packages/box-service/src/box-service.ts` 仍是旧 `cards + internal` 结构；
- `Chips-Host/src/main/services/register-host-services.ts` 只注册了 `box.pack / box.unpack / box.inspect`；
- `Chips-Host/src/main/services/register-schemas.ts` 尚未登记 `box.openView` 等运行时动作；
- `Chips-Host/src/main/core/file-association.ts` 对 `.box` 仍只有 `box.inspect` 级别预检查。

### 4.2 SDK

- `Chips-SDK/src/api/box.ts` 仍暴露 `full/shell/mixed` 与 `internal/external` 旧类型；
- `Chips-SDK/src/contracts/route-manifest.json` 只有三条旧箱子动作。

### 4.3 编辑引擎

- `Chips-EditingEngine/src/services/workspace-service.ts` 仍用“目录态 `.box`”创建箱子；
- 当前没有正式的箱子工作目录会话、条目管理、布局编辑与保存链路；
- 编辑引擎已有基础卡片插件动态加载模式，可作为布局插件宿主实现参考。

### 4.4 查看器

- `Chips-CardViewer/src/App.tsx` 当前仍是纯卡片查看入口；
- `Chips-CardViewer/manifest.yaml` 目前只声明 `file-handler:.card`；
- 没有 `.box` 会话管理与布局插件宿主层。

### 4.5 布局插件仓

- `Chips-BoxLayoutPlugin/` 当前为空目录，尚无正式工程、清单、测试和网格布局实现。
