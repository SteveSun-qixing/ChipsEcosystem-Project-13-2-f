# 工单058-Host安装包与文件关联入口未闭环

## 背景

本轮按“用户下载安装包 -> 安装 Host -> 双击 `.card` / `.box` 文件 -> 进入查看器或对应应用插件”的真实用户路径检查全生态协同时，发现 Host 侧虽然已经具备 Node/CLI 版文件关联处理能力，但 Electron 面向最终用户的入口并没有把这条链路正式接起来。

## 证据

1. `Chips-Host/src/main/core/file-association.ts` 已实现 `openAssociatedFile(...)`，可以：
   - 识别 `.card/.box`
   - 预解析对应文件
   - 找到声明了 `file-handler:<ext>` 的应用插件
   - 走 `plugin.launch` 打开目标插件窗口
2. `Chips-Host/src/main/cli/index.ts` 的 `open` 子命令会调用这条能力，因此 `chips open <file>` 可工作。
3. `Chips-Host/src/main/electron/app-entry.ts` 当前只处理：
   - `--workspace=...`
   - `--chips-launch-plugin=...`
   - `--chips-launch-payload=...`
4. `app-entry.ts` 没有处理：
   - 直接传入的 `.card/.box` 路径
   - macOS `open-file`
   - 其他系统级文件关联回调
5. 自动化也只覆盖 CLI 路径：
   - `Chips-Host/tests/e2e/cli.test.ts` 验证的是 `runCli(['open', cardFile])`
   - 并非“安装后 Host App 被系统拉起并打开文件”

## 影响范围

- 当前不能证明“用户安装 Host 后双击 `.card` 就能直接进入卡片查看器”。
- 即便查看器在拿到 `launchContext.launchParams.targetPath` 后可以正常渲染，也无法保证系统级入口能把该参数真正送到查看器。
- 这会直接影响安装分发、桌面文件关联和系统级快捷打开体验。

## 复现步骤

1. 核对 `Chips-Host/src/main/core/file-association.ts` 与 `Chips-Host/src/main/cli/index.ts`；
2. 核对 `Chips-Host/src/main/electron/app-entry.ts`；
3. 发现 Electron 主入口缺少文件路径与 `open-file` 处理；
4. 核对 `Chips-Host/tests/e2e/cli.test.ts`，确认当前只验证 CLI `open`。

## 期望行为

1. Electron 主入口接入系统级文件关联事件；
2. Host 安装后双击 `.card/.box` 文件时，能统一落到正式 `openAssociatedFile(...)` / `plugin.launch` 链路；
3. 为“安装后双击文件打开目标应用插件”补齐正式验收脚本。

## 暂行处理

- 当前仅将 `chips open <file>` 视为开发/CLI 可用；
- 不把它等同于“最终用户安装 Host 后系统文件关联已闭环”。

## 需要协作方

- Host Electron 入口维护方
- 安装分发维护方
- 查看器维护方
- 测试链路维护方

## 当前状态

部分闭环（2026-03-20复核）

## 处理结果（2026-03-19）

1. `Chips-Host/src/main/electron/app-entry.ts` 已补齐系统级文件关联入口收口：
   - 解析进程参数中的 `.card/.box` 文件路径；
   - 继续保持 `--chips-launch-plugin` 应用快捷方式启动链路不变；
   - 接入 macOS `open-file` 事件并在 Host 运行时就绪后统一落到正式文件关联处理链路；
   - 接入 `second-instance` 参数转发，使已运行 Host 能接收后续文件打开请求。
2. Electron 主入口对系统级文件打开请求统一复用 `openAssociatedFile(...)`，不新增旁路实现；
3. 已补充正式回归测试 `Chips-Host/tests/unit/app-entry.test.ts`，覆盖：
   - 启动参数中的 `.card` 文件；
   - 应用快捷方式参数优先级；
   - macOS `open-file` 预启动排队；
   - `second-instance` 文件打开转发。
4. 已按 Host 高风险链路要求完成验证：
   - `npm run build`
   - `npm test`
   - `npm run test:contract`
5. 结论：Host 安装后的 Electron 主入口已正式接入 `.card/.box` 文件关联收口链路，本工单阻断已解除。

1. 已确认 `Chips-Host/src/main/electron/app-entry.ts`、`Chips-Host/src/main/core/file-association.ts` 与 `Chips-Host/tests/unit/app-entry.test.ts` 中的入口代码和单元测试均已落地；

   - `openAssociatedFile(...)` 在 `.box` 未命中处理插件时仍只会创建空白原生窗口，不会自动进入正式查看器；
3. 因此本工单当前只能判定为“Electron 主入口代码侧已收口”，不能再将其表述为整体已闭环；
