## 2026-03-26｜编辑引擎白屏修复与主题图标字体资源链路补齐

### 1. 问题现象

1. 使用 `chipsdev run` 启动 `Chips-EditingEngine` 后，界面白屏不可用；
2. 控制台出现 React `Minified React error #31`，错误对象键为 `{ name, decorative, opsz }`；
3. 同时出现 `MaterialSymbolsOutlined[FILL,GRAD,opsz,wght].woff2` 资源 `ERR_FILE_NOT_FOUND`。

### 2. 根因定位

1. `Chips-EditingEngine/src/components/BaseWindow/ToolWindow.tsx` 仍把 `config.icon: IconDescriptor` 直接作为 React 子节点渲染，导致窗口初始化阶段直接抛错并白屏；
2. Host 主题运行时把 `dist/theme.css` 文本直接注入 preload 文档，但未重写其中的相对 `url(...)` 资源路径，导致图标字体在内联样式场景下失去原始相对路径基准并加载失败。

### 3. 本轮修复

1. 编辑引擎：
   - `ToolWindow` 头部图标改为统一使用 `RuntimeIcon` 渲染；
   - 新增 `tests/unit/tool-window.test.tsx`，覆盖 `IconDescriptor` 进入工具窗口头部时不会再被错误当作对象子节点渲染。
2. Host：
   - 新增 `src/main/theme-runtime/css-assets.ts`；
   - 在主题加载阶段引入 `rewriteThemeCssAssetUrls(...)`，把主题 CSS 中的相对资源路径重写为绝对 `file://` URL；
   - 新增 `tests/unit/theme-css-assets.test.ts`，覆盖相对路径重写、绝对路径保留、query/hash 保留三类情况。

### 4. 验证结果

1. `npm test --prefix Chips-EditingEngine -- tool-window` 通过；
2. `npm test --prefix Chips-Host -- theme-css-assets` 通过。

### 5. 当前结论

1. 本次白屏的直接崩溃点已修复；
2. 图标字体资源在主题 CSS 内联注入场景下的解析缺口已从 Host 根链路补齐；
3. 本轮修复属于生态正式图标系统链路问题修复，不引入兼容层或回退实现。

### 3. 当前剩余事项

1. `design-assets/Appicon/` 中基础卡片与布局插件源文件命名仍可继续收口，避免未来新增资产时再次漂移。
