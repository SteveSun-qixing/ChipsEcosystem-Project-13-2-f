# 阶段一：图标公共契约与文档基线冻结

## 1. 阶段目标

在开始大规模改代码前，先冻结图标系统的唯一正式口径，解决共享文档内部冲突，明确实施边界，避免后续各仓继续按旧图标模型扩散。

## 2. 涉及范围

1. `生态共用技术文档/主题系统`
2. `生态共用技术文档/组件库`
3. `生态共用技术文档/插件开发`
4. `生态共用技术文档/协议与接口标准`
5. `生态设计原稿` 中与图标实施方向直接相关的文档
6. `项目日志与笔记/工单067-*`

## 3. 当前基线

1. 图标与字体系统设计规范已经冻结；
2. 主题包开发指南仍保留旧 SVG / `icons.css` 口径；
3. 卡片插件、布局插件、编辑引擎装配标准仍保留 `icon?: string`；
4. 设置治理文档尚未明确治理页图标展示的正式要求。

## 4. 核心任务

1. 统一 `ChipsIcon`、图标描述符和 ThemePack 图标 token 的公共定义；
2. 明确运行时 UI 图标与 `manifest.ui.launcher.icon` 的职责分层；
3. 明确 emoji 清零范围；
4. 明确“不做兼容、不做双写、不保留 fallback”的实施原则；
5. 清除共享文档中的 emoji 和旧图标描述；
6. 确认若设计原稿需要同步收口，应在核对代码与共享文档后再改。

## 5. 交付物

1. 更新后的共享文档清单；
2. 工单067回写补充；
3. 本大任务阶段日志。

## 6. 验证

1. 共享文档之间不再出现两套图标口径；
2. 文档中不再保留“运行时 UI 图标用 SVG 类名，另一路再做 Material Symbols”的双轨叙述；
3. 正式文档搜索结果中不再出现阶段目标明确要求移除的 emoji。

## 7. 完成判定

只有当共享文档能够明确回答以下问题时，本阶段才算完成：

1. 统一图标组件是什么；
2. 统一图标输入模型是什么；
3. ThemePack 需要提供哪些图标能力；
4. Host 继续负责哪些系统级图标文件能力；
5. 哪些旧口径必须被直接替换且不保留兼容。
