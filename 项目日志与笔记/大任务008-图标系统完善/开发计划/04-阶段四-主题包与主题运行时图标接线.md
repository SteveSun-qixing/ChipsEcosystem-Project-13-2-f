# 阶段四：主题包与主题运行时图标接线

## 1. 阶段目标

让默认主题包与暗色主题包真正接入运行时图标资源和图标 token，使图标视觉参数由主题系统正式控制。

## 2. 涉及范围

1. `ThemePack/Chips-default`
2. `ThemePack/Chips-theme-default-dark`
3. `Chips-Scaffold/chips-scaffold-theme`
4. 主题系统相关共享文档

## 3. 当前基线

1. 两个正式主题包都只有 `icons/README.md`；
2. 主题包尚未提供真正的图标资源接入；
3. 默认主题字体基线与冻结规范仍需复核。

## 4. 核心任务

1. 为两个正式主题包接入运行时图标资源；
2. 补齐图标 token 默认值；
3. 补齐图标字体声明、变量轴默认值和主题映射；
4. 更新主题模板；
5. 更新主题包相关正式文档。

## 5. 交付物

1. 可被运行时正式消费的主题包图标资源；
2. 图标 token 默认值；
3. 主题模板更新；
4. 文档更新。

## 6. 验证

1. 默认主题与暗色主题切换后，图标视觉参数能够按主题变化；
2. 主题包目录不再只剩 README 占位；
3. 主题模板生成的新主题工程具备正式图标结构。

## 7. 完成判定

只有当主题系统可以真实控制运行时图标，而不是停留在文档说明层，本阶段才算完成。
