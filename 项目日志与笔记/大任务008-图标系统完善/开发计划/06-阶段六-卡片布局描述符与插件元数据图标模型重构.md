# 阶段六：卡片布局描述符与插件元数据图标模型重构

## 1. 阶段目标

把基础卡片插件、布局插件和编辑引擎内部描述符中的旧 `icon: string` / emoji 口径升级为正式语义图标描述符。

## 2. 涉及范围

1. `Chips-BaseCardPlugin/*`
2. `Chips-BoxLayoutPlugin`
3. `Chips-EditingEngine` 中基础卡片、布局清单和卡箱库相关类型
4. 卡片插件、布局插件、编辑引擎装配标准文档

## 3. 当前基线

1. `BasecardDefinition.icon?: string` 仍是旧字段；
2. 基础卡片插件当前以 emoji 表示显示图标；
3. 编辑引擎 `CardTypeDefinition` / `LayoutTypeDefinition` 仍使用 `icon: string`；
4. 卡箱库默认值仍会回退到 emoji。

## 4. 核心任务

1. 定义插件元数据图标正式类型；
2. 升级基础卡片、布局插件和编辑引擎内部描述符；
3. 替换 richtext/image 基础卡片的 emoji 图标；
4. 给布局插件补齐正式图标输出；
5. 更新相关文档和测试。

## 5. 交付物

1. 新的图标数据模型；
2. 基础卡片与布局插件元数据重构；
3. 卡箱库图标数据源重构。

## 6. 验证

1. 卡箱库能够从正式描述符展示图标；
2. 基础卡片与布局插件不再输出 emoji；
3. 相关类型检查与插件测试通过。

## 7. 完成判定

只有当“插件定义 -> 编辑引擎描述符 -> UI 展示”全链路都不再依赖旧 `icon: string` 模型时，本阶段才算完成。
