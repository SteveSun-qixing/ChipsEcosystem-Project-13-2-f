# 阶段八：编辑引擎全量图标替换与 Emoji 清退

## 1. 阶段目标

一次性完成 `Chips-EditingEngine` 的旧图标替换、emoji 清退、类型重构和相关文档测试更新，彻底消除该仓中散落的字符图标实现。

## 2. 涉及范围

1. 程序坞
2. 顶栏 / 布局切换 / 窗口菜单
3. 文件管理器 / 右键菜单 / 拖拽 / 空态 / 加载态
4. 卡箱库 / 历史面板 / 编辑面板 / 卡片窗口
5. 设置面板 / CoverMaker / 导出面板 / ThemePanel
6. 编辑器运行时错误态与占位态
7. 编辑引擎内部相关类型、样式、文档、测试

## 3. 当前基线

1. 当前搜索到 76 处 emoji / 旧字符图标；
2. 多数图标直接写在 JSX 字符串中；
3. 多个内部数据结构仍使用 `icon: string`；
4. 旧图标与业务逻辑混写，重构时需同步整理组件边界。

## 4. 核心任务

1. 把全部 emoji 图标替换为正式图标组件；
2. 将菜单、面板、状态位的图标数据模型改为正式图标描述符；
3. 统一空态、错误态、加载态图标样式与可访问性；
4. 清理旧字符箭头、锁定、主题切换等图标写法；
5. 更新相关样式、测试与项目文档。

## 5. 交付物

1. 全量图标替换后的编辑引擎代码；
2. `icon: string` 旧模型替换结果；
3. emoji 清零结果；
4. 测试与文档更新。

## 6. 验证

- `cd Chips-EditingEngine && npm run build && npm test && npm run lint && npm run validate`
- `rg -n -P '[\\p{Extended_Pictographic}]' Chips-EditingEngine`

## 7. 完成判定

只有当编辑引擎搜索结果中的 emoji 清零、所有图标入口统一接入正式图标体系且正式脚本通过时，本阶段才算完成。
