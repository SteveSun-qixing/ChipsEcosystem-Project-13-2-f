# 任务002-阶段六：联调、CI 与验收闭环

## 1. 背景

在前几个阶段完成命令行拆分、Host/SDK CLI 实现、生态仓库适配以及文档更新后，需要通过一轮系统性的联调与验收，确保整个生态在实际使用场景下工作正常。

本阶段目标是通过端到端流程验证、CI 门禁完善和验收记录，确认“两个命令：`chips` + `chipsdev`”的方案在工程与文档层面都已经完全落地。

## 2. 目标

1. 通过至少一条完整的“从零开始”的端到端路径验证：
   - 安装 Host 与 SDK；
   - 使用 `chipsdev` 创建工程、开发、构建、测试、验证、打包；
   - 使用 `chips` 安装并运行生成的插件/应用；
   - 在 Host 内验证主题/组件/功能正常工作。
2. 在主要仓库中建立稳定的 CLI 相关 CI 门禁：
   - 任一影响 CLI 契约的变更都会被 CI 抓住；
   - Host/SDK/脚手架/主题包/应用模板均有至少一条 CLI 冒烟测试流水线。
3. 形成任务闭环报告：
   - 罗列所有受影响仓库与文档；
   - 说明测试覆盖范围与遗留风险（如有）。

## 3. 范围

### 3.1 在范围内

- Host 与 SDK 仓库的联调；
- 脚手架、主题包、CardViewer 等代表性项目的端到端验证；
- 生态级 CI 配置与质量门禁；
- 任务 002 的整体验收记录。

### 3.2 不在范围内

- 对未来新引入仓库的命令行规范（新仓库应直接遵循已有契约与模板）。

## 4. 产出物

1. 联调与验收记录文档（可单独存放或附在本阶段文档末尾）：
   - 实际执行的端到端用例；
   - 执行环境说明（OS、Node 版本等）；
   - 运行日志与结果总结。
2. 更新后的 CI 配置：
   - 各主要仓库的 CLI 冒烟测试 job；
   - 失败时的报警策略（可按现有体系处理）。
3. 任务 002 闭环结论：
   - “命令行拆分已在整个生态落地”的确认说明；
   - 若存在未来需要跟进的优化点，记录为后续任务或工单。

## 5. 任务拆解

### 5.1 端到端场景设计

- 至少设计两类典型场景：
  1. **应用插件路径**：
     - 使用 `chipsdev create` 创建一个应用插件工程（例如基于 CardViewer 或简单应用模板）；
     - `chipsdev dev/server` 本地调试；
     - `chipsdev test/lint/validate`；
     - `chipsdev build/package` 产出 `.cpk`；
     - 使用 `chips plugin install` 安装；
     - 使用 Host 启动并访问该应用。
  2. **主题包路径**：
     - 使用 `chipsdev create` 创建主题包工程；
     - 完成最小修改与构建；
     - 使用 `chips theme validate` 校验契约；
     - 在 Host 中应用该主题验证界面正常。
- 根据需要，可补充卡片插件、布局插件等其它路径，但至少确保上述两条典型路径全绿。

### 5.2 手工联调执行

- 在一台“尽量干净”的开发机或容器环境中执行：
  - 安装 Host（提供 `chips`）；
  - 安装 SDK/CLI（提供 `chipsdev`）；
  - 按 5.1 中的场景逐步执行，并记录每一步的命令与结果。
- 记录所有发现的问题并回流到对应阶段处理：
  - 若属于 Host/SDK CLI 实现问题，回到阶段二或三；
  - 若属于仓库适配问题，回到阶段四；
  - 若属于文档与实际行为不一致，回到阶段五。

### 5.3 CI 门禁完善

- 为以下仓库至少各增加一条 CLI 冒烟测试 job：
  - `Chips-Host`：验证 `chips` 关键命令；
  - `Chips-SDK`/`chipsdev` 仓库：验证 `chipsdev` 核心命令；
  - 一个脚手架仓库（如 `chips-scaffold-theme`）：验证脚手架与 `chipsdev` 的协作；
  - `ThemePack/Chips-default`：验证主题构建与 Host 校验链路；
  - `Chips-CardViewer` 或类似应用模板：验证应用构建与运行。
- 确认：
  - 对 CLI 接口有破坏性变更时，至少一个以上仓库的 CI 会失败；
  - 版本升级流程中包含 CLI 冒烟测试环节。

### 5.4 验收与归档

- 汇总：
  - 所有受影响仓库列表；
  - 更新的文档列表；
  - 端到端用例与 CI job 列表；
  - 遗留问题与后续建议。
- 在 `项目日志与笔记/问题工单.md` 中记录任务 002 的最终状态与关键结论。

## 6. 验收标准

- 手工联调用例全部通过，无阻断性问题；
- 主要仓库的 CLI 冒烟测试 CI 运行稳定；
- 文档与实际行为一致，命令示例可直接复制执行；
- 在任务记录中明确标记“任务002-拆分命令行”已完成并归档。
