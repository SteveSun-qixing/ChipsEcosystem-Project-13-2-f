# 工单014：卡片查看器外层壳子主题变量链路未接通且存在写死深色兜底

## 1. 背景

用户在未安装主题包时预期查看器应回到官方默认白色主题，而当前查看器界面呈现黑色。经排查，问题集中在查看器应用壳层本身，而不是卡片内容 iframe。

## 2. 现场证据

### 2.1 查看器壳层写死深色 fallback

`Chips-CardViewer/src/components/CardViewerShell.tsx` 当前使用：

- `background: var(--chips-sys-color-surface, #101014)`
- `color: var(--chips-sys-color-on-surface, #f5f5f5)`

只要 `--chips-sys-color-*` 没有被注入，整个查看器壳层就会稳定退回黑底白字。

### 2.2 查看器未真正完成主题运行时接入

`Chips-CardViewer/src/App.tsx` 仅包裹了：

- `ChipsThemeProvider themeId="chips-official.default-theme" version="1.0.0"`

但当前接入缺少正式主题变量来源：

- 没有传入 `resolver`；
- 没有传入 `eventSource={window.chips}`；
- 没有在应用层执行变量注入。

根据组件库文档与 hooks 实现：

- `ChipsThemeProvider` 主要维护 theme runtime 状态与事件订阅；
- 变量注入应通过 `applyThemeVariables(...)` 或正式 resolver/运行时链路完成；
- 文档示例要求通过 `resolver` 与 `theme.changed` 事件把主题系统接入应用。

因此当前查看器并没有真正获得主题变量。

### 2.3 默认主题本身并非黑色

`ThemePack/Chips-default/theme.css` 明确声明：

- `color-scheme: light`
- `background-color: var(--chips-sys-color-surface, #ffffff)`

说明官方默认主题本应是白色浅色基线，而不是黑色。

## 3. 根因判断

这是“查看器应用层主题接入未完成 + 壳层 fallback 方向错误”的组合问题：

1. 查看器没有真正接上 Theme Runtime -> 变量注入链路；
2. 壳层 fallback 还写成了深色；
3. 因此在主题变量缺失时，界面被稳定渲染为黑色，与默认主题设计目标相反。

## 4. 影响范围

- `Chips-CardViewer` 第一视觉错误，直接偏离默认主题设计；
- 用户会误以为默认主题是黑色或主题包逻辑异常；
- 查看器应用层无法作为“正确接入主题系统”的生态示范工程；
- 后续其他应用若复制该壳层代码，会扩散错误接入方式。

## 5. 复现步骤

1. 启动 `Chips-CardViewer` 当前工程。
2. 在未完成主题变量注入的情况下打开应用窗口。
3. 观察查看器外层壳子为黑底白字。
4. 对照 `ThemePack/Chips-default/theme.css`，可确认官方默认主题本应为白底浅色。

## 6. 期望行为

- 查看器外层壳子应以官方默认白色主题作为视觉基线；
- 在主题系统未切换时，壳层 fallback 也应与默认主题方向一致，而不是深色；
- 查看器必须通过正式 Theme Runtime 链路获取变量，不允许继续仅靠硬编码色值维持界面。

## 7. 建议处理方向

1. 将查看器壳层 fallback 调整为默认主题一致的白色基线；
2. 按组件库与主题系统规范接入正式 `resolver` / `theme.changed` / 变量注入链路；
3. 避免在应用层继续写死系统色值，尤其避免与默认主题方向相反的 fallback；
4. 为查看器补充主题接入回归测试，覆盖：
   - 未切换主题时默认白色；
   - 主题变更事件触发后变量刷新；
   - 无 resolver / 无变量注入时的防退化行为。

## 8. 正式处理结果

- 查看器 manifest 已声明 `ui.window.chrome` 白色窗口基线；
- Host preload 已统一向插件窗口注入主题 CSS 与解析后的主题变量；
- 查看器壳层、拖拽区、加载态与错误态均改为消费主题变量，不再保留深色默认兜底。

## 9. 需要协作方

- CardViewer 应用仓库
- Host Theme Runtime / theme 服务维护方
- 组件库主题 hooks 维护方

## 10. 当前结论

查看器主题变量链路已打通，默认外层壳子已回归官方白色主题基线，且各反馈态统一消费正式主题变量。
