# 工单018：全局 `chips` 命令无法装载刚安装的主题插件，导致主题切换失败

## 1. 背景

用户明确要求：

- 打包暗夜主题包；
- 使用正式 `chips` 命令安装主题包；
- 使用正式 `chips` 命令切换到该主题包。

当前已完成暗夜主题包开发，主题包路径为：

- `/Users/sevenstars/Documents/ChipsCard/Develop/Project-13-2-f/ThemePack/Chips-theme-default-dark`

正式打包产物为：

- `/Users/sevenstars/Documents/ChipsCard/Develop/Project-13-2-f/ThemePack/Chips-theme-default-dark/dist/theme.theme.chips-official-default-dark-theme-1.0.0.cpk`

## 2. 现场事实

### 2.1 打包成功

在主题工程目录执行：

```bash
node ../../Chips-SDK/cli/index.js package
```

成功输出：

- `theme.theme.chips-official-default-dark-theme-1.0.0.cpk`

并确认包内包含：

- `manifest.yaml`
- `contracts/theme-interface.contract.json`
- `contracts/theme-min-functional-set.json`
- `dist/theme.css`
- `dist/tokens.json`

说明主题包结构符合当前仓库内正式打包契约。

### 2.2 全局 `chips` 安装与启用成功

在仓库根目录执行：

```bash
chips plugin install /Users/sevenstars/Documents/ChipsCard/Develop/Project-13-2-f/ThemePack/Chips-theme-default-dark/dist/theme.theme.chips-official-default-dark-theme-1.0.0.cpk
chips plugin enable theme.theme.chips-official-default-dark-theme
```

返回结果分别为：

- `pluginId: "theme.theme.chips-official-default-dark-theme"`
- `{ "ok": true }`

说明全局 `chips` 至少完成了插件安装记录与启用动作。

### 2.3 全局 `chips` 主题服务未识别新主题

继续执行：

```bash
chips theme list
chips theme current
chips theme apply chips-official.default-dark-theme
```

实际现象：

- `chips theme list` 仍只返回旧的 `chips-official.default-theme`；
- `chips theme current` 仍显示旧默认主题；
- `chips theme apply chips-official.default-dark-theme` 返回 `THEME_NOT_FOUND`。

这说明“插件安装成功”并没有进入当前全局 `chips` 所使用的主题运行时装载结果。

### 2.4 仓库内当前 Host CLI 链路是正常的

切换到仓库内 Host 工程后，执行：

```bash
npx tsx src/main/cli/index.ts plugin install <cpk路径>
npx tsx src/main/cli/index.ts plugin enable theme.theme.chips-official-default-dark-theme
npx tsx src/main/cli/index.ts theme list
npx tsx src/main/cli/index.ts theme apply chips-official.default-dark-theme
npx tsx src/main/cli/index.ts theme current
npx tsx src/main/cli/index.ts theme validate
```

全部成功，且：

- `theme list` 能看到 `chips-official.default-dark-theme`
- `theme apply` 成功
- `theme current` 返回 `chips-official.default-dark-theme`
- `theme validate` 通过

因此可确认：

1. 主题包自身没有结构问题；
2. 当前仓库代码中的 Host 主题装载逻辑是可工作的；
3. 问题出在全局 `chips` 实际执行链路与仓库当前代码不一致。

## 3. 初步根因判断

高概率原因是以下之一：

1. 全局 `chips` 二进制对应的是旧版 Host/Theme Runtime；
2. 全局 `chips` 使用的运行时工作区或装载逻辑与当前仓库代码不一致；
3. 全局 `chips` 的 `theme.list/theme.apply` 仍停留在旧主题模型，未读取正式主题插件记录；
4. 全局命令与仓库内 `Chips-Host` / `Chips-SDK` 的版本已发生错位。

## 4. 影响范围

- 当前用户无法按正式命令链路切换到暗夜主题；
- 主题包发布验收在全局 `chips` 环境中不可信；
- 生态会出现“代码已修复，但用户安装环境仍不可用”的交付断层。

## 5. 期望结果

- 全局 `chips plugin install` + `chips plugin enable` 后，`chips theme list` 必须能发现新主题；
- `chips theme apply chips-official.default-dark-theme` 必须成功；
- 全局 `chips` 的 Host/Theme Runtime 版本与仓库当前正式代码保持一致。

## 6. 建议排查方向

1. 确认全局 `chips` 命令实际指向的代码版本与构建产物路径；
2. 对比全局 `chips` 与仓库内 `Chips-Host/src/main/cli/index.ts` 的主题命令实现差异；
3. 核查全局 `chips` 运行时读取的工作区、`plugin-runtime.json`、主题发现逻辑；
4. 确认是否存在旧版内置默认主题兜底，绕过了已安装主题插件加载；
5. 若全局分发版本落后，需要先更新全局 `chips` 再重测暗夜主题安装与切换。

## 7. 当前结论

本次“使用正式 `chips` 命令安装并切换暗夜主题”的任务被生态链路问题阻断。

按照仓库规范，已登记工单并停止继续推进；等待该工单处理完成后，再继续进行正式主题安装与切换验收。

## 8. 正式处理结果

### 8.1 根因确认

最终确认，问题根因不是：

- 主题包 `manifest.yaml` 字段错误；
- Theme Runtime 主题发现算法错误；
- `chips theme.*` 服务实现错误；
- 暗夜主题包构建产物缺失。

真正根因是：

1. 生态根 `package.json` 只声明了 workspaces 和 Volta 版本；
2. 但没有显式声明根级工具依赖 `chips-host` 与 `chips-sdk`；
3. 在该状态下，Volta 的 `chips` shim 在生态根目录不会稳定命中当前工作区的本地 Host CLI；
4. 结果是 `chips` 回退到了旧的全局工具版本，输出与当前仓库 `Chips-Host` 代码不一致；
5. 因而表现为：
   - `chips plugin install` / `enable` 看似成功；
   - `chips theme list` / `apply` 却仍停留在旧主题模型。

### 8.2 正式修复

已完成以下修复：

1. 在生态根 `package.json` 中补齐：
   - `devDependencies.chips-host: "0.1.0"`
   - `devDependencies.chips-sdk: "0.1.0"`
2. 执行根工作区 `npm install`，使根级工具依赖与 lockfile 正式收敛；
3. 增强 `scripts/verify-ecosystem-workspace.mjs`：
   - 新增根级工具依赖校验；
   - 将 `chips-host` / `chips-sdk` 根依赖视为正式生态门禁；
4. 同步更新生态共用文档：
   - 开发环境配置指南；
   - 命令行拆分与使用指南；
   - Chips Host 命令行使用手册；
   - Chips Dev 开发者命令行手册。

### 8.3 验证结果

已完成以下验证：

#### 根目录命令解析

在生态根目录直接执行：

```bash
volta which chips

```

均已解析到：

- `/Users/sevenstars/Documents/ChipsCard/Develop/Project-13-2-f/node_modules/.bin/chips`
- `/Users/sevenstars/Documents/ChipsCard/Develop/Project-13-2-f/node_modules/.bin/chipsdev`

说明根目录命令不再回退到旧的全局工具版本。

#### 正式主题链路验证

在隔离 `CHIPS_HOME` 工作区中，使用正式 `chips` 命令完整执行：

```bash
chips plugin install <cpk>
chips plugin enable theme.theme.chips-official-default-dark-theme
chips theme list
chips theme apply chips-official.default-dark-theme
chips theme current
chips theme validate
```

全部成功，且：

- `chips theme list` 能正确列出 `chips-official.default-dark-theme`
- `chips theme apply` 成功
- `chips theme current` 返回暗夜主题
- `chips theme validate` 通过

## 9. 当前结论

工单已正式闭环。

当前生态中，`chips` 与 `chipsdev` 在根工作区与子工作区的解析约束已统一为：

- 根工作区负责声明正式工具依赖；
- 子工作区通过 `volta.extends` 继承根工具链；
- 所有命令优先命中当前生态工作区实现，而不是旧的全局工具版本。
