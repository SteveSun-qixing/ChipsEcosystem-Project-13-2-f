## 4. 根因判断

当前阻塞是一个组合型缺口，而不是单点 bug：

### 4.1 脚手架缺口

模块插件脚手架仓库没有建立完成，导致“正式项目从哪里起”的问题就没有答案。

### 4.2 Host 契约缺口

### 4.3 SDK 封装缺口

即便 Host 后续补了路由，如果 SDK 不同步补齐封装，应用层和插件层仍然没有正式调用入口。

### 4.4 导出链路未闭环

编辑引擎导出 `.card`、导出 HTML / PDF / 图片，本应是一条正式链路；当前却仍停留在 stub 阶段，说明基础设施尚未打通。

### 4.5 文档与仓库状态漂移

共用文档、脚手架文档、设计原稿和代码实现之间已经出现多处漂移：

- 文档说有模块脚手架，仓库没有；
- 文档说卡片服务有 pack/unpack/readMetadata，Host 对外动作没有；
- 设计说有文件转换系统，Host/SDK 没有正式接口；
- 编辑引擎 UI 已暴露导出入口，但实现仍然是 stub。
