# 工单063-SDK复合卡片iframe使用srcdoc继承宿主CSP导致渲染阻断

- 日期：2026-03-22
- 发现阶段：新系统下复测卡片查看器，`jsdom/esbuild` 依赖问题修复后继续打开 `.card`
- 问题类型：SDK 显示链路缺陷 / 文档加载模型错误 / CSP 继承导致 Host 正式渲染被阻断

## 背景

在安装包已补齐 `jsdom` 与 `esbuild` 后，卡片查看器再次打开 `.card`。最初排查确认，旧实现使用 `srcdoc` 直接继承宿主 CSP，导致 Host 正式内联启动脚本被阻断；第一次修复虽然把顶层 `srcdoc` 改成了 `blob:`，但在真实 Electron `file://` 壳层里继续复测后，`blob:` 复合文档中的内联脚本仍被宿主 `script-src 'self'` 阻断，问题并未真正闭环。

## 证据

1. 查看器控制台显示：
   - `card.render` 成功
   - 复合 iframe 已挂载
   - `chips.composite:ready` 已触发
2. 浏览器随后报错：

```text
Executing inline script violates the following Content Security Policy directive 'script-src 'self''
```

3. 第一阶段核对当前实现确认：
   - `Chips-SDK/src/api/card.ts` 当时通过 `iframe.srcdoc = body` 挂载 Host 返回的 HTML；
   - `Chips-CardViewer/index.html` 明确声明 `script-src 'self'`；
   - `srcdoc` 文档会继承父文档 CSP，因此 Host 复合卡片文档中的正式内联启动脚本被直接拦截。
4. 第二阶段继续复测确认：
   - 即使把顶层 `srcdoc` 改成 `blob:`，真实 Electron `file://` 宿主里仍会出现 `Executing inline script violates the following Content Security Policy directive 'script-src 'self''`；
   - 说明仅靠 `blob:` 包装仍不足以构成稳定隔离的正式文档上下文。
5. 继续复核 `Chips-Host/packages/card-service/src/card-service.ts` 后进一步确认：复合文档内部的基础卡片 iframe 也必须改成独立文件文档会话，不能继续依赖运行时 `blob:` / `srcdoc` 注入。

## 影响范围

- 卡片查看器中的复合卡片窗口无法正常执行 Host 生成的复合文档与编辑器文档启动脚本；
- 任何同样通过 SDK 正式 iframe 装载链路消费 Host HTML 视图、且应用壳层 CSP 禁止内联脚本的宿主，都会复现同类问题；
- 旧实现会让“Host 已成功渲染，但应用仍显示空白”成为系统性隐患。

## 复现步骤

1. 在应用壳层设置 `script-src 'self'` 的正式 CSP；
2. 调用 `client.card.compositeWindow.render(...)` 或 `client.card.editorPanel.render(...)`；
3. 让 SDK 通过 `iframe.srcdoc` 挂载 Host 返回的 HTML；
4. 观察浏览器拦截内联脚本，复合卡片或编辑器无法完成正式启动。

## 期望行为

- SDK 必须把 Host 返回的 HTML 视图装载为独立文件文档上下文，而不是继续走 `srcdoc` 或二次包装 `blob:`；
- Host 复合文档内部的基础卡片 iframe 也必须改为 render session 内的独立文档 URL 挂载，不能在单卡层继续使用 `srcdoc` / 运行时 `blob:`；
- Host 应为复合卡片和编辑器文档返回正式 `documentUrl + sessionId`，由 SDK 直接写入 `iframe.src`；
- SDK 需要在资源释放与事件监听上保证这条链路稳定，并通过正式 `card.releaseRenderSession` 回收 render session。

## 暂行处理

- 已将 `Chips-Host/packages/card-service/src/card-service.ts` 切换为 render session 模型：
  - Host 先把单卡文档写入 session 下的 `nodes/*.html`
  - 再把复合文档 / 编辑器文档写入 session 根目录
  - 正式返回 `documentUrl` 与 `sessionId`
- 已将 `Chips-SDK/src/api/card.ts` 切换为直接消费 `view.documentUrl`，并在 `dispose/pagehide/DOM 移除` 时调用 `card.releaseRenderSession(sessionId)`；
- 已移除复合卡片与封面链路里的 `allow-same-origin`，避免继续触发沙箱逃逸警告；
- 已补齐 `Chips-SDK/tests/card.test.ts`，覆盖 `documentUrl/sessionId` 与 render session 释放；
- 已补齐 `Chips-Host/tests/unit/card-service-rendering.test.ts`，覆盖复合文档节点文件、资源基准地址与运行时脚本口径；
- 已同步更新 SDK 技术文档与生态共用显示链路文档。
