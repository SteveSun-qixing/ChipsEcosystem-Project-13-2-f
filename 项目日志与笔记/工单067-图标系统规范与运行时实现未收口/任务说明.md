# 工单067-图标系统规范与运行时实现未收口

## 背景

在核查薯片生态当前图标相关链路时，发现“应用启动图标”与“运行时 UI 图标”处于两种不同成熟度：

1. 应用启动图标链路已具备正式安装、复制、快捷方式创建能力；
2. 运行时 UI 图标系统虽然已有冻结规范、设计资产与局部 token，但尚未形成统一组件、主题参数与资源注册闭环。

根据《`生态共用技术文档/主题系统/03-图标与字体系统设计规范.md`》，图标系统正式方向已经冻结为：

- Material Symbols Variable Font；
- 统一 `ChipsIcon` 组件；
- 视觉参数统一走主题 token；
- 禁止在组件和业务页面直接硬编码图标 SVG path。

但当前代码与文档实际状态仍未达到该目标，因此需要先登记工单，避免后续项目继续沿着旧口径扩散。

## 证据

### 1. 正式规范已冻结

- `生态共用技术文档/主题系统/03-图标与字体系统设计规范.md`
  - 明确要求：
    - 使用 Material Symbols Variable Font；
    - 统一通过 `ChipsIcon` 输出；
    - 图标参数使用 `icon.color / icon.weight / icon.fill / icon.grade / icon.opticalSize / icon.size`；
    - 禁止硬编码 SVG path。

### 2. 统一图标组件尚不存在

- 在仓库根目录执行：

```bash
rg -n --hidden --glob '!node_modules' '\\bChipsIcon\\b' .
```

- 结果仅命中文档，不命中任何运行时代码，说明统一图标组件尚未落地。

### 3. 组件库当前仍直接硬编码默认 SVG

- `Chips-ComponentLibrary/packages/components/src/index.js`
  - 存在 `createGlyphIcon(...)`；
  - `getDefaultIconContent(...)` 直接写死：
    - `chevron-down`
    - `close`
    - `expand`
    - `collapse`
    - `calendar`
  - 同时固定：
    - `width="12"`
    - `height="12"`
    - `strokeWidth`
    - `viewBox`

这与“统一图标组件 + 主题 token 驱动参数”的正式规范不一致。

### 4. 图标 token 只落了一小部分

- `Chips-ComponentLibrary/packages/tokens/tokens/comp/date-time.json`
- `Chips-ComponentLibrary/packages/tokens/tokens/comp/empty-state.json`

当前只存在局部的 `icon.color`，没有：

- `icon.weight`
- `icon.fill`
- `icon.grade`
- `icon.opticalSize`
- `icon.size`

也没有全局图标 token 治理层。

### 5. ThemePack 没有正式接入图标资源

- `ThemePack/Chips-default/icons/README.md`
- `ThemePack/Chips-theme-default-dark/icons/README.md`

两个主题目录都只有说明文档，没有真正的图标字体、映射文件或资源注册实现。

### 6. 设计资产已落库，但运行时未消费

- `design-assets/material-symbols/variablefont/`

目录中已经存在：

- `MaterialSymbolsOutlined[FILL,GRAD,opsz,wght].woff2`
- `MaterialSymbolsRounded[FILL,GRAD,opsz,wght].woff2`
- `MaterialSymbolsSharp[FILL,GRAD,opsz,wght].woff2`
- 对应 `.codepoints` 与 `SOURCE.md`

说明设计资产准备工作已完成，但当前 Host / ThemePack / 组件库 / 应用运行时没有正式接线。

### 7. 设置面板已经拿到图标路径，但界面没有消费

- `Chips-EcoSettingsPanel/src/shared/runtime/settings-runtime-service.ts`
  - `listAppPlugins()` 会调用 `this.client.plugin.getShortcut(record.id)`；
  - `PluginShortcutRecord` 包含 `iconPath?: string`。
- `Chips-EcoSettingsPanel/src/features/app-plugins/AppPluginsPage.tsx`
  - 页面只展示名称、状态、快捷方式状态和操作按钮；
  - 没有渲染插件图标。

说明 Host/SDK 一侧已有部分能力，但应用展示链路没有接上。

### 8. 公共文档内部存在旧口径漂移

- `生态共用技术文档/主题系统/01-主题包开发指南.md`

仍然保留：

- `icons.css`
- `svg/`
- `icon-sm/icon-md/icon-lg`
- “图标使用 SVG 格式”的旧说明

这与冻结的 Material Symbols + `ChipsIcon` 规范不一致，属于公共文档漂移。

## 影响范围

- `Chips-ComponentLibrary`
  - 当前默认 icon fallback 机制可能继续扩散到更多组件；
- `ThemePack/*`
  - 主题无法正式控制图标字重、填充、grade、光学尺寸；
- `Chips-EcoSettingsPanel`
  - 插件治理页面尚未形成图标展示体验；
- 后续编辑引擎、查看器、程序坞、工具窗口等
  - 若继续开发图标位，将面临继续写本地 SVG 还是等待统一组件的分叉；
- `生态共用技术文档/主题系统`
  - 当前已存在两套图标口径并存，继续扩展会放大漂移。
