# 工单067-社区服务器部署脚本与实际文件清单漂移导致一键安装文档无法收口

## 背景

当前正在复核 `Chips-CommunityPlatformServer/` 的产品文档与技术文档是否已经达到“全面、详细、文档即代码”的收口标准。

按工作区规则，在继续修改文档前，重新核对了：

- 当前代码目录；
- `deploy/` 部署资产；
- `docker-compose.yml`；
- 现有部署文档与大任务文档。

核对后确认：社区服务器当前“一键安装 / Docker 部署”口径存在正式阻断，文档不能继续按“已完成闭环”口径推进。

## 证据

### 1. Compose 依赖的 Dockerfile 不存在

`Chips-CommunityPlatformServer/deploy/docker-compose.yml` 当前声明：

- `web` 服务使用 `deploy/Dockerfile.web`
- `admin` 服务使用 `deploy/Dockerfile.admin`

但实际 `Chips-CommunityPlatformServer/deploy/` 目录下只有：

- `.env`
- `.env.example`
- `Dockerfile.server`
- `docker-compose.yml`
- `nginx/nginx.conf`

未发现：

- `Dockerfile.web`
- `Dockerfile.admin`

### 2. 计划文档中的部署清单与当前仓库不一致

`项目日志与笔记/大任务006-薯片社区服务器开发/00-大任务概述与开发计划.md` 仍把以下内容写成规划中的正式输出：

- `deploy/docker-compose.prod.yml`
- `docs/` 目录

但当前仓库未见这些文件。

### 3. 现有部署文档无法以真实文件清单为唯一依据

当前 `Chips-CommunityPlatformServer/技术文档/13-部署与一键安装说明.md` 虽描述了部署结构，但如果底层部署资产未收口，就不能把这份文档视为“最终可信文档”。

## 影响范围

### 直接影响

1. 无法确认 `docker-compose.yml` 是否可以完整构建并拉起前台与后台；
2. 无法对外宣称“社区服务器已具备一键无脑安装能力”；
3. 无法把部署类产品文档和技术文档标记为正式完成。

### 波及范围

1. `Chips-CommunityPlatformServer/deploy/docker-compose.yml`
2. `Chips-CommunityPlatformServer/技术文档/13-部署与一键安装说明.md`
3. `Chips-CommunityPlatformServer/需求文档/08-交付边界与非功能要求.md`
4. `项目日志与笔记/大任务006-薯片社区服务器开发/00-大任务概述与开发计划.md`
5. 后续所有生态共用文档中涉及社区服务器部署与安装的正式口径

## 复现步骤

1. 打开 `Chips-CommunityPlatformServer/deploy/docker-compose.yml`。
2. 查找 `web` 与 `admin` 服务的 `dockerfile` 字段。
3. 列出 `Chips-CommunityPlatformServer/deploy/` 当前文件。
4. 确认 `Dockerfile.web` 与 `Dockerfile.admin` 均不存在。
5. 对照部署文档与大任务文档，确认它们仍以部署闭环口径描述该项目。

## 期望行为

在继续收口文档前，先补齐并冻结真实部署资产：

1. 明确 `web` / `admin` 的正式镜像构建方案；
2. 若采用独立 Dockerfile，则补齐对应文件并验证；
3. 若不采用独立 Dockerfile，则同步修正 `docker-compose.yml`；
4. 重新核对部署文档、需求文档、大任务文档中的文件清单与安装步骤；
5. 只有在部署链路真实可执行后，再继续宣称“所有文档都已完成”。
