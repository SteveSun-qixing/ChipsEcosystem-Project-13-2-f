# 工单071-.env.example遗漏Host动态插件配置导致环境文档与实际配置漂移

## 背景

当前正在逐篇重新核对社区服务器技术文档。  
在复核环境配置文档时，同时对照了：

- `packages/server/src/config/env.ts`
- `deploy/.env`
- `deploy/.env.example`

发现示例配置文件未同步最新的 Host 动态插件配置段。

## 证据

### 1. 实际环境解析已支持 Host 动态插件配置

`packages/server/src/config/env.ts` 当前已解析：

- `HOST_CARD_PLUGIN_PATHS`
- `HOST_THEME_PLUGIN_PATHS`
- `HOST_MODULE_PLUGIN_PATHS`
- `HOST_ACTIVE_THEME_ID`

### 2. 实际开发配置已经包含这些字段

`deploy/.env` 中已存在上述 Host 配置段。

### 3. 示例配置文件缺失该配置段

`deploy/.env.example` 在本次修复前没有同步这组配置，导致：

- 示例文件与真实支持的环境变量集合不一致；
- 技术文档提到的配置项无法在样例中直接找到；
- 部署者替换默认插件来源时容易漏配。

## 影响范围

1. `Chips-CommunityPlatformServer/deploy/.env.example`
2. `Chips-CommunityPlatformServer/技术文档/03-环境配置与依赖清单.md`
3. `Chips-CommunityPlatformServer/技术文档/13-部署与一键安装说明.md`

## 复现步骤

1. 查看 `packages/server/src/config/env.ts`；
2. 查看 `deploy/.env`；
3. 查看 `deploy/.env.example`；
4. 可确认 `.env.example` 缺少 Host 动态插件配置段。

## 期望行为

1. `.env.example` 应完整体现当前服务端支持的正式环境变量；
2. 环境样例、技术文档与实际解析逻辑保持一致。
