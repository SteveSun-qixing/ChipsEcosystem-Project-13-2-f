# 工单073-社区平台前端旧版主题词典与多页面壳层未升级到服务器独立Web体系

## 背景

- 当前任务要求将 `Chips-CommunityPlatformServer` 的社区前端重新设计为极简个人主页形态，并归档旧版文档、重写新的需求手册、技术手册和开发计划；
- 根据用户最新目标，社区前端不再需要总首页、发现页、房间页、工作台聚合页等多余界面，只保留个人主页与工作区上传页；
- 用户进一步明确：社区服务器与用户设备上的 Host 是同根同源但完全独立的两套底层系统，服务器要为 Web 服务单独建设主题、图标、多语言等能力；
- 在核对当前代码后确认，社区前端还没有完成这套“服务器独立 Web 体系”的升级。

## 证据

### 1. 当前主题与多语言仍停留在旧版页面内置实现

- 文件：`Chips-CommunityPlatformServer/packages/web/src/contexts/AppPreferencesContext.tsx`
- 真实现状：
  - 词典 `dictionaries` 直接硬编码在前端代码中；
  - `themeId` 只允许 `chips-community.midnight` / `chips-community.paper` 两个应用内枚举；
  - 语言与主题均通过 `localStorage` 自行持久化；
  - 旧实现直接和页面代码绑定，没有形成服务器侧可独立治理的 Web 主题 / 多语言模块。

### 2. 当前视觉 token 与图标策略仍由页面代码直接散落维护

- 文件：`Chips-CommunityPlatformServer/packages/web/src/styles/tokens.css`
- 真实现状：
  - 页面底色、阴影、圆角、动效时长等变量均在应用内手写；
  - 图标系统也没有被抽离成服务器独立 Web 前端基础设施；
- 风险：
  - 若继续在这套样式层上堆叠新页面，会把旧版散落式主题 / 图标 / 词典写法继续固化到新前端中。

### 3. 当前公开页面结构远超用户最新要求

- 文件：
  - `Chips-CommunityPlatformServer/packages/web/src/App.tsx`
  - `Chips-CommunityPlatformServer/packages/web/src/components/GlobalNav.tsx`
  - `Chips-CommunityPlatformServer/packages/web/src/pages/HomePage.tsx`
  - `Chips-CommunityPlatformServer/packages/web/src/pages/DashboardPage.tsx`
  - `Chips-CommunityPlatformServer/packages/web/src/pages/CardUploadPage.tsx`
  - `Chips-CommunityPlatformServer/packages/web/src/pages/RoomPage.tsx`
- 真实现状：
  - 仍存在首页、登录、注册、卡片页、箱子页、个人空间页、房间页、工作台、上传中心等多页面结构；
  - `GlobalNav` 仍承担品牌入口、上传、工作台、语言、主题、管理员后台等全局壳层职责；
  - 首页还包含发现、搜索、最新卡片、最新箱子等总站式结构。
- 与新目标冲突点：
  - 用户已明确“不需要有任何其他多余的页面、功能板块、组件等”；
  - 社区公开入口应以 `@username` 个人页为中心，而不是总首页。

## 影响范围

1. 新一轮社区前端重构若直接复用当前壳层，会把错误的信息架构继续沉淀进代码；
2. 当前前端不能作为“服务器独立 Web 主题 / 图标 / 多语言体系已收口”的可信样板；
3. 新的需求手册、技术手册、开发计划必须显式把现状定义为待替换基线，而不是最终口径。

## 复现步骤

1. 打开 `Chips-CommunityPlatformServer/packages/web/src/contexts/AppPreferencesContext.tsx`；
2. 打开 `Chips-CommunityPlatformServer/packages/web/src/styles/tokens.css`；
3. 打开 `Chips-CommunityPlatformServer/packages/web/src/App.tsx`；
4. 打开 `Chips-CommunityPlatformServer/packages/web/src/components/GlobalNav.tsx`；
5. 打开 `Chips-CommunityPlatformServer/packages/web/src/pages/HomePage.tsx`、`DashboardPage.tsx`、`CardUploadPage.tsx`。

## 期望行为

1. 社区前端只保留必要认证页、个人主页、工作区上传页；
2. 社区服务器建设并消费自己的 Web 主题 / 图标 / 多语言体系；
3. 服务器前端与 Host 维持同源理念和插件通用，但两者不形成直接运行时依赖。

## 暂行处理

1. 本轮先归档旧版文档；
2. 新文档全部以“社区前端 V2 重构目标”撰写，并显式把旧版硬编码方案定义为待替换基线；
3. 后续代码开发以前，本工单需作为服务器独立 Web 体系的前置依赖持续跟踪。
