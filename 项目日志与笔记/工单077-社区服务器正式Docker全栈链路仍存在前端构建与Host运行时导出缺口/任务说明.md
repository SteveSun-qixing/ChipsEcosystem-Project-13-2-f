# 工单077-社区服务器正式Docker全栈链路仍存在前端构建与Host运行时导出缺口

## 背景

当前任务需要把最新的 `Chips-CommunityPlatformServer` 以正式 Docker / Compose 全栈方式部署到远程 1Panel 服务器，并且不再接受开发态源码挂载式运行作为最终交付。

在前一轮排查中，已经登记了 `工单076`，确认 `Dockerfile.server` 生产阶段错误复制了仓库中不存在的 `.chips-server-host` 运行时目录。

继续按“修复正式链路而不是绕过问题”的要求推进后，又进一步确认还有两类会直接阻断正式部署的正式链路缺口。

## 证据

### 1. 前端正式镜像构建在依赖安装阶段失败

`Chips-CommunityPlatformServer/deploy/Dockerfile.web` 与 `Chips-CommunityPlatformServer/deploy/Dockerfile.admin` 当前都在执行：

```dockerfile
RUN pnpm install --frozen-lockfile
```

但在此之前，这两个 Dockerfile 只复制了社区服务器根清单和 `packages/web` / `packages/admin` / `packages/server` 的包清单，并没有复制：

- `Chips-Host`
- `Chips-BaseCardPlugin/image-BCP`
- `Chips-BaseCardPlugin/richtext-BCP`
- `Chips-ModulePlugin/Chips-HtmltoImage-Plugin`
- `ThemePack/Chips-default`

而 `packages/server/package.json` 通过 `file:` 依赖这些路径，导致 `pnpm install` 在工作区解析阶段直接失败，报错：

```text
ENOENT: no such file or directory, scandir '/app/Chips-Host'
```

### 2. 服务端正式运行产物引用了 Host 源码路径

`Chips-CommunityPlatformServer/packages/server/src/services/host-integration.ts` 当前使用：

```ts
import { HostApplication } from 'chips-host/src/main/core/host-application';
```

这在开发态 `tsx` 运行时可以解析 TypeScript 源文件，但正式构建后的 `dist/services/host-integration.js` 会保留为：

```js
require("chips-host/src/main/core/host-application")
```

而 `node dist/server.js` 的正式运行模式下，Node 无法直接加载 `chips-host` 包中的 TypeScript 源文件，验证命令：

```bash
node -e "require('./Chips-CommunityPlatformServer/packages/server/dist/services/host-integration.js')"
```

会报错：

```text
Cannot find module 'chips-host/src/main/core/host-application'
```

## 影响范围

### 直接影响

1. `Dockerfile.web` 和 `Dockerfile.admin` 不能从当前提交直接完成正式镜像构建；
2. 即使服务端镜像构建阶段继续推进，`node dist/server.js` 的正式运行模式也不满足可发布要求；
3. 远程服务器无法通过仓库正式 Docker / Compose 全栈链路稳定重建。

### 波及范围

1. `Chips-CommunityPlatformServer/deploy/Dockerfile.web`
2. `Chips-CommunityPlatformServer/deploy/Dockerfile.admin`
3. `Chips-CommunityPlatformServer/packages/server/src/services/host-integration.ts`
4. `Chips-Host/package.json`
5. 社区服务器正式 Docker / Compose 部署文档与运行说明

## 复现步骤

1. 直接构建 `Chips-CommunityPlatformServer/deploy/Dockerfile.web`；
2. 观察 `pnpm install --frozen-lockfile` 阶段报 `Chips-Host` 缺失；
3. 直接构建 `Chips-CommunityPlatformServer/deploy/Dockerfile.admin`；
4. 观察同样错误；
5. 执行：
   - `node -e "require('./Chips-CommunityPlatformServer/packages/server/dist/services/host-integration.js')"`
6. 观察 `chips-host/src/main/core/host-application` 无法在正式 Node 运行模式下解析。

## 期望行为

1. 前端正式镜像安装依赖时只解析自身所需工作区包；
2. `chips-host` 对社区服务器提供稳定的正式运行时导出入口，而不是依赖源码路径；
3. 修复后重新验证：
   - `Dockerfile.server`
   - `Dockerfile.web`
   - `Dockerfile.admin`
   - `deploy/docker-compose.yml`
   - 远程 1Panel 服务器正式重建部署
