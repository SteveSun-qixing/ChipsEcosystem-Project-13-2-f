## 背景

- 本轮用户要求先提交当前未提交代码。
- 按工作区 `AGENTS.md` 要求，提交前需要做与改动范围匹配的正式验证。
- 在执行 `Chips-Scaffold/chips-scaffold-basecard` 的 `build/test/test:templates/test:e2e` 时，前三项通过，`test:e2e` 失败。
- 失败点不在本次 Host `BoxService` 代码改动本身，而是在 `Chips-SDK/src/api/platform.ts` 的类型收口。

## 证据

- `Chips-Scaffold/chips-scaffold-basecard/scripts/run-generated-e2e.cjs` 会先生成基础卡片正式模板工程，再在生成目录执行：
  - `npx tsc -p tsconfig.json --noEmit`
  - `npx vitest run`
- 当前在第一步 TypeScript 编译就失败，报错如下：

```text
../../../../../Chips-SDK/src/api/platform.ts(207,29): error TS2339: Property 'pluginId' does not exist on type '{}'.
../../../../../Chips-SDK/src/api/platform.ts(207,57): error TS2339: Property 'pluginId' does not exist on type '{}'.
../../../../../Chips-SDK/src/api/platform.ts(208,30): error TS2339: Property 'sessionId' does not exist on type '{}'.
../../../../../Chips-SDK/src/api/platform.ts(208,59): error TS2339: Property 'sessionId' does not exist on type '{}'.
../../../../../Chips-SDK/src/api/platform.ts(210,14): error TS2339: Property 'launchParams' does not exist on type '{}'.
../../../../../Chips-SDK/src/api/platform.ts(210,41): error TS2339: Property 'launchParams' does not exist on type '{}'.
../../../../../Chips-SDK/src/api/platform.ts(210,89): error TS2339: Property 'launchParams' does not exist on type '{}'.
../../../../../Chips-SDK/src/api/platform.ts(211,23): error TS2339: Property 'launchParams' does not exist on type '{}'.
```

- `Chips-SDK/src/api/platform.ts` 当前相关实现：
  - `window.chips.platform.getLaunchContext?: () => Partial<PlatformLaunchContext> | undefined`
  - `const raw = getLaunchContext();`
  - 随后访问 `raw?.pluginId`、`raw?.sessionId`、`raw?.launchParams`
- 但在脚手架生成工程的编译上下文里，上述返回值被推导为 `{}`，导致正式模板工程无法通过 TypeScript 编译。

## 期望行为

- SDK 的 `resolveBridgeLaunchContext` 在正式生成工程里应保持稳定可编译；
- 脚手架生成工程必须满足“生成后直接通过 `tsc` 与测试”的正式标准；
- `window.chips.platform.getLaunchContext()` 的桥接类型应与脚手架模板工程的 TypeScript 配置完全兼容，不得在 E2E 场景退化成 `{}`。
