# 工单085-应用插件当前窗口自控制与壳层动态变更接口缺口

## 1. 背景

在 `Chips-VideoPlayer` 收口播放器界面时，用户进一步提出了两个与宿主窗口能力直接相关的问题：

1. 应用窗口的原生标题栏能否自动隐藏；
2. 应用窗口能否根据视频自动调整窗口比例。

按工作区 AGENTS 规则，在继续实现前必须先核对共享技术文档、SDK 和 Host 当前正式能力，确认这是否属于插件侧可直接完成的功能。

## 2. 证据

当前核对结果如下：

1. `生态共用技术文档/插件开发/06-Manifest配置规范.md`
   - 已定义 `ui.window.chrome.titleBarStyle`，说明应用插件可以为窗口声明原生壳层外观基线；
   - 该口径更偏向“启动时默认配置”，不是对当前运行中窗口的动态更新接口。

2. `生态共用技术文档/协议与接口标准/05-系统接口标准.md`
   - 已定义 `window.open/focus/resize/setState/getState/close`；
   - `window.open(config.chrome)` 支持 `titleBarStyle` 与 `titleBarOverlay`；
   - 但没有定义 `window.setChrome` 或类似动作。

3. `生态共用技术文档/协议与接口标准/02-Bridge-API规范.md`
   - 已定义 `surface.resize(surfaceId, width, height)`；
   - `surface.open(target=plugin)` 当前仍要求 `window.control + plugin.manage`；
   - 文档没有提供应用插件读取“当前自己所在 surfaceId”的正式入口。

4. `Chips-SDK/src/api/window.ts` 与 `Chips-SDK/src/api/surface.ts`
   - SDK 已封装 `open/focus/resize/setState/getState/close`；
   - 这些 API 都要求调用方显式提供 `windowId` 或 `surfaceId`；
   - 当前没有 `client.window.self.*` 或 `client.surface.self.*` 形式的自窗口封装。

5. `Chips-VideoPlayer/manifest.yaml`
   - 当前视频播放器只声明了 `resource.read / file.write / platform.read / theme.read / i18n.read`；
   - 没有 `window.control`，也没有任何“当前窗口标识”来源。

## 3. 影响范围

该缺口会影响所有需要根据运行时内容主动调节当前宿主窗口的应用插件，包括但不限于：

- 视频播放器
- 图片查看器
- 文档查看器
- 未来需要沉浸式壳层切换的演示类应用

## 4. 复现步骤

1. 从应用插件页面侧尝试基于视频元数据实现“按视频比例自动调整当前窗口”；
2. 会发现 `window.resize` / `surface.resize` 都需要 `windowId/surfaceId`；
3. 再检查 `platform.getLaunchContext()`，只会拿到 `pluginId / sessionId / launchParams`，无法直接映射为公开可用的当前窗口标识；
4. 尝试实现“播放时动态隐藏原生标题栏”时，也只能找到 manifest 基线配置与 `window.open(config.chrome)`，找不到更新当前窗口壳层的正式动作。

## 5. 期望行为

生态需要补齐下列至少一类正式方案：

1. 提供“当前承载容器”正式只读入口：
   - 例如 `client.surface.getCurrent()` / `client.window.getCurrent()`
   - 返回 `surfaceId/windowId` 与必要的窗口元数据

2. 提供只作用于当前承载容器的安全控制接口：
   - 例如 `client.surface.self.resize(...)`
   - 或 `client.window.self.resize(...)`

3. 若支持运行时壳层切换，需要明确正式契约：
   - 例如 `surface.updatePresentation(...)` / `window.updateChrome(...)`
   - 或明确宣布当前版本只支持“启动时静态 chrome”，不支持运行时变更。

## 6. 暂行处理

当前播放器仓库只实现插件内部可控范围内的能力：

1. 前端工具栏自动隐藏；
2. 更多信息面板固定尺寸与内部滚动；
3. 保持原生窗口壳层仍由 Host 当前静态配置决定；
4. 不在插件页面侧尝试越权控制当前宿主原生窗口。

## 7. 需要协作方

- Host 维护方
- SDK 维护方
- 共享技术文档维护方
