# 工单086-chipsdev-package返回成功但未产出cpk文件

## 1. 背景

在音乐基础卡片插件 `Chips-BaseCardPlugin/music-BCP` 完成编辑器重构后，按工作区正式流程执行了：

1. `chipsdev build`
2. `chipsdev validate`
3. `chipsdev package`
4. `chipsdev plugin install <dist/*.cpk>`

按共享技术文档口径，第 3 步应生成真实 `.cpk` 包，第 4 步再基于该包安装到开发工作区。

## 2. 证据

实际执行结果如下：

1. `chipsdev package` 返回：

```json
{
  "message": "打包成功",
  "output": "/Users/sevenstars/Documents/ChipsCard/Develop/Project-13-2-f/Chips-BaseCardPlugin/music-BCP/dist/chips.basecard.music-0.1.0.cpk"
}
```

2. 随后核对目录：

```bash
ls -la /Users/sevenstars/Documents/ChipsCard/Develop/Project-13-2-f/Chips-BaseCardPlugin/music-BCP/dist
```

实际只有：

- `index.mjs`

3. 继续执行：

```bash
chipsdev plugin install /Users/sevenstars/Documents/ChipsCard/Develop/Project-13-2-f/Chips-BaseCardPlugin/music-BCP/dist/chips.basecard.music-0.1.0.cpk
```

返回：

```json
{
  "error": "Plugin source not found: /Users/sevenstars/Documents/ChipsCard/Develop/Project-13-2-f/Chips-BaseCardPlugin/music-BCP/dist/chips.basecard.music-0.1.0.cpk",
  "code": "PLUGIN_SOURCE_NOT_FOUND"
}
```

4. 当前《`生态共用技术文档/命令行工具/02-Chips-Dev开发者命令行手册.md`》仍明确写明：

- `chipsdev package`：按 `manifest.yaml` 与构建产物生成 `.cpk`

## 3. 影响范围

- 任何插件工程都会被这个“成功但无产物”的行为误导；
- 开发者会在安装阶段才发现前一步的打包结果并不真实；
- 文档、CLI 输出和真实工具行为已经不一致，会影响后续正式安装链路验证。

## 4. 复现步骤

1. 进入任意插件工程；
2. 执行 `chipsdev build`；
3. 执行 `chipsdev package`；
4. 检查返回的 `.cpk` 路径是否真实存在；
5. 再执行 `chipsdev plugin install <返回路径>`，观察是否报 `PLUGIN_SOURCE_NOT_FOUND`。

## 5. 期望行为

1. `chipsdev package` 成功时必须真实产出 `.cpk` 文件；
2. CLI 返回的 `output` 必须与真实文件路径一致；
3. 若当前版本不支持真正打包，不应返回成功；
4. 共享技术文档需要与真实行为保持一致。

## 6. 暂行处理

```bash
chipsdev plugin install /Users/sevenstars/Documents/ChipsCard/Develop/Project-13-2-f/Chips-BaseCardPlugin/music-BCP/manifest.yaml
```
