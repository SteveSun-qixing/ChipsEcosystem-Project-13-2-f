## 3. 总体路线

### 3.1 核心原则

- 先冻结公共契约，再进入代码实施；
- 先打通 Host 公开动作、schema、错误和事件基线，再推进具体运行时；
- SDK 与脚手架必须以 Host 正式契约为唯一来源，不允许各自演化第二套口径；
- 不做兼容设计，不保留回退链路，不为旧 `slot + mount` 口径续命。

### 3.2 串并行策略

主线关键路径：

- 阶段一 -> 阶段二 -> 阶段三 -> 阶段六 -> 阶段七 -> 阶段八

受控并行路径：

- 阶段四可在阶段二完成后启动，但只有在阶段三完成正式接口冻结检查点后才能最终收口；
- 阶段五可在阶段二完成后启动模板骨架和文档重写，但必须在阶段四 SDK 正式接口冻结后完成生成工程接入；
- 阶段六必须等待阶段三、阶段四、阶段五全部收口后再进入正式联调。

### 3.3 为什么这样排

- 当前真实代码中，Host 和 SDK 仍然暴露旧 `module.mount` 体系，脚手架仍生成 React/slot 运行时模板；
- 如果不先冻结契约、动作面、schema 和错误语义，Host、SDK、脚手架会反复返工；
- Host 是唯一运行时承载，因此模块能力链路必须先在 Host 收口，再由 SDK 和脚手架消费。

只有当某一阶段的文档、代码、测试和验证全部收口后，才进入下一阶段。不得为赶进度跳过文档和测试，也不得先实现再事后补正式契约。
