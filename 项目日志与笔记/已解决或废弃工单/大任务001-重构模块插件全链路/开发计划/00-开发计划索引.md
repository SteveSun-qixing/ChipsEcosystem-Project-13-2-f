# 大任务001开发计划索引

## 1. 目标

本目录用于承载“模块插件全链路重构”的实施计划细化稿。每个阶段单独成文，明确入口条件、仓库范围、任务拆解、可并行项、验证门禁和阶段退出标准。

## 2. 阶段清单

1. `01-阶段一-公共契约与实施基线冻结.md`
2. `02-阶段二-Host路由契约与Schema基线重构.md`
3. `03-阶段三-Host模块服务运行时重构.md`
4. `04-阶段四-SDK模块调用入口重构.md`
5. `05-阶段五-模块脚手架与创建链路重做.md`
6. `06-阶段六-参考模块与跨模块联调.md`
7. `07-阶段七-生态调用方接入与回归.md`
8. `08-阶段八-验收收口与旧链路清退.md`

## 3. 总体依赖关系

- 阶段一是全局阻塞阶段，后续所有代码任务都依赖它；
- 阶段二冻结 Host 公开动作、schema、错误和事件基线，是阶段三、四、五的共同前置；
- 阶段三是全链路主关键路径；
- 阶段四与阶段五可以并行推进，但都要受阶段二和阶段三检查点约束；
- 阶段六依赖阶段三、四、五全部完成；
- 阶段七依赖阶段六的参考链路验证结果；
- 阶段八负责统一验收、旧链路清退和文档收口。

## 4. 并行原则

- 允许并行的前提是写入范围不冲突、依赖边界明确、对外契约已经冻结；
- 任何并行任务都不得绕开阶段二的正式 route/schema/error 基线；
- 若并行过程中发现契约未冻结、代码与文档漂移、Host/SDK/脚手架口径不一致，立即中止并补工单或回到前一阶段收口。
