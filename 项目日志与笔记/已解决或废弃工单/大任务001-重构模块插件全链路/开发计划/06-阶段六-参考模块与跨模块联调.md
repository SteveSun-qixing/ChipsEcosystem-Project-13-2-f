# 阶段六：参考模块与跨模块联调

## 1. 阶段目标

通过正式参考模块验证新的 Host、SDK、脚手架链路已经可用于真实业务场景，并覆盖长期演进中最关键的三类使用场景。

## 2. 入口条件

- 阶段三、阶段四、阶段五全部完成；
- 模块创建、安装、注册、调用和 job 管理链路已可用。

## 3. 参考模块覆盖要求

至少落地三类长期场景的参考实现，但它们只是业务场景，不代表新的模块类型：

- 转换场景参考模块：例如导入、导出、格式转换、转码；
- 跨模块编排场景参考模块：由一个模块通过 Host 调用其他模块完成组合任务；
- 通用处理场景参考模块：例如规范化、解析、索引、抽取、校验。

同时至少覆盖：

- 一个同步方法；
- 一个异步 job 方法；
- 一条模块间调用链。

## 4. 任务拆解

### 4.1 参考模块选型

- 根据生态近期和长期价值选择正式参考模块；
- 每个模块明确 capability、method、schema、依赖和测试范围；
- 避免再引入“特殊模块类型”命名。

### 4.2 工程创建与实现

- 使用新脚手架创建工程；
- 实现模块入口、contracts、测试、README；
- 验证 Manifest 与 Host 注册链路；
- 补齐模块自身需求文档和技术文档。

### 4.3 联调验证

- 安装到 Host；
- 通过 SDK 调用；
- 运行模块间调用链；
- 验证 job 进度、取消、失败和回收。

### 4.4 经验回灌

- 把联调中暴露的问题回灌到 Host / SDK / 脚手架；
- 若发现正式契约缺口，先补工单再调整；
- 同步更新公共文档和示例。

## 5. 串并行安排

- 三个参考模块可以并行开发；
- 但第一个模块应优先承担打样职责，先走通全链路，再复制到其他参考模块；
- 模块间调用相关模块必须在被依赖模块 capability 稳定后再合流。

## 6. 验证门禁

- 参考模块自身测试通过；
- Host / SDK / 脚手架相关回归通过；
- 真实安装包或开发态安装链路可用；
- 模块链路日志与审计字段完整。

## 7. 阶段退出标准

- 已有可对外讲清楚的正式参考实现；
- 新链路已通过真实业务场景证明可用；
- 可以进入调用方接入阶段。
