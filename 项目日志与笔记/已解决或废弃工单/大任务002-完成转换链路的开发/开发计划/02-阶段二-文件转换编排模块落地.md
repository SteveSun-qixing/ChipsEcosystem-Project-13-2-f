# 阶段二：文件转换编排模块落地

## 目标

完成 `Chips-FileConversion-Plugin` 的统一入口、流水线规划、job 进度聚合和结果归一。

## 本阶段任务

- 用正式 capability 契约替换脚手架示例能力
- 实现请求归一、流水线规划和原子模块调用层
- 接入统一进度聚合与取消治理
- 完成 `card -> html/pdf/image` 与 `html -> pdf/image` 的矩阵路由
- 补齐编排模块单元测试和跨模块集成测试

## 完成标准

- 调用方能够只通过一个入口发起 3 类导出
- job 进度、错误和最终结果全部由编排模块统一输出
- 原子模块替换或多 provider 并存时，不影响调用方入口
