# 大任务003开发计划索引

## 1. 目标

本目录用于承载“箱子文件全链路开发”的实施计划细化稿。每个阶段单独成文，明确入口条件、涉及仓库、任务拆解、可并行项、验证门禁和阶段退出标准。

## 2. 阶段清单

1. `01-阶段一-全链路实施基线与仓库边界冻结.md`
2. `02-阶段二-Host箱子服务与布局会话运行时重构.md`
3. `03-阶段三-SDK箱子能力与类型契约重构.md`
4. `04-阶段四-布局插件脚手架工程基线与CLI接入.md`
5. `05-阶段五-网格布局插件工程与共享布局运行时落地.md`
6. `06-阶段六-查看器箱子消费链路接入.md`
7. `07-阶段七-编辑引擎箱子编辑链路接入.md`
8. `08-阶段八-样例工作区与端到端联调验收.md`
9. `09-阶段九-回归收口与发布级文档清理.md`

## 3. 总体依赖关系

- 阶段一是全局阻塞阶段，负责冻结实施入口、仓库边界、共享位置和验证方式；
- 阶段二冻结 Host 正式动作面、schema、错误、事件和 `.box` 运行时会话，是阶段三至阶段六的共同前置；
- 阶段三依赖阶段二；
- 阶段四依赖阶段一至阶段三，用于补齐布局插件脚手架正式工程基线；
- 阶段五依赖阶段四；
- 阶段六与阶段七都依赖阶段二、阶段三、阶段四、阶段五；
- 阶段八依赖阶段六与阶段七全部完成；
- 阶段九负责统一收口和旧口径清理。

## 4. 并行原则

- 允许并行的前提是写入范围不冲突、依赖边界明确、正式契约已冻结；
- Host 动作面、会话模型、布局导出契约、布局脚手架正式工程基线一旦冻结，后续阶段不得再反向改动；
- 若实施过程中发现共享契约仍未冻结、公共组件缺口、脚手架缺口或 Host/SDK 正式能力缺口，优先把问题、影响面和调整后的收口策略直接补进对应阶段文档与总计划，不再新增阻断式工单；
- 查看器与编辑引擎不得各自发明不同的布局插件宿主约定，必须在阶段一先冻结共享方案。

## 5. 动态收口规则

- 当前大任务采用“边实施、边修订计划”的推进方式；
- 任何新增的阶段前置、验证门禁、仓库写入范围变化，都必须先回写计划文档再继续编码；
- 计划文档本身成为当前任务的唯一动态收口入口，避免阶段执行和文档再次分叉。
