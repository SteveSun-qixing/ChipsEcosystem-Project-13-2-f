# 阶段七：编辑引擎箱子编辑链路接入

## 1. 阶段目标

让 `Chips-EditingEngine` 正式具备箱子创建、打开、编辑、保存、布局配置编辑与预览能力，并与 Host 新箱子服务、SDK 箱子 API 和网格布局插件形成完整创作链路。

## 2. 入口条件

- 阶段二、阶段三、阶段四、阶段五已完成；
- 查看器阶段可以并行进行，但不阻塞本阶段开始；
- 编辑引擎当前工作区与基础卡片运行时已核对。

## 3. 涉及仓库

| 仓库 | 本阶段职责 |
|---|---|
| `Chips-EditingEngine` | 实现箱子正式编辑链路 |
| `Chips-Host` | 如需补齐编辑链路相关箱子静态动作的最小支持，做闭环修正 |
| `Chips-BoxLayoutPlugin` | 网格布局编辑器与预览集成 |

## 4. 重点交付物

- 新版箱子工作区模型；
- 箱子文档会话与保存模型；
- 条目管理界面；
- 网格布局参数编辑与实时预览；
- 箱子资产管理与写回；
- 编辑引擎测试和内部技术文档更新。

## 5. 任务拆解

### 5.1 重写箱子创建与工作区识别模型

必须把编辑引擎当前“目录态 `.box`”模型改为正式模型：

1. 创建箱子时生成正式工作目录结构；
2. 保存或导出时通过 Host `box.pack` 产出 `.box` 文件；
3. 打开 `.box` 时通过 `box.unpack` 或正式工作目录会话进入编辑态；
4. 工作区文件树识别 `.box` 时不再把它当作普通目录逻辑。

要求：

- 不得继续以“目录直接等于正式箱子文件”作为长期实现；
- 不得把旧字段 `layout`、`cards` 原样延续到新结构。

### 5.2 建立箱子文档会话层

编辑引擎需要新增正式箱子文档会话，负责：

1. 维护 `metadata.yaml / structure.yaml / content.yaml / assets/`；
2. 记录脏状态、保存状态与关闭确认；
3. 负责导入、删除、引用和排序箱子资产；
4. 统一刷新布局预览与条目管理面板；
5. 对接 Host `box.pack / box.unpack / box.readMetadata / box.validate`。

### 5.3 实现条目管理器

首版正式编辑链路至少支持：

1. 向箱子新增 `file://` 卡片 URL；
2. 新增或编辑 `http:// / https://` URL 条目；
3. 调整条目顺序；
4. 启用 / 禁用条目；
5. 维护摘要快照字段；
6. 维护 `layout_hints`；
7. 删除条目与批量基础校验。

要求：

- 条目只写 URL 和摘要，不拷贝卡片实体；
- 不得重新引入内部/外部判断字段；
- 编辑器内显示的条目预览也必须遵循摘要优先。

### 5.4 集成布局插件编辑器与预览

编辑引擎需要把网格布局插件接入到正式编辑链路：

1. 通过共享布局宿主层加载布局插件；
2. 把当前条目摘要、配置快照、资产回调注入 `renderEditor`；
3. 在编辑面板中编辑布局参数；
4. 在预览区域实时渲染网格布局效果；
5. 切换布局配置时同步刷新 `content.yaml`。

### 5.5 箱子资产管理

编辑引擎需要新增箱子资产管理能力，用于支持布局自有资源：

1. 导入背景图、占位图等布局资源；
2. 删除未引用资源；
3. 维护 `asset_refs`；
4. 避免把运行时临时 URL 或绝对路径写入正式配置。

## 6. 串并行安排

本阶段可受控并行：

- 一路重构箱子工作区与文档会话层；
- 一路实现条目管理器；
- 一路集成布局插件编辑器与预览；
- 一路补保存、校验和资产管理；
- 最终统一收口到同一套文档会话模型。

## 7. 验证门禁

- `cd Chips-EditingEngine && npm run build`
- `cd Chips-EditingEngine && npm run test`
- `cd Chips-EditingEngine && npm run lint`
- `cd Chips-EditingEngine && npm run validate`

同时必须验证：

- 可创建新箱子；
- 可编辑条目和布局配置；
- 可保存并导出正式 `.box`；
- 预览链路使用正式布局插件而非编辑器私有 mock；
- 未保留目录态 `.box` 的正式写路径。

## 8. 阶段退出标准

- 编辑引擎已经成为正式箱子创作工具；
- 用户可以从编辑引擎创建并保存一个可被查看器打开的 `.box`；
- 若编辑引擎仍需要手工改 YAML 或只能保存目录态样例，则本阶段不算完成。

## 9. 当前实现收口（2026-03-24）

本阶段当前已收口以下正式实现：

1. `Chips-EditingEngine` 工作区文件树已切换为单文件 `.box` 识别，不再把目录态 `.box` 作为正式写入路径；
2. 新增 `box-document-service`，负责：
   - 在工作区隐藏运行目录中创建箱子会话工作目录；
   - 对接 `box.inspect / box.unpack / box.pack`；
   - 维护 `metadata.yaml / structure.yaml / content.yaml / assets/`；
   - 维护脏状态、保存状态、条目管理与资源管理；
3. 新增箱子编辑窗口 `BoxWindow`：
   - 支持 URL 条目新增、编辑、排序、启用禁用与删除；
   - 支持布局配置编辑；
   - 支持布局实时预览；
   - 支持保存并覆盖正式 `.box` 文件；
4. 编辑引擎已补齐 `box.read / box.write` 权限；
5. 查看器与编辑引擎已经收口到共享布局宿主层 `chips-box-layout-host`，不再各自维护一套布局插件加载实现。

当前阶段验证已覆盖：

- `cd Chips-EditingEngine && npm run build`
- `cd Chips-EditingEngine && npm run lint`
- `cd Chips-EditingEngine && npm run test`
- `cd Chips-EditingEngine && npm run validate`
