# 大任务008：图标系统完善

## 1. 任务目标

按已经冻结的生态设计原稿、生态共用技术文档和当前真实代码基线，正式完成薯片生态图标系统的全链路收口，使生态具备以下正式能力：

1. 形成统一的运行时图标组件与语义输入模型；
2. 正式消费 `design-assets/material-symbols/` 中的 Material Symbols 资产；
3. 由主题系统统一控制图标颜色、填充、字重、grade、光学尺寸和尺寸；
4. 清除组件库、应用插件、编辑引擎、基础卡片插件中的硬编码 SVG、旧图标字符串与 emoji；
5. 收口应用启动图标、插件元数据图标、运行时 UI 图标、脚手架图标资产和正式文档口径；
6. 完成所有历史占位实现、旧版图标替换与相关代码重构；
7. 以正式脚本、正式测试和正式文档交付完整闭环，不保留兼容设计、回退链路或 TODO。

## 2. 本任务覆盖范围

本大任务覆盖以下正式范围：

1. 图标公共契约与共享文档；
2. 图标设计资产、来源追溯与生成链路；
3. `Chips-ComponentLibrary` 统一图标组件、token、契约、测试；
4. `ThemePack/*` 默认主题包与主题模板图标能力；
5. `Chips-Host` / `Chips-SDK` / Bridge / PAL 中与图标相关的正式能力边界；
6. `Chips-EcoSettingsPanel`、`Chips-CardViewer`、`Chips-EditingEngine` 等应用插件的图标接入；
7. `Chips-BaseCardPlugin/*`、`Chips-BoxLayoutPlugin` 等插件元数据与显示链路的图标重构；
8. `Chips-Scaffold/*` 中 app / basecard / boxlayout / theme 模板的图标资产与生成结果；
9. 正式代码、正式模板、正式共享文档中的 emoji 清零；
10. 样例工作区、验证脚本与发布级文档收口。

## 3. 本任务不包含的范围

以下内容不在本大任务范围内：

1. 兼容旧图标模型、双写字段、回退机制或过渡层；
2. 临时 SVG fallback、演示版图标方案、占位字体文件或“先跑通再重构”的过渡实现；
3. 篡改第三方上游追溯文件内容，例如 `design-assets/material-symbols/README.md` 这类上游同步文件；
4. 与图标系统无关的业务功能扩展；
5. 无实际图标入口的仓库强行做无意义改动。

## 5. 已核对的真实代码基线

当前真实基线与正式目标之间存在以下明确差距：

### 5.1 统一运行时图标组件缺失

- 全仓不存在 `ChipsIcon` 正式实现；
- `Chips-ComponentLibrary/packages/components/src/index.js` 仍通过 `createGlyphIcon(...)` 和 `getDefaultIconContent(...)` 直接硬编码 SVG path；
- 当前图标默认实现固定 `12x12` 尺寸和固定 stroke 语义，不符合“图标参数由主题 token 控制”的冻结规范。

### 5.2 图标 token 与 ThemePack 未形成正式闭环

- `ThemePack/Chips-default/icons/` 和 `ThemePack/Chips-theme-default-dark/icons/` 当前只有说明文档，没有正式图标字体、映射文件与加载方案；
- `Chips-ComponentLibrary/packages/tokens` 当前只有局部组件级 `icon.color` token，缺少 `icon.weight / icon.fill / icon.grade / icon.opticalSize / icon.size` 的全链路落地；
- `生态共用技术文档/主题系统/01-主题包开发指南.md` 仍保留旧的 `icons.css + svg + icon-sm/icon-md/icon-lg` 口径。

### 5.3 应用启动图标链路基本可用，但与运行时图标系统割裂

- `Chips-Host` 已支持 `manifest.ui.launcher.icon` 的安装复制与快捷方式解析；
- `Chips-CardViewer`、`Chips-EcoSettingsPanel`、`Chips-EditingEngine` 均已声明 `assets/icons/app-icon.ico`；
- 但这条链路只覆盖操作系统入口，不等于运行时 UI 图标系统已完成。

### 5.4 插件与应用侧仍大量使用 emoji / 旧 icon 字段

- `Chips-EditingEngine` 当前搜索到 76 处 emoji / 旧字符图标；
- `Chips-BaseCardPlugin` 当前搜索到 4 处 emoji / 旧字符图标；
- `Chips-Scaffold` 当前正式技术文档中仍存在 emoji；
- `生态共用技术文档` 当前仍存在 emoji 状态标记；
- `BasecardDefinition.icon?: string`、编辑引擎 `CardTypeDefinition.icon: string`、布局定义与多个菜单项仍停留在旧字符串模型。

### 5.5 设置面板与治理链路未消费正式图标能力

- `Chips-EcoSettingsPanel` 已可通过 `PluginShortcutRecord.iconPath` 读取快捷方式图标路径；
- 但当前应用插件治理页没有显示插件图标，未形成治理 UI 的正式消费链路。

## 9. 当前收口结论（2026-03-25）

当前只能确认以下结论：

1. 应用启动图标链路已经基本闭环；
2. 运行时 UI 图标系统尚未正式收口；
3. 组件库、主题包、设置面板、编辑引擎、基础卡片插件和脚手架仍有大量旧图标与 emoji 债务；
4. 本大任务必须按“先公共契约、再共享基础设施、再各仓接入、最后全仓回归”的顺序推进，不能跳阶段实施。
