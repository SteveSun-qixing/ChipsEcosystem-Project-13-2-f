# 阶段九：脚手架模板与默认图标资产全面升级

## 1. 阶段目标

把正式图标体系沉淀到脚手架和默认图标资产中，确保后续新工程默认就是正确口径，不再继续传播历史图标债务。

## 2. 涉及范围

1. `Chips-Scaffold/chips-scaffold-app`
2. `Chips-Scaffold/chips-scaffold-basecard`
3. `Chips-Scaffold/chips-scaffold-boxlayout`
4. `Chips-Scaffold/chips-scaffold-theme`
5. 默认 `app-icon.* / basecard-icon.svg / layout-icon.svg`
6. README 模板、技术文档、生成样例测试

## 3. 当前基线

1. app / basecard / boxlayout / theme 模板图标口径不一致；
2. 主题模板仍停留在说明文档级别；
3. 脚手架文档中还存在 emoji 和旧图标说明。

## 4. 核心任务

1. 更新默认图标资产来源；
2. 更新模板中的图标目录、文档说明和生成输出；
3. 更新基础卡片与布局模板中的图标数据模型；
4. 更新主题模板，使其默认具备正式图标运行时结构；
5. 更新脚手架测试与生成样例。

## 5. 交付物

1. 升级后的脚手架模板；
2. 统一后的默认图标资产；
3. 更新后的脚手架文档与测试。

## 6. 验证

- `cd Chips-Scaffold/chips-scaffold-app && npm run build && npm test && npm run test:templates && npm run test:e2e`
- `cd Chips-Scaffold/chips-scaffold-basecard && npm run build && npm test && npm run test:templates && npm run test:e2e`
- `cd Chips-Scaffold/chips-scaffold-boxlayout && npm run build && npm test && npm run test:templates && npm run test:e2e`
- `cd Chips-Scaffold/chips-scaffold-theme && npm run build && npm test`

## 7. 完成判定

只有当新生成工程默认不再继承旧图标模型、旧 emoji 和旧文档口径时，本阶段才算完成。
