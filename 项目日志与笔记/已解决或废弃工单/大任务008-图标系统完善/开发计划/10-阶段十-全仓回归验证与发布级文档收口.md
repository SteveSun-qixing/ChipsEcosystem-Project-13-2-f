# 阶段十：全仓回归验证与发布级文档收口

## 1. 阶段目标

对所有受影响仓库执行正式验证，完成工作区级复验、文档一致性检查和最终收口，确保图标系统作为生态正式基础设施可以发布。

## 2. 涉及范围

1. 所有核心改造仓
2. 受控进入仓
3. 共享文档与项目文档
4. 样例工作区与成品测试空间

## 3. 当前基线

1. 前九阶段完成后会形成多仓联动改动；
2. 图标系统涉及资产、类型、主题、应用、脚手架和验证链路；
3. 必须通过跨仓回归才能确认没有剩余旧口径。

## 4. 核心任务

1. 逐仓运行正式脚本；
2. 对全仓执行 emoji / 旧图标模型残留搜索；
3. 对共享文档、项目文档和本大任务文档做最终一致性检查；
4. 对样例工作区和成品测试空间执行最终回归；
5. 回写工单067、本大任务日志和最终收口结论。

## 5. 交付物

1. 全部正式验证结果；
2. 最终清零搜索结果；
3. 收口后的共享文档与项目文档；
4. 最终任务日志和结论。

## 6. 验证

1. 所有仓库正式脚本通过；
2. `rg` 搜索结果中不再出现本任务要求清除的旧实现；
3. 工作区级验证通过。

## 7. 完成判定

只有当以下条件同时满足时，本大任务才允许结束：

1. 图标系统完整可用；
2. 旧图标模型和 emoji 已清零；
3. 共享文档与代码一致；
4. 脚手架和样例工程不再传播旧口径；
5. 工作区级验证通过；
6. 工单与任务文档已全部回写完成。
