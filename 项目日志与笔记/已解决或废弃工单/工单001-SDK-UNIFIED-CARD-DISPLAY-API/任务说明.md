# 工单001-SDK-UNIFIED-CARD-DISPLAY-API 任务说明

## 背景

薯片组件库已进入卡片显示 iframe 高级组件开发阶段，组件侧已完成 `CardCoverFrame` 与 `CompositeCardWindow` 的契约定义和运行时实现。当前阻断项为 SDK 统一显示接口未落地。

## 阻断问题

1. 缺失 `client.card.coverFrame.render` 实际实现。
2. 缺失 `client.card.compositeWindow.render` 实际实现。
3. 缺失 SDK 向复合 iframe 发布标准事件的协议落地：
   - `chips.composite:ready`
   - `chips.composite:node-error`
   - `chips.composite:fatal-error`
4. 缺失 `compositeWindow` 的 `mode` 参数限制口径（仅 `view | preview`）与非法值错误码契约。
5. 缺失 iframe origin 元信息输出约定，组件端 `allowedOrigins` 无法完成安全联调闭环。
6. 缺失接口返回值与错误码的统一契约测试。

## 任务目标

- 在 SDK 仓实现统一显示接口与稳定返回结构。
- 对齐组件库已定义的事件协议和标准错误模型。
- 对齐组件库已定义的 `mode` 口径与 origin 白名单安全约束。
- 提供可用于查看器/编辑器联调的最小可用版本。

## 组件库侧已完成准备

- 适配器校验：`@chips/card-runtime` `validateCardDisplayAdapter`
- 运行时数据加载：`loadCoverFrameData`、`loadCompositeWindowData`
- iframe 事件解析：`parseCompositeFrameMessage`
- iframe 安全策略：`resolveIframeSandboxPolicy`、`isAllowedFrameOrigin`
- 组件契约与 token 已对齐

## 交付验收建议

- SDK 接口可在组件库 demo 场景直接接入。
- 查看器与编辑器对同一卡片文件展示一致。
- 单节点错误可降级，整体窗口不崩溃。
- 契约测试与集成测试通过。
