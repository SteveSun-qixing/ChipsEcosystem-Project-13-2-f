# 任务005-卡片查看器插件依赖无法按生态仓库方式安装

- 日期：2026-03-08
- 发现阶段：卡片查看器插件构建准备
- 问题类型：依赖管理 / 仓库集成
- 问题描述：`/Users/sevenstars/Documents/ChipsCard/Develop/Project-13-2-f/Chips-CardViewer/package.json` 声明依赖 `@chips/component-library` 与 `chips-sdk`，但当前生态工作区内不存在可直接通过 npm 安装的 `@chips/component-library` 发布包。执行依赖安装时，npm 返回 `404 Not Found - GET https://registry.npmjs.org/@chips%2fcomponent-library`，导致卡片查看器工程无法按正常方式完成依赖安装。
- 影响范围：`Chips-CardViewer` 的依赖安装、构建、测试，以及后续所有依赖该包名的生态插件工程。
- 复现步骤：
  1. 进入 `/Users/sevenstars/Documents/ChipsCard/Develop/Project-13-2-f/Chips-CardViewer`。
  2. 执行 `npm install react@^18.2.0 react-dom@^18.2.0 --save-dev` 或常规 `npm install`。
  3. 观察 npm 在解析现有依赖时请求 `@chips/component-library`。
  4. 观察安装流程因仓库中不存在该发布包而终止。
- 期望行为：生态插件工程应能基于当前工作区或既定发布渠道稳定安装全部声明依赖，并进入可构建状态。
- 状态：已修复

## 修复记录

1. 在生态根目录建立正式 npm workspaces 工作区，统一管理 `Chips-CardViewer`、`Chips-SDK`、`Chips-ComponentLibrary` 等一方工程依赖。
2. `Chips-CardViewer/package.json` 改为声明正式 semver 依赖：
   - `@chips/component-library: ^0.1.0`
   - `chips-sdk: ^0.1.0`
3. `@chips/component-library` 聚合包补齐正式包边界，统一导出 hooks、components、card-runtime。
4. 脚手架模板改为输出正式 semver 依赖，不再写入 `file:` sibling 路径。
5. `chipsdev create` 收敛为“注册生态工作区工程”职责，不再生成临时路径依赖。
6. `chipsdev test` 修正为：
   - 无 `vitest.config.*` 时继续按 `chips.config.mjs` 的 `testsDir` 收敛测试范围；
   - 存在 `vitest.config.*` 时完全尊重工程自身配置，避免与 jsdom / include 规则冲突。
7. 基础卡片脚手架补齐正式工程契约：
   - `chips.config.mjs` 明确 `type: "card"`；
   - `manifest.yaml` 改为对齐实际构建产物 `dist/index.mjs`；
   - 模板补充 `jsdom` 与 `vitest.config.mts`，确保生成工程测试环境完整可运行。
8. `chips-sdk` 补齐 `jsdom` 运行依赖，保证工作区内由 `chipsdev test` 驱动的 DOM 测试可稳定执行。
