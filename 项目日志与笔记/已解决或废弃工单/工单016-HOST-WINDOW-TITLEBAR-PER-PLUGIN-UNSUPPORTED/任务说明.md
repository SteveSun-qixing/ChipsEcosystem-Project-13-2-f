# 工单016：Host 窗口管理暂不支持应用插件单独控制原生标题栏透明能力

## 1. 背景

用户提出：希望调查并评估“查看器窗口上方标题栏能否单独设置为透明”，以及当前 Host 的窗口管理架构是否支持每个应用插件单独设置。

经检查，当前答案是否定的：现有 Host 窗口 API 与 PAL/Electron 映射均未提供该能力。

## 2. 现场证据

### 2.1 对外窗口接口未暴露相关字段

当前标准窗口接口与 Host 实现支持的字段仅包括：

- `title`
- `width`
- `height`
- `url`
- `pluginId`
- `sessionId`
- `permissions`
- 以及 PAL 层的 `resizable` / `alwaysOnTop`

未看到以下原生窗口外观字段：

- `transparent`
- `frame`
- `titleBarStyle`
- `titleBarOverlay`
- `vibrancy`
- `trafficLightPosition`
- `backgroundColor`

### 2.2 PAL WindowOptions 未定义标题栏外观能力

`Chips-Host/packages/pal/src/types.ts` 的 `WindowOptions` 当前没有透明标题栏相关字段。

### 2.3 Electron BrowserWindow 创建时未映射此类选项

`Chips-Host/packages/pal/src/node-adapter.ts` 创建 `BrowserWindow` 时，仅传：

- `title`
- `width`
- `height`
- `resizable`
- `alwaysOnTop`
- `show`
- `webPreferences`

没有任何针对原生标题栏透明或覆盖层的配置。

## 3. 根因判断

这不是单个应用插件的参数遗漏，而是 Host 窗口管理能力尚未设计并落地到“每插件独立原生标题栏外观控制”层级。

目前 `pluginId/sessionId` 只用于会话绑定与 bridge 上下文，不代表窗口外观可以按插件单独配置。

## 4. 影响范围

- 查看器无法单独要求原生标题栏透明；
- 所有应用插件当前都受限于同一套普通窗口创建能力；
- 如果后续要支持透明标题栏、无框窗口、标题栏覆盖层等能力，需要先扩展 Host API、PAL 契约与 Electron 落地实现；
- 该能力若处理不当，还会影响拖拽区、窗口控制按钮、安全边界与跨平台一致性。

## 5. 复现与核查步骤

1. 查看 `生态共用技术文档/协议与接口标准/06-Chips-Host对外接口基线.md` 中 `window.open` 参数基线。
2. 查看 `Chips-Host/src/main/services/register-host-services.ts` 中 `window.open` 入参定义。
3. 查看 `Chips-Host/packages/pal/src/types.ts` 中 `WindowOptions`。
4. 查看 `Chips-Host/packages/pal/src/node-adapter.ts` 中 Electron `BrowserWindow` 创建参数。
5. 确认当前链路中没有任何一层支持 per-plugin 原生标题栏透明配置。

## 6. 期望行为

- 若生态决定支持该能力，应明确其作为正式 Host 窗口能力，而不是某个应用私有 hack；
- 能按插件或窗口实例显式声明标题栏透明/覆盖层等原生外观；
- Host/PAL/Electron 三层契约一致；
- 跨平台差异、窗口控制按钮、拖拽区、安全限制与测试矩阵需同步定义。

## 7. 建议处理方向

1. 先由生态架构层决定是否正式支持 per-plugin 原生标题栏外观能力；
2. 若支持，扩展：
   - Host `window.open` 接口契约；
   - PAL `WindowOptions`；
   - Electron `BrowserWindow` 映射；
   - 插件权限/安全边界约束；
   - 回归测试与跨平台验收矩阵；
3. 若不支持，应在文档中明确“当前所有应用插件统一使用 Host 默认窗口外观”。

## 8. 正式处理结果

- Host `window.open`、PAL `WindowOptions`、Electron `BrowserWindow` 映射已正式扩展 `chrome` 能力；
- 应用插件现可在 manifest `ui.window.chrome` 中声明默认窗口外观，并在运行时通过 `window.open({ chrome })` 覆盖；
- 查看器已按正式契约接入白色窗口背景、隐藏式标题栏与覆盖层配置，不再依赖私有 Electron hack。

## 9. 需要协作方

- Host 窗口管理维护方
- PAL 适配层维护方
- 生态架构与协议维护方

## 10. 当前结论

Host 窗口管理已支持应用插件按 manifest 与窗口实例单独声明原生标题栏外观能力，相关能力已成为正式生态契约的一部分。
