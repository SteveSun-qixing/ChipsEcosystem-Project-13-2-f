# 工单029：编辑引擎迁移核心模块缺失

> 当前状态：已闭环（2026-03-13）

- 工单编号：工单029-编辑引擎迁移核心模块缺失
- 日期：2026-03-12
- 发现阶段：编辑引擎从Vue迁移到React项目检查
- 问题类型：生态工具链 / 应用迁移 / 核心模块缺失
- 问题描述：
  在对编辑引擎项目进行从Vue到React的迁移检查时，发现新项目缺少多个旧项目的核心模块，导致编辑器核心功能无法正常工作。具体问题如下：

  **1. 核心模块完全缺失：**
  - `window-manager.ts` (旧项目481行) - 窗口管理器，负责窗口创建/管理/销毁
  - `card-service.ts` (旧项目499行) - 卡片服务，管理卡片创建/读取/保存/渲染
  - `drag-drop-manager.ts` (旧项目973行) - 拖放管理器，统一管理所有拖放操作
  - `editor.ts` (旧项目732行) - 编辑器主类，整合所有模块
  - `command-manager.ts` - 命令管理器
  - `event-manager.ts` - 事件管理器（部分实现但功能不完整）
  - `resource-service.ts` (旧项目264行) - 资源服务
  - `plugin-service.ts` - 插件服务
  - `sdk-service.ts` - SDK服务
  - `local-core-connector.ts` - 本地核心连接器
  - `base-card-content-loader.ts` - 基础卡片内容加载器

  **2. 组件功能使用mock数据，未连接真实状态：**
  - `EditPanel.tsx` - 使用mock数据，未连接CardContext
  - `CardSettingsDialog.tsx` - 使用mock数据
  - `EngineSettingsDialog.tsx` - 使用mock数据
  - `PluginHost.tsx` - 使用stub，getEditorComponent等为null

  **3. 文件服务不完整：**
  - `file-service.ts` 中 mkdir/delete/move/copy 均未实现，只是console.warn

  **4. 缺失的UI组件：**
  - `LayoutSwitcher.vue` - 布局切换器
  - `FileDropZone.vue` - 文件拖放区域
  - `NestableCard.vue` - 嵌套卡片
  - `SortableList.vue` - 可排序列表
  - `InsertIndicator.vue` - 插入指示器
  - `DropHighlight.vue` - 拖放高亮
  - `DragGhost.vue` - 拖放幽灵元素

- 影响范围：
  - 编辑引擎核心功能（卡片创建/编辑/保存）无法工作
  - 拖放功能不完整
  - 窗口管理功能不完整
  - 状态管理功能简化，无法支持完整业务逻辑

- 复现步骤：
  1. 检查 Chips-EditingEngine/src/core/ 目录
  2. 对比 归档/Chips-Editor/src/core/ 目录
  3. 观察新项目缺少多个核心模块

- 期望行为：
  - 完整迁移所有核心模块到新项目
  - 所有组件连接到真实状态管理
  - 文件服务完整实现

- 暂行处理：
  - 继续使用简化版状态管理（Context）
  - 组件使用mock数据运行基本UI

- 需要协作方：
  - 架构组确认核心模块设计
  - SDK组确认服务接口

- 状态：已闭环

## 正式处理结果

### 1. 核心职责已按新架构补齐

- `window-manager.ts`、`card-service.ts`、`drag-drop-manager.ts`、`command-manager.ts`、`card-initializer.ts` 已在 React 新项目中补齐；
- 旧 `editor.ts`、`connector.ts`、`base-card-content-loader.ts` 的职责没有机械复制，而是按新架构拆分到 `App.tsx`、Context、`workspace-service.ts`、Host/SDK 正式接口与卡片初始化/持久化链路中；
- `file-service.ts` 已正式封装 Host 文件能力，`mkdir/delete/move/copy` 不再是空实现。

### 2. 组件侧已移除核心 mock/stub 依赖

- `EditPanel.tsx` 已连接真实 `CardContext` 与基础卡片正式编辑器链路；
- `CardWindow.tsx` 已连接 SDK 复合卡片显示窗口；
- `CardSettingsDialog.tsx` 的元数据编辑已回到 `CardService.updateCardMetadata()` 正式持久化链路；
- 卡片设置主题面板已切换为 `client.theme.list()/getCurrent()` 正式主题数据，不再使用写死主题列表；
- 未被引用的旧 `PluginHost` stub 已移入归档目录，避免继续与正式链路混淆。

### 3. 迁移结论

- 本工单的目标不是复刻旧 Vue 文件清单，而是在 React 新架构下恢复旧项目的核心能力；
- 当前“卡片创建 / 基础卡片编辑 / 实时保存 / 复合卡片预览 / 主题与多语言运行时接线”的主链路已完成闭环；
- 后续若发现某个独立面板仍存在业务未完成功能，应拆分为新的具体工单，而不再作为“迁移核心模块缺失”继续挂起。
