# 工单051：chipsdev 插件安装 CPK 口径漂移

## 背景

用户当前要求在生态正式开发工作区中，使用 `chipsdev` 对富文本基础卡片插件执行完整替换：

1. 卸载旧插件
2. 重新打包
3. 重新安装到当前 `.chips-host-dev`

工作区 `AGENTS.md` 明确要求：

- 发现工具链不可用、文档与代码不一致时，必须先登记工单；
- 不允许使用临时方案或绕过正式链路继续推进。

## 证据

### 1. 已成功执行的步骤

工程目录：

- `/Users/sevenstars/Documents/ChipsCard/Develop/Project-13-2-f/Chips-BaseCardPlugin/richtext-BCP`

已执行命令：

```bash
npx chipsdev plugin uninstall chips.basecard.richtext
npx chipsdev build
npx chipsdev package
```

结果：

- `plugin uninstall` 返回：

```json
{ "ok": true }
```

- `build` 成功生成最新 `dist/index.mjs`
- `package` 返回：

```json
{
  "message": "打包成功",
  "output": "/Users/sevenstars/Documents/ChipsCard/Develop/Project-13-2-f/Chips-BaseCardPlugin/richtext-BCP/dist/chips.basecard.richtext-0.1.0.cpk"
}
```

### 2. `package` 返回成功但目标 `.cpk` 未真实落盘

继续执行：

```bash
ls -l /Users/sevenstars/Documents/ChipsCard/Develop/Project-13-2-f/Chips-BaseCardPlugin/richtext-BCP/dist/chips.basecard.richtext-0.1.0.cpk
```

实际返回：

```text
ls: /Users/sevenstars/Documents/ChipsCard/Develop/Project-13-2-f/Chips-BaseCardPlugin/richtext-BCP/dist/chips.basecard.richtext-0.1.0.cpk: No such file or directory
```

这说明当前首先复现的是：

- `chipsdev package` 返回成功，但 `.cpk` 并未真实生成。

该现象与既有 `工单044-CHIPSDEV-PACKAGE-SUCCESS-WITHOUT-OUTPUT-FILE` 同主题相关；本次在 `richtext-BCP` 上再次复现，说明不是之前 `image-BCP` 的一次性异常。

### 3. 按文档执行安装时失败

执行命令：

```bash
npx chipsdev plugin install /Users/sevenstars/Documents/ChipsCard/Develop/Project-13-2-f/Chips-BaseCardPlugin/richtext-BCP/dist/chips.basecard.richtext-0.1.0.cpk
```

返回：

```json
{
  "error": "Plugin source not found: /Users/sevenstars/Documents/ChipsCard/Develop/Project-13-2-f/Chips-BaseCardPlugin/richtext-BCP/dist/chips.basecard.richtext-0.1.0.cpk",
  "code": "PLUGIN_SOURCE_NOT_FOUND",
  "details": {
    "durationMs": 0,
    "route": "plugin.install"
  },
  "retryable": false
}
```

结合上面的文件系统核对结果，当前这次安装失败的直接原因至少包含：

1. `chipsdev package` 返回的 `.cpk` 路径实际上不存在；
2. 当前文档与实现对 `plugin install` 的输入口径也存在漂移。

### 4. 文档口径

共享文档当前明确写法为：

```bash
chipsdev plugin install /绝对路径/产物.cpk
```

涉及文档：

- `生态共用技术文档/命令行工具/02-Chips-Dev开发者命令行手册.md`
- `生态共用技术文档/开发与运维/03-命令行拆分与使用指南.md`

### 5. 实现口径

当前实现中，`plugin install` 实际仍按 `manifestPath` 处理：

- `Chips-Host/src/main/cli/index.ts`
  - `plugin install requires manifest path`
  - `runtime.invoke('plugin.install', { manifestPath })`
- `Chips-Host/src/main/services/register-host-services.ts`
  - `descriptor<{ manifestPath: string }, { pluginId: string }>('plugin.install', ...)`
  - `ctx.runtime.install(input.manifestPath)`

## 影响范围

1. 当前无法按共享文档定义的正式 `chipsdev package -> chipsdev plugin install <cpk路径>` 链路重装富文本基础卡片插件。
2. 当前 `.chips-host-dev` 中的 `chips.basecard.richtext` 已被卸载，但无法按正式链路重新装回。
3. 所有依赖开发工作区 `.cpk` 安装联调的插件工程都可能受到影响。
4. 当前若继续改用“manifest/source 路径安装”，会形成对共享文档公开口径的绕开，不符合工作区“禁止临时方案”的要求。

## 复现步骤

1. 进入 `Chips-BaseCardPlugin/richtext-BCP`
2. 执行 `npx chipsdev plugin uninstall chips.basecard.richtext`
3. 执行 `npx chipsdev build`
4. 执行 `npx chipsdev package`
5. 执行 `ls -l <package 输出的 .cpk 路径>`，确认文件并不存在
6. 执行 `npx chipsdev plugin install <package 输出的 .cpk 路径>`
7. 观察报错 `PLUGIN_SOURCE_NOT_FOUND`

## 期望行为

二选一，但必须统一：

1. `chipsdev package` 返回成功时，目标 `.cpk` 文件必须真实存在；
2. 保持共享文档口径不变，正式支持 `chipsdev plugin install <cpk路径>`；
3. 若正式设计已经改为仅支持 manifest/source 路径安装，则必须同步修正文档、帮助信息、示例命令和相关验收说明。
