# 工单060-生态正式验证门禁存在漂移与回归

## 背景

本轮对“整个生态当前是否已达到可完美协同”的检查，不只是看代码结构，也按各项目 `package.json` 中的正式脚本执行门禁验证。验证结果显示：当前生态并未达到“总仓与关键子项目门禁全绿”的状态。

## 证据

### 总仓验证

1. 根目录执行 `npm run verify:workspace` 直接失败；
2. 失败原因是 `scripts/verify-ecosystem-workspace.mjs` 要求存在：
   - `Chips-Scaffold/chips-scaffold-basecard/templates/card-text-basic/package.json.tpl`
3. 但当前 `Chips-Scaffold/chips-scaffold-basecard/templates/` 下只有 `card-standard`，不存在 `card-text-basic`。

### 设置面板验证

1. `Chips-EcoSettingsPanel` 执行 `npm run verify` 失败；
2. 失败点是 `npm run typecheck`；
3. 具体错误来自 `../Chips-SDK/src/api/module.ts` 的 TS2775，而不是设置面板自身测试失败；
4. 单独执行：
   - `npm test`
   - `npm run build`
   - `npm run validate`
   均通过，说明当前阻断是正式 `verify` 中的跨仓类型门禁。

### 编辑引擎验证

1. `Chips-EditingEngine` 执行：
   - `npm run build` 通过
   - `npm run lint` 通过
   - `npm run validate` 通过
   - `npm test` 失败
2. 当前至少 3 个用例失败，涉及：
   - 拖入基础卡片后新增配置默认字段与断言不一致；
   - 目录态 `.card` 文件持久化后的基础卡片配置与断言不一致。

## 影响范围

- 当前不能对外宣称“整个生态已完成正式门禁收口”；
- 总仓验证脚本失效会降低全生态验收结论可信度；
- 设置面板和编辑引擎都还存在发布前阻断项。

## 复现步骤

1. 根目录执行 `npm run verify:workspace`；
2. `Chips-EcoSettingsPanel` 执行 `npm run verify`；
3. `Chips-EditingEngine` 执行 `npm test`。

## 期望行为

1. 根工作区 `verify:workspace` 与真实脚手架模板目录保持一致并能稳定通过；
2. 设置面板 `npm run verify` 全绿；
3. 编辑引擎 `npm test` 与当前实现重新收口，全绿后再作为正式结论。
