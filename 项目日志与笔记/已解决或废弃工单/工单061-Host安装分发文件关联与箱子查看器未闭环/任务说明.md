# 工单061-Host安装分发文件关联与箱子查看器未闭环

## 背景

在 `大任务003-生态全链路梳理` 对 `工单058` 进行 2026-03-20 全量复核时确认：Host Electron 主入口代码虽然已经接入 `.card/.box`、macOS `open-file` 与 `second-instance` 文件转发，但真实“安装 Host 后双击文件”的产品级路径仍未完整闭环。

## 证据

1. `Chips-Host/src/main/electron/app-entry.ts` 已具备：
   - 解析进程参数中的 `.card/.box`
   - 接入 macOS `open-file`
   - 接入 `second-instance`
2. `Chips-Host/src/main/core/file-association.ts` 对 `.box` 的处理仍然是：
   - 先执行 `box.inspect`
   - 再查找 `file-handler:.box`
   - 若未命中，则回退到 `window.open`
3. 当前生态内没有任何正式应用插件声明 `file-handler:.box`；
4. `Chips-CardViewer/manifest.yaml` 当前只声明：
   - `file-handler:.card`
5. `Chips-Host/package.json` 当前没有安装分发层文件关联元数据，无法证明安装包已把 `.card/.box` 注册到系统。

## 影响范围

- 不能对外宣称“安装 Host 后双击 `.card/.box` 文件即可进入正式目标应用”；
- `.box` 当前没有正式消费方；
- 安装分发与桌面文件关联验收结论不完整。

## 复现步骤

1. 检查 `Chips-Host/src/main/electron/app-entry.ts`；
2. 检查 `Chips-Host/src/main/core/file-association.ts`；
3. 检查 `Chips-CardViewer/manifest.yaml`；
4. 检查 `Chips-Host/package.json`；
5. 对照 `生态共用技术文档/文件格式规范/02-箱子文件格式规范.md` 的“双击 .box”正式要求。

## 期望行为

1. 明确 Host 安装分发层如何正式声明 `.card/.box` 文件关联；
2. 明确并提供正式 `file-handler:.box` 的箱子查看器应用；
3. 为“安装后双击 `.card/.box`”补齐分发级与应用级正式验收。
