# 工单062-Host安装包缺少复合卡片渲染运行时依赖

- 日期：2026-03-21
- 发现阶段：新系统下执行生态正式链路联调，安装基础卡片与卡片查看器后打开 `.card`
- 问题类型：安装包发布链路缺陷 / Host 运行时依赖漏带 / 文档与产物漂移

## 背景

在新系统中安装基础卡片插件与卡片查看器后，拖入 `薯片生态介绍.card`，查看器前端能够收到 `card.render` 成功返回的复合 iframe，但复合卡片内部节点全部降级失败，最终触发 `CARD_RENDER_FAILED`。首轮修复 `jsdom` 后，再次复测又暴露出同类安装包缺口：`card-service` 运行时继续缺少 `esbuild`。

## 证据

1. 查看器控制台连续收到两个 `chips.composite:node-error`：
   - `nodeId: 6at8Obawde`
   - `nodeId: Qzea3RKc0A`
2. 首轮两个节点的 Host 侧错误一致：

```text
Cannot find module 'jsdom'
Require stack:
- /Applications/Chips.app/Contents/Resources/app/dist/packages/card-service/src/card-service.js
- /Applications/Chips.app/Contents/Resources/app/dist/packages/card-service/src/index.js
- /Applications/Chips.app/Contents/Resources/app/dist/src/main/core/host-application.js
- /Applications/Chips.app/Contents/Resources/app/dist/src/main/core/main-process.js
- /Applications/Chips.app/Contents/Resources/app/dist/src/main/electron/app-entry.js
```

3. 随后复测时，另一个节点级错误为：

```text
Cannot find module 'esbuild'
Require stack:
- /Applications/Chips.app/Contents/Resources/app/dist/packages/card-service/src/card-service.js
- /Applications/Chips.app/Contents/Resources/app/dist/packages/card-service/src/index.js
- /Applications/Chips.app/Contents/Resources/app/dist/src/main/core/host-application.js
- /Applications/Chips.app/Contents/Resources/app/dist/src/main/core/main-process.js
- /Applications/Chips.app/Contents/Resources/app/dist/src/main/electron/app-entry.js
```

4. 核对安装包产物确认：
   - `Chips.app/Contents/Resources/app/node_modules/` 当时仅包含 `yaml`
   - `packages/card-service/src/card-service.ts` 运行时实际直接 `require('jsdom')` 与 `require('esbuild')`
5. 核对 `Chips-Host/package.json` 确认：
   - 当时只声明了 `yaml`
   - `jsdom` 与 `esbuild` 都没有作为 Host 正式运行时依赖收口

## 影响范围

- 所有依赖 `card-service` 富文本 HTML 清洗、资源地址重写或运行时编译能力的复合卡片节点，在安装包环境中都会因为 `jsdom` / `esbuild` 缺失而失败；
- 问题只在安装包/正式发布产物中暴露，开发工作区由于根 `node_modules` 可见性，容易掩盖缺陷；
- 新系统首轮联调会直接误判为“基础卡片不能渲染”，并连带影响卡片转换链路的正式运行时。

## 复现步骤

1. 构建并安装旧口径的 `Chips.app` 安装包；
2. 安装基础卡片插件与卡片查看器；
3. 使用卡片查看器打开任意包含富文本基础卡片的 `.card`；
4. 观察查看器收到 `chips.composite:node-error` 与 `chips.composite:fatal-error`；
5. 检查 `/Applications/Chips.app/Contents/Resources/app/node_modules`，确认缺少 `jsdom`、`esbuild` 及其传递依赖 / 平台二进制包。

## 期望行为

- Host 安装包必须按 `Chips-Host/package.json` 的正式生产依赖递归打包运行时依赖，而不是手写零散白名单；
- `packages/card-service` 在安装包中可直接 `require('jsdom')` 与 `require('esbuild')`；
- 新系统中打开复合卡片与执行卡片转换时，不再因为 `MODULE_NOT_FOUND` 导致节点级渲染失败。

## 暂行处理

- 已将 `jsdom` 与 `esbuild` 收口为 `Chips-Host` 正式依赖；
- 已把 macOS installer 从“只复制 `yaml`”改为“递归收集 Host 正式运行时依赖闭包”；
- 已补安装器单测覆盖顶层依赖、传递依赖，以及不暴露 `package.json` 子路径的 npm 包解析场景；
- 已补真实运行时 smoke test，确认安装包产物内可直接 `require('jsdom')` 与 `require('esbuild')`。
