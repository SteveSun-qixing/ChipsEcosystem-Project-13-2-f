# 工单064-箱子格式与Host布局运行时链路未收口

## 背景

本轮任务要求先冻结新的箱子架构与公共契约，再同步更新生态共用技术文档。新的设计方向已经在 `生态设计原稿/06-箱子文件格式.md`、`生态设计原稿/10-布局插件结构.md`、`生态设计原稿/15-查看器设计.md` 中收口为：

1. 箱子正式形态是单个 `.box` 文件；
2. 箱子不保存被引用卡片实体；
3. 箱子统一记录卡片 URL、卡片摘要、布局配置和布局自身资源；
4. 布局插件默认以封面、名称、标签等摘要态消费卡片；
5. Host 按布局插件申请动态定位、加载和返回卡片信息与资源。

## 证据

### 1. Host box-service 仍是旧模型

- `Chips-Host/packages/box-service/src/box-service.ts`
  - `BoxInspection.cards` 仍为 `Array<{ id: string; path: string; internal: boolean }>`
  - `normalizeCards()` 仍依赖 `path` 与 `internal`

### 2. Host box 服务动作面未收口

- `Chips-Host/src/main/services/register-host-services.ts`
  - 当前只注册：
    - `box.pack`
    - `box.unpack`
    - `box.inspect`
- `Chips-Host/src/main/services/register-schemas.ts`
  - 只注册上述 3 条请求 schema

### 3. SDK 仍暴露旧箱子类型

- `Chips-SDK/src/api/box.ts`
  - `BoxInspectResult.type` 仍为：
    - `full`
    - `shell`
    - `mixed`
  - `cards` 仍为：
    - `path`
    - `type: internal | external`

### 4. 查看器没有正式 `.box` 消费入口

- `Chips-CardViewer/manifest.yaml`
  - 当前只声明：
    - `file-handler:.card`
  - 未声明：
    - `file-handler:.box`

### 5. 测试仍围绕旧链路

- `Chips-Host/tests`
- `Chips-SDK/tests`

当前箱子相关测试只覆盖：

1. `box.pack`
2. `box.unpack`
3. `box.inspect`

没有覆盖：

1. URL + 摘要模型
2. 布局插件按需申请卡片信息
3. 箱子自有资源解析
4. `.box` 正式消费入口

## 影响范围

### 直接影响

- 生态共用技术文档与真实代码存在两套箱子口径；
- 后续 Host / SDK / Viewer / Layout Plugin 实现将失去统一公共契约基线；
- 新箱子设计无法形成完整闭环验收路径。

### 波及仓库

- `Chips-Host`
- `Chips-SDK`
- `Chips-CardViewer`
- 未来布局插件仓库
- 编辑引擎中的箱子编辑链路

## 复现步骤

1. 阅读 `生态设计原稿/06-箱子文件格式.md`，确认新箱子架构；
2. 阅读 `Chips-Host/packages/box-service/src/box-service.ts`，确认仍为 `path/internal` 模型；
3. 阅读 `Chips-SDK/src/api/box.ts`，确认仍为 `full/shell/mixed`；
4. 阅读 `Chips-Host/src/main/services/register-host-services.ts`，确认 box 服务动作面仍只有 3 条；
5. 阅读 `Chips-CardViewer/manifest.yaml`，确认没有 `.box` 文件入口；
6. 检查相关测试，确认没有新模型覆盖。

## 期望行为

后续实现阶段需要以本轮新的公共文档为目标，统一收口：

1. Host `box` 服务动作面；
2. SDK `box` API 类型；
3. `.box` 正式消费应用入口；
4. Layout Plugin ↔ Host 按需取数接口；
5. route manifest / schema / tests / docs 示例。

## 暂行处理

本轮先完成生态共用技术文档重写，冻结正式公共契约。

在实现尚未跟进前：

- 不再把当前 Host/SDK/Viewer 代码当作箱子正式标准；
- 后续代码改造必须以新的共用技术文档为准；
- 不做兼容性设计，不保留旧三种箱子模型和旧导入方式。

## 处理结果（2026-03-24）

1. `Chips-Host` 已完成箱子服务重构，正式收口为 URL + 摘要快照 + 布局资源模型，并补齐：
   - `box.validate`
   - `box.readMetadata`
   - `box.openView`
   - `box.listEntries`
   - `box.readEntryDetail`
   - `box.resolveEntryResource`
   - `box.readBoxAsset`
   - `box.prefetchEntries`
   - `box.closeView`
2. `Chips-SDK/src/api/box.ts` 已删除旧 `full/shell/mixed` 与 `internal/external` 类型，统一为 `BoxInspectionResult`、会话化查看能力与按需资源接口。
3. `Chips-CardViewer` 已声明并接入 `.box` 正式消费入口，通过共享布局宿主层加载布局插件。
4. `Chips-EditingEngine` 已切换到单文件 `.box` 正式模型，编辑期仅使用隐藏会话工作目录，不再把目录态 `.box` 作为正式形态。
5. 共享文档、阶段计划、测试与正式验证脚本已同步收口，Host / SDK / Viewer / EditingEngine / LayoutPlugin 的相关验证均已通过。
