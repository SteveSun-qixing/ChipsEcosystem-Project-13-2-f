# 工单066：卡片转HTML与文件转换模块构建链路阻塞

## 背景

在重构薯片社区服务器上传流水线时，用户明确要求：

1. 服务器不再自建一条 HTML 转换链路；
2. 必须安装正式转换插件，并由插件执行 `card -> html`；
3. 插件应支持未来动态替换；
4. 上传时需要先把卡片资源上传到 CDN、替换卡片内资源链接、重新生成新的 `.card` 文件、删除原始 `.card`，再进入 HTML 转换。

按生态正式契约核对后，社区服务器要完成这条链路，至少需要接入：

- `Chips-CardtoHTML-Plugin`
- `Chips-FileConversion-Plugin`

## 证据

### 1. 两个模块的 manifest 都要求安装 `dist/index.mjs`

- `Chips-ModulePlugin/Chips-CardtoHTML-Plugin/manifest.yaml`
  - `entry: dist/index.mjs`
- `Chips-ModulePlugin/Chips-FileConversion-Plugin/manifest.yaml`
  - `entry: dist/index.mjs`

### 2. 当前仓库中不存在对应构建产物

核对目录后确认：

- `Chips-ModulePlugin/Chips-CardtoHTML-Plugin/` 下无 `dist/index.mjs`
- `Chips-ModulePlugin/Chips-FileConversion-Plugin/` 下无 `dist/index.mjs`

### 3. 正式构建命令不可执行

执行正式脚本：

```bash
cd Chips-ModulePlugin/Chips-CardtoHTML-Plugin && npm run build
cd Chips-ModulePlugin/Chips-FileConversion-Plugin && npm run build
```

两者均失败，错误为：

```text
Volta error: Could not locate executable `chipsdev` in your project.
npm error Lifecycle script `build` failed with error:
npm error code 126
```

这说明当前不是社区服务器接线错误，而是目标转换模块本身没有可安装产物，且官方构建链路未收口。

## 影响范围

### 直接影响

- 社区服务器无法切换到正式 `converter.card.to-html` 链路；
- 社区服务器无法切换到正式 `converter.file.convert` 编排链路；
- 无法按用户要求实现“插件动态安装、动态替换、统一转换效果”。

### 间接影响

- 当前社区服务器若继续推进，只能依赖现有自建转换路径；
- 这会继续导致导出 HTML 与编辑引擎正式导出结果不一致；
- 也会违反当前工作区“禁止临时开发、遇到工具链缺口先提工单并停止推进”的要求。

## 复现步骤

1. 打开：
   - `Chips-ModulePlugin/Chips-CardtoHTML-Plugin/manifest.yaml`
   - `Chips-ModulePlugin/Chips-FileConversion-Plugin/manifest.yaml`
2. 确认二者 `entry` 都指向 `dist/index.mjs`
3. 检查两个目录，确认没有对应 `dist/index.mjs`
4. 执行：
   - `npm run build`
5. 观察失败并出现 `Could not locate executable "chipsdev" in your project`

## 期望行为

1. 两个模块仓库都能通过各自 `package.json` 的正式 `build` 脚本成功构建；
2. 构建产物 `dist/index.mjs` 存在并可被 Host 正式安装；
3. 社区服务器随后改为：
   - 动态安装转换插件
   - 上传资源到 CDN
   - 改写卡片资源 URL
   - 重新打包生成新的 `.card`
   - 删除原始 `.card`
   - 调用正式转换模块生成 HTML

## 暂行处理

- 当前不继续在社区服务器里扩展自建 HTML 转换逻辑；
- 当前不手写临时模块打包脚本；
- 当前停止本轮服务器重构，等待模块正式构建链路恢复。
