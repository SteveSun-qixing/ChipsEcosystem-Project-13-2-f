# 工单066-箱子全链路正式实现前置能力未收口导致当前开发阻断

## 背景

当前用户要求“按照计划，完成整个项目的开发”，范围已覆盖：

1. `Chips-Scaffold/chips-scaffold-boxlayout`
2. `Chips-SDK`
3. `Chips-Host`
4. `Chips-BoxLayoutPlugin`
5. `Chips-CardViewer`
6. `Chips-EditingEngine`

并要求严格遵守已冻结的箱子新架构、共享技术文档与正式工程规范，不允许兼容设计、临时实现或自行拍板改变架构。

在正式进入编码前，按工作区规则重新核对生态设计原稿、生态共用技术文档、相关仓库代码、`manifest.yaml`、SDK CLI 与大任务003计划后，确认当前并不是“可以直接按阶段编码”的状态，而是仍存在生态级前置能力未收口问题，已经阻断本轮继续推进。

## 核心结论

当前箱子全链路正式实现的多个前置环节仍未收口：

1. Host 仍只提供旧 `box.pack / box.unpack / box.inspect`；
2. SDK `box` API 仍暴露旧 `full/shell/mixed` 与 `internal/external` 模型；
3. SDK CLI `chipsdev create` 仍不支持 `layout` 类型工程创建；
4. 查看器仍只声明 `.card` 文件处理，没有 `.box` 正式入口；
5. `Chips-BoxLayoutPlugin` 仍为空仓；
6. 这些现状与当前已冻结的共享文档、任务计划和布局插件文档口径并不一致。

这意味着：如果现在继续声称“完成整个项目开发”，就必须在多仓并发改造前，先重新确认并收口这些前置能力；否则会违反“不能自行决定改变架构设计”“文档与代码不一致时先登记工单并停止推进”的规则。

## 证据

### 1. Host 箱子运行时动作面仍是旧实现

- `Chips-Host/packages/box-service/src/box-service.ts`
  - `BoxInspection` 仍为 `cards: Array<{ id: string; path: string; internal: boolean }>`
  - `normalizeCards()` 仍依赖 `path/internal`
- `Chips-Host/src/main/services/register-host-services.ts`
  - 当前仅注册 `box.pack`、`box.unpack`、`box.inspect`
- `Chips-Host/src/main/services/register-schemas.ts`
  - 当前仅登记上述三条 `box.*` schema

与共享文档中已经冻结的 `box.openView / box.listEntries / box.readEntryDetail / box.resolveEntryResource / box.readBoxAsset / box.prefetchEntries / box.closeView` 正式动作面不一致。

### 2. SDK 仍停留在旧箱子模型

- `Chips-SDK/src/api/box.ts`
  - 仍定义 `BoxInspectResult`
  - 仍暴露 `type: "full" | "shell" | "mixed"`
  - 仍暴露 `cards: Array<{ path: string; type: "internal" | "external" }>`
- `Chips-SDK/src/contracts/route-manifest.json`
  - 当前仅包含 `box.inspect`、`box.pack`、`box.unpack`
- `Chips-SDK/需求文档/02-薯片 SDK 功能需求规格.md`
  - 仍以 `client.box.inspect()` 为正式公开能力

这与共享文档中已经冻结的会话化、摘要优先、按需动态取数模型不一致。

### 3. SDK CLI 尚未接入布局插件脚手架

- `Chips-SDK/cli/index.js`
  - `chipsdev create` 当前只支持：`app`、`card`、`module`、`theme`
  - 错误提示仍写为“当前支持：app、card、module、theme”

这意味着即使 `chips-scaffold-boxlayout` 的文档与计划已经建立，正式 CLI 创建入口仍未打通，无法进入“官方布局插件脚手架 -> 布局插件工程”的完整标准流程。

### 4. 查看器与布局插件消费入口仍未收口

- `Chips-CardViewer/manifest.yaml`
  - 当前只声明 `file-handler:.card`
  - 没有 `.box` 的正式消费入口
- `Chips-BoxLayoutPlugin/`
  - 当前为空目录，没有正式工程骨架

这与大任务003要求的“查看器正式消费 `.box` 并加载布局插件”的阶段目标不一致。

### 5. 大任务计划与真实代码当前仍处于“待实施”关系，而非“可直接收尾”

- `项目日志与笔记/大任务003-打通箱子文件全链路/00-任务总览与文档边界.md`
  - 已明确记录 Host / SDK 当前仍是旧箱子口径
- `项目日志与笔记/大任务003-打通箱子文件全链路/03-开发计划.md`
  - 已明确阶段二、阶段三、阶段四、阶段五分别承担前置能力改造

当前请求若直接进入“完成整个项目交付”，会跳过这些尚未完成的前置阶段核验，违反当前工作区规则。
