# 工单067-图标系统正式链路未落地导致规范与实现漂移

## 背景

在回答“薯片生态的图标相关链路现在是否可以正常使用、各处如何使用图标、是否符合规范与设计”这一问题时，按工作区要求先后复核了：

- `生态设计原稿/`
- `生态共用技术文档/`
- `Chips-Host`
- `Chips-ComponentLibrary`
- `ThemePack`
- `Chips-EditingEngine`
- `Chips-BaseCardPlugin`

复核后确认：当前图标链路分裂为“系统级应用启动图标链路”和“运行时 UI 图标链路”两部分，前者基本可用，后者未按冻结规范落地。

## 证据

### 1. 规范已冻结，但实现未收口

- `生态共用技术文档/主题系统/03-图标与字体系统设计规范.md`
  - 要求统一图标组件 `ChipsIcon`
  - 要求 Material Symbols Variable Font
  - 要求图标主题 token：`icon.color / icon.weight / icon.fill / icon.grade / icon.opticalSize / icon.size`
- `生态共用技术文档/组件库/02-组件契约标准.md`
  - 要求控制型图标支持外部槽位覆盖

### 2. Host 启动图标链路存在

- `Chips-Host/src/main/services/register-host-services.ts`
  - 存在 `resolvePluginIconPath(...)`
  - 会读取 `manifest.ui.launcher.icon`
  - 会按 `.icns/.png/.ico/.svg` 平台偏好探测变体
- `Chips-EditingEngine/manifest.yaml`
- `Chips-CardViewer/manifest.yaml`
- `Chips-EcoSettingsPanel/manifest.yaml`
  - 都声明了 `ui.launcher.icon: assets/icons/app-icon.ico`
- 以上应用插件的 `assets/icons/` 下也确实存在 `.ico/.icns/.png/.svg`

### 3. 主题包图标资产仍是占位状态

- `ThemePack/Chips-default/icons/README.md`
- `ThemePack/Chips-theme-default-dark/icons/README.md`
  - 当前仅说明“后续可放置具体图标文件”
  - 未形成正式 Material Symbols 字体、`icons.css`、统一资源注册表、`@font-face` 接线

### 4. 主题 token 未覆盖图标变量轴

- `ThemePack/Chips-default/dist/tokens.json`
- `ThemePack/Chips-theme-default-dark/dist/tokens.json`
  - 当前仅能看到少量组件级 `icon.color`
  - 未见冻结规范要求的 `icon.weight / icon.fill / icon.grade / icon.opticalSize / icon.size`

### 5. 组件库没有统一 `ChipsIcon`

- `Chips-ComponentLibrary/packages/components/src/index.js`
  - `createGlyphIcon(...)` 直接生成内联 SVG
  - 默认宽高写死 `12 x 12`
  - `close / expand / collapse / calendar` 等默认图标都来自硬编码 path

### 6. 业务页面大量绕过正式图标规范

- `Chips-EditingEngine/src/App.tsx`
  - 工具窗口图标直接使用 `📁 / ✏️ / 📦`
- `Chips-EditingEngine/src/components/BaseWindow/BaseWindow.tsx`
  - 窗口动作图标直接使用 `− / ▽ / △ / ×`
- `Chips-EditingEngine/src/components/EngineSettings/EngineSettingsDialog.tsx`
  - 分类图标直接使用 `⚙️ / 🎨 / ⌨️ / 📋`
- `Chips-EditingEngine/src/components/FileManager/FileItem.tsx`
  - 文件树图标直接使用 `📂 / 📁 / 🃏 / 📦 / 📄 / ▶`
- `Chips-BaseCardPlugin/image-BCP/src/index.ts`
  - `basecardDefinition.icon = "🖼️"`
- `Chips-BaseCardPlugin/richtext-BCP/src/index.ts`
  - `basecardDefinition.icon = "📝"`

## 影响范围

- 主题切换无法统一治理图标风格和变量轴；
- UI 图标跨平台字形会不一致；
- 页面层、组件库层、插件层同时存在多套图标输入方式；
- 无法把当前实现描述为符合冻结规范的正式图标系统；
- 后续若推进图标替换、主题联动、a11y 门禁，会缺少统一入口。

## 复现步骤

1. 先阅读 `生态共用技术文档/主题系统/03-图标与字体系统设计规范.md`；
2. 再检查 `ThemePack/*/icons/README.md` 与 `ThemePack/*/dist/tokens.json`；
3. 再检查 `Chips-ComponentLibrary/packages/components/src/index.js`；
4. 最后检查 `Chips-EditingEngine` 与 `Chips-BaseCardPlugin` 的图标使用点；
5. 对比 `Chips-Host/src/main/services/register-host-services.ts` 中启动图标链路，可确认“启动图标可用、UI 图标链路未收口”的现状。

## 期望行为

- 建立统一 `ChipsIcon` 组件；
- 主题包正式接入 Material Symbols Variable Font；
- Host / 主题运行时 / 组件库形成统一图标资源治理入口；
- 页面层只传语义图标名与必要语义参数，不再直接传 emoji / Unicode 字符；
- 补齐图标专项测试与可访问性门禁。

## 暂行处理

- 当前只保留“应用启动图标链路可用”的结论；
- UI 图标相关口径全部按“未正式收口”对外说明；
- 本工单闭环前，不继续新增分散图标方案。
