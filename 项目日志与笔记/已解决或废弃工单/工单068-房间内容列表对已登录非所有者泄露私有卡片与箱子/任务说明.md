# 工单068-房间内容列表对已登录非所有者泄露私有卡片与箱子

## 背景

当前正在按工作区要求，对 `Chips-CommunityPlatformServer/` 的全部产品文档和技术文档进行重新核对与重写。

按规则，在修改每一份文档前，需要先重新查看相关代码。  
在复核房间、卡片、箱子权限实现时，发现当前“房间内容列表”权限判断存在正式错误，已经阻断文档继续收口。

## 证据

### 1. 房间本身的访问控制与内容项访问控制分离

`RoomService.getAccessible(roomId, requesterId)` 只负责判断房间本身是否可访问。

它并不会进一步判断房间内的卡片或箱子是否应对该请求者可见。

### 2. `GET /api/v1/rooms/:roomId/contents` 直接把 `request.user?.userId` 透传给内容服务

`Chips-CommunityPlatformServer/packages/server/src/routes/rooms.ts` 当前实现：

- `CardService.listByRoom(room.id, request.user?.userId ?? null, qs)`
- `BoxService.listByRoom(room.id, request.user?.userId ?? null, qs)`

### 3. `CardService.listByRoom` 只按“是否已登录”控制是否过滤私有内容

`Chips-CommunityPlatformServer/packages/server/src/services/card.service.ts` 当前条件为：

```ts
where: and(
  eq(cards.roomId, roomId),
  requesterId ? undefined : eq(cards.visibility, 'public'),
  eq(cards.status, 'ready'),
)
```

这意味着：

- 未登录用户：只能看到公开卡片
- 任意已登录用户：不会再过滤私有卡片

### 4. `BoxService.listByRoom` 存在同样问题

`Chips-CommunityPlatformServer/packages/server/src/services/box.service.ts` 当前条件为：

```ts
where: and(
  eq(boxes.roomId, roomId),
  requesterId ? undefined : eq(boxes.visibility, 'public'),
)
```

同样会让任意已登录用户看到公开房间中的私有箱子。

## 影响范围

### 直接影响

1. 公开房间中的私有卡片可能被任意已登录非所有者看到；
2. 公开房间中的私有箱子可能被任意已登录非所有者看到；
3. 当前权限模型实际行为与产品规则不一致。

### 波及范围

1. `Chips-CommunityPlatformServer/packages/server/src/services/card.service.ts`
2. `Chips-CommunityPlatformServer/packages/server/src/services/box.service.ts`
3. `Chips-CommunityPlatformServer/packages/server/src/routes/rooms.ts`
4. 与房间内容、空间内容、权限控制有关的所有产品文档和技术文档

## 复现步骤

1. 用户 A 创建一个公开房间。
2. 用户 A 在该房间内上传一个 `visibility=private` 的卡片或箱子。
3. 用户 B 登录。
4. 用户 B 请求 `GET /api/v1/rooms/:roomId/contents`。
5. 观察响应中会返回本不应对用户 B 可见的私有内容。

## 期望行为

1. 房间内容列表应按“是否房间所有者”或“是否内容所有者”判断是否可见；
2. 非所有者即便已登录，也不应因为“有登录态”而看到私有内容；
3. 只有权限模型修复并验证完成后，相关产品文档和技术文档才能继续收口。

## 处理结果

1. 已修正 `Chips-CommunityPlatformServer/packages/server/src/routes/rooms.ts`：
   - `GET /api/v1/rooms/:roomId/contents` 不再只把 `request.user?.userId` 作为内容服务判断依据；
   - 现在会显式传入 `room.userId`，作为房间所有者 ID。
2. 已修正 `Chips-CommunityPlatformServer/packages/server/src/services/card.service.ts`：
   - `listByRoom(roomId, requesterId, ownerUserId, pagination)` 新增 `ownerUserId`；
   - 私有卡片过滤逻辑改为基于 `requesterId === ownerUserId` 判断。
3. 已修正 `Chips-CommunityPlatformServer/packages/server/src/services/box.service.ts`：
   - `listByRoom(roomId, requesterId, ownerUserId, pagination)` 新增 `ownerUserId`；
   - 私有箱子过滤逻辑同步改为基于房间所有者身份判断。
4. 已新增测试 `Chips-CommunityPlatformServer/packages/server/src/routes/rooms.test.ts`：
   - 验证房间内容列表查询会把 `room.userId` 透传到 `CardService.listByRoom(...)` 与 `BoxService.listByRoom(...)`；
   - 避免后续回归为“只要已登录即可看到私有内容”的错误行为。
