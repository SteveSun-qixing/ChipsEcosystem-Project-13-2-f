# 工单070-公开房间与空间页内容计数泄露私有或未完成内容数量

## 背景

当前正在重写社区服务器权限、房间、空间与 API 文档。  
在继续核对房间计数、空间页展示与房间详情页展示时，发现公开侧返回的 `cardCount` / `boxCount` 仍可能暴露私有内容数量或未完成卡片数量。

## 证据

### 1. `RoomService.getContentCounts` 当前只按 `roomId` 统计全部内容

旧实现会直接统计房间下所有卡片和所有箱子，不区分：

1. 请求者是否为房间所有者；
2. 卡片 / 箱子是否为私有；
3. 卡片是否已经 `ready`。

### 2. 这些计数会被公开接口直接返回

当前路由：

- `GET /api/v1/users/:username/rooms`
- `GET /api/v1/rooms/:roomId`
- `GET /api/v1/users/:username/space`

都会把 `RoomService.getContentCounts(...)` 的结果直接带到响应里。

### 3. Web 前端会直接显示这些计数

- `packages/web/src/pages/SpacePage.tsx`
- `packages/web/src/pages/RoomPage.tsx`

都会在公开页面展示房间卡片数与箱子数。

## 影响范围

1. 公开空间页会暴露房间里“还有多少不可见内容”；
2. 公开房间页会暴露私有卡片 / 箱子或未完成卡片的存在数量；
3. 当前 API 元数据与真实可见内容集合不一致，会阻断权限文档与 API 文档收口。

## 复现步骤

1. 用户 A 创建一个公开房间；
2. 在该房间中添加：
   - 私有卡片 / 私有箱子；或
   - 仍在 `pending` / `processing` / `error` 的卡片；
3. 使用用户 B 或访客访问公开房间详情、公开房间列表或公开空间页；
4. 观察 `cardCount` / `boxCount` 大于其实际可见内容数量。

## 期望行为

1. 非所有者看到的房间计数必须只反映其可见内容；
2. 公开卡片计数必须与公开房间内容列表一致，只统计 `public + ready` 卡片；
3. 公开箱子计数只统计 `public` 箱子；
4. 所有者访问时仍可看到完整计数。

## 暂行处理

1. 已登记总工单；
2. 已按请求者身份重构计数逻辑；
3. 已补充相关路由测试；
4. 已继续基于修复后的真实行为编写文档。

## 需要协作方

- 社区服务器后端维护方
- 前端维护方
- 文档维护方

## 处理结果

1. `RoomService.getContentCounts(...)` 已改为接收 `requesterId` 与 `ownerUserId`；
2. 非所有者分支现在只统计其可见内容；
3. `routes/rooms.ts` 与 `routes/space.ts` 已同步传入请求上下文；
4. 已新增测试，防止相关路由再次回归为错误计数行为。
