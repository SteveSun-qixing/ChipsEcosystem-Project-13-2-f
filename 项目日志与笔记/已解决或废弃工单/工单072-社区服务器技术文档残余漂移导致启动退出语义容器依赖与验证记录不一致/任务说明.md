# 工单072-社区服务器技术文档残余漂移导致启动退出语义、容器依赖与验证记录不一致

## 1. 背景

- 当前任务要求对 `Chips-CommunityPlatformServer/技术文档/` 全量逐篇复核；
- 每篇文档都必须先重新查看相关代码、部署文件、环境配置与实际验证链路，再决定是否需要改写；
- 在前几轮大修后，技术文档主体已经基本收口，但本轮收尾复核中又发现少量残余漂移。

## 2. 证据

### 2.1 服务端退出语义文档漂移

- 代码：`Chips-CommunityPlatformServer/packages/server/src/server.ts`
- 真实实现：
  - `SIGTERM` / `SIGINT`：调用 `shutdown(app)`，依次关闭 Fastify、Redis、PostgreSQL；
  - `uncaughtException` / `unhandledRejection`：只记录日志并 `process.exit(1)`，不先执行 `shutdown(app)`。
- 漂移点：
  - 旧版 `技术文档/02-服务端启动与运行机制.md` 把上述两类行为混写成统一的“优雅退出”口径。

### 2.2 容器构建依赖清单遗漏

- 代码：
  - `Chips-CommunityPlatformServer/deploy/Dockerfile.server`
  - `Chips-CommunityPlatformServer/packages/server/package.json`
- 真实实现：
  - `packages/server` 当前声明了本地依赖 `chips.module.chips.htmltoimage.plugin`；
  - `Dockerfile.server` 也显式复制了 `Chips-ModulePlugin/Chips-HtmltoImage-Plugin`。
- 漂移点：
  - 旧版环境 / 部署文档没有完整体现这项镜像构建依赖。

### 2.3 验证记录不准确

- 当前真实通过命令：
  - `npm_config_cache=/tmp/ccps-npm-cache npx pnpm@9 --filter @ccps/server test`
  - `npm_config_cache=/tmp/ccps-npm-cache npx pnpm@9 --filter @ccps/server build`
  - `npm_config_cache=/tmp/ccps-npm-cache npx pnpm@9 build`
  - `docker compose -f deploy/docker-compose.yml config`
- 漂移点：
  - 旧版 `技术文档/14-测试、验证与排障.md` 把前三条写成了简写 `pnpm ...`，不符合“本轮已实际通过”的真实记录。
