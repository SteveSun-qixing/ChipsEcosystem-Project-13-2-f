# 工单078-社区服务器正式全栈运行仍遗漏迁移资源与后台API反向代理

## 背景

在 `工单076` 与 `工单077` 已经把服务镜像构建和 Host 运行时导出问题修复后，继续对远端正式 Docker / Compose 全栈上线结果做稳态验证。

验证环境：

- 服务器：`111.92.241.17`
- 部署方式：`Chips-CommunityPlatformServer/deploy/docker-compose.yml`
- 验证目标：健康检查、管理员登录、后台受保护接口、前后台首页

继续验证后确认，正式可构建不等于正式可用，仍有两处正式运行链路遗漏。

## 证据

### 1. 服务端容器启动时缺少 Drizzle 迁移元数据

`ccps_server` 日志报错：

```text
Error: Can't find meta/_journal.json file
```

栈追踪显示失败点来自：

- `drizzle-orm/node-postgres/migrator`
- `packages/server/dist/db/migrate.js`

而当前服务端正式构建 `dist/` 产物里默认只有 TypeScript 编译出的 `.js/.d.ts` 文件，并不会自动把：

- `packages/server/src/db/migrations/*.sql`
- `packages/server/src/db/migrations/meta/_journal.json`
- `packages/server/src/db/migrations/meta/*_snapshot.json`

带到 `dist/db/migrations/`。

### 2. `/admin/api/*` 被错误回退到后台 SPA

使用管理员 access token 请求：

```text
GET /admin/api/v1/stats
```

返回内容为后台 HTML，而不是 JSON 统计数据。

核对 `deploy/nginx/nginx.conf` 后确认：

- 已有 `/api/ -> server:3000`
- 已有 `/admin/ -> admin:80`
- 但缺少 `/admin/api/ -> server:3000`

因此 `/admin/api/*` 会先命中 `/admin/` 规则，被转发到后台静态站点并回退到 `index.html`。

## 影响范围

1. `Chips-CommunityPlatformServer/packages/server/package.json`
2. `Chips-CommunityPlatformServer/scripts/copy-server-runtime-assets.mjs`
3. `Chips-CommunityPlatformServer/deploy/Dockerfile.server`
4. `Chips-CommunityPlatformServer/deploy/nginx/nginx.conf`
5. 所有依赖正式 `dist` 启动与 `/admin/api/*` 访问的部署环境

## 复现步骤

1. 使用正式 Docker / Compose 全栈方式启动当前仓库；
2. 观察 `ccps_server` 重复启动并查看日志；
3. 确认日志报 `Can't find meta/_journal.json file`；
4. 在管理员登录成功后请求 `GET /admin/api/v1/stats`；
5. 观察返回结果为后台 HTML，而不是 JSON。

## 期望行为

1. `@ccps/server build` 的正式 `dist` 产物必须自带运行时必需的迁移资源；
2. `Dockerfile.server` 在镜像内正式构建时也必须得到同样完整的 `dist`；
3. 主 Nginx 必须把 `/admin/api/` 优先代理到 `ccps_server`；
4. 修复后后台 API、管理员登录与健康检查应全部可用。
