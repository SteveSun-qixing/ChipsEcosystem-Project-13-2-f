# 工单080-箱子编辑会话并发打开导致工作目录冲突并暴露错误交互入口

## 背景

用户在编辑引擎中点击箱子后，右侧编辑面板加载箱子编辑器时，界面直接报错：

`Error invoking remote method 'chips:invoke': Error: __chips_ipc_error__:{"code":"INTERNAL_ERROR","message":"Unknown error","details":{"errno":-17,"code":"EEXIST","syscall":"mkdir"...}}`

同轮核对还确认，箱子名称、封面等文档级设置仍滞留在右侧编辑面板，没有对齐卡片的正式交互模型。

## 证据

1. 报错路径明确落在：
   - `chips:invoke -> file.mkdir -> EEXIST`
   - 工作目录位于 `.chips-host-dev/.chips-editing-engine/box-sessions/session-...`
2. `Chips-EditingEngine/src/services/box-document-service.ts` 原实现使用 `Date.now()` 直接拼接会话目录名；
3. `useBoxDocumentSession` 在组件加载阶段会主动调用 `boxDocumentService.openBox(...)`；
4. 右侧 `BoxEditorPanel` 原实现仍内嵌箱子名称、描述、标签与封面编辑。

## 影响范围

1. 箱子编辑器会在真实点击场景中直接失败；
2. 编辑引擎开发态下，重复 effect / 重复点击会放大问题；
3. 箱子交互和卡片正式链路不一致，文档级设置入口分裂。

## 复现步骤

1. 打开编辑引擎；
2. 创建一个箱子并放到桌面；
3. 点击箱子，让右侧编辑面板加载箱子编辑器；
4. 观察报错或不稳定行为；
5. 同时观察名称、封面等设置仍在右侧面板，而不是窗口右上角设置弹窗。

## 期望行为

1. 同一箱子会话打开必须幂等；
2. 工作目录命名必须避免同毫秒冲突；
3. 箱子文档级设置统一走窗口右上角设置弹窗；
4. 右侧编辑面板只负责布局配置和内容列表。

## 暂行处理

1. 在 `boxDocumentService` 中新增会话打开中的 Promise 复用；
2. 会话目录命名升级为 `时间戳 + generateId62()`；
3. `ensureDirectory` 在目录已被并发创建时按“存在即成功”处理；
4. 新增 `BoxSettingsDialog`，迁移箱子文档级设置；
5. 精简 `BoxEditorPanel` 职责。
