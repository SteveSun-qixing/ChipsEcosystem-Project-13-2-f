# 工单081-共享运行时宿主层分层与命名规则未收口

## 1. 背景

在继续完善箱子链路时，用户发现生态根目录存在一个独立目录 `Chips-BoxLayoutHost`，并质疑：

1. 为什么它不是放在 `Chips-Host` 内部；
2. 为什么布局插件需要这样一个单独共享项目，而“同样属于插件渲染/编辑体系”的基础卡片链路却没有对称的共享宿主层。

这不是单纯的目录命名问题，而是当前生态对“共享运行时适配包”缺少正式定义，导致布局链路和基础卡片链路出现组织方式不对称。

## 2. 已确认事实

### 2.1 `Chips-BoxLayoutHost` 的真实定位

经核对代码与文档确认：

- `Chips-BoxLayoutHost` 只有 `package.json`、`src/index.js`、`src/index.d.ts`，没有 `manifest.yaml`；
- 它不是可安装插件，也不是独立桌面应用；
- 它被 `Chips-EditingEngine` 与 `Chips-CardViewer` 同时依赖；
- 当前职责包括：
  - 查询已启用布局插件；
  - 装载布局插件入口模块；
  - 校验 `layoutDefinition` 正式导出；
  - 统一产出 Host 会话态 / 内存态布局运行时。

因此，它本质上是“跨应用复用的布局运行时适配包”。

### 2.2 基础卡片链路当前做法

基础卡片当前没有与 `chips-box-layout-host` 对称的共享包：

- 编辑引擎把基础卡片注册表、单卡 iframe 装配、编辑器挂载、资源操作等能力直接放在：
  - `Chips-EditingEngine/src/basecard-runtime/`
  - `Chips-EditingEngine/src/editor-runtime/`
- 查看器和其他 Host 托管场景则继续走 Host / SDK 公共链路。

共享文档当前把它定义为“官方编辑引擎正式例外”，但没有继续说明该例外的抽象原则。

## 3. 当前架构问题

### 3.1 包类型没有被正式分层

当前生态实际已经存在至少四类东西：

1. 可安装插件项目
2. Host 内部实现包
3. 应用本地运行时模块
4. 跨应用复用的共享运行时适配包

但文档和工作区目录规则没有把第 4 类单独定义出来。

### 3.2 命名和目录放置会误导开发者

`Chips-BoxLayoutHost` 虽然在技术上是合理拆包，但名字与目录位置会让人误判：

- 名字里带 `Host`，容易被理解成 `Chips-Host` 内部子项目；
- 位于生态根目录 `Chips-*` 顶层，外观上和真正的应用/插件项目没有明显区别；
- 缺少统一规则时，新增类似包时很可能继续凭感觉命名和放置。

### 3.3 布局链路与基础卡片链路不对称

布局链路选择“共享运行时包”，基础卡片链路选择“编辑引擎本地运行时例外”，两者都各自能成立，但目前缺少统一决议解释为什么：

- 哪些能力应抽成共享适配层；
- 哪些能力必须保留在应用本地；
- 什么时候可以作为正式例外存在；
- 例外需要满足什么前置条件。

如果不补规则，未来新增其他文档渲染/编辑能力时，容易继续出现第三种、第四种组织方式。

## 4. 影响

### 4.1 短期影响

- 新成员阅读目录时会误把 `Chips-BoxLayoutHost` 当成应用或 Host 子仓；
- 难以快速判断基础卡片链路为什么没有对应的共享包；
- 共享文档对“可复用运行时适配层”的边界表达不完整。

### 4.2 长期影响

- 不利于生态形成统一的平台型分层模型；
- 不利于未来新增查看器、编辑器、文档承载能力时做正确归属判断；
- 容易再次出现“逻辑相近、组织方式不同、理由只存在历史上下文里”的结构漂移。

## 5. 建议收口方向

本工单建议先做架构决议，再推进代码迁移。建议决议至少覆盖以下问题：

1. 正式定义“共享运行时适配包”这一包类型；
2. 规定其命名规范、目录归属和依赖方向；
3. 明确它与 `Chips-Host` 内核、应用本地运行时、可安装插件项目的边界；
4. 对基础卡片链路给出统一结论：
   - 是不是应该抽取对称共享层；
   - 如果不抽取，为什么这是正式长期例外。

## 6. 本轮不直接做的事

在没有统一决议前，本轮不直接执行以下动作：

- 不直接把 `Chips-BoxLayoutHost` 粗暴迁入 `Chips-Host/`；
- 不直接把基础卡片运行时从编辑引擎拆成新包；
- 不仅通过改名来掩盖真实分层问题。

## 7. 后续建议输出物

后续收口时，建议至少补齐以下正式产物：

1. 生态共享架构文档中的“共享运行时适配包”分层章节；
2. 工作区目录 / 命名规则；
3. 基础卡片链路与布局链路的一致性决议；
4. 如需迁移，再补对应实现计划与验证清单。
