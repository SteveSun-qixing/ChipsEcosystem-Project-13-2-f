- 问题类型：生态工具链 / Host 启动链路 / 应用静态资源装载契约
- 问题描述：
  - 在 `/Users/sevenstars/Documents/ChipsCard/Develop/Project-13-2-f/Chips-CardViewer` 目录执行 `chipsdev run` 时，CLI 会输出“Host 已在开发工作区中启动应用插件”，但命令立即返回终端，且没有出现任何插件窗口。
  - 根据 `生态共用技术文档/命令行工具/02-Chips-Dev开发者命令行手册.md`，`chipsdev run` 应完成 Host 启动、插件安装、会话初始化并通过 `window.open` 打开应用窗口；当前实际行为与文档和既有闭环结论不一致。
  - 进一步排查发现，`Chips-SDK/cli/index.js` 当前直接实例化 `HostApplication`，并未走 `Chips-Host/src/main/core/main-process.ts` 中负责 Electron `app.whenReady()` 与主进程生命周期绑定的 `HostMainProcess` 链路，导致开发态并未真正进入 Electron 主机窗口环境。

- 影响范围：
  - `chipsdev run` 当前无法形成“真实 Host + 真实应用窗口”的开发闭环，应用插件联调结论不可信。
  - 所有依赖 `chipsdev run` 启动 app 插件的工程都可能出现“CLI 提示成功但无窗口”的假成功现象。
  - 即使后续补齐 Electron 窗口启动，当前 manifest 安装链路与静态资源路径契约仍会导致应用页面空白或资源 404。

  5. 查看 `/Users/sevenstars/Documents/ChipsCard/Develop/Project-13-2-f/.chips-host-dev/plugins/com.chips.card-viewer/dist/`，仅有 `index.html`，无 `assets/` 目录。
